# Artikel Import Manager 
Artikel Import Manager 

Dieser Dialog öffnet sich im Artikel Schattenstamm über Stamm >> Import Manager und verwaltet alle Import-Quellen für den Import von Artikel- und Konditionsdaten. So kann pro Lieferant (= Datenlieferant, nicht Artikellieferant) ein Datensatz angelegt werden, welcher eingelesen wird. 

### Anlegen eines neuen Formats 
Zum Anlegen eines neuen Formats gehen Sie wie folgt vor: 

- Geben Sie den Lieferanten ein. Das Feld Import Format bleibt leer. - Wechseln Sie mit dem Button Formate in den Dialog Importformate . Hier wird immer das Format mit der Nr. 1 angezeigt. Um hier ein neues Format anzulegen, ist der Dialog zwingend mit F5 (Neu) zu leeren. Jetzt werden die Daten eingegeben und gespeichert. - Jetzt wechseln Sie mit dem Button Format (auf der Registerkarte Datenquelle ) in den Dialog Datenimport . Im Dialog Datenimport sind die Felder Beschreibung und Datei bereits gefüllt. 

### FTP Implementierung 
Im Rahmen der Produktpflege wurde die FTP-Implementierung von FtpWebRequest auf WinSCP gewechselt. Hintergrund ist die Empfehlung von Microsoft, die FtpWebRequest-Klasse nicht mehr zu verwenden. (siehe FtpWebRequest Klasse (System.Net) | Microsoft Learn ) 

Mittels der Implementierung der externen DLL werden FTP, S-FTP und FTP-S unterstützt. 
Wenn Sie eine eigene Verbindung zu den FTP-Verbindungen programmiert haben, so ist diese zu prüfen und ggf. anzupassen.

### Checkbox Direkter Import (ohne Vergleich) 
Der Import ohne gesetzte Checkbox kann beliebig oft auf einer Version durchgeführt werden, während der erneute Import derselben Version bei der Variante ohne Vergleich momentan unterbunden wird. Die Checkbox Direkter Import (ohne Vergleich) zielt nur auf den Vergleich der im Staging enthaltenen Daten und nicht der schon bereits in den Schattenstamm übernommenen Daten ab. Sie können dadurch steuern, ob Sie Datensätze aus unterschiedlichen 'Import-Versionen' zuvor miteinander vergleichen wollen, oder nicht. Diese Checkbox hat auch nicht direkt etwas mit den bereits im Artikelstamm enthaltenen Daten zu tun, es geht rein um die Daten die Sie in unterschiedlichen Import-Versionen eingespielt haben. 

Jedes Einlesen einer zu importierenden Datei erzeugt eine entsprechende Import-Version. Vor dem Import einer neuen Datei sollte die bestehende Import-Version entsprechend bearbeitet sein, d.h. in den Schattenstamm übernommen werden, wenn Sie die Checkbox Direkter Import (ohne Vergleich) verwenden. 

Wenn diese Checkbox aktiviert ist, werden die verschiedenen Import-Versionen nicht miteinander verglichen. Dies erhöht die Performance bei der Übernahme der Daten beim Import. 

Die alte Variante ohne Checkbox steht wegen der Abwärtskompatibilität zur Verfügung und wird nur für Nutzer benötigt, welche den Import über den Staging-Manager steuern ( Import bearbeiten ). Falls dies nicht der Fall ist, empfehlen wir den Import mit aktivierter Checkbox Direkter Import (ohne Vergleich) . 

Ein weiterer Unterschied ist, dass der alte Import ohne Checkbox auf einer Version beliebig oft durchgeführt werden kann, während der erneute Import derselben Version bei der neuen Variante momentan unterbunden wird. 

Tooltip: 

Wenn diese Checkbox aktiviert ist, wird ein ggf. bestehender Artikel komplett durch den neuen ersetzt. Auch bereits gefüllte Felder im Schattenstamm können geleert werden. 

Diese Variante ist schneller, da der Vergleich übersprungen wird.
