# Bestellbestätigung 
Bestellbestätigung 

Der Dialog Bestellbestätigung öffnet sich in der Bestellung über den Button Bestätigung auf der Registerkarte Allgemein , wenn ein Eintrag der Tabelle markiert ist. Dazu muss der Parameter Mehrfachbestätigung EK im Dialog Parameter Einkauf gesetzt sein. Dieser Dialog bietet die Möglichkeit, gesplittete Bestellungen bezüglich Liefertermin und Menge übersichtlich abzubilden und zu verwalten. Dies auch vor dem Hintergrund, dass solche Bestätigungen teilweise auf Papier gesendet werden oder per EDI. So brauchen Bestellpositionen bei mehreren Bestätigungen nicht mehr gesplittet zu werden und die historischen Daten bleiben erhalten. 

Folgende Daten können gepflegt werden: 

- Bestätigungsnummer - Bestätigungsdatum - Neues Lieferdatum 
Im Klartext heißt das: Sie senden eine Bestellung an den Lieferanten. Dieser erfasst einen entsprechenden Auftrag. Der Lieferant bestätigt den Auftrag. Wird dabei festgestellt, dass die Ware nicht komplett zu einem bestimmten Termin geliefert werden kann, wird die Auftragsbestätigung (entweder per Email, etc. oder automatisiert über EDI) gesplittet, mit Angabe wie viele Artikel wann eintreffen werden. Diese Information fließt in die Bestellbestätigung ein und damit in den gleichnamigen Dialog. Die Bestellbestätigung wird über die Reservierungstabelle der Bestellung mit den entsprechenden Daten gefüllt. 

Zur Erfassung einer neuen Bestellbestätigung muss eine neue Bestätigung erfasst werden. Hierzu legen Sie über den Button eine neue Zeile und die Bestätigungsnummer an. Der Lieferant und die Bestellung werden aus dem Vorgang übernommen. 

In der Bestellbestätigung können dann einzelne Bestätigungen (obere Tabelle) markiert werden. Die Bestellposition der zugeordneten Bestellung wird in die Tabelle Bestellposition eingetragen. Jetzt können Sie die genannte Teilmenge mit dem entsprechenden avisierten Lieferdatum (mit den Buttons zwischen den Tabellen) in die rechte Tabelle Bestätigungsposition übernehmen (dies funktioniert auch mit Drag&Drop). 

Der Bestätigungsstatus wird anschließend in die Bestellung , Registerkarte Positionen , Spalte ‚Bestätigungsstatus‘ eingetragen. 

Folgende Status stehen zur Verfügung, die automatisch zugeordnet werden: 

- Einfachbestätigung, wenn eine Bestellposition genau einer Bestätigungsposition mit einem Liefertermin zugeordnet ist. - Mehrfachbestätigung, wenn einer Bestellposition mehrere Bestätigungspositionen mit unterschiedlichen Lieferterminen zugeordnet sind. - Teilbestätigung, wenn eine Teilmenge bestätigt wird. Bei einer Teilmengenbestätigung entsteht also durch die reduzierte Reservierungsmenge ein zusätzlicher Dispobedarf, der zu weiteren Dispositionen / Bestellungen führen kann. 
Der Status der Bestätigungssituation steht in folgenden Dialogen zur Verfügung: 

- Position (Feld Bestätig. Status auf der Registerkarte Allgemein ) - Bestellauskunft (Checkbox Bestätigte Pos. und Spalte ‚Bestätigungsstatus‘) 
Über die Checkbox inkl. gelieferte werden auch bereits gelieferte Positionen (d.h. Status < 6) in der Tabelle angezeigt. 

Über die Checkbox inkl. historische werden auch die historisierten Positionen zum ausgewählten Bestätigungskopf-Datensatz in der Tabelle angezeigt. 

In der Tabelle der Bestätigungspositionen gibt es die Spalte 'FR'. Diese repräsentiert die Fixposnummer einer Bestell-Restposition auf die sich die entsprechende Bestätigungsposition (auch) bezieht.

### Feld Anzahl max. 
Über dieses Feld ist in den Suchdialogen die Anzahl der einzulesenden Datensätze limitiert. Die einzelnen Datensätze werden im Dialog entsprechend gesucht und nach Erreichen der maximalen Anzahl Datensätze abgebrochen. Auf diese Weise kann eine Suche über alle Datensätze verhindert werden, welche die gesamte Performance beeinträchtigen könnte. 

Diese Einstellung nehmen Sie im Dialog Parameter Allgemein auf der Registerkarte Allgemein vor. 

Siehe Kapitel Parameter Allgemein Registerkarte Allgemein . 

Diese Einstellung lässt sich in den entsprechenden Suchdialogen (z.B. Artikelsuche, Kundenauswahl) ändern bzw. ausschalten (ausgenommen Stammdatenprogramme). 

Tooltip: 

Anzahl Sätze, die maximal eingelesen werden.
