# Berechtigungsgruppe Registerkarte Allgemein

Grundeinstellungen für eNVenta ERP / Berechtigungsgruppe 

# Berechtigungsgruppe Registerkarte Allgemein 
Berechtigungsgruppe Registerkarte Allgemein 

Auf dieser Registerkarte legen Sie die Berechtigungen für die einzelnen Gruppen fest. 
### Feld Vorgabe Gült. 
Angabe der Anzahl Tage, wie lange eine Kondition gültig ist, wenn diese neu angelegt wird. Hierbei muss das Ablauf-Datum der Kondition angegeben werden und von diesem aus wird dann zurück gerechnet. 

Ein Eintrag im Feld Vorg. Gült. (= Vorgabe des Preisgültigkeits-Datums) hat Auswirkungen auf das Feld Bis Datum in den Dialogen 

- Preisgruppen Details - Rabatt Verkauf (Registerkarte Gültigkeit) - Sonderpreise - Rabattleiste - Rabatt Einkauf 
Da die Gültigkeiten von Preisen sehr genau abgestimmt werden, weil sie bei der Kalkulation eine große Rolle spielen, ist es wichtig, dass nur bestimmte Mitarbeiter dieses Recht erhalten. 

Wenn das Feld Bis Datum gefüllt wird, wird überprüft, ob das Feld Von Datum noch leer ist und anhand der Vorgabe im Feld Vorg. PG Gült. wird das Startdatum berechnet. 

Tooltip: 

Vorgabe der Spanne für die Gültigkeit in den Dialogen 
- Preisgruppen Details (DatumVon, DatumBis) 
- Rabatt Verkauf (Registerkarte Gültigkeit) 
- Sonderpreis-Verwaltung 
- Rabatt Einkauf (Registerkarte Gültigkeit) 

### Feld Max. Gült. 
Ein Eintrag in diesem Feld (Max Gült. = maximale Preisgruppen-Gültigkeit) hat ebenfalls Auswirkungen auf die o.g. Dialoge. Wenn dort das Datum Von/Bis gespeichert wird, wird geprüft, ob die Zeitspanne den Maximalwert im Feld Max PG Gült übersteigt. Wenn ja, darf die maximale Preisgruppengültigkeit nicht überschritten werden: Dies hängt von der Berechtigung des Mitarbeiters ab. 

Die Angabe in diesen beiden Feldern sind Tage. 

Siehe Kapitel Preisgruppen Details . 

Tooltip: 

Maximale Spanne für die Gültigkeit in den Dialogen 
- Preisgruppen Details (DatumVon, DatumBis) 
- Rabatt Verkauf (Registerkarte Gültigkeit) 
- Sonderpreis-Verwaltung 
- Rabatt Einkauf (Registerkarte Gültigkeit) 

### Checkbox Keine Retoure o. Auftrag (Diversartikel) 
Diese Checkbox steht im Zusammenhang mit dem Reklamationswesen. Wenn sie aktiviert wird, kann der Sachbearbeiter nur Retouren bearbeiten, die einen Bezug zu einem Auftrag mit Diversartikel haben. 

Tooltip: 

Keine Retoure ohne Auftragsbezug für Diversartikel 

### Checkbox Beschaffungsartikel Retouren Ber. 
Wenn Sie diese Checkbox aktivieren, können Sachbearbeiter dieser Gruppe die Retoure für Beschaffungsartikel freigeben. Der Grund dafür ist, dass die Retournierung für Beschaffungsartikel anders abläuft als für Lagerartikel. In den meisten Fällen werden keine Gutschriften generiert. Für eine bessere Kontrolle solcher Geschäftsfälle wird dieses Recht auf einzelne Mitarbeiter beschränkt. 

Siehe Kapitel Beschaffungsartikel in der Retoure . 

Tooltip: 

Beschaffungsartikel Retouren Berechtigung. 
Trotz der aktiven Prüfung ist es erlaubt eine Retourenposition ohne Belastungs-Rechnungsprüfung freizugeben. 

### Checkbox Rücknahme erlaubt 
Wenn Sie diese Checkbox aktivieren, ist den Sachbearbeitern dieser Berechtigungsgruppe erlaubt, Artikel zurück zu nehmen, welche im Artikelstamm als nicht-rücknahmefähig gekennzeichnet sind. Auf den Reports können dazu entsprechende Hinweistexte angedruckt werden. 

Siehe Kapitel Rücknahmetexte . 

Tooltip: 

Hiermit können Artikel mit dem Kennzeichen 'Keine Rücknahme' doch zurückgenommen werden. 

### Checkbox Belegspezifisches KL 
Wenn Sie diese Checkbox aktivieren, darf ein Sachbearbeiter mit der entsprechenden Berechtigung einem Kunden für einen bestimmten Auftrag ein Kreditlimit vergeben bzw. erhöhen. Dies gilt nur für Ausnahmefälle! 

Für weitere Details bezüglich Kreditlimit lesen Sie Kapitel Kreditlimit - Allgemeine Informationen . 

Tooltip: 

Berechtigung, im Bedarfsfall ein zusätzliches 
auftragsspezifisches Kreditlimit einzuräumen. 

### Checkbox Änderung Handlingskosten 
Wenn Sie diese Checkbox aktivieren, darf der Benutzer die Handlingkosten in der Auftragsposition, welche für das Kreditlimit und die auftragsbezogene Bestellung gelten, ändern. Dies gilt nur für das Berechnungsverfahren Erweitertes Kreditlimit . 

Tooltip: 

Berechtigung, die Handlingskosten einer Auftragsposition 
mit Bestellbezug zu ändern. 

### Checkbox LE genehmigen 
Wenn Sie diese Checkbox (LE = Leistungserfassung) aktivieren, ist der Sachbearbeiter berechtigt, Leistungen zu einem Projekt zu genehmigen. 

Tooltip: 

Leistungserfassungen genehmigen 

### Checkbox LE ausweisen 
Wenn Sie diese Checkbox (LE = Leistungserfassung) aktivieren, ist der Sachbearbeiter berechtigt, Leistungen zu einem Projekt auszuweisen. 

Siehe Kapitel Leistungen erfassen . 

Tooltip: 

Berechtigung, eine Leistung auf der Rechnung auszuweisen oder nicht. 

### Checkbox Änderung Paketgewicht 
Wenn Sie diese Checkbox aktivieren, ist den Sachbearbeitern dieser Berechtigungsgruppe erlaubt, das Paketgewicht manuell zu bestimmen. Z.B. wenn die Waage nicht zur Verfügung steht oder defekt ist. 

Tooltip: 

Berechtigung, das Paketgewicht in der Paketstraße zu ändern. 

### Checkbox Touren wieder öffnen 
Wenn Sie diese Checkbox aktivieren, darf der Sachbearbeiter dieser Gruppe bereits abgeschlossene Touren wieder öffnen. 

### Checkbox Fixes Artikel-Set ändern 
Wenn Sie diese Checkbox aktivieren, darf der Sachbearbeiter dieser Gruppe ein fixes Artikel-Set ändern. Er hat die Erlaubnis, Strukturänderungen an einem 'Fixen Artikel-Set' vorzunehmen, also: 

- Komponenten einfügen oder löschen - Mengenänderungen an den Komponenten 
(Dieses Recht hat grundsätzlich auch der Prozess-Administrator.) 

Tooltip: 

Erlaubnis, Strukturänderungen an einem 'Fixen Artikel-Set' vorzunehmen, also: 
- Komponenten einfügen oder löschen, 
- Mengenänderungen an den Komponenten, 
( dieses Recht hat grundsätzlich auch der Prozess-Administrator ) 

### Checkbox Autobelastung ändern 
Wenn Sie diese Checkbox aktivieren, darf der Sachbearbeiter dieser Gruppe eine automatisch aus der Rechnungsprüfung erstellte Belastung ändern. 

Tooltip: 

Autobelastung ändern: 
Mit dieser Berechtigung dürfen Autobelastungen aus der Rechnungsprüfung geändert werden. 

### Checkbox Manuelle Erfassung EK zulassen 
Über diese Checkbox wird gesteuert, ob der Sachbearbeiter im Einkauf manuell Positionen erfassen darf, welche laut Artikelstamm nicht für manuelle Erfassung im Einkauf zugelassen sind. 

Beispiel: 
Der Einkaufsleiter entscheidet, dass ein Zuschlag berechnet werden muss. Der "normale" Sachbearbeiter darf einen solchen Zuschlag nicht erfassen, da diese nur systemseitig generiert werden. Der Einkaufsleiter darf dies nun manuell erfassen, da dieser einer entsprechenden Berechtigunsgruppe zugewiesen ist. 

Die Kennzeichnung solcher Artikel erfolgt im Artikelstamm über die Registerkarte Allgemein mit der Checkbox keine manuelle Erfassung . 

Mit der Zulassung auf manuelle Erfassung wird auch das manuelle Löschen der Artikelpositionen im Vorgang erlaubt bzw. verhindert. 

Siehe Kapitel Artikelstamm auf der Registerkarte Allgemein. 

Tooltip: 

Manuelle Erfassung im Einkauf zulassen 

### Checkbox Manuelle Erfassung VK zulassen 
Über diese Checkbox wird gesteuert, ob der Sachbearbeiter im Verkauf manuell Positionen erfassen darf, welche laut Artikelstamm nicht für manuelle Erfassung im Verkauf zugelassen sind. 

Beispiel: 
Der Verkaufsleiter entscheidet, dass ein Zuschlag berechnet werden muss. Der "normale" Sachbearbeiter darf einen solchen Zuschlag nicht erfassen, da diese nur systemseitig generiert werden. Der Verkaufsleiter darf dies nun manuell erfassen, da dieser einer entsprechenden Berechtigunsgruppe zugewiesen ist. 

Mit der Zulassung auf manuelle Erfassung wird auch das manuelle Löschen der Artikelpositionen im Vorgang erlaubt bzw. verhindert. 

Siehe Kapitel Artikelstamm . 

Tooltip: 

Manuelle Erfassung im Verkauf zulassen 

### Checkbox Entkoppeln nach Teilfaktura 
Wenn eine Position aus dem Verbund bereits fakturiert ist, so muss der Benutzer auch das Recht zum Entkoppeln nach Teilfaktura haben. Dies stellen Sie über diese Checkbox ein. Es erscheint eine Fehlermeldung, wenn der Sachbearbeiter nicht die notwendige Berechtigung hat. 

Tooltip: 

"Entkoppeln von verbundenen Positionen bei Teilfaktura". 
Erläuterung: Entkoppeln von verbundenen Positionen auch dann, 
wenn eine oder mehrere Positionen aus dem Verbund 
bereits fakturiert sind. 

### Checkbox Entkoppeln von Pflicht-Zusatzp. 
Wenn Sie diese Checkbox aktivieren, ist der Sachbearbeiter berechtigt, sogenannte Pflichtpositionen (wie z.B. ein Wasserzähler und eine Eichgebühr) im Auftrag entkoppeln zu dürfen. 

Pflichtpositionen werden im Dialog Zusatzpositionen entsprechend gekennzeichnet. 

Tooltip: 

Entkoppeln von Pflicht-Zusatzpositionen 

### Checkbox Historiendaten Einsehen 
Wenn Sie diese Checkbox aktivieren, dürfen Sachbearbeiter dieser Gruppe Historiendaten einsehen. 

Siehe Kapitel Historisierung . 

Tooltip: 

Die Sachbearbeiter der Gruppe können die Historisierungs-Menüpunkte im Navigator sehen/aktivieren. 

### Checkbox Artikelübernahme anzeigen 
Wenn Sie diese Checkbox aktivieren, wird ein Artikel, der aus dem Artikelschattenstamm übergeben wurde, von einem berechtigten Sachbearbeiter im Verkauf qualifiziert. Es öffnet sich der Dialog für die Artikelübernahme. 

Siehe Kapitel Artikel übernehmen . 

Tooltip: 

Dialog für die Artikelübernahme anzeigen 

### Checkbox Keine Ret.-Änderung nach Druck 
Wenn diese Checkbox aktiviert ist, darf der Sachbearbeiter keine Änderungen an Retouren nach einem Belegdruck (Retourenbestätigung oder Abholavis) vornehmen. Folgendes ist dann nicht mehr gestattet: 

- Ändern einer Retourenmenge - Hinzufügen eine Position - Löschen einer Position 
Hinweis: Darf ein Mitarbeiter die Retouren aufgrund des gesetzten Hakens nicht ändern, hat aber das Recht, die bereits gedruckten Belege zu löschen, so kann er nach deren Löschung dennoch Änderungen vornehmen. 

Tooltip: 

Keine Retourenänderung nach Belegdruck. 
Ist dieser Parameter gesetzt, so darf der Benutzer in einer Retoure, 
nach dem Druck einer Abholavis oder einer Retourenbestätigung, 
folgende Aktionen nicht mehr durchführen: 

- Retourenmenge ändern 
- Hinzufügen einer Position 
- Löschen einer Position
