# Versandanschriften verwalten Registerkarte Logistik 
Versandadresse Registerkarte Logistik 

Auf dieser Registerkarte können die Einschränkungsmöglichkeiten des Kundenstamms, bezogen auf Tourenfindung und Transportweg, ebenfalls konfiguriert werden. Bei der Übernahme der Versandanschrift als Lieferanschrift in den Auftrag werden die entsprechenden Optionen übernommen. Basierend auf diesen Informationen wird die Tourenplanung entsprechend ausgeführt. 

Mit dem Button TE-Zuordnung können dem Kunden die Transport-Modalitäten zugeordnet werden. 

Konkret sind dies die folgenden Eigenschaften: 

- Routen (+ Button TE-Zuordnung ) - Versandweg - Transportweg - Kein Paketdienst - Kein Overnight - Keine Spedition - Keine Strecke - Kein Fuhrpark 
Folgendes ist zu beachten: 

Wird eine Versandanschrift, bezogen auf die o.g. Feldwerte, abweichend zum Kundenstamm konfiguriert und diese Versandanschrift im Auftrag ausgewählt, so findet auf Basis der in der Versandanschrift gemachten Konfiguration eine Tourenplanung statt. Dies gilt ebenso, wenn die Versandanschrift als „Standard“ gekennzeichnet ist und damit automatisch in eNVenta ERP eine Vorbelegung stattfindet. 

Wurden bereits Positionen erfasst, bevor die Versandanschrift geändert wurde, so findet die Tourenplanung auf Basis der nun geänderten Versandanschrift für die bereits existierenden Positionen erneut statt. 

Wird die Versandanschrift im Auftrag gelöscht, so bleiben etwaige Restriktionen der Versandanschrift im Auftragskopf stehen und können durch den Sachbearbeiter manuell angepasst werden. 

Das gilt explizit nur dann, wenn es sich um eine gespeicherte Versandanschrift handelt. Manuell erfasste Versandanschriften können wie bisher nur im Auftrag selbst erfasst und dann auf Auftragsebene angepasst werden. 

In der Retourenverwaltung kann nun eine Tour zur Abholung mittels des Buttons Tourenauswahl manuell ausgewählt werden.

### Button TE-Zuordnung 

Mit diesem Button können Sie der Versandanschrift ein Ladehilfsmittel zuordnen. So kann eine technische Entladung an der Versandanschrift definiert werden. 

Eine Zuordnung auf Versandanschrift übersteuert eine Zuordnung auf Kundenebene. 

Diese Zuordnungen werden automatisch in den Auftrag und die Retoure übernommen. Sie sorgen dort je nach Konfiguration für eine abweichende Tourenfindung.

### Combobox Stammtour 
Hier geben Sie an, mit welcher Tour (Route) (z.B. täglich, wöchentlich) der Kunde bzw. Lieferant immer (regelmäßig) beliefert werden soll. Die Touren, die zur Verfügung stehen, werden im Tourenstamm gepflegt. 

Siehe Kapitel Tourenstamm . 

Tooltip: 

Angabe der Stammtour 

### Feld Tour lfd Nummer 
Hier wird für die Tour eine Stopp-Nr. innerhalb der Tour angegeben. Damit definieren Sie, an wievielter Stelle der Kunde beliefert wird. 

Tooltip: 

Fortlaufende Nummer, wann der Stopp innerhalb der Tour angefahren wird. 

### Combobox Tourzyklus 
Wenn Sie hier den Tourzyklus angeben, wird bei der Generierung von Touren der Stammtour ein Tourzyklus zugeordnet (z.B. Vormittag, Nachmittag oder 1. und 2. Tour eines Tages). 

Siehe auch Kapitel Tourzyklus . 

Tooltip: 

Angabe, mit welchem Tourzyklus die Auslieferung erfolgt.

### Combobox Versandlager

### Checkbox Verdichten 
Wenn Sie diese Checkbox aktivieren, wird verhindert, dass bei der Tourenplatzeinlagerung die Ware zur Verdichtung vorgeschlagen wird. D.h. die kommssionierte und verpackte Ware wird zu einer anderen Umverpackung hinzugefügt. 

Tooltip: 

Wenn aktiviert, wird bei der Tourenplatzeinlagerung die Ware zur Verdichtung vorgeschlagen. 

### Checkbox Kein Bezugslager 
Wenn Sie diese Checkbox aktivieren, wird nicht über die Bezugslager auf das Versandlager umgelagert. 

Tooltip: 

Es erfolgt kein automatischer Vorschlag für eine interne Umlagerung vom Bezugslager, 
falls das Versandlager nicht lieferfähig ist. 

### Combobox Versandweg 
Hier wird der Versandweg angegeben. 

Zufuhr - Lieferung mittels eigenem Fuhrpark oder Paketdienstleister 
Strecke - Lieferung vom Lieferanten zum Kunden 
Abholung - Kunde holt Ware am Lager ab 

Tooltip: 

Angabe des Versandwegs: 
Zufuhr - Lieferung mittels eigenem Fuhrpark oder Paketdienstleister 
Strecke - Lieferung vom Lieferanten zum Kunden 
Abholung - Kunde holt die Ware am Lager ab 

### Combobox Transportweg 
Hier wird die Art des Transports angegeben. Mögliche Werte sind: 

leer = Automatisch 
0 = Fuhrpark 
1 = Paketdienst (Feld Versender und Versand Code müssen gefüllt sein) 
2 = Overnight Dienst (Feld Versender und Versand Code müssen gefüllt sein) 
3 = Spedition 

Tooltip: 

Transportweg - mögliche Werte 
leer = Automatisch 
0 = Fuhrpark 
1 = Paketdienst (Felder 'Versender' und 'Versand Code' müssen gefüllt sein) 
2 = Overnight Dienst (Feld 'Versender' und 'Versand Code' müssen gefüllt sein) 
3 = Spedition

### Checkbox Kein Paketdienst 
Wenn Sie diese Checkbox aktivieren, werden bei der Ermittlung der verfügbaren Touren keine Paketdienst-Touren berücksichtigt bzw. es können keine Touren mit diesem Transportweg angelegt werden. Die Eigenschaft „Kein Paketdienst“ kann global in den Versandparametern gesteuert werden sowie im Kundenstamm und im Artikelstamm. Sobald eine der Stammdaten die entsprechende Kennung hat, erfolgt keine Tourenermittlung über diesen Transportweg. 

Beim Kopieren eines Angebots / Auftrags wird diese Einstellung übernommen. 

Tooltip: 

Keine Lieferung per Paketdienst

Die Eigenschaft „Kein Overnight“ kann global in den Versandparametern gesteuert werden sowie im Kundenstamm und im Artikelstamm. Sobald eine der Stammdaten die entsprechende Kennung hat, erfolgt keine Tourenermittlung über diesen Transportweg. 

Tooltip: 

Keine Lieferung per Overnight 

### Checkbox Keine Spedition 
Wenn Sie diese Checkbox aktivieren, werden bei der Ermittlung der verfügbaren Touren keine Speditionstouren berücksichtigt bzw. es können keine Touren mit diesem Transportweg angelegt werden. Die Eigenschaft „Keine Spedition“ kann global in den Versandparametern gesteuert werden sowie im Kundenstamm und im Artikelstamm. Sobald eine der Stammdaten die entsprechende Kennung hat, erfolgt keine Tourenermittlung über diesen Transportweg. 

Tooltip: 

Keine Lieferung per Spedition 

### Checkbox Keine Strecke 
Wenn diese Checkbox aktiviert ist, kann der Kunde nicht per Strecke beliefert werden. 

Im Standard erfolgt keine Prüfung auf diese Einstellung! 

Die Prüfung auf Zulässigkeit steht nur über die erweiterten Abschlussprüfungen im WS-Package zur Verfügung. 

Tooltip: 

Keine Lieferung per Strecke
