# Aktionsdefinition 
Aktionsdefinition 

Dieser Dialog öffnet sich im Navigator >> System >> Administration >> Event Management >> Aktionsdefinition . Hier legen Sie fest, bei welchem Event (Ereignis) die Aktion (z.B. das Generieren einer Wiedervorlage oder Email) ausgelöst werden soll. 

Begriffsdefinition: 

Ereignis, Event = auslösendes Element / Ursache 
Aktion = ausgelöstes Element / Wirkung 

Dazu legen Sie im ersten Schritt die Kopfdaten der Aktion an. 

- Aktionsname: Vergeben Sie einen beliebigen Namen für die Aktion - Beschreibung: Beschreiben der Aktion - Event: Geben Sie das Event an, nach dem die Aktion ausgelöst werden soll. 
Die zur Verfügung stehenden Events sind momentan eine festgelegte Auswahl an Events, welche von eNVenta zur Verfügung gestellt werden. 

Im Kopf des Dialogs steht die Combobox Protokollierung zur Verfügung. Hier wählen Sie aus ob nur Fehler, alles oder nichts protokolliert werden soll. 

Nachdem Sie alle notwendigen Eingaben gemacht haben, also auch auf der entsprechenden Registerkarte, können Sie noch testen, ob der Datensatz korrekt übernommen worden ist. 

Beispiel: 
Sie haben als Event "Artikel erzeugt" angegeben. Jetzt öffnen Sie den Artikelstamm und suchen einen Artikel. Per Drag&Drop können Sie die Überschrift des Artikeldialogs auf den Aktionsdefinitionsdialog ziehen. Wird in dem Feld Test-Objekt der Name des Artikels angezeigt, können Sie über den Button neben dem Feld ihre Aktion testen.

### Dock Panel Ausführungsart 

Mit der Ausführungsart wird der Zeitpunkt bestimmt, an dem die Aktion ausgeführt wird. Und hier wiederum gibt es die Einstellungen Sofort (die Aktion wird in dem Moment ausgeführt, in dem das Ereignis eintrifft) und Gateway Adapter (das eingetragene Ereignis wird in der Datenbank gespeichert). Nun kann über das Gateway mit dem "EventManagement Adapter" für die Ereignisse die Aktion zeitgesteuert ausgeführt werden (benötigt die Gateway-Lizenz). 
### Radiobutton Sofort 
Wenn dieser Radiobutton aktiviert wird, wird die Aktion direkt nach dem Eintreten des Events ausgeführt. 

Tooltip: 

Die Aktion wird direkt nach dem Eintreten des Events ausgeführt. 

### Radiobutton Gateway Adapter 
Wenn dieser Radiobutton aktiviert ist, wird das eingetretene Event in die Datenbank geschrieben. Die Aktion kann anschließend im Gateway über einen Job Server ausgeführt werden. 

Tooltip: 

Das eingetretene Event wird in die Datenbank geschrieben und die Aktion kann dann im Gateway über einen Job-Server ausgeführt werden. 
(Erfordert die Gateway-Lizenz) 

### Dock Panel Event-Auswahl 

Durch die Ereignisdefinition können nun Ereignisse dynamisch selbst definiert werden. 

Deswegen kann die Aktionsdefinition zusätzlich zu den bestehenden Gateway-Ereignissen auch auf die selbst-erfassten Ereignisse reagieren. 

Durch aktivieren eines der Radiobuttons wird entschieden, welche Ereignisse in der Event-Combobox angezeigt werden; Ereignisse aus der Ereignisdefinition oder bestehende Gateway-Ereignisse. 
### Radiobutton Event Management

Die Events können im Menü über "Event -> Eventdefinition" selbst definiert werden.
