# Projekt Registerkarte Beteiligte 
Projekt Registerkarte Beteiligte 

Auf dieser Registerkarte werden alle Beteiligten und Kunden zu einem Projekt hinterlegt. 

Damit Kreditlimits nicht nur auf einzelne Kunden, sondern auch auf ein Projekt vergeben werden können (unabhängig von einzelnen Kunden) steht Ihnen die Spalte 'Kreditlimit' zur Verfügung. In dieser Spalte wird das projektbezogene Kreditlimit eingetragen bzw. angezeigt. 

Das Kreditlimit zu einem Projekt wird im Dialog Kreditlimit zugeordnet, nachdem Sie in diesem Dialog den Radiobutton Projekt aktiviert haben. 

Siehe Kapitel Kreditlimit . 

Dieses Kreditlimit wird kundenspezifisch gepflegt. Die Werte werden auf der Registerkarte Beteiligte angezeigt. Dazu steht der Button Kred. Limit (öffnet den Dialog Kreditlimit ) und in der Tabelle Kunden die Spalte ‚Kreditlimit‘ zur Verfügung. Damit erhalten Sie eine Übersicht, wie das Kreditlimit verteilt ist. 

Die Zuordnung von Belegen ist nur möglich, wenn der Kunde als Projektkunde angelegt ist. 

Es können nur Verkaufsbelege von Projektkunden zugeordnet werden. 

Ein Projektkunde kann erst gelöscht werden, wenn es keine Aufträge mehr zu diesem Projekt/Kunden gibt. Eine Fehlermeldung weist Sie daraufhin! 
### Tabelle Beteiligte 

In dieser Tabelle ist die Spalte ‚Funktion’ eine Combobox. Hier können Sie einen Eintrag, der in der Pickliste ‚Projektfunktion‘ hinterlegt ist, auswählen. 

Eine Eingabe in dieser Spalte ist Pflicht! 

Wird ein Eintrag dieser Tabelle markiert und Sie öffnen die Kontakt-Historie (Button Kontakte ) so wird die Projektnummer übergeben und daraus die Historie erstellt. 

Mit einem Doppelklick auf einen Tabelleneintrag öffnet sich der Kundenstamm.

Hier wird eine Adress-/Kunden- oder Lieferantennummer angegeben.

### Tabellenspalte Partner 
Hier können Sie einen Ansprechpartner angeben. 

Wenn es einen neuen Eintrag mit derselben AKL-Nummer und derselben Funktion gibt, kann dieser Eintrag nur gespeichert werden, wenn ein anderer Ansprechpartner eingetragen wird. So wird verhindert, dass 2 identische Datensätze erfasst werden. 

Der Ansprechpartner ist dazu bereits als Ansprechpartner in den Stammdaten angelegt worden. 

Auch beim nachträglichen Ändern wird verhindert, dass 2 identische Datensätze erfasst werden.

Angabe des Ansprechpartners

### Tabellenspalte Wunschdatum 
Wünscht der Kunde eine frühere Lieferung, wird der entsprechende Wunschtermin in das Feld Wunschdatum eingetragen. Im Programmteil Liefervorschlag kann das Programm gegebenenfalls bei frühzeitiger Lieferung den Wunsch des Kunden berücksichtigen und eine frühere Lieferung vorschlagen. 

Dieses Wunschdatum wird in der Tourendisposition nicht berücksichtigt; dort wird der Liefertermin zugrunde gelegt. 

Existieren 2 Positionen mit unterschiedlichem Lieferdatum, so werden diese Positionen zum „kleineren“ Datum berücksichtigt (um Kosten zu senken). 

Tooltip: 

Angabe des Wunschdatums 

### Tabellenspalte Kreditlimit 
Hier kann ein Standard-Kreditlimit für den Kunden (in System-Währung) festgelegt werden. Falls der Betrag überschritten wird, erhalten Sie eine Meldung, wenn Sie einen Auftrag zu diesem Kunden erfassen. Sie erhalten die Möglichkeit, eine Begründung für die Überschreitung einzugeben. 

Die absoluten Kopfrabatte werden in der Kreditlimitprüfung nicht berücksichtigt. 

Das Kreditlimit ist immer in "Haus-Währung" anzugeben! 

Tooltip: 

Angabe des gewährten Kreditlimits 

### Tabellenspalte Kein Vorschlag 
Wenn diese Checkbox gesetzt ist, wird das Projekt bei der Auftragserfassung nicht zur Auswahl vorgeschlagen und wird im Dialog Projektauswahl auch nicht aufgeführt. 

Tooltip: 

Wenn gesetzt, wird das Projekt bei der Auftragserfassung nicht zur Auswahl vorgeschlagen und wird im Dialog 'Projektauswahl' auch nicht aufgeführt.
