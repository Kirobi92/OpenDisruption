# Marketing-Aktion 
Marketing-Aktion 

Eine Marketing-Aktion ist z.B. eine Weihnachtsfeier, eine Messe, ein Newsletter. Eine Aktion besteht in der Regel aus mehreren Aktionsschritten. 

Dieses Menü bietet Ihnen die Möglichkeit, verschiedene Vertriebsaktionen mit einer genauen Beschreibung anzulegen. Damit ist gemeint, dass Sie hier z.B. Kunden, Interessenten oder Adressen sammeln können, um diese gezielt für eine Marketing-Aktion zu nutzen. Dies kann beispielsweise die Kundengewinnung sein, damit können Firmen- und / oder Produktneuheiten gezielt "verteilt" werden. Sie können die Marketing-Aktion auch nutzen, um Einladungen zu einer Veranstaltung oder eine Werbekampagne zu starten. 

Marketing-Aktionen können per Telefon, Post oder E-Mail verteilt werden. Dabei kann auch eine Datei, eine Broschüre, ein Brief oder Flyer angehängt werden. 

Im Zusammenhang mit den Marketing-Aktionen ist es sinnvoll Kundengruppen anzulegen, damit die Selektion gezielt gestaltet und gesteuert werden kann. Ebenso sind im Vorfeld die einzelnen Aktionssschritte für eine Marketing-Aktion zu definieren. Diese können Sie individuell gestalten und (als Empfehlung) legen Sie auch eine Schluss-Aktivität an, wie z.B. Aktionsschritt-Ende . Damit ist die "Zielerreichung" der Marketing-Aktion sofort ersichtlich. 
### Anlegen einer neuen Marketing-Aktion 
Wenn Sie eine neue Marketing-Aktion anlegen, können Sie den Namen der Aktion frei wählen (Feld Bezeichnung ) und bei Bedarf eine erklärende Beschreibung in das gleichnamige Feld eintragen. Wenn Sie die Aktion speichern, wird die Nummer automatisch vom System vergeben. 

Alle Informationen zu einem Kunden, einer Adresse können Sie in dessen Kontakt-Historie nachlesen. Über den Eintrag Chronologisch sehen Sie sämtliche Kontakte, die zum Kunden / Adresse erfolgt und erfasst sind. 

Siehe Kapitel Kontakt-Historie . 

### Projektbezug herstellen 
In der Marketing-Aktion kann ein Projekt zugeordnet werden. Dafür stehen im Kopf des Dialogs die Felder Projekt und Bezeichnung zur Verfügung. Wenn Sie ein Projekt zuordnen, wird die Marketing-Aktion speziell für eine bestimmte Gruppe Kunden angelegt. So können auch spezielle Preise und Konditionen für bestimmte Artikel, für bestimmte Kunden zusammengeführt werden. Diese Preise und Konditionen werden in der Rabattverwaltung Verkauf angelegt. 

Wird eine Marketing-Aktion mit einem Projekt verknüpft, sind alle Teilnehmer einer Marketing-Aktion (also nur Kunden) automatisch als Kunden des zugeordneten Projekts anzulegen. Wird ein Teilnehmer aus einer Aktion gelöscht und diese ist einem Projekt zugeordnet, wird der Teilnehmer ebenfalls aus dem Projekt gelöscht. 

Wird das Projekt aus einer Marketing-Aktion entfernt, so werden alle Teilnehmer der Aktion, die im Projekt als Kunden angelegt sind, aus dem Projekt entfernt. 

Wird eine Kunde im Projekt als Kunde hinzugefügt, so erhält dieser die Konditionen der Aktion, ohne dass er der Aktion direkt zugeordnet ist. Mit Entfernen des Projekts aus der Marketing-Aktion wird der Kunde allerdings nicht aus der Marketing-Aktion entfernt! 

### Auftragsbezug 
Aufgrund der Möglichkeit, ein Projekt mit einer Marketing-Aktion und den entsprechenden Kunden zu verknüpfen, bietet dies im Auftrag ebenfalls eine komfortable Möglichkeit eine Verbindung herzustellen. Dazu stehen auf der Registerkarte Kopfdaten die Felder Projekt und Aktion zur Verfügung. 

Die Zuordnung einer Aktion zu einem Vorgang ist ein manueller Prozess. Wird eine Aktionsnummer eingetragen, so ist zu prüfen, ob der Aktion auch ein Projekt zugeordnet ist. Wenn ja, wird die Projektnummer in den Auftragskopf übernommen und die Artikel werden entsprechend konditioniert. 

Alle nun zu erfassenden Positionen werden automatisch mit der Aktionsnummer des Kopfes versehen, sofern die Ermittlung der Kondition ergibt, dass es sich um einen Artikel handelt, welcher in der Projektkondition, über die Projektnummer der Aktion, zugeordnet wurde. 

Befindet sich der Artikel nicht in der Projektkondition, so wird die Aktionsnummer nicht in die Position übertragen. 

Sind zu diesem Zeitpunkt bereits Positionen erfasst – die Zuordnung der Aktion und des Projektes erfolgt also im Nachhinein – so kann die Aktionsnummer nur bei danach erfassten Positionen eingetragen werden. Wird der Auftrag mit hinterlegter Projekt- und Aktionsnummer neu kalkuliert, so sind die Positionen nach vorgenanntem Schema zu aktualisieren. 

Die Ermittlung findet auch beim Kopieren von Vorgängen statt. Sie folgt hier den Einstellungsmöglichkeiten des Kopierdialogs.
