# Abwicklung von Retouren und Reklamationen 
In eNVenta ist es möglich sowohl Retouren als auch Reklamationen abzuwickeln. Bei der Retoure handelt es sich um Vorgänge, bei welchen die Ware bewegt wird. Dazu zählen auch eine mögliche Abholung der zu retournierenden Ware als auch die Rückabwicklung mit dem Lieferanten. 

Die Bearbeitung von Retouren und Reklamationen kann auf verschiedene Mitarbeiter festgelegt werden. Z.B. übernimmt ein Lagerarbeiter den Wareneingang und ein anderer löst die Freigabe der Retoure aus. Dazu stehen verschiedene Buttons und auch die Berechtigungsgruppen zur Verfügung. 

Bei der Reklamation geht es um Beschwerden des Kunden ohne Warenbewegung, z.B. „Kratzer“ und eine entsprechende Preisreduktion, welche der Kunde haben möchte. Für beide Zwecke steht der Dialog Retoure im Modul Verkauf zur Verfügung. 

Zu den Funktionalitäten in der Retouren- bzw. Reklamationsabwicklung zählen die zentrale Abwicklung von 

- Lagerbuchung - Gutschrift - Nachlieferung bzw. Ersatzlieferung - Belastung (im Einkauf) - Materialrücksendung

### Allgemeine Vorgehensweise 
Öffnen Sie den Dialog Retoure , geben Sie die entsprechende Auftrags- oder Lieferscheinnummer ein, um den Vorgang einzulesen. Alternativ ist eine Suche über den Dialog Positionsübersicht nach Vorgängen zu einem Kunden möglich und darüber kann der entsprechende Vorgang eingelesen werden. Es können auch mehrere Aufträge in eine Retoure eingelesen werden. 

Achtung: Bei einer Reklamation (ohne Warenbewegung) erfolgt keine Lagerbuchung. Aus diesem Grund wird nach Einlesen der Retourenposition die Checkbox in der Spalte 'LBuchung' auf der Registerkarte Positionen zwingend deaktiviert. 

Sind die Daten des Vorgangs im Dialog Retoure erfasst, legen Sie eine Retourenart fest. Dies ist notwendig, um weitere Schritte einzuleiten bzw. statistische Auswertungen, z.B. Retourengrund, vornehmen zu können. 

Nachdem die Retourenpositionen angegeben sind, erfolgt ein (virtueller) Wareneingang für die Position, um anschließend die Retourenpositionen freigeben zu können. Anschließend erfolgt in der Regel eine Gutschrift; dafür steht der Button Gutschrift in der Retoure zur Verfügung. 

### Erfassen einer Gutschrift aus der Retoure 
Wenn Sie eine Gutschrift erfassen, ist zu überlegen, welchen Preis Sie dem Kunden erstatten. Dafür öffnen Sie zur markierten Position den Dialog Retourenposition (über den Button Ändern ) und geben hier einen entsprechenden Preis an. Dafür stehen im Dialog Retourenposition , Bereich Verrechnung , die Felder Preis Gutsch. und Rücknahme % zur Verfügung. So kann z.B. der gezahlte Preis abzüglich einer Rücknahmegebühr festgelegt werden. Diese Einstellung wiederum nehmen Sie im Dialog Parameter Verkauf , Registerkarte Retoure über den Bereich Abschlag in % vor. Dort geben Sie einen Prozentwert für Beschaffungsware oder Lagerware an. 

Es ist möglich, dass der eine oder andere Ihrer Kunden eine solche Rücknahmegebühr ablehnen. Für Kunden, die das nicht möchten, legen Sie das im Kundenstamm, Registerkarte Warenwirtschaft im Bereich Retoure fest. Dort ist zwingend 0 einzutragen, wenn der Kunde keine Rücknahmegebühr akzeptiert. 

Die Einstellung im Kundenstamm übersteuert die Einstellung im Dialog Retourenposition . 

Sind alle Angaben im Retouren-Dialog gemacht, erzeugen Sie mit dem Button Gutschrift eine Gutschrift. Nach einer Sicherheitsabfrage wird die Gutschrift erzeugt. 

Außerdem erscheint die Abfrage "Position ohne Lagerbuchung mit EK = 0 überschreiben?" , wenn es sich um eine Reklamation handelt. Diese beantworten Sie mit ja , weil es sich erstens um eine Reklamation und außerdem, durch eine eventuell vereinbarte Rücknahmegebühr, die Marge geschmälert wird und dies den EK verfälschen würde. 

Beispiel: 
Auftrag VK 100,-- / EK 30,-- Euro -> Marge 70,-- Euro 
Gutschrift VK 20,-- / EK 0,-- Euro 
Gesamt VK 80,-- / EK 30,-- Euro -> Marge nur noch 50,-- Euro 

Bevor Sie die Gutschrift abschließen, wählen Sie die Gutschriftsart sorgfältig aus. Dies hat Auswirkungen auf den weiteren Prozess und die Auswertung der Gründe für eine Gutschrift. Es ist immer eine Wertgutschrift zu verwenden, da die Lagerbuchung (Wareneingang) bereits mit der Retoure gebucht wurde. 

Anschließend erfolgt der Fakturaprozess.

Bei einer Retoure (evtl. auch bei einer Reklamation) geht es um die Rücknahme von Ware (Registerkarte Abholung ). D.h. die Rücknahme per Spedition oder Paket kann hier ohne Umwege eingeplant werden. Dazu kann auch eine Tourenbestätigung gedruckt werden, damit der Kunde informiert werden kann. 

Wenn die Ware nun zurück ist, hat ein Wareneingang gebucht zu werden, und eventuell ist auch eine Nachlieferung erforderlich (Button Nachlieferung ). D.h. es ist ein Auftrag zu erfassen, damit der Kunde Ersatz für die zurückgegebene Ware bekommt. 

Zur Erstellung eines Nachlieferungsauftrags über den Button Nachlieferung öffnet sich der Dialog zur Belegauswahl. Hier wählen Sie den Eintrag Nachlieferung Retoure aus. Es wird ein neuer Auftrag generiert. Wenn der Kunde bereits die Originalrechnung bezahlt hat, ohne Faktura! Falls eine neue Rechnung gestellt wird, ist eine Auftragsart inklusive Faktura zu wählen und die Originalrechnung (sollte diese bereits vorhanden sein) hat zwingend über eine entsprechende Wertgutschrift ausgeglichen zu werden. 

Wenn Sie aus dem Auftrag heraus noch etwas an der Position prüfen oder ändern wollen, öffnen Sie den Dialog Position (Position markieren und Button Ändern drücken). Auf der Registerkarte Retoure sehen Sie alle Einstellungen zur Position. So haben Sie auch eine ideale Möglichkeit, dem Kunden Auskunft zur Retourenposition zu geben. In diesem Zusammenhang bietet die gesetzte Checkbox Detailinfos in Vorgängen im Dialog Parameter Verkauf , Registerkarte Retoure detaillierte Informationen zur Ursprungsretoure innerhalb der Prozesse.

### Retoure mit Gutschriftsvermeidung 
Um eine Retoure zu erfassen, für die keine Gutschrift generiert werden soll, setzen Sie die Checkbox Gutschriftsvermeidung im Kopf des Dialogs Retoure . 

Die Checkbox Gutschriftsvermeidung kann dann gesetzt werden, wenn der ursprüngliche Auftrag noch nicht fakturiert ist. In diesem Fall kann eine Gegenposition (Ausgleichsposition) im Auftrag erzeugt werden. In diesem Fall erscheint bei Aktivierung des Buttons Gutschrift die Abfrage "Ausgleichspositionen für freigegebene Positionen erzeugen?" . Wenn Sie diese Abfrage bestätigen, wird die Gutschrift als Ausgleichsposition in den Auftrag übernommen. Es erfolgt eine gemeinsame Fakturierung, d.h. die Positionen gleichen sich aus (Wert 0). In diesem Fall zieht auch der virtuelle Retourenausgleich zur Abbuchung der Ware. ABER: Die Berücksichtigung von Kopfrabatten kann nicht erfolgen! 

### Auswirkungen einer Retoure auf den Einkauf 
Im Fall einer Retoure wird auch der Lieferant kontaktiert, da dieser die Ware bzw. das Geld zurückgeben muss. Aus diesem Grund ist es notwendig, einen Bezug zur ursprünglichen Bestellung herzustellen. 

Um diesen Vorgang zu automatisieren, können Sie im Dialog Parameter Verkauf auf der Registerkarte Retoure die Checkbox Autozuordnung Lagerware setzen. So werden die "ältesten" Artikel zugeordnet, um die Gewährleistung zu nutzen. Die Zuordnung der Lagerware ist auch manuell möglich. 

Um Kontakt mit dem Lieferanten herzustellen, ist eine Anfrage zu erstellen (dafür verwenden Sie am besten eine extra definierte Anfrageart). Diese Anfrage können Sie per E-Mail senden und bei Bedarf auch Bilder (die die Schäden zeigen) per Referenz-Link an die Position anheften. Informationen aus der ursprünglichen Bestellung werden automatisch in die E-Mail eingefügt.

Falls ein Schaden z.B. durch die Spedition passierte, ist es erforderlich auch mit dieser Kontakt wegen der Behebung des Schadens aufzunehmen. Beteiligte an einer Retoure werden auf der Registerkarte Beteiligte erfasst bzw. angezeigt. Markieren Sie den benötigten Beteiligten und erteilen Sie ihm Meldung (z.B. auch per E-Mail). 

### Belastung erfassen 
Zur Erfassung einer Belastung drücken Sie den Button Belastung . Es öffnet sich der Dialog Belastung erstellen . Hier können Sie eine Belastungsanzeige an den Lieferanten erstellen, um die Retoure an den Lieferanten weitergeben zu können. Zuvor muss die Retourenposition freigegeben und der Wareneingang gebucht sein. 

Diese Belastung ist dann im Einkauf im Dialog Belastung ersichtlich. 

Sind im Dialog mehrere Lieferanten zur Auswahl angeboten, achten Sie sehr sorgfältig darauf, dass Sie den richtigen Lieferanten auswählen, ansonsten erhält ein Lieferant die Belastung angezeigt für Ware, welche von einem anderen Lieferanten bezogen wurde. Sollte dies der Fall sein, so erscheinen mehrere Lieferanten im Dialog der Belastungserstellung und es erscheint eine Warnmeldung. 

Nach der Erstellung der Belastung wird die Nummer der Belastung zur Retourenposition eingetragen, damit diese nachvollziehbar ist. Ebenso kann dadurch keine doppelte Belastung an den Lieferanten erfolgen. 

Wenn Sie sich für eine Wertbelastung entscheiden, aktivieren Sie die Checkbox Wertbelastung im Dialog Belastung . Mit der Entscheidung für eine Wertbelastung ergibt sich ein sogenannter Wertefluss. Um den Wertefluss abbilden zu können, ist im Dialog Parameter Verkauf , Registerkarte Retoure im Feld Kunde eine entsprechend definierte Kundennummer auszuwählen (eine sogenannte Dummy-Kundennummer).
