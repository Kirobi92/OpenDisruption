# EDI Adapter 

### Elektronische Bestellung 
EDI (Electronic Data Interchange) ist ein internationaler Standard, um geschäftliche Daten interventionsfrei zwischen den Anwendungsprogrammen der beteiligten Partner auszutauschen. Grundvoraussetzung für diesen Datenaustausch ist die Abbildung einer Struktur auf eine andere, das heißt Erstellung eines Mappings zur Konvertierung der Nachrichten. Für eNVenta werden dieser Nachrichtentransportweg und die Grundlagen der Verarbeitung im Gateway parametrisiert. Es muss gewährleistet werden, dass jede gesendete Nachricht den Empfänger einmal erreicht. 

eNVenta verwendet InHouse-eigene Formate. 

Folgende EDI-Nachrichtentypen werden zwischen dem E/D/E und eNVenta ausgetauscht: 

Nachrichten Austausch 
| | | |
| --- | --- | --- |
| ORDERS | Bestellung (Order) | Export |
| ORDRSP | Auftragsbestätigung (Orders Response) | Import |
| DESADV | Lieferavis (Despatch Advice Message) | Import |
| INVOIC | Rechnung (Invoice) | Import |

Bestelländerungen werden telefonisch geregelt. 

Manuell oder automatisch generierte Bestellungen werden von eNVenta via FTP-Server an den E/D/E gesendet. 

E/D/E stellt die in eNVenta zu importierenden Daten auf deren FTP-Server zur Verfügung. 

Zu beachten bei INVOIC: 

Bei den bereitgestellten Prozessdaten wird vom E/D/E zusätzlich zur INVOIC die herkömmliche Rechnung in Papierform zur Verfügung gestellt. 

Beim Umschalten auf signierte Belege wird vom E/D/E zusätzlich zur INVOIC die eSignatur mit PDF zur INVOIC bereitgestellt, die Papierrechnung entfällt damit. 

Im Modul Werkzeuge >> Gateway befindet sich die gesamte Verwaltung. Im Menü Optionen >> Gateway Adapter müssen die jeweiligen Adapter angelegt sein. 

Siehe Kapitel Gateway Adapter . 

In der Spalte ‚AdapterName‘ muss genau der Name Adapter EDI des Adapters angegeben werden, so wie er auch als Klassenname (AdapterEDI) spezifiziert ist. Danach klicken Sie die Checkbox Aktiviert an. 

In den Picklisten unter System >> Allgemein >> Picklisten steht im Feld Einträge zum EDI Message Format die Formatierung ‚EDI‘. 

Zum EDI Message Type sind alle für den EDI benötigten EDI-Nachrichten vorhanden. 

Hier geben Sie an, welche EDI-Nachricht für den Kunden bzw. Lieferanten zur Verfügung stehen soll. Ebenso wird die Art der Kommunikationsrichtung hinterlegt. 

Unter Optionen >> EDI Parameter können allgemeingültige Einstellungen für den EDI Import bzw. Export vorgenommen werden. 

### Einstellungen im Dialog 'EDI Parameter Registerkarte Import' 

#### Import: 
Kein Postext 
Wenn diese Einstellung aktiv ist, dann werden aus der zu importierenden EDI Datei die DT17-Sätze nicht verarbeitet. 

Default Pfade 
Diese Pfade dienen zur leichteren Verwaltung der einzelnen Nachrichten beim Kommunikationsteilnehmer (Kunde/Lieferant). Beim Anlegen einer Nachricht für den Import werden diese Pfade als Standardpfade vorgegeben. 

Preise aktualisieren 
Die Preise (aus dem Artikelstamm) werden aus der importierten Datei als Preise angenommen. 

Wichtig: Es ist unbedingt darauf zu achten, dass nur EDI-Dateien im Inhouse Format im Importverzeichnis liegen. Der EDI-Adapter versucht alle in diesem Import-Verzeichnis befindlichen Dateien einzulesen und bricht ab, wenn eine Datei nicht korrekt gelesen werden kann . 

Siehe Kapitel EDI Parameter Registerkarte Import. 

### Einstellungen im Dialog 'EDI Parameter Registerkarte Export' 

#### Export: 
Rabatt/Zuschl Feld: 
Die Rabattprozent-Felder 1-6 werden an anderer Stelle in die EDI-Datei geschrieben, wenn die Werte die Länge von 8 Zeichen erreichen können. (Siehe auch separates Dokument zum EDI-Format.) 

DT13: 
Die Adress-Datensätze sollen beim Rechnungsexport in die Datei geschrieben werden. 

DT13SU: 
Die Lieferantenadresse soll beim Rechnungsexport gesondert in die Datei geschrieben werden. 

Kein Postext: 
Bei einer Bestellung sollen keine Positionstexte aus den Positionen in die Exportdatei geschrieben werden. 

Default Exportpfad: 
Wie beim Import kann ein Standard Exportpfad für alle Nachrichten der Kommunikationsteilnehmer hinterlegt werden. 

Unter Optionen >> Parameter Auftragsimport können gesonderte Einstellungen (Importprofile) für den Auftragsimport erstellt und verwaltet werden. 

Diese Einstellungen gelten für den ORDERS-Import. 

Siehe Kapitel EDI Parameter . 

Siehe Kapitel Parameter - Auftragsimport . 
### Einstellungen im Dialog Parameter - Auftragsimport 

#### Import-Profil: 
Ein frei zu vergebender Name für das Profil kann eingegeben werden. 

Fehlende Preise ergänzen: 
Alle fehlenden Positionspreise aus der Datei werden über das System neu ermittelt. 

Preise und Kond. erm: 
Alle Preise und Konditionen werden neu ermittelt. Wenn also diese Einstellung aktiv ist, dann ist die andere Einstellung Fehlende Preise ergänzen überflüssig und wird im Importvorgang ignoriert. 

EK ergänzen: 
Fehlende EKs in der Importdatei werden aus dem Artikelstamm ermittelt und zugewiesen. 

Artikel Verfügbarkeitsprüfung: 
Bei jeder Position wird die Mengenverfügbarkeit geprüft und ggf. als Fehlermeldung im Gateway Protokoll ausgegeben. 

Unter Optionen >> ISO-Einheiten sind alle Verkaufs-/Einkaufseinheiten zu pflegen: 

In der Spalte ‚Einheit VK‘ müssen alle internen Einheiten (bitte aus den Picklisten entnehmen) angelegt und die gültige ISO-Einheit für den Import, sowie Export eingetragen werden. Bei fehlenden Einheiten wird der EDI-Import / -Export abgebrochen. 

Siehe Kapitel ISO-Einheiten . 

Unter Optionen >> EDI-Teilnehmer können für einzelne Kunden oder Lieferanten Nachrichten aus der EDI-Nachrichten-Liste zugewiesen werden: 

Um einen Kommunikationsteilnehmer einzufügen, muss rechts neben der oberen Tabelle auf den Button geklickt werden. Der Dialog AKL-Auswahl öffnet sich. 

Anschließend werden die EDI Nachrichten angelegt. Der Kommunikationsteilnehmer muss markiert sein. Dazu muss rechts neben der unteren Tabelle der Button geklickt werden. 

Abhängig von der jeweiligen Nachricht des zugewiesenen Kunden - ob Import oder Export - sind Dateipfade verfügbar. 

Siehe Kapitel EDI-Teilnehmer . 

### Einstellungen in den Referenz Parametern: 

#### Für den Import: 
- Quellpfad für die zu importierenden Dateien - Zielpfad für die erfolgreich importierten Dateien - Fehlerpfad für die fehlerhaften Importe 
Für alle Pfade gilt: Diese Pfade müssen immer Pfade auf dem Broker und voll qualifiziert sein. 

Beispiel: 
\ServerXEDIimportImportKD4711 funktioniert hingegen nicht. 

Alle Dateien werden in UTF8 exportiert. Beim Import wird auch die eingehende Datei als UTF8-codierte Datei gelesen. Falls ein externer Konverter zum Einsatz gebracht wird, muss dieser Konverter die Dateien auch in UTF8 für den Import liefern. 

#### Für den Export: 
Exportpfad 
Verbands Nr.: Wenn eine Verbandsnummer eingetragen ist, dann erfolgt der Export über die Einstellungen des Verbands. Der Verband muss folglich auch in den Referenz-Parametern für die entsprechende Nachricht konfiguriert sein. Wenn unsere Nummer eingestellt ist, dann sollen nicht die Verbandseinstellungen verwendet werden, sondern die eigenen Exporteinstellungen. 

Zum manuellen Export / Import klicken Sie im Dialog Gateway Adapter Kontext in die Zeile Adapter EDI und klicken auf den Button , rechts neben der Tabelle. Es öffnet sich der EDI Adapter , wo Sie den Button Start drücken, um die EDI Export- / Import-Events auszulösen. 

### EDI Adapter 
Import 
Der Import wird über alle Kommunikationspartner (Kunden / Lieferanten) verarbeitet. Für jeden Kommunikationspartner wird geprüft, ob in dem jeweiligen konfigurierten Importordner Dateien existieren, die zu verarbeiten sind. 

Export 
Für den Export müssen die Ereignisse für den EDI Adapter im Menü Gateway Adapter Kontext aktiv sein und der Kommunikationsteilnehmer entsprechend in den EDI-Teilnehmern konfiguriert sein. 

Das automatische Auslösen des Adapters kann über den Job-Server erfolgen. 

Siehe Kapitel Job-Verwaltung . 

### Cultures angeben 
Für den Import und Export von Daten per EDI-Adapter kann die Culture angegeben werden. Diese Culture entscheidet z.B. darüber, welches Dezimaltrennzeichen für Zahlenwerte genommen wird. 

In der Pickliste „Laendereinstellung (CultureInfo)“ werden die verfügbaren Cultures hinterlegt. 

Diese Culture-Einstellungen können in den EDI-Teilnehmern bzw. Gateway-Parametern hinterlegt werden.
