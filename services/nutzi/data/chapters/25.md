# Update-Assistent 
Update-Assistent 

Beim Start von eNVenta öffnet sich der Dialog Update-Assistent , der Sie beim Update Ihrer Datenbank optimal unterstützt. Bei jedem Start von eNVenta wird geprüft ob eine neue Version vorliegt und wenn ja ob noch Update-Scripte ausstehend sind. Sollten Sie der erste Benutzer der neuen Version sein und keine Rechte zum Update auf der Datenbank haben, so erscheint eine entsprechende Meldung und Sie können eNVenta nicht starten. -> Wenden Sie sich bitte an Ihren Systemadministrator. 

Die notwendigen Änderungsskripte für die aktuelle Version werden über diesen Dialog automatisch aus der Differenz zwischen den DB-Tabellen und der DBTable-Schicht aus Framework Studio ermittelt. Dadurch wird ein manuelles Ausführen einzelner Updateskripte, wie Sie es von früheren Versionen kennen, bei der Erstellung neuer Spalten während der Entwicklung überflüssig. 

Das Update wird schrittweise ausgeführt. Dafür steht die Registerkarte Update-Schritte zur Verfügung. Hier führen Sie bitte die Schritte 1 bis 5 nacheinander aus. Für jeden Schritt erscheint ein Symbol, wenn der Schritt erfolgreich ausgeführt wurde. Überspringen Sie keinen Schritt, da ansonsten eNVenta evtl. die Daten nicht korrekt speichern kann, was zu Folgeproblemen führen kann. 
### Beschreibung der einzelnen Schritte: 

#### Schritt 1: 
Alle Skripte, welche vor der Version 3.3 vorhanden waren, werden hier ausgeführt. 
#### Schritt 2: 
Erweiterung der bestehenden Strukturen. Darunter fallen neue Tabellen, neue Spalten, Erweiterung der bestehenden Spalten (z.B. Decimal 15,0 auf Decimal 30,3), Anlage neuer Views… 

Alle unkritischen Änderungen können mit dem Button Ausführen sofort ausgeführt werden. Wenn es kritische oder keine möglichen Skripte gibt, so können diese über den Button Details betrachtet werden. 

Ziel ist es hier die Struktur soweit anzupassen, damit im nächsten Schritt die Dateninhalte aktualisiert werden können. 
#### Schritt 3: 
Ausführung von DML-Scripts. Hier werden alle Insert-, Update-, …. Befehle ausgeführt, um neue Tabellen mit Default-Werten zu füllen oder auch um Inhalte gezielt zu ändern. Diese Skripte werden automatisch für alle Mandanten bereitgehalten, wenn notwendig und entsprechend konfiguriert. 
#### Schritt 4: 
Nachbearbeitung der Struktur (vorausgesetzt, diese sind durch DML-Skripte gefüllt) so können nachträgliche Struktur-Skripte ausgeführt werden, z.B. Spalten als "Not NULL" deklariert werden. 
#### Schritt 5: 
Setzen der Version. Mit Klick auf den Button Ausführen wird die Version als 'aktualisiert' in der Datenbank vermerkt. Beim nächsten Start von eNVenta erscheint der Update-Assistent nicht mehr. 

Wenn beim Start noch ein Update-Script auszuführen ist, der Benutzer aber keine Berechtigung hat, so erscheint eine entsprechende Hinweismeldung. 
## Steuerelemente des Formulars 

### Bezeichnungsfeld Update-Assistent 
Tooltip: 

Dieser Dialog unterstützt Sie beim Update Ihrer Datenbank. 
Für mehr Informationden drücken Sie F1. 

### Button Update 

Wenn Sie diesen Button drücken, werden alle Update-Schritte ausgeführt. 

Tooltip: 

Es werden alle Update-Schritte ausgeführt. 

### Button Anwendung Neu Starten 

Wenn Sie diesen Button drücken, wird die Anwendung neu gestartet. 

### Button Anwendung Beenden 

Wenn Sie diesen Button drücken, wird die Anwendung beendet und geschlossen. 

### Button Ohne Update Starten 

Wenn Sie diesen Button drücken, wird die Anwendung ohne Update gestartet. 

Das ist nur bei Änderungen innerhalb der aktuellen Version möglich. Bei einer neuen Version oder einem neuen Package ist ein Update zwingend erforderlich! 

Tooltip:
