# Dokumenten-Server 
Dokumenten-Server 

In eNVenta ERP haben sämtliche Clients Zugriff auf ein und denselben Dokumenten-Server. Diesem Dokumenten-Server werden die Archive, die Sie im Einsatz haben, zugeordnet. 

Über System >> Allgemein >> Archiv >> Dokumenten-Server stehen alle erkannten Dokumenten-Server zur Verfügung, bzw. können weitere Dokumenten-Server eingebunden werden. In der Spalte 'Doku-Server Name' geben Sie die Bezeichnung des Dokumenten-Servers an und in der Spalte 'Url' geben Sie den absoluten Pfad zum Dokumenten-Server an. 

Dabei werden über den Button Channels alle angebundenen Archiv-Systeme zu dem markierten Dokumenten-Server aufgerufen. Ausführliche Informationen hierzu erhalten Sie im Dokument Installation Dokumenten-Server . 

Siehe Kapitel Channels . 

### Proxess (Dokumenten-Server) 
Um das Archiv-System Proxess nutzen zu können, sind in eNVenta Anpassungen vorzunehmen, die nachfolgend beschrieben werden: 

Im Dialog Dokumenten-Server steht die zusätzliche Spalte ‚Url 3.5‘ zur Verfügung. 

3.5 steht hier für die Versionsnummer des Dokumenten-Services. 

Barcode-Belege können direkt in das Archiv eingefügt werden. Das entsprechende Merkmal wird erkannt und die Belege als ganz normale Belege ohne Dateien eingefügt. Die Archiv-Mappe landet in diesem Fall nicht im externen Archivpool. 

### Barcode-Pool bei Proxess-Archiv 
Der eNVenta ERP -Dokumenten-Service ist nicht hart auf den Barcode-Pool festgelegt. Der Barcode-Pool (genauer gesagt die Bezeichnung des Dokumententyps) kann frei in der Config-Datei gewählt / eingestellt werden. 

Ebenso können mehrere Barcode-Pools angebunden werden. Für den Proxess-Channel gibt es Einträge mit dem Key "BarcodePoolName" in der DocumentService.exe.config. Hier werden mit dem SELBEN Key untereinander die verschiedenen Barcode-Pool-Namen angegeben, welche eNVenta berücksichtigen soll. 

Damit steht eine komfortablere Nutzung des Dokumenten-Servers zur Verfügung.
