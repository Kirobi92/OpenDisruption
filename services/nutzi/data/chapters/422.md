# Html Report 
Html Report 

Html Reports stehen in den verschiedenen Dialogen unter dem Form-Menü-Button zur Verfügung, z.B. Auftrag, Kundenstamm, Artikelstamm. 

Ein dynamischer Report wird zur Laufzeit generiert und kann mittels des Html Report-Designers innerhalb eines vorgegebenen Rasters an Ihre Ansprüche angepasst werden - ohne Einsatz von Framework-Studio. 

Bilder können eingebunden werden. Damit wird eine bessere Beauskunftung für Artikel möglich. 

Dafür werden die üblichen Bildformate wie PNG, JPEG, GIF etc. unterstützt. Das Bild wird in der bestehenden Auflösung angezeigt. 

Neben Ihren gewohnten Druckbelegen aus Crystal Reports können Ihre Berichte auch dynamisch dargestellt werden: Das bedeutet, in Ihren Berichten werden, ähnlich der Darstellung auf einer Internetseite, Links zu weiterführenden Informationen eingebettet, denen Sie bei Bedarf folgen können. Fast 100 Vorlagen für dynamische Reports werden im Standard mitgeliefert, darunter beispielsweise das Kundenstammblatt, Konditionsreport, Besuchsbericht etc. 

Einer der Vorteile dynamisch erstellter Reports besteht darin, dass Sie eigene Auswertungen ohne Programmierung entsprechend Ihren Bedürfnissen und Ihrer Berechtigung generieren können. 

Ebenfalls ein Vorteil ist die Nutzung der Html Reports in Verwendung der App „eNVenta2Go“ für mobile Geräte ohne Java-Client (Android und Apple-Geräte) – hier können Sie direkt aus der App auf Vorgänge in eNVenta zugreifen ohne ein eNVenta zu nutzen. Sie können unterwegs Berichte einsehen und entsprechend reagieren. 

Komplexe Berichte werden durch die eingebetteten Referenzlinks zu den Vorgängen transparent. 

Gedruckte Dokumente aus der Belegfolge (Verkauf, Einkauf und Produktion) werden direkt im Html Report Viewer angezeigt. Vorteil ist die direkte Anzeige des Inhalts des Reports anstatt den Dialog Druckbelege vorher öffnen zu müssen. 

Folgende Dokumente werden unterstützt: 

- Verkaufsbelege (Angebot, Auftragsbestätigung, Lieferschein, Packliste, Rechnung, Gutschrift) - Retourenbelege - Einkaufsbelege (Anfrage, Bestellung, Belastung …) - Produktionsbelege 
Aus dem Fokus-Element Verbindungen können die Belege ebenfalls aufgerufen werden. 

Auch die in manchen Html Reports enthaltenen Links auf Druckbelege werden unterstützt. 

Das Blättern in gedruckten Dokumenten wird unterstützt. 

Zusätzlich wird in der Belegfolge ein Symbol angezeigt, falls es sich bei dem Eintrag um einen Report handelt. 

### Funktionsweise der Html Reports 
Der Report Viewer zeigt einen Html-Report in einem Browser-Control an. Dabei werden zwei Blätterfunktionen angeboten: 

1. Blättern im Browser-Modus (Buttons links über dem Browser-Control) 
2. Logisches Blättern der Datenobjekte, wenn verfügbar (Buttons in Titelleiste) 

Ein Business-Objekt wie z.B. ein Auftrag oder ein Artikelstammsatz kann per Drag&Drop auf den Viewer gezogen werden. Wenn es einen geeigneten Report gibt, wird dieser angezeigt. Sollte es mehrere passende Reports geben, wird der erste gefundene Report mit der Markierung "Standard" gewählt (Checkbox im Report-Designer ). 

Der Viewer erlaubt eine Dokumentennavigation. Als Link unterstrichen dargestellte Elementwerte lassen eine Navigation in das entsprechende Dokument zu. (Zurück per Blättern-Funktion). 

Unter Optionen stehen die Einträge Html Report Designer und Bearbeiten zur Verfügung. Mit dem Menü Html Report Designer kann ein Report gestaltet werden. Mit der Option Bearbeiten öffnen Sie den Vorgang, welcher mit dem Html Report verknüpft ist, z.B. den Artikelstamm aus dem Report des Artikelstamm-Blattes heraus. Hierbei ist zu beachten, dass jeder Benutzer Zugriff auf die Reports erhält, wenn er die entsprechende Ansichts-Berechtigung zum Datenobjekt hat. So kann ein Verkäufer den Auftragsreport sehen, aber darin referenzierte Bestellungen sieht der Verkäufer nicht, da er keine Berechtigung für den Einkauf hat. Die Berechtigung erstreckt sich auf eingebettete Reports wie auch verlinkte Reports.

### Button Form-Menü 

Mit Klick auf diesen Button werden je nach Kontext und Berechtigung die unterschiedlichen Optionen des jeweiligen Dialogs hinter dem Button zur Auswahl angeboten. 

Die Funktionen für das Client Side Customizing sowie die Online-Hilfe stehen über diesen Button ebenfalls zur Verfügung.
