# Kapitel NV.ERP.Base.Common.cdSystemMM.shtCumuGroupDiscountFlagPurch

Wenn Sie diesen Parameter aktivieren, nutzen Sie die Möglichkeit zur Warengruppen-übergreifenden Rabattierung. Hierbei wird beim ersten Einfügen einer Bestellung geprüft, ob mehrere gleichartige Artikel (über die Assortierung-Gruppe im Artikelstamm definiert) in der Bestellung vorhanden sind und ob diese zu einer Staffelstufe in der Konditionierung zusammengefasst werden können. 

Jedem Artikel, der an der Assortierung teilnehmen soll, muss eine EK-Kumulationsgruppe im Artikelstamm (Registerkarte Einkauf ) zugeordnet werden. 

Die Einträge zur Kumulationsgruppe werden in einer Pickliste gepflegt. 

Siehe Kapitel Preisermittlung im Einkauf . 

Siehe Kapitel Rabatt Einkauf , Absatz Assortierung in der Rabattverwaltung. 

Tooltip: 

Wenn dieser Parameter gesetzt ist, können mehrere Artikel zu einer Kumulationsgruppe zusammengefasst werden. 
In der Rabattverwaltung können Rabatte für diese Kumulationsgruppen hinterlegt werden.
