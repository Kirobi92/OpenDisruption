# Kapitel NV.ERP.Base.Common.cdSystemMM.shtNewTransferPosting

Wenn Sie diesen Parameter aktivieren, kann zwischen ‚alter‘ und ‚neuer‘ Umlagerungslogik unterschieden werden. 

Neue Umlagerungslogik bedeutet, jede Umlagerung von einem Lager auf ein anderes Lager erfolgt grundsätzlich über entsprechende Bestellungen und Umlagerungsaufträge (Verbundobjekt). Hierzu müssen die entsprechenden Einstellungen in den Lagerorten hinterlegt sein. 

Alte Umlagerungslogik bedeutet hingegen, dass die Umlagerung organisatorisch erfolgen und eine entsprechende Verkaufsauftragsart genutzt werden soll, welche ein Ziellager definiert hat. 

Siehe Kapitel Interner Umlagerungsprozess . 

Ist dieser Parameter aktiviert, wird im Dialog Lager pro Artikel die Spalte ‚Std. Lager Umlagerung‘ überflüssig und daher ausgeblendet. 

Siehe Kapitel Lagerbestand . 

Tooltip: 

Mit diesem Parameter kann zwischen 'alter' und 'neuer' Umlagerung unterschieden werden. 
Dieser Parameter spielt eine Rolle im internen Umlagerungsprozess.
