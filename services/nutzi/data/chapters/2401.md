# Journal Detail 
Journal Detail 

Der Button Detail oder ein Doppelklick in eine Zeile des Dialogs Journal öffnet diesen Dialog. In diesem Menü erhalten sie ausführlichere Informationen zu den Buchungssätzen. 

Bei Änderung der Ausgleichsbuchung zu einem OP wird nur der Journalsatz in der Ausgleichsbuchung angepasst. Eine Anpassung des Buchungsdatums kann nur innerhalb der Periode erfolgen, alles andere wird abgelehnt. 

Bei jeder Änderung des Belegdatums werden die Skontokonditionen und Nettofälligkeiten nicht automatisch angepasst. Deshalb empfiehlt es sich, diese Buchung nachträglich anzupassen. Aus dem Dialog ‚Journal Detail‘ gelangen Sie über den Button ‚Zum Buchen‘ wieder in den Buchen-Dialog.

### Button Löschen 

Über diesen Button haben Sie grundsätzlich die Möglichkeit eine Buchung zu annullieren (löschen). Zum Beispiel wird eine Debitor-Rechnung einschließlich des OPs gelöscht. Dies ist nur möglich, wenn diese Rechnung nicht bereits ausgeglichen ist. 

Diese Möglichkeit haben Sie nicht, wenn diese Buchung zum Beispiel bereits journalisiert ist oder anderes damit passierte. Die Buchung enthält in diesem Fall einen Hinweis oberhalb der Tabelle. Wenn z.B. bereits eine Verbandsabrechnung erfolgt ist, müssen Sie die Verbandsabrechnung zu dieser Buchung erst rückabwickeln (annullieren). 

Bitte wenden Sie das Annullieren verantwortungsvoll an, bitte prüfen Sie sorgfältig.

### Button Storno 

Dieser Button öffnet den Dialog Storno . 

Dieser Button ist gesperrt, wenn nach Konto- bzw. Personenkontonummer selektiert wird, da nicht der gesamte Buchungssatz für die Prüfung der Stornofähigkeit zur Verfügung steht. 

Die Prüfung, ob ein Buchungssatz wirklich storniert werden kann, findet erst statt, wenn Sie den Button Storno drücken. D.h. eine automatische Prüfung im Hintergrund findet nicht statt. Falls der Buchungssatz nicht storniert werden kann, erhalten Sie eine Meldung. 

Siehe Kapitel Storno im Journal .

### Feld Hinweis 
Hier wird ggf. ein Hinweis ausgewiesen. Diese Hinweise deuten darauf hin, dass mit der Ursprungsbuchung bereits Weiteres passierte; z.B. dass sie journalisiert wurde bzw. eine Ausgleichsbuchung oder Verbandsabrechnung erfolgt ist. 

Diese Buchungen sind evtl. nur eingeschränkt änderbar/annullierbar. 

Ein Hinweis erfolgt auch für eine bereits stornierte Buchung. Damit wird verhindert, dass eine bereits stornierte Buchung noch mal storniert wird. 

Eine weitere automatische Stornobuchung wird so lange verweigert, solange diese Spalte in der zu stornierenden Buchung befüllt und die referenzierte Stornobuchung (noch) vorhanden ist. 

Dabei ist zu beachten, dass bestimmte Vorgänge die Zuordnung von stornierten Buchungen zur Stornobuchung aufheben. 

Beispiel: 
- Mit der stornierten Buchung zurück zum Dialog Buchen - Ändern der Betragsfelder der Stornobuchung 
Das Buchungsdatum und das Belegdatum können noch geändert werden. 

### Checkbox Automatische Verteilung 
Diese Checkbox ist standardmäßig aktiviert. D.h. die Beträge der Buchungssätze werden automatisch verteilt. Wird diese Checkbox deaktiviert, kann in jeder Buchungszeile der Betrag manuell geändert werden. Der Benutzer ist dann für die Soll-/Haben-Gleichheit selbst verantwortlich. 

### Checkbox Einheitlicher Buchungstext 
Ist diese Checkbox gesetzt, so wird der geänderte Buchungstext auf alle Journalzeilen übertragen. Ist die Checkbox nicht gesetzt, so gilt die Änderung nur für die aktuelle Zeile. 

Die Änderung des Buchungstextes ist unabhängig von der Checkbox auto. Verteilung . 

Tooltip: 

Hier wird ein einheitlicher Buchungstext angegeben.
