# Kopieren eines Belegs im Verkauf 
Beleg kopieren 

eNVenta ERP bietet eine einfache Möglichkeit, ganze Aufträge, Angebote oder Gutschriften zu kopieren. Dazu wird der zu kopierende Beleg in den entsprechenden Dialog eingelesen und der Dialog Beleg kopieren über das Menü Optionen >> Beleg kopieren oder den Button geöffnet. Geben Sie die Parameter zum Kopieren ein und führen Sie den Kopiervorgang mit dem Button durch. 

Es ist auch möglich Verkaufsbelege auf einen Kunden mit einer anderen Währung zu kopieren. Dabei werden die Preise in die neue Währung umgerechnet. Aus diesem Grund sind die Checkboxen Preis/Rabatt akt. und Stckl. re. Kalk. systemseitig gesetzt. 

Beim Kopieren eines Angebots für einen Kunden, der sein Kreditlimit überschritten hat, erscheint eine Hinweismeldung, nachdem der Auftrag erzeugt wurde. Wenn Sie die Meldung bestätigen, wird der Dialog Beleg kopieren geschlossen und kein zweiter Auftrag angelegt. Außerdem wird der erzeugte Auftrag auf „Gesamtsperre“ gesetzt. D.h. es kann keine Rechnung gedruckt werden. 

Wenn Sie ein Angebot in einen Auftrag kopieren möchten, welches gelöschte Artikel enthält, werden diese gelöschten Artikel nicht in den Auftrag eingetragen. 

Wenn Sie beim Kopieren eines Angebots in einen Auftrag, oder eines Auftrags in einen anderen Auftrag, versuchen das Lager zu wechseln, erhalten Sie einen Hinweis, falls das gewählte Lager für die Position nicht zugelassen ist. 

Beim Kopieren eines Angebots in einen Auftrag werden die Start- und Endetexte aus dem Angebot nicht in den Auftrag übernommen. 

Beim Kopieren von Verkaufsbelegen werden Start- und Endetexte immer dann neu ermittelt, wenn der Kunde oder der Belegtyp (Angebot -> Auftrag etc.) gewechselt wird. 

Beim Kopieren eines Streckenauftrags in einen Sofortauftrag wird der Versandweg aus der Auftragsart übernommen. Das Bezugslager im Streckenauftrag wird geleert. Damit das Lager aktualisiert wird, ist im Dialog Beleg kopieren die Checkbox Lagerort akt. standardmäßig gesetzt. 

Beim Kopieren eines Streckenauftrags in einen Auftrag wird die Lieferantennummer nicht mitgegeben. Ursache ist, dass bei Streckenaufträgen der Lieferant auf den Hauptlieferanten des Artikels gesetzt wird (ohne Ausnahme!). Dies kann nachträglich manuell geändert werden. Aber sobald der Versandweg auf Strecke gesetzt ist, wird die Lieferantennummer wieder auf diesen Wert gesetzt. Dies vor dem Hintergrund, dass Streckenaufträge im Allgemeinen immer mit dem Hauptlieferanten getätigt werden. 

Sollte der Hauptlieferant nicht definiert sein, wird im kopierten Auftrag auch keine Lieferantennummer eingetragen sein. 

Nach dem Kopieren eines Auftrags für einen Kunden mit Sperre aufgrund überschrittenen Kreditlimits erscheint eine entsprechende Warnmeldung. 

Artikel, die mit „Ab Werk“ gekennzeichnet sind, werden auch nur in einen „Ab-Werk-Auftrag“ übernommen. Dies gilt sowohl für das Kopieren aus Angeboten und Aufträgen!

### Button Speichern 

Mit diesem Button wird der Kopiervorgang gestartet. Falls eine Projektnummer angegeben ist, wird diese ebenfalls zum neuen Beleg gespeichert, die Liefer- und Versandadresse werden neu zugeordnet. Nach erfolgreicher Beendigung erhalten Sie eine entsprechende Meldung. Es wird automatisch eine neue Belegnummer aus dem jeweiligen Nummernkreis vergeben und der neue Beleg in den Dialog eingelesen. 

### Button Schließen 

Mit diesem Button schließen Sie den Dialog. Bitte denken Sie daran, Ihre Eingaben / Änderungen vorher zu speichern! 

### Combobox Belegart 
Über diese Combobox kann z.B. ein Auftrag in eine andere Belegart kopiert werden. Sie finden in der Liste der Belegarten außer den Auftragsarten auch die Einträge Gutschrift und Angebot . 

Beispielsweise kann durch Auswahl der Gutschrift aus dem Auftrag eine Gutschrift erzeugt werden. Auf diese Weise ersparen Sie sich die Arbeit, die Gutschrift zu einem Auftrag komplett neu zu erfassen. Bei Gutschriften wird eine Lagergutschrift erstellt, es sei denn der Parameter Wertgutschrift ist gesetzt. 

Grundsätzlich kann mit diesem Programmteil jede Verkaufs-Belegart in eine andere überführt werden.

### Feld Projekt 
Wenn beim Kopieren eines Belegs der Kunde geändert wird und es zu diesem Kunden ein oder mehrere Projekte gibt, kann nach Auswahl der Kundennummer die entsprechende Projektnummer hier eingetragen werden. Die gewählte Projektnummer wird dann entsprechend in den neuen Beleg (Auftrag, Angebot oder Gutschrift) übernommen. 

Sollte beispielsweise ein Angebot eine Projektnummer haben, und für den neu gewählten Kunden wäre diese Projektnummer ebenfalls gültig, erfolgt keine Projektauswahl, die „gültige“ Projektnummer wird mit kopiert. Wenn nicht, wird geprüft, ob die Projektnummer zum geänderten Kunden passt. Falls nicht, öffnet sich die Projektauswahl . 

Wird der Kunde geändert, so muss als erstes im Projekt nachgeschaut werden, ob der neue Kunde auch an diesem Projekt beteiligt ist (Registerkarte Beteiligte , Tabelle Kunden ). Falls ja, kann es stehen bleiben, falls nein, wird das Feld geleert und die Adresse geändert. 

Siehe Kapitel Projektauswahl .

### Checkbox Markierte Positionen 
Wenn Sie diese Checkbox aktivieren, werden nur die in der Tabelle markierten Positionen in den neuen Beleg kopiert. Wenn Sie keine Positionen markieren, wird auch nichts in den neuen Beleg übernommen. 

### Checkbox Stornierte Positionen 
Wenn Sie diese Checkbox aktivieren, können beim Kopieren des Leitbelegs auch stornierte Positionen mit kopiert werden. Diese Checkbox wird vorbelegt, wenn ein Splitt-Auftrag kopiert wird. 

Tooltip: 

Stornierte Positionen mit kopieren 

### Checkbox Mit Vorkassenpos 
Wenn Sie diese Checkbox aktivieren, werden Vorkasse-Positionen übernommen. 

Tooltip: 

Vorkassenpositionen werden übernommen 

### Checkbox Preis/Rabatt aktualisieren 
Ist diese Option eingeschaltet, werden für den neu zu erzeugenden Beleg alle Preise und Rabatte neu ermittelt. Dies ist z.B. interessant, wenn der Beleg auf einen anderen Kunden kopiert wird und dessen individuelle Konditionen berücksichtigt werden sollen. 

Beim Kopieren eines Angebots in einen Auftrag wird diese Checkbox in Abhängigkeit von der Bindefrist vorbelegt. Ist die Bindefrist gesetzt, wird diese berücksichtigt und die Checkbox Preis/Rabatt akt. nur bei abgelaufener Bindefrist vorbelegt. 

Ist keine Bindefrist definiert, wird diese Checkbox immer vorbelegt. 

### Checkbox Stückliste rekursiv kalkulieren 
Ist diese Option eingeschaltet, können für den neu zu erzeugenden Beleg alle enthaltenen Stücklisten neu kalkuliert werden. Hierbei werden der kalkulierte Verkaufspreis und der kalkulierte Einkaufspreis in die Position eingetragen. Diese Option kann nur in Verbindung mit der Option Preis/Rabatt akt. genutzt werden. 

Tooltip: 

VK von Stücklisten rekursiv kalkulieren 

### Checkbox EK aktualisieren 
Ist diese Option eingeschaltet, werden nur die Einkaufspreise neu ermittelt und in den neu zu erzeugenden Beleg eingetragen. 

### Checkbox MTZ Refresh 
Ist diese Option eingeschaltet, kann beim Kopieren eines Belegs der MTZ neu berechnet werden. Der MTZ-Refresh wirkt sich nur auf die MTZs aus, alle anderen Positionsrabatte bleiben unberücksichtigt. 

### Checkbox Zahlungsbedingung aktualisieren 
Wenn diese Checkbox aktiviert ist, so wird die Zahlungsbedingung im Zielbeleg mit der Zahlungsbedingung aus dem Kundenstamm aktualisiert. Das kann notwendig werden, wenn beispielsweise die Auftragsart geändert wurde. 

Ist die Checkbox nicht gesetzt, so bleibt die Zahlungsbedingung erhalten. 

Tooltip: 

Ist die Checkbox "Zahlungsbedingung aktualisieren" aktiv, so wird die Zahlungsbedingung aktualisiert im Zielbeleg. 
Ist sie nicht gesetzt, so bleibt die Zahlungsbedingung erhalten. 

### Checkbox Provision aktualisieren 
Ist diese Option eingeschaltet, werden für die Erstellung des neuen Belegs die Vertreterprovisionen aktualisiert. Sollte inzwischen ein anderer Vertreter hinterlegt worden sein, so wird dieser in den kopierten Auftrag übernommen. 

### Checkbox Steuer und Erlöskonto akt. 
Ist diese Option eingeschaltet, wird für die Erstellung des neuen Belegs die Steuer aktualisiert. 

Tooltip: 

Ist diese Option eingeschaltet, wird für die Erstellung des Zielbelegs die Steuer und das Erlöskonto aktualisiert. 

### Checkbox Kostenstellen aktualisieren 
Ist diese Option eingeschaltet, werden die Kostenstellen bzw. Kostenträger aktualisiert. 

Beim Kopieren eines Angebots in einen Auftrag wird diese Checkbox vorbelegt. 

### Checkbox Kursaktualisierung 
Ist diese Option eingeschaltet, wird beim Kopieren eines Belegs der Kurs aktualisiert. 

### Checkbox Porto neu kalkulieren 
Wenn Sie einen Verkaufsbeleg, dem Porto hinterlegt ist, kopieren, wird das Porto nur mitkopiert, wenn diese Checkbox nicht aktiviert ist. 

Tooltip: 

Ist diese Checkbox gesetzt, werden die Portokosten aktualisiert. Die Verpackungskosten werden dabei geleert. 

### Checkbox Lagerort und V-Weg akt. 
Ist diese Option eingeschaltet, werden alle Lagerorte aktualisiert. Insbesondere wenn die Zielauftragsart "Lager fix" hat, wird diese Checkbox beim Kopieren vorbelegt. Ansonsten nicht. 

Wenn sich beim Kopieren eines Auftrags die Belegarten unterscheiden, wird diese Checkbox automatisch gesetzt. Der Sachbearbeiter hat dann die Möglichkeit, die Checkbox wieder zu deaktivieren. 

Tooltip: 

Lagerort und Versandweg für Positionen aktualisieren 

### Checkbox Wertgutschrift 
Ist diese Option eingeschaltet, wenn der Beleg in eine Gutschrift kopiert werden soll, entsteht eine reine Wertgutschrift ohne Lagerbuchung. 

### Checkbox Rechnungsadresse akt. 
Ist diese Option eingeschaltet, wird die Rechnungsanschrift aus dem Kundenstamm aktualisiert. Zusätzlich wird der Ansprechpartner aus dem Kundenstamm übernommen. 

Sollte im Kundenstamm kein Ansprechpartner hinterlegt sein, wird der Ansprechpartner aus dem Ursprungsbeleg übernommen. 

### Checkbox Pos. Lieferanschrift kopieren 
Wenn Sie diese Checkbox aktivieren, wird die Lieferanschrift auf Positionsebene kopiert. 

Tooltip: 

Positions-Lieferanschrift kopieren 

### Checkbox Erfasser übernehmen 
Wenn Sie diese Checkbox aktivieren, wird der Erfasser aus dem Originalbeleg in die Kopie übernommen (z.B. wenn ein Angebot in einen Auftrag kopiert wird). 

Diese Einstellung ist wichtig für die korrekte Zuordnung der Kostenstellen, wenn beispielsweise aus dem Angebot ein Auftrag wird. 

Ist diese Checkbox bereits im Dialog Parameter Verkauf Registerkarte Allgemein gesetzt, ist sie im Dialog Beleg kopieren bereits vorbelegt. 

Siehe Kapitel Kopieren eines Belegs im Verkauf . 

Siehe Kapitel Parameter Verkauf . 

Tooltip: 

Falls gesetzt, wird der Erfasser aus dem Originalbeleg in die Kopie übernommen. 
Eine globale Version dieses Parameters lässt sich auf dem Dialog "Parameter Verkauf" finden.
