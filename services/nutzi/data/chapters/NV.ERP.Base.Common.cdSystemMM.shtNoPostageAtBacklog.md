# Kapitel NV.ERP.Base.Common.cdSystemMM.shtNoPostageAtBacklog

Ist dieser Parameter gesetzt, werden ab der zweiten Teilfaktura für einen Auftrag kein Porto und keine Verpackung mehr berechnet. Hierzu wird der Portobetrag nach dem ersten Rechnungsdruck auf 0 gesetzt. 

Wenn die Funktion Kein Porto bei Rückständen in den Parameter Verkauf nicht aktiviert ist, dann kann es in den Auftragsarten und im Kunde gesteuert werden. 

Ist der Verkaufsparameter aktiv, dann wird bei Rückständen nie Porto berechnet. 

Tooltip: 

Ist dieser Parameter gesetzt, werden ab der zweiten Teilfaktura für einen Auftrag kein Porto und keine Verpackung mehr berechnet. 
Portobetrag nach dem ersten Rechnungsdruck auf 0 gesetzt.
