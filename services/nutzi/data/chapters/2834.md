# Voraussagen Registerkarte Bedarfscluster 
Voraussagen Registerkarte Bedarfscluster 

Die Einteilung der Kunden in Cluster erfolgt hierbei anhand des eNVenta Machine Learning . Dieser Wert kann über die Anzahl Max. Cluster beeinflusst werden. Dazu kann in das Feld Max. Cluster eine maximale Zahl eingegeben werden (z.B. 80). Nach Klicken des Buttons Evaluieren wird nach einigen Minuten Wartezeit eine Graphik angezeigt. Der optimale Wert wird durch das rote Kreuz markiert und stellt die maximale Entfernung eines Punktes auf dem Bogen zur Sehne dar. 

Der Wert kann dann wieder in das Feld Max. Cluster eingegeben werden. Nun kann ein Job zur Generierung dieser Cluster gestartet werden (Button Job Server auf der Registerkarte einmalig, sofort) oder im Dialog-Header für eine regelmäßige Generierung mit diesen Daten. 

Achtung: wird der Wert Max. Cluster für die Evaluation zu hoch angegeben, bricht die Evaluation nach einiger Zeit mit einer Fehlermeldung ab. Bitte mit einem niedrigeren Wert noch einmal versuchen. 

Achtung: In JEDEN Kundenstammsatz wird die ermittelte Clusternummer oder NULL (Feld ClusterID ) eingetragen und der Stammsatz wird gespeichert. Sollte es einen oder mehrere Stammsätze mit inkonsistenten Daten geben, ist ein Speichern nicht möglich. Die Verarbeitung bricht ab. Die Fehlermeldung gibt Aufschluss über die Ursache. Der Vorgang muss nach Datenbereinigung erneut gestartet werden. 

Die so gewonnene Cluster-Zuteilung eines Kunden kann nun für verschiedene Zwecke genutzt werden. eNVenta Analytics nutzt dies für 

- Kundenspezifische Produktempfehlungen (Recommendation Engine) - Kundenspezifische Preisvorschläge (Price Prediction)

### Button Job starten 

Mit diesem Button starten Sie den Job. 

### Button Evaluieren 

Mit dieser Funktion kann die optimale Clusteranzahl ermittelt werden. Dazu wird in das Feld Max.Cluster eine maximale Zahl eingegeben werden (z.B. 80). Nach Klicken des Buttons Evaluieren wird nach einigen Minuten Wartezeit eine Graphik angezeigt. Der optimale Wert wird durch das rote Kreuz markiert und stellt die maximale Entfernung eines Punktes auf dem Bogen zur Sehne dar. Der Wert kann dann in das Feld Max. Cluster eingegeben werden. 

Nun kann ein Job zur Generierung dieser Cluster gestartet werden (Button Job Server auf der Registerkarte einmalig sofort) oder im Dialog-Header für eine regelmäßige Generierung mit diesen Daten. 

Achtung: Wird der Wert Max. Cluster zu hoch angegeben, bricht die Evaluation nach einer Zeit mit einer Fehlermeldung ab. Bitte mit einem niedrigeren Wert noch einmal versuchen. 

Achtung: In JEDEN Kundenstammsatz wird die ermittelte Cluster-Nummer oder NULL (Feld ClusterID ) eingetragen und der Stammsatz gespeichert. Sollte es einen oder mehrere Stammsätze mit inkonsistenten Daten geben, ist ein Speichern nicht möglich. Die Verarbeitung bricht ab. Die Fehlermeldung gibt Aufschluss über die Ursache. Der Vorgang muss nach Datenbereinigung erneut gestartet werden. 

### Feld Max. Bedarfsgrp. 
Hier werden die maximalen Bedarfsgruppen angegeben. Z.B. 200. Das bedeutet, dass nur Bedarfsgruppen mit einem Mindestumsatzanteil von 0,5% berücksichtigt werden (100% / 200 Gruppen). 

Tooltip:

### Feld Max. Cluster 
Hier wird die maximale Anzahl zu bildender Cluster angegeben. Bei fehlender Angabe ermittelt das System ein möglichst optimale Anzahl Cluster.
