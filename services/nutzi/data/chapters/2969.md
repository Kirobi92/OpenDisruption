# EDI Parameter Registerkarte Import 
EDI Parameter Registerkarte Import 

Auf dieser Registerkarte definieren Sie Einstellungen für den Import. 

Im Bereich ORDRSP gibt es die Combobox Wiedervorlage . 

So kann sichergestellt werden, dass im Fall einer Auftragsbestätigung, die per EDI eingeht, der entsprechende Einkaufsmitarbeiter informiert wird. 

Wenn sich die bestätigte Menge in der Importdatei von der bestellten Menge unterscheidet, wird die Position nicht bestätigt, dafür aber eine Wiedervorlage an den Erfasser generiert. 

Wird beim Import einer ORDRSP eine Menge größer der Bestellmenge bestätigt, so bekommt der entsprechende Sachbearbeiter ebenfalls eine Wiedervorlage. Diese Wiedervorlage ist mit der Bestellung (als Referenz) verknüpft, damit diese direkt angesehen werden kann. 

Die Parameter für die Wiedervorlage gelten nur für das alte Inhouse Format und nicht für das XML-Format! 

Um beim Austausch von EDI-Nachrichten Fehler bei der Übertragung zu vermeiden, wird die Nachricht zuerst mit der Dateiendung „tmp“ übertragen und anschließend auf dem Endsystem umbenannt. Dafür steht das Feld Dateiendung zur Verfügung. Wenn keine Dateiendung angegeben ist, so wird standardmäßig die Dateiendung „edi“ verwendet. 

Die Einstellung der Pfade dient als Standard-Einstellung, falls bei der einzelnen Definition pro Kunde bzw. Lieferant + Nachrichtenformat nichts hinterlegt wurde.

### Feld Temp-Pfad 
Hier wird das Verzeichnis angegeben, in dem die EDI-Dateien während ihrer Verarbeitung zwischengespeichert werden. 

Beim Import mehrerer Nachrichten im Format EDI (d.h. Inhouse-Format) vom FTP-Server wird die Datei in mehrere einzelne Dateien geteilt bevor diese anschließend einzeln verarbeitet werden. Wird ein Temp-Verzeichnis angegeben, werden die einzelnen Dateien nicht in dem Import-Verzeichnis erstellt,sondern in dem Temp-Verzeichnis. Dies ermöglicht bei einem FTP-Import-Verzeichnis eine schnellere Verarbeitung.

### Combobox Wiedervorlage 
Beim Import einer ORDRSP gibt es für den Versand der Wiedervorlage 4 Möglichkeiten: 

- Leer : es wird keine Wiedervorlage generiert - Allgemein : eine Wiedervorlage wird an den Sachbearbeiter gesendet, der auf der Registerkarte Allgemein ausgewählt ist. - Erfasser : die Wiedervorlage wird bei Abweichungen der Bestelldaten an den Erfasser der Bestellung gesendet. - Letzter Änderer : die Wiedervorlage wird bei Abweichungen der Bestelldaten an den Sachbearbeiter gesendet, der die Bestellung zuletzt geändert/bearbeitet hat. 
Diese Einträge werden in den Prozessregeln festgelegt. 

Tooltip: 

Wiedervorlage an: 
<leer> = Keine Wiedervorlage von ORDRSP Nachrichten. 
Zusammenfassung = Siehe Registerkarte Allgemein: Alle EDI Nachrichten werden gesammelt und als eine Wiedervorlage an diesen Sachbearbeiter gesendet. 
Erfasser = Einzelwiedervorlage einer jeden ORDRSP Nachricht an den jeweiligen Erfasser der einzelnen Bestellung. 

### Checkbox Automatische Rechnungsprüfung 
Wenn Sie diese Checkbox aktivieren, wird für eingehende Rechnungen sofort eine Rechnungsprüfung veranlasst. Die automatische Verbuchung erfolgt allerdings in einem weiteren Schritt anhand der definierten Regeln in der Rechnungsprüfung. 

Tooltip: 

- Automatische Rechnungsprüfung - 
Eine importierte Rechnung wird übernommen und automatisch gebucht, 
wenn die konfigurierten Gültigkeitsregeln keinen Fehler verursachen.
