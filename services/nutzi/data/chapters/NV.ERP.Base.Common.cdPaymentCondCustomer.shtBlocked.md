# Kapitel NV.ERP.Base.Common.cdPaymentCondCustomer.shtBlocked

Zahlbedingungen nur gelöscht werden, wenn sie nirgends mehr verwendet werden. Da dies im Arbeitsalltag, bei sehr vielen Vorgängen im Unternehmen, nur sehr schwer umgesetzt werden kann, gibt es die Möglichkeit, eine Zahlbedingung für den weiteren Gebrauch zu sperren. 

Diese Zahlbedingung wird dann nicht mehr angezeigt und kann auch in Auftrag, Bestellung etc. nicht mehr genutzt werden. Die Funktionalität der Zahlbedingung bleibt noch erhalten, damit die Vorgänge ordnungsgemäß abgearbeitet werden können. 

Eine gesperrte Zahlbedingung wird in den entsprechenden Comboboxen der Vorgangs-Dialoge nicht mehr angezeigt. 

Tooltip: 

Hiermit kann eine Zahlungsbedingung gesperrt werden. 
Sie kann somit nicht mehr in Comboboxen ausgewählt werden. 
Bestehende Vorgänge können dennoch weiterverarbeitet werden. 

Um eine Zahlungsbedingung zu sperren, darf diese weder im Kunden, Adressen oder Projektstamm ausgewählt sein.
