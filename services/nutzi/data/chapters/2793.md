# Online Analyse 
Online Analyse 

Die Online-Analyse erlaubt, im Sinne eines Data-Warehouse, eine mehrdimensionale Analyse von Erträgen, Umsätzen, Absatzmengen, Planzahlen, Prozesskosten etc. Dabei kann eine Drill-Down-Analyse bis hinunter auf den einzelnen Beleg erfolgen, oder die Entwicklung der Zahlen im zeitlichen Verlauf verfolgt werden. 

Funktionsprinzip 

Die Online-Analyse besteht aus einer Pivot-Tabelle, welche auf die Analyse-Datenbank zugreift. Als Analyse-Datenbank wird von eNVenta die Microsoft Analysis Services genutzt, welche Bestandteil des Produkts Microsoft SqlServer ist. Im Gegensatz zu einer relationalen Datenbank werden bei Analyse-Datenbanken Ergebnisse vorberechnet, so dass das Antwortverhalten für stark verdichtete Daten sehr schnell ist. 

Analyse-Datenbanken verwenden im Gegensatz zu transaktionsorientierten Datenbanken eine auf Abfragen optimierte Datenstruktur. Die Datenwerte, welche analysiert werden sollen, stehen in einer zentralen Fakten-Tabelle und die Tabellen, welche die Filter / Gruppierungswerte enthalten in referenzierten Dimensionstabellen. 

Snowflake - Schema 

Begriffserklärung: Faktentabelle 

Eine Faktentabelle enthält die Datenwerte, welche analysiert werden sollen. Die Werte werden durch einen Übernahmelauf aus der Geschäftsanwendung extrahiert, in die abfrageoptimierte Form transformiert und in einer eigenen Datenbank gespeichert. Die Fakten entsprechen den späteren Basiskennzahlen, also Kennzahlen, welche einfach aggregiert werden können. 

Begriffserklärung: Dimensionstabelle 

Die Dimensionstabelle enthält Eigenschaften, nach welchen gruppiert und gefiltert werden kann. Eine Dimension entspricht einer Auswertungsrichtung (z.B. Kunden, Artikel, Zeit, Projekt), also einer eigenständigen Entität, welche über einen Schlüsselwert mit der Faktentabelle verknüpft ist (z.B. die Kundennummer bei einem Auftrag). Zudem gibt es auch abhängige Dimensionen (in der Grafik die Tabellen „Hersteller“ und „Land“, welche nur indirekt mit den Fakten verknüpft sind). Man nennt diese Struktur auch „Snowflake-Schema“. Wenn es nur direkte Verknüpfungen gibt, nennt man das Schema „Star-Schema“. 

Eine Analyse-Datenbank geht in der Regel von diesem Modell aus und bildet hieraus eine Cube-Struktur. Eine Dimension entspricht einer Achse in diesem theoretischen Würfel (es sind auch mehr als 3 Achsen möglich). Allerdings sind nicht alle Eigenschaften eigene Dimensionen. Eine Region wäre z.B. von dem Standort des Kunden abhängig, also eine Gruppierung nach bestimmten Kunden. 

Theoretischer Datenraum 

Was auf einer relationalen Datenbank ein Datensatz ist, wird in einer analytischen Datenbank zu einer Datenstruktur! Die Datenwerte aus der Faktentabelle werden dann in diese Datenstruktur „einsortiert“. 

Gefüllter Würfel 

Anschließend werden zusätzlich die Aggregate für die eingefügten Datenwerte berechnet. Durch diese Vorberechnung greift das System bei einer Auswertung nicht mehr auf jeden einzelnen Datenwert zu, sondern wenn möglich auf die Zwischensummen. 

Gefüllter Würfel mit Zwischenergebnissen 

Damit arbeitet das System insgesamt sehr schnell, wenn es um stark verdichtete Daten geht. Wenn die Daten sehr detailliert werden, nähert sich das Antwortverhalten denen eines relationalen Systems. 

Pivot-Tabelle 

Um die Daten eines Cubes analysieren zu können, braucht es geeignete Werkzeuge. Eine Tabelle erlaubt üblicherweise die Darstellung von 1-2 Dimensionen. 

Um weitere Dimensionen darzustellen, wäre es z.B. möglich, unterschiedliche Tabellenblätter zu verwenden. Mit unterschiedlichen Tabellenblättern würden Sie sich für ein bestimmtes Blatt entscheiden (z.B. das aktuelle Jahr) und könnten dann die anderen 2 Dimensionen auf diesem Blatt ansehen. 

Sollen die Dimensionen in einer einzigen Tabelle angezeigt werden, wird das Kreuzprodukt aus den Dimensionen dargestellt. 

Auf diese Weise ist es möglich, beliebig viele Dimensionen in einer 2-dimensionalen Darstellung abzubilden. Die Kreuztabellen nennen sich auch Pivot-Tabellen und haben sich in der Praxis als üblicher Zugang zum Analyse-System etabliert. In eNVenta ERP ist eine Pivot-Tabelle im Bereich Online-Analyse integriert worden. Die Online-Analyse nimmt für die Auswertung Verbindung zu der Analyse-Datenbank auf, und erlaubt das einfache Erstellen von Abfragen und Berichten per Mausklick. 

Positionieren von Feldern 

Die Pivot-Tabelle besteht aus mehreren Bereichen. Links findet sich ein Baum mit Meta-Informationen, welche zum Erstellen der Abfragen genutzt werden. Der rechte Bereich ist wiederum in einen Kopf-Bereich und einen Datenbereich unterteilt. 

Die Knotenpunkte mit einer Checkbox können mit der Maus auf die Bereiche „Filterfelder“, „Datenfelder“, „Spaltenfelder“ oder „Zeilenfelder“ gezogen werden. Das momentan gezogene Feld erscheint in einer transparenten Vorschau. Sobald Sie mit dem Feld über den Bereich fahren, erscheint ein schwarzer Rahmen, der das Ziel skizziert. Sobald ein Feld auf einem Bereich positioniert wurde, wird eine Abfrage an die Datenbank gesendet. 

Die Felder können jederzeit per Maus auf einen anderen Bereich geschoben werden. Wenn es sich bei dem ausgewählten Feld um eine Hierarchie handelt (voneinander abhängige Felder), werden diese automatisch mit verschoben. 

Um ein Feld aus der Abfrage herauszunehmen, ziehen Sie dieses mit der Maus auf einen Bereich außerhalb der Feldbereiche. 

Soll eine Hierarchieebene herausgenommen werden, ziehen Sie die entsprechende Ebene einfach aus dem Feldbereich heraus. Alternativ kann auch die Checkbox neben der Ebene im Metadaten-Baum deaktiviert werden oder aus dem Kontextmenü im Tabellen-Kopfbereich die Option „Spalte ausblenden“ gewählt werden. 

Die Datenfelder, welche für die Auswertung genutzt werden können, finden sich unter dem Knotenpunkt „Kennzahlen“ im Metadaten-Baum. 

Filtern von Werten 
Der nächste Schritt ist üblicherweise, die ausgewählten Daten zu betrachten und weiter einzugrenzen. Dazu markieren Sie die relevanten Elemente im Kopfbereich der Tabelle und wählen aus dem Kontext-Menü den Eintrag „Ausgewählte Werte behalten“, um die Darstellung auf die ausgewählten Elemente einzuschränken. 

„Ausgewählte Werte filtern“ entfernt dagegen die ausgewählten Werte aus der Darstellung. 

Ist ein Feld gefiltert, erscheint daraufhin ein Filter-Icon auf dem Feld. 

Filter Icon 

Der Filter kann wieder entfernt werden, indem aus dem Kontext-Menü „Filter leeren“ gewählt wird, oder indem Sie den erweiterten Filter auf dem Feld aufrufen und den Auswahl-Filter deaktivieren. 

Filter leeren 

Gefüllter Würfel 

Durch den Filter wird der Cube auf die ausgewählten Elemente eingeschränkt. In der Konsequenz kann die Analyse-Datenbank nicht mehr auf alle vorberechneten Zwischenergebnisse zurückgreifen, sondern nur auf die, deren Dimension nicht am Filter beteiligt ist. 

Hierarchien und Felder 

Felder, welche nur eine logische Reihenfolge zulassen, werden als Hierarchien gruppiert. Der Vorteil einer Hierarchie besteht darin, dass Sie die Elemente im Tabellen-Kopf expandieren und kollabieren können. Damit ist eine Zugänglichkeit entsprechend der natürlichen Gruppierung garantiert (z.B. Jahr, Quartal, Monat, Tag). 

Felder können dagegen nicht durch Werte anderer Felder expandiert / kollabiert werden, sondern werden immer komplett dargestellt. Dafür können Felder der gleichen Dimension, welche nicht aus derselben Hierarchie kommen, auf unterschiedlichen Achsen dargestellt werden. 

Positionierung von Feldern auf verschiedenen Achsen 

Zwischensummen und Gesamtergebnis 

Sobald zwei Felder im gleichen Feld-Bereich positioniert werden, werden automatisch Zwischensummen angezeigt. 

Gesamtergebnis anzeigen 

Aus dem Kontext-Menü des Tabellen-Kopfs kann die Option Gesamtergebnis anzeigen genutzt werden. Am Ende der Tabelle wird dann eine Spalte 'Gesamtergebnis' angezeigt, welche das Aggregat der gesamten Spalte / Zeile anzeigt. Das Gesamtergebnis kann sowohl für Spalten als auch für Zeilen oder beide angezeigt werden. 

Um die Zwischensummen zu entfernen, klicken Sie aus dem Kontext-Menü des Feldes die Option Feldeinstellungen . 

Feldeinstellungen für Zwischenergebnisse 

Es öffnet sich ein kleiner Dialog mit weiteren Einstellungsmöglichkeiten für das Feld. Wird statt der standardmäßigen Vorbelegung „Automatisch“ die Option „Keine“ ausgewählt, verschwinden die Zwischenergebnisse. Die Zwischenergebnisse werden ebenfalls durch den Analyse-Server berechnet. Damit ist gewährleistet, dass auch Aggregate, welche mehr Informationen benötigen als aktuell dargestellt werden, korrekt sind (z.B. Durchschnittswerte). 

Feldeinstellungsdialog 

Erweiterte Filter und Sortiermöglichkeiten 

Alle Felder in den Filterfeldern, Spaltenfelder oder im Zeilenfelder-Bereich verfügen über einen Filter-Button. Wenn Sie auf diesen Button klicken, öffnet sich ein erweiterter Filter-Dialog. Hier können Sie zwischen einem berechneten Filter und einer Elementauswahl auswählen. 

Bezeichnungsfilter 
Alle Elemente, die angezeigt werden sollen, müssen die angegebene Zeichenfolge enthalten / mit ihr beginnen / mit ihr enden. Dabei muss es sich nicht unbedingt um die im Tabellen-Kopf-Bereich angezeigten Bezeichnungen handeln, sondern kann auch auf Schlüsselwerte gesetzt werden (z.B. Artikelbezeichnung wird angezeigt, Artikelnr. soll gefiltert werden). Die Schlüsselwerte sind i.d.R. über das Feld „Key0“ zu setzen, „Key1“ wird von den BranchKey-Schlüsseln genutzt. 

Wertfilter 
Alle Elemente, die angezeigt werden sollen, müssen für eine bestimmte Kennzahl einen Wert enthalten, der kleiner / größer / gleich / zwischen den angegebenen Werten ist. 

Top N 
Es sollen nur die ersten / letzten Elemente einer Dimension angezeigt werden, sortiert nach der ausgewählten Kennzahl. Es kann hier nach Anzahl (Top 10 Produkte), Prozent (die Artikel, welche 10% vom Umsatz ausmachen) und Summe (die Artikel, mit denen 3 Millionen Umsatz gemacht werden) selektiert werden. Dabei werden die Elemente nach der Kennzahl sortiert und entsprechend dem festgelegten Grenzwert ausgewählt. 

Element Auswahl 
Es werden alle Elemente der Dimension angezeigt. Über die Checkbox kann ein Element heraus- / hinzugenommen werden. Es werden auch Elemente angezeigt, für die keine Fakten vorliegen. 

Erstellen von Auswertungen 

Oft wird die Frage gestellt: Welche Auswertungen kann denn das MIS? Natürlich gibt es bestimmte Kennzahlen, welche für die Auswertungen genutzt werden müssen. Aber diese Kennzahlen lassen oft eine Vielzahl von Auswertungen zu. Im Verkauf sind beispielsweise der Umsatz und der Ertrag sehr vielseitig nutzbar, im Finanz-Cube ist der Betrag alles, was benötigt wird, um ein Unternehmen zu bewerten. 

Wichtig ist es deshalb, sich Gedanken darüber zu machen, was man gerne auswerten möchte, und welche Daten dafür relevant sind bzw. ausgegrenzt werden müssen: 

- Sollen Retouren berücksichtigt werden? - Sollen nur bestimmte Auftragsarten ausgewertet werden? - Sollen die Kunden unterschieden werden? - Welcher Zeitbereich ist betroffen? 
Der Aufbau einer Auswertung fängt i.d.R. immer mit dem Zeitbereich an. Die Zeitdimension berücksichtigt auch verschobene Geschäftsjahre und kann entweder nach dem Geschäftsjahr oder dem Kalenderjahr ausgewählt werden. Wird die Zeit nicht gesetzt, werden alle Daten, welche an das MIS übergeben wurden, ausgewertet. 

Hinweis: Jeder Würfel hat immer eine „Standard“- Zeitdimension. Im Verkaufswürfel ist dass das Fakturadatum, im Einkauf das Bestelldatum und im Finanz-Würfel das Buchungsdatum. Zusätzlich gibt es noch weitere Daten, welche für die Auswertung relevant sein können, z.B. das Wareneingangsdatum, das Lieferdatum, das Erfassungsdatum etc. 

Um den Auswertungszeitraum zu setzen, können Sie z.B. das Feld „Geschäftsjahr“ auf den Zeilenfeldern platzieren. Anschließend filtern Sie die Jahre, welche ausgewertet werden sollen. Wenn das Jahr nicht weiter interessiert, können Sie das Feld dann in den Bereich „Filterfelder“ verschieben. Damit bleibt der Filter aktiv, die Jahr-Darstellung taucht aber nicht mehr in der Auswertungstabelle auf. 

Filter auf dem Geschäftsjahr 

Der nächste Schritt besteht in der Auswahl der Kennzahlen, welche ausgewertet werden sollen. Es ist möglich, mehr als eine Kennzahl auszuwerten. Dazu platzieren Sie einfach die Kennzahlen im Bereich Datenfelder . 

Sind alle Filter, welche die anzuzeigenden Daten einschränken sollen, gesetzt, können Sie mit der Analyse der Daten anfangen. Dazu werden weitere Felder auf die Pivot-Tabelle gezogen. Im Ergebnis lassen sich so sehr flexible Analysen durchführen. 

Auswertung der Top 10 Artikel nach Umsatz und Ertrag 

So zeigt ein zusätzlicher Top 10 Filter auf die Artikel, z.B. dass der Ertrag für den wichtigsten Artikel deutlich schlechter ist, als z.B. bei dem zweit-wichtigsten. Ursache hierfür könnte z.B. die Kundengruppe sein, welche preissensibler ist, regionale Unterschiede in der Preisgestaltung, ein Großprojekt, welches die Marge „belastet“, höhere Einkaufspreise, hohe Nebenkosten beim Verkauf etc. Auch diese Fragen kann das MIS beantworten, indem Sie den Ertrag für den Artikel unter Berücksichtigung der anderen Dimensionen wie Kunde, Region, Projekt, Versandart usw. analysieren. 

Speichern von Auswertungen 

Wenn eine bestimmte Auswertung öfters benötigt wird, besteht die Möglichkeit, diese im Benutzerkontext oder allgemein zu speichern. Abfragen im Benutzerkontext sind nur sichtbar, wenn der gleiche eNVenta -Benutzer das MIS wieder öffnet. 

Tipp: Nicht nur fertige Auswertungen sind es wert, gespeichert zu werden, auch eine Ausgangs-Situation mit Filtern für das Geschäftsjahr etc. ist es wert, gespeichert zu werden. 

Um eine Abfrage zu speichern, wechseln Sie nach dem Erstellen der Abfrage auf die Registerkarte Abfragen verwalten und wählen dort eine existierende Abfrage zum Überschreiben aus, oder erstellen eine neue Abfrage über den Button auf der Registerkarte Abfragen verwalten . 

Begriffsdefinitionen 

Definition DataWarehouse: 

Das DataWarehouse hält Geschäftsdaten in einer Form, dass diese ‚abfrageoptimal‘ sind. Das Geschäftsdatenmodell von eNVenta ERP kann Informationen sehr verteilt speichern, so dass sich darauf keine sinnvolle Struktur für den Aufbau eines Cubes definieren lässt. Deshalb wird ein Teil der Daten aus der Geschäftsdatenbank umformatiert und in die Datenstruktur des DataWarehouse gespeichert. Die Umformatierung und das Kopieren der Daten übernehmen Integration-Services Pakete. Dieser Vorgang wird auch als ETL (Extraktion, Transformation, Laden) bezeichnet. 

Definition Dimension: 

Dimensionen sind Auswertungsrichtungen. Sie können z.B. den Umsatz aus dem Verkauf nach Kundengruppe / Kunden auswerten. Damit ist Kunde eine Dimension. 

Allgemein gilt: In der Fakten-Tabelle sind zumeist nur die Schlüssel auf Dimensionstabellen enthalten. In den Dimensionstabellen können zusätzlich Attribute (z.B. Land des Kunden) gespeichert sein. 

Definition Attribut: 

Ein Attribut beschreibt Elemente aus einer Dimension. Beispielsweise ist das Land, aus dem ein Kunde stammt ein Attribut. Man kann z.B. fragen: „Zeige mir alle Kunden an, welche 2015 bei mir eingekauft haben; gruppiere sie dabei nach dem Land, aus dem sie stammen.“ 

Definition Fakt: 

Fakten sind üblicherweise Zahlen, welche sich aggregieren lassen und als Kennzahlen ausgewertet werden können. Fakten müssen immer mit allen Dimensionen in Bezug gesetzt werden, weshalb die Fakten-Tabelle über gültige Schlüssel zu allen Dimensionstabellen verfügen muss. Nur so sind korrekte und sinnvolle Analysen möglich. Es muss auch genau festgelegt werden, was geschieht wenn kein Dimensionsschlüssel vorhanden ist. 

Allgemeine Einstellungen in der Online-Analyse 

Datenbank: 

Wählen Sie hier die Data-Warehouse-Datenbank aus, welcher Sie die Daten für die Analyse entnehmen. Im Moment gibt es nur eine Datenbank „NVMIS“, welche gewählt werden kann. 

Cube: 

Wählen Sie den Cube aus, von welchem aus die Daten analysiert werden sollen. 

Ein Cube oder Data Cube OLAP-Würfel ist ein in der Data-Warehouse-Theorie gebräuchlicher Begriff zur logischen Darstellung von Daten. Die Daten werden dabei als Elemente eines mehrdimensionalen Würfels (engl. cube) angeordnet. Die Dimensionen des Cubes beschreiben die Daten und erlauben auf einfache Weise den Zugriff. Daten können über eine oder mehrere Achsen des Würfels ausgewählt werden. Die Bezeichnung OLAP stammt aus der Datenanalyse, und bedeutet Online Analytical Processing. 

Diese Art der Darstellung ist für die Analyse von Daten von Vorteil, da auf verschiedene Aspekte (Dimensionen) der Daten auf gleiche Weise zugegriffen wird. Daher auch der Einsatz bei OLAP-Anwendungen, die die Daten in einem Data-Warehouse analysieren oder visuell aufbereiten. 

Im eNVenta ERP -Standard werden 3 Cubes (Verkauf, Einkauf, Finanzbuchhaltung) mitgeliefert. 

Bei Wechsel des Cubes werden dann auch die entsprechend verfügbaren Kennzahlen und Dimensionen in der Registerkarte Pivot aktualisiert. 

Gespeicherte Abfragen: 

Haben Sie bereits eine Abfrage zusammengestellt und möchten diese nochmals benutzen, kann diese über die Registerkarte Abfrage gespeichert werden und steht in der Combobox im Kopfbereich zur Verfügung. 

Die Abfragen werden zum entsprechenden Cube gespeichert, daher muss immer zuerst der Cube gewählt werden.
