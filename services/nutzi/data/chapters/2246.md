# Katalogverwaltung Registerkarte Ressourcen 
Katalogverwaltung Registerkarte Ressourcen 

Diese Registerkarte erlaubt die Definition von Ressourcen zu einer Gruppe / einem Produkt. Eine Ressource kann eine Datei (Bild, PDF, o.ä.) sein, ein Referenzlink (Verweis auf einen Lieferantendatensatz als Hersteller, Verweis auf einen Artikel, o.ä.) oder einfach ein Text. Je nach verwendeter Ressourcengruppe stehen unterschiedliche Möglichkeiten zur Verfügung. 

Da die Ressourcen den Server-File Mechanismus nutzen, ergibt sich ein Problem, weil die Ressourcen sowohl im Intranet (z.B. bei der Erstellung von Katalogen) als auch aus dem Internet (z.B. durch den WebShop) adressiert werden müssen. 

Der ServerFile-Mechanismus bietet die Möglichkeit Dateiarchive zu registrieren, welche auf bestimmte Benutzergruppen oder Nutzungszwecke zugeschnitten sind. Der Sachbearbeiter arbeitet nur noch mit diesen Archiven, welche auf einen Brokerpfad, Netzwerkpfad oder FTP-Server verweisen können. Zwar müssen die Zugriffsrechte / Freigaben immer noch für den Webshop-Account eingerichtet werden, es kann aber ausgeschlossen werden, dass "aus Versehen" mit Client-Ressourcen hantiert wird. 

Die Ressourcen werden deshalb über die Funktion Ressourcen exportieren auf einen Datei-Pfad geschrieben und stattdessen über den ServerFile-Pfad adressiert. 

Für die Umsetzung bereits gepflegter Pfade kann der Dialog Server Datei Pfad Konverter aus Optionen >> Konvertierung >> Ressourcen >> Server Datei verwendet werden. Dafür sind die Funktionen Suchen und Ersetzen für Tabellenwerte vorhanden. 

Bei Auswahl einer Verbindung wird nach Einträgen gesucht, welche dem physikalischen Pfad aus der Verbindung im Feld durch entsprechen. Dieser Teil des Pfades kann dann durch den Wert ersetzt werden, welcher mit dem Archivnamen aus der Verbindung vorbelegt wird. 

Sprache (Optional): 
Eine Ressource kann auf eine bestimmte Sprache eingeschränkt werden. Damit ist es möglich, für verschiedene Sprachen unterschiedliche Ressourcen zu definieren, z.B. längere Informationstexte in Deutsch und Englisch. 

Gruppe: 
Die Gruppen können in der Katalogverwaltung über den Menüpunkt Stamm >> Ressourcen Gruppen definiert werden. Im Standard werden bereits fünf verschiedene Ressourcengruppen ausgeliefert. Diese fünf Ressourcengruppen werden im eNVenta Shop automatisch verarbeitet. 

Ressourcen sind so zu organisieren, dass gleichartige Ressourcen (Produktbild Groß, Produktbild Klein, Produktbild Vorne, Produktbild Hinten, Produktbild Druck, Hersteller Link, Hersteller Info, usw.) derselben Gruppe zugewiesen werden. Damit ist die Klassifizierung einer Ressource möglich. In dem Dialog zur Gruppendefinition kann eingestellt werden, ob eine Gruppe / ein Produkt mehrfach über eine Ressource einer bestimmten Gruppe verfügen kann. 

Außerdem kann festgelegt werden, welche Ressourcentypen (Datei, Referenz, Text) für eine Ressourcengruppe angeboten werden sollen und Sie können einen Standard-Ressourcenschlüssel hinterlegen. Ist ein Standard-Ressourcenschlüssel definiert, werden alle Ressourcen dieser Gruppe mit diesem Schlüssel verschlüsselt. Dieser Schlüssel kann im Webshop für den NVResourceProvider genutzt werden, um bspw. über den Schlüssel Supplier.sWebsite mit Hilfe eines Referenz Links auf einen Lieferantenstammsatz zuzugreifen und von dort auf die Eigenschaft "Website" zurückzugreifen. Damit ist eine sehr flexible Möglichkeit geschaffen worden, um Informationen überall aus dem ERP-System nachlesen zu können. 

Ressourcen Typen: 
Datei: 
Eine Ressource kann über einen Pfad referenziert oder in die Datenbank geladen werden. Über Menü System >> ECommerce / Katalog >> Parameter Katalog kann ein Standard-Pfad eingestellt werden, welcher das Wurzelverzeichnis für einen Verwaltungspfad darstellt. 

Es ist darauf zu achten, dass das eNVenta ERP -Broker Benutzerkonto Leserechte auf dieses Verzeichnis besitzt. Optimalerweise handelt es sich hierbei um ein Netzlaufwerk, das von allen Computern in der Domäne gleich festgelegt ist, oder einen Netzwerkpfad. Verwenden Sie hier keinen lokalen Pfad (z.B. C:Bilder) bzw. nur, wenn Sie die Katalogverwaltung immer von demselben System aus pflegen und alle Bilder in die Datenbank laden. Eine ausgewählte Ressource kann über den Button Hochladen (Dialog Datei auswählen ) in der Datenbank gespeichert und über den Button wieder aus der Datenbank gelöscht werden. Der ursprüngliche Verweispfad bleibt hierbei erhalten. 

Typ: Datei 
Bezeichnung: 
Bezeichnung für die Ressource. Die Bezeichnung wird im unteren TreeView angezeigt und erleichtert die Verwaltung der Ressourcen. Bei Dateien entspricht die Bezeichnung dem Dateinamen unter dem die Ressource exportiert wird und wird standardmäßig mit dem Dateinamen der ausgewählten Datei vorbelegt. 

Pfad: 
Ein Verwaltungspfad für eine Ressource. Beim Exportieren wird die Ressource in das Zielverzeichnis + Pfad geschrieben. Deshalb hat dies ein relativer Pfad zu sein. Der Pfad wird auch genutzt, um z.B. ein Bild innerhalb des Webshops zu identifizieren. 

Quelle: 
Zeigt den Pfad an, aus dem die Voransicht geladen wurde, "Datenbank" wenn die Datei in die Datenbank hochgeladen wurde.

### Feld Auslaufdatum 
Das Datum, das Sie hier eingeben kennzeichnet, wann eine Ressource ungültig wird. 

Tooltip: 

Dieses Datum gibt an, wann eine Ressource ungültig wird.

Wenn Sie diese Checkbox aktivieren, ist die Ressource als Standardressource gekennzeichnet. 

Tooltip:
