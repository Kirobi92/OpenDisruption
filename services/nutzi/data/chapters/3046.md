# Trace 
Trace 

Dieser Dialog öffnet sich im Dialog Status Information über den Button Trace . Hier können Ereignisse der eigenen Session beobachtet werden. 

Sobald der Dialog geöffnet ist, ist die Protokollierung aktiv. Wenn der Dialog geschlossen wird, endet die Protokollierung. 

Über die Checkboxen kann eingestellt werden, welche Ereignisse protokolliert werden sollen. 

Exception: Es werden Exceptions protokolliert. Diese Checkbox ist standardmäßig gesetzt. 

Load: Es werden alle Datenbank-Loads mit ihrer Suchbedingung protokolliert. Bei Problemen kann so analysiert werden, ob z.B. Mandanten-Bedingungen oder Tabellen-Filter korrekt gezogen werden. 

Request Start / Request End: Beginn und Ende eines Requests werden protokolliert. Beim Ende des Requests wird die für den Request benötigte Zeit ausgegeben. 

StackTrace: Bei den Loads wird die Aufrufliste mit ausgegeben. So können Sie sehen, wie genau es zum Aufruf dieses Loads gekommen ist und evtl. Probleme oder Fehler besser eingrenzen.

### Checkbox Exception 
Wenn diese Checkbox aktiviert ist, werden Exceptions protokolliert. Diese checkbox ist standardmäßig gesetzt. 

### Checkbox StackTrace 
Wenn diese Checkbox aktiviert ist, wird bei den Loads die Aufrufliste mit ausgegeben. So können Sie sehen, wie genau es zum Aufruf dieses Loads gekommen ist und evtl. Probleme oder Fehler besser eingrenzen.

Wenn diese Checkbox aktiviert ist, wird der Beginn eines Requests protokolliert. Beim Ende des Requests wird die für den Request benötigte Zeit ausgegeben. 

### Checkbox Request End 
Wenn diese Checkbox aktiviert ist, wird das Ende eines Requests protokolliert. Beim Ende des Requests wird die für den Request benötigte Zeit ausgegeben.
