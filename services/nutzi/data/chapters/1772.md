# Betroffene Vorgänge 
Betroffene Vorgänge 

Wird eine Terminänderung in Form einer Lieferbestätigung durch den Lieferanten gebucht, liefert der Dialog Betroffene Vorgänge (zu öffnen über das Menü Optionen in der Bestellung ) eine Liste aller betroffenen Aufträge / Betriebsaufträge, die nicht mehr termingemäß geliefert werden können. 

Ist die Checkbox Kennz. Termin Verschiebung im Dialog Parameter Einkauf aktiviert, so wird bei Terminänderung nicht der Dialog Betroffene Vorgänge geöffnet. Die Prüfung der Terminverschiebung erfolgt in diesem Fall über den Verkauf im Dialog Positionsübersicht . Hier wird das Feld EK Bestätigung von sichtbar und zugänglich. Alle nicht lieferfähigen Auftragspositionen erhalten das aktuelle Datum. 

Siehe Kapitel Parameter Einkauf . 

Ist für eine Auftragsposition bereits ein Lieferschein vorhanden, so erscheint diese Position nicht! Dies gilt auch für fakturierte bzw. stornierte Auftragspositionen. 

Betroffene Vorgänge sind Vorgänge mit deterministisch negativer Bestandssituation zu dem jeweiligen Artikel bis zum angegebenen Liefertermin. Dies bedeutet, auch wenn sich die Lieferung verzögert, der Kunde aber mit der Reserve beliefert werden kann, erscheint dieser Dialog nicht automatisch. 

Mit einem Doppelklick in eine Zeile wird der entsprechende Vorgang (z.B. Auftrag) aufgerufen. 

Der Dialog öffnet sich nur, wenn es einen betroffenen Vorgang gibt. Ansonsten erscheinen entweder die Meldungen „keine termingeänderte Position gefunden“ oder „keine betroffenen Vorgänge gefunden“.

Tooltip: 

In diesem Dialog erhalten Sie eine Liste aller betroffenen Aufträge / Betriebsaufträge, die nicht mehr termingemäß geliefert werden können. 
Für mehr Informationen drücken Sie F1. 

### Button Form-Menü 

Mit Klick auf diesen Button werden je nach Kontext und Berechtigung die unterschiedlichen Optionen des jeweiligen Dialogs hinter dem Button zur Auswahl angeboten. 

Die Funktionen für das Client Side Customizing sowie die Online-Hilfe stehen hier ebenfalls zur Verfügung.
