# BMECat - Import 
BMEcat Import 

Um einen Katalog zu importieren, wird in der Katalogverwaltung im Menü Optionen >> BMEcat >> BMEcat Import ausgewählt. Die Konfiguration für einen Import wird gespeichert, um Aktualisierungen für den Katalog einspielen zu können. 

Im Dialog BMEcat Import steuern Sie die Optionen für den Import. 

Beim BMECat wird unterschieden zwischen Lieferantenkatalogen und Kundenkatalogen. Bei Lieferantenkatalogen (Verkäufer) geht es um Stammdatenpflege und den Einkauf und bei Kundenkatalogen (Käufer) dagegen um Reimport von Katalogen, welche durch externe Werkzeuge auf Basis der aktuellen ERP-Daten erstellt wurden. 

Durch einen Import-Vorgang in den Artikelstamm werden automatisiert massenhaft Stammdaten verändert. Allein durch diese Tatsache ist ein Import immer mit Vorsicht zu genießen. Die Änderungen können nicht einfach rückgängig gemacht werden. Es ist deshalb ein Kompromiss zwischen Datensicherheit und Automatisierung zu treffen. Der BMEcat geht in der Definition davon aus, dass durch einen Import bestehende Daten überschrieben werden können. Ein Problem ist die Tatsache, dass Artikel, welche in einem früheren Katalog enthalten waren, nicht gezielt abgekündigt werden, sondern einfach nicht mehr in dem neuen Katalog enthalten sind. Diese Artikel können nur über die letzte Änderung im Artikel Schattenstamm selektiert werden. 

Um den Aufwand für die Implementierung zu reduzieren und die Import-Funktionalität in eNVenta ERP zu vereinheitlichen, erfolgt der Import über den Artikel Schattenstamm. Der Schattenstamm folgt der Logik, dass die Daten im Schattenstamm ohne weitere Bearbeitung übernommen werden können. Wenn von der inhaltlichen Korrektheit der Daten ausgegangen wird, werden die Daten derzeit mit einer festen Programmierlogik in eNVenta übernommen. Der Benutzer hat derzeit keine Möglichkeit, die Veränderungen an den Stammdaten einzeln zu überprüfen und freizugeben bzw. zu verhindern. 

Es können nur Artikel importiert werden, wenn die Bestellnummern des Lieferanten mit den eNVenta Artikelnummern übereinstimmt: 

- Beim BMEcat Import werden die Bestellnummern des Lieferanten auch als Bestellnummern importiert. Hintergrund: Beim BMEcat Export werden Artikelnummern an den Kunden exportiert. Beim Import jedoch werden die Artikelnummern des Lieferanten importiert und das sind die Bestellnummern, die beim Lieferanten gültig sind. - Beim Katalogimport wird die Struktur eingelesen. Artikel werden zugeordnet, wenn diese beim Einlesen erkannt werden (siehe 1.). - Beim Import in den Schattenstamm kann keine Artikelgruppe mitgegeben werden, bei der Übergabe aus dem Schattenstamm in den Echtstamm ist die Artikelgruppe notwendig. D.h. auch hier können nur Artikel importiert werden, welche in eNVenta bereits vorhanden sind. 
Aktuell werden die bestehenden Daten im Schattenstamm überschrieben, bspw. Einheit Lager, Einheit VK. 

Ist kein Katalog ausgewählt, sind alle Felder / Registerkarten deaktiviert. 

Wenn im Artikelstamm keine Einträge zu Artikelnummern stehen, werden die Artikel in den Schattenstamm übernommen. 

Da der Verwendungszweck in eNVenta über die Ressourcengruppen abgegrenzt wird, kann im Dialog Ressourcen Gruppen ein Verwendungszweck angegeben werden. Beim Import werden diese Ressourcen dann automatisch dem Typ zugeordnet. 

Siehe Kapitel Ressourcen Gruppen . 

Beim Einlesen der Daten für den BMECat wird ein Fehler protokolliert, wenn die Ressource keinen Pfad enthält, und die Ressource beim Export ignoriert. 

Beim Ändern des Ressourcen-Typs wird die Bezeichnung der Ressource nicht mehr auf leer gesetzt.
