# Projekt Registerkarte Phasen 
Projekt Registerkarte Phasen 

Diese Registerkarte enthält die einzelnen Phasen zu einem Projekt oder ein ganzes Schema (Button Schema ). Das ist auch gemischt oder ergänzend möglich. Beispielsweise kann ein Projekt zunächst auch ein reines Vertriebsprojekt sein und zunächst nur die Phase oder das Phasenschema für die Akquisition beinhalten. Ein weiteres Schema kann hinzugefügt werden. Die neuen Phasen werden dann einfach an die bestehenden Phasen angehängt. 

Wenn der Betriebsauftrag einer Projektphase zugeordnet ist, so wird der Betriebsauftrag angezeigt. 

Per Drag & Drop können nicht nur Phasen verschoben werden. Auch Aufgaben, Betriebsaufträge und Leistungen können von einer Phase in eine andere Phase verschoben werden. Phasen und Aufgaben können auch zwischen den Phasen per Drag & Drop verschoben werden. Dazu muss der gewählte Eintrag auf die passende Überschrift der Zielphase „gezogen“ werden. 

Im Projekt kann ein Checklisteneintrag zu einer Aufgabe als erledigt markiert werden. Dazu öffnen Sie die Aufgabe in der Registerkarte Phasen . Eine erledigte Checkliste wird mit einem X gekennzeichnet, eine offene Checkliste mit einem O (rechtes Feld). 

Um den Checklisteneintrag einer Aufgabe zu ändern, klicken Sie doppelt auf eine Aufgabe auf der Registerkarte Phasen . Es öffnet sich der Dialog Aufgabe ändern . In der Tabelle Checkliste bearbeiten Sie die Checklisten. Die Positionsnummer kann manuell angegeben werden und legt die Reihenfolge der Einträge fest. Wenn Sie die Checkbox in der Spalte „Erledigt“ aktivieren, wird automatisch das Datum (Verarbeitungsdatum) festgehalten. 

Siehe Kapitel Aufgabe ändern . 

Eine Aufgabe kann direkt innerhalb des Treeviews mit der Tastenkombination <Strg+F4> kopiert werden. Die neue Bezeichnung der Aufgabe setzt sich zusammen aus dem Namen der kopierten Aufgabe plus den Zusatz Kopie . 

Rechts unten auf der Registerkarte gibt es einen Bereich, der die Referenzen anzeigt (wenn vorhanden), die zu diesem Projekt bestehen. 
### Button Neu 

Mit diesem Button können verschiedene Elemente eingefügt werden: 

- Phase, wenn eine Phase im Baum markiert ist - Aufgabe, wenn der Eintrag „Aufgaben“ einer Phase markiert ist - Einsatz, wenn der Eintrag „Einsätze / Leistungen“ einer Phase markiert ist. 

### Button Ändern 

Mit diesem Button können alle Elemente aufgerufen und geändert werden: 

- Phasen - Einsätze - Aufgaben - Calls - Betriebsaufträge 

### Button Löschen 

Dieser Button löscht die aufgerufenen Elemente, vorausgesetzt es sind keine Aufgaben mehr vorhanden. Das Löschen hängt vom Status des Projekts und des Elements ab. Betriebsaufträge können hier nicht gelöscht werden. 

### Button Schema 

Dieser Button öffnet den Dialog Auswahl Schema .

### Button Zuweisen 

Über diesen Button definieren Sie terminliche / personelle Zuweisungen von Aufgaben und Einsätzen. Es öffnet sich der Dialog Benutzer zuweisen . 

Siehe Kapitel Zuweisen eines Benutzers . 

### Button Service 

Über diesen Button lässt sich zu einer Aufgabe oder zu einem Call ein Einsatz oder eine Leistung generieren. Es erscheint der Schnellerfassungsdialog Einsatzerfassung für Serviceeinsätze mit vorbelegten Daten. 

Führt beispielsweise eine Aufgabe zu einem Arbeitseinsatz eines Mitarbeiters und soll dieser Einsatz nach Aufwand aufgenommen und evtl. als Dienstleistung abgerechnet werden, kann direkt zur Aufgabe der Einsatz vorgeneriert und einem Mitarbeiter zugewiesen werden. 

Siehe Kapitel Einsatz im Call Center erfassen .

### Button Eröffnen 

Dieser Button öffnet eine bereits abgeschlossene Aufgabe oder einen Call wieder.

### Button Kopieren 

Dieser Button öffnet den Dialog Kopieren / Verschieben ( Kopieren / Verschieben von Projektaufgaben ). In diesem Dialog kann der Sachbearbeiter mittels Checkboxen entscheiden, welche Elemente auf die Ziel-Projektaufgabe übernommen werden.

### Combobox Bereich 
Wenn Sie aus dieser Combobox einen Eintrag auswählen, wird die Struktur sofort zum ausgewählten Bereich eingelesen: 

- Es werden nur Phasen angezeigt, mit gleicher oder ohne Bereichskennung - Es werden nur Aufgaben angezeigt, mit gleicher oder ohne Bereichskennung. - Bei den Aufgaben werden nur solche angezeigt, bei denen die zugehörige Phase dem Bereich entspricht oder keine Kennung hat. 
Dadurch lassen sich Phasen und Aufgaben filtern. Dies dient der übersichtlicheren Darstellung bei größeren Projekten. Die Combobox wird aus dem Aufgaben- oder Phasenstamm vorbelegt. 

Zu jeder Phase gibt es immer 3 Untereinträge: 

- Aufgaben - Einsätze / Leistungen (Service-Einsätze aus dem Service-Modul) - Betriebsaufträge (aus dem PPS) 
Alle Einträge sind mit farbigen LED’s markiert: 

Rot Gelb Grün 
| Es ist noch nichts passiert, bzw. noch keine Freigabe oder Zuweisung geschehen. |
| --- |
| Es wurde bereits etwas begonnen, freigegeben, zugewiesen. |
| Alle Elemente sind abgeschlossen. |

Dazu gehört auch immer eine Liste mit den Calls, die zum Projekt erfasst wurden. 

Mit einem Doppelklick auf eine Phase öffnet sich das Menü Phase ändern . 

Siehe Kapitel Ändern eine Phase . 

Phasen können per Drag & Drop in der Reihenfolge verschoben werden. 

### Combobox Verantwortlich 
Hier wird vom Programm automatisch der Sachbearbeiter eingetragen, der den Datensatz zuletzt geändert hat. 

Wenn Sie hier einen Eintrag auswählen, werden die Phasen neu eingelesen. 

Tooltip: 

Zeigt den Sachbearbeiter, der als letztes den Datensatz bearbeitet hat. 

### Baumansicht Baumansicht 

In diesem Feld wird die Struktur der Phasen und Aufgaben angezeigt. Einer Phase können auch weitere Aufgaben zugeordnet werden. 

Über F9 oder Löschen kann eine Phase oder Aufgabe gelöscht werden. 

Handelt es sich bei der Struktur der Phase oder Aufgabe um einen Call, so ist abgesichert, dass ein Call erst nach einer entsprechenden Abfrage aus einem Projekt gelöscht werden kann. 

Phase Besuchsberichte : 

Damit kann vor allem der Kontakt zum Kunden gehalten und regelmäßige Kontakte können besser gepflegt und nachverfolgt werden. 

Bei Markierung eines Besuchs wird Ihnen im rechten Feld der verkürzte Inhalt des Besuchsberichts angezeigt. 

Mit Doppelklick auf einen Besuch öffnet sich der Dialog Besuchsbericht mit den Informationen zum gewählten Bericht. 

Dabei bedeuten die Farben der Punkte in der Baumstruktur folgendes für den Status des Besuchsberichts: 

- Grün - besucht - abgesagt - Gelb - vereinbart - bestätigt - Rot - geplant 
Siehe Kapitel Besuchsberichte . 

### Button Expandieren 

Dieser Button öffnet die komplette Strukturübersicht, d.h. es werden alle Aufgaben, Einsätze und Betriebsaufträge angezeigt. 

### Button Aktualisieren 

Mit diesem Button wird die Baumstruktur aktualisiert. 

### Feld Information 
Hier wird ein individueller Text zur internen Information angegeben. 

Ist ein Eintrag erfasst, wird auf der Registerkarte Memo mit einem Sternchen (*) gekennzeichnet, dass Text hinterlegt ist. So erkennen Sie sehr schnell, ob es eine interne Information zu einem Datensatz gibt. Dies gilt systemweit. 

Mit Doppelklick auf einen Besuch (im Bereich der Phase Besuchsbericht ) öffnet sich erstens der Dialog Besuchsbericht mit den Informationen zum gewählten Bericht. Zweitens wird Ihnen im rechten Feld der verkürzte Inhalt des Besuchsberichts angezeigt. 

Siehe Kapitel Besuchsbericht . 

Tooltip: 

Enthält interne Textinformationen.
