# Gateway Protokoll 
Gateway Protokoll 

Dieser Dialog öffnet sich im Gateway Adapter Kontext >> Optionen >> Gateway Protokoll . 

Hier kann der Verlauf der Import-/Exportverarbeitung im Gateway nachvollzogen werden. Die Anzahl der Einträge kann über folgende Felder eingegrenzt werden: 

- Adapter Name: Der Name des Adapters - Datum/Zeit von/bis: Zeitspanne von/bis - Max. Zeilen: Maximale Anzahl der darzustellenden Zeilen - Checkbox Information , Warnung und Fehler : Auswahl des Fehlertyps 
In der Tabelle erhalten Sie eine Auflistung aller gefundenen Datensätze. 

Zusammengehörende Aktivitätsschritte werden in der Spalte 'Schritte' fortlaufend nummeriert und durch eine grau/weiß Markierung gekennzeichnet. Zusammen verarbeitete Aktivitätsschritte werden übergeordnet mit einer Prozedur Nr. zusammengefasst und sind durch eine farbliche Markierung gekennzeichnet (Spalte 'Proz': hell/dunkel Türkis). 

Einträge, die in der Spalte 'E' (Export-Ereignis) mit einem E gekennzeichnet sind, können mit dem Button Ereignis wdh (Export Ereignis wiederholen) wiederholt werden. D.h. das Ereignis wird mit den ursprünglichen Parametern erneut erzeugt und gezielt durch den entsprechenden Adapter/Kontext erneut verarbeitet. 

Wird eine Zeile in der Tabelle markiert, erscheint in dem Feld Fehlermeldung eine detaillierte Fehlerbeschreibung, sofern ein Fehler vorhanden ist.

### Feld Max. Zeilen 
Über dieses Feld ist in den Suchdialogen die Anzahl der einzulesenden Datensätze limitiert. Die einzelnen Datensätze werden im Dialog entsprechend gesucht und nach Erreichen der maximalen Anzahl Datensätze abgebrochen. Auf diese Weise kann eine Suche über alle Datensätze verhindert werden, welche die gesamte Performance beeinträchtigen könnte. 

Diese Einstellung nehmen Sie im Dialog Parameter Allgemein auf der Registerkarte Allgemein vor. 

Siehe Kapitel Parameter Allgemein Registerkarte Allgemein . 

Diese Einstellung lässt sich in den entsprechenden Suchdialogen (z.B. Artikelsuche, Kundenauswahl) ändern bzw. ausschalten (ausgenommen Stammdatenprogramme). 

Tooltip: 

Anzahl Sätze, die maximal eingelesen werden. 

### Checkbox Information 
Wenn Sie diese Checkbox aktivieren, wird das Protokoll auf Informationen eingeschränkt. 

### Checkbox Warnung 
Wenn Sie diese Checkbox aktivieren, wird das Protokoll auf Warnungen eingeschränkt. 

Tooltip: 

Protokoll einschränken auf Warnungen. 

### Checkbox Fehler 
Wenn Sie diese Checkbox aktivieren, wird das Protokoll auf Fehler eingeschränkt. 

Tooltip: 

Protokoll einschränken auf Fehler. 

### Tabelle 

Die Tabelle listet die verarbeiteten Datensätze auf. 

Hier werden auch Meldungen aus den Dialogen Datenimport Adapter und Datenexport Adapter angezeigt, die über den Button Ausführen erzeugt wurden. 

Mit einem Doppelklick auf einen Eintrag wird der zugrunde liegende Beleg geöffnet. 
### Tabellenspalte Vorgang 
Tooltip: 

ID des Vorgang, durch welchem die Ereignisse gestartet wurden. 
Wird beispielsweise ein EDI-Adapter gestartet, bekommen alle Ereignisse, die zu diesem Zeitpunkt verarbeitet werden sollen die selbe ID 

### Tabellenspalte Anfangszeit 
Tooltip: 

Zeitpunkt zudem die Verarbeitung des Ereignisses gestartet wurde 

### Tabellenspalte Anwendungsname 
Tooltip: 

Der Name des Adapter-Typs über den die Verarbeitung durchgeführt wurde 

### Tabellenspalte Export/Import 
Hier wird angegeben, ob es sich um einen Export oder Import handelt. 

Tooltip: 

E = Export / I = Import 

### Tabellenspalte Schritt 
Tooltip: 

Manche Verarbeitungen werden in mehreren Schritten durchgeführt. 
U.A. bei dem Export im Adapter (der Anwendung) "cMessageConverterEDI" wird das Laden und Versenden in 2 Schritten angezeigt. 

### Tabellenspalte Modulname 
Tooltip: 

Zeigt an welches Modul des Adapters ausgeführt wurde. 
Beim EDI werden die einzelnen Nachrichtentypen als Modul aufgeführt. 
Beim AdapterGeneralIO die Export-/Importkonfigurationen 

### Tabellenspalte Aktivitätsparameter 
Tooltip: 

Kommaseparierte Liste der Werte, die zum asynchonen Laden des zu verarbeitenden Datensatzes notwendig sind. 

### Tabellenspalte Ereignis 
Tooltip: 

Gibt den Namen des Ereignis an, welches zur Ausführung des Vorgangs führte. 

### Tabellenspalte EreignisNr. 
Tooltip: 

Zugehörige Nummer zum "Ereignis" 

### Tabellenspalte Schlüssel 
Tooltip: 

Zusammengesetzer Schlüssel der verwendeten Tabellenspalten zum Event 

### Tabellenspalte Adapter-Kontext-Nr. 
Tooltip: 

Zeigt die Nummer des Adapter-Kontext im Gateway, mit welchem das Ereignis ausgeführt wurde. 

### Feld Fehlermeldung 
In diesem Feld wird die Fehlermeldung ausgegeben.
