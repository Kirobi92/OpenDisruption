# Artikel Registerkarte Logistik 
Artikel Registerkarte Logistik 

Auf der Registerkarte Logistik definieren Sie, wie mit dem Artikel in der Logistik zu verfahren ist. 
### Combobox Versandweg 
Hier kann ein Versandweg ausgewählt werden: 

- Zufuhr - Lieferung mittels eigenem Fuhrpark oder Paketdienstleister - Strecke - Lieferung vom Lieferanten zum Kunden - Abholung - Kunde holt die Ware am Lager ab 
Im Standard erfolgt keine Prüfung auf diese Einstellung! 

Die Prüfung auf Zulässigkeit steht nur über die erweiterten Abschlussprüfungen im WS-Package zur Verfügung. 

Tooltip: 

Angabe des Versandwegs: 
Zufuhr - Lieferung mittels eigenem Fuhrpark oder Paketdienstleister 
Strecke - Lieferung vom Lieferanten zum Kunden 
Abholung - Kunde holt die Ware am Lager ab 

### Checkbox Keine Tourenplanung 
Wenn Sie diese Checkbox aktivieren, ist der Artikel von der Tourenplanung ausgeschlossen; z.B. eine Dienstleistung. 

Tooltip: 

Für diesen Artikel wird keine Tourenplanung durchgeführt. 

### Checkbox Kein Fuhrpark 
Wenn Sie diese Checkbox aktivieren, wird der Artikel nicht mit einem LKW beliefert. 

Tooltip: 

Kein Fuhrpark 

### Checkbox Kein Paketdienst 
Wenn Sie diese Checkbox aktivieren, werden bei der Ermittlung der verfügbaren Touren keine Paketdienst-Touren berücksichtigt bzw. es können keine Touren mit diesem Transportweg angelegt werden. Die Eigenschaft „Kein Paketdienst“ kann global in den Versandparametern gesteuert werden sowie im Kundenstamm und im Artikelstamm. Sobald eine der Stammdaten die entsprechende Kennung hat, erfolgt keine Tourenermittlung über diesen Transportweg. 

Beim Kopieren eines Angebots / Auftrags wird diese Einstellung übernommen. 

Tooltip: 

Keine Lieferung per Paketdienst 

### Checkbox Kein Overnight 
Wenn Sie diese Checkbox aktivieren, werden bei der Ermittlung der verfügbaren Touren keine Overnight-Touren berücksichtigt bzw. es können keine Touren mit diesem Transportweg angelegt werden. Die Eigenschaft „Kein Overnight“ kann global in den Versandparametern gesteuert werden sowie im Kundenstamm und im Artikelstamm. Sobald eine der Stammdaten die entsprechende Kennung hat, erfolgt keine Tourenermittlung über diesen Transportweg. 

Tooltip: 

Keine Lieferung per Overnight 

### Checkbox Keine Spedition 
Wenn Sie diese Checkbox aktivieren, werden bei der Ermittlung der verfügbaren Touren keine Speditionstouren berücksichtigt bzw. es können keine Touren mit diesem Transportweg angelegt werden. Die Eigenschaft „Keine Spedition“ kann global in den Versandparametern gesteuert werden sowie im Kundenstamm und im Artikelstamm. Sobald eine der Stammdaten die entsprechende Kennung hat, erfolgt keine Tourenermittlung über diesen Transportweg. 

Tooltip: 

Keine Lieferung per Spedition 

### Checkbox Sperrgut 
Wenn Sie diese Checkbox aktivieren, wird der Artikel als Sperrgut-Artikel gekennzeichnet. Wenn dieser Artikel in einen Auftrag übernommen wird, ist zwingend eine Versandart zu wählen, welche Sperrgüter akzeptiert. 

Hierfür ist im Dialog Versandarten zwingend die Checkbox in der Spalte 'Sperrgut' zu aktivieren. Auf diese Weise können Sie z.B. sicherstellen, dass nicht versucht wird, Sperrgüter über einen Paketdienstleister zu versenden, sondern immer eine Spedition beauftragt werden muss. 

Im Auftrag erscheint bei der Erfassung eine entsprechende Fehlermeldung, falls versucht wird ein Sperrgut mit einer nicht zugelassenen Versandart zu versenden. 

Die Logik für Sperrgüter wird auch für den Webshop genutzt. 

### Checkbox Nicht IC/CC fähig 
Wenn Sie diese Checkbox aktivieren, ist der Artikel nicht für das Intercompany-Geschäft geeignet. 

Beim Auftragsabschluss wird für einen solchen Artikel keine automatische Verbundbestellung an das Lieferhaus erzeugt. 

### Checkbox Ungewollte Lagerware 
Wenn Sie diese Checkbox aktivieren, wird der Artikel als ungewollte Lagerware gekennzeichnet. Die Markierung erfolgt automatisch durch das System, wenn ein Diversartikel bestellt wurde und in das Lager einzulagern war, da eine direkte Durchschleusung an den Kunden (Cross-Docking) nicht vorgenommen werden konnte. In diesem Fall wird aus dem Diversartikel ein realer Artikel mit eigener Artikelnummer und alle Vorgänge zu diesem Diversartikel und Vorgang werden auf die neue Artikelnummer geändert. Alle Vorgänge bleiben dadurch erhalten und der Sachbearbeiter kann gezielt die weiteren Prozesse starten. 

Tooltip: 

Ungewollte Lagerware
