# Positionspflege 
Positionspflege 

Sollen mehrere Positionen eines Auftrags den gleichen Rabatt oder Zuschlag erhalten, kann dies mit dieser Option vereinfacht werden. Dieser Dialog wird im Auftrag über das Menü Optionen >> Positionspflege oder über die Tastenkombination <Strg> + <R> aufgerufen. 
## Steuerelemente des Formulars 

### Bezeichnungsfeld Positionspflege 
Tooltip: 

In diesem Dialog können Sie mehreren Positionen den gleichen Rabatt oder Zuschlag zuweisen. 
Für mehr Informationen drücken Sie F1. 

### Button Form-Menü 

Mit Klick auf diesen Button werden je nach Kontext und Berechtigung die unterschiedlichen Optionen des jeweiligen Dialogs hinter dem Button zur Auswahl angeboten. 

Die Funktionen für das Client Side Customizing sowie die Online-Hilfe stehen hier ebenfalls zur Verfügung.

### Feld Lieferdatum 
Dieses Feld wird beim Öffnen des Dialogs automatisch aus dem zugrundeliegenden Vorgang vorbelegt. Und zwar analog der Funktion zur Ermittlung des Lieferdatums im Auftragskopf, nachdem die Kundennummer im Auftrag eingetragen wurde. 

Entspricht das tatsächlich ermittelte Lieferdatum nicht dem gewünschten Lieferdatum, erhalten Sie eine entsprechende Meldung. 

Dieses Feld ist nicht aktiv, wenn bei der Auftragserfassung die Tourenermittlung aktiv ist und das Lager entsprechend über Touren geplant wird. Hier ist dann zwingend ein neues Datum im Feld Wunschdatum anzugeben. Auf Basis dieses Wunschdatums kann die neue Tourfindung erfolgen! 

Dieses Datum kann im Fall eines Streckengeschäfts, mit erfolgter Bestellung beim Lieferanten, nicht mehr geändert werden! 

Siehe Kapitel Tourendisposition . 

Tooltip: 

Angabe des Liefertermins 

### Feld KW 
Hier wird die Kalenderwoche angegeben. 

Bei Eingabe einer Kalenderwoche wird der Termin automatisch auf den Freitag in dieser Kalenderwoche gesetzt, es sei denn der Parameter Kein Termin aus KW im Dialog Parameter Verkauf ist gesetzt. 

Tooltip: 

Angabe der Kalenderwoche 

### Feld Wunschdatum 
Mit einem Eintrag in dieses Feld haben Sie die Möglichkeit, in Aufträgen mit sehr vielen Positionen das Wunschdatum für diese Positionen zu berücksichtigen. Damit können Sie dies in einem Arbeitsschritt durchführen. Dafür sind die entsprechenden Positionen zu markieren. 

Wenn Sie das Wunschdatum eingetragen und gespeichert haben, wird dieses Wunschdatum jeder einzelnen Position zugeordnet und im Dialog Position angezeigt. 

Siehe Kapitel VK-Positionen bearbeiten . 

### Checkbox Keine Dynamische Verfügbarkeit 
Wenn Sie diese Checkbox aktivieren, wird die Position von der dynamischen Verfügbarkeit ausgeschlossen, sofern diese über die VK-Parameter konfiguriert ist. So werden andere Werte angezeigt, wie z.B. andere Liefertermine und Läger. Sie haben die freie Auswahl. 

Bei der Neuberechnung wird die WBZ hinzugerechnet. 

Tooltip: 

Falls gesetzt, wird diese Position von der 'Dynamischen Verfügbarkeitsprüflogik' ausgeschlossen 
sofern diese über die VK-Parameter konfiguriert ist. 

### Feld Paketpreis 
Mit einem Eintrag in dieses Feld haben Sie die Möglichkeit, einen festen Preis für alle markierten Positionen zu vergeben. Der eingetragene Preis wird anteilig auf die einzelnen Positionen umgerechnet. 

Tooltip: 

Paketpreis, der auf die ausgewählten Positionen verteilt wird. 
Alternativ- oder bereits abgeschlossene Positionen werden nicht berücksichtigt. 

### Combobox Versandweg 
Hier wird der Versandweg angegeben. 

Zufuhr - Lieferung mittels eigenem Fuhrpark oder Paketdienstleister 
Strecke - Lieferung vom Lieferanten zum Kunden 
Abholung - Kunde holt Ware am Lager ab 

Tooltip: 

Angabe des Versandwegs: 
Zufuhr - Lieferung mittels eigenem Fuhrpark oder Paketdienstleister 
Strecke - Lieferung vom Lieferanten zum Kunden 
Abholung - Kunde holt die Ware am Lager ab 

### Combobox Transportweg 
Hier wird der Transportweg ausgewählt. Zur Verfügung stehen: 

0 = Fuhrpark 
1 = Paketdienst (Feld Sender und Versandcode müssen gefüllt sein) 
2 = Overnight Dienst (Feld Sender und Versandcode müssen gefüllt sein) 
3 = Spedition 

Tooltip: 

Transportweg - mögliche Werte 
leer = Automatisch 
0 = Fuhrpark 
1 = Paketdienst (Felder 'Versender' und 'Versand Code' müssen gefüllt sein) 
2 = Overnight Dienst (Feld 'Versender' und 'Versand Code' müssen gefüllt sein) 
3 = Spedition
