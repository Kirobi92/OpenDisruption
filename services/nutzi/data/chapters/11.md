# CSC - Auswählen und Anlegen von Triggern (Trigger Manager) 
Wenn der Trigger Manager geöffnet wird, werden im Bereich Control alle Controls aufgelistet, mit denen bereits ein Trigger definiert wurde. 

Zusätzlich wird immer ein Eintrag mit der Bezeichnung <None> eingefügt. Wenn dieser Wert ausgewählt und der Trigger Manager mit dem Button Ok geschlossen wird, wird dadurch explizit festgelegt, dass nun ungetriggerte Einstellungen (also Einstellungen, die ohne Triggerbedingung gültig sind) definiert und angezeigt werden sollen. 

Wenn beim Öffnen des Trigger Managers in der Client Side Customizing Console ein Control ausgewählt ist, das als Triggercontrol verwendet werden kann (siehe Trigger) und noch nicht verwendet wird, so wird ein Eintrag mit der Bezeichnung <Create: CONTROLNAME> erzeugt, wobei CONTROLNAME der Name des Controls ist. Auf diese Weise kann ein neues Control mit einem Trigger versehen werden. 

Im Trigger Manager muss zunächst in der Listbox Control die gewünschte Auswahl erfolgen. 

Danach muss durch die Selektion einer der Radiobuttons Existing Value und New Value entschieden werden, ob ein neuer Wert definiert werden soll oder ein bereits existierender Wert ausgewählt. 

Abhängig davon kann dann entweder in der Liste Existing value ein bereits existierender Triggerwert ausgewählt werden oder in der Combobox New value der neue Triggerwert eingegeben werden. Je nachdem, welche Werte für das ausgewählte Control gültig sind, können in der Combobox New value keine Vorschläge angezeigt werden oder es wird keine Freitexteingabe erlaubt. 

Mit dem Button Ok bestätigen Sie die Einstellungen, mit dem Button Cancel werden alle Angaben verworfen. In beiden Fällen wird der Dialog geschlossen. 

Hinweis: 
Trigger brauchen und können nicht manuell gelöscht zu werden. Das System entfernt einen Trigger automatisch, wenn keine Einstellung mehr existiert, die für diesen Trigger definiert wurde. Wenn ein Trigger nicht entfernt wird, obwohl Ihnen im aktuellen Scope keine Einstellungen angezeigt werden, so liegt dies daran, dass in einem anderen Scope noch Einstellungen existieren.
