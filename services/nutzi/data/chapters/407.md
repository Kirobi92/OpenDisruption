# Ereignisdefinition Registerkarte Tabelle 
Ereignisdefinition Registerkarte Tabelle 

Auf dieser Registerkarte werden alle Datensätze angezeigt, die den Selektionskriterien entsprechen. 
### Tabelle 

### Tabellenspalte Ereignisname 
Hier wird der Name des Events angegeben. 

Tooltip: 

Bezeichnung des Ereignisses 

### Tabellenspalte Komponente 
Hier wird der Komponententyp angegeben. 

Tooltip: 

Auswahl der Komponente, an der das Event definiert werden kann. 

Diese Komponenten können im Dialog "Event Komponenten" registriert werden. 

### Tabellenspalte Beschreibung 
Hier wird eine genaue Beschreibung angegeben. Sie können bis zu 32000 Zeichen angeben (bei Oracle: 2000). 

Tooltip: 

Beschreibung des Events 
Erläutern Sie, wann das Event gültig ist. 

### Tabellenspalte Aktiv 
Wenn Sie die Checkbox in dieser Spalte setzen, wird das Event aktiviert. 

Tooltip: 

Aktivieren oder deaktivieren Sie das Event 

### Tabellenspalte Eingefügt 
Das Ereignis ist nur gültig, wenn die Checkbox in dieser Spalte aktiviert ist. D.h. die ausgewählte Komponente ist neu. 

Beispiel: Sie erfassen eine neue Auftragsposition. 

Tooltip: 

Das Event ist nur gültig, wenn die ausgewählte Komponente neu ist. 
Beispiel: Sie erfassen eine neue Auftragsposition. 

### Tabellenspalte Geändert 
Das Ereignis ist nur gültig, wenn die Checkbox in dieser Spalte aktiviert ist. D.h. die ausgewählte Komponente geändert wurde. 

Beispiel: Sie ändern eine existierende Auftragsposition. 

Tooltip: 

Das Event ist nur gültig, wenn die ausgewählte Komponente geändert wurde. 
Beispiel: Sie ändern eine existierende Auftragsposition. 

### Tabellenspalte Gelöscht 
Das Ereignis ist nur gültig, wenn die Checkbox in dieser Spalte aktiviert ist. D.h. die ausgewählte Komponente wurde gelöscht. 

Beispiel: Sie löschen eine existierende Auftragsposition (kein Storno!). 

Tooltip: 

Das Event ist nur gültig, wenn die ausgewählte Komponente gelöscht wurde. 
Beispiel: Sie löschen eine existierende Auftragsposition (kein Storno). 

### Tabellenspalte Erfasser 
Hier wird der Sachbearbeiter angegeben, der den Datensatz erfasst hat. 

Tooltip: 

Sachbearbeiterkürzel des Erfassers des Datensatzes. 

### Tabellenspalte Erfassung 
Hier wird vom Programm automatisch das Datum der Erfassung eingetragen. 

Tooltip: 

Erfassungsdatum 

### Tabellenspalte Sachbearbeiter 
Hier wird vom Programm automatisch der Sachbearbeiter eingetragen, der den Datensatz zuletzt geändert hat. 

Tooltip: 

Sachbearbeiterkürzel des letzten Änderers des Datensatzes. 

### Tabellenspalte Änderung 
Hier wird vom Programm automatisch das Datum der letzten Änderung eingetragen. 

Tooltip: 

Datum der letzten Datensatzänderung 

### Feld Anzahl max. 
Über dieses Feld ist in den Suchdialogen die Anzahl der einzulesenden Datensätze limitiert. Die einzelnen Datensätze werden im Dialog entsprechend gesucht und nach Erreichen der maximalen Anzahl Datensätze abgebrochen. Auf diese Weise kann eine Suche über alle Datensätze verhindert werden, welche die gesamte Performance beeinträchtigen könnte. 

Diese Einstellung nehmen Sie im Dialog Parameter Allgemein auf der Registerkarte Allgemein vor. 

Siehe Kapitel Parameter Allgemein Registerkarte Allgemein . 

Diese Einstellung lässt sich in den entsprechenden Suchdialogen (z.B. Artikelsuche, Kundenauswahl) ändern bzw. ausschalten (ausgenommen Stammdatenprogramme). 

Tooltip: 

Anzahl Sätze, die maximal eingelesen werden. 

### Feld Anzahl Sätze 
Dieses Feld zeigt die Anzahl der tatsächlich eingelesenen Datensätze.
