# VK Rabatte/Zuschläge 
VK Rabatte/Zuschläge 

Dieser Dialog öffnet sich u.a. im Artikel >> Stamm >> VK Rabatte / Zuschläge und im Auftrag >> Stamm >> Rabatte / Zuschläge . Hier werden Rabatte/Zuschläge für Artikel angelegt; keine Rabatte für Kunden! 

In diesem Dialog können bis zu 99 verschiedene Rabatte / Zuschläge erfasst werden. Zu jeder Kondition wird ein Text (wie z.B. Bearbeitungsgebühr) angegeben, der auch in den Formularen gedruckt werden kann. Die hier hinterlegten Rabatte oder Zuschläge werden in den nachfolgend aufgeführten Modulen zugeordnet. Diese werden dann in folgenden Vorgängen entsprechend berücksichtigt: 

- Kunde - Kundengruppe - Artikel - Artikelgruppe - Angebot - Auftrag - Gutschrift 
Beispiel: 

VK Rabatte und Zuschläge - Beispiel 

Erklärung: 
Durch Nr. 12 werden 10 % des Angebots- oder Auftragswerts als Zuschlag berechnet, wenn das Gewicht kleiner als der angegebene Wert ist. 
Nr. 11 berechnet einen Aufschlag von 5 % auf den Auftragswert, wenn die Auftragsmenge einen Anbruch einer Verpackungseinheit verursacht. 
Nr. 10 erlaubt einen Auftragsrabatt von 3 %. Dieser wird durch die Checkbox Fragen erst nach manueller Genehmigung durch den Sachbearbeiter abgezogen. 

Es ist nicht möglich an den vier 'Rabatt-/Zuschlags-Zuordnungsstellen' (Kunde, Kundengruppe, Artikel, Artikelgruppe) einen automatischen Rabatt/Zuschlag vom Typ G oder P einem konkreten Objekt zuzuordnen. Diese automatischen Rabatte werden über alle Kunden und Artikel hinweg gezogen, falls die entsprechenden Kriterien zutreffen. Daher ist eine Zuordnung zu Kunden bzw. Artikel nicht notwendig. 

Hinweis: der Typ "R" (für Miete) ist KEIN automatischer Rabatt, sondern lediglich ein 'spezieller Zuordnungsrabatt'. 

Wenn Sie im Dialog VK Rabatte / Zuschläge versuchen, einen Rabatt auf Ebene P oder G zu übernehmen, kommt eine Hinweismeldung. 

Bitte beachten Sie, dass Sie die Daten nur dann ändern können, wenn Sie die Rabatte/Zuschläge aus dem Menü Stamm >> VK Rabatte/Zuschläge gewählt haben. 

Bei Aufruf über den Button R/Z aus anderen Dialogen heraus (z.B. Artikel ) können die Daten nicht geändert werden!

### Tabellenspalte Ebene 
Hier wählen Sie eine Ebene für die Rabattbedingung aus. Zur Verfügung stehen: 

- R = Rabatte / Zuschläge bei Vermietungen - P = Positionsautomatik (Rabatte/Zuschläge auf Positionsebene) 
Rabatte und Zuschläge auf Ebene P werden als Positionsrabatte bzw. -zuschläge ausgewiesen. 
Die Bedingungen werden generell für jede Angebots- und Auftragsposition geprüft und ggf. in den Auftrag übernommen. D.h. unabhängig davon, ob es hier irgendwelche Rabatt-Zuordnungen auf Kunden, Kundengruppen-, Artikel- oder Artikelgruppenebene gibt. Dazu sollte auch die Checkbox in der Spalte 'Fragen' gesetzt werden! 
Siehe auch Informationen zur Spalte 'Fragen'. - G = Gesamtbeleg (Rabatte/Zuschläge auf Kopfebene) 
Rabatte und Zuschläge auf Ebene G werden als Kopfrabatte bzw. -zuschläge ausgewiesen. 
Die Bedingungen werden generell für jede Angebots- und Auftragsposition geprüft und ggf. in den Auftrag übernommen. D.h. unabhängig davon, ob es hier irgendwelche Rabatt-Zuordnungen auf Kunden, Kundengruppen-, Artikel- oder Artikelgruppenebene gibt. Dazu sollte auch die Checkbox in der Spalte 'Fragen' gesetzt werden! 
Das bedeutet, dass nicht die Positionswerte sondern die Gesamtbelegwerte generell gegen die im Rabatt/Zuschlag definierten Grenzwerte verglichen werden. Siehe auch Information zur Spalte 'Fragen'. - <leer> 
Es erfolgt keine automatische Prüfung der Bedingungen. 
Die Rabatte und Zuschläge können auch manuell einer Position oder dem gesamten Auftrag zugeordnet werden. 

Wenn bei dem Artikel oder Kunden ein Rabatt / Zuschlag hinterlegt wurde, wird sie automatisch herangezogen (je nach Bedingung: Wert Menge Gewicht ...). 

Wenn hier ein Wert ausgewählt ist, bedeutet dies, dass der entsprechende Rabatt/Zuschlag auf der eingestellten Ebene automatisch geprüft und ggf. gezogen wird. Ist diese Spalte aber leer, dann kann diese Kondition manuell (auf beliebiger Ebene) geholt werden. 

Wenn keine Bedingung (trotz Zuordnung zum Kunden oder angegebener Ebene) angegeben ist, so wird der Positionsrabatt/-zuschlag immer eingetragen (Ausnahme: Checkbox Fragen ist aktiv). 

Die Prüfung erfolgt immer beim Einfügen bzw. 1. Speichern der betroffenen Auftrags-/Angebotsposition. 

Bei einer Zuordnung auf Artikel, Kunde, etc. erfolgt immer eine Zuordnung in Abhängigkeit der angegebenen Bedingung. Ist keine Bedingung angegeben, so erfolgt die Zuordnung immer. 

Tooltip: 

Automatische Rabattzuordnung: 
G - Rabatte / Zuschläge auf Kopfebene 
P - Rabatte / Zuschläge auf Positionsebene 
R - Rabatte / Zuschläge bei Vermietungen

### Tabellenspalte Anbruch 
Diese Bedingung trifft zu, wenn die Positionsmenge nicht ganzen Verpackungseinheiten entspricht, d.h. eine Verpackungseinheit angebrochen wird. Die Verpackungseinheit des Artikels wird im Artikel festgelegt. Geben Sie hier ein J für Ja ein, wenn eine Verpackungseinheit angebrochen wird und ein N für Nein. 

### Tabellenspalte Fragen 
Wenn die Checkbox in dieser Spalte gesetzt ist, wird die Übernahme der jeweiligen Bedingung in den Auftrag vorgeschlagen (wenn die Kriterien erfüllt sind).
