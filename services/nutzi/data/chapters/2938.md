# Prozessregeln 
Prozessregeln 

Der Dialog Prozessregeln öffnet sich über den Button neben der Combobox Prozessregel im Dialog EDI Teilnehmer . 

In den Prozessregeln wird definiert, wie die importierten Daten vom „Prozessobjekt“ in das „ eNVenta -Datenobjekt“ überführt werden. Hierbei werden Regeln definiert, wie z.B. der Kunde ermittelt wird oder auch wie mit Preisinformationen umgegangen wird. 

Pro EDI-Format sind bestimmte Typen an Regeln auszuwählen und ein möglichst sprechender Name zu vergeben. Dieser Name sollte eindeutig gewählt sein, da diese Prozessregeln pro EDI Teilnehmer gepflegt werden können. 

Die Prozessregeln sind in verschiedene Unter-Regeln aufgeteilt, aber es genügt die oberste Regel pro EDI-Format, z.B. Auftrag erzeugen, auszuwählen und entsprechend zu definieren. 

Pro Gruppierung in der Prozessregel kann über die entsprechende Checkbox die Regel aktiviert werden. Bei einigen Regeln können auch Vorgabewerte ausgewählt werden. Diese werden dann als Optionen rechts vom Baum angezeigt. 

Über die Buttons oberhalb der Baumanzeige kann die Reihenfolge der Regeln verändert werden, z.B. die Ermittlung des Kunden soll über die ILN und erst, wenn nichts gefunden wird, über die Kundennummer erfolgen. 

Für die Ermittlung der Auftragsart können Sie diese direkt beim Mapping hinterlegen und dabei z.B. auch mittels Ausdruckseditor komplexe Regeln hinterlegen. 

Für die Ermittlung der Auftragsart gelten daher folgende Prioritäten: 

- Priorität: Die zu verwendende Belegart wird in den Prozess-Daten gesetzt (z.B. im X-Mapping Editor ). Ist diese dort gesetzt worden, wird der Auftrag / das Angebot mit dieser Belegart angelegt. Falls diese ungültig ist, wird der Vorgang mit einer entsprechenden Fehlermeldung abgebrochen. - Priorität: Die Belegart wird aus dem Kundenstamm verwendet. - Priorität: In dem Prozess-Setting wird die Belegart statisch angegeben. Falls keine Belegart in den Prozess-Daten (X-Mapping) vorhanden ist und auch nicht im Kundenstamm, wird die fix hinterlegte Belegart genommen. 
Der Button Exportieren exportiert die aktuelle Konfiguration im XML-Format. 

Der Button Importieren importiert eine bereits exportierte Konfiguration wieder. 

Folgende Regeltypen (Combobox Typ der Regel ) stehen zur Verfügung: 

- Auftrag erzeugen - Adressprüfung - Ermittlung Artikel - Ermittlung Kunde - Lieferbedingung - Ermittlung Zahlungsbedingung - Auftrag stornieren - Auftrag erzeugen - Erstellung Auftragsposition - Ermittlung Auftrag - Auftrag ändern - Erstellung Wareneingang - Übereinstimmung Auftragsposition - Übereinstimmung Bestellposition - Gutschriftsabrechnung erzeugen - Erstellung Rechnungsprüfung

### Baumansicht Liste mit Regeln 

In diesem Feld werden die Regeln aufgelistet. 

Die Prozessregeln sind in verschiedene Unter-Regeln aufgeteilt, aber es genügt die oberste Regel pro EDI-Format, z.B. Auftrag erzeugen, auszuwählen und entsprechend zu definieren. 

Alle Regeln sind detailliert beschrieben. Lesen Sie dazu das Kapitel Prozessregel-Typen . 

Pro Gruppierung in der Prozessregel kann über die entsprechende Checkbox die Regel aktiviert werden. Bei einigen Regeln können auch Vorgabewerte ausgewählt werden. Diese werden dann als Option rechts vom Baum angezeigt. 

Damit auch externe Referenzen für Kunden und Lieferanten definiert werden können, die keine ILN haben, können Sie hier solch eine Referenz definieren. Hier stehen zur Prozessregel „Auftrag erzeugen“ 3 weitere Optionen zur Verfügung (Feld 1 – Feld 3), die alphanumerisch definiert werden können. 

Durch die entsprechende Konfiguration in den Prozessregeln wird gesteuert, mit welchem Feld im Kunden-/Lieferantenstamm der Wert verglichen wird. Z.B. eines der Code 1-6-Felder. 

Zur Identifizierung einer Lieferanschrift in der Prozessregel „Auftrag erzeugen“ kann die ILN genutzt werden. Falls diese nicht ausreicht, kann über den Parameter Adressübereinstimmung die Lieferanschrift geprüft werden. D.h. jedes einzelne Adressfeld (Firma 1, Ort, Straße etc.) kann für die Relevanz der Übereinstimmung geregelt werden. Dabei werden Leerzeichen zwischen den Wörtern ignoriert, die Wörter alphabetisch sortiert und eine Übereinstimmung bei Wahrscheinlichkeit ab … festgelegt werden. 

Sie haben die Möglichkeit sämtliche Adressen, die von Ihrem Kunden übermittelt werden zu speichern. Die entsprechende Einstellung nehmen Sie in den Prozessregeln unter dem Prozess Auftrag erzeugen vor. So steht beim Import neben dem Auftragskopf, den Auftragspositionen auch die Tabelle Zusatzanschriften zur Verfügung. 

Die Regeln zu dem Prozess 'Auftrag anlegen' werden wie folgt erweitert: 

Die erzeugten Anschriften stehen im Dialog Zus. Anschriften zur Verfügung. Und auch im Auftrag auf der Registerkarte Zusätzliche Anschriften stehen diese Kundenadressen zur Verfügung. Zu jedem Adress-Typ gibt es nur eine Adresse. Daher werden hier nicht mehrere Adresstypen pro Adresse zugelassen, sondern es wird je nachdem, ob es sich um eine Bestell-, Liefer- oder Rechnungsadresse in den Prozessdaten handelte, der entsprechende Typ gesetzt und kein weiterer.
