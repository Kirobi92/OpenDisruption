# Wiederkehrende Buchung anlegen / verwalten 
WdkB Verwalten 

Dieser Dialog öffnet sich über den Button im Dialog WdkB Übersicht . Hier legen Sie eine neue Buchung an, können Änderungen an einer Buchung vornehmen oder eine Buchung löschen.

Tooltip: 

In diesem Dialog legen Sie eine neue Buchung an, können Änderungen an einer Buchung vornehmen oder eine Buchung löschen.

### Combobox Gruppe 
Auswahl einer Gruppe, der die Buchung zugeordnet werden kann. (Kein Muss-Feld) 

Nur bei Neuanlage einer WdkB werden die Einstellungen ( Typ und Intervall ) der gewählten Gruppe übernommen (Vorschlag). Die übernommenen Werte können überschrieben werden. 

Bestehende WdkB können einer Gruppe zugeordnet werden. Die Einstellungen Typ und Intervall werden jetzt nicht mehr aus der Gruppe übernommen. Die Zuordnung zu einer Gruppe dient der Selektion in anderen Dialogen. Das Feld Gruppe ist ein Selektionskriterium im Dialog WdkB Übersicht und Wdk. Buch. (Buchen der WdkB-Vorschläge). 

Tooltip: 

Angabe der Gruppennummer

### Combobox Minus 
Dieses Feld wird eingeblendet, wenn im darüberliegenden Feld Betragsart Saldo aktuelle Periode gewählt ist. 

Das Feld ist vorbelegt mit der Auswahl 0 Perioden . Vom Buchungsdatum ausgehend wird die umzubuchende Periode ermittelt. 

Beispiel: 

Buchungsdatum 02.05.2022 

=> Saldo 31.01.2022 wird umgebucht. 
0 Periode = Mai 
1 Periode = April 
2 Periode = März 
3 Periode = Februar 
4 Periode = Januar

### Feld Intervall 
Anzahl, wie oft die Buchung durchgeführt werden soll, z.B. Typ – Monatlich: Intervall – 1, d.h. die Buchung wird jeden Monat 1x zur Ausführung vorgeschlagen. 

Standardmäßig werden im Typ und im Intervall die Daten aus der zugeordneten Gruppe übernommen. Diese können jedoch für jede Buchung individuell angelegt werden. Die individuelle Anlage hat eine höhere Priorität als die Gruppe. 

Tooltip: 

Angabe des Intervalls

### Feld Belegnummer 2 
Das Feld Belegnummer 2 und die Checkbox Nummerierung werden in Abhängigkeit des Fibu Parameters Belegnummer 2 für Debitoren bzw. Belegnummer 2 für Kreditoren bei den jeweiligen Buchungskreisen eingeblendet und beim WdkB Buchen verwendet. 

In Verbindung mit dem Fibu-Parameter Zahlungsverkehr mit Belegnr 2 erhält damit die Überweisung, die über den Dialog Zahlungsverkehr einen OP aus den WdkB ausgleicht, im Verwendungszweck die Belegnummer 2. 

Tooltip: 

Angabe der Belegnummer 2

In Verbindung mit dem Fibu-Parameter Zahlungsverkehr mit Belegnr 2 erhält damit die Überweisung, die über den Dialog Zahlungsverkehr einen OP aus den WdkB ausgleicht, im Verwendungszweck die Belegnummer 2.

### Feld Barcode 
Hier können Sie einen Barcode angeben, um einen Beleg mit wiederkehrenden Buchungen zu scannen und anschließend zu archivieren. Der entsprechende archivierte Beleg kann dann über das Journal aufgerufen werden. 

Beispiel: 
Am Jahresanfang wird eine Versicherungsrechnung über € 1200,00 (Buchungskreis KR) verbucht und zu dieser Buchung die Versicherungspolice mit einem Barcode versehen, eingescannt und ins Archiv übernommen. 

Aus betriebswirtschaftlichen Gründen werden diese € 1200,00 monatlich per wiederkehrender Buchung (Buchungskreis FB) von einem Sachkonto auf ein anderes gebucht. So kann aus dem Journal heraus, bei dieser Buchung, die Versicherungspolice im Archiv gefunden werden. 

### Feld Projekt 
Hier geben Sie die Projektnummer ein, falls diese wiederkehrende Buchung sich auf ein Projekt bezieht und in der wiederkehrenden Buchung berücksichtigt werden soll. 

Die Voreinstellung dazu nehmen Sie im Dialog Parameter Fibu auf der Registerkarte Allgemein vor (Checkbox Buchen mit Projektnummer ). 

Wenn Sie diese Checkbox nachträglich deaktivieren, werden bereits hinterlegte Projektnummern beim Buchen nicht mehr berücksichtigt! 

Siehe Kapitel Parameter Finanzbuchhaltung . 

Tooltip: 

Tragen Sie eine eindeutige Projektnummer ein.
