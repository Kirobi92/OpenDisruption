# Besuchskalender Registerkarte Kalender 
Besuchskalender Registerkarte Kalender 

Auf dieser Registerkarte werden alle Termine im Kalender dargestellt. Es werden der eigene sowie die freigegebenen Kalender anderer Benutzer angezeigt, wenn diese unterhalb der Monatsübersicht aktiviert sind. So können Termine mehrerer Benutzer verglichen werden. Die Benutzer werden in unterschiedlichen Farben angezeigt. 

Die Anzeige der Termine hängt von der Berechtigung ab. 

Auf der linken Seite sehen Sie die Kalenderansicht. Hier können Sie durch markieren einzelner Tage oder Wochen individuell den Zeitraum einstellen, den Sie betrachten möchten. Rechts von den Kalendertagen werden die Kalenderwochen in senkrechten Balken angezeigt. Wenn Sie auf eine Kalenderwoche klicken, wird Ihnen diese KW auf der rechten Seite angezeigt. 

Mit den Buttons unterhalb der Kalenderansicht können Sie einen Monat rückwärts blättern, vorwärts blättern und wenn Sie auf den Stern (*) klicken, wird der aktuelle Monat angezeigt und das Heute-Datum hervorgehoben. 

Ein Termin kann durch Ziehen mit der Maus bzw. mit dem Button neu erstellt werden. Per Drag & Drop können Termine verschoben, verlängert oder verkürzt werden. 

Das Verschieben eines Termins von einem Vertreter auf einen anderen Vertreter wird nicht unterstützt. 

Ein Termin im Kalender kann mit Doppelklick aufgerufen werden. Es öffnet sich der Dialog Besuchstermin . 

Eine Projektaufgabe im Kalender kann mit Doppelklick aufgerufen werden. Es öffnet sich der Dialog Aufgabe ändern . 

Siehe Kapitel Besuchstermin-Verwaltung . 

Siehe Kapitel Aufgabe ändern . 

Die Registerkarten Lieferanten und Adressen werden bei einer ersten Aktivierung einmalig geladen und stehen dann die gesamte Zeit zur Verfügung (solange der Besuchskalender geöffnet ist). 

Hier erfolgt keine Vorbelegung eines Vertreters, jedoch wird durch den selektierten Vertreter bzw. dem angemeldeten Benutzer (falls mehrere Vertreter ausgewählt sind) versucht einen Besuchsplan zu laden. 

Ohne Besuchsplan können Termine nicht erstellt werden, dementsprechend werden die Listen nicht gefüllt. 

Terminerstellung per Drag + Drop 

Ein markierter Kunde in der Vorschlagsliste in der neuen Tabelle im unteren Teil der Registerkarte Kalender kann per Drag + Drop in den Kalender gezogen werden, woraufhin dort ein Termin erstellt wird. 

Folgende Felder des Termins werden automatisch gefüllt: 

- AKL-Nummer: Kundennummer des markierten Kunden in der Tabelle - Besuchstag: Stelle, auf die im Kalender per Drag + Drop „geschoben“ wird. - Zeit: Anfangsuhrzeit ist die Zeit, auf die im Kalender per Drag + Drop „geschoben“ wird. - Dauer: die Dauer des Besuchstermins wird über den bestehenden Vertriebsparameter gesteuert ( System >> Vertrieb >> Parameter >>Registerkarte Allgemein >> Feld Besuchsdauer ) - Zeit bis: wird berechnet aus Zeit (Anfangszeit) + Dauer. Beispiel: Zeit = 10:00 Uhr, Dauer = 2 Stunden, Zeit bis = 12:00 
Diese Informationen werden im Dialog Besuchstermin angezeigt. 

Wenn mehrere Kunden in der Tabelle markiert werden, dann terminieren sich die anderen Kunden nahtlos hintereinander weg: 

- Startzeit zweiter Kunde = „Zeit bis“ vom ersten Kunden - Dauer aus dem Vertriebsparameter - „Zeit bis“ = Startzeit + Dauer - Startzeit dritter Kunde = „Zeit bis“ zweiter Kunde - Etc. 
Die Markierreihenfolge bestimmt die Folge, in der die Kunden bei einer Mehrfachauswahl terminiert werden. 

Wenn ein Termin manuell im Kalender erstellt wird, wird bei der AKL-Auswahl der Vertreter als Selektionskriterium angezeigt (Feld Vertreter ) und schon mit dem eingetragenen Vertreter vorbelegt. 

Status der Besuchstermine 

Hier wird auch der Status der Besuchstermine direkt angezeigt. Siehe Markierung im Screenshot. 

Am Anfang eines Termins wird der Status direkt aus dem Besuchsbericht übernommen und angezeigt. 

Termine, zu denen es noch keine Besuchsberichte gibt, werden am Beginn mit einem Minus-Zeichen gekennzeichnet. 

Die Anzeige des Status Besuch und der Kontaktperson wird durch // getrennt. 
### Checkbox Alle Termine 
Wenn Sie diese Checkbox deaktivieren, wird nur der Prozesstyp berücksichtigt, der auch erzeugt werden kann. Hier: nur Besuchsberichte in der Besuchsplanung. 

### Checkbox 5-Tage Woche 
Wenn Sie diese Checkbox aktivieren, hat die Woche 5 Tage. Samstag und Sonntag werden nicht berücksichtigt. 

### Checkbox Neutraler Termin 
Wenn Sie diese Checkbox aktivieren, kann auch ein manueller, kundenunabhängiger Termin, z.B. für Urlaub eingefügt werden. Ein neutraler Termin kann nur durch Ziehen mit der Maus im Kalender erzeugt werden; nicht über den Button ! 

Beim Generieren eines neutralen Termins erscheint ein Dialogfeld zur Erfassung der Terminbezeichnung. Hier kann auch ein vorhandener Termin gelöscht werden. 

### Button Satz hinzufügen 

Mit diesem Button kann für den selektierten Benutzer ein neuer Besuchstermin des vorgegebenen Typs (z.B. Besuchstermin aus Instanz Besuchsplanung) erstellt werden. Es öffnet sich der Dialog Besuchstermin . 

Siehe Kapitel Besuchstermin-Verwaltung . 

### Button Aktualisieren 

Mit diesem Button werden Termine aktualisiert.
