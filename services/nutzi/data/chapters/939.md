# Aufgabe ändern 
Aufgabe ändern 

Der Dialog Aufgabe ändern öffnet sich im Projekt auf der Registerkarte Phasen mit einem Doppelklick auf eine Aufgabe. Die Beschreibung der Aufgabe wird aus der Registerkarte Phasen übernommen. 

Eine neue Aufgabe kann angelegt werden, wenn der Eintrag „Aufgaben“ einer Phase ( Projekt Registerkarte Phase ) markiert ist und Sie den Button Neu drücken. Mit dem Button Neu öffnet sich der Dialog Aufgabe auswählen , wo Sie die passende Aufgabe übernehmen. Siehe nachfolgendes Beispiel: 

Neue Aufgabe anlegen - Beispiel 

Aus dem Dialog Aufgabe auswählen selektieren Sie die benötigte Aufgabe. Diese kann entsprechend Ihrer Bedürfnisse geändert werden. 

Bei einer zugewiesenen Aufgabe kann über die Buttons In Bearb. bzw. In QS der Status der Aufgabe entsprechend verändert werden. Der Status ist u.a. sichtbar in der Aufgaben-Übersicht und in den Projektphasen . 

Wenn Änderungen an einer Projektaufgabe durchgeführt werden, erfolgt das Löschen und Neu-Anlegen der Outlook Termine nur für relevante Änderungen. Dazu prüft eNVenta folgende Felder: 

- Status Neu - Datum Von - Datum Bis - Aufgabenname - Beschreibung - Projektnummer - Verantwortlicher - Keine Outlook Sync - Terminart (auf der Registerkarte Termine ) - Termintyp (auf der Registerkarte Termine ) - Keine Erinnerung 
Bei Terminen: 
- Status Neu - Datum - Zeit von - Zeit bis - Beschreibung - Ressource - Fahrzeug - Aufgabenname - Keine Outlook Sync - Terminart (auf der Registerkarte Termine ) - Termintyp (auf der Registerkarte Termine ) - Keine Erinnerung 
und synchronisiert lediglich die geänderten Werte (Reservierungen) mit Outlook. 

Eine Änderung in eNVenta ERP übersteuert die Änderung in Outlook. 

Die Einstellung der Projektaufgabe hat stets Priorität gegenüber der Einstellung aus dem Aufgabenstamm. 

Werden auf einer Projektaufgabe Termine gesetzt, werden nur für diese Reservierungen vorgenommen. Sind keine Termine gesetzt, so wird für den Verantwortlichen eine Reservierung vorgenommen, wobei als Start- und Endzeit das Datum/Zeit von bzw. Datum/Zeit bis der Aufgabe verwendet wird.

### Button Speichern 

Mit diesem Button speichern Sie die Einstellungen. Dabei wird eine Meldung angezeigt, falls eine zugeordnete Ressource außerhalb der normalen Arbeitszeit liegt (bezogen auf den Betriebskalender). 

### Button Vorheriger Satz 

Mit diesem Button kann über Aufgaben rückwärts geblättert werden. Hierbei kann durch alle Aufgaben aus dem aktuellen Projekt geblättert werden. 

Wird die Aufgabe aus der Aufgaben-Übersicht heraus geöffnet, kann durch alle Aufgaben der Tabelle in diesem Dialog geblättert werden. 

### Button Nächster Satz 

Mit diesem Button kann über Aufgaben vorwärts geblättert werden. Hierbei kann durch alle Aufgaben aus dem aktuellen Projekt geblättert werden. 

Wird die Aufgabe aus der Aufgaben-Übersicht heraus geöffnet, kann durch alle Aufgaben der Tabelle in diesem Dialog geblättert werden. 

### Button Schließen 

Mit diesem Button schließen Sie den Dialog. Bitte denken Sie daran, Ihre Eingaben / Änderungen vorher zu speichern! 

### Button Öffnen 

Der Button Öffnen ist nur aktiv, wenn die Aufgabe den Status beendet hat. Mit diesem Button kann eine Aufgabe für die Bearbeitung wieder geöffnet werden. 

### Button In Bearbeitung 

Über diesen Button kann der Status der Aufgabe auf In Bearbeitung geändert werden. 

### Button In QS 

Über diesen Button kann der Status der Aufgabe auf In QS geändert werden. 

Die Sichtbarkeit des Buttons hängt vom Aufgabentyp ab. 

### Button Beenden 

Dieser Button beendet eine Aufgabe und ist nur aktiv, solange die Aufgabe nicht im Status beendet ist. 

### Button Leistungsabr. 

Mit diesem Button rufen Sie den Dialog Leistungsabrechnung auf. Die Daten zur Projektaufgabe werden eingelesen, Phase und Projektaufgabe werden als Referenz übergeben. Die aktuelle Projektphase und die Aufgabe werden automatisch markiert und alle Leistungen zur Aufgabe werden in die Tabelle eingelesen.

### Button Archiv einfügen 

Mit diesem Button können Sie eine Archiv-Mappe zur Projektaufgabe einfügen. D.h. es öffnet sich der bekannte Dialog Archiv-Mappe einfügen , um ein Dokument, Bild etc. einem Vorgang hinzuzufügen. Die Funktion Drag&Drop steht dafür ebenfalls zur Verfügung. 

Die Archivierung zu einer Projektaufgabe ermöglicht eine feinere Steuerung der Archivierung von Dokumenten zu den relevanten Vorgängen anstatt der Archivierung über das gesamte Projekt. 

Wird eine Projektaufgabe verschoben und für die ursprüngliche Aufgabe gibt es ein Archiv-Dokument, wird zur neuen Aufgabe ein neues Archiv-Dokument erzeugt. 

Wird eine Projektaufgabe kopiert und verschoben, wird das Archiv-Dokument nicht kopiert! 

Siehe Kapitel Archiv-Mappe einfügen .
