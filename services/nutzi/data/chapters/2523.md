# Liquiditätsübersicht 
Liquiditätsübersicht 

Diesen Dialog finden Sie im Buchen >> Auswertungen >> Liquiditätsübersicht . Dieses Programm ist ein einfaches aber sehr effektives Arbeitstool, um den Bestand und die Entwicklung der Liquidität aufgrund vorhandener Informationen zu simulieren und zu planen. Als Datenquelle für die Liquiditätsentwicklung dienen die offenen Posten der Debitoren und Kreditoren, die Bestände der Geldkonten und die individuellen Planwerte. 

Der Drilldown auf ein Geldkonto öffnet den Dialog Offene Posten Sachkonto oder Journal . 

Die Übersicht kann beliebig oft gestartet werden und wird stets anhand des aktuellen Datenbestands angepasst. Darüber hinaus besteht stets die Möglichkeit, durch manuelle Änderungen (z.B. Änderungen der Zahlungsziele im OP, als nachhaltige Änderung, oder bloße temporäre Eingaben in den Datenfeldern) die Simulation durch Wenn-Annahmen anzupassen und zu testen. Als weitere Funktionalität steht auch hier die bekannte Drilldown-Funktion zur Verfügung. 

Hiermit wird die Möglichkeit geschaffen, frühzeitig Liquiditätsengpässe zu erkennen, um rechtzeitig Gegenmaßnahmen zu simulieren und letztendlich ergreifen zu können. 

Die Grundstruktur der Berechnung sieht wie folgt aus: 

Alter Geldbestand 
+ zu erwartende Einnahmen von Debitoren (aus OP-Debitoren) 
- zu leistende Zahlungen an Kreditoren (aus OP-Kreditoren) 
+/- Planwerte (individuell bestimmbar) 
+/- Bestand Geldkonten (Konten mit aktivierter Checkbox Geldkonto) 
= neuer Geldbestand 

Die Berechnung kann durch die Einstellungen im Kopf des Dialogs beeinflusst werden.

### Feld Erster Stichtag 
Dieses Feld ist frei wählbar, um auch Daten, die weit in der Zukunft liegen, anzeigen zu können. Infolgedessen werden die Tage im Bereich Bewegungen beim Typ Quartal anders ermittelt. Es wird von Jahresbeginn der Tag ermittelt und dann immer nur die zum Betrachtungszeitraum passenden Monate hinzugerechnet. Beispiel: Tag 80, Typ Quartal = 80. Tag des Jahres = 21.03. Die folgenden Fälligkeiten (der 21. Tag wird beibehalten) sind dann immer + 3 Monate (1 Quartal) = 21.06. / 21.09. / 21.12.

### Combobox Code 
Hier steht eine Gruppierungs- oder eine Selektionsfunktion zur Verfügung. Beide Funktionen beziehen sich auf die Code-Felder (Code1 –Code3) im Debitorenstamm. 

Gruppierungsfunktion: Bei Auswahl der Zahlen 1-3 (also für Code -Feld 1, 2, oder 3) werden nach drücken des Buttons nicht die einzelnen Debitoren angezeigt, sondern die Gruppensummen lt. Gruppendefinition im entsprechenden Code-Feld. 

Die danebenliegende Combobox muss dabei leer sein!!! 

Selektionsfunktion: Nach Auswahl des Codes (also ob sich die Selektion auf das Code -Feld 1, 2 oder 3 beziehen soll) kann in der danebenliegenden Combobox ein Stichwort selektiert werden. Nach drücken des Buttons werden nur die Debitoren mit dem entsprechenden Eintrag im Code -Feld angezeigt.
