# Parameter PService 
Parameter PService 

Dieser Dialog öffnet sich in der Kommissionierung >> Optionen >> PService Schnittstelle >> PService Parameter . Hier wird Ihnen eine Paketdienstschnittstelle zur Verfügung gestellt. 

Die Übergabe per CSV lässt keine mehrfachen Drucker bzw. Arbeitsplätze zu! Mehrere Arbeitsplätze werden nur mit dem Webservice unterstützt! 

Im Versand kann eine Webservice-Schnittstelle für den Druck ausgewählt werden. Wenn Sie hier den V-LOG Webservice ausgewählt haben, wird der Button VS-Drucker aktiv. Dieser Button öffnet den Dialog Versanddrucker V-LOG . 

Siehe Kapitel Versanddrucker V-LOG . 

Der Datenaustausch zwischen eNVenta und V-LOG geschieht grundsätzlich MHP V-Log Webservice. 

Dabei werden stets zwei Schritte durchgeführt: 

=> erstens werden die Versanddaten in die Schnittstellentabelle geschrieben und 
=> zweitens werden diese Daten dann in die Exportdatei (CSV) geschrieben. 

Bei Import der Rückmeldedaten von V-LOG werden die beiden vorgenannten Schritte in der umgekehrten Reihenfolge durchlaufen, also: 

=> erstens die Rückmeldedatei wird ausgelesen und 
=> zweitens diese Daten werden in die Schnittstellentabelle geschrieben 
=> (und drittens, die übernommenen Rückmeldedaten werden in die entsprechenden eNVenta Business Tabellen eingepflegt). 

Im Einzelnen gestaltet sich der Vorgang wie folgt: 

- Bei der Fakturafreigabe aus der Kommissionierung oder der Logistik werden - sofern zu der eingestellten Versandart das Attribut 'UebergVersender' gesetzt ist - die entsprechenden Datensätze zunächst in die Schnittstellentabelle 'Versender' geschrieben. - Wenn der entsprechende Vorgang über Lieferschein als 'höherwertigem Beleg' kommissioniert wurde, ist der LS der logische Träger des V-LOG-Versendungsvorganges und kommt daher mit dem Suffix 'L' ins Feld 'Versender.RefBelegnr'. Andernfalls geschieht dies mit der Auftragsnummer und dem Suffix 'A'. - Anschließend werden die Datensätze in die Text-Datei ins 'Übergabeverzeichnis' gestellt. Ist diese Datei dort bereits vorhanden, dann werden die neuen Datensätze angehängt; andernfalls wird die Datei neu generiert. ==> Wenn die Funktion „Auto Rückmeldung“ AKTIV ist, wird nun überprüft, ob noch etwa eine 'alte' Rückmeldedatei ('Melde.csv') im Übergabeverzeichnis steht. Diese wird dann verarbeitet und gelöscht, bevor die neue Datei generiert wird (Alternative zur Auto Rückmeldung wäre der Button PSRÜCK ). - Die von V-LOG zurückgemeldeten Daten werden von diesem Programm automatisch in die Rückmeldedatei abgelegt. Dabei wird pro gedrucktem Paketaufkleber jeweils ein separater Datensatz in diese Datei weggeschrieben. - Nach erfolgreichem Auslesen der Datei wird jeder darin enthaltene P-Satz mit seinen entsprechenden Daten an der richtigen Stelle in die Tabelle Versender eingepflegt. 'Richtige Stelle' heißt hier, dass der P-Satz mit der übereinstimmenden Versandgutnummer, der ja bereits beim Übergeben der Daten an die VS-Schnittstelle in der Versendertabelle angelegt wurde, gesucht wird und per Update um die zurückgemeldeten Daten ergänzt wird. Ist dies geschehen, dann wird das Attribut 'Verarbeitet' auf 1 gesetzt. - Als letzten Schritt werden nun die relevanten Rückmeldedaten wie Paketnummer, Gewicht und Verpackungsart aus der Tabelle Versender ausgelesen und in die Tabelle Versandgut eingepflegt. - Die Verzeichnisse für die beiden vorgenannten 'Transferdateien' können per Dialog Parameter VS / LO jeweils separat voreingestellt werden. Außerdem müssen dieselben Verzeichnisse (für beide Dateien) auch im PSS eingestellt werden. Weiterhin ist darauf zu achten, dass sich die Verzeichnisse auf dem lokalen Rechner (Client) befinden, so dass im Netzwerkbetrieb mit mehreren Clients keine Probleme auftreten. 
Weitere Hinweise: 
1. Der Dateiname kann nicht geändert werden. 
2. Der Pfad kann geändert werden.

### Feld V-LOG Mandant 
Hier geben Sie die eindeutige ClientID für die Schnittstelle zum Paketdienstleister an. Wenn nichts anderes vereinbart ist, kann die ID 701 verwendet werden. 

Tooltip: 

Angabe der eindeutigen ClientID für die Schnittstelle zum Paketdienstleister. 

### Combobox Fremdprogramm 
Hier geben Sie an, welche Schnittstelle für Paketdienstleister genutzt wird. Es stehen nur die Einträge leer und MHP V-LOG Webservice zur Verfügung. 

Von eNVenta ERP wird im Moment „MHP V-LOG“ als Schnittstelle unterstützt. 

Bei Angabe des "MHP V-LOG Webservice" muss die entsprechende Service-Adresse beim Publish von eNVenta ERP angegeben sein, damit die Verbindung hergestellt werden kann. Bei Angabe von "MHP V-LOG Webservice" müssen die entsprechenden Melde-Pfade in diesem Dialog gepflegt werden. 

Achtung: Die Angabe des Fremdprogramms muss mit dem Schnittstellen-Anbieter besprochen sein. 

Tooltip: 

Angabe der Schnittstelle, welche für Paketdienstleister genutzt wird. Entweder CSV-Schnittstelle oder Webservice für die Kommunikation von V-LOG. 

### Feld Hin-Pfad 
Hier geben Sie den Pfad für die Hinmelde-Datei ein, d.h. den Pfad zur Abspeicherung der Paketdaten für die Übergabe nach V-Log. 

Die Datei wird beim Export der Daten, also Übergabe an Versender bei Fakturafreigabe, generiert. Der Dateiname ist hierbei immer VLOGHIN_[User].csv. 

Tooltip: 

Hinmeldepfad für die Übermittlung der Daten an V-LOG

### Feld Timeout PService 
Diese Angabe ist nur bei "MHP V-LOG Webservice" zu pflegen. Diese gibt an, wie lange eNVenta ERP auf die Rückantwort der V-LOG-Software antworten soll. 

Tooltip: 

Maximale Zeit in Sekunden in der auf die Rückmeldedaten

(Default = 20 sec) 

### Checkbox Auto-Rückmeldung V-LOG 
Bei Nutzung des "MHP V-LOG Webservice" wird automatisch versucht innerhalb des Timeouts die Rückmeldung einzulesen. Hierzu ist der sogenannte "BlockBox-Betrieb" oder "Pooling-Betrieb" der V-LOG-Software notwendig. Für die Einrichtung der V-LOG-Software wenden Sie sich bitte an MHP bzw. sprechen Sie Ihren eNVenta -Lieferanten an, um den Kontakt herzustellen. 

### Feld Inhaltsbeschr. 
In diesem Feld geben Sie an, was in den Paketen verpackt ist. Dies wird bei einer Auslandssendung für die Zollinhaltserklärung übernommen. Da es keine Zuordnung der Artikel zum Paket gibt, wird ersatzweise immer diese Inhaltsbeschreibung an den Paketdienst übermittelt. 

Tooltip: 

Inhaltsbeschreibung für Auslandssendung 

### Field Panel X-Mapper 

Aufgrund der immer wieder hinzukommenden Anforderungen an eine Flexibilisierung der Schnittstelle zu V-LOG steht zusätzlich zur bestehenden Übertragung die Nutzungsmöglichkeit der Schnittstellendefinition mittels X-Mapper zur Verfügung. D.h. Datenimport und –export können über den X-Mapper abgebildet werden. Dafür stehen entsprechende Datenobjekte zur Verfügung. 

Über die Comboboxen 

- Export - Status - Storno 
können die X-Mapping-Settings konfiguriert werden. 

Folgende Settings können konfiguriert werden: 

- Export-Setting: Dies wird bei der Übertragung der Packstücke verwendet und bei der Abfrage der Rückinformationen. Wird bei einer Schnittstelle ein abweichendes Abfragetelegram benötigt, so müssen wir dies implementieren. - Import-Setting: Dies wird bei der Abfrage der Rückinformationen verwendet zum Mappen der zurückgegebenen Informationen in das Prozessdatenobjekt. - Storno-Setting: Dieses Setting wird verwendet um die Exportdaten im Stornofall zusammenzustellen 
Für die Nutzung dieser Übertragungslogik ist zwingend die Checkbox X-Mapper PService Logik zu aktivieren. Ist die Checkbox nicht aktiviert, erfolgt die Übermittlung der Daten weiterhin im fix definierten Format und ohne Einflussmöglichkeit. 
### Checkbox X-Mapper PService Logik 
Für die Nutzung der Übertragungslogik zum X-mapper, ist diese Checkbox zwingend aktivieren. Ist die Checkbox nicht aktiviert, erfolgt die Übermittlung der Daten weiterhin im fix definierten Format und Einflussmöglichkeit. 

Tooltip: 

Verwendung von X-Mapping zur Konfiguration der Schnittstelle
