# Anfrage Registerkarte Positionen 
Anfrage Registerkarte Positionen 

Auf dieser Registerkarte geben Sie die Artikel für eine Anfrage an den Lieferanten ein. 

Es gibt mehrere Möglichkeiten, eine Anfrageposition zu erfassen: 

- Eine Anfrageposition kann jederzeit mit dem Button Neu auf der Registerkarte Positionen erfasst werden. Das Programm fügt in die Positionstabelle eine neue Zeile ein. Durch Betätigen der Funktionstaste F7 wird ebenfalls eine neue Zeile in die Positionstabelle eingefügt. Der Cursor steht in der Spalte ‚Artikelnr’ zur Eingabe des gewünschten Artikels bereit. - Steht der Cursor in der letzten Spalte einer bereits erfassten Position, kann durch Weiterschalten mit Tab sofort eine neue Zeile eingefügt und mit der Erfassung der nächsten Artikelposition begonnen werden. Eine schnelle und reibungslose Anfrageerfassung wird dadurch wesentlich unterstützt. 
Legen Sie vor der Eingabe der Anfragepositionen fest, auf welches Lager die angefragte Ware gebucht werden soll. Dazu steht die Combobox Lagernr . zur Verfügung. Die entsprechende Lagernummer wird automatisch in jede neue Position eingetragen. 

Die Registerkarten am unteren Rand der Registerkarte Positionen geben Ihnen detaillierte Auskunft zum Artikel, zu Anfragen, zum Artikeltext und den Meldungen. 

Artikelsuche mit der Artikelnummer : 

- Im Artikelstamm wird nach einem Artikel mit der angegebenen Nummer gesucht. - Als nächstes wird ermittelt, ob es sich um eine EAN aus dem Artikel handelt. - Dann wird ermittelt, ob es sich um eine Artikelnummer des Lieferanten handelt (Bestellnummer in der Rabattverwaltung). - Falls noch kein zutreffender Artikel gefunden ist, wird eine Liste aller Artikel angezeigt, auf die der angegebene Artikelnummernteil zutrifft. - Gibt es nur einen zutreffenden Artikel, werden dessen Daten sofort in die Tabellenzeile eingespielt. Andernfalls ist eine Auswahl aus der angezeigten Liste notwendig. Dies geschieht wie bei der Kundenauswahl mit Doppelklick auf die gewünschte Zeile oder mit dem Button . 
Artikelsuche mit der Bezeichnung in den AnfragenIn der Spalte ‚Bezeichnung’ kann alternativ zur Artikelnummer ein Suchbegriff angegeben werden. eNVenta ERP ermöglicht eine Volltextsuche in der Artikelbezeichnung. 

Beispiele: 
Schraube: 
Es erscheinen alle Artikel, deren Bezeichnung mit Schraube beginnt. 

%schraub%: 
Volltextsuche - alle Artikel, deren Bezeichnung die Zeichenfolge ‚schraub’ enthält, werden ausgewählt. Dieser Suchmodus ist etwas langsamer, da der gesamte Artikelbestand durchsucht werden muss. 

Schraube%15mm: 
Hier werden alle Artikel ausgewählt, deren Bezeichnung mit Schraube beginnt, und die zusätzlich die Zeichenfolge 15mm enthalten. 

Groß- und Kleinschreibung wird nicht unterschieden. 

Wird der Vorgang abgebrochen, werden keine Positionen eingefügt. Auf diese Weise kann verhindert werden, dass durch eine Falscheingabe eine große Menge an Positionen erzeugt wird. 

Bei der Auswahl von Artikeln im Artikelsuchdialog können auch mehrere Artikel markiert werden. Für jeden gewählten Artikel wird dann eine Anfrageposition generiert. 

Artikelsuche mit weiteren Auswahlkriterien in den AnfragenGeben Sie in das Feld Bezeichnung ein Leerzeichen oder # ein und schalten Sie dann mit Enter oder Tab zum nächsten Feld weiter. Es erscheint ein Artikelsuchdialog mit weiteren Auswahlkriterien, wie Artikelgruppe und Code-Felder. Die Suchfelder können auch in Kombination miteinander verwendet werden. Der Button löst die Suche aus. 

Siehe Kapitel Allgemeine Funktionalitäten in eNVenta ERP / Artikelsuche . 

Positionsschnellerfassung in den AnfragenNachdem der Artikel eingespielt wurde, steht der Cursor in der Spalte ‚Menge’ zur Erfassung der angefragten Menge bereit. Schalten Sie nach der Eingabe zum nächsten Feld mit Enter oder Tab . 

Steht der Cursor in der letzten Spalte der Tabelle (Rabatt), kann durch Weiterschalten mit Tab sofort eine neue Zeile eingefügt und mit der Erfassung der nächsten Position begonnen werden. Eine schnelle und reibungslose Erfassung der Anfrage wird dadurch wesentlich unterstützt. 

Sind alle Positionen erfasst, ist es notwendig die gesamte Anfrage in die Datenbank einzufügen. Dazu drücken Sie bitte den Button . 

Jetzt werden die Daten auf Richtigkeit überprüft und in der Datenbank gespeichert. 

Erst danach können Formulare gedruckt oder eine Gesamtkalkulation der Anfrage angezeigt werden. 
### Button Neu 

Dieser Button fügt eine neue Zeile in die Tabelle ein, so dass Sie eine neue Position einfügen können. 

### Button Ändern 

Über diesen Button können Sie Änderungen an einer Position vornehmen. Auch die Kostenstellennummer kann hier optional auf Positionsebene eingegeben werden. Die Kostenstellennummer wird dann beim Kopieren einer Bestellung sowie beim Generieren einer neuen Bestellung aus dem Dispo-Pool entsprechend berücksichtigt. 

Siehe Kapitel Ändern einer Position in der Bestellung . 

### Button Löschen 

Mit diesem Button können Sie einen Eintrag der Tabelle löschen. 

Ist der Status einer Bestellung offen und ist die Bestellung noch nicht gedruckt und auch noch kein EDI-Versand erfolgt, wird über den Button Löschen die gesamte Bestellung gelöscht. 

Ist die Bestellung bereits an den Lieferanten abgeschickt worden (per Druck oder EDI), so muss die Bestellung auf Status 6 (storniert) gesetzt werden. Zuvor erscheint eine Abfrage mit entsprechenden Hinweisen. 

Auch die einzelnen Positionen können storniert werden. Bei Klick auf den Button Löschen werden diese entsprechend auf den Status 6 gesetzt. Alle entsprechenden Reservierungen werden gelöscht. Zuordnungen von Belegen auf diese Bestellung bzw. Bestellpositionen werden gelöscht, damit der Benutzer entscheiden kann, ob diese Position nochmals bestellt werden muss. 

Im Dialog Einkauf Übersicht können stornierte Bestellungen selektiert werden. So ist eine Überwachung möglich. Dazu gibt es die Checkbox Storniert .

### Button Stückliste 

Zu einem Stücklistenartikel öffnet sich mit diesem Button der Dialog Einkaufstückliste für weitere Einstellungen. 

Bereits bei der Erfassung der Bestellpositionen kann bestimmt werden, dass bei der Verbuchung des Wareneingangs die Stückliste geteilt werden soll. Das kann z.B. sinnvoll sein, wenn der Kauf eines montierten Artikels günstiger ist als der Einkauf von Einzelkomponenten, letztere aber tatsächlich benötigt werden. Diese Einstellung nehmen Sie im Dialog Position , Registerkarte Allgemein , im Feld Menge zerlegen , vor. 

Wenn Sie zu einer Stücklistenposition in der Tabelle den Button Stückliste drücken, öffnet sich der Dialog Einkaufsstückliste . Dabei muss es sich bei der Stückliste um eine Produktionsstückliste handeln!

### Combobox Positionsart 
Aus dieser Combobox wählen Sie die Positionsart aus. Zur Verfügung stehen: 

- Positionen - Gruppentitel - Zwischensumme - Text 
Position 
Die Positionsart P wird aktiviert, d.h. Sie erfassen einzelne Positionen. 

Gruppentitel 
Die Positionsart G (Gruppentitel) wird aktiviert, d.h. Sie erfassen eine Gruppierung von Artikelpositionen. Diese Gruppierung hat keine direkte Auswirkung auf Preise und Konditionen, bewirkt allerdings eine Anpassung der Darstellung auf den Belegen. 

Siehe Kapitel Gruppentitel . 

Zwischensum. 
Die Positionsart Z (Zwischensumme) wird aktiviert. Ist ein Gruppentitel mit seinen Positionen erfasst, erscheint im Auftrag eine Leerzeile. Diese Leerzeile wird im Druck der Rechnung mit der errechneten Zwischensumme gefüllt. 

Text 
Die Positionsart T (Text) wird aktiviert. Damit erfassen Sie Textpositionen. 

Siehe Kapitel Textpositionen .
