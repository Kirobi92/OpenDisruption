# Verbindungsmanager Registerkarte Allgemein

Allgemeine Funktionalitäten in eNVenta ERP / Datentransfer (Import und Export von Daten) / Verbindungsmanager 

# Verbindungsmanager Registerkarte Allgemein 
Verbindungsmanager Registerkarte Allgemein 

Auf dieser Registerkarte werden die Details zu einer Serververbindung angegeben. Bei der Angabe der Verbindungen ist zu beachten, dass der IIS-Dienst der eNVenta -Applikatonen und des JobServers die entsprechenden Rechte auf dem Ziel benötigt. Eine Einschränkung für einzelne Sachbearbeiter definieren Sie über den Button Berechtig . 

Hinweis: 
Bei der Filezilla Server-Version < 1.0 darf die folgende Option nicht aktiviert werden, da .Net dies nicht unterstützt. 

https://stackoverflow.com/questions/65636482/can-connect-to-ftp-using-filezilla-or-winscp-but-not-with-ftpwebrequest-or-flue 
### Feld Serverpfad 
In diesem Feld geben Sie den Pfad innerhalb des Servers an. Der Serverpfad sollte immer als UNC-Pfad angegeben werden: \ServerbezeichnungFreigabebezeichnung, z.B. \eNVenta-ServereNVenta-Freigabe. 

Falls Sie die Angaben nicht genau kennen, wenden Sie sich bitte an Ihren Administrator. 

Falls es sich bei der Verbindung um den Verbindungstyp „FTP“ handelt, so wird hier der entsprechende Pfad innerhalb des FTP-Servers angegeben. 

Tooltip: 

Der Pfad innerhalb des Servers. 

### Feld Server 
In diesem Feld geben Sie den Namen des Servers an. 

Tooltip: 

Der Name des Servers wird angegeben. 

### Feld Port 
In diesem Feld geben Sie den Port des Servers an. Bei FTP-Verbindungen ist dies zumeist der Port 25. 

Tooltip: 

Der Port des Servers 

### Feld Benutzer 
In diesem Feld geben Sie den Benutzernamen des Servers an. Mittels dieser Angabe erfolgt der Verbindungsaufbau zum FTP-Server. 

Tooltip: 

Der Benutzername des Servers 

### Feld Passwort 
In diesem Feld geben Sie das Passwort des Servers an. Mittels dieser Angabe erfolgt der Verbindungsaufbau zum FTP-Server. 

Tooltip: 

Das Passwort des Servers. 

### Feld Private Key für SSH 
Hier wählen Sie den privaten Schlüssel (Private Key) für die SFTP / SSH Verbindung aus. 

Der Dateiinhalt der Zertifikatsdatei hat zu beginnen mit 
---- BEGIN SSH2 ENCRYPTED PRIVATE KEY ---- 
und zu enden mit 
---- END SSH2 ENCRYPTED PRIVATE KEY ---- 

Tipp: Für die Erstellung der Keys empfehlen wir das kostenlose Programm Puttygen . 

Tooltip: 

Wählen Sie hier den privaten Schlüssel (Private Key) für die SFTP / SSH Verbindung aus. 

### Button Suche 

Über diesen Button können Sie den Private Key für SSH auswählen. 

### Button Löschen 

Mit diesem Button entfernen Sie den Private Key für SSH. 

### Feld Private Key Pass Phrase 
Hier geben Sie ein Passwort ein, wenn der private Schlüssel mit einem Passwort geschützt wurde. 

Tipp: Für die Erstellung der Keys empfehlen wir das kostenlose Programm Puttygen . 

### Checkbox FTP über SSL 
Wenn Sie diese Checkbox aktivieren, wird der FTP über SSL angesteuert. 

Tooltip: 

FTP über SSL (FTPS) ist eine Methode zur Verschlüsselung des File Transfer Protocol 

### Checkbox Zertifikat vertrauen 
Wenn Sie diese Checkbox aktivieren, erhalten Sie keinen Hinweis mehr, wenn das Zertifikat unbekannt ist. 

Tooltip: 

Soll dem Zertifikat vertraut werden? 

### Feld Conn. String 
Tooltip: 

Der Connection String wird benötigt um eine Verbindung mit einem Azure Blob Storage Account aufzubauen. 
Der Connection String kann über das Azure Portal erzeugt werden. 

### Feld Container 
Tooltip: 

Angabe des Containernamens innerhalb der Azure Blob Storage Verbindung. 
Innerhalb einer Verbindung können mehrere Container existieren und Container können Dateien und Ordner beinhalten.
