# Html Report Designer 
Html Report Designer 

Der Dialog Html Report Designer öffnet sich über das Menü Optionen im Html Report oder auch aus dem Navigator >> System >> Administration . Hier können Sie neue Reports gestalten oder auch bestehende Reports verändern. 

Damit Sie die mitgelieferten Reports verwenden können, müssen diese über die Initialisierung von eNVenta (Navigator Menü System >> Initialisierung >> eNVenta Init ) importiert werden. 

Siehe Kapitel eNVenta Init . 

### Mehrsprachigkeit 
Der gleiche Report kann in beliebig vielen Sprachen genutzt werden. Es sind keine Reportkopien für die Sprachen erforderlich. 

Die Übersetzung der Reports ist direkt im Report Designer möglich. Dazu wird nach Aufruf eines Reports einfach die Combobox Sprache umgestellt. Die fremdsprachigen Label der Felder werden automatisch aus den Objekten ermittelt und eingetragen. Diese können dann bei Bedarf angepasst werden. 

Achtung: Nur Änderungen werden in den beiden Tabellen gespeichert. Bei der Erstellung eines Reports werden die Labels erneut aus den beteiligten Objekten ermittelt und ggfs. durch die gespeicherten individuellen Änderungen ersetzt. 

Vorteil: Übersetzungsarbeiten sind nur für Anpassungen nötig. Dies funktioniert nur für Sprachen, die eNVenta als Übersetzungen bereithält (Standard: Deutsch und Englisch). 

Hinweis: Ist für eine Fremdsprache kein Label/Titel vorhanden, so wird der systemsprachige Text (deutsch) angezeigt. 

Voraussetzung: in der eNVenta Sprachtabelle müssen die ISO-Kürzel eingetragen sein! 

Im Dialog Html Report Auswahl kann durch Umschalten der Sprache der Titel der Selektion mehrsprachig gepflegt werden. 

### Drag&Drop Unterstützung 
Im Designer können Elemente von der linken in die rechte Liste gezogen und so in das Design eingefügt werden (Kopf, Zeile, Fuß). Aus technischen Gründen ist der umgekehrte Weg derzeit nicht möglich. 

Es kann ein Business Objekt auf den Designer gezogen werden. Es erscheint die Drop-Option "Report Daten zuweisen". Es werden zwei Fälle unterschieden: 
- Das zu dem Business Objekt gehörende Report Design ist schon geladen. In diesem Fall werden anstatt der Dummy-Daten die Daten des Business Objekts in der Vorschau verwendet. So können Sie zur Designzeit das Layout mit verschiedenen Daten testen. - Es ist ein anderes Report Design oder gar keines geladen. In diesem Fall wird zunächst das passende Report Design geladen und dann werden die Daten des Business Objekts als Vorschaudaten verwendet. Dies kann auch zur schnellen Suche eines Reports bzw. zur schnellen Nachbearbeitung eines Reports genutzt werden. 

### Customizing-Hinweis 
Soll ein weiteres Objekt/Objektliste in die Liste der Datenobjekte aufgenommen werden, ist folgendes erforderlich: 

- Neuer Eintrag in der Methode cHTMLReportManager.LoadBODefListInner() 
Format: Eindeutige Nummer fortlaufend, Einzelobjekt (optional), Collection-Objekt (optional), AccessUnit (optional), Delegate Collectionermittlung (optional) 
Achtung: Ist sowohl das Einzel-, als auch das Collection-Objekt angegeben, MUSS die Regel, wie aus dem Einzelobjekt die Collection zu ermitteln ist, angegeben werden (letzter Parameter Delegate). 
Bitte Nummern > 50000 verwenden, um Konflikte mit Standarderweiterungen zu vermeiden! - Eintrag eines Datenobjekt-Namens unter der eindeutigen Nummer in der Textcollection ctBusinessObjectDefName. - Soll im Viewer eine Blätterfunktion der Datenobjekte angeboten werden (nur für Einzelobjekt möglich), muss das Interface IcObjectScrollInterface (in General.Reminder) im Datenobjekt implementiert werden. (Beispiel siehe cdArticle). Es sind 4 Methoden zu implementieren: 
GetNextObject(), GetPreviousObject(), HasNextObject(), HasPreviousObject() - Neue oReference-Objekte müssen, damit sie automatisch geladen werden können, in der Methode this.Global.oBasics.GetObjectFromReference(…) ergänzt werden. - Sonderfall Hüllobjekte (z.B. wie cSalesOrder). Die Bereitstellung des eigentlichen Haupt-Datenobjekts muss sichergestellt werden (z.B. cdSalesOrderMaster). Dazu sind folgende Methoden zu ergänzen: 
- cHTMLReportManager.ConvertObject() 
- cHTMLReportManager.ConvertObjectReverse() 

### Vorgehensweise zur Erstellung eines Reports mit Subreport 
- Erstellen des Hauptreports. Dies muss ein Report mit einem Einzel-Datenobjekt sein. Reine Listen-Datenobjekte unterstützen keine Subreports. - Erstellen des Subreports. Hierbei muss ein Datenobjekt mit einer Liste gewählt werden. Einzelobjekte können nicht als Subreport verwendet werden. - Erstellen einer Report Auswahl. Das Quellobjekt muss dem Objekt des Hauptreports entsprechen. 
Als Html-Report muss der zuvor erstellte Subreport gewählt werden. 
Ein Titel muss angegeben werden, unter dem später der Link angezeigt wird. 
Es müssen eine oder mehrere Bedingungen angegeben werden, die die beiden Datenobjekte verbinden (Join), da sonst alle in der Datenbank vorhandenen Datensätze gelesen werden. - Im Hauptreport muss auf der Registerkarte Links ein neuer Eintrag gemacht werden. Die Spalte ‚Report-Auswahl‘ bietet nun den neuen Subreport zur Auswahl an. 
## Steuerelemente des Formulars 

### Bezeichnungsfeld Html Report Designer 
Tooltip: 

In diesem Dialog können Sie neue Reports gestalten oder auch bestehende Reports verändern. 
Für mehr Informationen drücken Sie F1. 

### Button Form-Menü 

Mit Klick auf diesen Button werden je nach Kontext und Berechtigung die unterschiedlichen Optionen des jeweiligen Dialogs hinter dem Button zur Auswahl angeboten. 

Die Funktionen für das Client Side Customizing sowie die Online-Hilfe stehen über diesen Button ebenfalls zur Verfügung. 

## Menüeinträge 

### Menü Belegfolge 
Über dieses Menü öffnet sich die Belegfolge. Alle Vorgänge der Belegkette inkl. der Druckbelege werden auf einem Zeitstrahl angezeigt. 

Siehe Kapitel Belegfolge . 

### Menü Html Report 
Über dieses Menü öffnet sich der Dialog Html Report . 

Siehe Kapitel Html Report . 

### Menü Html Report Wahl 
Über dieses Menü öffnet sich der Dialog Html Report Auswahl . Dort stehen sämtliche, bereits angelegte Html Reports zur Auswahl. 

Siehe Kapitel Html Report Auswahl . 

### Menü Hilfe 
Über dieses Menü gelangen Sie in die Online-Hilfe und Sie erhalten Informationen zu dem Dialog, aus dem heraus Sie die Hilfe geöffnet haben. 

### Menü Customizing 
Über dieses Menü gelangen Sie ins Client Side Customizing. 

Siehe Kapitel Client Side Customizing . 

### Menü Zusatzfelder konfigurieren 
Über dieses Menü erreichen Sie den Dialog Zusatzfelder definieren . Die dynamischen Zusatzfelder dienen der Ablage zusätzlicher Informationen zum Vorgang, ohne dass eine neue Programmierung notwendig ist. 

Siehe Kapitel Zusatzfelder definieren .

Über dieses Menü werden Ihnen Informationen zum Dialog angezeigt. Siehe nachfolgendes Beispiel: 

Meldungen 

### Button Referenz-Link 

Dieser Button steht zur Verfügung, um Dokumenten-Links sowie URL‘s als Info in eNVenta ERP zu hinterlegen oder um 2 Vorgänge/Referenzen miteinander zu verknüpfen. 

Siehe Kapitel Referenz-Links . 

### Button Drucken 

Dieser Button startet den Druck . Der Button wird erst aktiv, wenn Daten eingelesen sind. 

### Button Suche 

Dieser Button startet die Suche nach Daten; entweder ganz allgemein oder wenn Sie im Kopf des Dialogs entsprechende Selektionskriterien festgelegt haben, gezielt. 

### Button Neu 

Dieser Button leert die Daten in den Kopf-Feldern. 

Ist in den Parametern für die Sachbearbeiter die Checkbox 2 Schritt Neu aktiviert, bleiben die Suchkriterien in den Stammdialogen erhalten. 

Siehe Kapitel Parameter Sachbearbeiter . 

### Button Kopieren 

Dieser Button startet die Kopierfunktion. Es werden sämtliche Felder kopiert, welche keine Schlüsselfelder sind. 

### Button Speichern 

Dieser Button speichert Ihre Eingaben. 

### Button Vorheriger Satz 

Mit diesem Button können Sie rückwärts blättern. Die Reihenfolge beim Blättern bezieht sich auf die Registerkarte Tabelle , sofern diese vorhanden ist. 

### Button Nächster Satz 

Mit diesem Button können Sie vorwärts blättern. Die Reihenfolge beim Blättern bezieht sich auf die Registerkarte Tabelle , sofern diese vorhanden ist. 

### Button Löschen 

Dieser Button löscht Ihre Eingaben. Genauer gesagt, die markierte Zeile wird mit dem versehen und erst endgültig gelöscht, wenn Sie den Button drücken. 

### Button Schließen 

Dieser Button schließt den Dialog. In manchen Fällen werden Sie gefragt, ob Sie die Eingaben / Änderungen noch vorher speichern möchten. 

### Button Auswahl 

Dieser Auswahl öffnet den Dialog Html Report Auswahl . 

Siehe Kapitel Html Report Auswahl . 

### Feld Titel 
Hier geben Sie dem Report einen Namen. 

Tooltip: 

Angabe des Titels 

### Combobox Datenobjekt 
Hier wird das zugrunde liegende Datenobjekt angezeigt. 

Beispiel: Wenn diesem Html Report Designer der Kundenstamm zugrunde liegt, wird hier "Kunde" angezeigt. 

### Combobox Sprache 
Hier können Sie die Sprache für den Report auswählen. 

Tooltip: 

Angabe des Ländercode gemäß ISO 3166-1 alpha-2
