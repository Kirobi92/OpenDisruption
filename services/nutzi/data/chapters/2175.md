# Parameter Service Erinnerung 
Parameter Erinnerung 

Dieser Dialog öffnet sich im Navigator >> System >> Service >> Parameter Erinnerung . 

Für eine bessere Betreuung der Kunden in Bezug auf die Wartung der Serviceartikel können aus eNVenta automatisierte E-Mails an den Kunden erzeugt werden, um zu informieren, dass die Wartung des Serviceartikels fällig ist. 

Hier werden die Einstellungen für die Erinnerung an eine Wartung vorgenommen, wie z.B. eine Beschreibung, eine E-Mail-Vorlage, Karenztage etc. 

Es können mehrere Parameter-Einstellungen verwaltet werden, um z.B. verschiedene Reports für Kunden bzw. unterschiedliche Karenztage zu verwalten. 

Die Zuordnung der Erinnerungen für die Serviceeinsätze erfolgt über die Einsatzarten. Pro Einsatzart kann eine Parametrisierung der Erinnerung hinterlegt werden. 

Siehe Kapitel Einsatzarten . 

Die Mahntexte werden auf den Registerkarten Erinnerungstext 1-3 angelegt und sind zwingend dementsprechend pro Serviceeinsatz zuzuweisen. 

Der Kunde kann per E-Mail, Fax oder Druck erinnert werden. Dazu können auf der Registerkarte Allgemein dementsprechende E-Mailvorlagen, Faxvorlagen und ein Report hinterlegt werden. Zusätzlich können Erinnerungstexte für die Stufe 1, 2 und 3 angelegt werden. Dafür steht jeweils eine Registerkarte zur Verfügung. Pro Erinnerungsstufe kann die jeweilige Anzahl an Karenztagen hinterlegt werden. Ab der Erinnerungsstufe 3 wird immer der Erinnerungstext 3 verwendet. 

Im Kundenstamm und im Dialog Ansprechpartner kann eine Hierarchie festlegt werden, wie die Erinnerung erfolgen soll. Zuerst wird ein potentieller Erinnerungsweg des Ansprechpartners geprüft, danach der Erinnerungsweg im Kundenstamm. 

Die Combobox Erinnerungsweg im Ansprechpartner und im Kundenstamm bietet folgende Auswahlmöglichkeiten zu einem Erinnerungsweg im Service: 

- E-Mail, Fax, Druck - E-Mail, Druck, Fax - Fax, E-Mail, Druck - Fax, Druck, E-Mail - Druck, E-Mail, Fax - Druck, Fax, E-Mail 
Siehe Kapitel Ansprechpartner in den Stammdaten . 

Siehe Kapitel Kunde . 

### Generierung der Erinnerung 
Die Generierung der Erinnerung erfolgt immer, wenn ein neuer Serviceeinsatz fällig bzw. überfällig ist. Hierzu müssen die Serviceeinsätze entsprechend bereits im Vorfeld generiert werden, z.B. per Massenlauf. Nur auf Basis der generierten Serviceeinsätze erfolgt ein Versand der Erinnerungen an den jeweiligen Kunden bzw. Ansprechpartner des Serviceeinsatzes. 

Ob Serviceeinsätze fällig bzw. überfällig sind, ermittelt das System wie folgt: 

- Datum letzte Wartung oder wenn leer - Datum Inbetriebnahme + Anzahl Tage letzte Wartung(Periode) + Karenztage. - Wenn dieses Datum in der Vergangenheit liegt, so wird der Serviceeinsatz als überfällig angesehen und der Kunde informiert. 
Die Service E-Mails werden gruppiert/gebündelt nach dem Ursprungsauftrag und per Job erzeugt (dafür steht der Button zur Verfügung). 

In diesem Zusammenhang wurde die Anzeige des Status im Serviceeinsatz erweitert. Erst hat der Serviceeinsatz den Status offen . Wenn die Erinnerungsnachricht erzeugt wurde, erhöht sich der Status des Serviceeinsatzes auf Kunde erinnert . Und die Erinnerungsstufe wird auf 1 gesetzt.

### Button Drucken 

Dieser Button startet den Druck . Der Button wird erst aktiv, wenn Daten eingelesen sind. 

### Button Suche 

Dieser Button startet die Suche nach Daten; entweder ganz allgemein oder wenn Sie im Kopf des Dialogs entsprechende Selektionskriterien festgelegt haben, gezielt. 

### Button Neu 

Dieser Button leert die Daten in den Kopf-Feldern. 

Ist in den Parametern für die Sachbearbeiter die Checkbox 2 Schritt Neu aktiviert, bleiben die Suchkriterien in den Stammdialogen erhalten. 

Siehe Kapitel Parameter Sachbearbeiter . 

### Button Kopieren 

Dieser Button startet die Kopierfunktion. Es werden sämtliche Felder kopiert, welche keine Schlüsselfelder sind. 

### Button Speichern 

Dieser Button speichert Ihre Eingaben. 

### Button Vorheriger Satz 

Mit diesem Button können Sie rückwärts blättern. Die Reihenfolge beim Blättern bezieht sich auf die Registerkarte Tabelle , sofern diese vorhanden ist. 

### Button Nächster Satz 

Mit diesem Button können Sie vorwärts blättern. Die Reihenfolge beim Blättern bezieht sich auf die Registerkarte Tabelle , sofern diese vorhanden ist. 

### Button Löschen 

Dieser Button löscht Ihre Eingaben. Genauer gesagt, die markierte Zeile wird mit dem versehen und erst endgültig gelöscht, wenn Sie den Button drücken. 

### Button Schließen 

Dieser Button schließt den Dialog. In manchen Fällen werden Sie gefragt, ob Sie die Eingaben / Änderungen noch vorher speichern möchten. 

### Button Vorschau 

Über diesen Button öffnet sich der Dialog Serviceübersicht . So können Sie sich eine Übersicht über sämtliche Einsätze verschaffen. 

Siehe Kapitel Serviceübersicht .
