# Arbeitsplan 
Arbeitsplan 

Ein Arbeitsplan dient als „Schablone“, welche Arbeitsschritte für die Herstellung eines Stücklistenartikels durchlaufen werden müssen. Dazu werden verschiedene Arbeitsgänge zusammengefasst. 

Das im Kopf des Arbeitsplans anzugebende Produkt/Baugruppe muss im Artikel als Produktionsstückliste definiert sein. 

Es darf nur einen Arbeitsplan je Produkt/Baugruppe geben. 

Eine Einstellung von Kostenstelle und Zuschlagssätzen überschreibt die Einstellungen aus dem Dialog Parameter Produktion . 

Die Verwaltung von Arbeitsplänen findet auf mehreren Ebenen statt und ermöglicht so eine effiziente und flexible Vorgehensweise: 

- Allgemeiner Arbeitsgangstamm - Arbeitsplanstamm - Individueller Arbeitsplan je Betriebsauftrag (änderbare Kopie aus Stamm) - Kundenauftragsspezifischer Arbeitsplan (zur Vorplanung und individuellen Konfiguration bevor ein Betriebsauftrag entsteht) 
Ein Arbeitsplan bezieht sich immer auf eine Ebene einer Baugruppe bzw. Stückliste. Er enthält eine Liste von Arbeitsgängen, die in der vorgegebenen Reihenfolge abgearbeitet werden sollen. 

Im Arbeitsplan können auch Varianten zu jedem Arbeitsschritt (Posnr) verwaltet werden. Diese Varianten sind analog zu den Stücklistenvarianten zu sehen. Wird ein Betriebsauftrag in einer bestimmten Variante erzeugt, so wird auch der Arbeitsplan in dieser Variante übertragen. Die Variante 0 ist dabei immer die Basisvariante. Diese wird arbeitsgangspezifisch nur dann durch eine speziellere Variante ersetzt, wenn diese im Stammarbeitsplan existiert. (Es müssen also nur Abweichungen zum Standard gepflegt werden.)
