# Assortierung im Rabatt Verkauf 
Dies ist eine spezielle Option für Artikelgruppenrabatte. Normalerweise werden die Rabatte nur für eine einzige Auftrags- oder Angebotsposition berechnet, d.h. die Rabattstaffel wird auf die Menge der einzelnen Positionen angewandt. Wird diese Option jedoch gesetzt, werden die Mengen aller Positionen einer Kumulationsgruppe zusammengefasst und darauf für jede einzelne Position die entsprechende Rabattstaffel angewendet. Der sich daraus ergebende Rabatt wird dann in die Position eingetragen. 

Voraussetzung ist jedoch, dass in der einzelnen Kondition ebenfalls die Assortierung eingeschaltet ist. 

Ist die Option Assortierung eingeschaltet, hat dies Auswirkungen auf die Geschwindigkeit der Auftragserfassung. Die Rabattvergabe erfolgt nur einmalig zum Zeitpunkt der Ersterfassung des Auftrags (Button ) oder bei einer Neukalkulation. 

Bitte beachten Sie auch, dass der assortierte Rabatt alle eventuell vorhandenen Rabatte/Zuschläge der Position überschreibt. 

### Vorgehensweise für den Aufbau einer Assortierung im Rabatt Verkauf 
Der Schalter Assortierung im Programmteil Parameter Verkauf (Registerkarte Allgemein ) muss eingeschaltet werden. 

Die möglichen Kumulationsgruppen müssen geplant und mit der Picklistenverwaltung vorgegeben werden. 

Jedem Artikel, der an der Assortierung teilnehmen soll, muss eine Kumulationsgruppe im Artikel (Infos) zugeordnet werden. 

In den Konditionen muss ebenfalls die Assortierung eingeschaltet sein. 

Siehe Kapitel Parameter Verkauf Registerkarte Allgemein .
