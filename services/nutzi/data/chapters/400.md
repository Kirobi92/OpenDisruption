# Ereignisdefinition Registerkarte Bedingung 
Ereignisdefinition Registerkarte Bedingung 

Um das Event weiter einzuschränken, können auf der Registerkarte Bedingung weitere Einschränkungen definiert werden. 

Beispiel: 
Das Event soll nur gültig sein, wenn der angelegte oder geänderte Artikel kein Divers Artikel ist. Siehe Beispiel: 

Ereignisdefinition 

Mit dem -Button auf der rechten Seite der Tabelle kann eine neue Zeile hinzugefügt werden. 

In der Spalte ‚Feld‘ kann nun ausgewählt werden, auf welches Feld eine Bedingung erfasst werden soll. 

Mit den Spalten ‚Eingefügt‘, ‚Geändert‘ oder ‚Gelöscht‘ kann folgendes definiert werden: 

- Eingefügt: Das Feld war leer und wurde nun gefüllt. - Geändert: Der Wert des Feldes hat sich geändert. - Gelöscht: Das Feld war gefüllt und wurde nun geleert. 
Ist keine der Checkboxen gesetzt, wird der Wert immer verglichen. 

Mit dem Operator und dem Wert wird definiert, wie und mit welchem Wert der Inhalt des Feldes verglichen wird. 

Alle Bedingungen werden mit "UND" verknüpft. 

Somit trifft das Ereignis ein, wenn ein neuer Nicht-Divers Artikel angelegt wird. 
### Checkbox OR-Verknüpfung 
Wenn Sie diese Checkbox aktivieren, werden die Property-Bedingungen mit "OR" verknüpft. D.h. wenn eine der Bedingungen zutrifft, ist das Event eingetreten. 

Wenn Sie Bedingungen mit "AND" und "OR" abdecken wollen, kann das über den Ausdruckseditor vorgenommen werden. 

Siehe Kapitel Ausdruckseditor . 

Tooltip: 

Die angegebenen Property-Bedingungen werden mit OR verknüpft. 
D.h. wenn eine der Bedingungen zutrifft, ist das Event eingetreten. 

Ansonsten müssen alle Property-Bedingungen zutreffen.

### Tabellenspalte Eingefügt 
Wenn der Haken in dieser Spalte gesetzt ist, wird das Event nur ausgelöst, wenn die ausgewählte Komponente neu ist. 

Beispiel: 
Das Feld Hauptlieferant im Artikelstamm war leer und wird nun mit einer Lieferantennummer gefüllt. 

Tooltip: 

Das Event ist nur gültig, wenn in das ausgewählte Feld ein Wert eingetragen wird, 
aber davor kein Wert vorhanden war. 

z.B.: Das Feld "Hauptlieferant" im Artikelstamm war leer 
und wird nun mit einer Lieferantennr befüllt. 

### Tabellenspalte Geändert 
Wenn der Haken in dieser Spalte gesetzt ist, wird das Event nur ausgelöst, wenn die ausgewählte Komponente geändert wurde. 

Beispiel: 
Sie ändern eine existierende Auftragsposition. 

Tooltip: 

Das Event ist nur gültig, wenn in dem ausgewählen Feld der Wert geändert wird. 

z.B.: Das Feld "Hauptlieferant" im Artikelstamm hatte einen Wert 
und wird nun mit einer anderen Lieferantennr befüllt. 

### Tabellenspalte Gelöscht 
Wenn der Haken in dieser Spalte gesetzt ist, wird das Event nur ausgeöst, wenn die ausgewählte Komponente gelöscht wurde. 

Beispiel: 
Das Feld Hauptlieferant im Artikelstamm ist gefüllt und wird nun gelöscht. 

Tooltip: 

Das Event ist nur gültig, wenn aus dem ausgewählen Feld der vorhandene Wert gelöscht wird. 

Beispiel: Das Feld "Hauptlieferant" im Artikelstamm ist befüllt 
und wird nun herausgelöscht. 

### Tabellenspalte Operator 
In dieser Spalte wählen Sie den Vergleichsoperator aus. 

Beispiel: 
Feld "VK1" 
Operator "Kleiner" 
Wert "100" 

Tooltip: 

Vergleichen Sie den Wert im Feld mit einer Konstanten, die Sie im Feld 'Wert' angeben. 
Der Vergleichs-Operator kann hier angegeben werden. 

Z.B. 
Feld "VK1" 
Operator "Kleiner" 
Wert "100" 

### Tabellenspalte Wert 
Vergleichen Sie den Wert im Feld mit einer Konstanten, die Sie hier eingeben. 

Beispiel: 
Feld VK1 
Operator "Kleiner" 
Wert "100" 

Tooltip: 

Vergleichen Sie den Wert im Feld mit einer Konstanten, die Sie im Feld 'Wert' angeben. 
Die Konstante kann hier angegeben werden. 

z.B. 
Feld "VK1" 
Operator "Kleiner" 
Wert "100"
