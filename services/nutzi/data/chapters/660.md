# Lieferant Registerkarte Rechnungswesen 
Lieferant Registerkarte Rechnungswesen 

Auf dieser Registerkarte werden Informationen zum Lieferanten angelegt, die für den Zahlungsverkehr mit dem Lieferanten wichtig sind. 
### Feld Sammelkonto 
Hier wird anhand der zugewiesenen Währung das entsprechende Sammelkonto angegeben. Dieses Konto kann auch manuell geändert werden, allerdings ist unbedingt darauf zu achten, dass das Konto, welches hier eingetragen wird, auch besteht und in der richtigen Währung geführt wird. 

Wird die Währung geändert, so wird auch das zugehörige Sammelkonto automatisch angepasst. 

### Button Suche 

Dieser Button öffnet den Dialog Kontoauswahl , um komfortabel das gesuchte Konto auszuwählen. 

Siehe Kapitel Kontoauswahl . 

### Feld Std-Geg.-Kto 
In dieses Feld können Sie eine Kontonummer für eine standardmäßige Gegenbuchung eingeben. Diese wird im Buchen vorbelegt und ist änderbar. 

Diese Einstellung zieht ausschließlich bei manuellen Buchungen in der Finanzbuchhaltung.

### Combobox Lieferantenstatus 
Aus dieser Combobox können Sie einen Status für den Lieferanten auswählen. Diese Status stehen zur Verfügung, um z.B. bei einer Umfirmierung des Lieferanten, bei Verkauf oder Fusionierung bestehende Vorgänge abschließen zu können und neue zu „verhindern“. Die noch laufenden Bestellungen und auch noch nicht fakturierten Rechnungen müssen in einem solchen Fall gebucht werden können. Es dürfen aber keine neuen Bestellungen mehr auf einen solchen Lieferanten erfasst werden. 

Folgende Status stehen zur Verfügung und haben folgende Bedeutung: 

- 0 - Normal -> Lieferant ist "offen" - 1 - Gesamtsperre -> Es darf nichts mehr mit dem Lieferanten gemacht werden. (jetzige Sperrung) - 2 - Fakturasperre -> Der Warenverkehr darf noch abgewickelt, Eingangsrechnungen aber nicht verbucht werden. - 3 – Bestellsperre -> Bestehende Bestellungen dürfen noch abgewickelt werden, sofern bereits bestellt wurde (EDI/direkt); es dürfen aber keine neuen Bestellungen mehr erfasst werden. 
Der Lieferantenstatus wird im Dialog Lieferantenwahl in der Spalte ‚S‘ angezeigt. 

Im Lieferantenvorschlag werden bestell-gesperrte Lieferanten nicht berücksichtigt. 

Eine Bestellung kann nur gespeichert werden, wenn der Lieferant in einer neu erfassten Bestellung im Status 0 oder 2 ist oder bereits erfasste Bestellungen im Status 0, 2 oder 3 sind. 

Bestellungen von Lieferanten mit Gesamt- oder Bestellsperre können nicht kopiert werden! 

Im Gateway wird die Abfrage abgewiesen, wenn der Lieferant eine Bestellsperre hat. Ein Export wird abgelehnt. 

Bestellvorschläge können weiterhin erstellt werden. Bei einem gesperrten Lieferanten wird die Bestellung später im Bestellautomaten verhindert. 

Im Wareneingang wird die Prüfung auf Status != 1 geändert. 

Rechnungen für einen gesamt-gesperrten Lieferanten können weder genehmigt noch freigegeben werden. Rechnungen für Lieferanten mit Status 2 ( Fakturasperre ) dagegen können genehmigt, aber nicht freigegeben werden. 

Beim Generieren einer Bestellung aus der Rahmenbestellung wird geprüft, ob der Lieferant eine Bestellsperre hat. 

Das Ändern einer Rahmenbestellung ist bei Gesamtsperre eines Lieferanten nicht möglich. Bei Bestellsperre kann die Rahmenbestellung geändert werden, aber keine Bestellungen generiert werden. 

Bei Fakturasperre darf geändert werden, Bestellungen können generiert werden. 

Im Dialog Position (Verkauf) können keine Lieferanten mit Bestellsperre ausgewählt werden. 

DESADV -> INVPRT -> dürfen importiert werden, wenn nicht Gesamt- oder Bestellsperre gesetzt ist. Bestellpositionen dürfen angelegt werden. 

INVOICE -> dürfen in die Rechnungsübernahme übernommen werden, aber nicht in der Rechnungsprüfung angelegt und gebucht werden, wenn Faktura- und Gesamtsperre gesetzt ist. 

Tooltip: 

0, leer = Normal 
1 = Gesamtsperre 
2 = Fakturasperre 
3 = Bestellsperre 

### Checkbox Zahlsperre 
Wenn Sie diese Checkbox aktivieren, wird automatisch für jeden neuen OP (im Dialog OP Kreditoren Detail ) eine Zahlsperre gesetzt. 

Auf bereits gebuchte Lieferantenrechnungen wirkt sich das Setzen dieses Parameters nicht aus! 

Tooltip: 

Diesen OP nicht im Zahlungsvorschlag berücksichtigen. 

### Feld Zentralzahler 
Das Feld Zentralzahler wird wie das Feld Verband (Registerkarte Warenwirtschaft ) für die Verwendung der Verbandsabrechnung benötigt. Es bietet eine zusätzliche Auswahlmöglichkeit in der Verbandsabrechnung für Kreditoren. 

Siehe Kapitel Verbandsabrechnung Kreditor . 

### Feld ZZ-Mitglied 
Hier wird die Mitgliedsnummer beim Zentralzahler eingetragen. 

Tooltip: 

Gibt die Mitgliedsnummer beim Zentralzahler an. 

### Feld Druck Avis ab 
Mit einem Eintrag in diesem Feld (Anzahl der OPs) können für Bankeinzug und Zahlungsverkehr die Einstellungen aus den Parametern übersteuert werden. Auf diese Weise kann definiert werden, ob und ab welcher Position ein Avis gedruckt werden soll. 

Das Avis wird innerhalb einer Kontonummer, gruppiert nach Zahlungsverbindung, gedruckt. 

### Feld E-Mail Avis 
Hier wird eine E-Mail-Adresse für den Versand eines Avis eingetragen. 

Um sicherzugehen, dass eine E-Mail an die richtige Adresse gesendet wird, prüft eNVenta beim Speichern oder Ändern einer Mailadresse, ob die Mail-Adresse der „Norm“ entspricht. Falls nicht wird ein Hinweis ausgegeben. Dabei wird das Speichern nicht grundsätzlich verhindert. Auf diese Weise werden z.B. Outlook-Adressbucheinträge wie z.B. „#EW_ERP“ bei Bedarf gespeichert und genutzt. 

Dabei läuft die Prüfung ausschließlich bei Anlage und Änderung einer Mail-Adresse. Werden andere Einträge, z.B. des Kundenstammsatzes geändert, so kommt diese Meldung nicht. 

Werden Stammdaten über das Gateway importiert, so werden u.U. fehlerhafte Mailadressen importiert (ggfs. mit Warnung). Bei einem Import unter Verwendung des Jobservers erfolgt dies ebenfalls in Form einer Warnung (im Jobprotokoll). 

Tooltip: 

Angabe der E-Mail-Adresse. 
Bei der Angabe mehrerer Adressen, müssen diese durch Semikolon getrennt sein.
