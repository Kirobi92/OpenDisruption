# Opportunityübersicht Registerkarte Allgemein 
Opportunityübersicht Registerkarte Allgemein 

Auf dieser Registerkarte wird die Entwicklung des Opportunity-Bestands für den aktuellen Monat und die vorausgegangenen 11 Monate dargestellt. Es wird erkennbar, wie sich die Opportunities in der Vergangenheit entwickelt haben. 

Diese Darstellungen beziehen sich immer nur auf die Selektionskriterien Vertreter und Opportunitytyp . 
### Button Historie 

Dieser Button öffnet den Dialog Kontakt-Historie . Allerdings werden hier, bezogen auf die Opportunityübersicht, nicht die Kontakte nach Themen gegliedert, sondern nach den Stufen des Vertriebsprozesses. 

D.h. es wird auch eine Art Protokoll über die Stufenveränderungen dargestellt. Zu jeder Stufe werden dazu existierende Kontakte aufgelistet. Auf diese Weise wird der gesamte Vertriebsprozess transparent. 

Siehe Kapitel Kontakt-Historie . 

Tooltip: 

Öffnet die Kontakthistorie. 

### Button Stamm 

Wenn eine Zeile der Tabelle markiert ist, gelangen Sie über diesen Button in den Adress-bzw. Kundenstamm. 

Tooltip: 

Öffnet Adress-/Kundenstamm 

### Button Call 

Der Dialog zur Call -Erfassung öffnet sich. Die Kontaktdaten der Adresse bzw. des Kunden werden dabei übernommen. 

Tooltip: 

Öffnet die Call-Erfassung 

### Button Wiedervorlage 

Über diesen Button kann eine Wiedervorlage erzeugt werden. 

Siehe Kapitel Erfassung eines neuen Termins . 

Tooltip: 

Erzeugt eine Wiedervorlage 

### Button Angebot 

Mit diesem Button gelangen Sie in den Dialog Angebot . Die Kontaktdaten des Kunden werden übernommen. Wollen Sie ein Angebot für eine Adresse erstellen, werden Sie automatisch gefragt, ob die Adresse in einen Kunden umgewandelt werden soll. 

Tooltip: 

Erzeugt ein Angebot 

### Button Projekt 

Wenn Sie diesen Button drücken, können Sie einem Projekt eine Opportunity zuordnen. Dazu markieren Sie die entsprechende Zeile in der Tabelle und drücken diesen Button. 

Ist bereits ein Projekt zugeordnet, rufen Sie über diesen Button das Projekt auf. 

Tooltip: 

Ordnet dem Projekt eine Opportunity zu. 

### Button Erhalten 

Wenn Sie für eine markierte Zeile den Button erhalten drücken, wird der Eintrag als „erhalten“ markiert. Das bedeutet, diese Aktion war erfolgreich. 

Tooltip: 

Markiert eine Zeile der Tabelle als "erhalten". 

### Button Verloren 

Wenn Sie für eine markierte Zeile den Button verloren drücken, wird der Eintrag als „verloren“ markiert. Das bedeutet, diese Aktion ist gescheitert. 

Tooltip: 

Markiert eine Zeile der Tabelle als "verloren". 

### Tabelle Opportunities 

### Tabellenspalte ABC 
In dieser Spalte ist der Klassifizierungstyp eingetragen. Die Zuordnung geschieht über die Einstellung des Opportunity-Typs und berücksichtigt das Bestelldatum und/oder den zu erwartenden Umsatz. 

Tooltip: 

Ergebnis ABC-Analyse. 

### Tabellenspalte Bezeichnung 
Der Name der Opportunitystufe wird eingetragen. 

Tooltip: 

Angabe der Bezeichnung 

### Tabellenspalte Funnelstufe 
Die Funnelstufe wird angezeigt. Die Funnelstufe wird aus dem Opportunitytyp vorbelegt und beginnt immer bei der ersten Funnelstufe des Opportunitytyps. 

Eine Funnelstufe kann nur erhöht werden, eine Rückstufung der Opportunity ist nicht möglich. 

Tooltip: 

Angabe der Funnelstufe 

### Tabellenspalte Prozessstufe 
Die Prozessstufe wird angezeigt. Sie ist abhängig vom Opportunitytyp und der Funnelstufe. 

Eine Prozessstufe kann nur erhöht werden, eine Rückstufung der Opportunity ist nicht möglich. 

Tooltip: 

Angabe der Prozessstufe 

### Tabellenspalte Typ 
Hier wird der Typ eingetragen, entweder Kunde oder Adresse. 

### Tabellenspalte Firma 1 
Der Firmenname wird eingetragen. 

Tooltip: 

Zeile 1 des Firmennamens. 

### Tabellenspalte Datum Stufe 
Das Datum der Erreichung dieser Stufenkombination wird eingetragen. Das Feld wird farbig hinterlegt: 

Grün Gelb Rot 
| alles ok |
| --- |
| die erste Frist zur Erreichung der nächsten Funnel-/Prozessstufe ist erreicht. |
| die zweite Frist zur Erreichung der nächsten Funnel-/Prozessstufe ist erreicht. |

Die Einstellung der Fristen geschieht über Opportunitystufen. 

Tooltip: 

Das Datum der Erreichung dieser Stufenkombination wird eingetragen. Das Feld wird farbig hinterlegt: Grün - ok. Gelb - erste Frist erreicht. Rot - zweite Frist erreicht. 

### Tabellenspalte Bestelldatum 
Hier wird der erwartete Auftragsabschlusstermin eingegeben. Auch wenn kein definitives Datum bekannt ist, ist doch ein ungefährer Tag einzutragen. Nur so können eine ABC-Analyse, eine Zeitüberwachung und vor allem auch eine zeitlich aufgeschlüsselte Forecast-Bewertung funktionieren. 

Tooltip: 

Angabe des Bestelldatums 

### Tabellenspalte Projekt 
Der Opportunity kann ein Projekt zugeordnet werden. 

Tooltip: 

Tragen Sie eine eindeutige Projektnummer ein. 

### Tabellenspalte Wahrscheinlichkeit % 
Wird eine neue Opportunity-Stufe erreicht, wird hier automatisch der zur Stufe hinterlegte Wahrscheinlichkeitsprozentsatz eingetragen. Der Wert kann manuell geändert werden. Mit diesem Prozentsatz wird der angegebene Umsatz bewertet und der Forecast berechnet. 

Tooltip:
