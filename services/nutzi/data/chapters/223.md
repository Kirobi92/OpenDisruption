# Artikelauswahl Registerkarte Artikelstamm 
Artikelauswahl Registerkarte Artikelstamm 

Nach erfolgreicher Suche können aus der Registerkarte Artikelstamm ein oder mehrere Artikel mit der Menge, welche Sie in die Spalte ‚Menge“ eingegeben haben, in den Vorgang übernommen werden. Eine Eingabe im Feld Menge ist keine Pflicht. 

Außerdem ist es möglich mehrere markierte Artikel gleichzeitig zu übernehmen. Hierbei ist es unerheblich in welcher Tabelle bzw. Registerkarte diese Artikel markiert sind. 

Wenn bei der Übernahme von einem oder mehreren Artikeln in den Auftrag ein Problem auftritt, können Sie mit dem Button Neu (oder F7 ) auf der Registerkarte Positionen im Auftrag den Vorgang fortsetzen. Z.B. eine Meldung bei mangelnder Verfügbarkeit. 

Wird die Checkbox Auch gesperrte aktiviert, so werden auch gesperrte Artikel im Dialog Artikelauswahl angezeigt. 

Des Weiteren stehen unterhalb der Tabelle Registerkarten zur Verfügung. Dies sind die Registerkarten Artikelinfo, Alternativen, Zubehör und Cross Over : 

Artikelinfo: 
Hier erhalten Sie Detailinformationen zum Artikel, wie z.B. Artikelnummer, Bezeichnung und Artikelbild. 

Alternativen: 
Diese Daten werden nur eingelesen, wenn der Benutzer auf die Registerkarte klickt. Dabei werden die Alternativgruppe sowie die Alternativen aus dem Artikelstamm berücksichtigt. 

Gezeigt wird eine weitere Tabelle (analog der oberen) mit alternativen Artikeln. Hier kann ebenfalls die Menge eingegeben und ausgewählt werden. (Eine kombinierte Auswahl ist in allen Tabellen möglich!) 

Zubehör: 
Diese Daten werden ebenfalls nur eingelesen, wenn der Benutzer auf die Registerkarte klickt. Gezeigt wird eine weitere Tabelle (analog der oberen) mit Zubehörartikeln. Hier kann ebenfalls die Menge eingegeben und ausgewählt werden. (Eine kombinierte Auswahl ist in allen Tabellen möglich!) 

Cross Over: 
Diese Daten werden auch nur eingelesen, wenn der Benutzer auf die Registerkarte klickt. Gezeigt wird eine weitere Tabelle (analog der oberen) mit den wichtigsten 10 Cross-Over-Artikeln zum angewählten Artikel. Hier kann ebenfalls die Menge eingegeben und ausgewählt werden. (Kombinierte Auswahl in allen Tabellen möglich!) 

Optimierung: Beim hin-und-her-klicken zwischen den Registerkarten dürfen die Daten nur dann nachgelesen werden, wenn sie für diesen Artikel noch nicht eingelesen wurden! 

Per Shortcut < Strg +1-4 > können die Registerkarten gewechselt werden, der Cursor steht sofort in der Spalte ‚Menge‘ der ersten Zeile. Per Shortcut <Strg + 0> kann auch wieder zurück in die obere Tabelle gesprungen werden.

### Feld Suchbegriff 
Wenn Sie in einem Vorgang (Auftrag, Angebot, …) einen Teil einer Artikelnummer eingeben und sich die Artikelauswahl öffnet, wird bereits beim Öffnen der Suchbegriff in dieses Feld eingetragen. Bei Aktivierung einer der Registerkarten startet automatisch die Artikelsuche im jeweiligen Dialog. 

### Combobox Methode wählen

- Standard - Teilsuche - Ähnlichkeitssuche 
Voraussetzung ist, dass die globale Suche in den allgemeinen System-Parametern aktiviert ist.

### Feld Anzahl max. 
Über dieses Feld ist in den Suchdialogen die Anzahl der einzulesenden Datensätze limitiert. Die einzelnen Datensätze werden im Dialog entsprechend gesucht und nach Erreichen der maximalen Anzahl Datensätze abgebrochen. Auf diese Weise kann eine Suche über alle Datensätze verhindert werden, welche die gesamte Performance beeinträchtigen könnte. 

Diese Einstellung nehmen Sie im Dialog Parameter Allgemein auf der Registerkarte Allgemein vor. 

Siehe Kapitel Parameter Allgemein Registerkarte Allgemein . 

Diese Einstellung lässt sich in den entsprechenden Suchdialogen (z.B. Artikelsuche, Kundenauswahl) ändern bzw. ausschalten (ausgenommen Stammdatenprogramme). 

Tooltip: 

Anzahl Sätze, die maximal eingelesen werden.

### Tabelle Artikelliste 

In der Tabelle werden die Artikel entsprechend der Selektionskriterien angezeigt. 
### Tabellenspalte Trefferliste 
Hier werden die Treffer (mit Ziffer) angezeigt, die bei der Artikelsuche gefunden wurden. 

- Kundenartikelnummer - Fremdartikelnummer - Artikelnummer - GTIN - EAN - Einheitsnummer - EAN in Artikeleinheiten Verkauf - EAN in Artikeleinheiten Einkauf - GTIN in Artikeleinheiten Verkauf - GTIN in Artikeleinheiten Einkauf - Bestellnummer - Lucene-Suche - Leersuche. Es wurde nur mit den Inhalten der Felder gesucht. 
Für die Komponente und damit die direkte Übernahme des gefundenen Artikels wird die folgende Hierarchie verwendet: 

- Kundenartikelnummer - Artikelnummer - GTIN - EAN - Einheitsnummer - EAN in Artikeleinheiten - GTIN in Artikeleinheiten - Bestellnummer - Artikelnummer ähnlich - Suche nach der Einheitsnummer - Suche im Artikelindex 
Zu beachten: 
Wenn im Artikel „keine manuelle Erfassung“ gesetzt ist, so wird der Artikel im Vorgang nicht als Treffer aufgelistet. Die Ausnahme hiervon kann über die Berechtigungsgruppe gesteuert werden. Hier wird dann auch zwischen VK bzw. EK, je nach Vorgang, unterschieden.

### Tabellenspalte Einheit VK 
Hier wird die Verkaufseinheit angegeben. Diese wird bereits beim Einlesen der Daten grau hinterlegt, wenn nur eine Verkaufseinheit hinterlegt ist. Gibt es mehrere Verkaufseinheiten für den Artikel, so wird das Feld „weiß“ (ebenfalls bereits beim Einlesen) hinterlegt. Wenn die Verkaufseinheit grau hinterlegt ist, erkennen Sie sofort, dass eine Änderung der Verkaufseinheit nicht möglich ist. 

Als erstes wird immer die VK-Einheit angezeigt, welche als Standard VK-Einheit hinterlegt ist. Ist keine VK-Einheit als Standard hinterlegt, zieht sich das System die VK-Einheit aus dem Artikelstamm. 

Tooltip: 

Verkaufseinheit
