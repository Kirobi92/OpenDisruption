# Replikation 
Über die Replikation können Tabelleninhalte zwischen mehreren Mandanten synchron gehalten werden. Die Replikation ermöglicht es, gewisse Daten zwischen eigentlich voneinander abgeschotteten Mandanten auszutauschen, bzw. diese gemeinsam zu nutzen. 

Hierbei kann in eNVenta frei definiert werden, welche Datenstrukturen und Dateninhalte auf die einzelnen Geschäftseinheiten (Mandanten) „repliziert“ werden müssen. Dies bedeutet, dass in jeder definierten Ebene ein entsprechender Datensatz vorhanden ist, der per interner Replikation gepflegt wird. 

Zur Steuerung dieser Replikationsmechanismen verfügt eNVenta ERP über einen Verwaltungsdialog, der im Folgenden näher erläutert wird. Siehe auch: 

- Replikationsdefinition - Ausdruckseditor - Ausdruck XML - Replikationsgruppen - Status-Information - Trace - Replikationsservice - Fehler Replikations-Aufträge
