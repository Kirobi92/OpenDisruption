# Generischer IO Adapter Registerkarte Export 
Generischer IO Adapter Registerkarte Export 

Auf dieser Registerkarte definieren Sie die Einstellungen für den Export der Dateien. Die zur Verfügung stehenden Daten werden über den Transformer geladen. Hierbei stehen alle Komponenten mit Ihren entsprechenden Lade-Bedingungen aus eNVenta zur Verfügung. Ein Eingriff in die Lade-Bedingung ist hierbei nicht möglich und muss daher ggf. programmiert werden. 

Über die Einstellung X-Mapper erfolgt die Zuordnung der einzelnen Properties zum Exportob-jekt (XML, csv,…). 
### Button Exportieren 

Dieser Button führt den Export der Daten durch. Nach Abschluss erhalten Sie eine Meldung, die Sie über die Details des Exports informiert. 

Tooltip: 

Starten des Exportvorgangs für den aktuellen Adapter.

### Combobox Event 
Hier wird das Event angegeben, das vom Gateway empfangen wird. 

Um ein Event auswählen zu können, muss ein Ereignis (Event) zunächst in der Ereignisdefinition definiert und auch im verwendeten Gateway-Adapter-Kontext aktiviert werden. 

Als Datenstamm für den Export dient das Datenobjekt des ausgewählten Events. Dieses Datenobjekt kann entweder direkt per X-Mapping in den Export-Kanal übertragen oder eine Transformation dazwischen geschalten werden, die zum Event-Datensatz entsprechende Exportdaten lädt bzw. aufbereitet. 

Alternativ kann der Export auch ohne ein aufgetretenes Ereignis ausgelöst werden, in dem die Option "Ohne Event" aktiviert wird. 

In diesem Fall findet ein Export bei jedem Lauf des Adapter-Kontextes statt. Das Datenobjekt ist in diesem Fall der Konfigurationsdatensatz an sich. 

Es werden nur aktive Events angezeigt.
