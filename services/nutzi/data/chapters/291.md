# Lagerorte Registerkarte Disposition 
Lagerorte Registerkarte Disposition 

Auf dieser Registerkarte können Sie bestimmen, welche Artikel von einem Lager direkt disponiert werden (Bestellvorschlag erzeugen) und welche Artikel z.B. von einem Zentrallager beschafft werden (Bestellvorschlag wird nicht erzeugt). Dazu gibt es zum Lagerort eine Zuordnungstabelle für den Ausschluss von Lieferanten oder Artikelgruppen in der Disposition. 

In diesem Zusammenhang ist der Parameter Neue Umlagerung im Dialog Parameter Einkauf zwingend zu aktivieren. 

Siehe Kapitel Parameter Einkauf Registerkarte Allgemein . 

Wichtiger Hinweis: Beim erstmaligen Aktivieren dieser Funktionalität hat einmalig ein manueller Bestellvorschlag-Gesamtlauf mit der allgemeinen Laufnummer durchgeführt zu werden! Für jedes Lager separat! 

Beispiele für Umlagerungs-Szenarien: 

Ausgangssituation: 

Der Artikel ist in der Lagerort-Tabelle entsprechend für Umlagerung aufgelistet. Dies erfolgt wahlweise über die Artikelgruppe oder den Hauptlieferanten. Hierbei ist das Bezugslager das Zentrallager. Für das Außenlager ist die Checkbox kein BV zu setzen, um auszuschließen, dass Umlagerung UND Bestellvorschlag geschrieben werden, die Ware würde doppelt beschafft werden. 

- Beschaffungsartikel für Außen-Lager, Lagerware auf Zentrallager: Ob ein Artikel ein Beschaffungsartikel ist, bestimmt das Disopoverfahren. Dieses kann pro Lagerort eingerichtet sein. D.h. laut Dispoverfahren ist es ein Beschaffungsartikel auf dem Außenlager und am Zentrallager z.B. mit dem Dispoverfahren Deterministisch . 

Dies bedeutet, erst wenn ein Auftrag im Außenlager geschrieben wird, erfolgt die Ermittlung des Umlagerungsvorschlags (vom Zentrallager auf das Außenlager) via Bestellvorschlag. Ansonsten erfolgt keine Umlagerung auf das Außenlager, weil dieses z.B. gar nicht über die entsprechende Lagerfläche verfügt. 

Wird ein Auftrag im Außenlager erzeugt, so erfolgt bei Abschluss des Auftrags kein automatischer Bestellvorschlag. Hier wird über den Bestellvorschlag ein Umlagerungsvorschlag erzeugt. (Umlagerungsvorschläge sind vom perm. Bestellvorschlag ausgeschlossen.) Dies kann per Job-Server automatisiert werden und ist auch in der Praxis vorteilhaft, da ein eigener LKW nur 1-2 pro Tag die Ware umlagern wird. Wird nun der Bestellvorschlag für die Umlagerung generiert, so wird automatisch der Bestand auf dem Hauptlager geprüft und eine Reservierung geschrieben. Ist auf dem Hauptlager nicht genügend Bestand vorhanden, wird für das Hauptlager ein perm. Bestellvorschlag geschrieben, und die Bestellung an den Lieferanten ausgelöst. 
- Lagerware auf Außen-Lager und Zentrallager: Siehe 1., erfolgt aber nur unter der Voraussetzung, dass auf dem Außenlager nicht genügend Bestand vorhanden ist. 
- Immer Beschaffungsware bedeutet der Artikel hat in keinem Lager das Dispoverfahren gepflegt, sondern direkt im Artikelstamm das Dispoverfahren "Beschaffung" erhalten. In diesem Fall wird der Auftrag im Außenlager geschrieben, es wird wieder eine Umlagerung ausgelöst, doch diese löst wiederum eine Bestellung auf dem Zentrallager aus, da die Ware auch dort erst beschafft werden muss. Hier kann organisatorisch auch direkt beim Lieferanten aus dem Außenlager heraus bestellt werden, da dies Zeit und Kosten spart. 

Bei Änderung des Dispoverfahrens auf Beschaffungsartikel wird der entsprechende Eintrag in der Tabelle PROGNOSE gelöscht. 

Es gibt bei der Ermittlung des Bezugslagers keine Prioritäten für Hauptlieferant und Artikelgruppe. Es handelt sich um eine ODER-Prüfung. Sollten auf einen Artikel mehrere Kriterien auf mehreren Lagern zutreffen, so gewinnt der 1. Bestellvorschlag, welcher ermittelt wurde. 
Bestellungen aus einzelnen Filialen 

Damit eine Filiale eines Unternehmens einzelne Artikel selbst bestellen kann, kann hier nach einzelnen Artikeln selektiert werden. 

Einzelne Artikel von Bezugslagern können im Rahmen von Umlagerungsauftrag und Umlagerungsbestellung bezogen werden. Dafür gibt es die Kategorie Artikelnr , die Combobox Bezugslager und die Checkbox Kein Vorschlag . 

Es ist nicht möglich ein und denselben Artikel aus 2 verschiedenen Bezugslagern zu beziehen. Sollten Sie versehentlich ein 2. Bezugslager auswählen, erhalten Sie eine Meldung, die Sie darauf hinweist, dass dies nicht möglich ist. 
### Combobox Bezugslager 
Hier wird das Bezugslager angegeben (z.B. Quelllager einer Umlagerung). 

Tooltip: 

Definiert das Bezugslager ( z.B. Quelllager einer Umlagerung ) 

### Button Bezugslager übernehmen 

Mit diesem Button öffnet sich der Dialog Bezugslagerzuordnung . Dort können diejenigen Läger zugeordnet werden, die im Falle mangelnder Verfügbarkeit als mögliche Bezugsläger angeboten werden sollen. 

Tooltip: 

Übernimmt das Bezugslager in alle markierten Datensätze. 

### Checkbox Kein Bestellvorschlag 
Wenn Sie diese Checkbox aktivieren, wird für sämtliche Artikel der Tabelle kein Bestellvorschlag generiert. Diese Artikel werden immer nur über eine Umlagerung von einem anderen Lager bezogen. 

Tooltip: 

Wenn aktiviert, wird kein Bestellvorschlag generiert. 
Beeinflusst nicht, ob Umlagerungsvorschläge erzeugt werden. 

### Tabelle 

### Tabellenspalte Kategorie 
Hier geben Sie ein, um welche Kategorie es sich handelt: 

- Artikelgruppe oder - Hauptlieferant 

Tooltip: 

Wählen Sie eine Kategorie aus: entweder Artikelgruppe oder Hauptlieferant. 

### Tabellenspalte Bezeichnung 
Nachdem Sie in der 1. Spalte die ‚Kategorie‘ gewählt haben, öffnet sich mit der Tab-Taste je nach Auswahl entweder der Dialog Lieferantenauswahl oder Artikelgruppenauswahl . Hier wird dann entweder die Lieferantennummer oder die Abkürzung der Artikelgruppe eingetragen. 

Tooltip: 

Ist als Kategorie 'Hauptlieferant' gewählt, wählen Sie einen Lieferanten aus. 
Ist als Kategorie 'Artikelgruppe' gewählt, wählen Sie eine Artikelgruppe aus. 

### Tabellenspalte Wert Name 
Der Name zum Wert wird eingetragen, d.h. entweder der Name des Hauptlieferanten oder der Artikelgruppe. 

Tooltip: 

Bezeichnung zum Eintrag in der Spalte 'Wert' wird automatisch eingelesen. 

### Tabellenspalte Kein Bestellvorschlag 
Wenn Sie die Checkbox in dieser Spalte aktivieren, wird kein Bestellvorschlag generiert. Diese Artikel werden immer nur über eine Umlagerung von einem anderen Lager bezogen. 

Tooltip: 

Wenn aktiviert, wird kein Bestellvorschlag generiert. 
Beeinflusst nicht, ob Umlagerungsvorschläge erzeugt werden. 

### Tabellenspalte Bezugslager 
Aus der Combobox in dieser Spalte wählen Sie ein Bezugslager für die Disposition aus. 

Der bestehende Bestellvorschlag bleibt erhalten, um z.B. Nachbestückung aus einem ‚Handlager‘, erweiterte Dispositionen für einzelne Lieferanten mit erweiterter Karenzzeit, etc. zu erfassen. 

Dabei greift die „alte Logik“, die zusätzlich zu den bestehenden Bestellvorschlägen die manuellen Bestellvorschläge visualisiert. 

Tooltip: 

Ordnen Sie ein Bezugslager zu. 

### Button OK 

Dieser Button speichert Ihre Eingaben auf dieser Registerkarte. 

### Button Satz hinzufügen 

Mit diesem Button können Sie der Tabelle einen Eintrag hinzufügen. 

### Button Löschen 

Dieser Button löscht einen Eintrag der Tabelle.
