# Datenimport Registerkarte Allgemein 
Datenimport Registerkarte Allgemein 

Auf dieser Registerkarte definieren Sie die allgemeinen Einstellungen für den Import. 

CSV-Dateien können manuell und / oder per Job-Server importiert werden. 

Das CSV-Format ist flexibel konfigurierbar. Allerdings hat mindestens ein Pflichtfeld einer Pflichtfeldgruppe vorhanden zu sein. 

Der CSV-Auftragsimport unterstützt 2 Möglichkeiten CSV-Dateien zu verarbeiten: 

- einzelne CSV-Datei - alle CSV-Dateien in einem Verzeichnis 
Achten Sie darauf, dass Codierung, Format, Dezimal-Zeichen und Trennzeichen korrekt sind. Beachten Sie dabei die Kopfzeile und vergleichen Sie diese mit der Quelldatei. 

Die Umbenennung einer Datei findet nur im Falle eines Auftragsimports statt, bei allen anderen Komponenten kann keine Umbenennung stattfinden! 

Import einer einzelnen CSV-Datei 

Eine Möglichkeit für den CSV-Auftragsimport ist es, eine einzelne CSV-Datei auszuwählen. Diese wird verarbeitet und anschließend umbenannt, damit sie nicht noch einmal verarbeitet werden kann. 

Import aller CSV-Dateien in einem Verzeichnis 

Es kann auch manuell ein Verzeichnis angegeben werden. In diesem Fall werden alle .csv-Dateien in diesem Verzeichnis verarbeitet und die Dateien anschließend umbenannt. (Ausgenommen sind. DONE.csv und .ERROR.csv Dateien.) 

Die Dateien werden nach folgendem Muster umbenannt: 

Import erfolgreich: 
yyyy-MM-dd hh-mm ID-[AuftragsNr(n)-] [bisheriger Dateiname].DONE.csv 

Import missglückt: 
yyyy-MM-dd hh-mm [bisheriger Dateiname].ERROR.csv 

Beispiel --> Datei: SalesOrderImport.csv 

wird umbenannt in: 
"2018-11-28 12-30 ID-10000815 SalesOrderImport.DONE.csv" (1 Auftrag generiert) 
"2018-11-28 12-30 ID-10000816, 10000817 SalesOrderImport.DONE.csv" (2 Aufträge) 
"2018-11-28 12-30 ID-10000818-10000820 SalesOrderImport.DONE.csv" (n Aufträge) 
oder 
"2018-11-28 12-30 SalesOrderImport.ERROR.csv" (Kein Auftrag generiert) 

Importprotokoll in der verarbeiteten CSV-Datei 

In die verarbeitete und umbenannte CSV-Datei wird auch das Verarbeitungsprotokoll angehängt. So können Sie genau nachvollziehen, was mit einer konkreten Datei passiert ist.

Aus diesem Grund wird auch die Registerkarte Format ausgeblendet.

### Checkbox Kopfzeile 
Wenn Sie diese Checkbox aktivieren, wird die Kopfzeile ebenfalls importiert. 

### Checkbox Einfügen 
Wenn Sie diese Checkbox aktivieren, kann ein neuer Datensatz eingefügt werden. Beim Import wird automatisch über den PK (Primary Key = Schlüsselfelder zur Identifikation) der Komponente geprüft, ob der Datensatz bereits vorhanden ist. Wird ein Datensatz ohne PK importiert, kommt es zu einem Fehler. 

### Checkbox Ändern 
Wenn Sie diese Checkbox aktivieren, kann ein Datensatz geändert werden. 

### Checkbox Löschen 
Wenn Sie diese Checkbox aktivieren, kann ein Datensatz gelöscht werden. 

Das "Löschen" eines Merkmals bezieht sich auf dir konkrete Eigenschaft (Property) - in diesem Fall "Merkm. Bezeichnung" und sagt aus, dass der Wert in eNVenta durch den externen Wert (aus der CSV Datei) überschrieben werden darf, auch wenn dieser (externe Wert) leer bzw. null ist. 

Dadurch wird dieser Wert (DB-Spalte für Datensatz) "gelöscht". 

### Combobox Modus 
Der Import versucht aufgrund des abgebildeten Property-Typs den Import-Wert zu konvertieren. Ist dies nicht möglich, kann der Wert ignoriert, der Datensatz übergangen oder der Import-Vorgang abgebrochen werden. Zum Beispiel, wenn versucht wird eine Komma-Zahl als Datum einzugeben. 

Dazu wählen Sie den entsprechenden Eintrag aus dieser Combobox. 

- Wert ignorieren und fortsetzen 
Bei jedem Satz wird versucht alles einzulesen, was geht. Wenn ein einzelnes Feld bei einem Satz einen Fehler wirft, liest er die restlichen Felder trotzdem ein. 
Wenn z.B. bei einem Artikel der EK nicht eingelesen werden kann, weil er ein Wort statt einer Zahl vorfindet, liest er den Rest des Artikels ein. - Satz ignorieren und Fortsetzen 
Wenn unser Artikel-Beispiel am EK scheitert, wird der komplette Artikel verworfen und er geht gleich zum nächsten Artikel über. - Abbrechen 
Fehler führt zu Abbruch des gesamten Imports, aber das bereits importierte bleibt gespeichert. - Abbrechen und Rollback 
Fehler führt dazu, dass der komplette Import rückgängig gemacht wird und alles neu importiert werden muss.
