# Einkaufsarten Registerkarte Allgemein 
Einkaufsarten Registerkarte Allgemein 

Hier definieren Sie die allgemeinen Einstellungen für die Einkaufart. 

Bestellset 

Auf dieser Registerkarte kann die Einkaufart für Inhaltsartikel (= Artikel-Set) ausgewählt werden (Combobox EinkArtInh.Art. ). Über diese Einstellungen ist es möglich die Warenbewegung von der kaufmännischen Abwicklung zu trennen. 

Es handelt sich hierbei um eine Einkaufart, die lediglich zur Verwaltung der Inhaltsartikel eines Sets dient. Der Druck einer Bestellung oder deren Versand per EDI sind ausgeschlossen. Ebenso sind der automatische Druck und das manuelle Erfassen von Artikeln auf Basis dieser Einkaufart nicht möglich. Die Einkaufsart dient nur zur Abwicklung der Warenbewegung. 

Der Abschlussparameter ( Parameter Einkauf Registerkarte Allgemein ) ist zwingend zu aktivieren! 

Die Einkaufart für die Inhaltsbestellung muss folgende Parameter zwingend gesetzt haben: 

- EK-Aktualisierung, - Keine R’Prüfung - Ist Einkaufart Inhaltsartikel 
Enthält eine Bestellung ein Bestellset, kann dieser Bestellung keine weitere Position hinzugefügt werden. 

Grundsätzlich können Artikel-Set-Positionen in Bestellungen, Anfragen und Belastungen (Wertbelastungen) erfasst werden.

### Combobox Standardlager 
Hier wählen Sie ein Lager aus. 

Das angegebene Lager wird als Standardlager in Bestellung und Disposition automatisch übernommen, kann aber jederzeit vom Benutzer geändert werden.

### Checkbox Wareneingang/Lager 
Wenn Sie diesen Parameter aktivieren, werden beim Wareneingang die entsprechenden Mengen der einzelnen Bestellpositionen dem Lager zugebucht. Diese Checkbox ist immer zu aktivieren, außer es handelt sich wirklich um einen Vorgang ohne Lagerbuchung, wie z.B. eine Dienstleistung. 

Ist die Option nicht eingeschaltet, erfolgt keinerlei Lagerbuchung! Auch lagerführende Artikel werden dann nicht gebucht!

### Checkbox Materialabbuchung 
Dieser Parameter ist nur zu beachten, wenn Sie das Modul Produktion lizenziert haben! 

Dieser Parameter muss gesetzt werden, wenn beim Wareneingang der ersten Stufe das verbrauchte Material (dieses hat der Lieferant für die Produktion erhalten) abgebucht werden soll. 

Das Quelllager (Beistelllager des Lieferanten) darf dabei nur ein statisches Lager sein, kein chaotisches. Das Lager in der Einkaufart wird nicht gezogen, da sonst pro Lieferant/Hersteller eine Einkaufart vorhanden zu sein hat.

### Checkbox Umlagerung 
Diese Checkbox steht in Zusammenhang mit der Einkaufsart Interne Umlagerung . 

Wenn Sie diese Checkbox aktivieren, kann diese Belegart für Umlagerungsprozesse genutzt werden. So gekennzeichnete Belegarten können im Dialog Lagerorte , Registerkarte Automatisierung für interne Umlagerungen ausgewählt werden. 

Bei Nutzung von Multicompany ist das Kennzeichen für die Belegarten für interne und externe Umlagerungen in den Parametern Gesellschaft notwendig. Nur so gekennzeichnete Belegarten können dort ausgewählt werden. 

Wenn sie deaktiviert wird, erfolgt wieder ein Fallback (= Rückfall) auf alle Belegarten, sobald die Verschlüsselung von 'intern' auf 'nicht intern' wieder zurück wechselt. 

Beispiel: 
Wenn eine Einkaufsart mit Umlagerung im Bestellautomaten auf einer Position eingestellt ist, so erhält diese Position automatisch die Kennzeichnung Umlagerung . 

Bei der Generierung einer auftragsbezogenen Bestellung wird die Kennzeichnung Umlagerung an die Bestellung übergeben. 

Tooltip: 

Kennzeichnet die Belegart als Belegart für Umlagerungen. So gekennzeichnete Belegarten können im Dialog Lagerorte, Registerkarte Automatisierung für interne Umlagerungen ausgewählt werden.

### Checkbox Strecke 
Wenn Sie diesen Parameter aktivieren, können Strecken-Bestellungen bei der Erfassung der Eingangsrechnung gesondert behandelt werden. Dazu ist es notwendig den Prozess datentechnisch identifizieren zu können. 

Bei der Übernahme von Rechnungen nach eNVenta (B2B -> Rechnungsprüfung) findet im Falle von Streckenbestellungen die Anlage und Verbuchung virtueller Wareneingänge statt. 

Hierbei gelten folgende Bedingungen: 

- Lieferscheinnummer aus der Rechnung - Je Lieferscheinnummer ein Wareneingang - Die Einheiten der Rechnungsposition und der Bestellposition müssen übereinstimmen, sonst gibt es einen Fehler. - Die Liefermenge orientiert sich an der Menge aus der B2B-Rechnungsposition (Menge laut Rechnung). 
Teil-Lieferung: 

Ist im Artikelstamm das Feld Teillief. WE % gepflegt, so muss dies bei der Entscheidung, ob es sich um eine Voll- oder Teillieferung handelt, berücksichtigt werden. Hier kann ein Prozentsatz hinterlegt werden, ab dem eine Teillieferung als Volllieferung erkannt wird. D.h. die Bestellposition wird in Status 3 gesetzt und es wird keine Rückstandsposition erzeugt. 

Transaktionssicherheit: 

Es wird gewährleistet, dass sowohl die Übernahme der Rechnung, als auch das Verbuchen der virtuellen Wareneingänge innerhalb einer Transaktion ablaufen. Bei Fehlern in der Rechnungsanlage darf auch kein WE gebucht werden. 

Protokollierung: 

Relevante Informationen / Ereignisse werden protokolliert. Hierfür wird das Rechnungsprüfungsprotokoll verwendet (Button Report in der Rechnungsprüfung ). 

Ablehnung von Strecken-Rechnungen: 

Angenommen die Rechnung konnte übernommen und der virtuelle Wareneingang gebucht werden, die Rechnung soll nun aber abgelehnt werden, so muss im Fall von Streckenrechnungen dafür gesorgt werden, dass der virtuelle Wareneingang storniert wird. Dafür steht der Button Ablehnung in der Rechnungsprüfung zur Verfügung. 

Mehr Informationen zum Thema Streckengeschäft erhalten Sie im Kapitel Globalabschluss und Strecke . 

Tooltip: 

Wenn aktiviert, werden Strecken-Bestellungen bei der Erfassung der Eingangsrechnung gesondert behandelt. 

### Checkbox Keine Rechnungsprüfung 
Zu einer Bestellung mit dieser Einkaufart erfolgt keine Rechnungsprüfung, d.h. die Bestellung ist mit dem Wareneingang abgeschlossen. Z.B. Kostenlose Muster oder Ersatzlieferungen.

Wenn Sie diesen Parameter aktivieren, werden mit der Rechnungsprüfung die aktuellen EKs, Rabatte und Nebenkosten in die bei der Bestellposition eingetragene Rabatt-Kondition übernommen. Dieser Parameter kann in der Rechnungsprüfung nachträglich geändert werden. 

Tooltip:

### Checkbox EK-Aktualisierung 
Wenn Sie diesen Parameter setzen, werden die Einkaufspreise nach der Rechnungsprüfung aktualisiert. 

Tooltip: 

Wenn aktiviert, werden die Einkaufspreise nach der Rechnungsprüfung aktualisiert. 

### Checkbox Keine NK-Aktualisierung 
Wenn Sie diesen Parameter aktivieren, werden die Nebenkosten im Kundenauftrag und im Artikel nicht aktualisiert. Diese Vorgabe wird in die Rechnungsprüfung übernommen und kann dort übersteuert werden. 

Siehe Kapitel Rechnungsprüfung . 

Tooltip: 

Wenn aktiviert, dann werden die Nebenkosten im Artikelstamm nicht aktualisiert. 

### Checkbox Kein Intrastat 
Wenn Sie diese Checkbox aktivieren, erfolgt keine Berücksichtigung in der Intrastat. 

Wenn die Checkbox nicht gesetzt ist, ist zwingend ein Land anzugeben. Voraussetzung: Dieses Land ist in der Ländertabelle für Intrastat freigegeben. 

Tooltip: 

Wird nicht in der Intrastat berücksichtigt, wenn Checkbox aktiv ist.
