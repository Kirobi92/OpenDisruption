# Lagerbestandsbericht 
Lagerbestandsbericht 

Mit dem Inventory Report (Lagerbestandsbericht) werden regelmäßig Bestandsdaten für ein vorher zwischen Lieferant und Kunde vereinbartes Sortiment vom Kunden zum Lieferanten übertragen. Ziel ist es, dass der Lieferant für das vereinbarte Sortiment die Nachbevorratung seines Lagers aufgrund der mit der INVRPT Nachricht übermittelten Inhalte vornehmen wird. Eine gesonderte Bestellung ist nicht mehr nötig. 

Die Bestellungen werden im Voraus automatisch angelegt. Hierfür hat zwingend die Angabe Bestellung zu erfolgen. Dies ist die Anzahl, wie viele Bestellungen in eNVenta beim Export INVRPT automatisch auf Vorrat erstellt werden. Diese Bestellungen sind auf den Lieferanten erfasst und beinhalten keine Positionen. 

Wenn der Lieferant nun als Reaktion auf INVRPT eine ORDRSP oder DESADV sendet – er schickt uns also Ware - wird beim Import dieser Daten eine dieser leeren Bestellungen genommen und die entsprechende Artikelposition automatisch in die Bestellung geschrieben. 

Der Dialog Lagerbestandsbericht öffnet sich im Dialog EDI Teilnehmer über den Button neben dem Feld Art. Auswahl , wenn zum Lieferant der Nachrichtentyp INVRPT ausgewählt ist. 

Für diese Nachricht muss die Einkaufart angegeben werden. Hier gilt es eine Bestellart speziell für den INVRPT anzulegen und zu dieser Nachricht den jeweiligen Lieferanten einzustellen. Die Bestellung als Vorgang wird beim Wareneingang automatisch mit der hinterlegten Einkaufart angelegt, wenn der Lieferant die Ware aufgrund INVRPT liefert. 

Da es keinen direkten Vorgang gibt, der eine INVRPT für das Gateway zum Versand überträgt, wird das über einen Job erzeugt, um die Daten der Übertragung zu berechnen. Über den einzustellenden Job wird zuerst nur der Eintrag in die Gateway-Exporttabelle eingetragen. Der Job, der den EDI-Adapter auslöst, kümmert sich dann um den Versand. 

Der Versand erfolgt somit als 2-stufiger Job.

### Button Job-Server 

Mit diesem Button können Sie Jobs konfigurieren. Wenn der Job läuft, erzeugt er lediglich ein Event. Die Verabeitung dieses Events erfolgt jedoch asynchron; nämlich erst beim Lauf des EDI-Adapters. 

Siehe Kapitel Job Verwaltung . 

### Button Suche 

Dieser Button startet die Suche nach Daten; entweder ganz allgemein oder wenn Sie im Kopf des Dialogs entsprechende Selektionskriterien festgelegt haben, gezielt. 

### Button Neu 

Dieser Button leert die Daten in allen Feldern, nicht gespeicherte Änderungen gehen verloren. 

Ist in den Parametern für die Sachbearbeiter die Checkbox 2 Schritt Neu aktiviert, bleiben die Suchkriterien in den Stammdialogen erhalten. 

Siehe Kapitel Parameter Sachbearbeiter . 

### Button Speichern 

Dieser Button speichert bzw. verarbeitet die Eingaben. Vor einer endgültigen Speicherung prüft das System die Eingabe. Wenn fehlende oder fehlerhafte Daten gefunden werden, erhalten Sie eine Meldung und die Eingabe muss entsprechend korrigiert bzw. vervollständigt werden. 

### Button Neuer Satz 

Mit Klick auf diesen Button erscheint ein Dialog zur Auswahl aller Artikel, gefiltert nach dem Lieferanten als Hautlieferant, an den der Lagerbestandsbericht versendet werden soll. 

Es können auch Artikel von anderen Hauptlieferanten an den zu meldenden Lieferanten, oder auch Artikel, die keinen Hauptlieferanten besitzen, gemeldet werden. 

### Button Löschen 

Dieser Button löscht Ihre Eingaben. Genauer gesagt, die markierte Zeile wird mit dem versehen und erst endgültig gelöscht, wenn Sie den Button drücken. 

### Button Schließen 

Dieser Button schließt den Dialog. In manchen Fällen werden Sie gefragt, ob Sie die Eingaben / Änderungen noch vorher speichern möchten. 

### Button Freigeben 

Wenn Sie diesen Button drücken, wird ein Gateway-Event registriert. Dieses Event wird erst beim regulären Durchlauf eines zuständigen EDI-Adapter-Kontexts verarbeitet. 

Tooltip: 

Ein Vermerk für den Export eines Lagerbestandsberichts wird für das Gateway erzeugt. 
Der Bericht wird für den Versand per EDI freigegeben. 

### Checkbox WE Menge 
Wenn diese Checkbox aktiviert ist, wird die bestandswirksame Mengenänderung aktiviert. D.h. alle an einem Tag je Artikel durchgeführten Wareneingänge werden in der Position zusammengefasst. Der letzte, zurückliegende Wareneingang kann übertragen werden. Das Wareneingangsdatum kann zusätzlich übertragen werden. 

Tooltip: 

Wareneingangsmenge 

### Checkbox Off. Be Best. 
Wenn Sie diese Checkbox aktivieren, werden alle Lagerbestellungen des zu disponierenden Lagers zusammengefasst übertragen. Wie es zu der Bestellung kam (ob per Telefon, Fax, etc.) ist unerheblich. D.h. auch Bestellungen, die per Telefon oder Fax ausgelöst wurden, müssen übertragen werden. 

Tooltip: 

Offener Bestellbestand 

### Checkbox Warenrückstand 
Wenn Sie diese Checkbox aktivieren, ist der Warenrückstand in der reservierten Menge enthalten. Mit der Lieferung des Warenrückstands wird der gesamte vom Hersteller ermittelte Bedarf als Eillieferung geliefert werden. Der übermittelte Warenrückstand ist immer ein positiver Wert. 

Negative Werte werden nicht übermittelt. 

Tooltip: 

Warenrückstand 

### Checkbox Maximalbestand 
Wenn Sie diese Checkbox aktivieren, wird der maximale Lagerbestand übertragen. Es wird empfohlen dies nicht öfter als einmal im Monat zu übertragen. 

Tooltip: 

Maximal Bestand - Übertragung nur einmal monatlich. 

### Checkbox Mindestbestand 
Wenn Sie diese Checkbox aktivieren, wird der minimale Lagerbestand übertragen. Es wird empfohlen dies nicht öfter als einmal im Monat zu übertragen. 

Tooltip: 

Minimal Bestand - Übertragung nur einmal monatlich 

### Feld Anzahl Bestell. 
Mit einem Eintrag in dieses Feld wird festgelegt, auf wie viele Bestellnummern der Lieferant die aus dem Lagerbestandsbericht resultierenden Bestellpositionen aufteilen darf. 

### Feld Letzter Min/Max 
Dieses Feld gibt an, wann die letzte Übertragung gewesen ist. Die Mengen werden nur einmal im Monat übertragen. 

Tooltip: 

Letzte Übertragung von maximalen / minimalen Beständen.
