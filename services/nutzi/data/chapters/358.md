# Protokolldefinition 
Protokolldefinition 

Diesen Dialog finden Sie unter System >> Administration >> Protokolldefinition . Hier erhalten Sie eine Übersicht über alle überwachten Tabellen und welche Events sie überwacht (z.B. bei Update, Delete). Die Überprüfung der Tabellen erfolgt mittels Datenbank-Triggern. Diese Trigger überwachen jede Änderung, unabhängig ob dies über eNVenta ausgelöst wurde oder von extern, wie z.B. direkt per SQL-Befehl auf der Datenbank. Eine solche Protokollierung sollte nur für die wichtigsten Tabellen und Events eingerichtet werden, um z.B. eine Fehleranalyse zu unterstützen. Eine dauerhafte Protokollierung aller Tabellen und aller Aktionen durch die Anwender macht keinen Sinn und schränkt die Performance des gesamten Systems ein. 

Achtung: Bei einer Vielzahl von Triggern verlangsamt sich das System durch die Ausführungsdauer der Protokollierung pro Datenbank-Änderung und der Speicherverbrauch steigt extrem an. Der Trigger ist ein schneller Befehl, eine häufige Ausführung kann die Systemperformance trotzdem senken. Die Bereinigung der Protokollierungstabellen muss manuell durch den Administrator erfolgen. 

Die Einrichtung und Aktualisierung der Trigger kann hierbei über die Oberfläche erfolgen. 

Des Weiteren wird geprüft, ob die Trigger aktiviert sind, in welchem Status sich die Trigger befinden und welches die Keyfelder (Schlüsselfelder) sind. 

Hier erfolgt die Überwachung bzw. die Protokollierung von Feldern über bestimmte Werte auf Datenbankebene. Dabei kann der Anwender (Administrator) frei definieren, welche Daten überwacht werden sollen. Diese werden über die Checkbox Update in der Tabelle aktiviert.

Der Nachteil ist, dass sehr viele Daten gespeichert werden und dies die Laufzeit verlängert! 

Um das Ergebnis der Überwachung anzuzeigen, wählen Sie den Dialog Protokollverfolgung . Und speziell für Aufträge oder Artikel nutzen Sie die Dialoge Auftrag Cockpit oder Artikel Cockpit . 

Der Anwender (Administrator) benötigt dazu die Berechtigung, den V$SESSION View aufzurufen! 

Wenn die Checkbox SB Freigabe im Dialog Sachbearbeiter aktiviert ist, wird der letzte Bearbeiter im Trigger nicht aktualisiert bzw. berücksichtigt! 

Siehe Kapitel Sachbearbeiter . 

### Prozesse mit Zugriff auf die Oracle-DB 
Die Ermittlung der Prozesse mit Zugriff auf die Oracle-DB erfolgt über den „Type“ der Verbindung. Das bietet eine exaktere Protokollierung der Prozesse. Die Zugriffe auf die Datenbank werden hierfür in v$session entsprechend vom System geschrieben und können daher ausgelesen werden. So können auch Änderungen protokolliert werden, welche durch einen Service wie z.B. den Jobserver ausgelöst werden. 

Für die Ermittlung der eNVenta -Benutzer erfolgt die Ermittlung auch unabhängig der Prozessbezeichnung. Nur wenn kein eNVenta -Benutzer ermittelt werden kann, erfolgt der Rückfall auf den Prozess-Benutzer.

### Button Trigger 

Die Trigger werden bei vorhandener Berechtigung (der eNVenta- Sachbearbeiter hat zwingend auch Administrator auf der Datenbank sein) direkt auf dem Server angelegt und aktiviert. Dies erfolgt mit Klick auf den Button Speichern , Button Trigger und Ändern der Checkboxen Aktiv und Dokumentation Update . 

Ist keine Berechtigung vorhanden, kann der Trigger per Copy & Paste kopiert und manuell angelegt werden. 

Die Angabe der Keyfelder hat zwingend wie folgt auszusehen: 

- Keyfeld 1 = Belegnr - Keyfeld 2 = Belegart - Keyveld 3 = Fixposnr 

Tooltip: 

Aktualisiert, überprüft und synchronisiert den angezeigten Trigger auf der Datenbank. 
Die Synchronisation aktiviert den Trigger automatisch.

### Checkbox Update-Trigger synchronisieren 
Wenn die Checkbox gesetzt ist, wird das SQL-Statement des Update-Triggers mit der Datenbank synchronisiert und in der Definition gespeichert. 

Ist der Haken nicht gesetzt, wird der Trigger von der Datenbank entfernt, bleibt jedoch in der Definition bestehen. 

Die Synchronisation aktiviert den Trigger automatisch!
