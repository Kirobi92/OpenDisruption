# Funktionen im Webshop 

### Registrierung im Shop 
Wird die Registrierung im Shop angestoßen, gelangt der Benutzer auf die Seite Account . Hier kann er die für Ihn notwendigen Daten des Benutzerkontos eingeben. Hier geschieht auch die Validierung auf den Benutzernamen und im Falle des aktivierten Flags "eindeutige Emailadresse" auch, ob die Emailadresse bereits vorhanden ist. 

Bei Klick auf den Button Weiter wird der Benutzer auf die Seite Customer weitergeleitet, wenn alle Daten korrekt eingegeben sind. Hier kann der Kunde Profilinformationen anlegen. 

Die Validierung geschieht auf die fett hervorgehobenen Felder. 

Bei Klick auf den Button Weiter wird der Benutzer aufgefordert eine Adresse zu hinterlegen. 

Diese dient in erster Linie als Rechnungsadresse, welche in den Kundenstamm übernommen wird, aber auch als primäre Lieferanschrift des Benutzers. Während der Bestellung im Shop kann der Benutzer später auch eine abweichende Lieferadresse erfassen. 

Nachdem die Adresse erfasst und validiert wurde und der Benutzer auf Weiter geklickt hat, erscheint eine Übersicht, welche dem Kunden nochmals seine kompletten Daten anzeigt. 

Will der Benutzer rückwärts durch die Registrierung navigieren, so werden ihm seine vorherigen Eingaben nochmals angezeigt. Diese werden aus dem flüchtigen "cRegUser" Objekt gezogen. 

### Zuletzt gesehene Produkte 
Mit der Katalog-Navigation (linke Seite) können Sie die Produkte auswählen und ansehen. Je nachdem wie Sie die Katalogverwaltung von eNVenta ERP eingerichtet haben, wird ein Produkt im Webshop angezeigt. 

Mit dem Feature zuletzt gesehene Produkte werden immer die 3 zuletzt angesehenen Produkte angezeigt. Mit dem Pfeil, der sich in dieser Auswahl rechts befindet, ist es möglich weiter zurück zu gehen, um die restlichen Produkte, die zuletzt gesehen wurden, zu sichten. 

### Passendes Zubehör 
Der Dialog Zubehörartikel zeigt dem Kunden an, welche Artikel er zusätzlich zum Produkt kaufen könnte, die das Produkt ergänzen. So kann der Kunde beispielsweise schneller ein Zusatzprodukt finden, da hier die Möglichkeit geboten wird, dies zu hinterlegen. 

In eNVenta ERP kann im Artikelstamm unter Optionen der Zubehörartikel hinterlegt werden. Dieser ist ebenfalls noch in der Katalogverwaltung als Artikel zu hinterlegen. 

Wird kein Zubehörartikel gepflegt bzw. hinterlegt, wird dieser Dialog in der Produktansicht auch nicht aufgezeigt. 

Siehe Kapitel Zubehörartikel . 

### Artikelseite 
Die Artikelseite ermöglicht es Ihnen, Artikel im Shop anzuzeigen, die nicht offiziell im Webshop geführt werden, welche aber in eNVenta ERP im Artikelstamm gepflegt werden. 

### Crossselling 
Das Crossselling ermöglicht es Ihnen im Webshop einzusehen, welche Produkte von anderen Kunden öfters zu dem von Ihnen ausgewählten Produkt gekauft wurden. 

Die Analysedaten müssen von Hand in eNVenta gestartet werden, damit die Daten im Webshop vorhanden sind. 

### Filterung der Produktlisten 
Der Webshop bietet Ihnen die Möglichkeit Produkte zu filtern. 

Wenn eine Gruppe viele verschiedene Produkte hat, kann hier eingestellt werden wie viele Sie auf einer Seite sehen möchten. Auswählbar sind 9, 18 oder 45 Artikel pro Seite. 

Dieser Dialog ermöglicht es Ihnen, trotz eines großen Artikelsortiments einen besseren Überblick über die einzelnen Artikel zu erhalten. 

### Suchfilter 
Die Sucheinschränkung via SML (Sachmerkmalleiste) ermöglicht es nach Produkten zu filtern. 

Mit der SML(Sicht) können Sie im Webshop die Suche nach einem Produkt einschränken. 

Die hinterlegte SML–Sicht ist in diesem Beispiel in den Kategorien Diagonale, Fabrikat und Farbe hinterlegt worden. Somit können Sie die Produktsuche einschränken. Es werden alle SML im Webshop berücksichtigt. 

### Suchvorschläge 
Mit Hilfe der Suchvorschläge können Sie im Webshop einfacher nach einem Artikel suchen. Beispielsweise geben Sie die ersten zwei Buchstaben in das Suchfeld ein, dann schlägt er Ihnen alle weiteren Produkte vor, die im Webshop hinterlegt sind und mit diesen zwei Buchstaben anfangen. 

### Lesbare URL 
In der Adressleiste können Sie die angegebenen Artikel oben ablesen. 

### Paging 
Der Paging Mechanismus wurde in den Webshop mit eingebaut, um die Ladezeiten auf Grund der Preisfindung auf den einzelnen Listenseiten zu verringern. Er bildet nur eine definierte Zahl von Artikeln ab. Ebenso wird die Preisfindung nur für die angezeigte Anzahl an Artikeln vorgenommen und nicht für die komplette Artikelanzahl. 

### Gutscheine/ Gratisartikel 
Gutscheine bzw. Gratisartikel können Sie in eNVenta ERP hinterlegen. Der Dialog öffnet sich über Artikel >> Optionen >> Gutscheinartikel . Diese werden dann beim Starten des Bestellprozesses aufgeführt. 

Im Warenkorb können Sie unter der Auswahl Schnellerfassung einen Artikel mit Artikelnummer auswählen, falls Ihnen diese bekannt ist. Somit können Sie den Bestellvorgang verkürzen. 

Mit dem Button Bestellen kommen Sie in den Bestellvorgang. Mit dem Button Angebot erfragen kann sich Ihr Kunde ein Angebot zu den gewünschten Artikeln erstellen lassen. 

Unter der Option Gutschein-/Gratisartikel können Sie einen Gutscheincode einfügen und es gibt die Möglichkeit Gratisartikel zu erfassen. Diese werden dann unterhalb der Gutscheinerfassung in einer eigenen Box angezeigt. Die Anzeige dieser Gratisartikel erfolgt an Hand der ausgewählten Parameter in der eNVenta Gutscheinerfassung. 

Siehe Kapitel Gutscheinartikel . 

### Bestellprozess 
Sobald Sie einen Artikel in den Warenkorb gelegt haben, können Sie eine Bestellung durchführen. Diese Bestellung wird - wie die Registrierung - in einzelne Schritte aufgeteilt. 

In dieser Auswahl können Sie die Lieferadresse auswählen bzw. abändern. Mit dem Button Weiter --> gelangen Sie in das nächste Menü. 

In der Auswahl Versand können Sie auswählen, über welches Logistikunternehmen Ihre Bestellung versandt werden soll. 

In der Zahlungsauswahl können Sie die Zahlungsart auswählen, mit der Sie zahlen möchten. Der Kunde kann via Bankeinzug, Vorkasse oder per Rechnung bezahlen. 

Falls mehrere Bankverbindungen hinterlegt wurden, werden in der Auswahl Bankeinzug alle hinterlegten Bankverbindungen angezeigt, so dass die Bankverbindung hier nochmals auszuwählen ist. Standardmäßig wird hier die Standardbankverbindung immer als erste Option angezeigt. 

Wurde noch keine Bankverbindung hinterlegt, können Sie hier auch die Bankdaten eingeben mit BLZ, Kontonummer, IBAN und BIC. 

Haben Sie sich für eine Zahlungsmethode entschieden, können Sie den Bestellvorgang mit dem Button Weiter --> fortsetzen. 

Das Menü Bestellen gibt Ihnen einen Überblick über die Bestelldaten, die Sie davor ausgewählt hatten und über Ihre Bestellpositionen. 

Nachdem der Haken für die akzeptierten AGBs und die Widerrufsbelehrung gesetzt wurde, können Sie mit dem Button Abschicken die Bestellung abschließen. 

An die von Ihnen hinterlegte E-Mail-Adresse erhalten Sie dann eine Bestätigung zugeschickt.
