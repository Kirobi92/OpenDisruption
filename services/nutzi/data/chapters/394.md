# Aktionsdefinition Registerkarte E-Mail 
Aktionsdefinition Registerkarte Email 

Auf dieser Registerkarte definieren Sie, an wen im Fall einer Aktion eine E-Mail gesendet werden soll. Beispielsweise soll eine E-Mail (Versandbestätigung) bei Lieferscheindruck an die im Kundenstamm hinterlegte E-Mail-Adresse gesendet werden. 

Nach dem Versand eines Pakets kann dem Kunden eine Tracking-Nummer, bzw. den zugehörigen Link in einer E-Mail gesendet werden. Diese Tracking-Nummer wird im Dialog Versandarten in der Spalte 'Paketverfolgung URL' hinterlegt. Dafür lässt sich bei Bedarf auch eine entsprechende E-Mail Vorlage generieren. 

Mögliche Ereignisse (Events) für die Versendung einer Tracking-Nummer sind 
- Versand einer E-Mail über die Benachrichtigungen im Auftrag (Registerkarte Abschluss ) - Versand einer E-Mail aus dem Event Management (Gateway-Event „Lieferschein gedruckt“). 
Beim Email-Versand aus dem Auftrag heraus können die Links mit den folgenden Vorlagen (Templates) erzeugt werden: 

Plain-Text: 
{{LIST {{oObj.oSalesDocument.oSalesOrderReference.oShippingPackageColl}} {{"{{oObj.oSalesDocument.oSalesOrderReference.oShippingPackageColl:oTrackingUrl.sString}}"}}}} 

Html: 
{{LIST {{oObj.oSalesDocument.oSalesOrderReference.oShippingPackageColl}} {{"<a href="{{oObj.oSalesDocument.oSalesOrderReference.oShippingPackageColl:oTrackingUrl.sString}}">Paket {{oObj.oSalesDocument.oSalesOrderReference.oShippingPackageColl:oTrackingUrl.sStringDisplay}}</a>&#160;"}} }} 

Beim Versand aus dem Event Management, Event „Lieferschein gedruckt“ (cdSales-DocPrinted): 
Plain-Text: 
{{LIST {{oObj.oTrackingUrlColl}} {{"{{oObj.oTrackingUrlColl:sString}}"}}}} 

Html: 
{{LIST {{oObj.oTrackingUrlColl}} {{"<a href="{{oObj.oTrackingUrlColl:sString}}">Paket {{oObj.oTrackingUrlColl:sStringDisplay}}</a>&#160;"}} }} 

Im Einzelfall kann auch die Checkbox Aktiv (sie ist standardmäßig gesetzt) deaktiviert werden. Dann wird keine Aktion angestoßen. 
### Checkbox Aktiv 
Diese Checkbox ist standardmäßig gesetzt. Im Einzelfall kann sie auch deaktiviert werden. Dann wird keine Aktion angestoßen. 

### Feld Vorlage 
Hier geben Sie eine E-Mail-Vorlage ein. 

### Button Suchen 

Mit diesem Button können Sie aus dem Vorlagenstamm eine Email auswählen. 

Der Betreff und der Text sind danach nicht mehr editierbar.

### Feld von 
Hier wird die E-Mail-Adresse angegeben. Bei der Angabe mehrerer Adressen, müssen diese durch Semikolon getrennt werden. 

Beim Import von E-Mails, beim Vergleich / der Prüfung der E-Mail-Adressen, werden Leerzeichen am Ende und Beginn ignoriert. So können Absender leichter ermittelt werden und die E-Mail wird problemlos importiert. Ebenso werden bereits beim Speichern neuer oder geänderter E-Mail-Adressen automatisch die unnötigen Leerzeichen am Beginn und Ende der Mail-Adresse entfernt. 

Sammelverteiler wie z.B. Mail-Adresse Mahnung im Kundenstamm sind davon ausgenommen. 

Um sicherzugehen, dass eine E-Mail an die richtige Adresse gesendet wird, prüft eNVenta beim Speichern oder Ändern einer Mailadresse, ob die Mail-Adresse der „Norm“ entspricht. Falls nicht wird ein Hinweis ausgegeben. Dabei wird das Speichern nicht grundsätzlich verhindert. Auf diese Weise werden z.B. Outlook-Adressbucheinträge wie z.B. „#EW_ERP“ gespeichert und genutzt. Die Auswertung des Eintrags erfolgt wie gewohnt im Outlook Client. 

Dabei läuft die Prüfung ausschließlich bei Anlage und Änderung einer Mail-Adresse. Werden andere Einträge, z.B. des Kundenstammsatzes geändert, so kommt diese Meldung nicht. 

Werden Stammdaten über das Gateway importiert, so werden u.U. fehlerhafte Mailadressen importiert (ggfs. mit Warnung). Bei einem Import unter Verwendung des Jobservers erfolgt dies ebenfalls in Form einer Warnung (im Jobprotokoll).
