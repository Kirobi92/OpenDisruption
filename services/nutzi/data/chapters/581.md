# Parameter Office Registerkarte Outlook

Grundeinstellungen für eNVenta ERP / Parameter Office 

# Parameter Office Registerkarte Outlook 
Parameter Office Registerkarte Outlook 

Auf dieser Registerkarte können die Parameter für die beiden Bereiche Abgleich Wiedervorl. / Aufgaben und E-Mails-Import getrennt eingestellt werden. 
### Checkbox Outlook Online 
Wenn diese Checkbox aktiviert ist, wird der E-Mail Import und der Kontaktdatenexport über die Graph API, statt über das lokale Outlook, kommuniziert. 

Wichtige Hinweise: 
- Beim Import der E-Mails für die automatische Anlage von Calls erfolgt der Import der Nachrichtendatei als eml-Datei anstatt als msg-Datei, ggf. muss hier die Einstellung für die Archivierung angepasst werden. - Folgende Application Permissions werden in Azure Active Directory benötigt: Contacts.ReadWrite und Mail.ReadWrite - Bei der Übergabe der Kontakte werden die Faxnummern nicht in den Kontakt bei Outlook übergeben; dies sieht die aktuelle Version der Microsoft Graph Schnittstelle nicht vor. 

Tooltip: 

Ist diese Checkbox gesetzt, wird der E-Mail Import und der Kontaktenexport über die Graph API, statt über lokalen Outlook kommuniziert. 

### Checkbox Abgl. Aufgaben / WV beim Programmstart 
Wenn diese Checkbox gesetzt wird, werden beim Start von eNVenta ERP alle Aufgaben aus Outlook mit den eNVenta ERP -Wiedervorlagen abgeglichen. Noch nicht übernommene Aufgaben werden als Wiedervorlage übernommen. 

Ebenso werden bereits erledigte Aufgaben in MS-Outlook in der Wiedervorlage auf erledigt gesetzt. 

Umgekehrt werden erledigte Wiedervorlagen in MS-Outlook auf erledigt gesetzt. 

Alternativ zu diesem Parameter können Sie den Menüeintrag Outlook >> Wiedervorl. / Aufgaben abgleichen auf der Registerkarte Wiedervorlagen nutzen. 

### Feld Fälligkeitsdatum 
Bei der Generierung von Aufgaben in MS-Outlook aus eNVenta ERP wird immer ein Fälligkeitsdatum benötigt. Hier können Sie die Standardeinstellung vornehmen, in welchem Abstand dies berechnet werden soll. 

Beispiel: 

- Datum der Generierung der Wiedervorlage 05.09.2017 - Fälligkeitsdatum - Parameter: 5 Tage - Datum der Fälligkeit in MS-Outlook: 10.09.2017 

### Feld Importieren aus Ordner 
Hier können Sie eine abweichende Quelle für den Posteingang angeben. 

### Feld Verschieb. nach Ordner 
Hier geben Sie an, in welchen Ordner bereits importierte E-Mails verschoben werden sollen. 

Ein Eintrag ist Pflicht, um den doppelten Import zu verhindern. 

Eine E-Mail wird nicht importiert, wenn die Absender-Adresse mehr als einmal in der Tabelle E-Mail-Adressen vorhanden ist. 

Tooltip: 

Verschieben nach Ordner 

### Feld Max. Anz. an E-Mails 
Geben Sie hier an, wie viele E-Mails maximal importiert werden sollen. Die Sortierung ist hierbei absteigend, dies bedeutet die jüngsten E-Mails werden zuerst importiert. 

### Feld Max. Größe E-Mails (KB) 
Angabe bis zu welcher Größe die E-Mails importiert werden. 

Die Größe ist hierbei in Kilobyte anzugeben. 

Alternativ zu diesem Parameter können Sie den Menüeintrag Outlook >> E-Mails importieren auf der Registerkarte Wiedervorlagen nutzen. 

### Checkbox Import der E-Mails beim Programmstart 
Ist diese Checkbox gesetzt, werden alle E-Mails im Posteingang geprüft und bei Bedarf als Wiedervorlage und Call in eNVenta ERP importiert. Die Einstellungen, welche E-Mails übernommen werden, nehmen Sie in den nachfolgend beschriebenen Feldern Ordnername , Max. Anz. an Mails und Max. Größe Mails (KB) vor: 

### Checkbox E-Mail-Identifikation über deren Domänen 
Wenn dieser Parameter aktiviert wird, werden E-Mail-Adressen über Domänen gesucht. 

Beispiel A: 

Ein Kunde hat im Dialog Kunde in der allgemeinen E-Mail-Adresse die Adresse info@maxmustermann.de stehen. Nun kommt eine Email von support@maxmustermann.de. 

Die Adresse wird trotzdem dem Kunden zugeordnet, auch wenn die Adresse support@maxmustermann.de nirgends zugeordnet ist, da ja die Domäne „maxmustermann.de“ dem System bekannt ist. 

Beispiel B: 

Ein Kunde hat im Dialog Kunde in der allgemeinen Email-Adresse keine Adresse stehen. Aber beim Ansprechpartner „Herr Max Mustermann“ steht die Email-Adresse max.mustermann@maxmustermann.de. Nun kommt eine Email von support@maxmustermann.de. 

Die Adresse wird trotzdem dem Kunden zugeordnet, auch wenn die Adresse support@maxmustermann.de nirgends zugeordnet ist, da ja die Domäne „maxmustermann.de“ dem System über den Ansprechpartner bekannt ist.
