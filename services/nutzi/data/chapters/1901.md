# Rechnung in die Rechnungsprüfung übernehmen 
Rechnung übernehmen 

Dieser Dialog öffnet sich in der Rechnungsprüfung >> Optionen >> Rechnung übernehmen . Damit können im Business Gateway importierte Rechnungen übernommen werden. 

Für jede Position werden die entsprechenden Beträge in die Spalte 'PosBetrag' und in die Rabattfelder (des Dialogs Rechnungsprüfung ) eingetragen und bei Abweichungen rot dargestellt. 

Im Kopf des Dialogs ist die Lieferantennummer aus der Rechnungsprüfung voreingestellt. 

Der Import erfolgt, indem für jeden Lieferschein, der in der Rechnung enthalten ist, eine Suche durchgeführt wird und die entsprechenden Positionen aus dem Wareneingang eingespielt werden. 

Für jeden Lieferschein werden dann die Positionen in 3 Durchläufen abgeglichen: 
- Im 1. Durchlauf werden die Positionen ermittelt, bei denen Artikelnummer, Menge und Preis übereinstimmen. - Im 2. Durchlauf werden die Positionen ermittelt, die beim 1. Durchlauf übriggeblieben sind und bei denen Artikelnummer und Menge übereinstimmen. - Im 3. Durchlauf werden die Positionen ermittelt, die beim 2. Durchlauf übriggeblieben sind und bei denen die Artikelnummer übereinstimmt. 

### Eingangsrechnungen per Job Server überwachen 
Sie haben die Möglichkeit, elektronische Eingangsrechnungen, die noch nicht gleich freigegeben bzw. in der Rechnungsprüfung bearbeitet werden sollen, zu überwachen. In der heutigen Zeit kommt die elektronische Rechnung meist früher als die Warensendung und daher sollte nicht vergessen werden die Rechnung zu prüfen, damit diese möglichst unter Wahrung der Skonto- bzw. Zahlungsfristen bezahlt werden kann. Mittels des Job-Servers haben Sie die Möglichkeit die Überwachung systemunterstützt durchzuführen. 

Dabei ist wichtig, dass in der importierten elektronischen Rechnung die Lieferscheinnummer auf Positionsebene eingetragen ist, da sonst keine automatische Zuordnung über die Rechnungsprüfung erfolgen kann. Diese Lieferscheinnummer muss entsprechend vom Ersteller der Rechnung mitgeliefert werden. 

Die Überwachung der eingegangenen Rechnungen kann automatisch über den Job Server erfolgen. Dafür steht der Button zur Verfügung.

### Button Job-Server 

Mit diesem Button öffnen Sie die Job Verwaltung , um einen Job für die automatische Übernahme der Rechnungen zu generieren. 

Für die automatische Übernahme der Rechnung kann eine Grenze an Versuchen festgelegt werden, damit diese nicht endlos in einer Schleife hängenbleiben. Dazu können Sie eine Zeitspanne festlegen (z.B. 3 Tage nach Rechnungseingang). Konnte die Rechnung nach dieser Zeit nicht automatisch übernommen werden, wird sie an einen Sachbearbeiter zur weiteren Bearbeitung weitergereicht. 

Ebenfalls kann für die Übernahme auch die Skontofälligkeit (z.B. jeweils einen Tag vor Skontofälligkeit 1-3) oder Nettofälligkeit (z.B. 1 Tag vor Nettofälligkeit: Belegdatum (Valutadatum, wenn vorhanden) + Nettotage aus der Zahlungsbedingung des Lieferanten -1) der Rechnung als Zeitpunkt der Rechnungsübernahme festgelegt werden. Ist der entsprechende Zeitpunkt erreicht, erfolgt eine Meldung an den Sachbearbeiter zur manuellen Bearbeitung. 

Sollte eine Rechnung ihr Limit an Versuchen erreicht haben, wird diese an den entsprechenden Sachbearbeiter weitergeleitet. Beide Jobs, der für die automatische Übernahme und der für die Neuversuche, sind unabhängig voneinander konfigurierbar. Dadurch kann der EDI Import z.B. alle 30 Minuten und der u.U. aufwändige Wiederversuch 1 Mal täglich erfolgen. 

Die Parameter werden direkt im Dialog bearbeitet. Folgende Parameter und ihre Funktion lauten: 

Parameter Wert Name 
| | | |
| --- | --- | --- |
| Nur Testdurchlauf (keine Datenänderungen) | False | bisTest |
| Sachbearbeiter für die Wiedervorlagen | SYSADM | sClerkIDReminder |
| Anzahl Tage vor Warnung wegen Ablauf von Zahlungsfristen | 1 | sDaysBeforeWarning |
| Maximale Anzahl der Tage für Wiederholungsversuche | 5 | sMaxDaysTryingAssume |
| SQL-Filter für zu übernehmende Rechnungen | sSqlWhere | |

### Combobox R-Lager 
Hier wählen Sie eine Lager-ILN aus. 

Mit der Übermittlung der Lager-ILN kann gesteuert werden, an welchen Standort Ware gebracht werden soll. Deshalb ist es notwendig, dass die Lager-ILN in den Bestellungen per EDI an den Lieferanten übermittelt werden. 

Da der Großteil der Rechnungen bereits per EDI übermittelt werden, können fehlgeschlagene Rechnungsprüfungen anhand der Lager-ILN eingegrenzt werden. 

Tooltip: 

Der Lagerort, welcher mit der ILN aus der Rechnung über die Rechnungs-ILN im Lagerort ermittelt wurde.
