# Dateinamen Generator 
Dateinamen Generator 

Dieser Dialog öffnet sich im Datenexport Adapter über den Button neben dem Feld Datei , um Bestellungen per CSV zu exportieren. Voraussetzung ist, dass als Typ Bestell Export ausgewählt ist. Beim CSV-Bestellexport werden die Informationen aus dem Bestellkopf und der Bestellposition benötigt. 

Hier wird anhand von angelegten Variablen ein Muster für die Dateinamen erstellt. Die CSV-Datei kann manuell oder per Job-Server aus einer Bestellung generiert werden.

### Feld Dateinamenvorlage 
In diesem Feld wird der Dateiname angegeben. 

Wenn Dateien in regelmäßigen Zyklen über den Gateway Export Adapter exportiert werden (z.B. für den SCT Webshop) wird dem Dateinamen ein Zeitstempel (z.B. im Format yyyymmdd – dies ist individuell definierbar) mitgegeben. Dies vor dem Hintergrund, dass alte Exportdateien evtl. nochmals kontrolliert werden müssen, um die Nachvollziehbarkeit bzw. die Vollständigkeit gewährleisten zu können. 

Tooltip: 

Die Vorlage für einen Dateinamen ist die Definition eines Dateinamens, der Platzhalter enthalten kann, die zur Ausführungszeit durch den Wert der jeweiligen Variable ersetzt werden. 

Beispiel: 

Vorlage für Dateiname: {DATE}_DOC_{ID}.xml 

Wert der Variable 'DATE' = "20230405" 
Wert der Variable 'ID' = "47110815" 

Endgültiger Dateiname: "20230405_DOC_47110815.xml" 

### Tabelle 

### Tabellenspalte Variable 
Hier wählen Sie die Variable aus. Variablen vom Typ "Objekt-Eigenschaft" werden auch dafür verwendet, um die Datensätze zu gruppieren, die in eine Datei geschrieben werden. 

Tooltip: 

Name der Variable. Der Name der Variable wird in der Dateinamenvorlage innerhalb { } angegeben und zur Laufzeit durch ihren Wert ersetzt. 

### Tabellenspalte Bezugsvariable 
Tooltip: 

Wird eine Bezugsvariable gesetzt, dann bezieht sich die aktuelle Variable auf ein Property innerhalb des Objektes welches über die Bezugsvariable selektiert wird. 
Daher macht es nur Sinn, Bezugsvariablen auf Variablen zu setzen, die sich auf eine Komponente bzw. eine Kollektion beziehen. 

### Tabellenspalte Typ 
Hier wird der Typ der Variablen angegeben. 

### Tabellenspalte Eigenschaft 
Hier wählen Sie die Eigenschaft aus. Dabei ist es wichtig, dass die Eigenschaft zum Typ passt, für eine evtl. Gruppierung der Datensätze für den Export. 

### Tabellenspalte Format 
Hier wählen Sie das Datumsformat aus (z.B. JJJJMMTT).

### Feld Komponententyp 
Hier wird der Typ des Datenobjekts angegeben, welches als Datenquelle für die Erzeugung eines dynamischen Dateinamens genutzt werden kann. 

### Feld Konfig. Schl. 
Dies ist die eindeutige ID dieser Konfiguration (GUID) für den Dateinamen. Ist noch keine ID gesetzt, handelt es sich noch um einen neuen, ungespeicherten Datensatz. 

Tooltip: 

Konfigurationsschlüssel 

### Feld Export Nr. 
Dies ist ein Primary Key-Feld der DB-Table Gw_DataExport. Es ist die ID der Datenexport-Adapter-Konfiguration. Ein Wert > 0 zeigt somit zu welchem Datenexport-Adapter diese FileName-Konfiguration gehört. Da der Filename-Generator nicht nur im Datenexport verwendet wird, kann der Wert auch 0 sein. In diesem Fall dient der Konfigurationsschlüssel als Referenz-ID auf die Konfiguration.
