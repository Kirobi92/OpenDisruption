Pro Knoten in der Zuordnung kann eine Transformation eingestellt werden. Hierbei können auch mehrere Transformationen in Reihe geschaltet werden, z.B. Sonderzeichen entfernen und dann Leerzeichen entfernen. 

Im X-Mapping kann der Ausdruckseditor über eine Transformation aufgerufen werden. 

Dieser Dialog dient zum Erstellen und Bearbeiten von Ausdrücken. Je nachdem über welchen Dialog der Ausdruckseditor geöffnet wird, werden die Spalten entsprechend angepasst angezeigt. Im X-Mapping Editor kann der Ausdruckseditor jedoch nur beim Export genutzt werden. Hier existiert ein eNVenta -Objekt, welches mit Hilfe dieses Editors entsprechend dargestellt werden kann. Beim Import ist das Gegenstück ein XPath-Ausdruck. Hier gibt es kein eNVenta -Objekt, daher muss das Element entsprechend angesteuert werden. 

Der Ausdruckseditor wird genutzt, um an einem eNVenta -Quellobjekt Operationen durchzuführen und als Ergebnis einen einfachen Datentyp (String, Bool, Decimal, ...) zurück zu bekommen.

Im X-Mapper gibt es eine "Listen-Funktion" in der Transformation. 

Diese wird über Input-Parameter bedient und ermöglicht z.B. die Summenbildung über einen gewünschten Wert. 

- Parameter bestimmt die Funktion: Sum, Max, Min, Avg, Count, First, Last - Parameter bestimmt, welcher Wert aus dem einzelnen Element für die Funktion genommen werden soll. Dazu soll der Ausdruckseditor verwendet werden, der mit einem einzelnen Listen-Element arbeitet. - Parameter bestimmt einen Filter (optional), ähnlich wie beim 2. Parameter wird mittels des Ausdruckseditors pro Listen-Element ein bool-Rückgabewert ermittelt. - Parameter bestimmt eine Sortierung (optional). Diese ist nur für die Funktionen First / Last relevant. Auch hier soll mittels des Ausdruckseditors der Wert für die Sortierung "ausgewählt" werden. Damit muss die Sortierung dem jeweiligen Typ entsprechen.

Splitt Transformationen 

Es steht eine Transformation mit dem Namen „Split string“ mit nachfolgenden Optionen zur Verfügung. D.h. hier erfolgt das Splitten von strings, so dass aus einer Wortfolge, evtl. mit # getrennt, nur ein einzelner Wert genutzt wird. 

1.	Trennzeichen: Hier kann eine Zeichenkette eingegeben werden, anhand welcher der String getrennt werden soll. 

2.	Regex Ja/Nein: String wird anhand eines Regex-Statements getrennt. 

3.	Element-Nr.: Gibt die Elementnummer ein, die vom gesplitteten string verwendet werden soll. 

4.	Von hinten: Zählt die Elementnummer, beginnend vom Ende des Strings. 

5.	Bei ungültigen Werten: Gibt an, was im Fehlerfall geschehen soll. Hierbei gibt es folgende Optionen: 

1)	'error': Fehler, 
2)	'error if input not empty': Fehler, nur, wenn eingehender Wert nicht leer ist, 
3)	'output = input': der eingehende Wert wird wieder ausgegeben, 
4)	'empty': es wird ein leerer Wert zurückgegeben. 

Wenn kein gültiger Wert angegeben wurde, verhält sich die Transformation wie bei der Option 'error'. 

Beispiel: Es wird der Wert 001#002#030#040 in einem Feld der Datei geliefert. Es handelt sich hierbei um die Rabatte zu einer Rechnungsprüfung. Hier müssen nun die entsprechenden Zahlenwerte ausgelesen werden. Durch die Angabe des Trennzeichens # werden die Elemente getrennt. 

Im Folgenden muss dann das Feld mehrmals auf die entsprechenden Elemente der internen Datenstruktur gemappt werden. Beim ersten Mapping geben Sie die Element-Nr 1 ein, bei, zweiten Mapping die Element-Nr. 2 usw.
