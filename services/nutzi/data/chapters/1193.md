# EDI Auftrag auswählen 
EDI Auftrag auswählen 

Im Auftrag >> Optionen >> EDI Auftrag übernehmen öffnet sich der Dialog EDI Auftrag auswählen . 

In diesem Dialog werden die importierten ORDERS bzw. ORDCHG (Auftrag bzw. Auftragsänderungen) angezeigt. Hierbei werden nur ORDERS angezeigt, welche nicht automatisch als Auftrag angelegt wurden. Dies kann aufgrund der Prozessregeln definiert sein oder aufgrund fehlerhafter Daten. Zum Bearbeiten und zur endgültigen Übernahme kann eine Datei per Doppelklick bzw. Button geöffnet werden. 

ORDCHG-Dateien werden niemals automatisch übernommen und müssen über diesen Dialog immer nachbearbeitet werden. Dies ist notwendig, da beim Import der Daten der aktuelle Bearbeitungsstand eines Auftrags dem Kunden (Absender der ORDCHG-Datei) nicht bekannt ist, z.B. 'Lieferung hat das Lager bereits verlassen'. 

Durch die Bearbeitung der Daten wird die Ursprungsdatei nicht verändert! Die Daten werden nur innerhalb des B2B-Bereichs in der Datenbank geändert; die Originaldaten sind unverändert. 

Mit einem Doppelklick auf eine Zeile können Sie die betreffende Nachricht öffnen, um Details zu bearbeiten. 

Dieser Prozess wird nur beim Import via X-Mapper unterstützt. Bei einem Import über das EDI-Inhouse-Format gibt es die Möglichkeit der Nachbearbeitung nicht. 

Siehe Kapitel EDI Auftrag / Auftragsänderung .

### Button Suche 

Mit diesem Button starten Sie die Suche. 

Sie können ganz allgemein suchen, indem Sie den Button drücken, gleich nach Öffnen des Dialogs. Oder Sie geben im Kopf des Dialogs die Selektionskriterien vor und drücken dann diesen Button.

Siehe Kapitel Parameter Sachbearbeiter . 

### Button Auswahl 

Dieser Button übernimmt eine markierte Zeile in den zugrundeliegenden Dialog. 

### Button Schließen 

Dieser Button schließt den Dialog. 

### Button Erledigt 

Um die Protokollierung von Import und Export per EDI zu gewährleisten, können Sie nicht mehr benötigte Datensätze über diesen Button aus der Tabelle entfernen. Es erfolgt keine physische Löschung! 

Wenn der Auftrag importiert wurde, werden diese EDI-Aufträge automatisch als erledigt gekennzeichnet. Eine manuelle Kennzeichnung ist nur dann notwendig, wenn der Auftrag nicht per EDI angelegt wurde, sondern manuell (z.B. weil telefonisch zusätzlich gemeldet). 

In der Protokolltabelle wird vermerkt, wenn ein Datensatz manuell auf erledigt gesetzt ist.
