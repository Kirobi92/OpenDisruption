# Planung 
Planung 

In diesem Dialog können Sie Ihre vorhandenen Kennzahlen aus dem MIS nutzen, um zukünftige Kennzahlen zu planen bzw. zu ermitteln. 

Um die Möglichkeit des Schreibzugriffs zu nutzen, muss der SQL-Server 2008 Enterprise Edition installiert sein. 

SqlServer 2008 unterstützt nur das Rückschreiben von Werten für summierte Kennzahlen. 

Außerdem muss sich das Rückschreiben auf die feinste Granularität beziehen, also im Fall von Verkauf und Einkauf auf Positionsebene, im Fall von Finanz-Cube auf pro Konto, Tag, Kostenstelle, Kostenart, Kostenträger, etc. 

Wird eine summierte Zelle durch einen neuen Wert überschrieben, wird der neue Wert auf die untergeordneten Zellen im Verhältnis ihrer bisherigen Werte zurückgeschrieben. 

Die Summe der Elemente ergibt dann wieder den zugewiesenen Wert. Das Modell impliziert aber, dass sich ein zu planender Wert auf einen existierenden Satz an „Basisdaten“ bezieht. Wird der Wert ohne existierende Datenbasis in die Datenbank geschrieben, muss entschieden werden, wie der neue Wert auf die „potentiellen“ Datenfelder verteilt wird. Wenn die Aufteilung nicht genauer spezifiziert wird, erhalten alle verfügbaren Datenfelder einen Bruchteil der zugewiesenen Summe. Da die Anzahl der potentiellen Datenfelder aber sehr hoch ist (das Produkt aller Elemente in allen Dimensionen) kann diese Möglichkeit nicht direkt genutzt werden. 

Da eine Planungs-Dimension zu Beginn der Planung noch keine Werte besitzt, muss zunächst geprüft werden, wie die Planwerte zustande kommen sollen. In der Standard-Implementierung übernimmt diesen Job ein SSIS Package mit dem Namen „CopyPriorYearToScenario“. Dieses Package kann über den Job-Server von eNVenta ERP ausgeführt werden, wenn Sie in dem Modul Planung aus dem Menü Optionen den Eintrag „Plandaten übernehmen“ auswählen. 

Hierbei werden im DataWarehouse die Ist-Daten aus dem Vorjahr als Soll-Daten im Planjahr übernommen und anschließend die Cubes neu aufbereitet. 

Damit stehen die Ist-Daten aus dem Vorjahr als Plan-Daten für das Planjahr zu Verfügung. 

Die Zeit-Dimension muss das komplette Planjahr umfassen. 

Das Plan-Jahr steht nur zur Verfügung, wenn der Job-Server das Package erfolgreich ausgeführt hat. Abhängig vom Umfang des Datenbestands kann die Bereitstellung einige Minuten dauern. 

Sind die Daten kopiert, die Cubes erfolgreich verarbeitet und die Anzeige über den Such-Button aktualisiert, wird der Planungs-Dialog inkl. Werte dargestellt. 

Bedienung 

Im Menü Planung kann aus den Comboboxen Datenbank und Cube der zu bearbeitende Cube ausgewählt werden (wie in der Online-Analyse). 

Links wird in einem Baum eine Strukturansicht der OLAP-Datenbank angeboten. Wenn Sie ein Level oder eine Hierarchie auswählen, sind unterhalb des Baums alle in der Hierarchie / dem Level enthaltenen Elemente zu sehen. Über die Blätter-Funktionalität können weitere Elemente angezeigt werden, das Eingabe-Feld ermöglicht die Suche nach Elementen. 

Wird ein Element per Doppelklick ausgewählt, wird es automatisch als Cube-Filter gesetzt. Alle weiteren Operationen beziehen sich in diesem Fall auf die eingeschränkte Datenmenge. 

Der Cube-Filter ist die Tabelle oben rechts, in dieser werden alle gesetzten Filter angezeigt und können durch Klick auf den Button wieder gelöscht werden.

### Button Suche 

Mit diesem Button starten Sie die Suche. 

Sie können ganz allgemein suchen, indem Sie den Button drücken, gleich nach Öffnen des Dialogs. Oder Sie geben im Kopf des Dialogs die Selektionskriterien vor und drücken dann diesen Button. 

### Button Neu 

Dieser Button leert die Daten in allen Feldern, nicht gespeicherte Änderungen gehen verloren. 

Ist in den Parametern für die Sachbearbeiter die Checkbox 2 Schritt Neu aktiviert, bleiben die Suchkriterien in den Stammdialogen erhalten. 

Siehe Kapitel Parameter Sachbearbeiter . 

### Button Speichern 

Dieser Button speichert bzw. verarbeitet die Eingaben. Vor einer endgültigen Speicherung prüft das System die Eingabe. Wenn fehlende oder fehlerhafte Daten gefunden werden, erhalten Sie eine Meldung und die Eingabe muss entsprechend korrigiert bzw. vervollständigt werden. 

### Button Löschen 

Dieser Button löscht Ihre Eingaben. Genauer gesagt, die markierte Zeile wird mit dem versehen und erst endgültig gelöscht, wenn Sie den Button drücken. 

### Button Schließen 

Dieser Button schließt den Dialog. In manchen Fällen werden Sie gefragt, ob Sie die Eingaben / Änderungen noch vorher speichern möchten.
