# X-Mapping Editor Registerkarte CSV-Zeile

Werkzeuge / EDI X Mapping / X-Mapping Editor 

# X-Mapping Editor Registerkarte CSV-Zeile 
X Mapping Editor Registerkarte CSV-Zeile 

Auf dieser Registerkarte werden die Angaben für die CSV-Zeile definiert. 

In einer CSV-Zeile können auch weitere CSV-Spalten enthalten sein als fix definiert. D.h. die CSV-Zeile ist flexibler, falls z.B. der Datenlieferant das Mapping ändert. Dabei werden die in der Konfiguration nicht definierten Spalten einfach ignoriert. Trotz der Erweiterung sollte immer eine Prüfung des Mappings erfolgen, falls der Datenlieferant seine gelieferten Daten anders formatiert liefert. Ansonsten besteht die Gefahr, dass die Spalte 1 nicht mehr in Feld A landen darf, sondern in Feld B landen muss, dies aber im Mapping nicht nachgezogen wurde. 

Trotzdem besteht die Möglichkeit, dass die Spaltenanzahl fixiert werden kann, d.h. es gibt keine Abweichungen, die der X-Mapper einfach ignoriert. Auf diese Weise erhalten Sie eine Sicherheit, dass die Gesamtanzahl der Spalten immer identisch sein muss und ansonsten der Import der Daten mit entsprechender Fehlermeldung abgewiesen wird. 

Beispiel: 

- Man definiert eine Zeile mit z.B. 4 Spalten. In der Datei sind aber 6 Spalten enthalten. Die 4 definierten Spalten werden verarbeitet (d.h. sie können gemappt werden) und die anderen zwei werden ignoriert. Es werden nur die 4 Spalten importiert, welche ein Mapping besitzen, unabhängig ob es die ersten 4 Spalten sind oder an einer anderen Stelle. - Eine CSV-Zeile hat 10 Spalten. Fürs Mapping sind jedoch nur z.B. 3 Spalten relevant: die erste, zweite und die 7. Spalte. Es ist möglich nur 3 Spalten zu definieren. Bei der 3. geben Sie als Spaltennummer „7“ ein. Dadurch bezieht sich diese Spalte nicht auf die 3. Spalte, sondern gleich auf die 7. - Wird die Spaltenanzahl auf 10 fixiert, muss diese Zeile trotzdem immer 10 Spalten enthalten, auch wenn nur die 1. + 2. + 7. verwendet werden. - Wird die Spaltenanzahl offengelassen, muss die Zeile mindestens 7 Spalten oder beliebig viele weitere enthalten. 
### Button CSV-Datenstruktur übernehmen 

Tooltip: 

Zur Generierung der CSV-Datenstruktur, wird die erste Zeile als Titelzeile, d.h. als 
Zeile mit den Spaltenüberschriften getrennt durch das ";" (Semikolon) interpretiert. 
Die Spalten werden generiert und erhalten den Namen, der aus der entsprechenden 
Spalte in der ersten Zeile ausgelesen wurde. 

HINWEISE: 
- das ";" (Semikolon) darf nicht innerhalb der Spaltenüberschriften im Namen vorkommen. 
- weitere Zeilen werden genutzt um zu ermitteln, ob die Werte der jeweiligen Spalte in " " (Anführungszeichen) stehen oder nicht. 
- die zu importierende CSV-Datei darf nicht größer als 50 MB groß sein. 
Importieren Sie sonst nur die ersten Zeilen, die Sie in eine neue Datei Kopieren/Einfügen können. 

### Feld Spaltenanzahl Fix 
In dieses Feld geben Sie die Anzahl der Spalten an. 

Tooltip: 

Die CSV-Zeile muss genau die angegebene Anzahl von Spalten enthalten. 
Wenn dieser Wert leer gelassen wird, muss die Zeile nur mindestens die Spaltenanzahl aufweisen, 
die sich aus der letzten definierten CSV-Spalte ergibt. 

### Button Fehlende Spalten hinzufügen 

Mit diesem Button lassen sich die entsprechenden CSV-Felder generieren. 

### Checkbox Werte in " " 
Die Werte der einzelnen Spalten stehen in Anführungszeichen, z.B.: "Wert". 

Diese Einstellung definiert diese Option für alle Spalten der Zeile. 

Sie können allerdings für einzelne Spalten diese Option anders definieren. 

Tooltip: 

Die Werte der einzelnen Spalten stehen in Anführungszeichen, z.B.: "Wert". 

Diese Einstellung definiert diese Option für alle Spalten der Zeile. 
Sie können allerdings für einzelne Spalten diese Option anders definieren.
