# Massendruck mit mehreren E-Mail-Adressen

Grundeinstellungen für eNVenta ERP 

# Parameter Nachrichten 
Parameter Nachrichten 

Dieser Dialog öffnet sich im Navigator >> System >> Allgemein >> Parameter Nachrichten . 

Da der Versand von Nachrichten inzwischen auf den unterschiedlichsten Medien laufen kann, gibt es den Dialog Parameter Nachrichten , um die verschiedenen Anbieter/ Einstellungen zu verwalten. Hier definieren Sie die Standardeinstellungen für den Nachrichtenversand, d.h. der Provider wird ausgewählt und die entsprechenden Parameter hinterlegt. 

Die Parameterwerte können als Text, Zahl oder Datum gespeichert werden. 

Ein Provider kümmert sich um den Versand der E-Mail, nicht um deren Speicherung etc. Der Manager prüft bei der User-Interaktion, ob ein Job für den Nachrichten-Versand eingerichtet ist. Ist das nicht der Fall, wird die Nachricht sofort durch den Broker versendet. Ist dagegen hier ein Job konfiguriert, wird die Nachricht so gespeichert, dass der Job diese beim nächsten Versand mit berücksichtigt. 

Nachfolgend weitere Erläuterungen zu den Einstellungen: 

Typ E-Mail / Anbieter SMTP E-Mail Provider (linke Tabelle): 

Wenn diese Einstellung gewählt ist, werden in der rechten Tabelle weitere Eingaben notwendig: 

Übermittlung: 
Art, wie die E-Mail an den SMTP-Server übergeben werden soll. 

Netzwerk: 
Direkte Übergabe an den Server per SMTP-Protokoll. 

PickUp-Verzeichnis: 
Speicherung als Datei im Verzeichnis. Dieses wird im Feld Pfad angegeben. 

IIS-PickUp-Verzeichnis: 
Speicherung als Datei im PickUp-Verzeichnis des IIS-Servers. 

Pfad: 
Angabe des Pick-Up-Verzeichnisses, wenn die Übermittlung "PickUpVerzeichnis" ausgewählt wurde. 

Host: 
Angabe des Servers des SMTP-Servers (IP-Adresse oder Servername); meist der Exchange-Server. 

Port: 
Angabe des genutzten SMTP-Ports (in der Regel 25) 

Benutzer: 
Angabe des Benutzerkontos für die Authentifizierung am SMTP-Server 

Passwort: 
Angabe des Passworts für das Benutzerkonto. 

Domäne: 
Angabe der zu verwendenden Netzwerk-Domäne. 

SSL: 
Angabe, ob eine SSL-Verschlüsselung der Verbindung zum SMTP-Server verwendet werden soll. 

Standard Absender: 
Beim Versand über SMTP können die Bezeichnung und die E-Mail-Adresse des allgemeinen Absenders verwendet werden. Bei Verwendung von SMTP, anstatt des lokalen Outlooks, wird immer diese Adresse als Absender verwendet. 

SMTP als Standard: 
Bei Angabe "JA" wird der SMTP-Server zum Versand von E-Mails genutzt, unabhängig davon ob der Sachbearbeiter ein lokales Outlook hat. In diesem Fall gibt es auch keine Kopie im Ordner Gesendete Objekte . Zur Änderung können die Office-Parameter pro Sachbearbeiter/Arbeitsplatz genutzt werden. 

Bei Angabe "NEIN" wird das lokale Outlook zum Versand der E-Mails pro Arbeitsplatzrechner verwendet. Zur Änderung können die Office-Parameter pro Sachbearbeiter/Arbeitsplatz genutzt werden. 

Für die Anbindung von Office 365 für den Mailversand sind zwingend die Verbindungsdaten als SMTP-Daten entsprechend einzurichten! 

Hinweis: Beim Versand einer Mail erscheint eine Meldung, falls im Dialog Parameter Nachrichten ein Benutzer und ein Passwort hinterlegt sind, obwohl der SMTP-Server dies nicht unterstützt. Hier muss ggf. nach dem Update eine Anpassung der Parameter Nachrichten erfolgen, damit diese Meldung nicht beim Versand der Mails erscheint.

### Registerkarte Allgemein 

Parameter Nachrichten Registerkarte Allgemein 

Auf dieser Registerkarte kann eingestellt werden, wie die Anhänge gespeichert werden. Bei Versand von vielen E-Mails inkl. Anhängen ist die Datenbanktabelle der E-Mails oftmals die größte Tabelle und verbraucht entsprechend viel Speicherplatz. Aus diesem Grund gibt es die alternativen Speichermöglichkeiten. Bei Nutzung der alternativen Speichermöglichkeit ist der Administrator verantwortlich, für eine entsprechende Datensicherung zu sorgen. 
### Combobox Verwaltung von E-Mail Anhängen 
Der Eintrag, den Sie hier auswählen, gibt an in welcher Form die E-Mail Anhänge verarbeitet werden sollen. Folgende Einträge stehen zur Verfügung: 

- Anhänge als Datei speichern : Anhänge (Druckbelege und Extra Anhänge wie Broschüren, Werbematerial etc.) werden extern auf die Festplatte gespeichert. Hierzu kann im Feld Pfad für E-Mail Anhänge das Verzeichnis definiert werden und im Feld Zeiteinheit der Ordnerstruktur die Verzeichnisstruktur (Tag/Monat/Jahr). Der Pfad muss für jeden Benutzer erreichbar sein. - Archivierte Druckbelege nicht als Anhang in der Datenbank speichern : Sofern ein Archiv eingerichtet ist, werden nur die Druckbelege dort gespeichert. Dateien landen nicht sofort im Archiv, sondern werden über den Archiv-Pool abgearbeitet. Zwischenzeitlich werden die Dateien aber in der Datenbank gespeichert, bis die Übernahme ins Archiv erfolgt ist; danach werden sie aus der Datenbank gelöscht. - Mails und Anhänge in der Datenbank speichern (Standard) : Anhänge und Druckbelege werden wie bisher in die Datenbank gespeichert. - Mails in der Datenbank speichern, keine Anhänge : Mails werden in der Datenbank gespeichert, aber die Druckbelege sowie die Anhänge werden nicht in der Datenbank gespeichert. Somit muss ein Administrator für die extra Speicherung sorgen. - Keine Mails und Anhänge in der Datenbank speichern: Die Speicherung der kompletten E-Mails übernimmt nicht eNVenta . 
Anhänge können wie gehabt nachträglich wieder geöffnet werden. Je nachdem, ob sie sich im Archiv oder in der Datenbank befinden, wird ein anderer Dialog aufgerufen.

SSL-Verschlüsselung 
Für die SSL-Verschlüsselung stehen verschiedene Einträge zur Verfügung. Siehe nachfolgende Abbildung. 

SSL Auswahl 

Die einzelnen Optionen werden nachfolgend beschrieben: 

Keine: 
Es wird keine SSL- oder TLS-Verschlüsselung verwendet. 

Automatisch: 
Es wird versucht automatisch eine TLS oder SSL Verbindung mit dem Server auszuhandeln. Wenn der Server weder SSL noch TLS unterstützt, wird die Verbindung ohne Verschlüsselung fortgesetzt. 

SslOnConnect: 
Die Verbindung nutzt sofort die SSL- oder TLS-Verschlüsselung. 

StartTls: 
Stellt die Verbindung sofort nach der Erkennung und der Fähigkeiten des Servers auf TLS-Verschlüsselung um, aber nur, wenn der Server die STARTTLS Erweiterung unterstützt. 

StartTlsWhenAvailable: 
Stellt die Verbindung sofort nach der Erkennung und der Fähigkeiten des Servers auf TLS-Verschlüsselung um, aber nur, wenn der Server die Starterweiterung unterstützt. 

Bei der Verwendung der TLS-Verschlüsselung wird grundsätzlich die Version 1.2 bzw. größer verwendet. Sollten diese Versionen nicht verfügbar sein, werden auch die Version 1.1 bzw. 1.0 genutzt. Die Nutzung der TLS-Version hängt vom verwendeten IMAP-Service ab. 

Standard Absender 
Hier kann ein Standard-Absender für die E-Mail angegeben werden. Sobald hier ein Eintrag erfolgt, wird immer diese Absenderadresse verwendet. 

Wenn Sie beim Massendruck mehrere E-Mail-Adressen verwenden wollen, ist folgendes zu beachten: 

- Lassen Sie diese Spalte leer. - Die E-Mail Adresse des Sachbearbeiters, welcher den Massendruck / Job anlegt, wird als Absender verwendet. 
Das hat den Nachteil, dass man ggf. Sachbearbeiter nur für den Versand mit unterschiedlichen Adressen anlegen muss.

Hier wird der Typ der Nachricht angegeben, z.B. E-Mail oder Fax. 

### Tabellenspalte Anbieter 
In dieser Spalte kann der aktive Anbieter ausgewählt werden. 

Für den Nachrichtenversand stehen die Typen Fax , SMS und E-Mail zur Verfügung. Aus der Combobox in der Spalte ‚Anbieter‘ wählen Sie den entsprechenden Anbieter aus. 

Für den SMS- und Fax-Versand besteht keine verfügbare Schnittstelle, diese muss zu jedem Anbieter separat angebunden werden. 

Für den Versand von E-Mails ist es notwendig, dass der E-Mail Provider SMTP ausgewählt ist. Andernfalls funktioniert der E-Mail Versand nicht über einen zentralen Server!
