### Checkbox Factoring aktiv 
eNVenta unterstützt das verdeckte Factoring (auch stilles Factoring genannt). 

Verdecktes Factoring bedeutet, dass offene Forderungen an eine Factoringbank verkauft werden, ohne dass der Debitor darüber informiert wird. Diese Factoring-Variante wird einerseits genutzt, um die Beziehungen zu den Kunden nicht zu belasten und andererseits, um Abtretungsverbote zu umgehen. Das Mahnwesen bleibt im Unternehmen angesiedelt. 

Um am stillen Verfahren teilzunehmen hat ein Unternehmen zwingend bestimmte Voraussetzungen zu erfüllen und auch die Besonderheiten dieser Factoring-Variante sind zu berücksichtigen. 

Mit dieser Checkbox wird das Factoring freigeschaltet (aktiviert). Nach dem nächsten Start von eNVenta steht dann 
- im Navigator >> System >> Fibu die Auswahl Factoring Parameter - im Dialog Kunde die Registerkarte Factoring - im Buchen >> OP die Auswahl Factoring zur Verfügung. 

Tooltip: 

Aktiviert das Factoring

Tooltip: 

Info, ob ein OP in einem Zahlungs-, Mahn-, Verbandsabrechnungs- oder Zinsrechnungs-Vorschlag enthalten ist. 

### Checkbox Debitorenbuchhaltung Zentral 
Wenn Sie diesen Parameter aktivieren, werden bei Debitoren im Dialog OP Ausbuchung alle offene Posten aller Niederlassungen angezeigt, ansonsten nur die OPs mit der Niederlassung, die im vorausgehenden Dialog Buchen ausgewählt war. 

Wenn die Checkbox Debitorenbuchhaltung zentral gesetzt ist, greift beim Einbuchen die Niederlassung aus dem Sammelkonto und nicht die des Kunden. 

Tooltip: 

Niederlassung der Debitoren-OPs gemäß Sammelkonto 

### Checkbox Erw. KSt-/KTr-Zuord. Ausb./Rz. 
(Bedeutung: erweiterte Kostenstellen- und Kostenträgerzuordnung Skontoausbuchung) 

Beim Ausbuchen von Differenzbeträgen für Debitoren und Kreditoren (damit Skontobeträge aus Warenkäufen in die Kostenrechnung einfließen können) über den Button Skonto im Dialog OP-Ausbuchung wird die Verschlüsselung bei aktiviertem Parameter wie folgt geprüft: 

- Sachkontenstamm - Steuerschlüssel - Währung 
Es wird nur auf die erste Kostenstelle / den ersten Kostenträger aus dem Sachkonto geprüft! 

Die Prüfung im Einzelnen: 

Kostenstellen: 
mit Kostenstellen-Zwang 
- Im ausgebuchten Beleg ist keine Kostenstelle hinterlegt. - Jetzt wird geprüft, ob im entsprechenden Skonto-Konto ( Sachkontenstamm , Registerkarte Kostenrechnung ) eine Kostenstellen-Verschlüsselung hinterlegt ist. - Wenn JA, wird diese in Bezug auf den Skonto verwendet. - Wenn NEIN, wird im entsprechenden Steuerschlüssel geprüft, ob dort eine Kostenstelle hinterlegt ist. - Wenn JA, wird diese verwendet. - Wenn NEIN, wird in der entsprechend verwendeten Währung nachgeschaut, ob dort eine Kostenstelle hinterlegt ist. - Wenn JA, wird diese verwendet. - Wenn NEIN, wird die Buchung mit der bereits vorhandenen Meldung abgewiesen. 
ohne Kostenstellen-Zwang 
Ablauf wie oben, dann aber ohne Kostenstelle. 

Kostenträger: 
mit Kostenträger-Zwang 
- Im ausgebuchten Beleg ist kein Kostenträger hinterlegt. - Jetzt wird geprüft, ob im entsprechenden Skonto-Konto ( Sachkontenstamm , Registerkarte Kostenrechnung ) eine Kostenträger-Verschlüsselung hinterlegt ist. - Wenn JA, wird diese in Bezug auf den Skonto verwendet. - Wenn NEIN, wird die Buchung mit der bereits vorhandenen Meldung abgewiesen. 
ohne Kostenträger-Zwang 
Ablauf wie oben, dann aber ohne Kostenträger 

Tooltip: 

Erweiterte Kostenstellen- /Kostenträger-Zuordnung bei Ausbuchung/Rückzahlung in Skonto-/Kursdifferenz-Buchung

### Feld Kred. Vorerfass. Puffer 
Pufferzeiten ermöglichen die Kontrolle über die Einhaltung von Skonto-Fristen. 

Das Feld Kred. Vor.erfass Pufferzeit legt den Standardwert für die Pufferzeit fest, nach der eine vorerfasste Buchung zur Zahlungserinnerung vorgelegt werden soll, damit die Frist bis zum Ablauf der Skonto-Gewährung nicht versehentlich überschritten wird. 

Dieses Feld ist nur sichtbar bei eingeschalteter Kreditorenvorerfassung. 

Tooltip: 

Wert für die Kreditorenvorerfassung-Pufferzeit definieren. 

### Feld Ausz./Kassenimp. Karenzbetr. 
Hier können Standardwerte für die Auszug-/Kassenimport-Bearbeitung definiert werden. Diese Angaben werden in die Auszug-/Kassenimport-Bearbeitung übernommen und können dort temporär überschrieben werden. 

Tooltip: 

Hier können Standardwerte für den Auszug-/Kassenimport definiert werden. Diese Angaben werden in den Dialog Auszug-/Kassenimport übernommen und können dort temporär überschrieben werden. 

### Feld Ausz./Kassenimp. Karenz-% 
Hier kann die Angabe der Karenz in % erfolgen. Der Prozentsatz wird auf den Auszug-/Kassenimport-Zahlbetrag (OP-Betrag abzüglich Skonto +/- Karenzbetrag) für die Automatikfunktion angewandt. Diese Angaben werden in die Auszug-/Kassenimport-Bearbeitung übernommen und können dort temporär überschrieben werden. 

Wenn Karenzprozent und -betrag angegeben werden, so ist dies eine UND-Abfrage, also beide Kriterien müssen erfüllt sein. 

Tooltip: 

Hier können Standard-Prozentwerte für den Auszug-/Kassenimport definiert werden. Diese Angaben werden in den Dialog 'Auszug-/Kassenimport' übernommen und können dort temporär überschrieben werden.

### Checkbox Mahndaten übern. 
Wenn Sie diesen Parameter aktivieren, ist es möglich bei der Anlage eines neuen Debitoren-OPs (aus einer Debitoren-Ausgleichsbuchung) das ursprüngliche Mahndatum und die ursprüngliche Mahnstufe an den OP zu übergeben. So können Sie verhindern, falls der Rest-OP nicht rechtzeitig bezahlt wird, er nicht mit Mahnstufe 1 beginnt. So ist eine schnellere Übergabe an das Mahnwesen möglich. 

Nach Aktivierung dieses Parameters stehen im Dialog OP Anlage die beiden Felder Mahnstufe und Mahndatum zur Verfügung. 

Tooltip: 

Mahndaten aus dem Ursprungs-OP übernehmen
