# Anzahlung und Abschlagszahlung 
eNVenta ERP bietet die Möglichkeit für einen Auftrag eine Anzahlung oder Abschlagszahlung zu vereinbaren. 

Anzahlung und Abschlagszahlung unterscheiden sich von der Logik her nicht voneinander. Eine Anzahlung ist im Grunde genommen eine einmalige Abschlagszahlung. 

Nachdem die erste Abschlagsposition in den Auftrag generiert wurde, kann das Kennzeichen nicht mehr geändert werden. 

Im Dialog Parameter Verkauf , Registerkarte Allgemein kann die Checkbox Vorkasse über VZ-Gruppe aktiviert werden. So können im Auftrag Anzahlung und Abschlagszahlung verarbeitet werden. 

Vorkasse-Aufträge und Anzahlungen bzw. Abschlagszahlungen schließen sich aus. 

Anzahlung bzw. Abschlagszahlung kommen nicht in die Sammelrechnung. 

Nachdem ein Auftrag angelegt und eine Position erfasst ist, kann für Anzahlung bzw. Abschlagszahlung über den Menüpunkt Optionen >> Vorauszahlung der Dialog Vorkasse aufgerufen und die dort erfasste Einstellung in den Auftrag übernommen werden. Voraussetzung ist, dass der Auftrag bereits angelegt und zumindest eine Position erfasst wurde. 

Bei Aufruf des Menüeintrags wird zusätzlich die Option zur Aktualisierung einer bestehenden Abschlagszahlung angeboten, falls eine offene, bestehende Abschlagsposition markiert wurde.

Bei der Erfassung der ersten Abschlagszahlungsposition kann eNVenta ERP nicht „wissen“, ob es sich um einen Abschlag von mehreren noch zukünftig zu erfassenden Abschlägen oder um eine einmalige Anzahlung handelt. Daher gibt es in den Auftragsarten die Checkbox Abschlagszahlung . Diese Kennzeichnung wird in den Auftrag übernommen, so dass sie dort wahlweise gesetzt oder abgewählt werden kann. 

Die Höhe des Anzahlungs- / Abschlagsbetrags wird manuell vom Sachbearbeiter (nach Rücksprache mit dem Kunden) eingetragen. Dazu gibt es die Möglichkeit einen Zahlungsplan zu erstellen. Mit Hilfe des Zahlungsplans können Sie einem Kunden einen Report zur Verfügung stellen, welcher alle bisherigen Abschlagsrechnungen und deren entsprechenden Ausgleiche auflistet. 

Eine Anzahlung bzw. ein Abschlag wird ganz normal fakturiert und kann als solcher auch per EDI übertragen werden. 

Siehe Kapitel Abschlagszahlungen per EDI . 

#### Schlussrechnung 
Im Auftrag selber gibt es die Checkbox Schlussrechnung (Registerkarte Kopfdaten ). Diese ist nur wählbar, wenn es sich um einen Auftrag handelt, der als Abschlag gekennzeichnet wurde. Diese Kennung wird benötigt, um die letzte Rechnung zu erzeugen. Wurde die letzte Position geliefert bzw. alle Positionen können fakturiert werden, so wird die Checkbox Schlussrechnung automatisch gesetzt und damit auch die letzte Rechnung für diesen Auftrag gedruckt. 

Siehe Kapitel Auftrag . 

Wird das Kennzeichen Abschlag entfernt, wird die Checkbox Schlussrechnung unzugänglich, wird es gesetzt, so wird die Checkbox Schlussrechnung zugänglich. 

Siehe Kapitel Schlussrechnung . 

Im Fall von Anzahlung/Abschlagszahlung entfällt die Freigabe des Auftrags, da der Auftrag in diesem Fall aufgrund der Anzahlung in der Regel bereits bearbeitet wird. In diesem Fall findet eine Prüfung statt, ob es sich um einen Abschlagsauftrag handelt oder nicht. Falls ja, so wird außerdem geprüft, ob mit dem Ausgleich des Abschlags die Gesamtforderung des Auftrags beglichen wurde. Wenn dies zutrifft, ist das Kennzeichen Schlussrechnung im Auftrag zu setzen.
