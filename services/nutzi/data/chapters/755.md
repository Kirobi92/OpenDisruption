# Kalkulationsgruppen 
Kalkulationsgruppe 

Dieser Dialog öffnet sich in der Artikelkalkulation >> Optionen >> Kalkulationsgruppen . 

Zweck der Kalkulationsgruppen: 

Es müssen viele Artikelkalkulationen zum gleichen Stichtag gestartet werden, z.B. kann das Importieren eines neuen Katalogs x-fache Artikelkalkulationen mit unterschiedlichen Kalkulationsschemata nach sich ziehen. 

Beispielsweise werden Artikelgruppen mit unterschiedlichen EK Konditionen einzeln kalkuliert. Basierend auf den unterschiedlichen Einkaufspreisen (Basispreis, KEK, etc.) werden je VK-Kondition Kalkulationen für unterschiedliche Preisgruppen erstellt. 

Diese zusammengehörenden Artikelkalkulationsschemata sollen in einer Gruppe zusammengefasst werden. Die Artikelkalkulation für einen Katalog wird dann nur über die Kalkulationsgruppe angestoßen. 

Dies verschafft dem Benutzer Transparenz und eine Vereinfachung der Abläufe. 

Um eine Kalkulationsgruppe anzulegen, geben Sie die Kalkulationsgruppennummer und –bezeichnung manuell ein. 

Über die Positionsnummer wird die Reihenfolge der Artikelkalkulationen in der Kalkulationsgruppe gesteuert. 

Die Reihenfolge ist wichtig, wenn Kalkulationen fortlaufend aufgebaut werden. 

Es ist möglich eine Einzelkalkulation in der Kalkulationsgruppe zu definieren. Ziel ist es, einzelne Artikel sollen innerhalb eines Kalkulationsschemas oder einer Kalkulationsgruppe nachkalkuliert werden können. Dies ist notwendig, da täglich neue Artikel angelegt werden oder einzelne Preise von Datenlieferanten nachkorrigiert werden. 

In diesem Zusammenhang wird die Begriffsdefinition wie folgt festgelegt: 

Artikel Kalkulationsgruppe 
Die bereits existierende Artikel-Kalkulationsgruppe fasst diejenigen Artikel zusammen, welche mit einer Artikelkalkulation kalkuliert werden sollen. 

Kalkulationsschema 
Das bereits existierende Kalkulationsschema definiert Vorlagen, um einzelne Artikelkalkulationen zu definieren. 

Kalkulationsgruppe 
Die neue Kalkulationsgruppe fasst diejenigen Artikelkalkulationen zusammen, welche aus Gründen der einfacheren Handhabung zusammen ausgeführt werden sollen.

### Bezeichnungsfeld Kalkulationsgruppe 
Tooltip: 

In diesem Dialog können Artikelkalkulationen in Gruppen zusammengefasst werden. 
Für mehr Informationen drücken Sie F1. 

### Button Form-Menü 

Mit Klick auf diesen Button werden je nach Kontext und Berechtigung die unterschiedlichen Optionen des jeweiligen Dialogs hinter dem Button zur Auswahl angeboten. 

Die Funktionen für das Client Side Customizing sowie die Online-Hilfe stehen über diesen Button ebenfalls zur Verfügung.

### Button Job-Server 

Mit diesem Button gelangen Sie in die Job-Verwaltung, um eine zeitgesteuerte Kalkulation für mehrere Artikelkalkulationen anzustoßen. 

Siehe Kapitel Job Verwaltung .

Dieser Button steht zur Verfügung, um Dokumenten-Links sowie URL‘s als Info in eNVenta ERP zu hinterlegen oder um 2 Vorgänge/Referenzen miteinander zu verknüpfen.

### Button Drucken 

Dieser Button startet den Druck. Sind keine Daten eingelesen, erhalten Sie eine Meldung. 

### Button Suche 

Dieser Button startet die Suche nach Daten; entweder ganz allgemein oder wenn Sie im Kopf des Dialogs entsprechende Selektionskriterien festgelegt haben, gezielt. 

### Button Neu 

Dieser Button leert die Daten in den Kopf-Feldern. 

Ist in den Parametern für die Sachbearbeiter die Checkbox 2 Schritt Neu aktiviert, bleiben die Suchkriterien in den Stammdialogen erhalten. 

Siehe Kapitel Parameter Sachbearbeiter . 

### Button Kopieren 

Dieser Button startet die Kopierfunktion. Es werden sämtliche Felder kopiert, welche keine Schlüsselfelder sind. 

### Button Speichern 

Dieser Button speichert Ihre Eingaben. 

### Button Vorheriger Satz 

Mit diesem Button können Sie rückwärts blättern. Die Reihenfolge beim Blättern bezieht sich auf die Registerkarte Tabelle , sofern diese vorhanden ist. 

Der Button ist nur aktiv, wenn in der Trefferliste (Tabelle) mehrere Einträge vorhanden sind bzw. eingelesen sind. 

### Button Nächster Satz 

Mit diesem Button können Sie vorwärts blättern. Die Reihenfolge beim Blättern bezieht sich auf die Registerkarte Tabelle , sofern diese vorhanden ist. 

Der Button ist nur aktiv, wenn in der Trefferliste (Tabelle) mehrere Einträge vorhanden sind bzw. eingelesen sind. 

### Button Löschen 

Dieser Button löscht Ihre Eingaben. Genauer gesagt, die markierte Zeile wird mit dem versehen und erst endgültig gelöscht, wenn Sie den Button drücken. 

### Button Schließen 

Dieser Button schließt den Dialog. In manchen Fällen werden Sie gefragt, ob Sie die Eingaben / Änderungen noch vorher speichern möchten. 

### Button Alle kalkulieren 

Mit diesem Button werden alle Artikelkalkulationen dieser Kalkulationsgruppe nacheinander durchgeführt. Es erscheint eine entsprechende Abfrage, die Sie mit Ja beantworten. 

Tooltip: 

Es werden alle Artikelkalkulationen dieser Kalkulationsgruppe nacheinander durchgeführt. 

### Button Job starten 

Über diesen Button werden alle Artikelkalkulationen dieser Kalkulationsgruppe (im Hintergrund über den Job Service) nacheinander durchgeführt.
