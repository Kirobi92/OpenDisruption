# Kostenträger 
Kostenträger 

Der Kostenträger beschreibt einen bestimmten Ausgabenzweck, d.h. wofür verschiedene Arten von Ausgaben (= Kosten) im Unternehmen angefallen sind. Dies sind in aller Regel die Endprodukte, denen mit Hilfe der Kostenrechnung die verschiedenen Kosten zugerechnet werden. 

Kostenträger können als Artikel oder Artikelgruppen aus der Warenwirtschaft von eNVenta ERP übernommen werden. Natürlich können auch selbstdefinierte Kostenträger angelegt werden. Dies können z.B. auch Kunden oder Projekte sein. 

### Ändern/Löschen eines Kostenträgers 
Sie können die im vorigen Kapitel beschriebenen Einstellungen zu Kostenträgern jederzeit ändern und Ihre Änderungen mit dem Button speichern. Beachten Sie dabei die im vorigen Kapitel zu den einzelnen Einstellungen beschriebenen Konsequenzen. 

Das Löschen eines Kostenträgers mit dem Button ist nur möglich, solange der Kostenträger noch nicht in der Kostenrechnung gebucht wurde. Wurden schon Buchungen ausgeführt, akzeptiert das Programm die Löschung nicht. 

Das Löschen wird ebenfalls abgelehnt, wenn zu dem Kostenträger noch offene Verkaufsvorgänge existieren.

Angabe der Kostenträgernummer 

### Checkbox Gesperrt 
Wenn Sie diese Checkbox aktivieren, ist der Kostenträger gesperrt und wird im Buchen Kore nicht mehr angezeigt. Ein manuelles Buchen in der Kostenrechnung ist dann nicht möglich und eine Übernahme wird prinzipiell verweigert; es erscheint eine Fehlermeldung. 

Tooltip: 

Aktiviert: Setzt den Datensatz auf 'gesperrt'. 

### Feld Bezeichnung 
Dieses Feld wird nach Eingabe einer Kostenträgernummer automatisch mit dem Namen gefüllt. 

Wenn Sie einen neuen Kostenträger anlegen, geben Sie hier die passende Bezeichnung an. 

Tooltip: 

Angabe der Bezeichnung 

### Feld Kürzel 
Hier können Sie dem Kostenträger ein Kürzel zuweisen. 

### Feld Gültig von 
Über das Datum, das Sie hier eintragen, wird der Bearbeitungszeitraum eingeschränkt. 

Sind Datumswerte gefüllt, wird das Buchungsdatum auf diesen Zeitraum geprüft. 

Beim Setzen der Datumswerte Gültig ab und Gültig bis wird auf die Datenbank-Tabelle KORE_JOURNAL geprüft, ob nicht bereits Buchungen außerhalb dieser Datumswerte vorhanden sind, die kumuliert einen Wert ≠ 0.00 ergeben. 

### Feld Gültig bis 
Wenn Sie hier ein Datum eintragen, wird der Bearbeitungszeitraum eingeschränkt. 

Sind Datumswerte gefüllt, wird das Buchungsdatum auf diesen Zeitraum geprüft. 

Beim Setzen der Datumswerte Gültig ab und Gültig bis wird auf die Datenbank-Tabelle KORE_JOURNAL geprüft, ob nicht bereits Buchungen außerhalb dieser Datumswerte vorhanden sind, die kumuliert einen Wert ≠ 0.00 ergeben. 

### Feld Erfassung
