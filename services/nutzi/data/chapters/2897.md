# X Mapping Struktur 

In der Mitte des Dialogs wird die Mapping-Struktur angezeigt. 

Bei der Mapping Struktur handelt es sich ebenfalls um einen Baum von Knoten und Unterknoten. Ein Mapping-Knoten verbindet eine Objekt-Eigenschaft und einen Datei-Knoten miteinander und definiert die Mapping-Optionen wie Transformationen etc. 

Der Ablauf des Mappings erfolgt von oben nach unten (erstes Mapping wird zuerst gemappt, letztes als letztes). 

Der Button Test öffnet den Dialog X Mapping Test , um das Mapping zu prüfen. 

Das Symbol definiert die Map-Richtung. 

Das Symbol definiert eine Schleife. 

Siehe Kapitel X Mapping Test . 

### Spezifische Elemente 
Wenn ein Element mehr als einmal vorkommen kann (=Liste), kann es nötig sein, ein bestimmtes Element daraus zu "spezifizieren", d.h. zu definieren, welches Element aus der Liste gemeint ist. Zum Beispiel: 

- Das erste, zweite, dritte oder x-te Element in der Liste - Das letzte, vorletzte, drittletzte oder x-letzte Element in der Liste - Das Element, bei dem ein bestimmtes Attribut oder Element einen bestimmten Wert enthält. 
Ein "Spezifisches Element" gleicht also einer SQL-Bedingung auf einer Tabelle, bei der nur bestimmte Sätze selektiert werden. Da potentiell mehrere Elemente den Bedingungen entsprechen können, muss zusätzlich immer eine Einschränkung auf die Nummer der Elemente erfolgen: z.B. erstes, zweites, letztes, vorletztes … Element.
