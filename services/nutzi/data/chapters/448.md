# Protokolle Aufräumen Registerkarte Allgemein 
Protokolle Aufräumen Registerkarte Allgemein 

Auf dieser Registerkarte definieren Sie die Anzahl Tage für die Datensätze, die gelöscht werden sollen und das Export-Setting für einzelne Komponenten. 
### Feld Alter in Tagen 
Als Filter wird eine Anzahl an Tagen angeben, die das Alter der Daten festlegen, die bereinigt werden sollen. 

Wird eine Lösch-Konfiguration gespeichert, bei der die Tage so gewählt sind, dass extrem viel gelöscht wird (> 50.000), während gleichzeitig das Protokoll aktiviert ist, erhält der Nutzer eine Warnung. 

Tooltip: 

Es werden nur Einträge gelöscht, die älter sind als die hier angegebene Anzahl von Tagen. 

### Checkbox X-Mapper Export 
Wenn Sie diese Checkbox aktivieren, kann ein Export-Setting für einzelne Komponenten generiert werden. Auf Basis des X-Mapping können die aus der Datenbank zu löschenden Daten in ein beliebiges Datenformat exportiert/gesichert werden. Die Combobox X-Mapping ID und das Feld Export Pfad werden zugänglich und die entsprechenden Werte können eingegeben werden. 

Tooltip: 

Hiermit kann eine Datei erstellt werden, welche die gelöschten Einträge enthält. Die Struktur dieser Archiv-Datei wird über den Xmapper festgelegt. Falls aktiviert, werden ein Pfad zur Ausgabe sowie ein Setting benötigt. 

### Combobox X-Mapping ID 
Hier wird der Identifizierer eines Mappings angegeben, das mit dem X-Mapper erstellt wurde. 

Tooltip: 

Identifizierer eines Mappings, das mit dem X-Mapper erstellt wurde. 

### Button Öffnen 

Mit diesem Button verzweigen Sie in den X-Mapping Editor . 

### Feld Export Pfad 
Hier wird der Serverpfad angegeben. 

Tooltip: 

Angabe des Serverpfads 

### Button Suche 

Mit diesem Button öffnet sich der Dialog Ordner auswählen , um den gewünschten Export Pfad zuzuweisen.
