# eNVenta Init Registerkarte Import 
eNVenta Init Registerkarte Import 

Auf dieser Registerkarte haben Sie die Möglichkeit, eNVenta ERP zu initialisieren. Dazu geben Sie in den oberen Feldern Ihre Firmendaten, Währung, Geschäftsjahr, Monate des Geschäftsjahrs, Saldovortragskonten und Sammelkonten ein. 

Die Felder Währung , Geschäftsjahr , Erster Monat und Letzter Monat sind unbedingt zu füllen. Dabei ist zu beachten, dass die Währung zum eingestellten Land passen muss und dass Sie den Anfangs- und den Ende-Monat Ihres Geschäftsjahres eingeben (welches bekanntlich vom Kalenderjahr abweichen kann). 

Wenn die Initialisierung zu einem späteren Zeitpunkt nochmals durchgeführt wird, überschreibt dies alle bisherigen Einstellungen im oberen Teil. Die Input Items sind nicht betroffen. 

Der untere Teil der Registerkarte ist unabhängig vom oberen Teil. 
### Button Start Init 

Über diesen Button kann die Initialisierung gestartet werden. Wenn die Eingaben nicht in Ordnung sind, werden Sie mit einer entsprechenden Fehlermeldung darauf hingewiesen. 

Dieser Vorgang kann jeweils einige Minuten andauern, da über die Import-Items ganze Tabellen neu angelegt und gefüllt werden.

### Feld E-Mail 
Hier wird die E-Mail-Adresse angegeben. Bei der Angabe mehrerer Adressen, müssen diese durch Semikolon getrennt werden. 

Beim Import von E-Mails, beim Vergleich / der Prüfung der E-Mail-Adressen, werden Leerzeichen am Ende und Beginn ignoriert. So können Absender leichter ermittelt werden und die E-Mail wird problemlos importiert. Ebenso werden bereits beim Speichern neuer oder geänderter E-Mail-Adressen automatisch die unnötigen Leerzeichen am Beginn und Ende der Mail-Adresse entfernt. 

Sammelverteiler wie z.B. Mail-Adresse Mahnung im Kundenstamm sind davon ausgenommen. 

Um sicherzugehen, dass eine E-Mail an die richtige Adresse gesendet wird, prüft eNVenta beim Speichern oder Ändern einer Mailadresse, ob die Mail-Adresse der „Norm“ entspricht. Falls nicht wird ein Hinweis ausgegeben. Dabei wird das Speichern nicht grundsätzlich verhindert. Auf diese Weise werden z.B. Outlook-Adressbucheinträge wie z.B. „#EW_ERP“ gespeichert und genutzt. Die Auswertung des Eintrags erfolgt wie gewohnt im Outlook Client. 

Dabei läuft die Prüfung ausschließlich bei Anlage und Änderung einer Mail-Adresse. Werden andere Einträge, z.B. des Kundenstammsatzes geändert, so kommt diese Meldung nicht. 

Werden Stammdaten über das Gateway importiert, so werden u.U. fehlerhafte Mailadressen importiert (ggfs. mit Warnung). Bei einem Import unter Verwendung des Jobservers erfolgt dies ebenfalls in Form einer Warnung (im Jobprotokoll). 

Tooltip:

Bei der Angabe mehrerer Adressen, müssen diese durch Semikolon getrennt sein.

### Checkbox Nur Import Items 
Wird diese Checkbox aktiviert, werden die Einstellungen im Firmenstamm und die bestehenden Html Reports nicht verändert und nur die Daten aus dem Import-Schema importiert, wenn der Button Start Init geklickt wird. 

Siehe Kapitel Import Item Auswahl .

In diesem Feld werden die Import Items angezeigt. 
### Tabellenspalte Name

Tooltip: 

Angabe einer Beschreibung 

### Button Satz hinzufügen 

Über diesen Button können Input-Items hinzugefügt werden (es öffnet sich der Dialog Import Item Auswahl ). 

Die Input-Items werden farblich dargestellt: 

- Grün = es sind bisher keine Datensätze vorhanden. - Rot = es existieren bereits Datensätze. 
Standardmäßig werden von Nissen & Velten Software GmbH zwei Vorlagen für SKR03 und SKR04 mitgeliefert. 

### Button Löschen 

Über diesen Button können bereits ausgewählte Input-Items wieder vom Init ausgeschlossen werden.
