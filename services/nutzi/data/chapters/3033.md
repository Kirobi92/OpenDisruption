# Replikationsdefinition 
Replikationsdefinition 

In diesem Menü werden die zu replizierenden Tabellen festgelegt. Sie finden das Menü im Navigator unter System >> Mandanten >> Replikationsdefinition . 

Für jeden Mandanten kann in diesem Dialog definiert werden, welche Tabellen repliziert werden. Dabei können Sie auf Feldebene bestimmen, ob der Feldinhalt beim Insert oder Update repliziert werden soll. 

Ebenso können Sie hier Regeln definieren, die steuern, ob eine Replikation erfolgen soll oder nicht. 

Beispiel: 
Sie können definieren, dass die Replikation einer Tabelle nur bei Artikeln stattfinden soll, die mit R beginnen. So kann z.B. jeder Mandant zusätzlich noch seine eigenen Artikelnummern (in einem bestimmten Nummernkreis) anlegen und pflegen, ohne dass die anderen Mandanten diese Artikelstammsätze sehen würden. 

Im Fall der Replikation ist nun der Artikelstammsatz für jeden Mandanten einmal verfügbar und wird über die Replikation für die definierten Felder synchron gehalten. 

Wenn eine Tabelle nicht zu 100% gemeinsam genutzt werden kann, da diverse Informationen unterschiedlich sind, müssen diese, bis auf die unterschiedlichen Felder, synchronisiert werden. 

Z.B. Artikelstamm. Viel gemeinsames, aber jede Geschäftseinheit hat einen eigenen Bereich oder unterschiedliche Preise usw. 

Wenn Daten aus mehreren Instanzen in einen „virtuellen Mandanten“ kopiert werden müssen, steht ebenfalls die Funktion „Replikation“ zur Verfügung. 

Prinzipiell werden definiert: 

- die Tabelle - der Quell-Mandant - der Ziel-Mandant - explizit die betroffenen Datenfelder - ob die Felder bei einem Insert und/oder Update reagieren sollen - Bedingungen festlegen (allgemein, bei Insert oder bei Update), wann eine Reaktion stattfinden soll - Bedingungen für Manipulationen (bei Insert oder Update) und die Manipulation an sich (hier wird von einer Transformation gesprochen) 
Dabei stellt das System sicher, dass keine Zirkelbeziehung bzw. eine „Dauerschleife“ entsteht. 

Wählen Sie als nächstes aus, von welchem Mandanten die Replikation auf einen anderen Mandanten übertragen werden soll. 

### Replikation zur Laufzeit 
Die Verarbeitung der Replikation läuft asynchron über einen Service, d.h. die Verarbeitung der Änderung in einem Mandanten wartet nicht bis alle entsprechenden Replikationsaufträge verarbeitet sind, sondern die Verarbeitung der Replikationsaufträge findet zeitversetzt und unabhängig davon statt. 

### Transformationen 
Neben den beschriebenen Ausdrücken können in einer Replikationsdefinition bereits Datentransformationen definiert werden. So besteht die Möglichkeit für die Ereignisse ‚Insert‘ und ‘Update‘ jeweils eine Transformation mittels des Formeleditors anzulegen. Bei der Verarbeitung von Replikationsaufträgen zur Laufzeit werden die definierten Transformationsregeln dann angewendet. 

Über eine Transformationsregel kann z.B. folgendes Szenario abgebildet werden: 

In dem einen Mandanten ist der Artikel in der Artikelgruppe „XY“ ein Reichweitenartikel, in allen anderen Mandanten ist dies ein Beschaffungsartikel. Dennoch sollen die Artikel gemeinsam genutzt werden (müssen also repliziert werden). So wird für den Insert (Artikelanlage) und das Update (Änderung) eine Transformationsregel angelegt. Diese definiert, dass das Dispositionsverfahren von Artikeln, deren Artikelgruppe mit 10 beginnt, auf den Wert 5 gedreht wird. Bei den restlichen Artikeln erfolgt keine Transformation der Dispositionsmethode.

## Menüeinträge 

### Menü Bearbeiten 
Hier gibt es einige Menü-Punkte, die es ermöglichen mehrere Spalten gleichzeitig zu bearbeiten. 

### Menü Bearbeiten / Auswahl Spalten 
Die beiden Felder "Insert" und "Update" werden verändert. 

- Alle neuen: alle Tabellen-Spalten, die als "Neu" gekennzeichnet sind, werden gesetzt. - Alle: alle Tabellen-Spalten werden gesetzt. - Keine: alle Tabellen-Spalten werden deaktiviert. - Umkehren: bei allen Tabellen-Spalten wird der Schalter umgesetzt; ‚Gesetzt‘ wird zu ‚deaktiviert‘ und ‚deaktiviert‘ wird zu ‚gesetzt‘. - Alle zurücksetzen: bei allen Spalten wird der Schalter auf den zuletzt gespeicherten Wert zurückgesetzt. Die definierten Ausdrücke bleiben davon unberührt. 

### Menü Bearbeiten / Auswahl Spalten (Insert) 
- Es wird nur das Feld "Insert" verändert. - Menü-Punkte wie oben: Es wird jedoch nur der "Insert"-Schalter geändert. - = Update: die Insert-Schalter werden gleich den Update-Schaltern gesetzt. - = gesetzte Updates: Bei allen Tabellen-Spalten, bei denen der Update-Schalter gesetzt ist, wird auch der Insert-Schalter gesetzt. Die anderen Tabellen-Spalten bleiben unverändert. 

### Menü Bearbeiten / Auswahl Spalten (Update) 
- Es wird nur das Feld "Update" verändert. - Menü-Punkte wie oben: Es wird jedoch nur der "Update"-Schalter geändert. - = Insert: die Update-Schalter werden gleich den Insert-Schaltern gesetzt. - = gesetzte Inserts: Bei allen Tabellen-Spalten, bei denen der Insert-Schalter gesetzt ist, wird auch der Update-Schalter gesetzt. Die anderen Tabellen-Spalten bleiben unverändert. 

### Menü Bearbeiten / Spalten aktualisieren 
Über diesen Menüeintrag können Sie die Tabelle prüfen. Nicht vorhandene Spalten werden so in die Tabelle eingefügt. 

Im Anschluss daran muss der Sachbearbeiter diese neuen Spalten prüfen. Über den Button (Suche, Sortierung) entscheidet der Sachbearbeiter dann, wann eine Replikation stattfinden soll.
