# Mobile Datenerfassung

# Mobile Datenerfassung 
Moderne Mobile Datenerfassungsgeräte (im Folgenden als Scanner bezeichnet) unterstützen kein Java und können damit nicht den Java Client von eNVenta konsumieren. 

Aus diesem Grund stellt eNVenta einen HTML Client zur Verfügung, welcher direkt im Browser ausgeführt werden kann bzw. eine Android Mobile Client App, welche auf Android basierenden Scannern installiert werden kann. 

Die Darstellung und Bedienung eines vollständigen eNVenta auf diesen Endgeräten wäre aber sowohl von der Bedienung als auch von der Prozesssteuerung nicht optimal. Aus diesem Grund wurden für die Scanner optimierte Masken geschaffen. 

Genauere Informationen zum HTML Client und der FS Mobile Client App finden sie unter folgendem Link: 

FS Dokumentation des HTML Client 

Durch diese Fokussierung auf den HTML Client wurden Listendarstellungen mit dem List View Control realisiert. Dies verhindert eine Darstellung im Java Client. 

Bitte beachten Sie dies bei Ihren Test. 

Benutzen Sie immer die Application Mobile Scanner und verwenden Sie als Client den HTML Client oder die FS MobileClient Application. 

Programmbereiche, die mit dem Scanner optimal unterstützt werden können, werden in diesem Bedienkonzept realisiert. (z.B. LVS Prozesse, Produktionsrückmeldung ...) Näheres zum Bedienkonzept und den Prozessen die angeboten werden, finden Sie unter diesem Dokumentationspunkt. Siehe auch: 

- Bedienkonzept - Allgemeine Dialoge - Lagerprozesse - Basisform und Funktionen
