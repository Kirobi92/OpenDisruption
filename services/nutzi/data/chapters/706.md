# Lagerbestand (Lager pro Artikel) 
Lager pro Artikel 

Dieser Dialog öffnet sich über den Button Lager im Artikelstamm. Hier können für einen Artikel pro Lager / Lagerplatz unterschiedliche Melde-, Mindest- und Maximalbestände gepflegt werden. 

Die Einträge in diesem Dialog haben eine höhere Priorität als die Melde-, Mindest- und Maximalbestände aus dem Artikelstamm. 

Ein Artikellager kann nicht gelöscht werden, solange Bestand auf diesem Lager ist!

Tooltip:

### Tabellenspalte Lagersp. GEK 
Hier wird der gewichtete mittlere Einkaufspreis angegeben, der bei der Rechnungsprüfung neu errechnet werden kann. Dieser wird erst bei Abschluss einer Inventur berechnet anhand der neuen Menge 'Inventur IST' und dem (bei Positionen, falls gewünscht) neuen 'Bew. EK'. 

Dieser berechnete Wert wird dann in den 'GEK, Lager' zurückgeschrieben. 

Der GEK wird wie folgt berechnet: 

(((Buchbestand – noch nicht rechnungsgeprüfter WE) x alter GEK) + (Rechnungsprüfungsmenge x neuer EK))) / (Buchbestand – noch nicht rechnungsgeprüfter WE + Rechnungsprüfungsmenge) 

Hinweis: Als Alternative zum gewichteten Einkaufspreis gibt es auch den lagerspezifischen GEK. 

Die Errechnung des GEKs findet auf der Datenbank per SQL statt. 

In diesem Zusammenhang beachten Sie bitte auch die Informationen zur Checkbox Alle Lager im Dialog Lagerorte . 

Tooltip: 

Lagerspezifischer gewichteter mittlerer Einkaufspreis

### Tabellenspalte Verf. Losgröße 
Mit Hilfe der Combobox in dieser Spalte ist es möglich, ein global vorgegebenes Losgrößen-Verfahren optional auf Lager+Artikel-Ebene zu übersteuern. 

### Tabellenspalte Max. Sollreichw. 
Hier wird die errechnete Sollreichweite angegeben. Diese kann bedarfsgerecht angepasst werden. 

Rechnet eNVenta auf Basis der gegeben Werte eine Sollreichweite aus, die über der definierten maximalen Sollreichweite liegt, dann wird die Sollreichweite auf die maximale Sollreichweite begrenzt. Ist in den Parametern Einkauf, am Lagerort, beim Artikel als Losgrößenverfahren „bis Maximalbestand“ angegeben und es gibt einen Maximalbestand am Lager und Artikel, so definiert dieser die maximal zu bestellende Menge für dieses Lager. Das wird bei der Generierung des Bestellvorschlages dahingehend berücksichtigt, dass die maximale Menge am Lager die maximal zu bestellende Menge für dieses Lager einschränkt, unabhängig von der Sollreichweite. 

Eine mathematisch sinnvolle Losgröße nutzt nichts, wenn sie aus Platzgründen nicht eingelagert werden kann. 

Hinweis: Der Bestellautomat kann optional auf VPE runden. Wird der Maximalbestand durch die VPE-Rundung überschritten, so ist dies zulässig. 

Tooltip: 

Wenn hier ein Wert in Tagen vorgegeben wird, dann wird eine in der Reichweitenoptimierung 
berechnete Sollreichweite ggf. auf diesen Wert gekürzt. 

### Feld Anzahl max. 
Über dieses Feld ist in den Suchdialogen die Anzahl der einzulesenden Datensätze limitiert. Die einzelnen Datensätze werden im Dialog entsprechend gesucht und nach Erreichen der maximalen Anzahl Datensätze abgebrochen. Auf diese Weise kann eine Suche über alle Datensätze verhindert werden, welche die gesamte Performance beeinträchtigen könnte. 

Diese Einstellung nehmen Sie im Dialog Parameter Allgemein auf der Registerkarte Allgemein vor. 

Siehe Kapitel Parameter Allgemein Registerkarte Allgemein . 

Diese Einstellung lässt sich in den entsprechenden Suchdialogen (z.B. Artikelsuche, Kundenauswahl) ändern bzw. ausschalten (ausgenommen Stammdatenprogramme). 

Tooltip: 

Anzahl Sätze, die maximal eingelesen werden.
