# Fehler Replikations-Aufträge 
Fehler Repl.-Aufträge 

Der Button Fehlerh. Repl. im Replikationsservice öffnet den Dialog Fehler Repl.-Aufträge . Hier können nun ein oder mehrere Aufträge markiert und über den Button Ereignis wdh . wiederholt werden. Der Auftrag wird kopiert und erhält den Status 5 (neuer Status = wiederholt, damit nachvollzogen werden kann, dass es sich um einen fehlerhaften und wiederholten Auftrag handelt). 

Der kopierte Auftrag erhält den Status 1 (offen für Bearbeitung) und wandert wieder in den Dialog Replikationsservice wo er erneut verarbeitet wird. 

Es werden nur die Aufträge angezeigt, die für die aktuelle Geschäftseinheit bestimmt sind. 
## Steuerelemente des Formulars 

### Bezeichnungsfeld Fehler Repl.-Aufträge 
Tooltip: 

In diesem Dialog können die fehlerhaften Replikations-Aufträge analysiert werden. 
Für mehr Informationen drücken Sie F1. 

### Button Form-Menü 

Mit Klick auf diesen Button werden je nach Kontext und Berechtigung die unterschiedlichen Optionen des jeweiligen Dialogs hinter dem Button zur Auswahl angeboten. 

Die Funktionen für das Client Side Customizing sowie die Online-Hilfe stehen über diesen Button ebenfalls zur Verfügung.

### Button Suche 

Mit diesem Button starten Sie die Suche. 

### Button Löschen 

Mit diesem Button können fehlerhafte Aufträge gelöscht werden. Dazu werden die entsprechenden Aufträge selektiert und mit diesem Button gelöscht. 

Dabei werden diese Aufträge in die interne Historie verschoben und erhalten eine entsprechende Kennung. 

### Button Schließen 

Dieser Button schließt den Dialog. In manchen Fällen werden Sie gefragt, ob Sie die Eingaben / Änderungen noch vorher speichern möchten. 

### Button Ereignis wdh. 

Über diesen Button können ein oder mehrere Aufträge markiert und wiederholt werden. Der Auftrag wird kopiert und erhält den Status 5 (neuer Status = wiederholt, damit nachvoll-zogen werden kann, dass es sich um einen fehlerhaften und wiederholten Auftrag handelt). 

Der kopierte Auftrag erhält den Status 1 (offen für Bearbeitung) und wandert wieder in den Dialog Replikationsservice wo er erneut verarbeitet wird. 

### Checkbox Alle Repl.-Aufträge anzeigen 
Wenn diese Checkbox aktiviert ist, werden alle Replikationsaufträge angezeigt. 

Außerdem wird der zuständige Sachbearbeiter bei fehlgeschlagenen Replikationsaufträgen informiert. 

In diesem Zusammenhang steht auch die Spalte 'Auftr. Status' zur Verfügung, um die Einträge entsprechend zu filtern und zu sortieren. 

### Tabelle 

In dieser Tabelle werden die Aufträge aufgelistet. Dabei stehen die ältesten Aufträge oben. 

Wenn Sie einen Auftrag in der Tabelle selektieren, werden unter der Tabelle die Details zu dem Auftrag angezeigt. 

Bei der Selektion mehrerer Aufträge werden keine Details angezeigt.
