# Betriebskalender 
Kalender Schicht 

Der Betriebskalender erlaubt das Anlegen und Führen beliebig vieler Kalender. Der allgemeine übergeordnete Kalender (Basiskalender) eines Unternehmens kann Arbeitszeiten, Feiertage und Pausenregelungen sowie betriebliche Besonderheiten enthalten. Außerdem können spezifische Kalender für Ressourcen und Ressourcengruppen angelegt werden. Zudem können Schichten, die länger als 24.00 Uhr dauern, entsprechend abgebildet werden. 

Es muss immer mindestens 1 Kalender angelegt sein (Basiskalender mit Nummer 0). Pro Kalender wird die Arbeitszeit angegeben. Die Arbeitszeiten mehrerer Kalender werden aufsummiert, d.h. ein Basiskalender kann angelegt werden. So müssen nur noch abweichende Angaben erstellt werden. 

Pro Kalender können mehrere Schichten angegeben werden. Es muss immer wenigstens eine Schicht angegeben werden. 

Die Verwaltung der Kalender ist allgemein gültig und kann für verschiedene Zwecke verwendet werden. 

Beispiel: 

- Disposition von Monteuren oder Fahrzeugen im technischen Kundendienst (Servicemodul) - Kapazitätsterminierung in der Produktion (Kapazitätsstellen und Betriebsmittel) - Raumbelegungen 
Damit bei der Terminierung von Ressourcen in der Fertigung der richtige Kalender verwendet wird, muss jeder Ressourcengruppe ein Kalender zugeordnet werden. 

Um Kapazitätsveränderungen in einem Simulationsszenario durchspielen zu können, wird in allen Tabellen des Kalenders eine Simulationsnummer mitgeführt. In dieses Feld wird grundsätzlich eine 0 eingetragen, solange es sich nicht um eine Simulation handelt. 

Jeder Kalender wird für ein Jahr angelegt und kann später für weitere Jahre erweitert werden. Dazu muss wieder ein neuer Datensatz mit der gleichen Kalendernummer und einem abweichenden Jahr angelegt werden. Dabei ist es möglich, die Einstellungen eines anderen Jahres zu übernehmen. 

Das Löschen eines Kalenders ist nur möglich, wenn keine Schicht zu diesem Kalender mehr existiert und auch keine Zuordnung zu einer Ressourcengruppe. 

### Basiskalender / Kalender Nr. 0 
Um die Verwaltung von Kalendern zu erleichtern gibt es die Möglichkeit, einen Basiskalender anzulegen. Dieser Kalender bekommt die Kalendernummer 0. Alle im Basiskalender gemachten Einträge gelten grundsätzlich auch in allen anderen Kalendern, solange kein kalenderspezifischer Eintrag existiert. In anderen Kalendern müssen also nur abweichende Einträge gemacht werden. 

Ist einer Ressourcengruppe kein Kalender zugeordnet, gilt der Basiskalender. 

### Betriebskalender anlegen und definieren 
Zur Definition eines neuen Betriebskalenders sind folgende Eingaben notwendig: 

- Anlegen eines Kalenders mit einer neuen Kalendernummer und dem dazugehörenden Jahr. - Anlegen einer oder mehrerer Schichten (ohne Schichtbetrieb nur eine Schicht). - Anlegen von Wochentagen, wenn bei bestimmten Tagen die Arbeitszeit abweicht (z.B. Samstag, Sonntag). - Anlegen von allgemeinen Pausenregelungen, die für jeden Tag gleichermaßen gelten. - Anlegen von Abweichungen, die nur für einen Kalendertag / eine Schicht gelten, z.B. Feiertag. - Anlegen von abweichenden Pausen, die für einen Tag / eine Schicht gelten. 
Die Schritte 1 und 2 sind zwingend einzugeben. 

### Betriebskalender kopieren 
Ein Kalender kann mit allen anhängenden Daten (Wochentag, Schichten, Pausen, abweichenden Tagen und abweichenden Pausen) kopiert werden. Dazu öffnen Sie das Menü Stamm >> Kalender .

### Feld Schicht 
Hier geben Sie eine Nummer für die Schicht ein. Dies ist ein Pflicht-Feld! 

Die Schichtnummer hat pro Kalendernummer und Schicht eindeutig zu sein. 

Jeder Kalender/Jahr muss mindestens eine Schicht beinhalten. Dies ist auch erforderlich, wenn es im Betrieb keine Schichtenregelung gibt. In diesem Fall wird die Arbeitszeit/Kapazität mit nur einer Schicht abgebildet. 

Tooltip: 

Angabe der Schicht

Hier wird der Name der Schicht angegeben. 

Tooltip: 

Name der Schicht 

### Feld Zeit von (hh:mm) 
Hier geben Sie den Beginn der Schicht (ohne Pausen) ein. 

### Feld Zeit bis (hh:mm) 
Hier geben Sie das Ende der Schicht ein. 

Sonderfall: 

Eine Schicht kann sich auch in den nächsten Tag hineinziehen. In diesem Fall ist die Zeit Bis kleiner als die Zeit Von . 

Tooltip: 

Zeigt die Zeit-bis an.
