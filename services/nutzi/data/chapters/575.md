### Feld Wiedervorlagen 
In diesem Feld kann angegeben werden, in welchem Abstand die Wiedervorlage aktualisiert werden soll. Die Zeit wird in Sekunden angegeben. Wenn Sie hier nichts eingeben, werden automatisch 30 Sekunden eingestellt. 

Das Check-Intervall kann angegeben werden zwischen 1 und 32767 Sekunden. Die Wiedervorlagen werden in diesem Intervall geprüft und angezeigt, wenn ein neuer Eintrag hinzugekommen ist. 

Tipp: 
Wenn jemand sehr viele Wiedervorlagetermine in der Liste hat, ist es sinnvoll, hier einen möglichst hohen Wert zu verwenden (120-300 sec.). Das minimiert die Laufzeit. 

Tooltip: 

Intervall für die Aktualisierung der Termine im Navigator-Bereich. (Sekunden)

### Feld Verlauf Anzahl 
Diese Zahl gibt an, wie viele Einträge von demselben Typ angezeigt werden sollen. 

Der Wert in Verlauf Anzahl darf nicht größer sein, als der Wert in Verlauf Total . Es wird ggf. nach unten korrigiert. 

Tooltip: 

Diese Zahl gibt an, wie viele Einträge von demselben Typ angezeigt werden sollen. 

### Feld Verlauf Total 
Diese Zahl gibt an, wie viele Einträge insgesamt angezeigt werden sollen. 

Werden falsche Werte eingegeben, werden diese beim Speichern automatisch korrigiert: 

- Verlauf Anzahl: min 1 - max 100 - Verlauf Total: min 1 - max 500 
'Verlauf Anzahl‘ darf nicht größer sein als ‚Verlauf Total‘. Es wird ggf. nach unten korrigiert. 

Tooltip: 

Diese Zahl gibt an, wie viele Einträge insgesamt angezeigt werden sollen.

### Feld Liz.Warn. Tage 
In diesem Feld geben Sie an, wie viele Tage vor Ablauf einer Lizenz gewarnt werden soll: 

- nichts angegeben - 30 Tage - 0 - keine Warnung - Anzahl der Tage 
Wenn eine der Package-Lizenzen die Grenze unterschreitet, kommt nach der Anmeldung an eNVenta ERP eine Hinweismeldung. 

Beispiel: Achtung! Einige Lizenzen laufen in 25 Tagen ab. Bitte überprüfen Sie Ihre Lizenzen oder wenden Sie sich an Ihren Administrator. 

Tooltip: 

Hier geben Sie an, wie viele Tage vor Ablauf einer Lizenz gewarnt werden soll.

### Combobox Dialog-Position 
Der Java-Client speichert die Positionen und Größen der Dialoge. Über eine Form-Action kann in dieser Combobox folgendes Verhalten ausgewählt werden: 

- nichts speichern: es werden keine Positionen gespeichert - absolut: d.h. alle Positionen werden gespeichert (Bildschirm-Koordinaten) - relativ zum Haupt-Dialog : relative Positionen speichern (Koordinaten bezogen auf den Haupt-Dialog) 
Die Informationen werden immer gespeichert, egal welcher Modus gerade aktiv ist. Der Modus steuert nur, ob die gespeicherten Informationen ausgewertet werden. So kann es z.B. vorkommen, dass der Client für einen Dialog eine „exotische" Position speichert. Wenn der Benutzer dann die Logik aktiviert, wird diese „exotische" Position „plötzlich" wiederhergestellt. 

Der Client merkt sich für den Dialog nicht nur den Namen des Forms, sondern die Dokumenteneinstellung. D.h. die Einstellung hängt vom Benutzer ab, funktioniert also erst nach dessen Anmeldung. Außerdem spielt es eine Rolle, aus welchem Kontext heraus der Dialog geöffnet wurde. So werden z.B. für die Artikel-Auswahl, je nachdem von wo aus sie geöffnet wurde (Auftrag, Angebot, Bestellung, CallCenter, etc.), andere Angaben gespeichert. 

Beim Zurücksetzen des Layouts werden die gespeicherten Informationen verworfen. 

Standardmäßig wird der Modus nichts speichern gezogen. So muss der Benutzer diese Funktion aktivieren, wenn er sie nutzen möchte. 

Tooltip: 

Dialog-Position: 
Wählen Sie aus, wie Positionen und Größen der Dialoge gespeichert 
werden sollen. Zur Verfügung stehen: 
• Nicht speichern: es werden keine Positionen gespeichert

Koordinaten)

### Checkbox 2 Schritt Neu 
Wenn Sie diese Checkbox aktivieren, bleiben die Suchkriterien in den Stammdialogen erhalten. D.h.beim 1. Klick auf den Button bleiben die Suchkriterien erhalten. Der Button wird gekennzeichnet und zwar so: . In diesem Fall können Suchkriterien modifiziert werden, um eine neue Suche zu beginnen. Wenn Sie nun nochmals auf den Button klicken, bleiben die neuen Suchkriterien erhalten. Erst beim 2. Klick auf werden die Suchkriterien komplett gelöscht. 

Auch wenn ein gefundener Datensatz verändert und gespeichert wird, bleiben die Suchkriterien erhalten. 

Suchkriterien im Kopfbereich als auch auf den Registerkarten der Stamm-Dialoge bleiben erhalten. 

Ist die Checkbox nicht aktiv, werden die Suchkriterien bei Klick auf den Button geleert. 

Siehe Kapitel Standardbuttons . 

Tooltip: 

Wenn Sie diese Checkbox aktivieren, bleiben die Suchkriterien in den Stammdialogen erhalten. 

### Checkbox Alte TAB-Steuerung bei Langtext-Feldern 
Wenn Sie diese Checkbox aktivieren, kann jeder Sachbearbeiter für seine Arbeit die Logik für die Tabulator-Taste einstellen. Diese Funktion kann auch während der laufenden Anwendung eingestellt werden. 

Funktionsweise dieser Checkbox: 

- Nicht gesetzt (Standardwert): mit TAB / UMSCHALT+TAB wird zum nächsten / vorherigen Feld gesprungen; mit CTRL+TAB wird ein Tab-Zeichen geschrieben. - Gesetzt: mit CTRL+TAB / UMSCHALT+CTRL+TAB wird zum nächsten / vorherigen Feld gesprungen; mit TAB wird ein Tab-Zeichen geschrieben. 

Tooltip: 

Wenn Sie diese Checkbox aktivieren, kann jeder Sachbearbeiter für seine Arbeit die Logik für die Tabulator-Taste einstellen.
