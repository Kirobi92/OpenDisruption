# Zahlungsarten im Webshop 
Zahlungsarten 

Damit eine Zahlungsart im Shop verwendet werden kann, ist es notwendig, dem Kunden ein bestimmtes Zahlungsprofil zu hinterlegen. Um dies zu vereinfachen, können direkt im Webshop >> Optionen >> Zahlungsarten die Zahlungsprofile für den Kunden angelegt werden. 

Die Zahlungsarten werden für den Webshop unabhängig von der Zahlungsbedingung des Kunden definiert. Die Zahlungsarten werden über Profile organisiert. Ein Profil kann mehrere Zahlungsarten beinhalten. Die Zahlungsarten können pro Profil mit einer Zahlungsbedingung verschlüsselt werden. 

Beispiel: 
Profil "Privatkunde" 
-> Zahlungsart "Rechnung" -> Bedingung "1: Zahlung netto sofort" 

Profil "Geschäftskunde" 
-> Zahlungsart "Rechnung" -> Bedingung "2: Zahlung netto 15 Tage" 
### Auswahl des Zahlungsprofils: 
Dem EComm-Benutzer kann direkt ein Zahlungsprofil zugewiesen werden. Diese Zuweisung hat die höchste Priorität. Zudem kann dem Zahlungsprofil ein "Filter" hinzugefügt werden. Der Filter muss auf die Eigenschaften des Kunden zutreffen. 

Beispiel: 
Profil "Privatkunde DE" -> Filter Kunde: "Privatkunde = 1 AND Land = 'DE' 

Über diesen Filter ist eine dynamische Zuordnung von Profilen zu Kunden möglich. Es werden nur Profile mit Filterbedingung ausgewertet. 

Wenn kein passendes Profil gefunden wird, gibt es pro Webshop immer ein Standard-Profil "Standard", auf welches dann zurückgegriffen wird. Dieses Standard-Profil enthält als Vorbelegung eine Zahlungsart "Rechnung", mit der 1. gefundenen Zahlungsbedingung für Rechnungen. 
#### Einschränkungskriterien: 
- Land (z.B.: Alle Kunden aus dem Land "DE") - Privat (z.B.: Nur für Privatkunden) - Zahlungsbedingung (z.B.: Alle Kunden mit Zahlungsbedingung X ) 
Die Zahlungsarten Paypal und Kreditkarte werden nicht unterstützt! Diese Zahlungsarten können über das Customizing hinzugefügt werden.

### Combobox Zahlungstyp 
Aus dieser Combobox können Sie einen Zahlungstpyen auswählen. 

Damit ist es möglich, Zahlungsprofile pro Zahlungstyp erstellen zu können. Das Profil zieht, wenn dem Kunden eine Bedingung zugewiesen wurde, welche dem Zahlungstyp entspricht. 

Die Zahlungstypen in eNVenta sind: 

- Rechnung - Nachnahme - Bankeinzug - Vorkasse 
Der Zahlungstyp wurde für die Bedingungen eindeutig bestimmbar gemacht. Wenn einem Kunden eine Zahlungsbedingung hinterlegt ist, kann über die Zahlungsbedingung der Zahlungstyp ermittelt werden. Das Profil bietet als Filter die Zahlungstypen an. Damit kann ein Profil für alle Bedingungen dieses Typs angelegt werden. Existiert ein Profil für eine Zahlungsbedingung, hat dieses höhere Priorität als das Profil für den Typ. 

Wenn in der Combobox Zahlungstyp ein Eintrag vorhanden ist, wird der Eintrag aus dem Feld Zahlungsbedingung entfernt. 

Ist ein Zahlungstyp ausgewählt, werden die Zahlungsbedingungen auf diesen Typ gefiltert. 

Die Prioritätenreihenfolge ist:
