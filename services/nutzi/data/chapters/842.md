# Zusätzliche Anschriften 
Zusätzliche Anschriften 

Zusätzlich zu den bereits angelegten Versandanschriften können weitere Adressen des Kunden hinterlegt werden. Dafür steht der Dialog Zusätzliche Anschriften zur Verfügung. Dieser Dialog öffnet sich im Kunde >> Stamm >> Zusätzliche Kundenadressen , wenn die Kundendaten in den Dialog eingelesen wurden. Damit lassen sich z.B. Anschriften für verschiedene Zwecke anlegen. Beispiel: Anschrift für Angebote (der Adressat hat sein Büro evtl. in einem ausgelagerten Verwaltungsgebäude). 

Dieser Dialog steht auch in Auftrag, Angebot, Gutschrift, Retoure über Stamm >> Zusätzliche Anschriften zur Verfügung. In diesem Zusammenhang wurden die genannten Dialoge um die Registerkarte Zusätzliche Adresse ergänzt. 

Pro Vorgang darf nur ein Adresstyp hinterlegt werden. 

Im Kopf des Dialogs werden neue Anschriften definiert. Im Bereich Adresstyp wählen Sie den gewünschten Adresstyp aus. Folgende Adresstypen stehen zur Verfügung: 

AN = Angebotsanschrift 
BA = Bestellanschrift 
AB = Auftragsbestätigungsanschrift 
LA = Lieferanschrift 
RA = Rechnungsanschrift 
RP = Rechnungspostanschrift 

Was ist die Rechnungspostanschrift? 

Eine Rechnungspostanschrift wird erforderlich, wenn zwar die Rechnung auf die Firma (z.B. Konzern Mustermann in PLZ / Ort) laufen muss, die Abteilung oder die Firma, welche die Rechnungen erhält und bearbeitet u.U. eine andere Firmenbezeichnung und eine andere Postanschrift hat. 

Wird eine Adresse vom Typ „RA“ und/oder „RP“ eingefügt, wird automatisch der Auftragskopf für die Rechnung ergänzt, so dass pro RA und/oder RP eine Rechnung erstellt wird. Damit wird eine Sammelrechnung umgangen. 

Die zusätzlichen Anschriften müssen manuell/per Customizing in den Reports hinzugefügt werden! 

Beim Kopieren des Auftrags werden nachwievor alle Adressen aus den Stammdaten neu ermittelt. 

Beim Bestellen aus dem Auftrag ändert sich nichts; nur bei Streckenlieferung wird die Lieferanschrift an die Bestellung übergeben. 

Es können alle Adressen in die Zusatztabelle eingefügt werden, außer den Lieferanschriften. Die Lieferanschriften werden in der Registerkarte Anschriften ausgewählt und können dort ggfs. angepasst werden. 

Die abweichende Rechnungsanschrift hat keine Auswirkung auf das Mahnwesen. Die Mahnungen laufen immer auf den Kundenstammsatz, nicht auf die abweichenden RA oder RP. 

Für die Rechnungsanschrift gibt es kein MwStKZ / UstID/ ILN, da diese Adresse reine Adressstammdaten darstellt. Angaben zur juristischen Herkunft sind im Kundenstammsatz zu hinterlegen.

BA = Bestellanschrift

Die zusätzlich angelegten Adressen werden dem Report zur Verfügung gestellt. 
### Checkbox (AN) Angebotsanschrift 
Wenn Sie diese Checkbox aktivieren, wird das Angebot an diese Adresse gesendet. 

Ansonsten hat diese Adresse keine weitere spezielle Funktion. 

Tooltip: 

Angebotsanschrift (AN): 
Abweichende Anschrift, an welche das Angebot gesendet wird. 
Ansonsten hat diese Adresse keine weitere spezielle Funktion. 

### Checkbox (BA) Bestellanschrift 
Wenn Sie diese Checkbox aktivieren, wird die Bestellung direkt an den Arbeitsplatz des Bestellers gesendet. 

Ansonsten hat diese Adresse keine weitere spezielle Funktion. 

Tooltip: 

Bestellanschrift (BA): 
Abweichende Anschrift, wo der Besteller sitzt. 
Ansonsten hat diese Adresse keine weitere spezielle Funktion. 

### Checkbox (AB) Auftragsbestätigungsanschrift 
Wenn Sie diese Checkbox aktivieren, wird die Auftragsbestätigung an diese Adresse gesendet. 

Ansonsten hat diese Adresse keine weitere spezielle Funktion. 

Tooltip: 

Auftragsbestätigungsanschrift (AB): 
Angabe, wohin die Auftragsbestätigung gesendet wird. 
Ansonsten hat diese Adresse keine weitere spezielle Funktion. 

### Checkbox (RA) Rechnungsanschrift 
Wenn Sie diese Checkbox aktivieren, wird die Rechnung an diese Adresse gesendet. 

Diese abweichende Adresse hat keine Auswirkungen auf das Mahnwesen, da Mahnungen immer auf den Kundenstammsatz laufen und nicht auf abweichende Rechnungsanschriften. Dies gilt auch für die ZM u.v.a. 

Für die Finanzbuchhaltung sind die Rechnungsanschrift und Lieferanschrift aus den Kopfdaten maßgebend. 

Tooltip: 

Rechnungsanschrift (RA): 
Unterschiedliche Rechnungsanschriften zu einer juristischen Person. 
ACHTUNG: Die abweichende Rechnungsanschrift hat keine Auswirkung auf das Mahnwesen. 
Die Mahnungen laufen immer auf den Kundenstammsatz, nicht auf die abweichende RA! Ebenso ZM u.v.a. 
Relevant für die Finanzbuchhaltung ist die Rechnungsanschrift und Lieferanschrift aus den Kopfdaten! 

### Checkbox (RP) Rechnungspostanschrift 
Wenn Sie diese Checkbox aktivieren, wird die Rechnung an diese Rechnungsanschrift gesendet. 

Diese abweichende Rechnungsadresse hat keine Auswirkungen auf das Mahnwesen, da Mahnungen immer auf den Kundenstammsatz laufen und nicht auf abweichende Rechnungspostanschriften. 

Tooltip: 

Rechnungspostanschrift (RP): 
Abweichende Rechnungsanschrift für Rechnungen. 
ACHTUNG: Die abweichende Rechnungsanschrift hat keine Auswirkung auf das Mahnwesen. 
Die Mahnungen laufen immer auf den Kundenstammsatz, nicht auf die abweichende RP! 
Es ist als reine Postanschrift zu verstehen. 

### Checkbox (LE) Leistungsempfängeranschrift 
Wenn Sie diese Checkbox aktivieren, handelt es sich um eine Leistungsempfängeranschrift. D.h. eine abweichende Anschrift des Leistungsempfängers. 

Diese Adresse hat sonst keine weitere spezielle Funktion. 

Der Leistungsempfänger ist im Property oSalesProcessAddress_ARA zu finden. 

Tooltip: 

Leistungsempfängeranschrift (LE)
