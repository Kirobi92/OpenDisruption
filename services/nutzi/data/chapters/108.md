# Zoom-Elemente zum Beratungsassistenten

eNVenta ERP / Beratungsassistent in eNVenta 

# Zoom-Elemente zum Beratungsassistenten 
Die nachfolgenden Zoom-Elemente bieten die Möglichkeit, Informationen zum Beratungsassistenten bzw. eGate in den Fokus zu stellen. 

## Vertrieb / CRM 

### Preisspiegel 

#### Inhalt 
Dem Verkäufer sollen in einer Beratungssituation unterstützende Informationen zur Preissituation und Preisentwicklung eines Produktes angeboten werden. Damit kann die Preissituation des Produktes, bezogen auf die letzten x Monate, auf einen Blick beurteilt werden. 
#### Ziel / Nutzen 
Dies erlaubt eine schnelle Einstufung des Preises und gibt zusätzliche Sicherheit, einen angemessenen Preis zu finden. Damit wird eine Preisfindung auch für Verkäufer möglich, die den Kunden oder das Produkt noch nicht sehr gut kennen. Zu niedrige Preise können vermieden werden. 

#### Darstellung 
- Basislinie: Spanne vom niedrigsten bis zum höchsten Preis im Betrachtungszeitraum - Graue Balken: Häufigkeitsverteilung der Preise innerhalb der Spanne - Grauabstufungen: Unterteilung der Preis-Häufigkeiten nach geringen, hohen und mittleren Mengen - Grün : Durchschnittspreis - Rot : Letzter Preis des Kunden (wenn Kunde angegeben) mit Datum - Blaue Linie: Preistrend(-kurve) im Betrachtungszeitraum 
#### Parameter 
Titel - Titel des Elements 
Anzahl Monate - Zeitraum der Preisermittlung und Analyse (Standard: 12 Monate) 
Ausschlusskunden - kommaseparierte Liste der Kundennummern, die nicht berücksichtigt werden sollen. Auf diese Weise können z.B. interne Verrechnungskunden ausgeschlossen werden. 
#### Drill Down 
keiner 
#### Aktualisierung 
Die Aktualisierung der Daten erfolgt nur durch Anforderung des Benutzers. In Angebot / Auftrag / Gutschrift kann über den Mini-Button " B " eine Aktualisierung der Daten zur markierten Position ausgelöst werden. 
Alternativ kann auch eine Auftragsposition oder ein anderes Datenobjekt, das mindestens eine Artikelreferenz enthält, per Drag&Drop auf den Preisspiegel gezogen werden. 

## Vertrieb / CRM 

### Produktinfo 

#### Inhalt 
Anzeige einer Produktinformation auf Basis eines dafür vorgesehenen HTML-Reports. Dieses Element kann alternativ zur Anzeige der Original Produktseite aus eGate verwendet werden. 

#### Ziel / Nutzen 
Bereitstellung von umfassenden und relevanten Produktinformationen für den Berater. Zugang zu Alternativartikeln, Zubehör etc. durch Links. 
Wird der Artikel per Drag&Drop auf den Auftrag/Angebot/Gutschrift gezogen, entsteht eine neue Belegposition. 
#### Darstellung 
Anzeige einer HTML-Seite, die mit dem HTML-Report-Designer erstellt und angepasst werden kann. Auf diese Weise können passgenaue Produktinformationen zusammengestellt werden. 
#### Parameter 
Report - Auswahl des darzustellenden Reports für das Datenobjekt "Artikel". 
#### Drill Down 
Verlinkung auf andere Datenobjekte wie Alternativartikel etc. 
#### Aktualisierung 
Die Aktualisierung der Daten erfolgt nur durch Anforderung des Benutzers. In Angebot / Auftrag / Gutschrift kann über den Mini-Button " B " eine Aktualisierung der Daten zur markierten Position ausgelöst werden. 

Alternativ kann auch eine Auftragsposition oder ein anderes Datenobjekt, das mindestens eine Artikelreferenz enthält per Drag&Drop auf die Produktinfo gezogen werden. 

## Vertrieb / CRM 

### Kundennachfrage 

#### Inhalt 
Auf einer Zeitleiste werden die Vorgänge des Kunden der letzten Monate, Quartale oder Wochen zu dem aktuellen Artikel angezeigt. Folgende Vorgänge werden unterstützt: 
- Aufträge - Angebote - Gutschriften - Produktseitenaufrufe ( eGate ) 
#### Ziel / Nutzen 
Bei der Beurteilung der Kundensituation in einer Beratung ist auf einen Blick erkennbar, ob der Kunde den Artikel bereits einmal gekauft hat oder nicht, bzw. ob er dies regelmäßig tut. 
Es ist ebenfalls erkennbar, wenn ein Kunde den Artikel bereits mehrmals angefragt hat, aber dann nicht (bzw. woanders) kauft. 
#### Darstellung 
Der Zeitstrahl skaliert in Abhängigkeit der Käufe automatisch. So werden entweder Quartale, Monate oder Kalenderwochen dargestellt. Das Tagesdatum wird mit einer senkrechten, gestrichelten Linie markiert. 

Geht man mit der Maus auf ein Angebots- bzw. Auftragssignal, so werden in einem Tooltip die Menge und der Status der Vorgangsposition angezeigt. 

Oberhalb der Zeitleiste erscheinen die Aufträge ( grünes Symbol), Gutschriften ( rotes Symbol) und unterhalb werden die Angebote ( rotes Symbol) und die Produktseitenaufrufe ( blaues Symbol) dargestellt. 
#### Parameter 
Shop ID - Nummer des konfigurierten eGate Webshops 
Titel - Titel des Elements 
#### Drill Down 
Keiner 
#### Aktualisierung 
Die Aktualisierung der Daten erfolgt nur durch Anforderung des Benutzers. In Angebot / Auftrag / Gutschrift kann über den Mini-Button " B " eine Aktualisierung der Daten zur markierten Position ausgelöst werden. 

Alternativ kann auch eine Auftragsposition oder ein anderes Datenobjekt, das mindestens eine Kunden- und eine Artikelreferenz enthält, per Drag&Drop auf die Kundennachfrage gezogen werden. 

## eGate 
Dieses Zoom-Element steht Ihnen nur zur Verfügung, wenn eGate lizenziert ist! 
### eGate Historie 

#### Inhalt 
Das Element zeigt folgende Daten des Kunden aus dem Webshop eGate an. 
- Warenkörbe: offene Warenkorbpositionen evtl. aus mehreren Warenkörben - Gesehen: zuletzt angesehene Produkte des Kunden in eGate - Gesucht: letzte Suchen (Suchbegriffe) und Anzahl Treffer 
#### Ziel / Nutzen 
Der Berater / Vertriebsmitarbeiter bekommt zusätzliche Informationen zur Interessenssituation des Kunden. Dies ist insbesondere interessant, wenn der Kunde selbst im Webshop recherchiert und dann Beratungsbedarf hat. 
#### Darstellung 
Die drei verschiedenen Sichten werden als Registerkarten angeboten. Es erscheint jeweils eine Liste mit Ergebnissen. 
#### Parameter 
Shop ID - Nummer des konfigurierten eGate Webshops 

Anzahl - maximale Anzahl der angezeigten Ergebnisse. Dies gilt nur für "Gesehen" und "Gesucht". Dabei werden jeweils die aktuellsten Ergebnisse angezeigt. (Standard: 10) 

Es werden grundsätzlich bis zu maximal 100 Warenkorbpositionen angezeigt. 
#### Drill Down 
Kein Drill Down. Per Drag&Drop können Artikel aus den Sichten "Warenkorb" und "Gesehen" auf den Vorgang (Angebot / Auftrag / Gutschrift) oder auf ein anderes Element (z.B. Produktinfo) gezogen werden. 
#### Aktualisierung 
Die Aktualisierung der Daten erfolgt nur durch Anforderung des Benutzers. Im Angebot / Auftrag / Gutschrift kann über den Mini-Button "B" eine Aktualisierung der Daten zur markierten Position ausgelöst werden. 

Alternativ kann auch eine Auftragsposition oder ein anderes Datenobjekt, das mindestens eine Kundenreferenz enthält per Drag&Drop auf die eGate Historie gezogen werden. 

## eGate 

### Shopseite 

#### Inhalt 
Das Element zeigt die Produktseite aus dem eGate Webshop zum aktuellen Produkt an. Die Suchfunktion der Shopseite kann zur Recherche nach anderen bzw. weiteren Produkten genutzt werden. 
#### Ziel / Nutzen 
Der Berater erhält die gleiche Informationssicht, die auch der Kunde hat, wenn er sich im Webshop informiert. Das erleichtert das Gespräch. Eine zusätzliche Eingabe der Artikelnummer und Suche im Webshop entfällt. 
#### Darstellung 
Das Element kommuniziert direkt mit dem Webshop und bringt die Originalseite zur Ansicht. In Höhe und Breite sollte das Element im Beratungsassistenten so eingerichtet werden, dass der Inhalt der Shopseite in angemessener Größe darstellbar ist. 
#### Parameter 
Shop ID - Nummer des konfigurierten eGate Webshops 
Titel - Titel des Elements 
#### Drill Down 
Keiner. Der Artikel einer Produktseite kann per Drag&Drop auf der Artikelbezeichnung auf einen offenen Vorgang (Angebot / Auftrag) gezogen werden. Es entsteht eine neue Vorgangsposition. 
#### Aktualisierung 
Die Aktualisierung der Daten erfolgt nur durch Anforderung des Benutzers. In Angebot / Auftrag / Gutschrift kann über den Mini-Button " B " eine Aktualisierung der Daten zur markierten Position ausgelöst werden. 

Alternativ kann auch eine Auftragsposition oder ein anderes Datenobjekt, das mindestens eine Artikelreferenz enthält, per Drag&Drop auf die Shopseite gezogen werden. 

## Allgemein 

### Arrangements 

#### Inhalt 
Das Element zeigt folgende, bestehende Vereinbarungen / Arrangements mit dem Kunden an. 

- Projekte: Offene Projekte / Objekte des Kunden - Rahmen: laufende Rahmenaufträge zu dem aktuellen Artikel oder dessen Artikelgruppe - Aktionen: laufende Marketingaktionen, bei denen der Kunde Teilnehmer ist. 
Durch Klick auf eine Marketingaktion können dazu auch die Aktivitätsschritte angezeigt werden. 
#### Ziel / Nutzen 
Im Beratungsgespräch kann auf die laufenden Vereinbarungen / Arrangements Bezug genommen werden. Dies ist insbesondere dann hilfreich, wenn viel mit Projekten oder Rahmenvereinbarungen gearbeitet wird. Eine Zuordnung zu einem Projekt oder Aktion ist einfach per Drag&Drop möglich. 
#### Darstellung 
Die drei verschiedenen Sichten werden als Registerkarten angeboten. Es erscheint jeweils eine Liste mit Ergebnissen. Der Registerkarte erhält einen Stern, wenn Daten vorhanden sind. 
#### Parameter 
Titel - Titel des Elements 
#### Drill Down 
Das Projekt, der Rahmenauftrag oder die Marketingaktion können per Doppelklick aufgerufen werden. Per Drag&Drop wird das Projekt oder die Aktion auf einen Vorgang (Auftrag / Angebot) gezogen und somit zugeordnet. 

#### Aktualisierung 
Die Aktualisierung der Daten erfolgt nur durch Anforderung des Benutzers. In Angebot / Auftrag / Gutschrift kann über den Mini-Button " B " eine Aktualisierung der Daten zur markierten Position ausgelöst werden. 

Alternativ kann auch eine Auftragsposition oder ein anderes Datenobjekt, das mindestens eine Kundenreferenz enthält, per Drag&Drop auf die Arrangements gezogen werden.
