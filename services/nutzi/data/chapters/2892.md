# Auswahl der Konfiguration in EDI 
Alle Dateien aus einem Verzeichnis heraus werden betrachtet. Anhand der nachfolgend vorgestellten drei Schritte werden Prüfungen durchgeführt. Gibt es keinen Treffer, wird ein Fehler ausgegeben. 

Schritt 1 
Eine Prüfung erfolgt auf den xPath Ausdruck im EDI Teilnehmer . Gibt es dann mehrere Mapping-Möglichkeiten, geht es weiter mit Schritt 2. Wenn es jedoch keinen Treffer gibt, wird ein Fehler ausgegeben. 

Über den Info-Button hinter dem Feld XPath-Filter können ein bzw. mehrere XML-Dateien zur Prüfung ausgewählt werden. 

Schritt 2 
Es wird nun geprüft, ob die Struktur mit der Datei übereinstimmt. Die Knoten (Pflichtknoten) werden auf Übereinstimmung geprüft. Außerdem wird der in der Baumstruktur angegebene Fixwert überprüft. Gibt es dann mehrere Mapping-Möglichkeiten geht es weiter mit Schritt 3. Wenn es jedoch keine Übereinstimmung gibt, wird ein Fehler ausgegeben. 

Schritt 3 
Gibt es mehrere mögliche Mappings, werden weitere Prüfungen durchgeführt. Innerhalb des Mappings können über den Button Erweitert zusätzlich xPath-Ausdrücke hinterlegt werden. Im ersten Schritt ist der xPath Ausdruck in Verbindung mit dem EDI-Teilnehmer und dem Nachrichtentyp. Ein Mapping kann auch für einen anderen Nachrichtentyp genutzt werden, daher besteht hier noch die Möglichkeit einen xPath-Filter zu hinterlegen. 

Schritt 4 
Wenn es nach den ersten 3 Schritten immer noch mehrere Möglichkeiten gibt, folgt als letzter Schritt die Prüfung über die Prozessregeln. Hier findet eine Kundenermittlung anhand verschiedener Kriterien statt. Wenn ein EDI Teilnehmer angelegt ist und dessen Kundennummer der aus der Kundenfindung entspricht, wird das im Teilnehmer verzweigte Mapping gewählt. Gibt es keinen Treffer, wird ein Fehler ausgegeben. 

Hier wird die KL Nummer mit der Kundennummer aus dem entsprechenden XML der ORDERS (Import) verglichen. 

Siehe Kapitel EDI-Teilnehmer .
