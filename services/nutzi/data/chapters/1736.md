# Bestellungen Registerkarte Positionen 
Bestellung Registerkarte Positionen 

Auf dieser Registerkarte geben Sie die Positionen für eine Bestellung ein. Gehen Sie dabei in folgenden Schritten vor: 

- Lieferantennummer eintragen/suchen - Bestellkopfdaten eingeben - Bestellpositionen erfassen - Bestellerfassung abschließen - Zusätzliche Bestellinformationen 
Auf dieser Registerkarte gibt es vier verschiedene Positionsarten. Aus der entsprechenden Combobox können Sie wählen, welche Positionsart Sie einfügen möchten: 

- Artikelpositionen - Gruppentitel - Zwischensummen - Textpositionen 
Bevor Sie die erste Bestellposition erfassen, müssen Sie die Auswahl der korrekten Bestellart, der gewünschten Währung (es geht nur Kunden- oder Systemwährung) und die Anpassung des Kurses vorgenommen haben, da keine nachträgliche Änderung mehr möglich ist. 

Es gibt mehrere Möglichkeiten, eine Bestellposition zu erfassen. 

Eine Bestellposition kann jederzeit mit dem Button Neu auf der Registerkarte Positionen erfasst werden. Das Programm fügt in die Positionstabelle eine neue Zeile ein. Der Cursor steht in der Spalte ‚Artikel’ zur Eingabe des gewünschten Artikels bereit. 

Durch Betätigen der Funktionstaste F7 wird ebenfalls eine neue Zeile in die Positionstabelle eingefügt. 

Im rechten Bereich der Registerkarte werden die Spalten 'Basis Bestellt', 'EK Basis Netto', 'Basis Rabattbetrag' und 'Basis Rabatt%' mit den Werten gefüllt, die aktualisiert wurden. Die Werte werden bei jeder Änderung neu ermittelt, um diese dann mit der Rechnung vergleichen zu können. 

Diese Werte können nicht vom Benutzer manuell geändert werden. Auf diese Weise kann in der Rechnungsprüfung nachvollzogen werden, falls ein Benutzer manuell die Preise oder Konditionen verändert hat. 

Wenn ein Rabatt größer 10.000 eingegeben wird, wird dieser Rabatt begrenzt auf 9.999,99! 

Steht der Cursor auf der letzten Spalte einer bereits erfassten Position, kann durch Weiterschalten mit Tab sofort eine neue Zeile eingefügt und mit der Erfassung der nächsten Artikelposition begonnen werden. Eine schnelle und reibungslose Bestellerfassung wird dadurch wesentlich unterstützt. 

Legen Sie vor der Eingabe der Bestellpositionen fest, auf welches Lager die bestellte Ware gebucht werden soll. Dazu steht die Combobox Lagernr . zur Verfügung. Die entsprechende Lagernummer wird automatisch in jede neue Position eingetragen. 

Werden Positionen nach dem Abschluss storniert, werden sie an den Schluss der Tabelle gesetzt, nachdem Sie die Änderungen gespeichert haben. 

Erfassen von Zwischensummen 

Ist ein Gruppentitel mit seinen Bestellpositionen erfasst und Sie aktivieren den Radiobutton Zwischensumme , erscheint in der Bestellung eine Leerzeile. Diese Leerzeile wird im Druck mit der errechneten Zwischensumme gefüllt. 

Erfassen von Belastungspositionen in der BestellungEs ist möglich (analog zum Auftrag) in der Bestellung negative Mengen einzugeben. Auf diese Weise können auch Belastungspositionen direkt in der Bestellung erfasst werden. Dies kann hilfreich sein, wenn z.B. Gebinde mit dem Wareneingang zurückgegeben werden sollen, bzw. sonstige Belastungen gleich gegengerechnet werden sollen. 

Im Wareneingang kann eine negative Menge nicht verändert werden. 

Basis-Felder 

Die Spalten 'Basis Bestellt', 'Basis Rabattbetrag', 'Basis Rabatt %', 'EK Basis Netto' sind sogenannte Schattenfelder. So kann sichergestellt werden, dass der Bearbeiter immer noch beauskunften kann, zu welchen Konditionen geordert wurde. 

Außerdem kann bei der Rechnungsprüfung per EDI (über Job Server) oder manuell, verglichen werden, ob die Gültigkeitsregeln eingehalten wurden. 

Bestellsets 
Wird ein Bestell-Set in einer Bestellung erfasst, so wird mit den Komponenten des Sets bei Abschluss der Bestellung eine separate Bestellung über die Inhaltsartikel ausgelöst. Über eine eindeutige Referenz zwischen der Bestellung der Inhaltsartikel und der „ursprünglichen“ Set-Bestellung sind diese miteinander verknüpft (ersichtlich über die Registerkarte Artikel-Sets ). Dazu wird die Bestellnummer der Set-Bestellung in die Bestellposition des Inhaltsartikels eingetragen. Als Einkaufspreis der Set-Bestellpositionen wird der EK der Artikel-Sets wertmäßig auf die Inhaltsartikel verteilt. Es handelt sich um einen Netto-Netto EK. Mit diesem umgerechneten EK wird später auch der Wareneingang gebucht. 

Die Bestätigung des Lieferanten für die Bestellung erfolgt auf Basis des Bestell-Sets. Das bestätigte Lieferdatum wird automatisch in die Bestellposition der Set-Bestellpositionen der Inhaltsbestellung übertragen und der Status entsprechend auf 2 gesetzt. 

Ein Storno einer Bestellung mit Bestellsets kann wie gewohnt ausgeführt werden. Bei Stornierung der kompletten Bestellung werden auch die einzelnen Positionen storniert. 
### Button Neu 

Dieser Button fügt eine neue Zeile in die Tabelle ein, so dass Sie eine neue Position einfügen können. 

### Button Ändern 

Über diesen Button können Sie Änderungen an einer Position vornehmen. Auch die Kostenstellennummer kann hier optional auf Positionsebene eingegeben werden. Die Kostenstellennummer wird dann beim Kopieren einer Bestellung sowie beim Generieren einer neuen Bestellung aus dem Dispo-Pool entsprechend berücksichtigt. 

Siehe Kapitel Ändern einer Position in der Bestellung . 

### Button Löschen 

Mit diesem Button können Sie einen Eintrag der Tabelle löschen. 

Ist der Status einer Bestellung offen und ist die Bestellung noch nicht gedruckt und auch noch kein EDI-Versand erfolgt, wird über den Button Löschen die gesamte Bestellung gelöscht. 

Ist die Bestellung bereits an den Lieferanten abgeschickt worden (per Druck oder EDI), so muss die Bestellung auf Status 6 (storniert) gesetzt werden. Zuvor erscheint eine Abfrage mit entsprechenden Hinweisen. 

Auch die einzelnen Positionen können storniert werden. Bei Klick auf den Button Löschen werden diese entsprechend auf den Status 6 gesetzt. Alle entsprechenden Reservierungen werden gelöscht. Zuordnungen von Belegen auf diese Bestellung bzw. Bestellpositionen werden gelöscht, damit der Benutzer entscheiden kann, ob diese Position nochmals bestellt werden muss. 

Im Dialog Einkauf Übersicht können stornierte Bestellungen selektiert werden. So ist eine Überwachung möglich. Dazu gibt es die Checkbox Storniert .

### Button Stückliste 

Zu einem Stücklistenartikel öffnet sich mit diesem Button der Dialog Einkaufstückliste für weitere Einstellungen. 

Bereits bei der Erfassung der Bestellpositionen kann bestimmt werden, dass bei der Verbuchung des Wareneingangs die Stückliste geteilt werden soll. Das kann z.B. sinnvoll sein, wenn der Kauf eines montierten Artikels günstiger ist als der Einkauf von Einzelkomponenten, letztere aber tatsächlich benötigt werden. Diese Einstellung nehmen Sie im Dialog Position , Registerkarte Allgemein , im Feld Menge zerlegen , vor.

### Combobox Positionsart 
Aus dieser Combobox wählen Sie die Positionsart aus. Zur Verfügung stehen: 

- Positionen - Gruppentitel - Zwischensumme - Text 
Position 
Die Positionsart P wird aktiviert, d.h. Sie erfassen einzelne Positionen. 

Gruppentitel 
Die Positionsart G (Gruppentitel) wird aktiviert, d.h. Sie erfassen eine Gruppierung von Artikelpositionen. Diese Gruppierung hat keine direkte Auswirkung auf Preise und Konditionen, bewirkt allerdings eine Anpassung der Darstellung auf den Belegen. 

Siehe Kapitel Gruppentitel . 

Zwischensum. 
Die Positionsart Z (Zwischensumme) wird aktiviert. Ist ein Gruppentitel mit seinen Positionen erfasst, erscheint im Auftrag eine Leerzeile. Diese Leerzeile wird im Druck der Rechnung mit der errechneten Zwischensumme gefüllt. 

Text 
Die Positionsart T (Text) wird aktiviert. Damit erfassen Sie Textpositionen. 

Siehe Kapitel Textpositionen .
