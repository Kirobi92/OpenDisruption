# Static Cache

# Static Cache 
Static caching ist, wenn ein Browser eine Ressource anfordert. Der Server, der die Ressource bereitstellt, kann dem Browser mitteilen, wie lange er die Ressource vorübergehend speichern oder zwischenspeichern soll. Hierbei werden die Daten nicht auf dem Server, sondern auf dem Broker gelagert. Das ist wichtig, da ein Server ggf. mehrere Broker hosten kann. Bei dem Browser handelt es sich um den eNVenta -Client. 

Um sicherzustellen, dass die Daten trotzdem aktuell sind, werden hierbei 2 Prüfungen ausgeführt: Erfolgt ein Zugriff auf die entsprechenden Komponenten, so wird geprüft, ob die angeforderten Daten geladen sind. Falls die Daten nicht geladen sind, erfolgt automatisch ein Nachlesen der Daten von der Datenbank. Des Weiteren wird jeder Cache nach spätestens 5 Minuten automatisch aktualisiert, um das Nachladen von der Datenbank bei direktem Zugriff zu vermeiden. 

Es gibt hier noch eine 3. Möglichkeit: Wenn in den Pflegedialogen ein Datensatz geändert wird, wird beim nächsten Zugriff auf die Daten, von diesem Broker, auch neu geladen. (Beispiel: Wenn Sie eine neue Auftragsart anlegen, ist diese sofort im Auftrag verfügbar und nicht erst nach 5 Minuten.) 

Es handelt sich hier um eine Performanceverbesserung durch Cachen. D.h. der Static Cache hält ausgewählte Komponenten wie z.B. die Währungstabelle, die Auftragsarten, VK-Rabattgruppen, Lagerorte, Lieferantengruppen, Einkaufarten etc. bereit, um sie bei Bedarf abzurufen. 

Achtung: Bestehendes Customizing kann weiterhin genutzt werden. Um die Vorteile des Static Cache auch im eigenen Customizing zu nutzen, sind die entsprechenden Code-Stellen zu überarbeiten. Nähere Informationen finden Sie im separaten Dokument Static Cache For Customizer .
