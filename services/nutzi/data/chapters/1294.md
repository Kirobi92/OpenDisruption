# Rahmenabrufe im Rahmenauftrag 
Rahmenabrufe 

Wählen Sie zuerst im Dialog Rahmenauftrag die gewünschte Rahmenauftragsposition aus, zu der Sie einen Abruf erfassen möchten. Wenn Sie eine Position markiert haben, gelangen Sie über den Button Abrufe auf der Registerkarte Positionen in diesen Dialog, welcher der Erfassung und Verwaltung von Abrufen dient. Es werden alle wichtigen Informationen zur Position sowie vorhandene Abrufe angezeigt. 

Auf der rechten Seite des Dialogs werden bis zu 6 Rabatttexte, Rabattprozente und/oder Rabattbeträge angezeigt. 

Bei der Neuanlage eines Rahmenabrufs ist zwingend ein Lieferdatum einzugeben. 

Beim Erfassen von Auftragspositionen in der Auftragsverwaltung wird anhand der Artikelnummer automatisch eine Prüfung auf Existenz einer Rahmenauftragsposition durchgeführt. Existiert zum Kunden eine Rahmenauftragsposition zum entsprechenden Artikel, so wird eine Übersicht über alle Abrufe des Rahmenauftrags automatisch eingespielt. Aus dieser Übersicht können Sie offene Abrufe mit den Konditionen des Rahmenauftrags in die Auftragsverwaltung übernehmen. Sie haben außerdem die Möglichkeit, neue Abrufe zu generieren. 

Abrufe werden bei Warengruppen identisch wie bei Artikelpositionen gehandhabt. 

Mit Aktivierung der Checkbox Nur offene Abrufe anzeigen , werden nur Rahmenaufträge in die Tabelle eingespielt, die noch offen sind. 

In eNVenta ist es möglich, dass ein Rahmenauftrag aus mehreren Rahmenbestellungen bedient werden kann. 

Wird nun eine Auftragsposition in einem Auftrag für einen Kunden erfasst, für den ein Rahmenauftrag und ein entsprechender Rahmen für den Artikel (Rahmenauftragsposition auf Artikelgruppenebene mit mehreren Streckenabschlüssen) existiert, ist es im Dialog Rahmenabrufe möglich zu wählen, aus welchem Streckenabschluss die Position bedient werden soll. 

Hierzu wird die entsprechende Position "ohne Spezifikation" in der Tabelle markiert und über den Button ein neuer Abruf mit der entsprechenden Menge hinzugefügt. 

Dieser Abruf ist nun dem entsprechenden Streckengeschäft zugeordnet und beim Speichern des Auftrags bzw. der Auftragsposition wird ein entsprechender, einkaufsseitiger Abruf in der zugehörigen Rahmenbestellung erzeugt. 

Wird der Auftrag bzw. die Auftragsposition nicht gespeichert bzw. gelöscht, so wird der entsprechende, automatisch erzeugte Rahmenbestellungs-Abruf ebenfalls wieder gelöscht! 

Wird nun auftragsbezogen bestellt (Button Bestellen im Auftragskopf), so wird für diese Position im Dialog Bestellen / Disponieren der entsprechende Lieferant aus der Rahmenbestellung vorbelegt. 

Siehe Kapitel Rahmenabrufe in der Rahmenbestellung . 

Dieser darf in diesem Fall nicht verändert werden. 

Wenn Sie versuchen den Lieferanten im Dialog Bestellen / Disponieren über den Button L-Auswahl zu ändern, wird eine Fehlermeldung ausgegeben: "Der Lieferant kann nicht geändert werden, da es sich um einen Streckenabruf handelt. Die Position muss ggf. aus dem Auftrag gelöscht und ohne Streckenbezug neu erfasst werden". 

Des Weiteren ist es möglich, Abrufe aus dem Rahmenauftrag für eine Auftragsposition zu erzeugen, die NICHT aus einem Streckenabschluss bedient werden ("manueller Abruf"). 

Hierzu steht der Button Leerabruf zur Verfügung. Bei Klick auf diesen Button wird ein neuer Abruf im gewählten Rahmenauftrag erzeugt, der NICHT mit Strecke bedient wird, d.h. der keinem Artikel ohne Spezifikation zugeordnet ist. Wird diese Position aus dem Auftrag heraus auftragsbezogen bestellt, so ist es im Dialog Bestellen / Disponieren weiterhin möglich, den entsprechenden Lieferanten, bei dem bestellt werden soll, manuell über den Button L-Wahl zu wählen. 

Es ist explizit ausgeschlossen, dass eine Auftragsposition aus 2 verschiedenen Rahmenaufträgen bzw. Rahmenabrufen bedient wird, d.h. pro Auftragsposition gibt es nur EINEN zugeordneten Abruf. 

Der Button Leerabruf ist nur dann nicht zugänglich, wenn die übergeordnete Rahmenposition vom Typ = "W" ('Artikelgruppe') ist. 

Der Button Aktualisieren sorgt dafür, dass der komplette Abruf-Prozess (der hinter der Tabelle steht) neu aus der Datenbank geladen wird. Dies ist u.a. dann sinnvoll, wenn eine Exception geworfen worden ist. 

Auftragsposition: 

Sobald eine Auftragsposition global rahmenkonditioniert ist, wird deren Lieferant auf den 'Ziel-Lieferanten' der verknüpften Rahmenbestellung 'gedreht'. Solange diese Globalstrecke der Auftragsposition zugeordnet bleibt, ist der Lieferant nicht änderbar. 

Siehe Kapitel Globalabschluss und Strecke .

Dieser Button leert den Dialog für eine neue Eingabe.

### Button Speichern 

Mit diesem Button speichern Sie Ihre Eingaben / Änderungen. 

Gleichzeitig wird die Summe über alle Abrufe ermittelt und die Tabelle mit den Rahmenauftragspositionen aktualisiert. 

### Button Neuer Satz 

Mit diesem Button fügen Sie weitere Daten in den Dialog ein. 

Ein Abruf muss generell mit Abrufmenge und Liefertermin eingefügt werden, alle anderen Spalten werden automatisch gefüllt. 

Zu einem Artikel oder einer Warengruppe können mehrere Abrufe am selben Tag eingetragen werden. Es erfolgt keine Prüfung auf den Termin. 

### Button Löschen 

Dieser Button löscht Ihre Eingaben. Genauer gesagt, die markierte Zeile wird mit dem versehen und erst endgültig gelöscht, wenn Sie den Button drücken. 

### Button Auswahl 

Mit diesem Button übernehmen Sie offene Abrufe in den Rahmenauftrag übernehmen. 

### Button Schließen 

Dieser Button schließt den Dialog. 

Vergessen Sie nicht Ihre Eingaben zuvor zu speichern!
