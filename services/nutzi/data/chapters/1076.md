# Auswahl Zusätzlicher Anschriften 
Auswahl Zusätzliche Anschriften 

Dieser Dialog öffnet sich über den Button in den Verkaufsdialogen, wie z.B. Auftrag und Angebot und dort auf der Registerkarte Zusätzliche Anschriften . Hier können Sie nach zusätzlichen Adressen selektieren. Im Kopf des Dialogs stehen dazu noch weitere Selektionskriterien zur Verfügung. 
## Steuerelemente des Formulars 

### Bezeichnungsfeld Auswahl Zusätzliche Anschriften 
Tooltip: 

In diesem Dialog können Sie nach zusätzlichen Adressen zu einem Verkaufsvorgang suchen. 
Für mehr Informationen drücken Sie F1. 

### Button Form-Menü 

Mit Klick auf diesen Button werden je nach Kontext und Berechtigung die unterschiedlichen Optionen des jeweiligen Dialogs hinter dem Button zur Auswahl angeboten. 

Die Funktionen für das Client Side Customizing sowie die Online-Hilfe stehen hier ebenfalls zur Verfügung.

### Checkbox (AN) Angebotsanschrift 
Wenn Sie diese Checkbox aktivieren, wird das Angebot an eine abweichende Adresse gesendet. 

Ansonsten hat diese Adresse keine weitere spezielle Funktion. 

Tooltip: 

Angebotsanschrift (AN): 
Abweichende Anschrift, an welche das Angebot gesendet wird. 
Ansonsten hat diese Adresse keine weitere spezielle Funktion. 

### Checkbox (BA) Bestellanschrift 
Wenn Sie diese Checkbox aktivieren, wird die Bestellung an eine abweichende Adresse gesendet, wo der Besteller sitzt. 

Ansonsten hat diese Adresse keine weitere spezielle Funktion. 

Tooltip: 

Bestellanschrift (BA): 
Abweichende Anschrift, wo der Besteller sitzt. 
Ansonsten hat diese Adresse keine weitere spezielle Funktion. 

### Checkbox (AB) Auftragsbestätigungsanschrift 
Wenn Sie diese Checkbox aktivieren, wird die Auftragsbestätigung an eine abweichende Adresse gesendet. 

Ansonsten hat diese Adresse keine weitere spezielle Funktion. 

Tooltip: 

Auftragsbestätigungsanschrift (AB): 
Angabe, wohin die Auftragsbestätigung gesendet wird. 
Ansonsten hat diese Adresse keine weitere spezielle Funktion. 

### Checkbox (RA) Rechnungsanschrift 
Wenn Sie diese Checkbox aktivieren, wird die Rechnung an eine abweichende Anschrift (einer juristischen Person) gesendet. 

Die abweichende Rechnungsanschrift hat keine Auswirkung auf das Mahnwesen. 

Die Mahnungen laufen immer auf den Kundenstammsatz, nicht auf die abweichenden Rechnungsanschriften! Ebenso die ZM u.v.a. 

Relevant für die Finanzbuchhaltung sind Rechnungsanschrift und Lieferanschrift aus den Kopfdaten! 

Tooltip: 

Rechnungsanschrift (RA): 
Unterschiedliche Rechnungsanschriften zu einer juristischen Person. 
ACHTUNG: Die abweichende Rechnungsanschrift hat keine Auswirkung auf das Mahnwesen. 
Die Mahnungen laufen immer auf den Kundenstammsatz, nicht auf die abweichende RA! Ebenso ZM u.v.a. 
Relevant für die Finanzbuchhaltung ist die Rechnungsanschrift und Lieferanschrift aus den Kopfdaten! 

### Checkbox (RP) Rechnungspostanschrift 
Wenn Sie diese Checkbox aktivieren, wird die Rechnung an eine abweichende Anschrift (einer juristischen Person) gesendet. 

Die abweichende Rechnungsanschrift hat keine Auswirkung auf das Mahnwesen. 

Die Mahnungen laufen immer auf den Kundenstammsatz, nicht auf die abweichenden Rechnungspostanschriften! Ebenso die ZM u.v.a. 

Relevant für die Finanzbuchhaltung sind die Rechnungsanschrift und Lieferanschrift aus den Kopfdaten! 

Tooltip: 

Rechnungspostanschrift (RP): 
Abweichende Rechnungsanschrift für Rechnungen. 
ACHTUNG: Die abweichende Rechnungsanschrift hat keine Auswirkung auf das Mahnwesen. 
Die Mahnungen laufen immer auf den Kundenstammsatz, nicht auf die abweichende RP! 
Es ist als reine Postanschrift zu verstehen. 

### Checkbox (LE) Leistungsempfängeranschrift 
Wenn Sie diese Checkbox aktivieren, handelt es sich um eine Leistungsempfängeranschrift. D.h. eine abweichende Anschrift des Leistungsempfängers. 

Diese Adresse hat sonst keine weitere spezielle Funktion. 

Der Leistungsempfänger ist im Property oSalesProcessAddress_ARA zu finden.
