# Bestand 
Bestand 

Mit dem Button Bestand , den Sie in verschiedenen Dialogen finden, kann zu dem gewählten Artikel ein Dialog mit Bestandsinformationen aufgerufen werden. 

Je nachdem wo Sie den Dialog aufrufen, kann der Dialog Bestand auch als modaler Dialog erscheinen. 

eNVenta ERP unterstützt verschiedene Verfahren zur Berechnung und Darstellung von Artikelbestand und -bedarf. 

Es gibt kein Verfahren, das grundsätzlich besser ist als ein anderes. Die Wahl des richtigen Verfahrens hängt von der Lagerorganisation und dem Artikel selbst ab. 

Für jeden einzelnen Artikel kann das am besten geeignete Verfahren ausgewählt werden. 

Bitte prüfen Sie, welche der beiden folgenden Beschreibungen am ehesten auf den jeweiligen Artikel zutrifft und wählen Sie demzufolge das richtige Verfahren: 

- Der Artikel hat möglichst ständig in ausreichender Menge am Lager verfügbar zu sein, damit eine sofortige Lieferfähigkeit gegeben ist. Für die Mehrheit der Aufträge ist eine sofortige Auslieferung vorgesehen, deshalb wählen Sie hier das Bestellpunktverfahren . Die Angaben in den Feldern Lagerplatz, Meldebestand, Mindestbestand und Maximalbestand werden aus dem Menü Lager pro Artikel übernommen. Ist für dieses Lager keine Angabe im Dialog Lager pro Artikel vorhanden, werden die Angaben aus dem Artikelstamm übernommen. Der Dialog Lager pro Artikel ist beschrieben im Kapitel Lagerbestand (Lager pro Artikel) ) . - Sie bestellen oder produzieren die Ware erst nach Erhalt eines entsprechenden Auftrags. Die Mehrheit der Aufträge sind Terminaufträge, d.h. zur Auslieferung für einen bestimmten Termin vorgesehen. Eine sofortige Lieferfähigkeit braucht nicht unbedingt gegeben sein. Der Lagerbestand soll möglichst minimiert werden. Soll dazu der Einkauf oder die Produktion der Ware termingenau für den Bedarf, d.h. zu einem möglichst späten Zeitpunkt erfolgen, wählen Sie das deterministische Verfahren. 

### Auftragsbezogener Bestand 
Es ist möglich bevorratete Ware, die in einem anderen Auftrag auftragsbezogen bestellt worden ist, anderweitig zu verkaufen. Dafür gibt es eine entsprechende interne Unterscheidung in der Reservierung für den Auftrag, damit die Ware trotz offener Bestellung verkauft werden kann. (Dies gilt nur in Abhängigkeit des Lieferdatums des anderen Auftrags und ob dieses eine rechtzeitige Lieferung durch den Lieferanten gewährleistet.) 

Für die Sicht auf die Verfügbarkeit gilt in diesem Fall folgende Regel: 

Verfügbar = Buchbestand - Reservierung + auftragsbezogen bestellte Menge (Status < 3) 

Die auftragsbezogene Menge, die noch nicht wareneingangsgebucht ist, erhöht den verfügbaren Bestand. 

Die Ermittlung der verfügbaren Menge basiert auf den Reservierungen in der Datenbank. Um nun die auftragsbezogen bestellte Menge mit berücksichtigen zu können (von der Reservierung auszunehmen), wird diese Information in die Reservierung geschrieben. 

Es wird immer die Summe der auftragsbezogen bestellten Menge in die jeweilige Reservierung geschrieben. 

Dazu steht auf der Registerkarte Statisch das Feld Auftragsbez . zur Verfügung. In diesem Feld wird die Menge des Artikels angezeigt, die aus dem Auftrag heraus bestellt wurde, aber noch nicht geliefert ist. Diese Menge wird auf der Registerkarte Lagerliste in der Spalte ‚Auftragsbez.‘ ebenfalls angezeigt.

| --- | --- | --- | --- | --- |
| 10 | 8 | 0 | 2 | 2 |

Bestellung 14118204 -> 4 Stück

### Zuteilbare Menge 
Der Fall noch nicht lieferfreigegebene (Teil-) Reservierungsmenge wird bei der Berechnung besonders berücksichtigt. 

Es werden die noch nicht realisierten Rückstandsmengen (lieferfreigegebene Unterlieferungspositionen mit Rest-Kennzeichen = ‚R‘) mitberücksichtigt. Diese Teilmenge ist auf der einen Seite in der Reservierungsmenge der Ursprungsposition voll enthalten; wird aber erst zu einem noch nicht feststehenden zukünftigen Zeitpunkt benötigt. Damit kann sie zum aktuellen Zeitpunkt für andere Zwecke eingesetzt und folglich ‚zugeteilt‘ werden. Damit erhöht sich die zuteilbare Menge in diesem Fall – im Vergleich zur bisherigen Logik – um diese Rückstandsmenge. 

Anders verhält es sich bei lieferfreigegebenen Unterlieferungspositionen mit RestKZ = ‚S‘: 

Hier ist bereits die Reservierungsmenge auf die ‚zu liefernde Menge‘ dieser Position reduziert. Und im Auftrag wird Rückstandsmenge auch niemals benötigt und kann folglich nicht für andere Zwecke eingesetzt werden. Im Ergebnis wird also die ‚zuteilbare Menge‘ in diesem Falle – anders als im obigen ‚R‘-Falle - nicht erhöht. 

Außerdem wird noch nach ‚APos.RestKZ‘ gruppiert. 

Diese Logik bildet folgende Fälle ab: 

Fall Auftragsposition Reservierung, Feld Menge Teilmenge nicht freigegeben Status 1 Status 2 
| | | | |
| --- | --- | --- | --- |
| | | | |
| 1 | Bestellt = Geliefert | AP.Bestellt | AP.Bestellt |
| 2 | Bestellt > Geliefert, 'R' | AP.Bestellt | AP.Bestellt |
| 3 | Bestellt > Geliefert, 'S' | AP.Geliefert | AP.Geliefert |
| | | | |
| 4 | Bestellt = Geliefert | AP.Bestellt | 0 |
| 5 | Bestellt > Geliefert, 'R' | AP.Bestellt | AP.Bestellt - AP.Geliefert |
| 6 | Bestellt > Geliefert, 'S' | AP.Geliefert | |

Für die Logik sind die ‚Teilmenge!= 0-Fälle‘: 1, 2, 3 und 5 relevant. 

Beispiel: 

DB = DEV01, Lagernummer = 1, Artikelnummer = ‚XYZ‘, 
Aufträge: 19092018074629, 19092018074630 

Der Buchbestand auf Lager 1 = 50 Stück 
A1 19092018074629: Status = 2, Bestellt = 26, Geliefert = 25, R 
A2 19092018074630: Status = 2, Bestellt = 25, Geliefert = 24, R 

Daraus ergibt sich: 

Summe Reservierung (Lager 1, Artikel ‚XYZ‘) = 51 
Bestandsauskunft, Menge verfügbar = 50 – 51 = -1 (korrekt) 

A1: Menge nicht lieferfreigegen (Fall 5 ob.) = 26-25 = 1 
A2: Menge nicht lieferfreigegen (Fall 5 ob.) = 25-24 = 1 

Damit sieht die zuteilbare Menge nach der aktuellen Korrektur in der Bestandsauskunft folgendermaßen aus: 

Menge zuteilbar = -1 + 2 + 0 = 1 (Bemerkung: Bislang wird hier -1 angezeigt => falsch)

### Feld Prüfzeitpunkt 
Hier können Sie das Datum ändern, um Verkaufs- und Einkaufsprozesse, die in der Zukunft liegen, berücksichtigen zu können. Das hier eingetragene Datum findet auf der Registerkarte Statisch und in der Lagerliste Verwendung. 

Dieses Feld steht nur bei dynamischer Verfügbarkeit zur Verfügung! 

Mehr zur Berechnung des Prüfzeitpunkts erhalten Sie im Kapitel Dynamische Verfügbarkeitsprüfung . 

Tooltip: 

Prüfzeitpunkt: 

Hier kann der Betrachtungszeitraum für das Lager angegeben werden. 
Wird ein Datum in der Zukunft angegeben, so werden alle Reservierungen in diesem Zeitraum berücksichtigt. 
Liegt das Datum in der Vergangenheit oder auf dem aktuellen Datum, erfolgt keine Eingrenzung. 
Bei Eingabe eines Datums in der Vergangenheit, wird das Datum automatisch auf das Tagesdatum gesetzt. 

### Checkbox Auch Qualitäten 
Wenn Sie diese Checkbox aktivieren, wird die Combobox Lagernr um die Link-Lager (Darstellung der Qualitäten einer Ware und deren Abgrenzung) erweitert. Auf diese Weise kann gezielt B-Ware, also Ware schlechterer Qualität, verkauft werden. 

Tooltip: 

Wenn aktiv, werden hier auch Link-Läger angezeigt, auf denen Ware in anderer Qualität liegt.
