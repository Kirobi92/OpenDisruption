# Ereignisdefinition 
Ereignisdefinition 

Dieser Dialog öffnet sich im Navigator >> System >> Administration >> Eventmanagement >> Ereignisdefinition . 

In der Ereignisdefinition ist es möglich, beliebige Ereignisse in ganz eNVenta zu definieren. Diese Ereignisse müssen nun nicht erst umständlich programmiert werden, sondern können über die Oberfläche von eNVenta ERP definiert werden. Hierfür ist es notwendig, die benötigten Komponenten zu registrieren. Erst nach der Registrierung stehen die Komponenten in der Combobox zur Verfügung. Die Event Komponenten sind in der Ereignisdefinition aufrufbar über Stamm >> Event Komponenten . 

In der Ereignisdefinition ist es möglich einen Namen und eine Beschreibung anzugeben, sowie die betreffende Komponente auszuwählen. Für das Ereignis sollte eine möglichst sprechende Bezeichnung angegeben werden, damit diese in der Aktionsdefinition einfach zugeordnet werden kann. 

Zudem kann entschieden werden, ob das Ereignis nur zutrifft, wenn das Objekt neu eingefügt, geändert, oder gelöscht wurde. 

Siehe Kapitel Event Komponenten .

### Button Drucken 

Dieser Button startet den Druck . Der Button wird erst aktiv, wenn Daten eingelesen sind. 

### Button Suche 

Dieser Button startet die Suche nach Daten; entweder ganz allgemein oder wenn Sie im Kopf des Dialogs entsprechende Selektionskriterien festgelegt haben, gezielt. 

### Button Neu 

Dieser Button leert die Daten in den Kopf-Feldern. 

Ist in den Parametern für die Sachbearbeiter die Checkbox 2 Schritt Neu aktiviert, bleiben die Suchkriterien in den Stammdialogen erhalten. 

Siehe Kapitel Parameter Sachbearbeiter . 

### Button Kopieren 

Dieser Button startet die Kopierfunktion. Es werden sämtliche Felder kopiert, welche keine Schlüsselfelder sind. 

### Button Speichern 

Dieser Button speichert Ihre Eingaben. 

### Button Vorheriger Satz 

Mit diesem Button können Sie rückwärts blättern. Die Reihenfolge beim Blättern bezieht sich auf die Registerkarte Tabelle , sofern diese vorhanden ist. 

### Button Nächster Satz 

Mit diesem Button können Sie vorwärts blättern. Die Reihenfolge beim Blättern bezieht sich auf die Registerkarte Tabelle , sofern diese vorhanden ist. 

### Button Löschen 

Dieser Button löscht Ihre Eingaben. Genauer gesagt, die markierte Zeile wird mit dem versehen und erst endgültig gelöscht, wenn Sie den Button drücken. 

### Button Schließen 

Dieser Button schließt den Dialog. In manchen Fällen werden Sie gefragt, ob Sie die Eingaben / Änderungen noch vorher speichern möchten. 

### Button Export 

Mit diesem Button gelangen Sie in den Dialog Export Event Management . Aus diesem Dialog wird der Export aus dem System zentral gesteuert.

Bezeichnung des Ereignisses 

### Feld Beschreibung 
Ist dem Ereignis eine Beschreibung hinterlegt, wird der Text hier angezeigt. 

Tooltip: 

Beschreibung des Events

### Combobox Komponente 
Hier wird die Komponente des gewählten Ereignisses angezeigt. 

Bevor Sie hier Komponenten in die Ereignisdefinition einbinden können, sind diese in eNVenta anzulegen. 

Siehe Kapitel Ereignis Komponenten . 

Tooltip: 

Auswahl der Komponente, an der das Event definiert werden kann. 

Diese Komponenten können im Dialog "Event Komponenten" registriert werden. 

### Combobox Gruppe 
Dies ist die eindeutige ID dieser Konfiguration (GUID) für den Dateinamen. Ist noch keine ID gesetzt, handelt es sich noch um einen neuen, ungespeicherten Datensatz. 

Tooltip: 

Gruppe, welcher das Ereignis zugeteilt wurde. 

### Checkbox Aktiv 
Über diese Checkbox wird gesteuert, ob das Ereignis zur Laufzeit ausgewertet werden soll. 

Tooltip: 

Aktivieren oder deaktivieren Sie das Event 

### Checkbox Eingefügt 
Wenn Sie diese Checkbox aktivieren, trifft das Ereignis nur zu, wenn das Objekt neu eingefügt wurde. 

Tooltip: 

Das Event ist nur gültig, wenn die ausgewählte Komponente neu ist. 
Beispiel: Sie erfassen eine neue Auftragsposition. 

### Checkbox Geändert 
Wenn Sie diese Checkbox aktivieren, trifft das Ereignis nur zu, wenn das Objekt geändert wurde. 

Tooltip: 

Das Event ist nur gültig, wenn die ausgewählte Komponente geändert wurde. 
Beispiel: Sie ändern eine existierende Auftragsposition. 

### Checkbox Gelöscht 
Wenn Sie diese Checkbox aktivieren, trifft das Ereignis nur zu, wenn das Objekt gelöscht wurde. 

Tooltip: 

Das Event ist nur gültig, wenn die ausgewählte Komponente gelöscht wurde. 
Beispiel: Sie löschen eine existierende Auftragsposition (kein Storno). 

### Feld Erfasser

### Feld Änderung

### Feld Sachbearbeiter 
Hier wird vom Programm automatisch der Sachbearbeiter eingetragen, der den Datensatz zuletzt geändert hat. 

Tooltip: 

Sachbearbeiterkürzel des letzten Änderers des Datensatzes. 

### Feld Test-Datensatz 
Hier können Sie das definierte Event testen. 

Beispiel:

Wird in dem Feld Test-Objekt der Name des Artikels angezeigt, können Sie über den Button Ihr Event testen.

### Button Testen 

Mit diesem Button können Sie das definierte Ereignis testen. Hierzu muss das Test-Objekt gefüllt sein. 

Achtung: Der Test kann nur ausgeführt werden, wenn das Test-Objekt einen gültigen Status hat. 

Beispiel: 

Im Ereignis ist nur "Eingefügt" aktivert. 

Somit darf das Test-Objekt noch nicht gespeichert sein.
