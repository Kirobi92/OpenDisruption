# EDI X Mapping 
Für den elektronischen Datenaustausch gibt es mehrere Datenformate, welche parallel existieren. Um die verschiedenen Datenformate (ASCII, csv, XML etc.) importieren bzw. exportieren zu können, gibt es ein konfigurierbares Mapping in eNVenta . Es ist möglich ein flexibles Mapping für den Import und Export zu definieren. Für den Datenaustausch wird standardmäßig in den Ressourcen von eNVenta das Setting für das XML-Format OpenTrans-Format 2.1 mitgeliefert. Dieses kann aus den Ressourcen des Brokers in eNVenta importiert werden. Wenn Änderungen am Format gemacht werden, so gelten diese immer ersetzend vom Standard. Dies bedeutet, auch bei Änderungen in zukünftigen Versionen werden diese nicht automatisch übernommen, was den Vorteil der Stabilität der Konfiguration hat. 

Um den unterschiedlichen Katalogformaten gerecht zu werden, gibt es keine fest vorgegebenen Feldzuordnungen. Die einzelnen Felder des digitalen Katalogs (Quelldatei) werden den jeweiligen Datenfeldern in eNVenta einmalig je Katalog zugeordnet. Dies wird MAPPING genannt.

### Ansprechpartner 
Ansprechpartner aus der ORDERS können mit den vorhandenen Ansprechpartnern eines Kunden abgeglichen, ergänzt und/oder gespeichert werden. eNVenta prüft die Ansprechpartner und übergibt sie bei ausgehenden Dokumenten über das X-Mapping. 

Werden in einer Nachricht mehrere Ansprechpartner mitgesendet, dann werden alle mit gesendeten Ansprechpartner gespeichert/geprüft. Grundsätzlich wird aber immer nur der erste Ansprechpartner aus der Nachricht in den Auftrag übernommen (Rechnungsanschrift, Lieferanschrift, Auftragskopf). 

Sind in den Kopfdaten und in den Positionsdaten Ansprechpartner enthalten, dann werden immer die jeweiligen Ansprechpartner aus den Positionsdaten (BUYER_PARTY, INVOICE_PARTY, DELIVERY_PARTY) weiterverarbeitet. 

In diesem Zusammenhang gibt es die Möglichkeit, eine Prozessregel für Partner zu definieren. 

### Basiswissen 
Der Daten-Import setzt sich in eNVenta aus zwei voneinander unabhängigen Modulen zusammen: 

- Business-Logik zu den eigentlichen betrieblichen Prozessen, wie z.B. Auftrag erzeugen (ORDERS), Auftrag ändern/stornieren (ORDCHG), Auftragsbestätigung (ORDRSP), Wareneingang (DESADV), Rechnung erstellen (INVOIC) etc. - X Mapper, um aus jedem beliebigen Daten-Format prozessrelevante Daten zu importieren bzw. exportieren. 
Der EDI-Adapter nutzt beide Komponenten um Daten aus EDI-Dateien zu lesen / schreiben und die entsprechende Kern-Business-Logik damit aufzurufen. Beim Export von EDI-Nachrichten gibt es eigentlich keine Prozess-Logik, sondern die entsprechenden Daten aus dem Beleg müssen über den X Mapper „raus geschrieben“ werden. Diese Daten werden über ein eigenes Objekt dem X Mapper zur Verfügung gestellt. Der EDI-Adapter kümmert sich um das Laden dieser Daten und nutzt den X Mapper, um diese in den Datenexport zu übertragen.

Die EDI-Datei, welche importiert werden soll, plus das X Mapping erzeugen das Datenobjekt. Das Datenobjekt plus die Prozessregeln ergeben das fertige Objekt in eNVenta . 

1) Prozess-Input: Daten + Regeln 
Grundsätzlich erwartet ein Prozess zwei Dinge als Input: Erstens, die Daten für den Prozess und zweitens, ein Regelwerk in welcher Variation der Prozess ausgeführt werden soll. 

Daten 
Die Daten entsprechen einem „Formular“ für einen bestimmten Prozess (z.B. Auftrag erfassen). Sie kommen in der Regel von „außen“ z.B. über eine EDI Nachricht oder Webservice-Anfrage.Diese Daten werden über eine eigene Komponente (Klasse) bereitgestellt. Eine Instanz dieser Komponente (Klasse) muss mit Daten gefüttert werden. Dies kann z.B. über den X Mapper aus einer Datei erfolgen. 

Regeln 
Jeder Prozess erwartet neben dem Daten-Objekt mit den „Formular“-Daten auch ein Objekt mit den Regeln für den Prozess. Die Prozessregel definiert die Variation des Prozesses und könnte mit den Richtlinien verglichen werden, die ein Sachbearbeiter von seinem Vorgesetzten erhält. Diese Regeln werden in eNVenta intern definiert, um den automatischen Prozess zu konfigurieren. 

2) Prozess-Output: Prozess-Ergebnis + Protokoll 
Prozess-Ergebnis 
Je nach Prozess ist das Ergebnis-Objekt von einem anderen Typ – z.B. beim Prozess „Auftrag erzeugen“ ist das Ergebnis-Objekt der neu erzeugte Auftrag. (Ein Prozess kann auch kein Ergebnis-Objekt haben). 

Protokoll 
Neben dem Prozess-Ergebnis-Objekt liefert der Prozess auch einen Prozess-Status (erfolgreich, teil-erfolgreich oder fehlerhaft) zurück und außerdem ein Protokoll-Objekt mit den entsprechenden, benutzerfreundlichen Meldungen zum Ablauf des Prozesses. 

### Vorgehensweise Import EDI 
Für den Import einer Datei müssen folgende Voraussetzungen immer erfüllt sein: 

- Der Absender muss bekannt und eindeutig identifizierbar sein. - Dem Absender muss ein eindeutiges EDI-X-Mapping zugeordnet sein. - Dem Absender müssen Prozessregeln zugeordnet sein, nach welchen Kriterien die Logikdaten aus den Prozessdaten erzeugt werden. 
Für die Erkennung des Absenders muss ein X Mapping eingerichtet sein. Sollte dies nicht vorhanden sein, so muss ein neues X Mapping eingespielt werden. Für das Einspielen des Mappings gibt es mehrere Möglichkeiten: 

- Import Standard-Mapping - Einlesen der XML-Datei, welche importiert werden soll, und daraus die automatische Erkennung der Struktur. - Manuelle Anlage des Mappings. 
zu 1. 
Im Dialog X Mapping Auswahl kann mittels der Checkbox Aus Vorlage das X Mapping aus den mitgelieferten Vorlagen übernommen werden. Diese Mappings sind an das Standardformat OpenTrans2.1 angelehnt und liefern neben der Struktur der XML-Datei auch die Verknüpfung zu den eNVenta -Objektstrukturen. Mit Übernahme eines solchen Mappings wird eine Kopie davon erstellt und manuelle Änderungen an der Struktur können vorgenommen werden. Es gibt zusätzlich auch Strukturen für den Import von csv- bzw. ASCII-Formaten. Anhand der Bezeichnung der Struktur ist die Verwendung erkennbar anhand der Bezeichnung der Struktur. 

zu 2. 
Per Drag&Drop kann eine beliebige XML-Datei in den rechten Bereich des Dialogs X Mapping Editor eingefügt werden. Sobald diese Datei eingelesen wurde, wird im rechten Bereich des Dialogs die Struktur der Datei angezeigt. Nun kann manuell noch die Zuordnung der einzelnen Eigenschaften der XML-Datei (Tags) zu eNVenta erfolgen. Hierzu müssen die Eigenschaft eNVenta im linken Baum und der Tag im rechten Baum markiert sein und der Button über der Zuordnungstabelle im mittleren Bereich geklickt werden. Alternativ kann per Drag&Drop die eNVenta -Eigenschaft mit dem Tag verbunden werden. 

Alternativ zum Einlesen der Datei per Drag&Drop kann mittels Button über dem rechten Bereich, eine Datei eingelesen werden. Hierzu öffnet sich der Dialog Server Datei Auswahl , um eine XML-Konfiguration einzulesen. Leserechte auf dem Verzeichnis werden hierbei vorausgesetzt. 

Diese Funktion wird nur für XML-Dateien unterstützt, weitere Datenformate müssen manuell oder über eine Vorlage angelegt werden. 

zu 3. 
Mit dem Button über dem rechten Bereich beginnen Sie, die Struktur anzulegen. Immer unterhalb der Ebene der aktuellen Markierung wird bei erneutem Klick auf den Button ein neuer Tag / eine neue Ebene hinzugefügt.
