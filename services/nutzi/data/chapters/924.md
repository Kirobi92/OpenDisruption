# To-Do Liste Registerkarte Allgemein 
ToDo Liste Registerkarte Allgemein 

Auf dieser Registerkarte pflegen Sie die einzelnen Tätigkeiten, die zu erledigen sind. 
### Button Offen 

Mit diesem Button wird die Position in den Status offen gesetzt. Dies kann allerdings nur der Projektleiter veranlassen. 

Tooltip: 

Setzt die Position in den Status "Offen". 

### Button Wartet 

Wartende ToDos müssen selbstständig geprüft werden, ob hier evtl. eine andere Person bzw. Organisation die Aufgaben erledigt hat. 

Tooltip: 

Kennzeichnet ToDo zur Prüfung. 

### Button Genehmigt 

Dieser Button setzt den Status auf genehmigt . Der Name des Sachbearbeiters wird dokumentiert. 

Tooltip: 

Setzt die Position in den Status "genehmigt". 

### Button Abgelehnt 

Dieser Button setzt den Status auf abgelehnt . Der Name des Sachbearbeiters wird dokumentiert. 

Tooltip: 

Setzt die Position in den Status "abgelehnt". 

### Button Erledigt 

Dieser Button setzt den Status auf erledigt . Der Name des Sachbearbeiters wird dokumentiert. 

Tooltip: 

Setzt die Position in den Status "erledigt". 

### Button Geprüft 

Dieser Button setzt den Status auf geprüft . Der Name des Sachbearbeiters wird dokumentiert. 

Tooltip: 

Setzt die Position in den Status "geprüft". 

### Button Abgeschl. 

Mit diesem Button setzen Sie das ToDo auf abgeschlossen . Der Name des Sachbearbeiters wird dokumentiert. 

Tooltip: 

Setzt die Position in den Status "abgeschlossen". 

### Button Gen. Call 

Mit diesem Button wird ein neuer Call mit Bezug zu diesem Projekt generiert. Dabei wird der Projektkunde als Kunde in den Call übernommen, ebenso wie die Projektnummer. Die Beschreibung aus der ToDo Liste wird in das Thema des Calls übernommen. Die erzeugte Call-Nummer muss in die ToDo Liste eingetragen werden. 

Wenn mehrere Kunden zur Auswahl stehen, öffnet sich zunächst der Dialog Kundenauswahl zur Selektion eines Kunden. Wenn Sie hier einen Kunden auswählen, wird dieser in den Dialog Call übernommen, ebenso wie die Projektnummer. 

Ist bereits eine Call-Nummer eingetragen, so wird kein neuer Call generiert, sondern der bereits vorhandene Call aufgerufen. In diesem Fall heißt der Button auch einfach Call . 

Zusätzlich wird die Beschreibung aus dem ToDo-Eintrag in das Thema des Calls übertragen. Die erzeugte Call-Nummer muss in den ToDo-Datensatz eingetragen werden. Ist bereits eine Call-Nummer eingetragen, so wird kein neuer Call generiert, sondern einfach der vorhandene aufgerufen. Der Button heißt auch in diesem Fall einfach Call . 

Tooltip: 

Call generieren 

### Button Aufgabe 

Ist noch keine Aufgabe zugeordnet, so erscheint ein Dialog zur Auswahl einer Aufgabe aus diesem Projekt und somit wird eine Zuordnung möglich. Die beiden Felder werden gefüllt. Sonst wird die bereits zugeordnete Aufgabe geöffnet. 

Siehe Kapitel Projektaufgabe . 

Tooltip: 

Projektaufgabe zuweisen 

### Tabelle 

### Tabellenspalte Pos 
Hier wird die Nummer der Position eingetragen. 

Tooltip: 

Positionsnummer 

### Tabellenspalte S 
Hier wird die Ziffer des Status eingetragen. Zur Verfügung stehen: 
- Neu - Offen - Wartet auf ... - Genehmigt - Erledigt - Geprüft - Abgeschlossen - Abgelehnt - Gelöscht 

Tooltip: 

Status: 

1 = Neu 
2 = Offen 
3 = Wartet auf... 
4 = genehmigt 
5 = erledigt 
6 = geprüft 
7 = abgeschlossen 
8 = abgelehnt 
9 = gelöscht 

### Tabellenspalte Status 
Hier wird zur Status-Ziffer der Name des Status eingetragen bzw. angezeigt. 

Tooltip: 

Bezeichnung des Status 

### Tabellenspalte Bereich 
Hier können Sie eine frei zu definierende Einteilung der ToDos nach Bereichen vornehmen. Die Liste stellt alle bisher eingetragenen Bereiche für das Projekt zur Auswahl zur Verfügung und kann entsprechend ergänzt bzw. überschrieben werden. 

Tooltip: 

Geben Sie einen Bereich ein für die Einteilung des ToDos. 

### Tabellenspalte Beschreibung 
Hier können Sie zum Bereich noch eine Beschreibung eingeben. 

Tooltip: 

Geben Sie eine Beschreibung zum Bereich ein. 

### Tabellenspalte Klasse 
Zur weiteren Spezifikation kann hier eine Klasse ausgewählt werden. Diese Klassen werden im Dialog ToDo Klassen definiert. Dieser Dialog öffnet sich über Projekt >> Stamm >> ToDo Klassen . 

Siehe Kapitel ToDo Klassen . 

Tooltip: 

Wählen Sie eine Klasse zur weiteren Spezifikation. 

### Tabellenspalte Priorität 
Hier ordnen Sie dem ToDo eine Priorität zu. Prioritäten können frei definiert werden. Vorschlag: 1-9 (1 ist die höchste Priorität) 

Tooltip: 

Priorität von 1 bis 9 

1 bedeuted: Hohe Priorität 

### Tabellenspalte Aufgabe 
Hier stellen Sie die Verknüpfung zu einer Projektaufgabe her. Über den Button Aufgabe kann diese geöffnet bzw. eine bestehende Projektaufgabe zum ToDo verknüpft werden. 

Tooltip: 

Stellen Sie eine Verknüpfung zu einer Aufgabe her. 

### Tabellenspalte Call 
Hier können Sie eine Verknüpfung zu einem bestehenden Call herstellen und diesen aufrufen. Wenn das Feld leer ist kann ein neuer Call aus dem ToDo heraus angelegt werden. 

Tooltip: 

Angabe der Call-Nummer 

### Tabellenspalte Intern 
Wenn Sie die Checkbox in dieser Spalte aktivieren, ist das ToDo nur für interne Zwecke und wird nach außen nicht angezeigt. 

(Ein Zugriff von außen müsste entsprechend ausprogrammiert werden, bzw. im Reporting berücksichtigt werden.) 

Tooltip: 

Wenn aktiviert, wird das ToDo nur intern verwendet. 

### Tabellenspalte Kritisch 
Wenn Sie die Checkbox in dieser Spalte aktivieren, kennzeichnen Sie das ToDo als kritisch. 

Tooltip: 

Wenn aktiviert, gilt das ToDo als kritisch. 

### Tabellenspalte Ansprechpartner

Geben Sie eine Kontaktperson an.
