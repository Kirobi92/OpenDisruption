# Lohndaten übernehmen 
Adapter Lohndaten 

In der OP-Übernahme öffnet sich über das Menü Optionen >> Lohndaten importieren der Dialog Adapter Lohndaten . Der Adapter dient der Transformation von Daten aus einer Datei in Datensätze des Belegjournals. Die Lohnimport-Schnittstelle wird über das Gateway eingebunden. Voraussetzung für die Nutzung dieses Menüs ist die Lizensierung des Lohnadapters. 

Der Import-Vorgang läuft in mehreren Schritten ab: 

- Öffnen des Dialogs - Wahl des zu importierenden Formats (Datev10/Datev12) - richtet sich danach, ob die Import-Datei 10 oder 12 Spalten enthält. - Selektieren einer Datei - Eingabe von Belegnummer und ggf. Buchungstext (bitte dazu die Erläuterungen für das Feld Buchungstext bachten!); das Buchungsdatum wird mit dem Tagesdatum vorbelegt. - Betätigen des Buttons Einlesen : Die Daten werden in die Tabelle eingelesen. Die Datensätze werden nach den in den Anforderungen definierten Prüfroutinen validiert, Fehler im Fehlertext angezeigt. Fehlerhafte Datensätze werden rot , korrekte grün dargestellt. Ein Fehler während dieses Vorgangs hat zur Folge, dass der Datensatz, bei dem der Fehler auftritt, nicht in die Tabelle geladen wird. Dem Gateway-Protokoll, welches über den Button Protokoll geöffnet wird, können nähere Informationen entnommen werden. Pro Adapter können immer nur Einträge eines Tages angezeigt werden, was zur Folge hat, dass evtl. Datensätze mehrerer Import-Läufe zu sehen sind. - Datenmanipulation durch den Benutzer (nach jeder Wertänderung einer Spalte wird die Prüfroutine erneut durchgeführt). - Durch Betätigen des Buttons Verarbeiten werden die Datensätze ins Belegjournal gespeichert. Datensätze, welche nicht gespeichert werden konnten, bleiben in der Tabelle stehen, der entsprechende Fehler ist wieder dem Protokoll zu entnehmen. - Nach dem Schließen des Dialogs werden die gespeicherten Sätze in der Übernahme geladen. Somit kann der Benutzer die entsprechenden Sätze sofort buchen.

### s+p Lohndaten 
Um Lohndaten ohne Anpassung der Datei in eNVenta einlesen zu können, gibt es das Format s+p Lohndaten . 

Datei-Beschreibung für das Format s+p Lohndaten: 
Typ CSV-Datei, Komma-getrennt, Datei enthält 9 Spalten 

Spalte Feldbezeichnung Beispiel eNVenta Adapter Lohndaten: R'karte Import Lohn, Tabelle, Spalte 
| | | | |
| --- | --- | --- | --- |
| 1 | Betrag | 1000,12 | Betrag |
| 2 | Gegenkonto | 1755 | => Haben-Konto |
| 3 | Belegnummer | 1 | Belegnummer: ist das Dialog-Eingabefeld leer, wird der Importwert übernommen. |
| 4 | Belegfeld 2 | 000000 | nicht zugewiesen |
| 5 | Buchungsdatum | 2412 | => Belegdatum: Import (TTMM) wird um Geschäftsjahr ergänzt. |
| 6 | Konto | 3790 | => Soll-Konto |
| 7 | Kost1 | 1507 | Kostenstelle |
| 8 | Kost2 | - | nicht zugewiesen |
| 9 | Buchungstext | SV Beiträge AN (KV) | Buchungstext: ist Dialog-Eingabefeld leer, wird Importwert übernommen |

Bitte beachten: Gilt nur für das Format s+p Lohndaten, Dialog Eingabefeld : 

- Belegnummer - Buchungstext 
Ist das Dialog-Eingabefeld leer, wird der Import-Wert übernommen.
