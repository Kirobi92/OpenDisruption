# Zinsrechnung Registerkarte Zinsvorschlag 
Zinsrechnung Registerkarte Vorschlag 

Auf dieser Registerkarte werden die Daten entsprechend Ihrer Selektionskriterien eingelesen. 
### Button Löschen 

Mit diesem Button wird ein offener Posten vom Zinslauf ausgeschlossen. 

Tooltip: 

Aus diesem Lauf ausschließen 

### Button Beibehalten 

Wenn Sie auf diesen Button klicken, werden alle Zeilen - außer den selektierten - gelöscht. 

Tooltip: 

Alle Zeilen außer den selektierten löschen. 

### Button Ausschluss 

Mit diesem Button wird ein offener Posten grundsätzlich vom Zinslauf ausgeschlossen. 

Tooltip: 

Nie berechnen 

### Button Nullwerte löschen 

Mit diesem Button werden alle Null-Werte vom Lauf ausgeschlossen. 

Tooltip: 

Alle Nullwerte aus diesem Lauf löschen. 

### Combobox Zinslauf 
Hier kann ein gespeicherter/angelegter Vorschlag ausgewählt werden. 

### Feld Zinslauf 
Hier wird die Bezeichnung angegeben. 

Tooltip: 

Angabe der Bezeichnung 

### Checkbox Wiederholung 
Wenn Sie diese Checkbox aktivieren, haben Sie die Möglichkeit bereits versendete Zinsrechnungen erneut zu erstellen. 

Bei einer Selektion mit dieser aktivierten Checkbox werden in der Tabelle nur bereits abgearbeitete Zinsrechnungen angezeigt. 

### Feld Summe Zinsb. 
Hier wird die Summe über die angefallenen Zinsen angezeigt. 

Tooltip: 

Summe Zinsbetrag 

### Tabelle Linke Tabelle 

In dieser Tabelle wird der Kunde mit den summierten Einzel-OPs dieses Zinsvorschlags ausgewiesen. 

Mit einem Klick auf die linke obere Ecke werden alle OPs dieses Zinsvorschlags in der rechten Tabelle angezeigt. 

Klicken Sie auf einen Kunden, werden nur diese OPs in der rechten Tabelle angezeigt. 

Sie können auch blockweise oder selektiv Kunden markieren, hierzu werden dann in der rechten Tabelle die OPs angezeigt.

### Tabellenspalte E-Mail FiBu 
Hier wird die E-Mail-Adresse der Abteilung 'Finanzbuchhaltung' angegeben. 

Um sicherzugehen, dass eine E-Mail an die richtige Adresse gesendet wird, prüft eNVenta beim Speichern oder Ändern einer Mailadresse, ob die Mail-Adresse der „Norm“ entspricht. Falls nicht wird ein Hinweis ausgegeben. Dabei wird das Speichern nicht grundsätzlich verhindert. Auf diese Weise werden z.B. Outlook-Adressbucheinträge wie z.B. „#EW_ERP“ bei Bedarf gespeichert und genutzt. 

Dabei läuft die Prüfung ausschließlich bei Anlage und Änderung einer Mail-Adresse. Werden andere Einträge, z.B. des Kundenstammsatzes geändert, so kommt diese Meldung nicht. 

Werden Stammdaten über das Gateway importiert, so werden u.U. fehlerhafte Mailadressen importiert (ggfs. mit Warnung). Bei einem Import unter Verwendung des Jobservers erfolgt dies ebenfalls in Form einer Warnung (im Jobprotokoll). 

Tooltip: 

Angabe der E-Mail-Adresse der Fibu-Abteilung
