# Betriebsauftrag Registerkarte Kopfdaten 
Betriebsauftrag Registerkarte Kopfdaten 

Auf dieser Registerkarte definieren Sie die Einstellungen für das Lager. 

Im Bereich Terminierung können Sie eine Ziffer für die Priorität (z.B. 1 für die höchste Priorität), den frühesten bzw. einen fixen Termin eingeben. 

Die Angabe der Priorität und der Termine ist keine zwingende Eingabe, wird aber bei der Nutzung der Masseneinplanung als Grundlage benötigt. Damit dies möglich ist, gibt es in den Produktions-Parametern die Möglichkeit eine Standard-Priorität vorzugeben, welche auf die Betriebsaufträge angewendet wird, falls dort keine Priorität angegeben wurde. 
### Combobox Verkauf 
Wählen Sie aus dieser Combobox das Lager aus, auf welches die Waren nach der Produktion gebucht werden. Voreingestellt ist das Lager aus den Parametern.

### Combobox Phase 
Zu einem eingetragenen Projekt im Betriebsauftrag wird in dieser Combobox die Projektphase eingelesen. Die Zuordnung eines Betriebsauftrags ist phasenbezogen möglich, um eine versetzte Einführung / Fertigung zu ermöglichen. 

Wenn der Betriebsauftrag einer Projektphase zugeordnet ist, so wird der Betriebsauftrag auch in der Projektverwaltung Register Phasen angezeigt.

### Feld Priorität 
Hier können Sie eine Ziffer für die Priorität eingeben (z.B. 1 für die höchste Priorität). Der Eintrag in diesem Feld hat Einfluss auf die Sortierreihenfolge bei der Neuplanung von Betriebsaufträgen. 

Bleibt dieses Feld leer, zieht die Einstellung der Priorität aus den Parametern der Produktion (Feld Priorität Standard ). 

Siehe Kapitel Parameter Produktion .

### Feld Frühester Termin 
Hier geben Sie ein Datum ein für wann der Betriebsauftrag frühestens einzuplanen ist. 

### Feld Fixer Termin 
Hier geben Sie ein Datum ein, das bei der Neuplanung von Betriebsaufträgen die Sortierreihenfolge beeinflusst. 

### Checkbox Gesperrt 
Über diese Checkbox können Sie einen Betriebsauftrag sperren. 

Wird ein Betriebsauftrag gesperrt, der bereits freigegeben oder terminiert ist, so wird der Betriebsauftrag geparkt, d.h. alle Terminierungen und Ressourcen-Reservierungen werden gelöscht; Material-Reservierungen bleiben bestehen. 

Ist ein Betriebsauftrag gesperrt, kann dieser nicht neu eingeplant werden. Auch eine automatische Einplanung oder Verschiebung von Betriebsaufträgen ist dann nicht möglich. 

Solange es noch keine Buchungen zum Betriebsauftrag gibt, wird er nicht auf geparkt gesetzt, sondern auf offen . 

Tooltip: 

Aktiviert: Setzt den Datensatz auf 'gesperrt'.
