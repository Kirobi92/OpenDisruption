# Client Side Customizing Console 
Client Side Customizing Console 

Um bei Umgebungen mit vielen Rollen und Benutzern die Bedienung zu vereinfachen, werden bei der Auswahl eines Scopes Listboxen statt Comboboxen verwendet. Der Dialog ist auch etwas größer. 

Im Feld Current Trigger steht die aktuelle Angabe des Triggers, für welche nun eine Änderung im ClientSideCustomizing vorgenommen wird bzw. wurde. 

Über die Trigger kann der Dateninhalt beliebiger Felder überwacht werden und aufgrund der angegebenen Daten werden dann die Aktionen des Triggers an der Oberfläche ausgeführt. Auf diese Weise kann z.B. im Auftrag eingestellt werden, dass bei Auftragserteilung via Fax (Feld Bestellart ) eine Faxnummer im Auftrag eingegeben sein muss oder das Feld Projektnr farbig hervorgehoben wird. 

### Einfügen von Tabellenbedienelementen 
Es ist möglich, dynamisch neue Tabellenspalten ohne Programmieraufwand in den eNVenta ERP -Dialogen hinzuzufügen. 

Jeder zuständige IT-Mitarbeiter kann mit dem Spaltenkonfigurationsassistenten einfach und effizient neue Spalten in Tabellenbedienelementen (Grid) hinzufügen, wenn diese an der am Tabellenbedienelement registrierten Klasse als Attribut vorhanden sind und nicht bereits im Grid existieren. Für das Hinzufügen einer Spalte muss das Grid (Tabelle) markiert sein, anschließend steht die Eigenschaft ChildControls zur Verfügung. 

Dabei kann der Anwender alle Standardwerte einer Spalte definieren, die der Entwickler normalerweise in der Entwicklungsumgebung setzen kann. 

Dazu gibt es im TreeView (Baumstrukturbedienelement) die Möglichkeit, die Datenquelle, zu der eine Spalte gewünscht ist, zu wählen. Hierbei ist auch zu beachten, dass der Bezeichner der Datenquelle den Anzeigenamen und den Attributnamen enthält, damit die Eigenschaften für den IT-Mitarbeiter schnell identifizierbar sind. 

Beispiel: 
Child Controls Choose => (grdTable_oArticleDescription_shtLanguageID) 

-> Die Sprache würde mit Nummer (ID) angezeigt werden. Wenn Sie die Bezeichnung der Sprache wollen, ist dies zwingend einzustellen (dasselbe mit Dispo KZ usw.). 
### Vorgehensweise zum Einrichten 
Table Column Wizard 

In der Administrationsoberfläche des Client Side Customizing muss, wenn ein Tabellenbedienelement im Baumbedienelement gewählt wurde, auf dessen Registerkarte Properties als neue Eigenschaft „childControls“ angelegt werden. Diese enthält einen ListView (Listenbedienelement) mit den existierenden dynamischen Spalten. 

Ein Klick auf eine bestehende Spalte öffnet den Spaltenkonfigurationsassistent mit den Standardwerten (read-only / nur lesbar). 

Ein Klick auf den Button „+“ öffnet den Assistent im Hinzufügemodus. 

Auf der ersten Seite wird der Datenquellen-Treeview dargestellt, in welchem per Klick ein Attribut ausgewählt wird oder falls notwendig ein Event zum Nachladen des Baumes gefeuert. 

Die Registerkarte heißt Datenquellenauswahl (Datasource selection) und hat einen Abbrechen - (Cancel) und Weiter -(Next)Button haben. 

List Datasource zeigt den gewählten technischen Namen an. 

List Display Attribute zeigt den anzuzeigenden Wert an. 

List Value Attribute gibt den in der Datenbank gespeicherten Wert an. Mit diesem Wert wird die Anzeige (List Display Attribute) geladen. 

Ändert der Anwender den Namen oder den Text des Tooltips nicht, werden die vom Inhalt her identischen Attribute leer in die Datenbank gespeichert und können somit mehrsprachig aus dem Metadatentyp gelesen werden. 

Bestehende Änderungen müssen auf der Registerkarte Changes (übersetzt Änderungen) angezeigt werden.
