# OCI-Technische Anbindung 
Die OCI-Schnittstelle ist separat zu lizenzieren! 

### Was ist OCI? 
OCI (Open Catalog Interface) ist eine Schnittstelle von SAP, welche eine direkte Einkaufsmöglichkeit aus SAP in einem Katalogsystem (z.B.: NVShop) bereitstellt. 

Die eigentliche Bestellung erfolgt dann nicht per Übergabe des Warenkorbes in eNVenta , sondern wird zuerst in das Einkaufsmodul von SAP übernommen und wird zu einem späteren Zeitpunkt schriftlich, per Telefon, per Fax oder Email an den Shopbetreiber übermittelt. Der Bestellprozess des NVShops wird bei der OCI Übergabe nicht benötigt. 
### Vorgehensweise 

#### Voraussetzungen: 

- Es wird nur die Version „SRM Server Application“ von SAP unterstützt. - Andere Versionen von SAP müssen durch Entwicklungsaufwand des SAP Entwicklers angebunden werden. - Es wird nur OCI Version 4.0 unterstützt (OCI 4.1 beinhaltet eine XML-Struktur, 4.0 ist eine reine Html-Struktur) 
Folgende Felder müssen im SAP SRM angelegt werden. Die Benennung der Felder muss exakt übernommen werden: - User - Password - OCI_VERSION (optional) - OPI_VERSION (optional) - HOOK_URL (Diese wird bei SRM Server automatisach übermittelt - http_content_charset - returntarget(optional) 
Die Übertragung dieser Felder erfolgt als http Get Anfrage aus dem SAP in den NVShop auf die Seite ~/OCI/OCILogin.aspx. 

~/OCI/OCILogin.aspx (~ steht für die Domain, unter welcher der Shop läuft z.B.: www.nissen-velten.de/oci/ocilogin.aspx) 

### Einstellungen in eNVenta ERP 

#### Benutzerverwaltung: 
- Für den Kunden, welcher OCI verwenden möchte, ist zwingend ein zusätzlicher Benutzer anzulegen, welcher vom Typ „Multiuser“ ist. - Dieser Benutzer ist nicht dafür gedacht, um sich damit am Shop manuell anzumelden. - Der MultiUser ist so konfiguriert, dass mehrere Sachbearbeiter parallel mit dem selben Benutzeraccount am NVShop arbeiten können. 
Siehe Kapitel Konfiguration der Webshop Benutzer Registerkarte OCI . 

### Hinweise: 
Es ist zwingend erforderlich, dass der Browser des Sachbearbeiters, welcher mit OCI arbeiten möchte Cookies zulässt. Der Membershipprovider von ASP.net erfordert zwingend Cookies. Sind im Browser des Sachbearbeiters die Cookies deaktiviert, so schlägt die Anmeldung am NVShop fehl.
