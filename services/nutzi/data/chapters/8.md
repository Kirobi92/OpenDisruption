# Client Side Customizing (CSC) 
Das Client Side Customizing öffnet sich über den Form-Menü-Button ( ) und dient der einfachen und schnellen Anpassung der Benutzeroberfläche an die Gegebenheiten Ihres Unternehmens. Eingabefelder und Tabellenspalten können neu positioniert werden, unsichtbar geschaltet werden oder Beschriftungen und Farben geändert werden, um nur einige Beispiele zu nennen. Alle Einstellungen können auch an Bedingungen geknüpft werden. So ist es z.B. möglich, abhängig von der ausgewählten Auftragsart andere Eingabefelder mit Standardwerten zu versehen oder Pflichtfelder festzulegen, welche beim Speichern gefüllt sein müssen. 

Es gibt 4 Gültigkeitsbereiche (Scopes), für die Änderungen definiert werden können: 

- für alle Benutzer - für alle Benutzer, die den Client in einer bestimmten Sprache starten - für alle Benutzer, die eine bestimmte Rolle besitzen - für einen bestimmten Benutzer 
Analog der obigen Liste übersteuern Einstellungen für einen bestimmten Benutzer die rollenspezifischen Einstellungen, welche wiederum die sprachspezifischen Einstellungen übersteuern. Niedrigste Priorität haben die allgemeinen Einstellungen für alle Benutzer. 

Ein Trigger stellt eine Bedingung dar, unter der Einstellungen im Client Side Customizing angewendet werden. Ein Trigger wird durch Control und Wert definiert. Das heißt, dass alle dem Trigger zugeordneten Einstellungen dann angewendet werden, wenn an dem Control der Wert zugewiesen wurde. 

Beispiel: 
Der Trigger mit der Bezeichnung „edtsFormOfAddress: Herr“ bedeutet, dass am Control edtsFormOfAddress (Andrede) der Wert „Herr“ zugeordnet sein muss, damit die dem Trigger zugeordneten Einstellungen angewendet werden. Es ist unerheblich, ob der Wert durch den Benutzer oder durch eNVenta selbst im Control eingetragen wird. 

Alle Einstellungen, die unabhängig von Triggern angewendet werden sollen, müssen ohne die Angabe eines Triggers vorgenommen werden. 

Als Triggercontrol können folgende Controls verwendet werden: 

- einzeilige Textfelder (die kein Passwort und kein Datum darstellen) - Checkboxen - Listboxen (die keine Mehrfachauswahl erlauben) - Comboboxen 
Grundsätzlich gilt, dass Einstellungen dann gültig werden, wenn sowohl geforderter Scope als auch geforderter Trigger aktuell Gültigkeit haben. 

Achtung: 
Zwei Trigger, welche die gleiche Eigenschaft eines Controls verändern, sind grundsätzlich problematisch! Das Verhalten ist wie folgt: 

Wird ein Trigger gültig, so werden die Eigenschaften zugeordnet. Verliert ein Trigger seine Gültigkeit und die Eigenschaft wurde zuletzt durch ihn geändert, so wird der Originalzustand wiederhergestellt, wie er vorgefunden wurde, bevor irgendein Trigger die Eigenschaft geändert hat. Diese Regel gilt nicht für Default Values. Diese werden niemals auf den Originalwert zurückgesetzt und nur einmalig gefüllt (siehe Absatz Default Values im Kapitel CSC Registerkarte Properties / Erklärung der Eigenschaften ). 

#### Weitere allgemeine Hinweise: 
Alle Änderungen bleiben beim Neustart von eNVenta ERP erhalten. 

Die Auswahl eines Controls erfolgt entweder durch Anklicken des Controls in eNVenta ERP direkt oder durch Auswahl in der Baumstruktur der Client Side Customizing Console. Ist ein Control ausgewählt, werden unterhalb der Baumstruktur in einer Tabelle alle veränderbaren Eigenschaften aufgeführt. 

Positionsänderungen können zur Laufzeit direkt im eNVenta ERP -Dialog oder in der Baumstruktur der Controls mit der Maus per Drag & Drop durchgeführt werden. Durch mehrfaches Klicken mit der Maus auf die gleiche Position im eNVenta ERP kann jeweils das nächste umschließende Control ausgewählt werden. Für alle Eigenschaften (Schriftart, Farbe, ...) stehen in der Client Side Customizing Console komfortable Editoren zur Verfügung. 

Der Zugriff für die Anpassung des Client Side Customizing wird über die AccessUnit mit dem technischen Namen auClientCustomizing („Client Side Customizing") gesteuert und steht am Besten nur dem Administrator zur Verfügung. 

Die Einstellungen der verschiedenen Ebenen sind frei kombinierbar. Die Prioritäten-Reihenfolge ist je spezieller desto höher. Die Einstellungen werden dauerhaft gespeichert und beziehen sich immer auf den Benutzer. D.h. egal von welchem Rechner der Benutzer sich anmeldet, es stehen ihm die gleichen Einstellungen zur Verfügung. 

Abhängig vom Typ des ausgewählten Controls (z.B. Textfeld, Combobox, Button, …) stehen Ihnen unterschiedliche Möglichkeiten der Veränderung zur Verfügung. Eine Übersicht über die änderbaren Eigenschaften finden Sie im Kapitel CSC – Registerkarte Properties . 

Änderungen im Client Side Customizing werden direkt im Dialog angezeigt (WYSIWYG). 

Das Verschieben von Menüeinträgen wird nicht unterstützt! 

Wenn Sie hierzu noch Fragen haben, wenden Sie sich an Ihren Administrator. 

Um die Bediener-Freundlichkeit zu erhöhen, bzw. bestimmte Pflichtfelder zu definieren, ist es grundsätzlich möglich, an folgenden Control-Types Standardwerte vorzubelegen und die Controls als Pflichtfeld zu deklarieren: 
- Editfield - Combobox - Listbox 
Außerdem lässt sich ein Standardwert für Checkboxen vorbelegen. 

Wird im Client Side Customizing „true“ oder „false“ als Defaultwert für eine Checkbox gesetzt, so wird dieser unabhängig vom ursprünglichen Wert gesetzt. 

Es findet keinerlei syntaktische Prüfung statt. D.h., es ist notwendig die Daten im richtigen Format anzugeben und im Sinne der Businesslogik einen gültigen Wert darzustellen. 

Es wird empfohlen neu vorgenommene Einstellungen sofort im Anschluss zu prüfen. 
#### Wichtiger Hinweis: 
Spalten und Felder, die von eNVenta nicht standardmäßig dargestellt werden, können von Ihnen "auf eigenes Risiko" sichtbar gemacht werden. Nissen & Velten kann nicht gewährleisten, dass die angezeigten Werte korrekt sind. 

Ein "Absturz" bzw. die Darstellung von unplausiblen Werten bedeutet, dass Sie diese Felder nicht verwenden sollte. 

Angaben zu Formaten: 

- Dezimalzahlen: Als Dezimaltrennzeichen ist zwingend der Punkt zu verwenden. - Datumsformat: TT.MM.JJJJ hh:mm:ss (Zeit ist optional) oder * für 'jetzt' (heute) - Boolsche Werte: z.B. bei der Checkbox true oder false
