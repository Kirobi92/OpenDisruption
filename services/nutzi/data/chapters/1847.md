# Inventur Registerkarte Inventurliste 
Inventur Registerkarte Inventurliste 

Auf dieser Registerkarte werden die Positionen für die Inventur eingefügt. 

Der Button Neu fügt eine neue Zeile in die Tabelle ein. Dabei wird unterhalb der markierten Zeile ein neuer Datensatz eingefügt, welcher eine Kopie der markierten Zeile enthält. Hierbei werden Lagerplatz, Artikelnr, etc. übernommen und können angepasst werden. 

Das Ist-Feld der Inventur wird mit dem Sollwert vorbelegt, wenn der Cursor in das Ist-Feld platziert wird und dort noch kein Wert drinsteht (NULL). Der Istwert ist markiert, so dass Sie ihn direkt überschreiben können. Stimmt er mit dem Sollwert laut Zählliste überein, so kann das Feld mit Enter oder Tab (oder Pfeil runter) verlassen werden, ohne dass Sie etwas eintragen müssen. Die Spalte 'Inventur-Ist' wird auf Rot gesetzt, wenn sich der Ist-Bestand um mehr als 50% nach oben oder unten vom Soll-Bestand unterscheidet. Diese Funktion wird aufgerufen, wenn Sie eine Suche anstoßen oder wenn sich eine Spalte innerhalb der Tabelle ändert. In diesem Fall kann es notwendig werden, nachzuzählen. Einstellungen dazu nehmen Sie auf der Registerkarte Kontrolle vor und drucken eine Nachzählliste (Kontrollliste) aus. Nach erfolgreicher Nachzählung ist die Checkbox in der Spalte 'NZ erl.' zu aktivieren. 

Wenn in einer initialisierten Inventur nachträglich eine Zusatzposition zu einem Chargenartikel eingegeben wird, ist zwingend auch die Chargennummer in die entsprechende Spalte einzufügen. Daraufhin öffnet sich der Dialog Chargenzugang , um die Charge zu buchen. Hier ist auch nur der Button zum Speichern zugänglich. Sämtliche Felder zur Chargengenerierung und die Buttons , , sind nicht zugänglich, da es sich um eine Leerchargen-Initialisierung handelt. 

Siehe Kapitel Chargenzugang . 

Die neue Position, welche manuell hinzugefügt wurde, erhält zusätzlich das Kennzeichen „Z-Pos“ (= Zusatzposition). 

Bei jeder neu hinzugefügten Zeile wird geprüft, ob diese in die Inventur aufgenommen werden darf. Hierfür müssen folgende Kriterien erfüllt sein, ansonsten wird dies mit einer Fehlermeldung abgelehnt: 
- Artikel darf nur einmalig in der Inventur vorhanden sein - Bei Diversartikeln muss die Bezeichnung eindeutig sein (Kennzeichen in der Spalte ‚D-Art.‘) - Eine Charge muss eindeutig sein (Kennzeichen in der Spalte ‚C-Art.‘) - Der Artikel darf in keiner anderen offenen Inventur enthalten sein 
Alle Positionen / ohne InvSoll = 0/NULL / nur InvSoll = 0/NULL: 

Die Radiobuttons sind nur für die Anzeige bei einer bestehenden Inventur vorgesehen und haben keine Auswirkung auf die Selektion der Artikel bzw. Lagerplätze. Standardmäßig ist der Radiobutton alle Positionen gesetzt. Sie können die Einstellung aber auch auf ohne InvSoll = 0/NULL setzen, d.h. es werden manuell ausgeschlossene Positionen (Status 5) und offene Chargenpositionen mit einem Inventur-Soll von 0 oder NULL ausgeschlossen. Bei einer Einstellung auf nur InvSoll = 0/NULL werden nur Positionen mit einem Buchbestand von 0 oder NULL angezeigt. 

Auch wenn alle Positionen verbucht wurden, muss der Abschluss der Inventur manuell getätigt werden, da noch immer Positionen manuell erfasst werden können.

### Button Markierte buchen 

Über diesen Button werden alle markierten Positionen der Inventur gebucht. Hierbei werden die entsprechenden Einträge ins Lagerjournal geschrieben und der Positionsstatus der markierten Zeilen auf 3 gesetzt. Bei der Verbuchung der Mengendifferenz wird der EK aus der Kalk-Basis des jeweiligen Artikels verwendet. 

Unmittelbar vor dem Verbuchen findet eine Sicherheitsprüfung statt, ob der Artikel noch im Artikelstamm existiert (da alle wichtigen Daten in separate Inventurtabellen geschrieben werden und somit losgelöst sind). War der Verbuchungsversuch erfolgreich (InvDiffMenge != Null/0 und Dispo-Flag != 4), dann beträgt der Positionsstatus 6; andernfalls 5. 

Anschließend erfolgen die Verbuchung der Chargen, die Platzzuordnung (bei chaotischen Lagern) und die Buchung der Differenzmenge. 

Wird eine Position erfolglos verbucht, weil die Inventur-Differenz = 0 ist, wird diese Position auf Status 5 gesetzt. Um eine solche Position wieder zu öffnen, markieren Sie die Zeile und drücken den Button Pos. öffnen . Diese Position ist dann wieder in Status 3. Jetzt kann erneut ein Wert eingegeben bzw. geändert werden. 

Tooltip: 

Bestandsbuchung über alle markierten Positionen

### Tabellenspalte Seite 
In der Inventur ist es möglich, über den Parameter Einträge pro Inv (Dialog Parameter Einkauf ) festzulegen, wie viele Artikel auf eine Seite gedruckt werden sollen. Die Angabe der Anzahl Artikel sollte entsprechend gewählt werden, damit die passende Anzahl an Artikeln auf einer Seite gedruckt wird. Im Zweifelsfall sollte die Zahl entsprechend kleiner gewählt werden. 

In dieser Spalte wird die konkret gedruckte Seite (laut Berechnung) angegeben. 

Beispiel: Ist in den Parametern des Einkaufs die Anzahl Artikel pro Seite mit 9 angegeben, so wird mit dem ersten Datensatz begonnen und die Seite 1 zugewiesen. Alle folgenden Datensätze erhalten ebenso die Seite 1 zugewiesen, bis die „Grenze“ von 9 erreicht ist. Der 10te Datensatz erhält nun die Seite 2, usw. 

Auf dem Report wird die Seitenzahl als Gruppe eingefügt. Die Gruppe ist so konfiguriert, dass beim Start einer neuen Gruppe automatisch eine neue Seite eingefügt wird. Im Gruppenkopf wird der Wert der Gruppe als Seite angedruckt.

### Tabellenspalte GEK 
Hier wird der gewichtete mittlere Einkaufspreis angegeben, der bei der Rechnungsprüfung neu errechnet werden kann. Dieser wird erst bei Abschluss einer Inventur berechnet anhand der neuen Menge 'Inventur IST' und dem (bei Positionen, falls gewünscht) neuen 'Bew. EK'. 

Dieser berechnete Wert wird dann in den 'GEK, Lager' zurückgeschrieben. 

Der GEK wird wie folgt berechnet: 

(((Buchbestand – noch nicht rechnungsgeprüfter WE) x alter GEK) + (Rechnungsprüfungsmenge x neuer EK))) / (Buchbestand – noch nicht rechnungsgeprüfter WE + Rechnungsprüfungsmenge) 

Hinweis: Als Alternative zum gewichteten Einkaufspreis gibt es auch den lagerspezifischen GEK. 

Die Errechnung des GEKs findet auf der Datenbank per SQL statt. 

In diesem Zusammenhang beachten Sie bitte auch die Informationen zur Checkbox Alle Lager im Dialog Lagerorte .
