# Export Datanorm im Artikel 
Export Datanorm 

Wichtiger Hinweis: 

Die vorhandene Exportart Standard (für ein statisches Mapping) wird ab der eNVenta-Version 4.6 nicht mehr unterstützt. 

Ab eNVenta 4.4 werden Sie darauf nach dem Export bzw. Durchlauf des Jobs mit folgender Meldung hingewiesen. Sie lautet: 

Achtung: Sie verwenden die Exportart „Standard“. Diese ist abgekündigt und wird in einer zukünftigen Version nicht mehr zur Verfügung stehen. 

Für den Export steht später nur noch die Möglichkeit der Exportart Konfiguration zur Verfügung. Über diese fortbestehende Möglichkeit haben Sie mindestens den gleichen Funktionsumfang bzw. profitieren von einer verbesserten Möglichkeit der Datanorm-Verwaltung. 

Dieser Dialog öffnet sich im Artikel >> Optionen >> Export Datanorm . Hier können Artikeldaten im Format Datanorm 4.0 und 5.0 exportiert werden. 

Der Export in der Version Datanorm 5.0 ist nur möglich, wenn in der Exportart 'Konfiguration' ausgewählt ist und die entsprechenden Formate für Version 5.0 unter 'Satzarten' angelegt wurden. 

Siehe Kapitel Datanorm Satzarten . 

Folgende Satzarten werden von Datanorm unterstützt: 

- V - Vorlaufsatz - K - Kundenkontrollsatz - A - Hauptsatz 1 - B - Hauptsatz 2 - P - Preisänderungssatz - S - Warengruppensatz - T - Langtextsatz - Z - Staffelpreiszuschläge / -abschläge 
Der Artikel wird nur dann bei einem Datanorm Export berücksichtigt, wenn im Artikelstamm auf der Registerkarte Allgemein die Checkbox Datanorm gesetzt ist. 

Beim Export von Artikeln erfolgt die Sortierung fix nach Artikelnummern. Aufgrund der fixen Sortierung nach Artikelnummer wird auch eine bessere Vergleichbarkeit der Exportdateien gewährleistet. 

Diversartikel werden grundsätzlich nicht berücksichtigt. 

Wenn sich relevante Datenfelder des Artikelstammsatzes geändert haben, wird dies vom System erkannt. Auf diese Weise kann gesteuert werden, dass nicht immer alle Artikelstammdaten übertragen werden müssen, sondern nur die relevanten. Es wird automatisch aktualisiert, wenn sich beim Speichern des Artikelstammsatzes eines der folgenden Felder geändert hat: 

- Bezeichnung - Langtext (Beschreibung) - Rabattgruppe - EAN - Einheit VK - VK pro - Artikelgruppe - wenn der Status ausgelaufen oder das Gelöscht-Kennzeichen gesetzt ist.

### Dock Panel Preise 

Wenn Sie Nettopreis aktivieren, wird aus den Konditionen (inkl. Rabatten) ein Nettopreis gerechnet. Entsprechend verhält es sich, wenn Sie Bruttopreis aktivieren. 

### Dock Panel Transaktionsart 

Hier können Sie wählen, ob alles generiert werden soll oder nur die Datensätze mit Änderungen (Neuanlage, Änderung, Löschung, Auslaufartikel). Dies bezieht sich nur auf geänderte Artikelstammdaten, nicht aber auf Preise/Rabatte. 

### Checkbox Warengruppen 
Wenn Sie diese Checkbox aktivieren, werden die zugehörigen Warengruppen der selektierten Artikel in einer gesonderten Datei exportiert. 

### Checkbox Keine Sonderpreise 
Wenn Sie diese Checkbox aktivieren, werden Artikel mit Sonderpreisen nicht berücksichtigt. 

Tooltip: 

Keine Berücksichtigung von Sonderpreisen. 

### Checkbox Nur Lagerartikel 
Wenn Sie diese Checkbox aktivieren, werden nur Artikel mit dem Dispo-Kennzeichen deterministisch, Reichweite oder Bestellpunkt exportiert. 

### Checkbox Keine Langtexte 
Wenn Sie diese Checkbox aktivieren, wird die Artikelbeschreibung eines Artikels nicht in die DATANORM-Datei geschrieben.

Hier wählen Sie die Codierung des Datensatzes aus.

### Combobox Version 
Hier stehen Datanorm 4.0 und 5.0 zur Verfügung.

### Combobox Exportart 
Hier stehen die Exportarten Standard und Konfiguration zur Auswahl. 

Wenn Sie Konfiguration ausgewählt haben, ist der Export in der Version Datanorm 5.0 möglich. 

Tooltip: 

Exportarten 

Standard: 
Die Datanorm-Dateien werden anhand des statisch definierten Formats erstellt. 
Dieses statische Format ist nur für die Datanorm Version 4.0 verfügbar. 

Konfiguration: 
Die Datanorm-Dateien werden anhand des konfigurierten Formats erstellt. 
(Konfiguration über Button "Satzarten")
