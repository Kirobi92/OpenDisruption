# XML-Struktur 
Die Datei-Struktur beschreibt den Aufbau der Import- bzw. Export-Datei. Hierbei entspricht dies bei einer XML-Datei der zu mappenden XML-Struktur, bei einem Text-Format (ASCII oder csv) entspricht dies grob den einzelnen Zeilen oder Bereichen. 

Eine XML-Struktur ist kein XML-Schema! Die XML-Struktur gleicht eher einer Vorlage (Template), die allerdings ebenfalls gewisse Merkmale eines XML-Schemas beinhaltet. Die XML-Struktur ist also eine XML-"Vorlage", die etwas mit einem XML-Schema gemischt wird. 

Jeder Knoten im Baum ist eine Instanz der Komponente IcXmlNodeTmpl. Diese hat wiederum eine oChildNodeColl (IcXmlNodeTmplColl) mit den Kinder-Knoten. 

Jeder Knoten hat unter anderem folgende grundlegende Informationen: 

Ist das Vorkommen des Knotens ungleich '1', so handelt es sich um eine Liste. Ein solcher Knoten kann nur über eine Mapping-Schleife gemappt werden. 
'0' = optionales Feld 
'1' = Ein XML Element (XElement) 
'2' = Ein XML Attribute (XAttribute) 
'-1' = Ein spezifisches Element (siehe nachfolgenden Absatz) 
'0' = Ein typenloser Container-Knoten (benötigt nur für den TreeView) 
1 = Element 
2 = Attribut 
100 = Text-Dokument 
110 = Sequenz 
120 = Zeile (CSV) 
125 = Feld (CSV) 
130 = Zeile (ASCII) 
135 ( Feld (ASCII) 
150 = Regulärer Ausdruck
