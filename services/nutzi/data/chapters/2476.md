# Weitere OP's Kreditoren 
Weitere OP's Kreditoren 

Dieser Dialog öffnet sich im Buchen >> OP >> OP-Aufstellung , wenn Sie den Radiobutton Kreditor anklicken, einen Vorschlag auswählen und dann den Button Weitere OP's drücken. 

In diesem Dialog werden die Teilzahlungs-OPs nie ausgewiesen, sondern deren Wert in den OP-Zusatz-Infos in der Spalte ‘Bereits bezahlt‘ und ‘Noch offen‘ des Ursprungs-OPs ausgewiesen. Diese Teilzahlungs-OPs werden nicht als eigenständige OPs behandelt, sondern können nur zusammen mit dem Ursprungs-OP in den Verarbeitungsdialog übernommen werden. Dies betrifft: 

- Weitere OPs Kreditoren aus dem Dialog Zahlungsvorschlag oder OP-Aufstellung - OP Kreditor Auswahl aus dem Dialog Verbandsabrechnung Kreditor 
Wenn Sie feststellen, dass die Zuweisung eines Teilzahlungs-OPs zu einem Ursprungs-OP nicht zutreffend ist, können Sie die Verknüpfung zum Ursprungs-OP über den Dialog OP Kreditor Detail , Button Teilzahlung aufheben lösen.

### Feld Anzahl max. 
Über dieses Feld ist in den Suchdialogen die Anzahl der einzulesenden Datensätze limitiert. Die einzelnen Datensätze werden im Dialog entsprechend gesucht und nach Erreichen der maximalen Anzahl Datensätze abgebrochen. Auf diese Weise kann eine Suche über alle Datensätze verhindert werden, welche die gesamte Performance beeinträchtigen könnte. 

Diese Einstellung nehmen Sie im Dialog Parameter Allgemein auf der Registerkarte Allgemein vor. 

Siehe Kapitel Parameter Allgemein Registerkarte Allgemein . 

Diese Einstellung lässt sich in den entsprechenden Suchdialogen (z.B. Artikelsuche, Kundenauswahl) ändern bzw. ausschalten (ausgenommen Stammdatenprogramme). 

Tooltip: 

Anzahl Sätze, die maximal eingelesen werden.
