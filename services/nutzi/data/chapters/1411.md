# Parameter Miete 
Parameter Miete 

Unter System >> Miete >> Parameter Miete definieren Sie Grundeinstellungen für die Arbeit mit diesem Modul. 

Wichtig ist: in den Auftragsarten ist dasselbe Ziellager einzustellen, wie in den Mietparametern! 

Da es nicht korrekt ist, dass beim Erzeugen eines Auftrags aus der Vermietung heraus der Einkaufspreis aus dem Artikelstamm in die Auftragsposition übernommen wird, steht in der Miete das Feld KEK % zur Verfügung und im Mietobjekt auf der Registerkarte Preise ebenfalls das Feld KEK % . 

Bei einer Erstvermietung wird Ware vom Neuteilelager ins virtuelle Mietlager gebucht. Dabei findet eine Entlastung des Neuteilelagers um den Einkaufspreis des Artikels im Neuteilelager statt. Der Artikel wird mit dem entsprechenden Einkaufspreis des Neuteilelagers auf das virtuelle Mietlager für Neuteile umgebucht. 

Bei einer Verlängerung der Vermietung bleibt der Einkaufspreis des virtuellen Mietlagers bestehen, da auch keine weitere Lagerbuchung stattfindet. 

Der Einkaufspreis auf der Auftragsposition muss immer 0 sein, der Ertrag 100%. Prüfungen wie "kein EK vorhanden" auf der Position sind nicht möglich.

### Combobox VML Neuware 
VML bedeutet Virtuelles Mietlager 

Hier wählen Sie das Lager aus, auf dem das Mietobjekt für die Dauer des ersten Mietvorgangs hinterlegt ist. Dieses Lager hat zwingend in den Auftragsarten (für die Erstvermietung) hinterlegt zu sein. 

Siehe Kapitel Auftragsarten . 

Tooltip: 

Virtuelles Mietlager Neuware: 
Wird ein Mietobjekt zum ersten mal vermietet, so befindet es sich für die Dauer des Mietvorgangs auf dem hier hinterlegten Lager. 
Dieses Lager muss in der Auftragsart für die Erstvermietung hinterlegt werden. 

### Combobox VML Gebraucht. 
Virtuelles Mietlager für Gebrauchtware 

Ein Mietobjekt, das aus einer Vermietung zurückkommt, erhält das Kennzeichen Gebraucht und wird auf ein Lager für gebrauchte Mietobjekte gesetzt. Dieses wählen Sie hier aus. Für die Dauer des nächsten Mietvorgangs befindet sich das Mietobjekt auf diesem Lager. 

Dieses Lager ersetzt bei der Vermietung von gebrauchten Mietobjekten das in der Auftragsart für die Erstvermietung hinterlegte virtuelle Mietlager Neuware . 

Siehe Kapitel Auftragsarten . 

Tooltip: 

Virtuelles Mietlager 'Gebrauchtware': 
Wird ein Mietobjekt erneut vermietet und hat das Kennzeichen "Gebraucht" gesetzt, so befindet es sich für die Dauer des Mietvorgangs auf dem hier hinterlegten Lager. 
Dieses Lager ersetzt bei der Vermietung von gebrauchten Mietobjekten das in der Auftragsart für die Erstvermietung hinterlegte virtuelle Mietlager 'Neuware'. 

### Combobox Erstauftrag 
Hier wählen Sie für die Erstvermietung eines Mietobjekts die Auftragsart aus. Für diese Auftragsart ist zwingend das virtuelle Mietlager Neuware als Ziellager zu hinterlegen. 

Siehe Kapitel Auftragsarten . 

Tooltip: 

Diese Auftragsart wird für die Erstvermietung von Mietobjekten verwendet. 
In der Auftragsart muss das "virtuelle Mietlager Neuware" als Ziellager hinterlegt sein. 
Lagerbuchung und Liefer-/Fakturafreigabe müssen ebenfalls gesetzt sein. 

### Combobox Folgeauftrag 
Für Folgeaufträge (die Verlängerung von Mietvorgängen) wird hier eine eigene Auftragsart ausgewählt. 

Siehe Kapitel Auftragsarten . 

Tooltip: 

Diese Auftragsart wird für die Verlängerung von Mietvorgängen verwendet. 
Da beliebig viele Verlängerungen möglich sind, muss diese Auftragsart ohne Lagerbuchung, aber mit automatischer Liefer- und Fakturafreigabe geschlüsselt werden. 
Da keine Lagerbuchung stattfindet, werden in der Auftragsart selbst keine Lagerorte angegeben.

### Combobox Auftragsart Kauf. 
Hier wählen Sie die Auftragsart aus, welche für den Verkauf eines Mietobjekts an einen Kunden genutzt wird. Dies geschieht aus dem virtuellen Mietlager heraus. Dieses Mietlager wird in den Auftragsarten hinterlegt und dazu ist noch die Lagerbuchung zu aktivieren. 

Siehe Kapitel Auftragsarten . 

Tooltip: 

Auftragsart Kaufen: 
Wird ein Mietobjekt vom Kunden gekauft, so geschieht dies aus dem "virtuellen Mietlager" heraus. 
Dieses ist in der entsprechenden Auftragsart als Lager zu hinterlegen. 
Lagerbuchung muss ebenfalls aktiviert sein. 

### Combobox Auftragsart Verl. 
Falls ein Mietobjekt vom Kunden verloren geht, so ist hier die entsprechende Auftragsart auszuwählen. Das entsprechende virtuelle Mietlager sollte hier ebenfalls hinterlegt sein, damit das Mietlager um dieses Mietobjekt entlastet werden kann. 

Siehe Kapitel Auftragsarten . 

Tooltip: 

Auftragsart 'Verlust': 
Wird ein Mietobjekt vom Kunden verloren, so muss dieser es in der Regel bezahlen. 
Damit das virtuelle Mietlager von dem Mietobjekt entlastet wird, ist in der hier hinterlegten Auftragsart ebenfalls das virtuelle Mietlager als Lager zu hinterlegen. 
Lagerbuchung muss ebenfalls aktiviert sein. 

### Combobox Externes ML 
Hier wählen Sie ein externes Mietlager aus. Solch ein externes Mietlager wird benötigt, wenn es um die Vermietung von Massenmietobjekten geht, wie z.B. Schalungen. 

Bei eventuellen Fehlbeständen auf dem eigenen Lager, kann hier aus einem anderen Lager die benötigte Menge aufgefüllt werden. Dieses externe Lager kann auch einem Kooperationspartner gehören. 

Tooltip: 

Externes Mietlager für Massenmietobjekte. 
Von dem hier hinterlegten Lager werden etwaige 
Fehlbestände bei einer Vermietung gebucht, 
wenn eine Kooperationsvereinbarung mit einem 
anderen Lieferanten für dieses Mietobjekt besteht. 
Es wird ein unendlicher Bestand auf diesem Lager angenommen.

### Checkbox Seriennr. aus MO 
Wenn Sie diese Checkbox aktivieren, wird die Seriennummer verwendet, die im Mietobjekt hinterlegt ist. Diese spezielle Seriennummer ist losgelöst von der Seriennummernverwaltung im Verkauf. 

Tooltip: 

Nur Seriennummer aus Mietobjekt 
Wenn gesetzt, wird die beim Mietobjekt hinterlegte Seriennummer verwendet 
und nicht die eNVenta ERP – Standardseriennummernverwaltung. 
Das Ein- und Ausbuchen von Seriennummern entfällt damit. 

### Checkbox Monat mit 30 Tagen 
Wenn Sie diese Checkbox aktivieren, wird ein Monat immer mit 30 Tagen gerechnet. D.h. Monate mit 31 bzw. (im Februar) 28 Tagen werden auch mit 30 Tagen gerechnet. 

Tooltip: 

Monat kalkulatorisch 30 Tage 
Wenn gesetzt, wird ein Monat immer als ein Zeitraum von 30 Tagen angesehen. 
Wenn nicht gesetzt, richtet sich die Länge eines Monats nach dem tatsächlichen Datum. 
Beispiel: 01.02. – 28.02. repräsentiert bei gesetzter Checkbox Monat 30 Tage keinen kompletten Monat. 
Dient der Vorbelegung bei der Neuanlage von Mietobjekten. Kann je Mietobjekt geändert werden.

### Checkbox FA Bis Zu Akt. 
Wenn Sie diese Checkbox aktivieren, wird ein Auftrag bis zum Beginn / zur Generierung des Folgeauftrags erzeugt und berechnet. 

Tooltip: 

Folgeauftrag bis zum aktuellen Datum 
Wenn gesetzt, wird bei Folgeaufträgen ein Auftrag bis zum Datum der Erstellung des Folgeauftrags generiert und abgerechnet. 
Wenn nicht, werden Aufträge mit periodenweiser Abrechnung bis zum aktuellen Datum generiert. 

### Checkbox Folgeauftrag abweichend 
Wenn Sie diese Checkbox aktivieren, wird bei Folgeaufträgen der jeweils andere Berechnungszeitpunkt gewählt. Z.B. kann bei Erstmiete über den erwarteten Mietzeitraum eine Erstrechnung per Vorfakturierung erfolgen. Alle weiteren werden dann per Nachfakturierung berechnet. 

Dies dient der Vorbelegung bei der Neuanlage von Mietobjekten und kann je Mietobjekt geändert werden. 

Beispiel: 
Die Erstrechnung kann per Vorfakturierung erfolgen, die Folgerechnungen per Nachfakturierung. 

Tooltip: 

Folgeauftrag anders 
Wenn gesetzt, wird bei Folgeaufträgen der jeweils andere Berechnungszeitpunkt gewählt. 
Z.B. kann bei Erstmiete über den erwarteten Mietzeitraum eine Erstrechnung per Vorfakturierung erfolgen. 
Alle weiteren werden dann per Nachfakturierung berechnet. 
Dient der Vorbelegung bei der Neuanlage von Mietobjekten. Dies kann je Mietobjekt geändert werden. 

### Combobox Fakturierung

### Feld BS/Tag 
Wenn ein Mietobjekt einen Betriebsstundenzähler hat, geben Sie hier die Anzahl der Betriebsstunden ein, die das Mietobjekt pro Tag laufen darf. 

Wird ein Mietobjekt über mehr Stunden genutzt, als hier eingetragen, erhalten Sie eine Meldung. Evtl. werden dem Kunden weitere Stunden in Rechnung gestellt. 

Tooltip: 

Betriebsstunden/Tag 
Dient der Vorbelegung der Betriebsstunden pro Tag bei Maschinen mit Betriebsstundenzählern. 
Wird eine entsprechend gekennzeichnete Maschine länger als die hier angegebene Anzahl Stunden an einem Tag verwendet, 
so erfolgt ein entsprechender Hinweis. 

### Feld KEK % 
Hier geben Sie den Prozentsatz für einen fiktiven "Einkaufspreis" der Vermietposition im Auftrag ein. Aus dem Positionswert der Position wird anhand dieses Prozentsatzes ein EK errechnet und in die Position eingetragen. 

Wird dieses Feld leer gelassen oder beinhaltet es eine 0, so wird ein EK von 0 in die Position eingetragen. 

Tooltip: 

Prozentsatz zur Ermittlung eines fiktiven „Einkaufspreises“ 
der Vermietposition im Auftrag. 
Aus dem Positionswert der Position wird anhand 
dieses Prozentsatzes ein EK errechnet und in die 
Position eingetragen. Wird dieses Feld leer gelassen 
oder beinhaltet es eine 0, so wird ein EK von 0 in 
die Position eingetragen. 

### Feld Rückg. Zeit bis 
Hier kann eine Uhrzeit angegeben werden, bis zu welcher die Berechnung der Rückgabe für den Vortag gilt. So kann bei einer Erfassung der Rückgabe vor z.B. 09:00 Uhr entschieden werden, dass nur 1 Miet-Tag abgerechnet wird. Die Prüfung erfolgt immer, wenn hier eine Uhrzeit angegeben ist. 

Tooltip: 

Bestimmt die Uhrzeit (hh:mm), bis zu welcher die Berechnung der Rückgabe für den Vortag gilt.
