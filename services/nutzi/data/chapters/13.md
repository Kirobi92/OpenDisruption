# CSC - Registerkarte Properties

Wissenswertes über eNVenta ERP / Client Side Customizing (CSC) 

# CSC - Registerkarte Properties 

### Erklärung der Eigenschaften 
Generell können Eigenschaften, welche mit drei Punkten enden (z.B. Font …), nicht direkt gesetzt werden. Vielmehr lassen sich diese Eigenschaften aufklappen, wodurch Detaileigenschaften angezeigt werden, welche einzeln geändert werden können (z.B. bold, italic, underline, height). 

#### Background 
Geben Sie hier die Hintergrundfarbe an. Die Farbe wird im RBGA-Modus angegeben und für jeden Farbkanal sowie für den Alpha-Kanal sind Werte zwischen 0 und 255 möglich. Sie können diese Werte direkt eingeben oder über das Feld daneben die Farbpalette öffnen. Hier wählen Sie die Farbe aus und übernehmen sie anschließend mit Klick auf den Button OK . 

#### BorderColor 
Geben Sie hier die Farbe des Rahmens an. Die Funktionsweise ist gleich wie bei der Eigenschaft Background . 
#### BorderRadius 
Geben Sie hier den Krümmungsradius des Randes an den Ecken des Controls an. Diese Angabe beeinflusst den Rahmen und den Hintergrund des Controls. Einige Controls (z.B. Tabellen) werden ggf. über diese Begrenzung hinaus gezeichnet. 

Folgende Detaileigenschaften können angeben werden: 

#### BottomLeft 
Geben Sie hier den Radius der linken, unteren Ecke an. 
#### BottomRight 
Geben Sie hier den Radius der rechten, unteren Ecke an. 
#### TopLeft 
Geben Sie hier den Radius der linken, oberen Ecke an. 
#### TopRight 
Geben Sie hier den Radius der rechten, oberen Ecke an. 
#### BorderThickness 
Geben Sie hier die Detaileigenschaften zur Dicke des Rahmens am Control an. 
#### Bottom 
Geben Sie hier die Dicke des Rahmens an der unteren Seite des Controls an. 
#### Left 
Geben Sie hier die Dicke des Rahmens an der linken Seite des Controls an. 
#### Right 
Geben Sie hier die Dicke des Rahmens an der rechten Seite des Controls an. 

#### Bottom 
Geben Sie hier die Dicke des Rahmens an der oberen Seite des Controls an. 

#### CellPadding 
Geben Sie hier den Abstand an, der in den Zellen einer Tabelle vom Zellenrand zum Inhalt eingehalten werden soll. Folgende Detaileigenschaften können gesetzt werden: 
#### Bottom 
Geben Sie hier den Abstand einer Zelle in einer Tabelle vom unteren Zellenrand zum Inhalt an. 
#### Left 
Geben Sie hier den Abstand einer Zelle in einer Tabelle vom linken Zellenrand zum Inhalt an. 
#### Right 
Geben Sie hier den Abstand einer Zelle in einer Tabelle vom rechten Zellenrand zum Inhalt an. 
#### Top 
Geben Sie hier den Abstand einer Zelle in einer Tabelle vom oberen Zellenrand zum Inhalt an. 

#### Container (nicht auf der Registerkarte 'Properties' visualisiert) 
Die Eigenschaft Container verweist auf das Control, in dem ein Control als Unterelement eingehängt ist. Diese Eigenschaft wird auf der Registerkarte Properties nicht angezeigt. Die Eigenschaft kann jedoch per Drag&Drop im Control-Treeview oder im Designer geändert werden, indem das Control in ein anderes Control verschoben wird. Die Änderung wird dann auf der Registerkarte Changes angezeigt. 

#### ChildControls 
Es ist möglich, dynamisch neue Tabellenspalten ohne Programmieraufwand in den eNVenta ERP -Dialogen hinzuzufügen. 

In der Listenansicht der Eigenschaft ChildControls werden alle bereits per Client Side Customizing hinzugefügten dynamischen Tabellenspalten angezeigt. Durch Doppelklick auf einen Eintrag werden im Spaltenkonfigurationsassistenten ( Table Column Wizard ) die vorgenommenen Einstellungen der Tabellenspalte (schreibgeschützt) angezeigt. 

Ein Klick auf den Button „+“ öffnet den Spaltenkonfigurationsassistenten im Hinzufügemodus. Hinzugefügt werden können Spalten, deren Eigenschaft an der Klasse, der in der Tabelle dargestellten Daten vorhanden ist, und die nicht bereits in der Tabelle angezeigt werden. 

Auf der ersten Seite des Assistenten ( Datasource selection ) wird in der Baumstruktur die Eigenschaft per Mausklick ausgewählt, zu der eine neue Tabellenspalte angelegt werden soll. Nach der Bestätigung mit dem Button Choose können auf der zweiten Seite des Assistenten (Settings) die Eigenschaften angegeben werden, welche auch von den Entwicklern definiert werden können. 

Hinweis: Ändert der Anwender den Namen der Tabellenspalte oder den Text des Tooltips nicht, werden die Beschriftungen mehrsprachig aus der ausgewählten Eigenschaft ausgelesen. 

#### ContentAlignment 
Geben Sie hier an, wie Controls in einem WrapPanel angeordnet werden sollen. Die folgenden Detaileigenschaften können gesetzt werden: 
#### Horizontal 
Geben Sie hier an, wie die Controls bei ausreichendem Platz horizontal angeordnet werden sollen. 

Mögliche Werte sind Left (links), Center (zentriert), Right (rechts) und Fill (füllend). 
#### Vertical 
Geben Sie hier an, wie die Controls bei ausreichendem Platz vertikal angeordnet werden sollen. 

Mögliche Werte sind Top (oben), Middle (mittig), Bottom (unten) und Fill (füllend). 

#### DefaultValue 
Hier können Sie Standardwerte vorbelegen. Dies ist nur bei den folgenden Controls möglich: 
- Einzeiligen Textfeldern (die kein Passwort haben) - Listboxen (die keine Mehrfachauswahl erlauben) - Comboboxen - Checkboxen 
Bei Datumsfeldern können ausschließlich relative Datumsangaben vorbelegt werden (z.B. heute = *, gestern = *-1, übermorgen = *+2). 

Bei Checkboxen können die Werte true (Haken gesetzt) und false (Haken nicht gesetzt) vorbelegt werden. 

Wenn ein Standardwert konfiguriert ist, wird dieser zur Laufzeit trotzdem nicht gesetzt, wenn 
- im Control bereits ein Wert gesetzt ist (auch wenn dieser zuvor durch einen Trigger gesetzt wurde) - das Control nicht editierbar ist - das im Form dargestellte Objekt (z.B. Auftrag, Kunde, Artikel) zuvor bereits gespeichert wurde, siehe Formstatus 
Die Default-Werte für Comboboxen und Checkboxen sind standardmäßig true und false . Es reicht aber nicht aus, diese im CSC zu setzen, da es in eNVenta viele Dialoge gibt, in denen Parameter übergeben werden, die dies wiederum übersteuern. Hierzu muss daher in FS vorab noch die Methode geprüft werden. In der Regel werden bei Methoden deren Bezeichnung mit „SetParam…“ anfangen, weitere Parameter mit übergeben. Wenn die Daten nach diesen Parametern gesetzt werden müssen, muss über FS in der entsprechenden SetParam-Methode am Ende der Aufruf „this.CscApplyDefaultValues(); “ eingefügt werden. Danach können Sie dann im CSC die Werte unter ‚Default Value‘ setzen. 

Eine Möglichkeit zur Prüfung, wann Werte durch den Code fest hinterlegt sind, ist folgende: Mit der Tastenkombination <Strg+Shift+F12> können Sie sich eine Übersicht über alle Customizings anzeigen lassen, die in der aktuellen Situation durch das CSC durchgeführt wurden. Wenn dort ein Wert gesetzt wird, aber dieser nicht im Dialog zu sehen ist, so hat die Business-Logik diesen übersteuert (z.B. weil in „SetParam…“ ein neues Datenobjekt gesetzt wurde). 

Außerdem sehen Sie in dieser Übersicht auch in der letzten Zeile des Dialogs einen ‚Formstatus‘, aus dem hervorgeht, ob ein Customizing möglich ist oder nicht. 

Bitte lesen Sie hierzu noch das Kapitel CSC – Übersicht über aktuelle Einstellungen ; hier wird der Formstatus genauer erläutert. 

#### DisplayLength 
Geben Sie hier die minimale bzw. maximale Größe eines Texts darstellenden Controls schriftunabhängig an. Dazu wird horizontal die Anzahl Zeichen und vertikal die Anzahl Zeilen angegeben. 

Folgende Detaileigenschaften können gesetzt werden: 
#### MaxLength 
Geben Sie hier die Anzahl Zeichen an, die in diesem Control bei maximaler Breite dargestellt werden sollen. 
#### MaxLines 
Geben Sie hier (nur bei mehrzeiligen Texteingabefeldern) die Anzahl Zeilen an, die in diesem Control bei maximaler Höhe angezeigt werden sollen. 
#### MinLength 
Geben Sie hier die Anzahl Zeichen an, die in diesem Control bei minimaler Breite dargestellt werden sollen. 
#### MinLines 
Geben Sie hier (nur bei mehrzeiligen Texteingabefeldern) die Anzahl Zeilen an, die in diesem Control bei minimaler Höhe angezeigt werden sollen. 
#### DockPanelItemSize 
Geben Sie hier an, welchen anteiligen Platz dieses Control innerhalb des umschließenden DockPanels erhalten soll. Diese Eigenschaft steht an jedem Control genau dann zur Verfügung, wenn das Control direkt in ein DockPanel eingehängt ist. 
#### DockPanelOrientation 
Geben Sie an einem DockPanel an, wie dieses orientiert sein soll. Es gibt drei mögliche Werte: 

Horizontal: Die Controls werden nebeneinander angeordnet. 

Vertical: Die Controls werden untereinander angeordnet. 

HorizontalOrVertical: Die Controls werden nebeneinander angeordnet, solange die Breite ausreichend groß ist, andernfalls untereinander. 

#### Enabled 
Geben Sie hier an, ob die Editierbarkeit des Controls eingeschränkt werden soll. Bei Buttons und Menüs bedeutet Editierbarkeit , dass sie angeklickt werden können und bei anderen Controls, dass deren Wert durch den Benutzer geändert werden kann. 
Controls, welche in eNVenta nicht editierbar sind, können nicht per Client Side Customizing editierbar geschaltet werden. 

#### FieldRowLabelMode 
Geben Sie an einer FieldRow an, nach welchem Schema Beschriftungen angezeigt werden sollen. Mögliche Werte sind: 

Generated: Vor jedes Control wird bei Bedarf eine Beschriftung generiert. 
Generated_Merged: In der ersten Spalte werden alle Beschriftungen getrennt durch einen Separator angezeigt. 
None: Es werden keine Beschriftungen angezeigt. Die erste Spalte bleibt leer. 
None_Aligned: Es werden keine Beschriftungen angezeigt. Das erste Control wird in der ersten Spalte angezeigt. 
None_Fill: Es werden keine Beschriftungen angezeigt. Alle Controls werden (unabhängig von der ersten Spalte) über die gesamte Breite des FieldPanels verteilt. 
#### FieldRowSize 
Geben Sie hier den Höhenanteil einer FieldRow an, den diese im Verhältnis zu den anderen FieldRows innerhalb des FieldPanels erhalten soll. 

#### Font 
Setzen Sie hier folgende Detaileigenschaften zur Schrift: 
Bold 
Geben Sie hier an, ob die Schrift fett dargestellt werden soll. 

Family 
Geben Sie hier an, welche Schriftart verwendet werden soll. 

Height 
Geben Sie hier die Höhe der Schrift an. 

Italic 
Geben Sie hier an, ob die Schrift schräg gestellt (kursiv) sein soll. 

Underline 
Geben Sie hier an, ob die Schrift unterstrichen werden soll. 

#### Foreground 
Geben Sie hier die Farbe für den Vordergrund bzw. Schriftfarbe bei Labeln und Buttons an. Funktionsweise ist gleich wie bei Background. 

#### GridRowHeight 
Geben Sie hier die Höhe der Zeilen einer Tabelle an. 
#### HeaderBackground 
Geben Sie hier die Hintergrundfarbe der Spaltenüberschriften einer Tabelle an. 

#### Label 
Geben Sie hier alle Eigenschaften an, die für den Fall, dass für das Control in einem FieldPanel eine Beschriftung generiert wird, für die generierte Beschriftung gelten sollen. 
Dazu können folgende Detaileigenschaften gesetzt werden, die bereits in diesem Kapitel beschrieben wurden: 
- Background - Font - Foreground - Margin - MaxSize - Minsize - Padding 

Außerdem können die folgenden Detaileigenschaften gesetzt werden: 

#### TextAlign 
Geben Sie hier die Textausrichtung an. 
#### Visible 
Geben Sie hier an, ob für das Control bei Bedarf eine Beschriftung erzeugt werden soll. 

#### Margin 
Geben Sie hier den Abstand an, den ein Control zu anderen Controls halten soll. Folgende Detaileigenschaften können gesetzt werden: 

#### Bottom 
Geben Sie hier den Abstand am unteren Rand des Controls an. 
#### Left 
Geben Sie hier den Abstand am linken Rand des Controls an. 
#### Right 
Geben Sie hier den Abstand am rechten Rand des Controls an. 
#### Top 
Geben Sie hier den Abstand am oberen Rand des Controls an. 

#### MaxSize 
Setzen Sie hier die maximale Größe eines Controls in Pixeln. Diese Angabe wird nur bei Controls empfohlen, welche ihre Größe nicht anhand des dargestellten Textes angeben können. Ansonsten verwenden Sie die Detaileigenschaften (MaxLength und MaxLines) der Eigenschaft DisplayLengh. 

Detaileigenschaften: 

#### Height 
Geben Sie hier die maximale Höhe in Pixeln an. 
#### Width 
Geben Sie hier die maximale Breite des Controls in Pixeln an. 
#### MinSize 
Setzen Sie hier die minimale Größe eines Controls in Pixeln. Diese Angabe wird nur bei Controls empfohlen, welche ihre Größe nicht anhand des dargestellten Textes angeben können. Ansonsten verwenden Sie die Detaileigenschaften ( MinLength und MinLines ) der Eigenschaft DisplayLengh . 

Detaileigenschaften: 

#### Height 
Geben Sie hier die minimale Höhe des Controls in Pixeln an. 
#### Width 
Geben Sie hier die minimale Breite des Controls in Pixeln an. 

#### OptimizeGeneratedLabels 
Setzen Sie diese Eigenschaft an einer FieldRow, wenn generierte Beschriftungen vereinfacht werden sollen, die mit identischen Worten anfangen. 
Beispiel: Aus „Datum von / Datum bis“ soll „Datum von / bis“ werden. 

Order (nicht auf der Registerkarte Properties visualisiert) 
Diese Eigenschaft legt an einem Container die Reihenfolge der Unter-Controls fest. Diese Eigenschaft wird auf der Registerkarte Properties nicht angezeigt. Die Eigenschaft kann jedoch per Drag&Drop im Control-Treeview oder im Designer geändert werden, indem ein Control an eine andere Position im Container verschoben wird. Die Änderung wird dann auf der Registerkarte Changes angezeigt. 

#### Padding 
Geben Sie hier den Abstand an, den der Inhalt eines Controls zu seinem Rand haben soll. Folgende Detaileigenschaften können gesetzt werden: 

#### Bottom 
Geben Sie hier den Abstand vom Inhalt zum unteren Rand des Controls an. 
#### Left 
Geben Sie hier den Abstand vom Inhalt zum linken Rand des Controls an. 
#### Right 
Geben Sie hier den Abstand vom Inhalt zum rechten Rand des Controls an. 
#### Top 
Geben Sie hier den Abstand vom Inhalt zum oberen Rand des Controls an. 

#### PreferredSize 
Geben Sie hier die bevorzugte Größe einer Eingabemaske an. Diese Größe ist die initiale Vorbelegung, in der modale Dialoge geöffnet werden. Folgende Detaileigenschaften können gesetzt werden: 
#### Height 
Geben Sie hier die bevorzugte Höhe an. 
#### Width 
Geben Sie hier die bevorzugte Breite an. 

#### Required 
Hier legen Sie fest, ob dieses Feld ein Pflichtfeld ist. Bei jedem Speichern in einem Dialog werden alle Pflichtfelder geprüft und ggf. eine Meldung mit den Feldern ausgegeben, die fälschlicherweise nicht gefüllt wurden. Pflichtfelder können nur folgende Controls sein: 
- einzeilige Textfelder (die kein Passwort darstellen) - Listboxen (die keine Mehrfachauswahl erlauben) - Comboboxen 

#### ShowHorizontalLines 
Geben Sie hier für eine Tabelle an, ob die Zeilen jeweils durch horizontale Linien voneinander abgesetzt werden sollen. 
#### ShowVerticalLines 
Geben Sie hier für eine Tabelle an, ob die Spalten jeweils durch vertikale Linien voneinander abgesetzt werden sollen. 

#### Shortcut 
Zuweisung eines Tastenkürzels zum Control. Dies ist besonders bei Buttons für eine schnelle Bearbeitung von Vorteil. 

#### Spacing 
Geben Sie hier an, welchen Abstand die Controls eines DockPanels zueinander haben. Folgende Detaileigenschaften können gesetzt werden: 
#### Horizontal 
Geben Sie hier den horizontalen Abstand zwischen den Controls an. 

#### Vertical 
Geben Sie hier den vertikalen Abstand zwischen den Controls an. 
#### SynchronizeColumns 
Geben Sie hier für ein FieldPanel an, ob die Controls der einzelnen FieldRows wie in einer Tabelle spaltenweise untereinander ausgerichtet werden sollen. 

#### Tooltipp Text 
Zuweisung der Funktionsangabe, die mit diesem Button ausgelöst wird. Sie erscheint, wenn der Mauszeiger für mehr als 5 Sekunden über dem Element still gehalten wird. 

#### Visibility 
Schränken Sie hier die Sichtbarkeit eines Controls ein. Gültige Werte sind: 

Visible: Keine Einschränkung 
Hidden: Das Control ist nicht sichtbar. Der Platz für das Control wird frei gehalten. 
Collapsed: Das Control ist nicht sichtbar. Der Platz für das Control wird freigegeben und kann von anderen Controls genutzt werden. 

#### WrapArrangement 
Geben Sie hier die Ausrichtung eines WrapPanels an. Gültige Werte sind: 
Vertical: Die Controls werden nebeneinander angeordnet. Wenn der Platz nicht ausreicht, wird eine neue Zeile begonnen. 
Horizontal: Die Controls werden spaltenweise untereinander angeordnet. 

Bestehende Änderungen werden auf der Registerkarte Changes angezeigt. 

Mit dem Button werden die Änderungen gespeichert und sind mit der nächsten Anmeldung des Benutzers verfügbar.
