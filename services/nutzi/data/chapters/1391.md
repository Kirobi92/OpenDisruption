# Gesellschaft 
Gesellschaft 

Die Gesellschaftsverwaltung für eine gemeinsame Warenwirtschaft steht nur zur Verfügung, wenn die entsprechende Lizenz freigeschaltet ist! 

Dieser Dialog öffnet sich im Navigator >> System >> Allgemein >> Gesellschaftsverwaltung >> Gesellschaft . Hier werden die Gesellschaften angelegt und verwaltet. Dieser Dialog ist dem Firmenstamm sehr ähnlich. Alle Daten, welche pro Gesellschaft getrennt verwaltet werden, müssen hier gepflegt werden. Diese Daten dienen u.a. dem Druck der Reports, der Definition der Rechnungsnummern, den organisatorischen Lagern und Belegarten. 

Über diesen Verwaltungsdialog können Sie die notwendigen Daten der Gesellschaft erfassen. Die Gesellschaftsdaten beinhalten alle Daten, die normalerweise im Firmenstamm hinterlegt sind. Es kann auf alle „Rewe-spezifischen Daten“ verzichtet werden, da die Warenwirtschaft in der Finanzbuchhaltung pro Gesellschaft wieder getrennt wird und daher jede Gesellschaft ihre eigene Finanzbuchhaltung mit eigenen Firmendaten hat. 

Pro Gesellschaft können folgende übergreifende Gesellschafterdaten gepflegt werden: 

- Rechnungsnummernkreis und Format (aus diesem Nummernkreis werden alle Rechnungsnummern pro Gesellschaft gezogen, so dass die Rechnungsnummern innerhalb einer Gesellschaft fortlaufend sind). Die Rechnungsnummern zwischen den Gesellschaften müssen klar abgegrenzt sein. Es darf nicht die gleiche Rechnungsnummer bei einer anderen Gesellschaft nochmals vorkommen. - Footer1 - Footer5 - Fakturalager - QS Lager - NB Lager - „Versandlager“ Standard-Logistik (siehe Parameter Einkauf , Checkbox Neue Umlagerung ) - Retourenausgleichslager - Streckenlager. Dieses ist eine Fallback-Zuordnung. Da das Lager eine Dimension für die Ermittlung der Kostenstelle ist, muss es je Lager, für das die Kostenstelle unterschieden wird, ein passendes Streckenlager geben. - Logo - Gesellschaftskürzel zum Visualisieren in den Dialogen. Eine alphanumerische Angabe ist möglich, diese sollte auch möglichst sprechend sein, da diese immer zur Anzeige verwendet wird. - Alle Felder des Firmenstammes - ILN-Nummer der Gesellschaft - Mandantenkey des Rewe-Mandanten für die Übergabe an das Rechnungswesen 
Für die Rechnungen steht ein eigener Nummernkreis zur Verfügung. Bei der Speicherung der Gesellschaft werden alle notwendigen Felder geprüft und ggf. erscheint eine Meldung falls ein Pflichtfeld nicht gefüllt ist. 

Mehr zum Thema Gesellschaftsverwaltung erhalten Sie im Kapitel Gesellschaftsverwaltung für Multicompany .

### Button Drucken 

Dieser Button startet den Druck . Der Button wird erst aktiv, wenn Daten eingelesen sind.

Dieser Button startet die Suche nach Daten; entweder ganz allgemein oder wenn Sie im Kopf des Dialogs entsprechende Selektionskriterien festgelegt haben, gezielt. 

### Button Neu 

Dieser Button leert die Daten in den Kopf-Feldern. 

Ist in den Parametern für die Sachbearbeiter die Checkbox 2 Schritt Neu aktiviert, bleiben die Suchkriterien in den Stammdialogen erhalten. 

Siehe Kapitel Parameter Sachbearbeiter . 

### Button Kopieren 

Dieser Button startet die Kopierfunktion. Es werden sämtliche Felder kopiert, welche keine Schlüsselfelder sind. 

### Button Speichern 

Dieser Button speichert Ihre Eingaben. 

### Button Vorheriger Satz 

Mit diesem Button können Sie rückwärts blättern. Die Reihenfolge beim Blättern bezieht sich auf die Registerkarte Tabelle , sofern diese vorhanden ist. 

### Button Nächster Satz 

Mit diesem Button können Sie vorwärts blättern. Die Reihenfolge beim Blättern bezieht sich auf die Registerkarte Tabelle , sofern diese vorhanden ist. 

### Button Löschen 

Dieser Button löscht Ihre Eingaben. Genauer gesagt, die markierte Zeile wird mit dem versehen und erst endgültig gelöscht, wenn Sie den Button drücken. 

### Button Schließen 

Dieser Button schließt den Dialog. In manchen Fällen werden Sie gefragt, ob Sie die Eingaben / Änderungen noch vorher speichern möchten. 

### Button Check Ges. 

Wenn Sie diesen Button drücken, wird geprüft, ob alle relevanten Felder für die Nutzung "Multicompany" gefüllt sind. 

Dazu gehören neben den Parametern auch die Prüfung auf vollständige Zuordnung von: 

- Kunde zu Gesellschaft - Sachbearbeiter zu Gesellschaft - Lager zu Gesellschaft - Definierte Nummernkreise je Gesellschaft 
Wenn die Konfiguration unvollständig ist oder nicht alle Zuordnungen durchgeführt wurden, kann die Funktionalität für Multicompany nicht aktiviert werden.
