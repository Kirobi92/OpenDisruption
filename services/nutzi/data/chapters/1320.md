# Massendruck im Verkauf 
Massendruck 

Der Massendruck bietet eine sehr mächtige Funktionalität zum Batch- bzw. Massendruck auftragsbegleitender Formulare, wie z.B. Massenlieferscheindruck oder Massenfakturierung. 

Es können alle auftragsbegleitenden Formulare vom Angebot über die Rechnung bis zur Gutschrift in Serie gedruckt werden. Dabei ist es möglich, anhand flexibler und komfortabler Selektionsmöglichkeiten genau festzulegen, für welche Angebote, Aufträge oder Gutschriften ein Formular gedruckt werden soll und für welche nicht. Zusätzlich kann die Sortierfolge der Drucke festgelegt werden. 

Damit beim Rechnungs-Massendruck einer großen Anzahl von Belegen lange Laufzeiten vermieden werden, arbeitet eNVenta wie folgt: 

Der Rechnungs-Massendruck findet ähnlich wie der Rechnungs-Einzeldruck. D.h. der Druckprozess erfolgt ohne die Teil-Transaktionen (z.B. Archivierung). Die Teil-Transaktionen werden in eigenen Abläufen abgewickelt. Auf diese Weise werden lange Laufzeiten beim Drucken deutlich minimiert. 

Dabei ist weiterhin sichergestellt, dass alle Verbuchungsprozesse (Rechnungsnummer ziehen, Status setzen, Rechnungsnummer Zuteilung, Lagerbuchung, Druckbeleg-Datensätze) in einer Transaktion ablaufen und im Fehlerfall auch zurückgerollt werden (sofern der Fehler innerhalb dieser Logik auftritt). 

Fehler innerhalb des Drucks (Beispiel bei der Aufbereitung der Crystal-Datei) führen nicht zum Rollback, sondern lediglich noch zu einem Eintrag im Massendruckprotokoll). 

Definierte Referenz-Links werden auch berücksichtigt. 

### Regeln für den Druck einer Rechnung 

#### Bei Einzeldruck: 
Die E-Mail-Adresse für die Rechnung (Feld R. E-Mail ) aus dem Kundenstamm wird in den Dialog E-Mail senden vorbelegt. 

Wenn dieses Feld nicht gefüllt ist, wird die E-Mail-Adresse des Ansprechpartners aus dem Auftragskopf übernommen. 

Ist auch diese nicht gefüllt, wird die allgemeine E-Mail-Adresse aus dem Kundenstamm übernommen. 

#### Bei Massendruck 
Hier gilt dieselbe Logik wie im Einzeldruck. 

==> Im Einzeldruck entscheidet der User, ob das per Mail (mit Rückfallkaskade) oder auf andere Weise verschickt wird. 
==> Im Massendruck können Sie das aber durch die Definition / Einschränkung per SQL steuern. 

Wenn Sie den oben beschriebenen Rückfall im Massendruck nicht möchten, muss dort auf das Feld Rechnungsemail geprüft werden. 

Wenn Sie im Dialog Massendruck Definition z.B. in der Combobox TABELLE die KUNDEN auswählen und dort folgendes einschränken: 
==>[sEmailInvoice] IS NULL 

Siehe auch Kapitel Ändern und Löschen eines Massendrucklaufs . 

Was den Druck von Logos etc. auf den Reports betrifft, so ist folgendes zu beachten: 

Es können alle gängigen Bild-Formate (bmp, gif, jpeg, png, tif, …) ausgewählt werden. 

JPEG, GIF und PNG werden 1:1 gespeichert und auch so an den Report übergeben. 

Alle anderen nicht internetfähigen Formate (bmp, tif, …) werden verlustfrei in ein PNG umgewandelt. 

Empfehlung: 

Verwenden Sie PNG als Format. 

Vermeiden Sie nach Möglichkeit Bilder auf Subreports. 

Nutzungsbeispiele: 

In Ihrem Unternehmen werden 2-mal im Monat Rechnungen gedruckt. Es handelt sich dabei überwiegend um Sammelrechnungen. Einige Ihrer Kunden fordern von Ihnen, für eine bestimmte Artikelgruppe eine separate Rechnung erstellt zu bekommen. Diese Artikel können nun Bestandteil unterschiedlicher Aufträge sein. Mit Hilfe einer geeigneten Selektionsbedingung lässt sich ein Rechnungslauf definieren, der alle Artikel einer bestimmten Artikelgruppe aus den Aufträgen zu einer Sammelrechnung zusammenfasst. 

Es sollen alle Rechnungen mit einem Wert über € 200,-- gedruckt werden. 

Ein bestimmter Artikel ist am Lager eingetroffen. Für alle Aufträge, die diesen Artikel betreffen, soll ein Lieferschein, bzw. ein Teillieferschein gedruckt werden. 

Sie möchten für alle Aufträge, für die noch keine Auftragsbestätigungen erstellt wurden, Auftragsbestätigungen drucken, bzw. automatisch per Fax an die Empfänger versenden. 

Die Liste der Möglichkeiten lässt sich fast beliebig fortsetzen. Dieser Programmteil kann bei geschickter Nutzung in erheblichem Maß zu einer rationellen organisatorischen Abwicklung in Ihrem Unternehmen beitragen! 

Geben Sie für den Wiederholungsdruck eine Sammelrechnung ein, dann wird insgesamt ein Exemplar gedruckt und nicht eines pro aufgelistetem Auftrag.

### Zusammenstellung eines Massendruck-Programms 
Um ein Programm für den Massendruck zusammenzustellen, ist es erforderlich die gewünschten Läufe aus der Auswahlliste in die Programmliste zu übertragen. 

Bitte beachten Sie, dass die Reihenfolge der Übertragung auch die Reihenfolge der Abarbeitung ist. 

Markieren Sie einen oder mehrere Läufe in der Auswahlliste mit der Maus. Mit dem Button werden die Läufe in gleicher Reihenfolge in die Programmliste übernommen bzw. an das Ende der Programmliste angefügt. 

Die Programmliste kann mit der Taste geleert werden. 

In der Spalte ‚Anzahl' kann die Anzahl der Kopien für jeden Druck angegeben werden. Standardwert ist hier immer 1. Ändern Sie einfach den Wert in der Tabelle wie gewünscht. 

### Start und Abarbeitung eines Programms 
Mit dem Button wird das zusammengestellte Programm abgearbeitet. Falls die Definition eines Massendrucklaufs einen Fehler enthält, erscheint eine Fehlermeldung, der Lauf wird sofort abgebrochen. Prüfen Sie dann die Definition des Laufs. Der Fehler ist durch die Selektionsbedingung in dem Feld SELECT ... WHERE (im Dialog Massendruck def. ) entstanden. Der Inhalt entspricht nicht den Konventionen von SQL oder muss in der Reihenfolge umgestellt werden. Bitte achten Sie darauf, dass die Bedingungen zuerst genannt werden, welche die Felder aus der Auftragsposition betreffen. 

Folgende Spalten der Programmtabelle geben Ihnen Auskunft über Stand und Erfolg des Massendrucks: 

Gedruckt: 
Zeigt die Anzahl bereits gedruckter Formulare. 

Status: 
Der Status kann folgenden Wert haben: 
- Bereit (noch nicht bearbeitet) - Aktiv (wird gerade bearbeitet) - Abbruch (der Lauf ist wegen eines Fehlers oder durch den Benutzer abgebrochen worden) - Fertig (der Lauf wurde erfolgreich beendet) 
%: 
Zeigt den Fortschritt bei der Bearbeitung des Laufs in Prozent. 

Zu Beginn jedes einzelnen Drucks erscheint ein kleiner Dialog und zeigt den Button Abbruch . Soll die Abarbeitung des Programms gestoppt werden, drücken Sie bitte sofort nach Erscheinen die Taste Abbruch . 

Bei Zusatzformularen beginnt der Druck bei einem erneuten Start wieder von Beginn an! Bereits gedruckte Formulare werden noch einmal gedruckt.

### Feld Valutadatum 
Valutadatum, das bei der Berechnung der Fälligkeiten gezogen werden soll. 

Ist das Valutadatum in der Vergangenheit, wird dieses bei Faktura auf das Fakturadatum gesetzt. Das Fakturadatum entspricht dem Druckdatum. 

Das Fakturadatum ist grundsätzlich das Druckdatum, sowohl in der Tabelle DRUCKBELEGE als auch in der Tabelle AUFTRAGSPOS. 

Liegt das Valutadatum in der Zukunft, so wird dies für die Berechnung des Zahlungsziels genommen und bleibt auch im Auftragskopf und den Druckbelegen erhalten. 

Tooltip:
