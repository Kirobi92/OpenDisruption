# IIS-Timeout für eNVenta ERP einstellen 
Wenn Sie per ASP 2.0 auf den IIS zugreifen, ist standardmäßig ein Session-Timeout von 20 Minuten eingestellt, das bedeutet nach 20 Minuten Inaktivität schließt sich eNVenta ERP . 

Um dies zu ändern, ist es zwingend erforderlich folgendes auf dem Broker zu ändern: 

- Aufruf der Internet-Informations-Dienste, z.B. über Start - Ausführen: Eingabe von "inetmgr". - Webseite aufrufen (bei uns eNVenta ERP ) und dort über das Kontextmenü die Eigenschaften aufrufen. - Auf der Registerkarte Verzeichnis die Konfiguration öffnen. - Im Dialog Anwendungskonfiguration auf der Registerkarte Optionen nun das Sitzungs-Timeout definieren oder ganz rausnehmen. - IISReset durchführen. 
Sessions, die länger inaktiv sind (z.B. hängengeblieben sind) als der eingestellte max. Session Timeout, werden automatisch gelöscht. Diese Einstellung kann durch den Administrator direkt auf der Datenbank eingestellt werden und nicht über die Oberfläche. 

Die Konfiguration erfolgt per SQL in der Daten-DB SYSTEM_BASIS:BrokerTimeout. Dieser Wert wird in Stunden angegeben.
