# Optionen auf der Registerkarte Termine

eNVenta ERP 

# Registerkarte Wiedervorlage 
Registerkarte Wiedervorlage 

Diese Registerkarte wird über das Symbol geöffnet. 

Auf der Registerkarte Wiedervorlage werden die aktuellen Wiedervorlagen des Benutzers und seiner Abteilung dargestellt. Die offenen, noch nicht abgearbeiteten Wiedervorlagen erscheinen mit weißem Hintergrund. Neue, noch nicht gelesene Nachrichten werden mit grünem Hintergrund in der Auswahlliste dargestellt. Das Datum der Wiedervorlage wird hinsichtlich seiner Priorität farblich (bei einer normalen Priorität weiß, bei einer wichtigen Priorität gelb und bei einer sehr wichtigen Priorität rot ) schattiert. 

#### Weiterleitung von Wiedervorlagen 
Bei Wechsel des Sachbearbeiters oder der Abteilung (durch die entsprechende Combobox über der Tabelle der Wiedervorlage) öffnet sich automatisch der Dialog Wiedervorlage erfassen . Dadurch wird auch automatisch der Button zur Speicherung markiert. Die Weiterleitung der Wiedervorlage wird wie bisher mit Enter bzw. Leertaste bestätigt. Durch das Öffnen des Dialogs kann der Sachbearbeiter einen Text erfassen, ohne die alten Texte zu verändern und auch die Uhrzeit der Weiterleitung wird notiert. 

#### Weiterleitung von Wiedervorlagen an die Abteilung 
Bei einer Wiedervorlage, welche beim Sachbearbeiter erscheint, aber der Empfänger eine Abteilung ist, sind die Buttons zur Erledigung bzw. Löschung nicht aktiv; nur der Button zur Weiterleitung ist aktiv. So kann ein Sachbearbeiter den Vorgang zu sich holen, nicht aber für alle Sachbearbeiter auf erledigt setzen, ohne den Vorgang tatsächlich zu bearbeiten. 

Kennzeichnen Sie eine Wiedervorlage als erledigt , so wird sie aus der Liste Ihrer Wiedervorlagen entfernt. Sie bleibt jedoch als erledigte Wiedervorlage in eNVenta erhalten. 

Das gilt ebenso, wenn Sie eine Wiedervorlage zur Genehmigung erhalten. In dem Fall geht zusätzlich eine entsprechende Information – ebenfalls per Wiedervorlage - an den Anforderer der Genehmigung zurück, um diesen über die Genehmigung des Vorgangs zu informieren. 

#### Weitere Informationen 
Kennzeichnen Sie eine Wiedervorlage als gelöscht , so wird diese aus Ihrer Wiedervorlage entfernt. Sie bleibt jedoch als erledigte Wiedervorlage in eNVenta erhalten. 

Wenn Sie eine Wiedervorlage zur Genehmigung erhalten und diese ablehnen, geht eine entsprechende Information – ebenfalls per Wiedervorlage - an den Anforderer der Genehmigung zurück, um diesen über die Ablehnung des Vorgangs zu informieren. 

Leiten Sie eine Wiedervorlage weiter, so öffnet sich der Dialog Wiedervorlage erfassen (Kapitel Erfassung eines neuen Termins ), damit Sie den Empfänger der Wiedervorlage auswählen können. Dies kann wie oben beschrieben ein anderer Sachbearbeiter oder eine andere Abteilung sein. 

Über den Button können Sie eine Wiedervorlage ohne Bezug zu einem Vorgang erstellen und einem Sachbearbeiter (einer Abteilung) zusenden. Anschließend können Sie diesen bei sich als erledigt markieren, sofern der Vorgang mit der Weiterleitung an den Kollegen für Sie abgeschlossen ist. 

Möchten Sie darüber informiert werden (oder soll ein Kollege informiert werden), wenn der Mitarbeiter die von Ihnen erstellte Wiedervorlage nicht zeitnah erledigt hat, so ist dies ebenfalls über eine Warnung möglich. 
Wiedervorlage mit Warnung erfassen 

In diesem Beispiel erhält der Benutzer 710 eine Wiedervorlage am 08.01.2022, sofern der Mitarbeiter Hirling den Termin bis dahin nicht abgearbeitet hat. Geben Sie bei Warnung Tage „0“ ein, so werden Sie am Folgetag per Wiedervorlage informiert. 

#### Outlook 
Oben links auf der Registerkarte Termine sehen Sie den Menüeintrag Outlook . Wenn Sie darauf klicken, stehen 3 Optionen zur Verfügung: 

- Wiedervorl / Aufgaben abgleichen - E-Mails importieren - Ressourcen / Kalender abgleichen 
Mit diesen Optionen lassen sich Wiedervorlagen, Aufgaben, E-Mails und Ressourcen zwischen Microsoft Outlook und eNVenta ERP synchronisieren. Damit vermeiden Sie eine doppelte Verwaltung bzw. verringern die Gefahr, dass ein Termin „verloren“ geht. 

Sie haben die Möglichkeit Aufgaben, Wiedervorlagen und E-Mails abzugleichen. Dies ist besonders hilfreich, wenn die Kommunikation in Ihrem Unternehmen und mit externen Kontakten auf E-Mails basiert. 

Eine Integration von Outlook in eNVenta ERP ist nur möglich mit Microsoft Outlook ex 2003. 

Für die Verknüpfung von Outlook und eNVenta ERP benötigen Sie das Menü Parameter – Office . 

Lesen Sie hierzu die Kapitel Integration Outlook in eNVenta ERP . 

Siehe Kapitel Parameter Office . 

#### Optionen 
Im Navigator auf der Registerkarte Termine stehen unter Optionen die Funktionalitäten Auf ungelesen setzen und WV als Favorit zur Verfügung. 

Auf ungelesen setzen: 
Damit können bereits "geöffnete" Wiedervorlagen wieder auf ungelesen gesetzt werden. 

WV als Favorit: 

Wenn eine Wiedervorlage selektiert ist und Sie darauf klicken, wird die Wiedervorlage zu den Favoriten hinzugefügt. Wenn die selektierte Wiedervorlage keine Referenz hat (wie das z.B. bei manchen EDI-Meldungen der Fall ist) erscheint eine Fehlermeldung, dass die Wiedervorlage nicht als Favorit gespeichert werden kann. 

Genehmigung von Wiedervorlagen: 

In eNVenta ist es möglich, Wiedervorlagen bewusst einem Genehmigungsvorgang zu unterwerfen. Hierfür muss die Wiedervorlage mit dem Typ Zur Genehmigung an einen entsprechend berechtigten Sachbearbeiter gesendet werden. Diesem berechtigten Sachbearbeiter stehen die Buttons genehmigt und abgelehnt zur Verfügung. So wird sichergestellt, dass auch der Sachbearbeiter, der die Wiedervorlage erfasst hat, über die Genehmigung bzw. Ablehnung informiert wird. 

Wiedervorlagen, die genehmigt werden müssen, können nicht über den Button Erledigt auf erledigt gesetzt werden. Der Button ist in diesem Fall inaktiv. D.h. umgekehrt, Wiedervorlagen, die genehmigt werden müssen, können über die Buttons genehmigt bzw. abgelehnt gesteuert werden. 

Für alle „normalen“ Wiedervorlagen sind die Buttons genehmigt und abgelehnt nicht zugänglich. 

Durch Klick auf den Button genehmigt oder abgelehnt wird eine Wiedervorlage zurück an den Erfasser der Wiedervorlage gesendet. Diese hat im Betreff und im WV-Text jeweils den Begriff GENEHMIGT (bzw. ABGELEHNT) vor dem ursprünglichen Betreff stehen. Wird der Betreff dadurch zu lang, so wird er entsprechend am Ende gekürzt. 

Die Wiedervorlage, welche zurück an den Erfasser gesendet wird, ist vom Typ „zur Info“ und kann mit dem Button Erledigt abgeschlossen werden. 
## Steuerelemente des Formulars 

### Button Erledigt / Genehmigt 

Bei Wiedervorlagen vom Typ „Zur Bearbeitung“ bedeutet der Button „Erledigt“, beim Typ „Zur Genehmigung“ entsprechend „Genehmigt“. 

### Button Löschen / Abgelehnt 

Bei Wiedervorlagem vom „Zur Bearbeitung“ beutet der Button „Gelöscht“, beim Typ „Zur Genehmigung“ entsprechend „Abgelehnt“. 

### Button Wiedervorlage Weiterleiten 

Dieser Button wird für die Weiterleitung der Wiedervorlage genutzt. Wenn Sie diesen Button drücken, öffnet sich der Dialog Erfassung eines neuen Termins . 

### Combobox Mitarbeiter 
Hier wird vom Programm automatisch der Mitarbeiter eingetragen, der angemeldet ist. Wenn Sie einen anderen Sachbearbeiter auswählen, können Sie dessen Wiedervorlagen einsehen. 

Tooltip: 

Zeigt den Sachbearbeiter, der als letztes den Datensatz bearbeitet hat. 

### Checkbox Ganze KW 
Wenn Sie diese Checkbox aktivieren, kann von Tagesanzeige auf Wochenanzeige umgeschaltet werden. 

### Checkbox Auch erl. Term. 
Wird diese Checkbox gesetzt, werden auch die bereits erledigten Termine für den Tag bzw. für die KW angezeigt. 

### Bezeichnungsfeld Offen: 
Hier wird die Anzahl der offenen, noch nicht erledigten Wiedervorlagen angezeigt. 

### Bezeichnungsfeld Offene WV 
Hier wird die Anzahl der neuen, noch nicht gelesenen Wiedervorlagen angezeigt. 

### Button Suche 

Dieser Button öffnet den Dialog Wv-Übersicht . Hier erhalten Sie eine Übersicht über sämtliche Wiedervorlagen. 

Der Zugriff auf diesen Dialog kann durch den Administrator gesperrt bzw. erlaubt werden. 

Siehe Kapitel Wv-Übersicht . 

Tooltip: 

Wv-Übersicht 

### Feld Betreff 
Hier wird der Betreff aus dem Call angezeigt, der im Moment in der Tabelle aktiviert ist. 

### Feld Betreff 
Hier wird der Betreff des Calls angegeben (Thema). 

Tooltip: 

Call-Betreff 

### Feld Nachricht 
Hier wird angezeigt, was im Dialog Wiedervorlage erfassen in das Feld Nachricht eingetragen wurde. 

### Feld Termin 
Wenn der Termin / die Terminbearbeitung für den markierten Vorgang verschoben werden soll, geben Sie ein Datum ein. 

### Combobox Termin-Typ 
Hier wird der Bearbeitungsstand des Termins angegeben: 
- zur Bearbeitung - zur Info - zur Genehmigung 

### Combobox Priorität 
Hier wird die Priorität der Wiedervorlage angegeben. Zur Verfügung stehen: - normal - wichtig - sehr wichtig 

Tooltip: 

Priorität des Termins 

### Combobox Abteilung 
Wenn Sie hier eine Abteilung auswählen, werden in der Tabelle die Termine für diese Abteilung angezeigt. 

Wird die Abteilung geändert, wird automatisch die Combobox Sachbearbeiter geleert. 

### Combobox Sachbearbeiter 
Hier wählen Sie einen Sachbearbeiter aus. 

Der markierte Termin wird dann an diesen Sachbearbeiter verschoben. 

Tooltip: 

Zeigt den Sachbearbeiter, der als letztes den Datensatz bearbeitet hat. 

### Tabelle Termine 

In dieser Tabelle werden das Fälligkeitsdatum, der Vorgang (Call oder Projekt) und der Sachbearbeiter, der den Vorgang weitergeleitet hat, angezeigt. 

Für ungelesene Wiedervorlagen gilt folgende Farbgebung: 

- Wiedervorlagen an den Stellvertreter werden grau dargestellt. - Ungelesene Wiedervorlagen an den Sachbearbeiter werden hellgrün dargestellt. - Wiedervorlagen an die Abteilung werden blaugrün dargestellt. 
### Tabellenspalte Termin 
Hier können Sie eingeben, wann die Wiedervorlage beim Empfänger ankommen soll. Das Tagesdatum ist voreingestellt. 

### Tabellenspalte Vorgang 
Hier wird der betroffene Vorgang angegeben. 

### Tabellenspalte Sachb. von 
Hier wird vom Programm automatisch der Sachbearbeiter eingetragen, der den Datensatz zuletzt geändert hat. 

Tooltip: 

Zeigt den Sachbearbeiter, der als letztes den Datensatz bearbeitet hat. 

### Tabellenspalte Status 
Hier wird der Status der Position angegeben. 

1-offen 
2-offen, zur Lieferung freigegeben 
3-offen, zur Faktura freigegeben 
4-fakturiert 
5-geliefert (ohne Berechnung) 
6-storniert 

Tooltip: 

Angabe des Positionsstatus 

### Bezeichnungsfeld Datenbank Info 
In diesem Feld wird die Datenbank angezeigt, auf der Sie sich befinden. 

Tooltip: 

Aktuelle DB und aktueller Benutzer 

## Menüeinträge 

### Menü Outlook 

### Menü Outlook / Abgleich Wiedervorl. / Aufgaben 
Mit dieser Option lassen sich Wiedervorlagen, Aufgaben, E-Mails und Ressourcen zwischen Microsoft Outlook und eNVenta ERP synchronisieren. 

### Menü Outlook / E-Mails importieren 
Über dieses Menü können E-Mails importiert werden. 

### Menü Outlook / Ressourcen / Kalender abgleichen 
Über dieses Menü können Ressourcen und Kalender abgeglichen werden. 

### Menü Outlook / Feiertage exportieren 
In diesem Menü werden alle Feiertage des Default-Betriebskalenders (Kalendernr 0) für alle Ressourcen exportiert, welche einen gültigen Kalender-Alias hinterlegt haben. 

Es werden alle Kalender aktualisiert, nicht nur der eigene! 

Bei Neuanlage von Ressourcen-Alias muss die Funktion wieder neu ausgeführt werden. 

### Menü Optionen 

### Menü Optionen / Auf ungelesen setzen 
Mit diesem Menü setzen Sie eine Wiedervorlage wieder auf "ungelesen". 

### Menü Optionen / WV als Favorit 
Über dieses Menü können Sie eine bestimmte (markierte) Wiedervorlage als Favorit festlegen. 

Tooltip: 

Selektierte Wiedervorlage als Favorit speichern

### Menü ? / Hilfe
