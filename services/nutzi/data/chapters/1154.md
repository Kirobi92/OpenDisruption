# Automatische Lieferfreigabe 
Lieferzuteilung 

Dieser Dialog öffnet sich im Auftrag >> Vorgänge >> Lieferzuteilung . Hier können mehrere Auftragspositionen automatisch zur Lieferung freigegeben werden. Hierbei werden alle Positionen auf ihren Status überprüft und anhand der angegebenen Kriterien wird geprüft, ob diese zur Lieferung freigegeben werden können (Status 2). Ebenso werden Sperrungen von Kunden (Liefer- oder Gesamtsperre) und der Abschluss des Auftrags (falls aktiviert) berücksichtigt. 

Die Entscheidung, welcher Auftrag zur Lieferung freigegeben wird, hängt ab von der Kundennummer und dem gewünschten Lieferdatum der Auftragsposition (das Lieferdatum, welches als nächstes kommen soll, wird zuerst beliefert). 

Es wird geprüft, ob der Gesamtwert pro Kunde, welcher im Kundenstamm , Registerkarte Logistik , im Feld Min. Sendewert hinterlegt ist, eine Lieferzuteilung ermöglicht. Wird der Mindestsendewert nicht erreicht, werden die Positionen zu diesem Kunden nicht freigegeben. Wird zum gewählten Kunden ein weiterer Auftrag erfasst und der Mindest-Sendewert wird erreicht, werden beide Aufträge freigegeben. 

Damit ein Fehler bei der Freigabe einer Auftragsposition nicht die gesamte Lieferzuteilung zum Stillstand bring, arbeitet das System wie folgt: 
- Schlägt das Speichern einer einzelnen Position fehl, dann wird dies protokolliert und der gesamte Freigabe-Prozess wird mit der nächsten Position fortgesetzt. - Fehlerhafte Belege können über den Button Report schnell ermittelt werden. 
Beim Laden der Daten werden gesperrte Lagerplätze nicht berücksichtigt. D.h. der Bestand des Lagers wird um die Summe der Bestände auf gesperrten Lagerplätzen reduziert. So ist für den Sachbearbeiter ersichtlich, welcher Bestand tatsächlich noch für den Verkauf zur Verfügung steht. 

Es werden nur Positionen mit einer möglichen Volllieferung angezeigt. Erst wenn die Checkbox Auch Teillieferung gesetzt ist, werden zusätzlich auch Positionen angezeigt, welche mit einer Teillieferung beliefert werden könnten. 

Anhand der vorgegebenen Suchkriterien wird der älteste noch offene Auftrag ermittelt. Für den Kunden dieses Auftrags werden alle offenen Positionen aufgrund der verfügbaren Mengen geprüft und zur Lieferung vorgeschlagen. Ist der ermittelte Gesamtwert der Positionen kleiner als der im Menü Kunde hinterlegte Mindestsendewert, wird der Vorschlag für den Kunden zurückgesetzt. 

Anschließend wird der nächst ältere Auftrag ermittelt und dieser Kunde komplett nach obigem Algorhythmus abgearbeitet. 

Die so entstandene automatische Lieferzuteilung kann jetzt manuell bearbeitet und anschließend freigegeben werden. 

Es werden hier keine Platzzuordnungen für Positionen im Status 1 vorgenommen. 

Zwischenzeitlich gesperrte Aufträge werden bei der automatischen Lieferzuteilung protokolliert! 

Es ist zu verhindern, dass die Menge_geliefert in der Position angepasst und Status erhöht wird, wenn Vorab Kommisionieren und Komplettliefern aktiviert ist UND eine Unterliefersituation entstehen würde. -> Damit wird verhindert, dass eine Packliste für eine Teilmenge gedruckt wird (über Job), bevor die gesamte Menge vorhanden ist.

### Checkbox Auftragsbezogen bestellt 
Bei einer auftragsbezogenen Bestellung kann es vorkommen, dass die Lieferfreigabe der Auftragsposition beim Wareneingang nicht stattfindet. 

Wenn Sie diese Checkbox aktivieren, kann eine Lieferzuteilung von Ware, die auftragsbezogen bestellt und vereinnahmt wurde, auch automatisch erfolgen. 

Diese Zuteilung kann auch über den Job-Server ausgeführt werden. 

Achtung: 
Eine Handelsstückliste wird beim Bestellen in ihre Komponentenartikel aufgelöst. Dadurch kann es für die einzelnen Komponentenartikel auch unterschiedliche Wareneingänge geben. Wird einer dieser Wareneingänge freigegeben, wird pauschal versucht die zugehörige Auftragsposition freizugeben. Das ist erst dann erfolgreich, wenn alle Komponenten vereinnahmt wurden. Die Entscheidung, ob eine Lieferfreigabe der Handelsstückliste möglich ist, fällt damit nicht mehr nur in der Verfügbarkeitsprüfung, sondern auch über die Lieferzuteilung. 

Tooltip: 

Auch Ware zuteilen, die auftragsbezogen bestellt wurde, deren Wareneingang jedoch nicht zur Freigabe der Position geführt hat.

Eventuelle Prüfungen werden dabei berücksichtigt. Chargen und Platzzuordnungen werden automatisch zugeteilt.

### Checkbox Sys. Lieferdatum 
Wenn diese Checkbox gesetzt ist, hat die jeweilige Auftragsposition ein vom System vorgegebenes Lieferdatum. D.h. das Lieferdatum wurde nicht vom Sachbearbeiter beeinflusst. Die entsprechende Kennzeichnung wird automatisch gesetzt, wenn im Dialog Verfügbarkeit Steuerung die Option Termin ändern ohne Freigabe ausgewählt wurde. 

Der Prozess der Packlistenerstellung im Massendruck richtet sich immer nach dem Lieferdatum der jeweiligen Position. Dies kann in dem Moment problematisch sein, in dem die Position aufgrund einer theoretischen Wiederbeschaffungszeit bei der Erfassung ein deutlich späteres Lieferdatum erhält, dieses aber dann (viel) früher eintrifft. Hier kann eine so entstandene Position die Lieferung eines kompletten Auftrags nach hinten verschieben, da sich eNVenta ERP nach dem spätesten Lieferdatum der Auftragsposition des jeweiligen Auftrags richtet. 

Wenn die Lieferzuteilung möglich wäre, aber eine auftragsbezogene Bestellung vorhanden ist, so erscheint der Auftrag nicht im Dialog Lieferzuteilung ( Automatische Lieferfreigabe ). 

Tooltip: 

System Lieferdatum. 
Das Flag zeigt an, ob die jeweilige Auftragsposition ein vom System vorgegebenes Lieferdatum beinhaltet. 

### Checkbox Rückstandsaufl. 
Wenn Sie diese Checkbox aktivieren, kann eine Rückstandsauflösung im Auftrag ausgeführt werden. 

Über diese Funktion der Rückstandsauflösung können Auftragspositionen, welche bereits im Status 2 oder 3 sind, nachträglich angepasst werden, solange diese noch nicht in der Kommissionierung (Packliste oder Lieferschein gedruckt) sind. (Gleiches gilt auch für die Auftragspositionen im Status 1.) Diese Auftragspositionen hatten zum Zeitpunkt der Erfassung evtl. nur einen Teil der Waren zur Verfügung und würden dann eine Rückstands-Position erzeugen. Ist nun aber genügend Ware vorhanden, lässt sich dies durch die Checkbox Rückstandsauflösung vermeiden, da die zu liefernde Menge auf die ursprünglich bestellte Menge erhöht wird. 

Gleiches gilt auch, wenn eine Komplettlieferung erst erfolgen kann, wenn auch die letzte Position den notwendigen Rest zur Lieferung bekommt und somit keinerlei Rückstandsposition generiert wird. 

Beispiele, welche mit der Checkbox gelöst werden können: 

- Ein eNVenta ERP Kunde erfasst für seinen Kunden einen Auftrag über 10 Stück einer Position, deren Lieferdatum in 14 Tagen liegt. Zum aktuellen Zeitpunkt ist keine Ware vorhanden. 
2 Tage nach Auftragserfassung kommt eine Teillieferung von 5 Stück durch den Lieferanten. Diese Menge wird per automatischer Lieferzuteilung zugeordnet und der Auftrag auf Status 2 gesetzt. Ein paar Tage später werden erneut 5 Stück geliefert. Die Zuteilung erfolgt nun, weil diese Option gesetzt ist. - Ein eNVenta ERP Kunde erfasst einen Komplettauftrag über mehrere Positionen. Es gibt eine Teillieferung seitens des Lieferanten für eine Position, die zugeteilt wird. Später erfolgen Zuteilungen für alle anderen Positionen des Auftrags. Dieser Auftrag wird nun geliefert, weil diese Option gesetzt ist. 
Es kann vorkommen, dass ein Lieferant nicht die volle Menge liefert und es auch nicht zu erwarten ist, dass die Ware noch geliefert wird. Hier wird der Wareneingang mit dem Teillieferungskennzeichen "V" gebucht (d.h. keine Rückstandsposition). In diesem Fall kann die Lieferzuteilung eine Mengenerhöhung durch zuteilbaren Lagerbestand vornehmen. 

Tooltip: 

Rückstandsauflösung mit Lagerware: 
Ist diese Checkbox gesetzt, so wird die bereits zugeteilte Menge um die Restmenge, sofern vorhanden, erhöht. 
Dadurch wird eine Rückstandsbildung bei Lieferschein- oder Rechnungsdruck vermieden.
