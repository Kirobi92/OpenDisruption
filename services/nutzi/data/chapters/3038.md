# Ausdruckseditor 
Ausdruckseditor 

Dieser Dialog öffnet sich u.a. über die -Buttons z.B. in den Dialogen Aktions-, Ereignisdefinition und Replikationsdefinition oder über den Button Formel im Dialog Massenänderung . 

Dieser Dialog dient zum Erstellen und Bearbeiten von Ausdrücken, damit auf Basis von Bedingungen die Daten gelesen, geändert oder geschrieben werden können. 

Ein Ausdruck muss immer einen Wert mit einem bestimmten Typ zurückgegeben. Dieser Typ ist abhängig vom Anwendungsfall. Wenn es sich um einen Ausdruck zu der Entscheidung handelt, ob repliziert werden soll, dann wird ein "bool" erwartet. Wenn es sich um einen Transformations-Ausdruck handelt, dann orientiert sich der Typ z.B. an der betreffenden Tabellen-Spalte. (FSstring, FSDateTime, FSdeciml, …) Über die Anzeige im Feld Typ ist der erwartete Ziel-Typ sichtbar. 
### Replikationsdefinition 
Wenn es sich um einen Ausdruck zu der Entscheidung handelt, ob repliziert werden soll, dann wird ein "bool" erwartet. Wenn es sich um einen Transformations-Ausdruck handelt, dann orientiert sich der Typ z.B. an der betreffenden Tabellen-Spalte (FSstring, FSDateTime, FSdecimal, …). 

Bei jeder Neuanlage oder Änderung eines Replikationsdienstes hat zwingend der Dienst neu gestartet zu werden.

Funktionen im Ausdruckseditor 
#### 1.	Contains. 

Zur Nutzung einer Transformation. Hier wird ein Wert zurückgegeben, der angibt, ob eine angegebene untergeordnete Zeichenfolge in dieser Zeichenfolge vorkommt. 
Beispiel: Es wird überprüft, ob die eingegebene Zeichenfolge „1“ enthält oder nicht. 
Beispiel: 
a)	Im Feld Import wird eine Folge „123“ angegeben. Die Eingabe ist korrekt und wird akzeptiert. 
b)	„abc“ wird angegeben. Die Eingabe ist falsch und wird nicht akzeptiert. 
Mit „Contains“ kann überprüft werden, ob der angegebene Wert an 1. Stelle den an 2. Stelle angegebenen Wert enthält. Ist dies der Fall wird true zurückgegeben, ansonsten false . 
Beispiel zu b): 
Der Wert an 1. Stelle enthält den Text „Fischers Fritz fischt 12 frische Fische“. 
a.	Wird nun im 2. Wert „Fritz“ angegeben, ist die Bedingung wahr. 
b.	Wird im 2. Wert „Hans-Peter“ angegeben, ist die Bedingung nicht wahr. 
#### 2.	IndexOf. 
Zur Nutzung einer Transformation für den Import zur Suche nach "abc". 

Gibt einen Wert zurück, der angibt, ob eine angegebene untergeordnete Zeichenfolge in dieser Zeichenfolge vorkommt. 

a)	Der Wert „321“ wird angegeben. Das Alphafeld1 wird auf 2 gesetzt. 
b)	Der Wert „abc“ wird angegeben. Es wird „-1“ zurückgegeben. 
Mit „IndexOf“ kann ermittelt werden, an welcher Stelle des 1. Werts der 2. Wert zum ersten Mal vorkommt. Zurückgegeben wird der Index des ersten Vorkommens. 
Beispiel zu b): 
Der Wert an 1. Stelle enthält den Text „Fischers Fritz fischt 12 frische Fische“. 
a.	Wird nun im 2. Wert „Fritz“ angegeben, wird der Wert 9 zurückgegeben. 
b.	Wird im 2. Wert „Hans-Peter“ angegeben, wird der Wert -1 zurückgegeben. 

#### 3.	RegexMatchFirst. 
Zur Nutzung einer Transformation für den Import zur Suche nach "123" 
Regex ist „[0-9]+“, d.h. es werden Ziffern zurückgegeben. 

a)	Wert „abc123“ angeben. Es wird „123“ zurückgegeben. 
b)	Wert „abc“ angeben. Es wird Null zurückgegeben. 
Mit „RegexMatchFirst“ kann eine Teilzeichenfolge aus dem Wert an 1. Stelle ermittelt werden. Hierfür kann im Wert an 2. Stelle ein Regulärer Ausdruck verwendet werden. 
Beispiel zu b): 
Der Wert an 1. Stelle enthält den Text „Fischers Fritz fischt 12 frische Fische“. 
a)	Wird nun im 2. Wert „[0-9].*“ angegeben, wird „12 frische Fische“ ausgegeben. 
[0-9] steht für eine Zahl zwischen 0 und 9 
„.*“ für eine beliebige Anzahl, beliebiger Zeichen. 
#### 4.	RegexIndexOf. 
Zur Nutzung einer Transformation für den Import zur Suche nach Ziffern. 
Gibt den NULL-basierten Index des ersten Vorkommens eines angegebenen Unicode-Zeichens oder einer angegebenen Unicode-Zeichenfolge in dieser Instanz an. Die Methode gibt -1 zurück, wenn das Zeichen oder die Zeichenfolge in dieser Instanz nicht gefunden wird. 

a)	Wert „abc123“ angeben. Es wird „3“ zurückgegeben. 
b)	Wert „abc“ angeben. Es wird „-1“ zurückgegeben. 
Mit „RegexIndexOf“ kann ermittelt werden, an welcher Stelle des 1. Werts der 2. Wert zum ersten Mal vorkommt. Hierfür kann im Wert an 2. Stelle ein Regulärer Ausdruck verwendet werden. Zurückgegeben wird der Index des ersten Vorkommens. 
Beispiel zu b): 
Der Wert an 1. Stelle enthält den Text „Fischers Fritz fischt 12 frische Fische“. 
a.	Wird nun im 2. Wert „[0-9]“ angegeben, wird der Wert 22 zurückgegeben. Dies entspricht dem Index der Zahl 1. 
b.	Wird im 2. Wert „[3-9]“ angegeben, wird der Wert -1 zurückgegeben, da keine Zahl zwischen 3 und 9 vorhanden ist. 
#### 5.	StartsWith. 
Zur Nutzung einer Transformation für den Import zur Suche nach "abc". 
a)	Wert „123“ angeben. Es wird „true“ zurückgegeben. 
b)	Wert „321“ angeben. Es wird „false“ zurückgegeben. 
Mit „StartsWith“ kann überprüft werden, ob der angegebene Wert an 1. Stelle mit dem an 2. Stelle angegebenen Wert beginnt. Ist dies der Fall wird true zurückgegeben, ansonsten false . 
Beispiel zu b): 
Der Wert an 1. Stelle enthält den Text „Fischers Fritz fischt frische Fische“. 
a.	Wird nun im 2. Wert „Fischers“ angegeben, ist die Bedingung wahr. 
b.	Wird im 2. Wert „Fritz“ angegeben, ist die Bedingung nicht wahr. 
#### 6.	RegexIsMatch. 
Zur Nutzung einer Transformation für den Import zur Suche nach Ziffern. 
a)	Wert „abc123“ angeben. Es wird „1“ zurückgegeben. 
b)	Wert „abc“ angeben. Es wird „0“ zurückgegeben. 
Mit „RegexIsMatch“ kann überprüft werden, ob der angegebene Wert an 1. Stelle den an 2. Stelle angegebenen Wert enthält. Hierfür kann im Wert an 2. Stelle ein Regulärer Ausdruck verwendet werden. Ist dies der Fall wird true zurückgegeben, ansonsten false. 
Beispiel zu b): 
Der Wert an 1. Stelle enthält den Text „Fischers Fritz fischt frische Fische“. 
Das Zeichen „$“ steht im Regulären Ausdruck für das Zeilenende. 
a.	Wird nun im 2. Wert „Fische$“ angegeben, ist die Bedingung wahr. 
b.	Wird im 2. Wert „Fischers$“ angegeben, ist die Bedingung nicht wahr. 

Für das Testen dieser Funktionen im X-Mapping Editor klicken Sie dort den Button Test , geben im Feld Import etwas an und klicken dann auf Import . In der Registerkarte Daten können Sie dann das Ergebnis ansehen. 

### Ausdruckseditor mit Elementen 
Es gibt hier weitere Möglichkeiten für den Import von Daten beim X Mapping. Der Ausdruckseditor kann beim Import auf Daten angewendet werden. Beim Export der Daten kann dieser Dialog analog verwendet werden. 

Tipp: Im Ausdruckseditor des X-Mappers ist es bei Import möglich auf Werte außerhalb des gemappten Knotens zuzugreifen, wenn Sie über den Button „ ../ “ den gemappten Knoten verlassen und auf einen anderen Wert zugreifen. 

Beispiel: 
XML-Mapping: ZUGFeRD 1.0 Import EXTENDED XML 
XML-Knoten: AppliedTradeAllowanceCharge 

-> Ausdruckseditor 
-> Über „ ../ “ den gemappten Knoten verlassen und auf einen anderen Wert zugreifen. 

Der Ausdruckseditor für Elemente wird verwendet, um aus einem Bereich der Datei (beginnend bei einem entsprechenden Knoten) einzelne Elemente auf ein Property in eNVenta zuzuordnen. Diese Zuordnung kann dann regelbasierend erfolgen, z.B. wenn eine 1 im tag steht, wird der Wert A eingetragen, ansonsten Wert B usw. 

Der Ausdruckseditor kann auch eingehende Objekte und deren Aufbau anzeigen und daraus Ausdrücke aufbauen. Beim Speichern werden dann statt der verwendeten Properties XPath-Daten des Objekts gespeichert. Dabei werden sie als <Xpath> gekennzeichnet. 

Der Aufruf des Ausdruckseditors mit Daten kann nur aufgerufen werden, wenn der Knoten kein Endknoten ist.

### Baumansicht Aufbau 

Aufbau des Baums 

Ganz oben im Baum gibt es immer einen Knoten mit dem Namen "root". Darunter befindet sich der eigentliche Ausdruck. 

Wenn unterhalb vom "root"-Knoten kein Ausdruck existiert, dann wird dies als ein leerer Ausdruck angesehen. 

So können Sie also einen kompletten Ausdruck löschen, indem Sie den ersten Knoten unterhalb von "root" entfernen. 

Unterhalb von "root" darf es maximal einen Knoten geben. Dieser Knoten enthält den Rückgabewert an die aufrufende Funktion. Der Datentyp des Rückgabewerts ist hierbei fix vorgegeben und muss eingehalten werden. Ansonsten kommt es zu einer Fehlermeldung in der weiteren Verarbeitung. 

Während der Bearbeitung des Ausdrucks können Sie auch vorübergehend mehrere Knoten unterhalb von "root" platzieren, um sie dort zu puffern. Am Ende der Bearbeitung darf es aber maximal einen einzigen Knoten geben. 

Das Resultat dieses Knotens ist somit auch das Resultat des gesamten Ausdrucks. 

Anzeige eines Knotens im Baum 

Ein Knoten besteht aus folgenden Teilen: 
Name des Knotens 
"Wert einer Konstanten" 
(Name einer Spalte oder eines Properties) 
<Typ des Teil-Ausdruckes> bzw. < # ERROR # > bei Fehlern 

Beispiel: 
StartsWith <bool> 
Col (Artikelnr) <FSstring> 
Constant "A1" <FSstring> 

Hier wird geprüft, ob die Artikelnummer mit einem bestimmten Text anfängt. 

Im Detail: 

StartsWith ist eine Funktion, die als Rückgabewert ein "bool" hat. 

Die beiden Unter-Knoten geben jeweils einen Wert mit dem Typ "FSstring" aus. 

Der erste Knoten beinhaltet die Spalte "Artikelnr" und der 2. Knoten beinhaltet eine Konstante mit dem Wert "A1". 

Im Falle von Unverträglichkeiten bei den Dateitypen oder aber auch bei strukturellen Fehlern im Baum wird als Typ ein Fehler zurückgegeben. 

Beispiele: 
StartsWith < # ERROR # > [ Artikel beginnt mit einer 3 ?] 
Col (Artikelnr) <FSstring> 
Constant "3" <FSlong> 

Hier liegt z.B. ein Fehler vor, weil die Funktion StartsWith genau 2 Parameter mit dem Typ "FSstring" erwartet. Der 2. Parameter erfüllt dieses Kriterium nicht. 

Baum bearbeiten 

Wenn Sie einen neuen Knoten hinzufügen, dann wird dieser immer unterhalb des ausgewählten Knotens eingefügt. 

Neue Knoten können Sie aus der Registerkarte Auswahl Vorlagen und aus der Registerkarte Auswahl Spalten einfügen. 

Knoten können im Baum per Drag&Drop verschoben werden. Dabei wird der ausgewählte Knoten in einen anderen verschoben. Der Knoten landet immer am Ende, hinter den evtl. existierenden Unter-Knoten. Das Programm führt beim Verschieben einige Prüfungen durch. Wenn der Verschiebe-Vorgang nicht möglich ist, dann passiert nichts. Es werden bei diesen Prüfungen aber nicht alle Belange berücksichtigt (wie z.B. falsche Datentypen). 

Bei einigen Funktionen ist die Reihenfolge der Parameter von Bedeutung. Wenn Sie die Reihenfolge ändern möchten, dann tun Sie dies, indem Sie den ersten Knoten woanders hin verschieben und anschließend wieder zurück unter den ursprünglichen Knoten. Somit ist aus dem ersten Parameter der letzte geworden. 

Den ausgewählten Knoten können Sie mit dem Button entfernen. Diese Aktion können Sie nicht rückgängig machen. Wenn Sie den falschen Knoten gelöscht haben, dann müssen Sie den Dialog ohne zu speichern schließen und den Ausdruck erneut bearbeiten. Einige Knoten können nicht gelöscht werden (z.B. "root", "Condition" unterhalb von "If"). In diesem Fall wird der Löschen-Button deaktiviert. 

Der gesamte Baum wird intern als XML gespeichert. Sie können auch für den ausgewählten Knoten mit dem Button <XML> dieses XML bearbeiten. Dabei öffnet sich ein Dialog (--> Ausdruck XML ). Mit diesem können Sie z.B. ganze Baum-Teile in die Zwischenablage packen und später an anderer Stelle wieder einsetzen. Das ist aber nur von erfahrenen Benutzern durchzuführen, da bei diesem Vorgehen sämtliche Prüfungen umgangen werden können. Wenn am Ende kein gültiger Ausdruck entsteht, dann kann dieser auch nicht übernommen werden. 

Siehe Kapitel Ausdruck XML .
