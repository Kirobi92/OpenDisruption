# Grundeinstellungen für das Mapping 
Grundeinstellungen 

Der Button (Grundeinstellungen) im Dialog X Mapping Editor öffnet diesen Dialog. Hier definieren Sie die grundlegenden Einstellungen für das Mapping. Diese Vorgaben gelten dann für alle Properties und sollten nicht ohne Rücksprache mit dem Kommunikationspartner nachträglich geändert werden.

Tooltip:

### Combobox Zeilenumbruch 
Für den Import und Export von Datensätzen mit mehreren Zeilen ist es möglich, eindeutig festzulegen, wie der entsprechende Zeilenumbruch gesteuert wird. Dies ist pro Schnittstelle bzw. Mapping, welche mittels X-Mapping bedient wird, zu definieren. 

Hier geben Sie an, mit welchen ASCII-Code der Zeilenumbruch erfolgt. In Windows überlicherweise CR+LF, bei anderen Systemen kann dies abweichend sein. Geben Sie hier die entsprechende Kennung ein. Wenn Sie keine Kennung angeben, so wird von CR+LF ausgegangen. 

Hier gibt es 3 Möglichkeiten. Diese Combobox wird nur sichtbar, wenn es sich beim Mapping um ein Text-Mapping handelt. D.h. im Dialog X-Mapping Editor ist als Strukturtyp Text zu wählen! 

Nach Auswahl des Zeilenumbruchs, entweder 

- Windows (CR + LF) - UNIX (LF) - Macintosh (CR) 
aktivieren Sie noch optional die Checkbox Fügen Zeilenende am Ende des Dokuments hinzu . 

Standardardmäßig bzw. wenn nichts angegeben ist wird Windows (CR+LF) genutzt. 

Siehe Kapitel X-Mapping Editor . 

### Checkbox Zeilenumbruch am Ende des Dokuments 
Nach Auswahl eines Werts aus der Combobox Zeilenumbruch aktivieren Sie noch optional diese Checkbox. 

### Combobox Textumbruch 
In eNVenta werden sowohl im X-Mapper als auch im Datenimport alle Zeilentrenner (CR, LF und CR+LF) gleichbehandelt und als echte Zeilenenden interpretiert. Eine differenzierte Handhabung für den X-Mapper kann hier in den Grundeinstellungen gesteuert werden. 

Standardmäßig wird der Wert „UNIX (LF)“ genutzt. 

Hinweis: Dies verwendet eNVenta in einem mehrzeiligen Feld für den Zeilenumbruch. 

Wählen Sie das ASCII-Zeichen für einen einfachen Textumbruch an. Dies kann bei mehrzeiligen Texten innerhalb eines Datenfeldes vorkommen und wird entsprechend nicht als Trenner der Inhalte betrachtet.

### Feld X-Filter 
Hier können Sie einen XPath-Ausdruck (eine Wahr-/Falsch-Bedingung) hinterlegen, um zu filtern, welche Dokumente überhaupt für diese Konfiguration gültig sind. 

Der Ausdruck wird vor dem Mapping ausgewertet. Wenn die Prüfung nicht richtig (d.h. false) sein sollte, wird nicht gemappt. 

Im XPath-Ausdruck können Sie z.B. prüfen, ob bestimmte Elemente vorhanden sind, einen bestimmten Wert enthalten und vieles weitere mehr. Die Möglichkeiten sind ähnlich wie bei SQL. 

Beispiel: 
//BuyerParty/ID='BP1' or //BuyerParty/ILN='4711' 

In diesem Fall muss es in dem Dokument einen Xml-Knoten 'BuyerParty' geben, dessen 'ID'-Knoten den Wert 'BP1' oder dessen 'ILN'-Knoten den Wert '4711' enthält. 

Hinweis: Der XPath-Ausdruck muss ohne Namespace-Bezüge angegeben werden und wird auch ohne Berücksichtigung der Namespaces ausgewertet. 

Weitere Dokumentation zur Benutzung von XPath finden Sie unter: 

www.w3.org/TR/xpath
