### Feld Sammelkonto 
Bei der Neuanlage eines Kunden wird anhand der zugewiesenen Währung das entsprechende Sammelkonto eingetragen. Dieses Konto kann auch manuell geändert werden, allerdings sollte unbedingt beachtet werden, dass das Konto, welches hier eingetragen wird, auch besteht und in der richtigen Währung geführt wird. 
Wird die Währung des Kunden geändert, so wird auch das zugehörige Sammelkonto automatisch angepasst.

### Combobox Kundenstatus 
Hier wird der Status des Kunden festgelegt. Folgende Status stehen zur Verfügung: 
- Normal : Alle Funktionen stehen für den Kunden zur Verfügung. - Gesamtsperre : Es können keine Vorgänge zum Kunden erfasst werden. - Liefersperre : Zum Kunden können zwar Aufträge erfasst, aber keine Lieferungen ausgeführt werden. - Fakturasperre : Für den Kunden können Lieferscheine gedruckt werden. Achtung: aber keine Rechnungen. 

Tooltip: 

0, leer = Normal 
1 = Gesamtsperre 
2 = Liefersperre 
3 = Fakturasperre 

### Combobox Sperrgrund 
Aus dieser Combobox können Sie einen Grund für die Sperrung eines Kunden auswählen. So sehen Sie auf einen Blick, warum der Kunde gesperrt ist. Diese Combobox wird erst zugänglich, wenn der Kundenstatus auf Gesamt-, Liefer- oder Fakturasperre steht. Der Sachbearbeiter, der einen Sperrgrund einfügt, wird gespeichert. Diese Änderungen werden in einer Historie gespeichert. 

Tooltip: 

Sperrgrundnummer zum Kundenstatus

### Feld Betrag 
Hier kann ein Kreditlimit für den Kunden festgelegt werden. Falls der Betrag überschritten wird, erhalten Sie eine Meldung, wenn Sie einen Auftrag zu diesem Kunden erfassen. Sie erhalten die Möglichkeit, eine Begründung für die Überschreitung einzugeben. 

Die absoluten Kopfrabatte werden in der Kreditlimitprüfung nicht berücksichtigt. 

Das Kreditlimit ist immer in "Haus-Währung" anzugeben! 

Tooltip: 

Betrag des Kreditlimits 

### Checkbox Strikt 
Ist zusätzlich zu einem Eintrag im Feld Kreditlimit WW der Parameter Strikt gesetzt, können bei Überschreitung des Kreditlimits keine Aufträge erfasst werden, es sei denn in den Parametern des Verkaufs ist ein Passwort für das strikte Kreditlimit hinterlegt und dieses wird bei der Auftragserfassung eingegeben. 

Siehe Kapitel Parameter Verkauf . 

### Checkbox Auf Verbandsebene prüfen 
Wenn Sie diese Checkbox aktivieren, wird das Kreditlimit nicht auf den Kunden, sondern den Verband geprüft. 

### Feld Betrag 
Hier können Sie dem Kunden ein Sonderkreditlimit (in System-Währung) einräumen. 

Ist das Sonderkreditlimit vom Datum her noch aktuell, erscheint dieser Betrag in der Bonität im Feld Kreditlimit . Der Betrag wird u.U. noch zu anderen bestehenden Kreditbeträgen hinzuaddiert. 

Tooltip: 

Angabe des Betrags 

### Feld Gültig bis 
Mit diesem Feld können Sie ein Sonderkreditlimit zeitlich eingrenzen. 

Ist das Sonderkreditlimit vom Datum her noch aktuell, erscheint dieser Betrag in der Bonität im Feld Kreditlimit . Der Betrag wird u.U. noch zu anderen bestehenden Kreditbeträgen hinzuaddiert. 

In das Feld S-Kreditlimit bis kann auch ein negativer Betrag eingegeben werden, da so das Kreditlimit eines Kunden „gedrückt“ werden kann. 

Tooltip: 

Geben Sie ein Bis-Datum ein. 

### Feld Text 
Hier kann ein Text zum Sonder-Kreditlimit eingetragen werden. 

### Feld Druck Avis ab 
Mit einem Eintrag in diesem Feld (Anzahl der OPs) können für Bankeinzug und Zahlungsverkehr die Einstellungen aus den Parametern übersteuert werden. Auf diese Weise kann definiert werden, ob und ab welcher Position ein Avis gedruckt werden soll. 

### Checkbox Voraus-Avis 
Wenn diese Checkbox aktiviert ist und wenn im Einzugsvorschlag ( Bankeinzug ) auf der Registerkarte Selektionen 1 die Checkbox Voraus-Avis aktiviert ist, nur diese Kunden selektiert. 

Die Checkbox Voraus-Avis dient dazu, die SEPA Vorab-Benachrichtigung zu erstellen. Die separate SEPA Vorab-Benachrichtigung wird versendet, wenn z.B. dem Zahlungspflichtigen auf der Rechnung der Einzugstermin nicht mitgeteilt wird. 

Tooltip: 

Voraus-Avis (SEPA Pre-Notification) 

Die Pre-Notification ist die verpflichtende Information des Gläubigers an den Zahler über die „anstehende Lastschrift". 
Der Zahlungspflichtige hat so die Möglichkeit, für eine ausreichende Deckung auf seinem Konto zu sorgen. 

Im Dialog Kunde, Registerkarte Rechnungswesen, steuert die Checkbox "Voraus-Avis", ob der Kunde bei der Erstellung von Voraus-Avisen berücksichtigt werden soll. 

Ist im Dialog Einzugsvorschlag, Registerkarte Selektionen 1, die Checkbox „Voraus-Avis“ aktiviert, werden Voraus-Avise erstellt. 
Soll für diesen Einzugsvorschlag die Bankdatei erstellt und der Vorschlag verbucht werden, muss die Checkbox „Voraus-Avis“ 
im Einzugsvorschlag auf der Registerkarte Selektionen 1 deaktiviert werden. 

### Feld E-Mail Avis 
Hier wird eine E-Mail-Adresse für den Versand eines Avis eingetragen. 

Um sicherzugehen, dass eine E-Mail an die richtige Adresse gesendet wird, prüft eNVenta beim Speichern oder Ändern einer Mailadresse, ob die Mail-Adresse der „Norm“ entspricht. Falls nicht wird ein Hinweis ausgegeben. Dabei wird das Speichern nicht grundsätzlich verhindert. Auf diese Weise werden z.B. Outlook-Adressbucheinträge wie z.B. „#EW_ERP“ bei Bedarf gespeichert und genutzt. 

Dabei läuft die Prüfung ausschließlich bei Anlage und Änderung einer Mail-Adresse. Werden andere Einträge, z.B. des Kundenstammsatzes geändert, so kommt diese Meldung nicht. 

Werden Stammdaten über das Gateway importiert, so werden u.U. fehlerhafte Mailadressen importiert (ggfs. mit Warnung). Bei einem Import unter Verwendung des Jobservers erfolgt dies ebenfalls in Form einer Warnung (im Jobprotokoll). 

Tooltip: 

Angabe der E-Mail-Adresse. 
Bei der Angabe mehrerer Adressen, müssen diese durch Semikolon getrennt sein.

Siehe Kapitel Unterstützung One-Stop-Shop .

### Checkbox Unternehmer / Juristische Person 
Diese Checkbox steht im Zusammenhang mit der Änderung im Umsatzsteuergesetz § 13b. Bei der Ermittlung der Steuerschlüssel in Verkauf und Einkauf wird immer geprüft, ob es sich beim Kunden um einen Unternehmer handelt. 

Wenn eine Bagatellgrenze auf dem Steuerschlüssel hinterlegt wurde, diese aber unterschritten ist, wird der Unternehmer wie ein Privatkunde behandelt. 

Wenn die Bagatellgrenze überschritten ist, werden die Steuerschlüssel unter Spezifische Umsatzsteuer , Registerkarte Unternehmer verwendet. 

Bevor Sie einen Kunden als Unternehmer kennzeichnen, halten Sie ggf. Rücksprache mit Ihrem Steuerberater. 

Siehe Kapitel Bagatellgrenze . 

Siehe Kapitel Spezifische Umsatzsteuer . 

Tooltip: 

Falls gesetzt, dann wird dieser Kunde bei der Belegkalkulation 
steuerrechtlich (gemäß §13b) als Unternehmer behandelt.
