# Mietobjekt 
Mietobjekt 

Dieser Dialog dient zur Verwaltung von Mietobjekten. Das Mietobjekt beschreibt den Artikel, der vermietet werden soll und alles was dazu gehört (sogenannte Zusätze). 

Beispiel: Bagger mit Schaufel und Kraftstoff (Diesel). 

Neue Mietobjekte können angelegt oder bereits vorhandene gelöscht werden. Gelöscht werden können jedoch nur Mietobjekte, für die es noch keine Vorgänge im System gibt. Sind bereits Vorgänge für das Mietobjekt vorhanden, so kann es nur gesperrt aber nicht gelöscht werden. Der Dialog kann auch als Suchdialog genutzt werden. 

Beim Anlegen eines neuen Mietobjekts ist zwingend ein Bezugslager anzugeben, damit die Vermietung generiert werden kann. 

Ein Mietobjekt kann nur für Artikel festgelegt werden, die im Artikelstamm als ‚Mietartikel‘ gekennzeichnet wurden. 

Siehe Kapitel Mietartikel . 

Standard-Mietobjekte mit Ressourcenverwaltung können nur 1-mal vermietet werden. Daher findet eine Reservierung über die Ressourcenverwaltung statt. 
### Retoure 
Hier wird die bestehende Retourenverwaltung im Bereich Verkauf genutzt. Dieser kann ein Auftrag bzw. ein Mietauftrag (Vermietung, Mietvertrag) mitgegeben werden. Im Detailbereich erscheinen die einzelnen Positionen, wo die Rückkaufposition zurückgeholt und verbucht werden kann.

### Combobox Klasse 
Hier wählen Sie eine Mietobjekt-Klasse aus. Zur Verfügung stehen: 

- Standard: Dieses Mietobjekt kann nur einmal in der Vermietung vorkommen (z.B. Bagger). Optional kann dies mit der Ressourcenverwaltung kombiniert werden. - Versicherung: Für Mietobjekte, die versichert werden müssen und der Kunde keine eigene Versicherung nachweisen kann, wird diese MO-Klasse gewählt. - Paketmietobjekt: Diesen Eintrag wählen Sie aus, wenn Sie ein Mietobjekt haben, dem noch weitere Mietobjekte zugewiesen sind (auf der Registerkarte Zusätze ). - Massenartikel: Diesen Eintrag wählen Sie aus, wenn dieses Mietobjekt mehrfach in der Vermietung vorkommen kann (z.B. Schalungsteile). 
Das Paketmietobjekt ist ein Sonderfall, für den es keine Bestandsführung gibt und dient als Hülle für die unter Zusätze hinterlegten Artikel. 

Der Status im Mietobjekt wird vom System automatisch gesetzt, je nachdem, wo sich das Mietobjekt befindet. Er kann auch manuell gesetzt werden. 

Ausnahme: Wird ein Paketmietobjekt ohne Inhaltspositionen gespeichert, so geht der Status auf Nicht verwendbar und muss manuell gesetzt werden, wenn der Fehler korrigiert ist. 

Tooltip: 

Mietobjekt Klasse 
Definiert die Klasse des Mietobjektes. 
Standard – Dieses MO kann nur einmal in einer Vermietung vorkommen (z.B. Bagger). Optional mit der Ressourcenverwaltung kombinierbar. 
Versicherung – Dieses MO stellt eine Versicherungsposition dar. So gekennzeichnete MOs können als Zusätze bei MOs, 
welche versicherungspflichtig sind, hinterlegt werden und werden automatisch in die Vermietung übernommen, wenn der Kunde keine eigene Versicherung nachweist. 
Paketmietobjekt – Dieses MO stellt ein „Hüllobjekt“ dar. D.h., so gekennzeichnete MOs bilden den Rahmen für die unter Zusätze hinterlegten Positionen 
und ermöglichen die Kalkulation eines „Paketpreises“ für die Vermietung. 
Massenartikel – Dieses MO kann mehrfach in einer Vermietung vorkommen. (z.B. Schalungsteile) 

### Combobox Status 
Diese Combobox steht nur dem Prozess-Administrator zur Verfügung. Dieses Kennzeichen kann im Dialog Sachbearbeiter (Registerkarte Allgemein ) gesetzt werden. Für andere Sachbearbeiter ist die Combobox nicht zugänglich. 

Hier wählen Sie den Status des Mietobjekts aus. Folgende Status stehen zur Verfügung: 

- Offen : das Mietobjekt ist in Ordnung und kann vermietet werden. - Vermietet : das Mietobjekt wurde einem Mietvertrag zugeordnet, ist somit vermietet und steht nicht zur Verfügung - Gesperrt : das Mietobjekt wurde gesperrt. Dies kann vorkommen, wenn sich das Mietobjekt z.B. derzeit in Wartung oder in Reparatur befindet. - Nicht verwendbar : das Mietobjekt darf nicht mehr verwendet werden, da es einen Zustand erreicht hat, der eine Weitervermietung ausschließt. 
Siehe Kapitel Sachbearbeiter .
