# Importieren vom Offline Scanner 
Importieren vom Offline Scanner 

In Auftrag und Thekenerfassung unter Optionen finden Sie den Eintrag Importieren von Offline Scanner . Damit wird die Schnittstelle für den Offline-Scanner geschaffen. 

Es öffnet sich der Dialog Importieren vom Offline Scanner . Damit lassen sich im Material-Abholbereich ebenfalls Kommissionierungen für eingescannte Positionen eines Auftrags ausführen. 

Es stehen 2 Offline-Scanner zur Verfügung, die von eNVenta genutzt werden können! Dies sind der Scanner DENSO BHT-800 von OPAL und der Offline-Scanner Unitech 630. 

Für jeden möglichen Scanner ist eine eigene Kommunikation aufzubauen! 

Folgende Aufgaben kann der Unitech Scanner übernehmen: 

- Übernahme der Daten (vollständig eNVenta gesteuert) - Löschen der auf dem Scanner gespeicherten Daten (vollständig von eNVenta gesteuert) 
eNVenta liest die Daten vom Scanner und löscht sie bei entsprechend gesetzter Option shtDeleteAuto automatisch, ansonsten auf Knopfdruck in eNVenta und grundsätzlich nur beim Unitech-Scanner (Löschen der Daten am Scanner nur über Administratormodus möglich!). 

Mit dem Button Starten starten Sie die Scanner-Abfrage, aber Achtung, damit werden bisherige Ergebnisse gelöscht!!! 

Beim Einlesen wird versucht, die Artikel anhand der Artikelnummer, der EAN und der Fremd-Artikelnummer zu erkennen. Falls ein Artikel nicht korrekt im System erkannt werden konnte, so wird eine entsprechende Fehlermeldung ausgegeben. 

### Daten aus CSV- bzw. XML-Dokumenten importieren 
Sie haben die Möglichkeit, Daten zusätzlich aus CSV- oder XML-Dokumenten in den Auftrag zu importieren. 

In diesem Zusammenhang ist auch die Spalte ‚Menge‘ editierbar und so kann die Menge geändert werden. 

Diese Anpassungen im Dialog werden direkt in den Auftrag übernommen. 

Für den Import aus CSV- oder XML-Dokumenten müssen Formate (die mit X-Mapping generiert wurden) erstellt und hinterlegt werden. So weiß eNVenta , an welcher Stelle sich das Artikelnummernfeld und das Mengenfeld in der Datei befinden. 

Im Dialog Parameter Scanner müssen diese Einstellungen auf der Registerkarte DataFile gepflegt werden. 

Das Feld X-Mapping Setting ID auf der Registerkarte DataFile steht zur Verfügung, um ein Setting auszuwählen. Mit dem -Button kann das Setting editiert und bearbeitet werden. 

Dann ist im Dialog Parameter Scanner noch die Codierung anzugeben. In unserem Fall ist das die UTF-8 . 

Jetzt wählen Sie noch die Übertragungsart aus. Es kann zwischen Server und lokal gewählt werden. Allerdings ist Server die empfohlene Einstellung, so dass die Daten zentral und von mehreren Anwendern (Scanner) aktualisiert werden. 

Als Dateiname wird hier der absolute Pfad der Datei angegeben. 

Siehe Kapitel Parameter Scanner .
