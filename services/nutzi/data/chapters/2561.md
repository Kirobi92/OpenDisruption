# OP-Übernahme Registerkarte Fakturaübernahme 
OP-Übernahme Registerkarte Fakturaübernahme 

Wenn in der Warenwirtschaft ein Auftrag bzw. eine Gutschrift (Verkauf) fakturiert wird, steht diese Faktura im Belegjournal der Warenwirtschaft. Das Belegjournal ist somit die Grundlage für die Übernahme und referenziert hier auf ein Kennzeichen, welches sich als Feld auf der Datenbanktabelle befindet. Alle Fakturen, welche als Kennzeichen ein „N“ für Nein haben, werden bei Selektion hier in die Fakturaübernahmetabelle temporär eingelesen. Alle anderen, die ein „J“ für Ja haben, bleiben außen vor, da diese bereits schon übernommen worden sind. Natürlich können Sie die Selektion auch gemäß den hier vorhandenen Feldern einschränken, so dass nur die gewünschten Fakturen kommen. Zusätzlich können Sie auch manuell gewünschte Fakturen von der Übernahme ganz ausschließen bzw. vorübergehend ausschließen. 

Wenn im Auftrag die Zahlungsbedingung eines Kunden geändert wird, so wird diese Zahlungsbedingung in die OP-Übernahme eingetragen. D.h. die Spalten ‚Nettotage‘, ‚Skonto‘ und ‚Tage‘ werden entsprechend der geänderten Zahlungsbedingung gefüllt. Damit hat die geänderte Zahlungsbedingung Priorität vor der ursprünglichen Kunden-Zahlungsbedingung. 

Bei Übernahme der Faktura wird der Datensatz geprüft, welcher einen ZM-relevanten Steuerschlüssel hat, ob eine Ust-IdNr. vorhanden ist. 

Wenn JA , wird die USt-IdNr. im Journal und im Menü OP Debitoren übernommen. 

Wenn NEIN , wird die Übernahme verweigert. Die Spalte ‚Ust-IdNr‘ ist editierbar, so dass die Umsatzsteuer-Nr. nachgetragen werden kann. 

Nach der Übernahme werden alle übernommenen Posten aus der jeweiligen Tabelle gelöscht. Konnten ein oder mehrere Posten nicht übernommen werden, bleiben sie in der Tabelle stehen. In der letzten Spalte der Tabelle erscheint nun für jeden nicht übernommenen Posten ein Fehlertext mit wichtigen Hinweisen. 

Über die Buttons Nicht Üb ... oder Ausschluss haben Sie die Möglichkeit Daten von der Übernahme auszuschließen. 

Zuerst markieren Sie die Datensätze, die Sie nicht übernehmen wollen, direkt in der Tabelle. 

Danach können Sie durch Betätigen der Buttons Nicht Üb... oder Ausschluss die Datensätze aus der Tabelle entfernen. 
### Button Nicht übernehmen 

Dieser Button löscht die Datensätze nur temporär für diese Übernahme. 

Tooltip:

### Button Ausschluss 

Mit diesem Button werden die selektierten Datensätze endgültig von der Übernahme in die Finanzbuchhaltung ausgeschlossen. 

Tooltip:

### Tabellenspalte USt-IdNr. 
Hier wird die Umsatzsteuer-Identifikationsnummer des Kunden angegeben. 

Ist die Funktion System >> Parameter Allgemein >>UST-ID via WS prüfen aktiviert, so wird die Umsatzsteuer-Identifikationsnummer bei der Eingabe in den Dialogen Kunde, Lieferant, Angebot, Auftrag, Gutschrift, Bestellung etc. per Webservice geprüft. 

Achtung: Diese Prüfung ist rechtlich nicht bindend. Eine rechtlich bindende Prüfung der Umsatzsteuer-Identifikationsnummer kann nur schriftlich bei der zuständigen Behörde erfolgen.
