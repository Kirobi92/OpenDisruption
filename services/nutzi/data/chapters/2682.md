# Buchen Kore 
Buchen Kore 

Mit diesem Menü kann direkt in der Kostenrechnung gebucht werden (z.B. IBL, Kalkulatorische Kosten etc.). 

Wie in der Finanzbuchhaltung bedarf es immer einer Belegnummer, einem Buchungsdatum, einer Kostenarten- bzw. einer Leistungsnummer, eines Betrags sowie eines Buchungstextes. 

Wenn die Checkbox Fix aktiviert wird, bleiben die Felder fest für weitere Buchungen vorbelegt. 

Der Buchungsbetrag kann auf mehrere Kostenstellen und bei Bedarf gleichzeitig auf mehrere Kostenträger verteilt/gebucht werden. 

Gesperrte Objekte stehen nicht zur Verfügung! 

Die einzelnen Eingabefelder und Aktionsmöglichkeiten in diesem Dialog werden im Folgenden genau beschrieben. Die Buchungen erfolgen für Kostenarten und Leistungen auf die gleiche Art. Deshalb wird nachfolgend, der besseren Lesbarkeit wegen, zum Teil auf die Unterscheidung zwischen Kostenarten und Leistungen verzichtet und der Begriff 'Kostenarten' auch für Leistungen verwendet. 

Die Periode bzw. das Jahr werden entsprechend der Eingabe des Buchungsdatums angepasst. Evtl. erhalten Sie eine Meldung, dass sich die Periode bzw. das Jahr geändert haben. 

Nun folgen die Eingaben zum Buchen auf Kostenarten bzw. Leistungen. Zunächst ist die Eingabe der Kostenartennummer erforderlich. Nach Eingabe der Kostenartennummer wird die Bezeichnung der Kostenart automatisch nachgelesen. 

Der Verteilungszeitraum wird gemäß der im Kostenartenstamm hinterlegten Angaben vorbelegt, kann aber auch manuell durch Auswahl aus der Combobox verändert werden. 

Siehe Kapitel Kostenarten . 

Das Buchungsdatum wird mit dem aktuellen Tagesdatum vorgegeben, aber auch hier ist eine abweichende Eingabe möglich. Nun ist noch die Eingabe der Belegnummer, des Betrags und des Buchungstextes erforderlich. Als Buchungstext kann entweder ein individueller Text eingegeben oder durch Auswahl aus der Combobox ein Standardtext ausgewählt werden, sofern Standardtexte zuvor in den Stammdaten angelegt wurden. Aktivieren Sie die Checkbox Fix , bleibt der Buchungstext für nachfolgende Buchungen eingetragen. 

Das Buchungsdatum kann mit den Datumswerten auf dem Kostenträger abgeglichen und die Buchung mit entsprechender Meldung ggf. abgelehnt werden. 

Werden Splittbuchungen erstellt (Verbuchung auf mehrere Kostenstellen und/oder Kostenträger), so wird die komplette Buchung abgelehnt.
