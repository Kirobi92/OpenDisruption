# Parameter MIS Registerkarte Integration Services 
Parameter MIS Registerkarte Integration Services 

Nachdem die Verbindung konfiguriert wurde, können die ersten Daten in die Registerkarte Integration Services übernommen werden. 

Als erstes müssen Sie die Zeit-Dimension generieren, welche für MIS -Analysen zur Verfügung stehen soll. Geben Sie hierfür in den Feldern Zeit von und Zeit bis das Start- und Ende-Datum der Analyse an. Das Ende geben Sie möglichst großzügig an, wenn Sie die Planung nutzen; mindestens bis zum Abschluss des nächsten Geschäftsjahres. 

Beispiel: Geschäftsjahr = Kalenderjahr 

Dann geben Sie 01.01.2017 bis 31.12.2020 ein, wenn Sie die Daten ab dem 01.01.2017 analysieren möchten und auch ein Jahr in die Zukunft planen. 

Beispiel: Geschäftsjahr vom 01.04. bis 31.03. des Folgejahres 

Dann geben Sie 01.04.2017 bis 31.03.2021 ein, damit auch das kommende Geschäftsjahr komplett geplant werden kann. 

Geben Sie als nächstes den Lade-Zeitraum für die Vorgangsdaten ein. So können Sie bei größeren Zeiträumen die Übernahme der Daten in das MIS begrenzen. 

Bei der Übernahme der Daten in das MIS werden immer alle Dimensionen (Artikel, Kunden, Lieferanten, Konten…) übernommen, unabhängig des Ladezeitraums. Der Ladezeitraum bezieht sich nur auf die Fakten bzw. Vorgangstabellen; im Verkauf auf das Erfassungsdatum des Auftrags, in der Bestellung auf das Erfassungsdatum der Bestellung, in der Finanzbuchhaltung auf das Datum der Buchung. 

Haben Sie den Job-Server installiert, können Sie die Daten direkt aus diesem Dialog übernehmen. Klicken Sie hierfür auf den Button Zeit gen , damit die Zeit generiert wird und anschließend auf den Button Ausführen , damit alle weiteren Daten in das MIS übernommen werden. 

Haben Sie keinen Job-Server installiert, so verlassen Sie die Anwendung und öffnen die Integration Services im SQL-Server. Wählen Sie das Integration Service Paket „Generate Date“ aus und führen dieses aus. 

Wurde der Job-Server nicht lizenziert, müssen alle Einstellungen direkt im SQL-Server vorgenommen werden. 

Der Job-Server muss so definiert sein, dass er Zugriff auf den SQL-Server hat.
