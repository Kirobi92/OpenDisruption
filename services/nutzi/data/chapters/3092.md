# Importkonfiguration Registerkarte Datenquelle 
Importkonfiguration Registerkarte Datenquelle 

Auf dieser Registerkarte wählen Sie aus der Combobox den Pfad aus, von wo die Daten hergeholt werden (lokal oder Server). In Abhängigkeit Ihrer Selektion erscheint ein Feld, in das Sie den kompletten Pfad der Datei eingeben oder über den Button suchen. 

Die Angabe der Pfade ist essentiell, denn sie bestimmen, wohin die Daten gesendet werden (= Basis). Diese Informationen landen dann im Artikel Schattenstamm. 

In der Tabelle darunter werden mit dem Button Neu neue Zeilen erstellt, welche Informationen über die zu importierenden Daten beinhalten. Dazu zählt die verfügbare Komponente. Es stehen Adapter für die Objekte Artikel, Artikelmerkmale, Artikelbilder, Fremdartikelnummern, Artikelpreisgruppe und Artikeleinheiten zur Verfügung. 

Es ist nicht möglich, ein „Mischformat“ anzulegen: ein Format, das sowohl X-Mapper als auch den Weg über Formate enthält! 

Bei der Anlage des ersten Datenimportformates als X-Mapper sind nur X-Mapper-Formate innerhalb der Importkonfiguration möglich. Um dies entsprechend zu unterstützen ist der Import und Export von kompletten Importkonfigurationen inkl. X-Mapper möglich. 

Importformate 

Pro Datenquelle kann ein eigenes Import-Format definiert werden, je nach Datenlieferant ist ggf. ein eigenes Import-Format notwendig. 

Das Feld Posnr. wird automatisch gefüllt. 

Das Feld Bezeichnung wird vom Benutzer gefüllt. 

Ist ein neues Mapping angelegt worden, speichern Sie dieses und wechseln mit dem Button Format in den Dialog Daten Import . 
### Button Neu 

Über diesen Button wird ein Eintrag für die zu importierende Komponente angelegt. 

### Button Löschen 

Dieser Button löscht einen Eintrag der Tabelle. 

### Button Datei 

Mit diesem Button gelangen Sie in den Dialog Datei auswählen , um die Datei in die Import Konfiguration (Spalte 'Dateiname') zu übernehmen. 

Siehe Kapitel Datei auswählen . 

### Button Format 

Dieser Button öffnet den Dialog Datenimport . Dort werden die Felder der Quelldatei den Datenfeldern in eNVenta zugeordnet. Hier können auch neue Formate erstellt oder auch bestehende geändert bzw. auch gelöscht werden. 

Siehe Kapitel Datenimport für Artikel . 

### Button X-Mapping 

Mit diesem Button öffnet sich der Dialog X-Mapping Editor . Dort können Sie für die verschiedenen Bestandteile des Artikel-Schattenstamms (Artikel, Artikeleinheiten, Artikelpreisgruppen, …) die Zuordnung zwischen der Import-Datei auf der rechten Seite und dem Import-Objekt auf der linken Seite vornehmen. 

Beim Export und Import werden die Konfigurationen für den Artikel Schattenstamm-Datenimport auf die X-Mapping-Definitionen berücksichtigt. 

Siehe Kapitel X-Mapping Editor . 

### Button Suche 

Mit diesem Button können Sie das Verzeichnis suchen, wo die zu importierenden Daten liegen. 

### Feld Pfad 
Hier wird der genaue Pfad des Verzeichnisses eingegeben. Sie können bis zu 250 Zeichen angeben. 

Tooltip: 

Angabe einer Beschreibung 

### Feld Pfad 
Hier wird eine genaue Beschreibung angegeben. Sie können bis zu 32000 Zeichen angeben (bei Oracle: 2000). 

Tooltip: 

Angabe einer Beschreibung 

### Feld Select...Where 
Hier wird die Where-Bedingung angegeben, zur Steuerung von Importen aus externen Datenbanken in das Schattenstamm-Staging. 

Tooltip: 

Where-Bedingung zur Steuerung von Importen aus externen Datenbanken in das Schattenstamm-Staging. 

### Tabelle Dateien 

Entsprechend der Selektion werden die Dateien eingelesen bzw. kann die Komponente entsprechend ausgewählt werden. 

Wird ein bestehendes Mapping übernommen, kann dieses aus der Combobox in der Spalte 'Format' ausgewählt werden. 
### Tabellenspalte Pos 
Hier wird die Positionsnummer angegeben. Diese Nummer muss je Vorgang eindeutig sein! 

Tooltip: 

Positionsnummer 

### Tabellenspalte Bezeichnung 
Hier wird die Bezeichnung angegeben. 

Tooltip: 

Angabe der Bezeichnung 

### Tabellenspalte X-Mapper 
Wenn Sie die Checkbox in dieser Spalte aktivieren, werden unnötige Felder ausgeblendet. Die Bearbeitung des Import-Formats erfolgt dann über das X-Mapping und nicht über den Button Format . Des Weiteren werden Werte auf der Registerkarte Standard Werte ignoriert. 

Tooltip: 

Wenn aktiv, wird anstelle eines Import-Formats 
ein X-Mapping zum Importieren verwendet. 

### Tabellenspalte Komponente 
Hier wird der Komponententyp angegeben. 

Tooltip: 

Angabe der Ziel-Komponente, in welche die Quelldaten überführt werden sollen. 

### Tabellenspalte Format 
Hier wird die Auswahl des über den Button Format festgeleten Datenimport-Formats angezeigt. 

Tooltip: 

Auswahl des über den Button "Format" festgelegten Datenimport-Formats. 

### Tabellenspalte X-Mapping ID 
Hier wird der Identifizierer eines Mappings angegeben, das mit dem X-Mapper erstellt wurde. 

Tooltip: 

Identifizierer eines Mappings, das mit dem X-Mapper erstellt wurde. 

### Tabellenspalte Dateiname 
Hier wird der Dateiname angegeben. 

### Tabellenspalte Pfad Verarbeitet 
Tooltip: 

Verzeichnis, in welches erfolgreich verarbeitete Dateien verschoben werden. 

### Tabellenspalte Fehlerpfad 
Tooltip: 

Verzeichnis, in welches Dateien verschoben werden, bei deren Verarbeitung ein Fehler aufgetreten ist. 

### Tabellenspalte Quell Komponente 
Hier wird der Komponententyp angegeben. 

Tooltip: 

Bei O2O-Mappings die Quell-Komponente, aus welcher die umzumappenden Datensätze stammen. 

### Tabellenspalte Datei Kodierung 
Hier wird die Zeichencodierung angegeben. 

Siehe Kapitel Formate für Exportdateien - Encodings .

Zeichenkodierung 

### Tabellenspalte Einzelausführung 
Der Dateiinhalt wird aufgeteilt. Ist ein Startwert vorhanden, wird dieser zum Teilen der Artikel verwendet. 

Die Zeile beginnt mit Startwert bis nächste Zeile mit Startwert. 

Ist kein Startwert vorhanden, wird die Datei zeilenweise verarbeitet.
