# Bestellungen Registerkarte Kopfdaten 
Registerkarte Kopfdaten 

Auf dieser Registerkarte erhalten Sie die notwendigen Daten zum Lieferanten, wie z.B. die Zahlungsart, die Art der Lieferung, die Versandart. Diese Daten werden im Lieferantenstamm festgelegt. 

Differenzierung zwischen Komplett- und Teillieferung 

Es kann zwischen Komplett- und Teillieferung differenziert werden. Dafür steht die Checkbox Komplettlieferung gewünscht auf dieser Registerkarte zur Verfügung. Wenn die Checkbox gesetzt ist, ist eine Komplettlieferung gewünscht. 

Mit Setzen dieser Checkbox wird der Lieferant über den Wunsch nach Komplettlieferung in Kenntnis gesetzt. Der Vorteil ist hier, dass keine mehrfache Bezahlung von Frachtkosten, aufgrund von Teillieferungen, mehr vorkommen kann. Es entfällt auch das mühsame Erfassen des Wunschs nach Komplettlieferung als Starttext oder als Textposition. Der Lieferant kann diese Information nun auch nicht mehr übersehen, da die gesetzte Checkbox für eine elektronische Übermittlung sorgt. 

Damit die Entscheidung bezüglich Komplettlieferung nicht jedes Mal bei einer Bestellung getroffen werden muss, was u.U. zeitaufwändig sein kann bzw. auch mal vergessen werden könnte, gibt es die Möglichkeit, dies auf Lieferantenebene zu unterbinden. Im Lieferant auf der Registerkarte Warenwirtschaft steht ebenfalls die Checkbox Komplettlieferung gewünscht zur Verfügung. 

Dieses Vorgehen betrifft vor allem Bestellungen. Bei Prozessen ohne EDI kann die Information bezüglich Komplettlieferung optional auf dem Report ausgegeben werden. Die Report-Schnittstelle ist entsprechend erweitert, damit die Kennzeichnung auf dem Report verwendet werden kann.

### Feld Valutadatum 
In diesem Feld können Sie ein Valutadatum für die Rechnungsprüfung eintragen bzw. vorbelegen. Ab dem eingetragenen Valutadatum beginnt die Fälligkeit gemäß Zahlungsvereinbarung zu laufen. Wenn dies nun für eine Bestellung in 60 Tagen ist, dann tragen Sie dort ' *+60 ' ein. 

Tooltip: 

Geben Sie ein Valutadatum für die Rechnungsprüfung an. 

### Combobox Zahlung 
Hier wird die Zahlungsbedingung angegeben. Es werden Nummer und Text angezeigt. 

Tooltip: 

Gibt die gewünschte Zahlungsbedingung an. 

### Checkbox Keine Zentralregulierung 
Diese Checkbox ist nur sichtbar, wenn es sich bei dem Lieferanten um einen Verbandslieferanten handelt. 

Das Kennzeichen hat folgende Wirkung: 

- Der OP wird im Zahlungslauf für den Verbandslieferanten berücksichtigt. - Der OP wird von der Verbandsabrechnung ausgeschlossen. 

Tooltip: 

Keine Zentralregulierung (Verbandsabrechnung bzw. Zentralzahlung)

### Feld Beistellauftrag 
Für eine bessere Transparenz in der Anwendung und dem Handling von Beistellung und Materialrücksendung kann direkt aus der Beistellung in die Materialrücksendung gewechselt werden. Diese Informationen sind in den beteiligten Belegen (Auftrag und Bestellung) sichtbar. 

Neben dem Feld zum Beistellauftrag steht der -Button zum Öffnen des Belegs zur Verfügung. D.h. für einen Wechsel aus dem Auftrag in die Bestellung und umgekehrt. 

Tooltip: 

Angabe der Belegnummer 

### Button Öffnen 

Mit diesem Button öffnen Sie den Beleg. D.h. für einen Wechsel aus dem Auftrag in die Bestellung und umgekehrt. 

### Checkbox Nicht mahnen 
Wenn Sie diese Checkbox aktivieren, wird der entsprechende Artikel/Lieferant im Mahnvorschlag des Einkaufs nicht berücksichtigt; d.h. nicht gemahnt. 

Dies macht dann Sinn, wenn z.B. Frachtkosten über eine Artikelnummer definiert sind. Diesen Artikel kann der Lieferant ja nicht liefern. 

Tooltip: 

Wenn aktiviert, wird der Vorgang nicht angemahnt. 

### Checkbox Keine WBZ aktualisieren 
Wenn die Bestellung bei der automatischen Berechnung der Wiederbeschaffungszeit nicht berücksichtigt werden soll, setzen Sie diese Checkbox. Gründe dafür können sein, dass z.B. Eilbestellungen oder Lieferschwierigkeiten eines Lieferanten die Wiederbeschaffungszeit verfälschen würden. 

Siehe Kapitel Wiederbeschaffungszeit . 

Wird die Checkbox Keine WBZ aktualisieren in den Versandarten gesetzt, so wird diese Checkbox auch automatisch in der Bestellung aktiviert. 

Tooltip: 

Wenn aktiviert, wird die WBZ nicht aktualisiert. 

### Checkbox Kein EDI 
Ist diese Checkbox gesetzt, wird der Versand per EDI nicht gestattet. 

Tooltip: 

Wenn aktiviert, wird der Versand per EDI nicht gestattet. 

### Checkbox Kein Intrastat 
Wenn Sie diese Checkbox aktivieren, erfolgt keine Berücksichtigung in der Intrastat. 

Wenn die Checkbox nicht gesetzt ist, ist zwingend ein Land anzugeben. Voraussetzung: Dieses Land ist in der Ländertabelle für Intrastat freigegeben. 

Tooltip: 

Wird nicht in der Intrastat berücksichtigt, wenn Checkbox aktiv ist. 

### Combobox Währung 
Hier können Sie die Währung festlegen. Wurde zum Kunden / Lieferanten bereits ein Vorgang erfasst oder eine Buchung vorgenommen, kann die Währung nicht mehr geändert werden. 

Wird die Währung geändert, so wird auch das zugehörige Sammelkonto automatisch angepasst. 

Wird hier nichts eingetragen, wird automatisch die Systemwährung eingetragen. 

Tooltip: 

Angabe der Währung

### Field Panel #fpnMasterDataMainFieldsDiscount# 

Bis zu 3 Rabatte oder Zuschläge können zu einer Bestellung angegeben werden. Sie beziehen sich immer auf die gesamte Bestellung und werden in der Reihenfolge der Zeilen in der Tabelle berechnet. Ein Zu- oder Abschlag kann entweder prozentual oder als absoluter Betrag angegeben werden. 

Falls zu dem Lieferanten ein Mindermengenaufschlag hinterlegt ist (Dialog Bestellwertrabatte ), wird dieser in das Feld Rabatt 1 eingetragen. Falls hier schon ein Eintrag vorhanden war, wird dieser mit dem Mindermengenaufschlag überschrieben. 

Zuschläge müssen mit negativem Vorzeichen eingegeben werden.
