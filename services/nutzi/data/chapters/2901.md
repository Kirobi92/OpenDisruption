# X-Mapping Editor Registerkarte Filter 
X Mapping Editor Registerkarte Filter 

Auf dieser Registerkarte haben Sie die Möglichkeit das Mappen von Listenelementen zu filtern. 

Das Filtern kann notwendig werden, wenn z.B. eine Collection von Auftragspositionen auch Text-Positionen enthält, die rausgefiltert werden sollen. 

Die Registerkarte Filter wird aktiv, wenn es sich bei der aktuell selektierten MapNode (markierte Zeile in der Mitte oben) um eine Schleife (d.h. Mapping einer Liste) handelt. 

Im Feld X-Filter kann ein Filter für den Export bzw. Import angegeben werden. Dabei kommt der Ausdruckseditor zum Einsatz. Als Daten-Input für den Ausdruckseditor (Ergebnis-Typ -> bool) gilt für den Export die entsprechende Komponente der Komponentenkollektion und für den Import das entsprechende Element. 
### Checkbox Filter Export 
Wenn Sie diese Checkbox aktivieren, wird ein Filter für den Export aktiv. 

Tooltip: 

Filter für den X-Export 

### Button Filter Bearbeiten 

Über diesen Button kann ein Filter für das Mappen von Objekt-Daten zu XML bearbeitet werden. 

Wird ein Filter über diesen Button gesetzt, wird die Checkbox automatisch aktiviert. 

Tooltip: 

Öffnet einen Dialog zum Bearbeiten des Filters für das Mappen von Objekt-Daten zu XML. 

Sie können durch einen Filter nur bestimmte Elemente aus der Liste mappen. 

### Button Filter Löschen 

Mit diesem Button kann ein Filter gelöscht werden. 

### Checkbox Filter Import 
Wenn Sie diese Checkbox aktivieren, wird ein Filter für den Export aktiv. 

Tooltip: 

Filter für den X-Import 

### Button Filter Bearbeiten 

Über diesen Button kann ein Filter für das Mappen von Objekt-Daten zu XML bearbeitet werden. 

Wird ein Filter über diesen Button gesetzt, wird die Checkbox automatisch aktiviert. 

Tooltip: 

Öffnet einen Dialog zum Bearbeiten des Filters für das Mappen von XML zu Objekt-Daten. 

Sie können durch einen Filter nur bestimmte Elemente aus der Liste mappen. 

HINWEIS 
Falls zusätzlich ein XPath-Filter angegeben wurde, werden beide Filter berücksichtigt, d.h. beide Bedingungen müssen zutreffen. 

### Button Filter Löschen 

Mit diesem Button kann ein Filter gelöscht werden. 

### Feld X-Filter 
Hier kann ein Filter für den Export bzw. Import angegeben werden. Dabei kommt der Ausdruckseditor zum Einsatz. Als Daten-Input für den Ausdruckseditor (Ergebnis-Typ -> bool) gilt für den Export die entsprechende Komponente der Komponentenkollektion und für den XML-Import das entsprechende XML-Element. 

In dem XPath-Ausdruck können Sie z.B. prüfen, ob bestimmte Elemente vorhanden sind, einen bestimmten Wert enthalten und vieles weitere mehr. Die Möglichkeiten sind ähnlich wie bei SQL. 

BEISPIEL 
//BuyerParty/ID='BP1' or //BuyerParty/ILN='4711' 

In diesem Fall muss es in dem Dokument einen XML-Knoten 'BuyerParty' geben, dessen 'ID'-Knoten den Wert 'BP1' oder dessen 'ILN'-Knoten den Wert '4711' enthält. 

HINWEIS 
Der XPath-Ausdruck muss ohne Namespace-Bezüge angegeben werden und wird auch ohne Berücksichtigung der Namespaces im XML ausgewertet. 

Weitere Dokumentation zur Benutzung von XPath finden Sie unter: 
www.w3.org/TR/xpath
