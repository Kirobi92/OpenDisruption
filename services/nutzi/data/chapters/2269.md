# BMECat - Import Registerkarte Erweitert 
BMEcat Import Registerkarte Erweitert 

Auf dieser Registerkarte kann eingestellt werden, wie die Artikel in den Schattenstamm importiert werden sollen. Die Preise können in EK-Konditionen oder Preisgruppen importiert werden. Der wesentliche Unterschied ist, dass Preisgruppen allgemein nicht lieferantenspezifisch sind und keine Mengeneinheiten unterstützen. Der Import in Preisgruppen bietet sich deshalb vor allem dann an, wenn die Artikel hauptsächlich über den Kataloglieferanten bezogen werden. 

Lieferantenkataloge können als EK-Konditionen oder in eine Preisgruppe importiert werden, Kundenkataloge können nur in eine Preisgruppe importiert werden. 

Ist „Gültig ab“ gesetzt (in der Tabelle Preise importieren ), wird zunächst geprüft, ob der Gültigkeitszeitraum bereits existiert. Ist das nicht der Fall, wird ein neuer Gültigkeitszeitraum angelegt. Existiert der Zeitraum bereits, werden die Werte in dem Gültigkeitszeitraum überschrieben. Die Staffeln werden ebenfalls bezogen auf den Gültigkeitszeitraum angelegt / aktualisiert. 

Im BMEcat Import wird ausgewählt, ob die Preise aus dem BMECat in eine Preisgruppe oder die EK-Kondition übernommen werden sollen. Beim Einlesen der Datei wird zunächst analysiert, welche Preislisten mit welchen Gültigkeiten existieren. 

Der Sachbearbeiter muss jetzt festlegen, wie die Preise verarbeitet werden sollen. Die Auswahl kann pro Preistyp festgelegt werden. Es ist zulässig, dass keine Preisgruppe ausgewählt wird. Zusätzlich kann der Benutzer noch eine Kombination als EK-Kondition markieren. Ist die Position markiert, wird der Preis in die EK-Kondition übernommen. Ist bei dem markierten Datensatz zusätzlich eine Preisgruppe ausgewählt, wird der Preis nicht in die EK-Kondition übernommen, sondern in die Preisgruppe, und die ausgewählte Preisgruppe in das Feld Preisgruppe bzw. Gültigkeit -> Preisgruppe der EK-Kondition eingetragen. 

Die Auswahl der Preisgruppen pro Preisliste ist mit der Importkonfiguration speicherbar. Dazu gibt es in der BMECatConfig ein Text-Datenfeld, in welches eine XML-Datei mit den Config-Details gespeichert werden kann. 

Die Konfiguration befindet sich auf der Registerkarte Erweitert und ist nur aktiv, wenn auf der Registerkarte Allgemein die Option Artikel und auf der Registerkarte Artikel die Option Preise aktiviert ist. 

Für das komfortablere Einlesen von Artikeldaten (per CSV-Datei) in den Artikel Schattenstamm und damit auch gleich in den Artikelstamm über den BMEcat gibt es in eNVenta folgende Funktionalitäten: Die Checkbox Schattenstamm und die Combobox Default Artikelgr .

### Tabelle Link-Typ 

In dieser Tabelle wird angegeben, welche Daten wie importiert werden sollen. Die Ressourcen aus der BMEcat-Datei sind als Referenztypen zugewiesen. 

Tooltip: 

Hier wird festgelegt, wie die möglichen BMECat-MIME_PURPOSE in eNVenta umgesetzt werden sollen. MIMEs besitzen optional einen PURPOSE, welcher die Funktion der MIME-Ressource vorgibt. Falls der Katalog Zuordnungen enthält, müssen diese vollständig sein. 

Wird ein MIME_PURPOSE einem eNVenta-Linktyp zugeordnet, gilt dieser für sämtliche im Katalog aufgeführten Instanzen eines bestimmten MIME_PURPOSE. 

Hinweis: MIME_PURPOSE hat meist Produktbezug. Ausnahme ist der MIME_PURPOSE "logo", welcher aus dem Lieferanten-Header kommen kann (BMECat 1.2).
