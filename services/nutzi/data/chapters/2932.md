# Datenimport Adapter 
Datenimport Adapter 

Wenn Sie die Zeile mit der Adapter Nr. Datenimport (im Dialog Gateway Adapter Kontext ) markiert haben und den Button aktivieren, öffnet sich der Datenimport Adapter . Der Import verhält sich wie der Export. Allerdings steht hier keine Where Condition zur Verfügung. 

Der Import wird pro Objekt ausprogrammiert. 

Hinweis: Zur Verfügung gestellte Objekte müssen in der Datenbank eingetragen werden. 

### Manueller Datenimport 
Um den Datenimport manuell durchzuführen, öffnet der Sachbearbeiter im Gateway Adapter Kontext den Dialog Datenimport Adapter . 

Dazu muss zuerst ein Eintrag der oberen Tabelle markiert und dazu der Button gedrückt werden. 

Der Sachbearbeiter wählt das Importformat (Adapter) aus. 

Dann wählen Sie aus der Combobox Import Format die Konfiguration aus, bzw. legen die Bezeichnung für eine neue fest. 

Beim Klick auf den Button F. Import wird die Datei auf den Server übertragen. Der Manager übergibt die Datei an den Adapter mit SetFile und setzt die Versionsinformation über die Methode GetImportVersion(). Wenn keine Information vorhanden ist, wird der aktuelle Tag als Versionsnummer genommen. 

Bei Klick auf einen Verarbeiten -Button werden die Positionen wie beschrieben verarbeitet. Nach dem Verarbeiten wird die DataImportConfiguration mit den Werten LastImportDate, LastImportCreationDate (Erstelldatum der Datei), LastImportVersion aktualisiert. 

Nach dem Import der zu ändernden Daten muss analysiert werden, welche Felder von einer Änderung betroffen sind. Die Werte CanInsert, CanUpdate, CanDelete aus der Adapter-Konfiguration steuern hierbei die Vorbelegung. Ist ein Datensatz bereits im Schattenstamm vorhanden, und ist der Wert abweichend zu dem neuen Wert, wird in die Tabelle DataImportDetailSelection ein Datensatz eingetragen. Bei der Verarbeitung werden später nur Werte berücksichtigt / in den Schattenstamm übernommen, welche hier einen Eintrag haben. 

Der Fortschritts-Balken informiert während des Imports über den aktuellen Verarbeitungsfortschritt. 

Ist die Verarbeitung abgeschlossen, müssen die Datensätze mit Änderungen vom Sachbearbeiter nachbearbeitet werden. Ziel des Entwurfs ist, eine möglichst effiziente Weiterverarbeitungsmöglichkeit zu schaffen.
