# Rechnungsimport

Grundeinstellungen für eNVenta ERP 

# PROXESS 
PROXESS 

Dieser Dialog bietet eine Schnittstelle für eine Archivanbindung zu Proxess. Auf diese Weise werden z.B. Rechnungen nach dem Scan in eine XML-Datei umgewandelt. Diese Datei kann anschließend über die Funktion Rechnung übernehmen (in der Rechnungsprüfung >> Optionen ) in die Rechnungsprüfung übernommen werden. Wenn die automatische Verbuchung im Lieferantenstamm aktiviert ist, werden die Rechnungen nicht nur angelegt, sondern automatisch in der Rechnungsprüfung validiert und ggf. freigegeben. Für die Validierung der Rechnungsprüfung müssen die Gültigkeitsregeln in der Rechnungsprüfung entsprechend vorhanden sein. Bei der automatischen Anlage der Rechnungsprüfung wird immer geprüft, ob der Wareneingang zur Rechnung bereits verbucht wurde. Hierfür wird anhand der Lieferscheinnummer aus der XML-Datei der passende Wareneingang ermittelt. 

Sollte der Wareneingang noch nicht erfolgt sein, so landet die importierte Rechnung in einem Sammel-Pool (Dialog Rechnung übernehmen ) und kann nachträglich manuell oder automatisch mit einem Wareneingang verknüpft werden. 

Dies ist eine komfortable Möglichkeit Rechnungen zu importieren und verringert die Fehleranfälligkeit gegenüber dem manuellen Import erheblich. 

Siehe Kapitel Rechnung in die Rechnungsprüfung übernehmen . 

Siehe Kapitel Wareneingang . 

### Rechnungsimport über Proxess 
Beim Import von XML-Rechnungsdaten werden die Rechnungen vom Barcode Pool automatisch in den Dokumententyp Kreditoren-Rechnung verschoben. Hier wurden die Daten bereits manuell mit einem Barcode in Proxess erfasst und werden nach dem erfolgreichen Import einer Kreditoren-Rechnung via XML in den korrekten Dokumententyp Kreditoren-Rechnung verschoben. Im Anschluss sind die Archivbelege dann in der Rechnungsprüfung einzusehen. 

Beim Import von Proxess-Rechnungsdateien (XML-Rechnungen) wird auch der Mandanten-Key geprüft, wenn in der Datei vorhanden und eNVenta mit Mandanten-Management läuft. Diese Logik sichert den Import mehrerer Mandanten vom gleichen Import-Verzeichnis ab. 

Achtung: Generell wird empfohlen für jeden Mandanten ein eigenes Quellverzeichnis anzugeben, da hierdurch entsprechende Berechtigungen gesteuert werden können. 

### Archivierung mit Proxess Channel 
Bei Archivierung einer Datei mit Proxess, wird nur noch der reine Dateiname OHNE Dateierweiterung verwendet. 

Auch mit Proxess können Dateien verändert und versioniert werden.

### Button Job-Server 

Mit diesem Button starten Sie den Job. 

Der Job prüft das Import-Verzeichnis auf neue XML-Dateien und versucht für jede Datei den Import durchzuführen. Sollte der Import auf einen Fehler laufen, weil z.B. die XML-Datei keinen korrekten Inhalt hat, so wird eine Fehlermeldung an den Sachbearbeiter bzw. die Abteilung versendet und die Datei wird in den Fehler-Ordner verschoben. 

Konnte die Rechnungsprüfung nicht automatisch verbucht werden, obwohl aktiviert, so erfolgt eine Wiedervorlage an den Sachbearbeiter bzw. die Abteilung als Warnung, damit eine manuelle Nachbearbeitung erfolgt. 

Siehe Kapitel Job Verwaltung . 

### Button Suche 

Mit diesem Button starten Sie die Suche. 

Sie können ganz allgemein suchen, indem Sie den Button drücken, gleich nach Öffnen des Dialogs. Oder Sie geben im Kopf des Dialogs die Selektionskriterien vor und drücken dann diesen Button. 

### Button Neu 

Dieser Button leert die Daten in allen Feldern, nicht gespeicherte Änderungen gehen verloren. 

Ist in den Parametern für die Sachbearbeiter die Checkbox 2 Schritt Neu aktiviert, bleiben die Suchkriterien in den Stammdialogen erhalten. 

Siehe Kapitel Parameter Sachbearbeiter . 

### Button Speichern 

Dieser Button speichert bzw. verarbeitet die Eingaben. Vor einer endgültigen Speicherung prüft das System die Eingabe. Wenn fehlende oder fehlerhafte Daten gefunden werden, erhalten Sie eine Meldung und die Eingabe muss entsprechend korrigiert bzw. vervollständigt werden. 

### Button Löschen 

Dieser Button löscht Ihre Eingaben. Genauer gesagt, die markierte Zeile wird mit dem versehen und erst endgültig gelöscht, wenn Sie den Button drücken. 

### Button Schließen 

Dieser Button schließt den Dialog. In manchen Fällen werden Sie gefragt, ob Sie die Eingaben / Änderungen noch vorher speichern möchten.
