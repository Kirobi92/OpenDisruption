Lagerverwaltungssystem (LVS) / Thekenerfassung 

# Thekenerfassung Registerkarte Abschluss 
Registerkarte Abschluss 

In vielen Projekten muss sichergestellt werden, dass ein Vorgang (z.B. Auftrag) erst dann zur Lieferung freigegeben wird, wenn alle Daten vollständig und alle Bedingungen (Prüfungen) erfüllt sind. Dazu gibt es die Funktionalität Abschluss . 

Um die Abschluss-Logik zu aktivieren, wird in den Parametern des Verkaufs die Checkbox Auftragsabschluss aktiviert. Beim Speichern des Vorgangs wird automatisch die Gesamtsperre gesetzt, diese kann nicht manuell verändert werden. Außerdem wird die Registerkarte Abschluss in folgenden Dialogen aktiv: 

- Auftrag - Angebot - Gutschrift 
Siehe Kapitel Parameter Verkauf Registerkarte Allgemein . 

Die Vorgehensweise wird nachfolgend am Beispiel Auftrag erläutert: 

Gibt es nach der Erfassung eines Auftrags keine Fehler, kann mit dem Button Abschluss (auf der Registerkarte Abschluss ) der Auftrag freigegeben werden. In diesem Fall wird automatisch der Status Gesamtsperre des Auftrags auf Normal gesetzt. 

Diverse Prüfungen werden ausgeführt und die entsprechenden Meldungen dazu auf dieser Registerkarte angezeigt. 

Folgendes wird dabei geprüft: 

- Kreditlimit, wenn der Parameter Erweiterte Kreditlimitprüfung gesetzt ist ( Parameter Allgemein , Registerkarte Wawi ). Wenn überschritten, wird entweder eine Warnung oder eine Fehlermeldung ausgegeben. 
Ein überschrittenes Kreditlimit kann in unserem WS-Package über die erweiterte Abschlusslogik zusätzlich als Info, Warnung oder Fehler konfiguriert werden. - Zahlungsbedingung. Wenn diese fehlt, wird eine Fehlermeldung ausgegeben. - Rechnungsadresse. Wenn sie fehlt, wird eine Fehlermeldung ausgegeben. - Lieferanschrift. Ist diese unvollständig oder ungültig, wird eine Warnung ausgegeben. - Lieferbedingung. Ist diese leer, wird eine Warnung ausgegeben. - Tour. Wenn die Adresse nicht zur PLZ passt, wird eine Warnung ausgegeben. - Einkaufspreis. Ist dieser 0, wird eine Information ausgegeben. - Verkaufspreis. Ist dieser 0, wird eine Information ausgegeben. - Ist der Rohertrag <= 0, wird eine Information ausgegeben. - Steuerschlüssel. Ist dieser leer, wird eine Information ausgegeben. - Verkaufseinheit. Ist diese leer, wird eine Information ausgegeben. - Einkaufseinheit. Ist diese leer, wird eine Information ausgegeben. - Auftragsstückliste. Ist diese vollständig aufgelöst, wird eine Information ausgegeben. - Positionsmenge. Ist diese zu gering bzw. zu hoch, wird eine Information ausgegeben. - Set-Artikel. Wenn ein Fehler enthalten ist, wird eine Information mit den falschen Artikeln ausgegeben. - Automatischer Verbundprozess. Ist dieser noch nicht generiert, wird eine Warnung ausgegeben. - Mindestmarge. Ist diese unterschritten, wird eine Warnung ausgegeben. - Maximale Marge. Ist diese überschritten, wird eine Warnung ausgegeben. - Lieferdatum und Fixtermin. Weichen diese ab, wird eine Warnung ausgegeben. - IC/CC-Teilnehmer. Ist dem IC/CC-Teilnehmer der Artikel nicht bekannt, wird ein Fehler ausgegeben. 
Ist die Abschlussprüfung aktiviert, kann ein Auftrag jederzeit erfasst und geändert werden, unabhängig einer Kreditlimitsperre. Allerdings kann der Auftrag nicht abgeschlossen werden und somit kein logistischer Prozess gestartet werden! 

Trotzdem erscheint zu Beginn der Neuerfassung eines Auftrags eine Hinweismeldung, falls das Kreditlimit bereits überschritten ist. 

Die Entscheidung, ob es sich um eine Streckenlieferung handelt, wird ebenfalls hier getroffen, nämlich durch Aktivieren der Checkbox Streckenlieferung . Diese Checkbox gehört zur InterCompany-Adapter-Funktionalität und schaltet den logistischen Weg frei für den automatischen InterCompany-Verkehr. 

Diese Checkbox muss gesetzt werden, bevor der Button Abschluss betätigt wird. 

Anschließend wird eine auftragsbezogene Bestellung für den Zielmandanten erzeugt. Bei nachträglichen Erweiterungen (neue Positionen) wird nur dafür eine neue Bestellung generiert. Auftragspositionen, die bereits bestellt sind, dürfen modifiziert werden. Es erscheint eine Warnung und die Bestellung muss evtl. angepasst werden. 

Mehr zum Thema Streckengeschäft erhalten Sie im Kapitel Globalabschluss und Strecke . 

Die Bestellung wird bei Klick auf den Button Abschluss generiert und muss nicht über den Button Bestellen manuell generiert werden. 

Wenn es sich um eine Streckenbestellung handelt, ist ein Löschen der Auftragsposition nicht möglich. 

Wurde der Auftragsabschluss gesetzt, so kann nachträglich die Gesamtsperre gesetzt UND auch wieder entfernt werden. Die Gesamtsperre kann manuell nicht aufgehoben werden, solange noch Fehler vorhanden sind. 

In dieser Prüfroutine können in der Tabelle entsprechende Meldungen generiert werden. Diese Meldungen können 3 verschiedene Status haben: 

- Info : Unkritische Meldung, hier kann z.B. angezeigt werden, wie viele Aufträge vom Sachbearbeiter noch offen sind, wie hoch der Deckungsbeitrag ist usw. All die Informationen, die zum Schluss eines Auftrags zusammengefasst und nochmals dem Mitarbeiter mitgeteilt werden sollen. Der Info-Status verhindert keine Freigabe. Die Zeile der Meldung ist weiß. - Warnung : Diese Meldungen sind schon kritischer zu betrachten, würden aber eine Freigabe des Auftrags nicht verhindern. Dies betrifft Meldungen wie Kreditlimit überschritten, der Kunde hat aber kein striktes Limit oder es gibt Auftragspositionen mit mehr als X EUR Auftragswert usw. Hier soll dem Mitarbeiter visualisiert werden, dass es sich z.B. um einen Eingabefehler handeln könnte. Damit die Zeile mehr Aufmerksamkeit erzielt, wird diese Gelb dargestellt. - Fehler : Ab diesem Status kann der Auftrag nicht freigegeben werden – er bleibt gesperrt bis Fehler behoben worden sind. Mögliche Fehler können z.B. sein: Striktes Kreditlimit überschritten, Kunde will Kostenstellenrechnung und die notwendigen Daten sind nicht vorhanden oder Auftrag hat Vorkasse, Geld ist noch nicht da usw. 
Kommt es beim Abschluss des Vorgangs zu einer Fehlermeldung ( rot markierter Eintrag) ist zunächst dieser Fehler zu beheben und die Anpassung zu speichern. Die rote Markierung verschwindet. Der Vorgang kann abgeschlossen werden. 

Wenn ein Sachbearbeiter einen Vorgang aufruft, bei dem im Abschluss die nachträglich hinzugefügte Position nur eine WARNUNG ausgibt, (und somit keine Änderung / Speicherung im Auftrag erfolgt), ist diese Position zwingend über den Button Start Proz abzuschließen. 

Ist die Abschluss-Logik aktiviert, kann kein Produktionsauftrag generiert werden, wenn der Verkaufsauftrag noch nicht freigegeben wurde. 

Für einen Kunden mit der Zahlungsbedingung Vorkasse wird der Auftrag anschließend auf Liefersperre gesetzt! 

Siehe Kapitel Vorkasse . 

Es erfolgt keine automatische Gesamtsperre. 

Um auch nach Freigabe eines Vorgangs (Button Abschluss ) Positionen hinzuzufügen, steht Ihnen der Button Start Proz. zur Verfügung. Für neu hinzugekommene Positionen wird eine neue Belegkette (Verbundprozess) generiert. Dies kann mehrmals geschehen, solange der Auftrag nicht vollständig fakturiert ist. 

Außerdem werden auf der Registerkarte Abschluss Warnungen für neue Positionen angezeigt, die noch nicht in einem Verbundprozess sind. So hat der Benutzer die Möglichkeit entsprechend zu agieren. 

Wird eine Position nach Abschluss des Vorgangs gelöscht, so erhält diese Position den Status 6, anstatt komplett gelöscht zu werden. Auf diese Weise ist nachvollziehbar, falls nachträglich eine Änderung vorgenommen wurde. 

Im Zusammenhang mit der Tourenplanung gilt folgende Prüfung in der Abschluss-Logik des Auftrags : 

Ist ein Fixtermin im Auftragskopf gesetzt, wird dieses gegen alle Lieferdatums-Angaben aller Auftragspositionen geprüft und wenn mindestens ein Datum abweicht, so ergibt dies folgende Warnmeldung: 

"Abweichung zwischen Lieferdatum Position xxx und Fixtermin - bitte prüfen." 

Für nachträglich generierte Positionen in Verbundaufträgen in eNVenta läuft der Lieferscheindruck durch. Nach Eingang der Rechnung im Handelshaus werden entsprechende manuelle Prozesse durchgeführt, um die zusätzlichen Kosten zu verarbeiten.
