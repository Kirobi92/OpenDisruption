# Event Management 
eNVenta bietet ein sogenanntes Event Management an. Hier können sogenannte Aktionen per E-Mail oder Wiedervorlage an die entsprechenden Sachbearbeiter gesendet werden, ohne dass in eNVenta etwas zu programmieren ist. 

Gibt es ein entsprechendes Event in eNVenta , so löst dieses eine entsprechende Aktion aus. 

Begriffsdefinition 
Ereignis, Event: Auslößendes Element/Ursache 
Aktion: Ausgelöstes Element/Wirkung 

Großer Vorteil: Es wird kein Job benötigt, welcher laufend Objekte/Vorgänge überwacht - die Aktion erfolgt sofort bei Eintritt des Ereignisses. 

Dialoge zum Event Management werden im Navigator über System >> Administration >> Event Management aufgerufen. Dieses stellt eine Aktionsdefinition, ein Aktionsprotokoll und die Ereignisdefinition zur Verfügung. 

Events können auf unterschiedlichen Ebenen definiert werden. Für jedes Event wird dann geprüft, ob dafür Meldungen definiert sind. Diese Meldungen werden dann an Empfänger über den definierten Weg versendet. 

Beispiele: 
- Ein Wareneingang ist nicht vollständig, der zuständige Mitarbeiter muss unverzüglich darüber informiert werden. - Ein Warenkorb konnte nicht komplett freigegeben werden. Ursache muss geklärt werden. - Automatische Wiedervorlage zur Pflege der Artikelbilder kann an den zuständigen Sachbearbeiter gesendet werden, wenn ein neuer Webshop-Artikel angelegt wird bzw. ein vorhandener Artikel zum Shop-Artikel wird. 
Diese Ereignisse (Events) lösen automatisch bestimmte Aktionen aus. 

Events oder Ereignisse können auch im Gateway Adapter bzw. direkt im Event Management angelegt werden. 

Wenn Sie mit dem Event Management arbeiten, legen Sie zuerst das Ereignis an und anschließend die Aktion. 

Zur Anlage eines neuen Ereignisses gehen Sie wie folgt vor: 
Geben Sie dem Ereignis einen Namen und anschließend eine Beschreibung, um was es bei dem Ereignis geht. Anschließend wählen sie eine Komponente an und legen fest, wann es ausgelöst werden soll und zu guter Letzt aktivieren Sie es. 

Zur Anlage einer neuen Aktion gehen Sie wie folgt vor: 
Geben Sie einen Aktionsnamen und eine Beschreibung der Aktion an, ordnen die Protokollierung und das auslösende Event an. 

Beispiel für eine Aktion 
Es wird eine Benachrichtigungs-Email an den Kunden versandt, sobald der Lieferschein gedruckt wird (an die im Kundenstamm hinterlegte Email-Adresse). 

Anschließend nehmen Sie Einstellungen für eine Wiedervorlage oder eine E-Mail vor. Dafür stehen die entsprechenden Registerkarten zur Verfügung.

### CallStack integrieren 
Die Änderung der Inhalte von Properties ist eine Kernfunktion von eNVenta . Um zu erkennen, welche Programmstelle die jeweilige Änderung am einzelnen Property auslöst und um die Ursache für eine Datenänderung zu erkennen, gibt es die Möglichkeit den CallStack in die Dokumentation aufzunehmen. (Unter einem CallStack versteht man in der Softwaretechnik und Informatik einen besonders genutzten Stapelspeicher. Dieser enthält den Zustand, der zur Laufzeit eines Programms gerade aufgerufenen Unterprogramme.) 

Wenn das Objekt oSystemBase mit dem Indiv-Property „sCallStack“ im Event Management genutzt wird, so wird hier der CallStack entsprechend ausgegeben. 

Beispiel: 
Definition eines Events auf dem Artikelstamm. Bei Änderung des Wertes LEK wird der CallStack via E-Mail versendet. Zur Auslösung der Änderung geben Sie eine Rechnungsprüfung für den Artikel mit abweichendem LEK frei. Siehe auch:
