# Prioritätenreihenfolge in der EK Rabattverwaltung 
Bei jeder Position einer Bestellung erfolgt eine komplette Konditionierung anhand der ermittelten Daten. Hierfür werden neben den Daten der Bestellung (Lieferant, Artikel,…) auch die Daten eines evtl. hinterlegten Auftrags berücksichtigt. Damit die Konditionierung einen eindeutigen Datensatz für die Preisfindung finden kann, erfolgt die Suche über eine Preisfindungshierarchie. Hierbei wird immer die erste zutreffende Kondition ermittelt und mit dieser wird dann der Preis für die Bestellposition ermittelt. 

Die Hierarchie wird hierbei immer von oben kommend durchlaufen und sucht nach der speziellsten Kondition für die Bestellposition. 

Hierfür gibt es in eNVenta ERP die Regel: 
Je spezieller die Kondition, desto höher ist die Priorität. 

Dabei ist die Pyramide in mehrere Bereiche unterteilt: 

Kundenbezogene Konditionen 
Artikelkonditionen 
Einkaufsgruppen-Konditionen 
Konditionspyramide im EK Rabatt 

Im Kundenbereich werden alle Konditionen hinterlegt, welche genutzt werden, falls es sich um eine auftragsbezogene Bestellung handelt. Nachdem eine solche Kondition gefunden wurde, wird die Konditionssuche abgebrochen und der gefundene Datensatz wird für die Berechnung der Kondition übernommen. 

Im Artikelbereich werden alle Konditionen hinterlegt, welche nur für den Lieferanten und den Artikel gelten. In diesem Bereich können auch Konditionen für die verschiedenen Einkaufseinheiten des Artikels hinterlegt werden, z.B. Paletten-Konditionen und Stückkonditionen für einen Artikel. 

Im Bereich der Einkaufsgruppen können Konditionen für mehrere Artikel bzw. Artikelgruppen zusammengefasst werden. 

Wird keine Kondition für einen Artikel gefunden, so wird die Kondition direkt auf den Lieferanten übernommen. 

Wurde in der Konditionspyramide keine Kondition gefunden, so wird automatisch der letzte Einkaufspreis in die neue Bestellposition übernommen. 

### Kriterien der Konditionsarten 
Bei der Hinterlegung einer Kondition ist die Angabe eines Lieferanten immer eine Pflichtangabe. Es können keine Konditionen ohne Lieferantenbezug angegeben werden. Daneben kann eine Kundennummer hinterlegt werden. Diese wird nur dann gezogen, wenn es sich um eine auftragsbezogene Bestellung handelt. 

Über die Artikelnummer kann eine Kondition für einen Artikel angelegt werden. 

Über die Einkaufart können Konditionen für verschiedene Einkaufsprozesse hinterlegt werden, z.B. Unterschied für Streckenbestellung und Lagerbestellung. 

In der Combobox Einheit EK stehen alle Einheiten für den angegebenen Artikel zur Auswahl zur Verfügung. Diese Einheiten müssen dem Artikel zuvor im Artikelstamm hinterlegt werden. 

Über die Combobox Rabattgruppe kann eine Gruppe von Artikeln ausgewählt werden. Diese Artikel müssen zuvor über den Button Rab Gruppe (im Dialog Rabatt Einkauf ) angelegt werden. 

Weitere wichtige Regelungen: 

Enthält die höchste vorhandene Kondition keinen Basispreis, wird bei Prozentangaben in der Mengenstaffel immer der LEK aus dem Artikelsatz zugrundegelegt. 

Eventuell vorhandene Konditionen niedrigerer Priorität werden in keiner Weise berücksichtigt, selbst wenn sie günstigere Preise oder eine geeignetere Mengenstaffel enthalten! 

Die ermittelten Preise und Rabatte werden vom Programm automatisch in die entsprechenden Felder der Bestellposition eingetragen. Sie sind gewissermaßen nur ein Vorschlag des Systems. Der Benutzer hat natürlich die Möglichkeit, diese Rabatte und Preise manuell zu ändern. Somit bleibt dem Anwender immer die höchste Priorität vorbehalten.
