# X-Mapping Editor Registerkarte Mapping 
X-Mapping Editor Registerkarte Mapping 

Auf dieser Registerkarte nehmen Sie Einstellungen für das Mapping vor. 

Mit den beiden Pfeil-Buttons neben der gemappten Eigenschaft / dem gemappten X-Knoten, kann die aktuell markierte Eigenschaft / X-Knoten als gemappt gesetzt werden. Auf diese Weise muss ein Mapping nicht gelöscht und neu eingefügt werden, falls Sie sich z.B. in der Struktur verklickt haben. 

Das ganze geht auch per Drag&Drop, wenn Sie die Eigenschaft bzw. den X-Knoten auf das entsprechende Mapping im mittleren Bereich ziehen. 
### Feld Eigenschaft 
Hier wird die zugeordnete Eigenschaft des eNVenta -Objekts bzw. die Property-Bezeichnung angezeigt. 

Tooltip: 

Der Pfad zur gemappten Komponenten-Eigenschaft wird angegeben. 

### Button Gemappte Eigenschaft ersetzen 

Mit diesem Button wird die gemappte Eigenschaft auf die derzeit in der linken Datenstruktur ausgewählte Eigenschaft gesetzt. 

Das ganze geht auch per Drag&Drop, wenn Sie die Eigenschaft bzw. den X-Knoten auf das entsprechende Mapping im mittleren Bereich ziehen. 

Tooltip: 

Die gemappte Eigenschaft wird auf die derzeit in der linken Datenstruktur ausgewählte Eigenschaft gesetzt. 

### Feld X Path 
Hier wird der X-Pfad angezeigt. Die Anzeige erfolgt als XPath-Ausdruck. 

### Button Gemappten X-Knoten ersetzen 

Mit diesem Button wird der gemappte X-Knoten auf den derzeit in der rechten Datenstruktur ausgewählten Knoten gesetzt. 

Das ganze geht auch per Drag&Drop, wenn Sie die Eigenschaft bzw. den X-Knoten auf das entsprechende Mapping im mittleren Bereich ziehen. 

Tooltip: 

Der gemappte X-Knoten wird auf den derzeit in der rechten Datenstruktur ausgewählten Knoten gesetzt. 

### Combobox Schleifentyp 
Der Schleifentyp gibt an, wie sich das Mapping beim Abarbeiten einer Schleife (Collection oder Liste von XML-Elementen) verhalten soll: 

- "Vorhandene Ziel-Elemente nutzen, neue Elemente wenn nötig anlegen" - Sollten in der Ziel-Liste bereits Elemente vorhanden sein, erfolgt das Mapping auf diese. - Fehlen Elemente, so werden neue für das Mapping erzeugt. - "Immer neue Ziel-Elemente anlegen" - Es werden immer neue Elemente für das Mapping erzeugt, ungeachtet ob bereits welche existieren. - "Nur vorhandene Ziel-Elemente mappen" - Es werden nur die Elemente gemappt, die auch in der Ziel-Liste vorhanden sind. - Elemente, die in der Quell-Liste vorhanden sind, aber nicht in der Ziel-Liste werden ignoriert.

### Combobox Map-Richtung 
Hier geben Sie die Richtung an: 

Gilt dieses Mapping nur für das Mapping von der Komponente zum XML, nur vom XML zur Komponente oder sowohl als auch? 

Da zumeist pro Richtung ein eigenes Mapping angelegt wird, kann die Einstellung auf dem Standard "Beide" bleiben.

### Combobox Wertzugriff 
Legt fest, unter welchen Bedingungen die Zieleigenschaft durch das Mapping verändert werden darf. Diese Eigenschaft muss immer dann gesondert gesetzt werden, wenn eine Eigenschaft mehrmals zugeordnet ist. Beispiel: die ID der Datei wird als EAN, GTIN und Artikelnummer in der eNVenta -Objektstruktur zugeordnet und der erste sinnvolle Treffer soll genutzt werden. 

Die Optionen sind aufsteigend sortiert, d.h. dass jede Option alle vorherigen Optionen mit einschließt (implizit). 

'Setzen' 
Der zu setzende Wert wird nur dann übernommen, wenn der bisherige Wert noch leer oder null ist. 

'Ändern' 
- Der zu setzende Wert wird nur dann übernommen, wenn dieser nicht leer oder null ist. - Ein bestehender Wert auf der Zieleigenschaft wird ggf. überschrieben aber niemals auf null gesetzt. - (Schließt die Option 'Setzen' ein) 
'Löschen' 
- Der zu setzende Wert wird immer übernommen. Ein bestehender Wert auf der Zieleigenschaft wird ggf. überschrieben. - (Schließt die Optionen 'Setzen' und 'Ändern' ein. Auch Löschen / Leer ist erlaubt.)

### Combobox Fehlermodus 
Wie soll es sich auf das gesamte Mapping auswirken, wenn dieses Mapping fehlschlägt? Zur Selektion stehen: 

Information 

- Im Protokoll des Mapping erscheint eine entsprechende Meldung als Information. - Das Mapping gilt als erfolgreich durchgeführt mit Informationen. 
Warnung 

- Im Protokoll des Mapping erscheint eine entsprechende Meldung als Warnung. - Das Mapping gilt trotzdem als erfolgreich durchgeführt - allerdings mit Warnungen. 
Fehler - Im Protokoll des Mapping erscheint eine entsprechende Meldung als Fehler. - Das Mapping gilt nicht als erfolgreich durchgeführt.

### Combobox Culture 
Hier wählen Sie die Culture. Sie stellt Informationen über eine bestimmte Kultur bereit (die bei der nicht verwalteten Codeentwicklung als locale bezeichnet wird). 

Zu diesen Informationen gehören der Name der Kultur, das Schriftsystem, der verwendete Kalender sowie die Formatierung für Datumsangaben und sortierte Zeichenfolgen. 

Die zugeordnete Culture für den Mapping-Knoten übersteuert hierbei die Culture des Mappings im Dialog Grundeinstellungen . 

Siehe Kapitel Grundeinstellungen für das Mapping .

### Feld Format 
Im Feld Format kann ein gezieltes Datenformat angegeben werden, z.B. beim Export immer auf 3 Nachkommastellen auffüllen oder wie bei Import das Datum zu interpretieren ist. Hier muss nicht zwingend eine Eingabe erfolgen. Ist das Feld leer, wird die Einstellung der jeweiligen Culture für die Interpretation der Inhalte übernommen. 

Datum 

z.B.: dd.MM.yyyy HH:mm:ss --> 23.11.2019 14:20:45 

https://msdn.microsoft.com/de-de/library/8kb3ddd4 

Dezimalwert 

z.B.: #,0.000 --> 1.000,000 

Für Dezimalwerte gilt das feste Format nur für den Export. 

Beim Import wird nur die Spracheinstellung berücksichtigt, die wiederum mehrere mögliche Formatdefinitionen unterstützt.

### Feld Typ 
Hier wird der Datentyp des gemappten Felds angezeigt. Je nachdem, welcher Datentyp vorhanden ist, stehen unterschiedliche Möglichkeiten für Format, Transformation etc. zur Verfügung. Der Datentyp dient nur zur Information und kann nicht im X Mapping verändert werden. 

### Checkbox Mapping nur, wenn Quellwert vorhanden 
Wenn Sie diese Checkbox aktivieren, definieren Sie, dass ein Mapping nur stattfindet, wenn der Quellwert vorhanden ist. 

Ist kein Wert vorhanden (null oder leer) so erfolgt kein Mapping. 

Verhalten ist dann so, als gäbe es dieses Mapping nicht.
