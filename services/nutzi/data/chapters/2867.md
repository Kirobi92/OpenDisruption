# Ressource Registerkarte Synchronisation 
Ressource Registerkarte Synchronisation 

Auf dieser Registerkarte ist es möglich die Reservierungen einer Ressource (Maschinen, Personen, Werkzeug,…) mit einem Outlook-Kalender zu steuern. 

Durch die Angabe des Kalender-Alias kann gezielt ein Mitarbeiter einer Ressource zugewiesen werden. 

Geben Sie hierbei immer die Bezeichnung des Kontos auf dem Exchange-Server an – es werden immer nur genaue Übereinstimmungen für den Abgleich herangezogen. 

Der Abgleich der Ressourcen wird global aus der Wiedervorlage, Menü Outlook >> Ressourcen / Kalender abgleichen aufgerufen. 

Hierbei werden grundsätzlich alle Ressourcen abgeglichen, welche einen Kalender-Alias hinterlegt haben. Dieser Vorgang kann dementsprechend einige Momente dauern. 

Beim Speichern von Reservierungen in eNVenta ERP werden mehrtägige Termine entsprechend den Pausen gestückelt, sodass es nicht einen mehrtägigen Termin, sondern mehrere eintägige Termine gibt. 

Liegt ein Termin über mehrere Tage verteilt als Reservierung vor, so ist dies in Outlook ein zusammenhängender Termin. Wird der Termin in Outlook geändert, so wird dies korrekt in eNVenta ERP aktualisiert. Bei einem nicht-manuellen Termin (z.B. Serviceeinsatz) wird allerdings immer der Eintrag von eNVenta ERP nach Outlook aktualisiert. 

Es werden immer alle Termine von eNVenta ERP und Outlook in beiden Richtungen aktualisiert. Damit die Termine aus Outlook in eNVenta ERP übernommen werden können, müssen diese eine Kategorie mit der Bezeichnung „eNVenta ERP“ beinhalten, z.B. „eNVenta ERP – Serviceeinsatz“. Die Kategorien werden im Normalfall automatisch beim ersten Abgleich erstellt, falls noch nicht vorhanden. Wenn dies nicht funktioniert, wenden Sie sich bitte an Ihren Administrator. 

Ist ein Termin aus eNVenta ERP in Outlook übernommen worden, so hat eNVenta ERP immer die höhere Priorität, solange es sich nicht um einen manuell angelegten Termin handelt. Ändern Sie z.B. die Zeiten eines Serviceeinsatzes innerhalb von Outlook, wird dies nicht in eNVenta ERP übernommen; Outlook wird beim nächsten Abgleich wieder an eNVenta ERP angepasst. 

Bei manuellen Terminen, z.B. Überstunden-Ausgleich, gewinnt immer die letzte Änderung, unabhängig ob in Outlook oder eNVenta ERP . 

Werden private Termine aus Outlook in eNVenta ERP übernommen, steht immer nur die Bezeichnung „Privater Termin“ in der Reservierung. 
### Feld Kalender-Alias 
Hier geben Sie den Kalender-Alias ein. Dadurch kann gezielt ein Mitarbeiter einer Ressource zugewiesen werden. 

### Checkbox Keine manuellen Reservierungen synchronisieren 
Wenn Sie diese Checkbox aktivieren, werden keine Reservierungen in Outlook übernommen, welche manuell in eNVenta ERP angelegt wurden, also nicht auf einem Serviceeinsatz, Betriebsauftrag,… beruhen. 

Tooltip: 

Keine manuell angelegten Reservierungen synchronisieren. 

### Checkbox Keine Kontakte exportieren 
Wenn Sie diese Checkbox aktivieren, werden keine Kontaktdaten aus dem Serviceeinsatz, Projekt,… nach Outlook exportiert. Haben Sie die Checkbox nicht aktiviert, so werden immer die Kontaktdaten zu einer Reservierung an Outlook übertragen, damit diese bei einem Offline-Betrieb, z.B. im Außendienst immer verfügbar sind.
