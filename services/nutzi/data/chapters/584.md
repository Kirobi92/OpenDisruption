# Parameter Ressourcenabgleich 
Parameter Ressourcenabgleich 

Dieser Dialog öffnet sich über den Button Ressourcenabgleich im Menü Parameter Office . 

Wenn Sie diesen Dialog öffnen, werden zunächst die Benutzerdaten angegeben, die für den Ressourcenabgleich verwendet werden. Dabei muss es sich um einen User mit Exchange-Admin-Rechten handeln, da er sonst nicht auf fremde Kalender oder Kontaktordner zugreifen kann. Außerdem muss noch das Protokoll (z.B. „http“), der Name des Exchange-Servers (z.B. „nvdc“), die IIS-Website des Exchange-Servers (z.B. „exchange“), sowie die URL-Namen des Kalenders (z.B. „Kalender“ bei deutschen Exchange-Servern, „Calendar“ bei englischen) und der Kontakte angegeben werden. 

Wenn alle Daten eingegeben wurden, wird unten eine URL-Vorschau angezeigt. 

Der Ressourcenabgleich kann über die Wiedervorlage ( eNVenta ERP Registerkarte Wiedervorlage ) gestartet werden ( Menü Outlook >> Ressourcen/Kalender abgleichen ). 

Damit der Ressourcen-Abgleich genutzt werden kann, muss OWA (Outlook Web Access) auf dem Exchange-Server eingerichtet werden. Über die WebDAV-Erweiterung (Erweiterung des HTTP-Protokolls) können alle Kalender angesprochen werden, indem man WebDAV-Requests an den Exchange-Server schickt und die darauffolgenden WebDAV-Responses auswertet. 

Damit ein Abgleich der Ressourcen-Reservierungen mit den Outlook-Kalendern der Benutzer stattfinden kann, muss die Verbindung zum Exchange-Server eingerichtet werden. 

Auf der linken Seite geben Sie den Exchange-Nutzer an, welcher die Verbindung verwaltet. Es ist hierbei zu empfehlen, einen separaten User anzulegen, welcher über die Rechte „Exchange-Admin“ verfügt, da er sonst nicht auf fremde Kalender oder Kontaktordner zugreifen kann. 

Daneben müssen Sie noch das Übertragungsprotokoll angeben, mit welchem die Daten ausgetauscht werden, im Normalfall ist dies „http“. 

Als Server geben Sie den Namen des Exchange-Servers an. 

In der IIS-Website geben Sie die Root auf dem Exchange-Server an. 

Die TLS-Version ist abhängig von den Einstellungen des Applikationsservers. 

Im Rahmen der Produktpflege hat Microsoft die Basic Authentication für die EWS Exchange Anbindung abgekündigt. Hierbei ist nicht die komplette EWS-API abgekündigt, sondern deren Basic Authentication. Aufgrund dessen wurde die Schnittstelle auf OAuth umgebaut. Im Rahmen dessen müssen die entsprechenden neuen Verbindungsdaten im Dialog hinterlegt werden. 

Nachfolgend eine Liste der unterstützten Exchange-Server Versionen: 

- Exchange Server 2013 / 2016 - Exchange Server 2019 - Exchange Server Online 
Zusätzlich zur Basic Authentication wird auch die OAuth2 Authentication für den Mail-Versand über SMTP unterstützt. 

Wenn Sie nichts auswählen, ist das Verhalten analog zu Basic Authentication. 

Wenn Sie oAuth für Exchange Online auswählen und speichern, werden zusätzlich die Einträge für Anwendung-ID und Verzeichnis-ID angezeigt.

### Combobox Exch.-Server 
Hier wählen Sie den Namen des Exchange-Servers aus. Diese Angabe ist notwendig, damit die korrekte Anbindung von eNVenta ERP genutzt werden kann. 

Nachfolgend eine Liste der unterstützten Exchange-Server Versionen: 

- Exchange Server 2013 / 2016 - Exchange Server 2019 - Exchange Server Online 
Wird als Exchange Server der 2013er oder höher gewählt, werden die Eingabefelder für die Kontakt- und Kalender-URL ausgeblendet, da diese Info dann nicht mehr benötigt wird. Es müssen weiterhin das Protokoll (typischerweise https ), der Exchange-Server, die IIS-Website (typischerweise exchange.asmx) und natürlich die Zugangsdaten für den Benutzer (mit Zugriff auf alle Kalender und Kontakte) angegeben werden.

### Checkbox Kommunikation über Graph 
Wenn diese Checkbox gesetzt ist, wird mit Exchange Online über die Graph API, statt über EWS kommuniziert. 

Achtung: Vor der nächsten Synchronisation (Manuell und über Jobserver, ggf. Jobserver deaktivieren) müssen die bestehenden Reservierungen umgestellt werden. Für die Erkennung der Objekte über Graph benötigen diese eine entsprechende ID. Um dies zu erreichen steht der Button Migrieren zur Verfügung. Bei Klick auf den Button erfolgt eine Umstellung aller bestehenden Reservierungen über alle Outlook-Kalender hinweg. Der Vorgang kann entsprechend mehrere Minuten dauern und sollte außerhalb der Arbeitszeiten erfolgen. 

Es erfolgen folgende Abgleichungen, wie auch über EWS: 

- Abgleich von neutralen Terminen in beiden Richtungen. - Abgleich von Besuchsterminen. - Abgleich von Projektaufgaben und Projektterminen. 

Tooltip: 

Ist diese Checkbox gesetzt, wird mit Exchange Online über die Graph API, statt über EWS kommuniziert. 
Um dies zu nutzen, müssen vor erstmaliger Verwendung alle eNVenta-Reservierungen migriert werden (siehe Button "Migrieren").
