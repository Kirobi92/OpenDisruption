# GAP Position ändern Registerkarte Anschrift 
Position Registerkarte Anschrift 

Diese Registerkarte steht zur Verfügung, für den Fall dass einzelne Positionen eines Auftrags an verschiedene Adressen gesendet werden sollen. Hier können Sie unterschiedliche Lieferadressen auf Positionsebene definieren. 
### Combobox Lieferanschrift 
Hier wird die Nummer der Lieferadresse angegeben. 

Bei einem Adress-Splitt werden Abrufpositionen nicht auf Status 1 zurückgesetzt, wenn der verfügbare Bestand nicht ausreicht. Der Status 2 wird beibehalten, wenn die Quellposition bereits den Status 2 hat. Die Freigabe der Position ist meist zum Erfassungszeitpunkt begründet gewesen und daher muss dieser Status beibehalten werden. Meist ist dieser auch bereits mit einer Liefersperre versehen und daher bleibt die Ware fest für diese Abrufposition reserviert. Bei Status 1 wäre diese Position wieder für andere Aufträge frei, was dem Gedanken des Abrufauftrags widerspricht.

### Feld Firma 1 
Hier wird die Firmenadresse angegeben. 

Für die Eingabe der Adresse stehen bis zu 4 Zeilen zur Verfügung (Firma 1 - 4). 

Die Eingabe der Firma 1 ist Pflicht! 

Der Firmenname wird automatisch eingelesen, wenn Sie eine AKL-Nummer eingeben.

### Feld GEO-Koordinaten 
Hier werden geografische Daten zur besseren Orientierung angegeben. Damit lassen sich Besuche o.ä. besser planen und Sie können die Karte zur besseren Orientierung und Festlegung einer Besuchs-Reihenfolge nutzen. 

Das Format der GEO-Koordinaten lautet: 
XX.yyyyyy,AA.bbbbbb 

Beispiel für Stockach, Nissen&Velten: 
47.848029,09.005334

### Feld Kunde 
Hier wird die Kundennummer angegeben, unter welcher der Kunde in der Warenwirtschaft geführt wird. 

Bei der Neuanlage wird das Feld Kunde automatisch aus dem zugehörigen Nummernkreis unter Berücksichtigung des Parameters Kunde = Konto aus dem Dialog Parameter Allgemein gefüllt. Die Nummer kann alternativ auch manuell vergeben werden.

### Feld USt-IdNr. 
Hier wird die Umsatzsteuer-Identifikationsnummer des Kunden angegeben. 

Ist die Funktion System >> Parameter Allgemein >>UST-ID via WS prüfen aktiviert, so wird die Umsatzsteuer-Identifikationsnummer bei der Eingabe in den Dialogen Kunde, Lieferant, Angebot, Auftrag, Gutschrift, Bestellung etc. per Webservice geprüft. 

Achtung: Diese Prüfung ist rechtlich nicht bindend. Eine rechtlich bindende Prüfung der Umsatzsteuer-Identifikationsnummer kann nur schriftlich bei der zuständigen Behörde erfolgen.

### Feld MwSt.-Kz 
Dieses Feld legt die Steuerkategorie fest: 

- Inland - EU - Ausland 
In Abhängigkeit dieses Kennzeichens werden bei den Vorgängen (Angebot, Auftrag, Gutschrift) der entsprechende Steuerschlüssel und das zugehörige Erlöskonto voreingestellt. In der Buchhaltung wird der richtige Steuerschlüssel ermittelt.
