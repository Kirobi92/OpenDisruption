# Kunde Registerkarte Warenwirtschaft 
Kunde Registerkarte Warenwirtschaft 

Auf dieser Registerkarte werden zahlreiche Informationen zum Kunden hinterlegt, die für den Geschäftsverkehr im Allgemeinen wichtig sind, wie z.B. Zahlungsbedingung. 

ZUGFeRD Version 2.1.1 / XRechnung 

Mit Hilfe der ZUGFeRD-Version 2.1.1 können X-rechnungskonforme elektronische Rechnungen erzeugt werden. Diese elektronischen Rechnungen können an die entsprechenden Regierungsbehörden (B2G) übermittelt werden. Ab dem 27.11.2020 besteht eine Verpflichtung Rechnungen nur noch als elektronische Rechnungen an die Bundesverwaltung zu übermitteln. Eine Ausnahme besteht nur, wenn der Rechnungswert 1000,00 € netto nicht übersteigt. 

Mit Hilfe der ZUGFeRD Version 2.1.1 vom 01.07.2020 wird die zu Grunde liegende Norm EN-16931, entspricht der X-Rechnung, entsprechend erfüllt. Diese entsprechend erzeugten Mails können per Mail an den Rechnungssteller versendet werden bzw. im Verwaltungsportal des Bundes hochgeladen werden. 

Für die Nutzung der ZUGFeRD-Version 2.1.1 muss die entsprechende XML-Vorlage eingelesen sowie die Leitweg-ID im Kundenstamm hinterlegt werden. Die Leitweg-ID dient dem zentralen Empfänger der Rechnung zur Identifikation des Rechnungsempfängers, analog einer ILN. Dies ist durch Auswahl der Version „ZUGFeRD 2.1.1 / XRechnung“ möglich. Ebenso muss das zugehörige XML-Mapping hinterlegt werden – eine entsprechende Vorlage wird mitgeliefert. 

Seit dem 01.01.21 ist mindestens die XRechnungs-Version 2.0.0 relevant. Ab dem 01.07.21 ist die XRechnungs-Version 2.0.1 die Mindestversion – diese ist implementiert und abwärtskompatibel zur XRechnungs-Version 2.0. Ab dem 01.02.2022 ist die neu hinzugefügte Version 2.1.1 gültig und ersetzt die älteren Versionen. 

eNVenta unterstützt auch die UBL Sprachnotation. Diese Sprachnotation kann alternativ verwendet werden. 

Die Ressource hat in den Vorlagen für das XMapping den Namen N-VOIC_Export_XRechnung_UBL.xml 

Achtung: Bei Verwendung der UBL Sprachnotation wird nur der Export von Rechnungen unterstützt, ein Export von Gutschriften ist nicht möglich. 

Dieses kann auch unabhängig von ZUGFeRD, z.B. für den EDI-Export verwendet werden. 

Die Auslieferung erfolgt als Ressource zum Einlesen im X-Mapping Editor . Nach dem Einlesen der neuen Vorlage müssen die Stammdaten im Kundenstamm auf die neue Version aktualisiert werden.

### Feld Handlingskosten 
Hier geben Sie an, mit welchem Faktor der Kunde am Risiko einer auftragsbezogenen Bestellung beteiligt wird. Der Anteil wird bei der Berechnung des „erweiterten“ Kreditlimits für Auftragspositionen im Status 1 und auftragsbezogenen Lagerbestellung fällig. 

Der angegebene Prozentsatz wird mit dem EK-Umsatz multipliziert. 

Handlingskosten werden bei der Kreditlimitprüfung für Auftragspositionen im Status 1 berücksichtigt! 

Diese Handlingskosten können bereits im Dialog Parameter Allgemein auf der Registerkarte Wawi angelegt werden, damit die Pflege nicht pro Kunde / Lieferant erfolgen muss. 

Siehe Kapitel Parameter Allgemein .

### Checkbox Kein Sonderpreis 
Wenn Sie diese Checkbox aktivieren, wird für diesen Kunden kein Sonderpreis gezogen. Dies betrifft sowohl die Sonderpreisverwaltung als auch die Sonderpreise in der Konditionsstaffel und dem Artikelstamm. 

Der Sonderpreis kann auch durch die Auftragsart unterbunden werden. Deshalb muss bei der Auftragserfassung geprüft werden, ob der Parameter evtl. im Kundenstamm gesetzt werden muss. 

Tooltip: 

Keine Berücksichtigung von Sonderpreisen. 

### Checkbox Keine Schnittkosten 
Wenn Sie diese Checkbox aktivieren, werden standardmäßig keine Schnittkosten berechnet. 

Sobald ein Schnitt notwendig ist (es wird eine Teilmenge einer Charge verwendet) wird ein Schnitt berechnet. 

Für jeden Schnitt wird dem Kunden eine zusätzliche Position in den Auftrag generiert mit dem Artikel 'Schnittkosten'. 

Über die Konfektionierung im WS-Package kann der Vorgang/Prozess gesteuert werden. 

Tooltip: 

Standardmäßig werden keine Schnittkosten berechnet.

### Checkbox Strikte Pos-Prüfung 
Abhängig vom Verkaufsparameter Prüfen, ob Artikel schon beauftragt , wird in der Auftragserfassung geprüft, ob bereits eine offene, noch nicht gelieferte Position zu diesem Kunden und Artikel existiert. Ist dies der Fall und die strikte Positionsprüfung gesetzt, wird die Erfassung einer neuen Position abgelehnt. 

Siehe Kapitel Parameter Verkauf . 

### Checkbox Kein Porto bei Rückständen 
Ist dieser Parameter gesetzt, werden ab der zweiten Teilfaktura für einen Auftrag kein Porto und keine Verpackung mehr berechnet. Hierzu wird der Portobetrag nach dem ersten Rechnungsdruck auf 0 gesetzt. 

Wenn die Funktion Kein Porto bei Rückständen in den Parameter Verkauf nicht aktiviert ist, dann kann es in den Auftragsarten und im Kunde gesteuert werden. Wenn im Vorgang einer der beiden Parameter zutrifft, so wird kein Porto bei Rückständen berechnet. 

Ist der Verkaufsparameter aktiv, dann wird bei Rückständen nie Porto berechnet.

### Feld Leitweg-ID 
Hier muss die Leitweg-ID hinterlegt werden. Die Leitweg-ID ist ein Kennzeichen einer elektronischen Rechnung zur eindeutigen Adressierung von öffentlichen Auftragsgebern in Deutschland. 

Diese wird von der Bundesbehörde mitgeteilt und muss entsprechend auch bei jeder Rechnung mitgegeben werden. 

Neben der Leitweg-ID muss die zugehörige ZUGFeRD-Version angegeben werden. Dies ist durch Auswahl der Version „ZUGFeRD 2.1.1 / XRechnung“ möglich. Ebenso muss das zugehörige XML-Mapping hinterlegt werden – eine entsprechende Vorlage wird mitgeliefert. 

Tooltip: 

Die Leitweg-ID ist ein Kennzeichen einer elektronischen Rechnung zur eindeutigen Adressierung von öffentlichen Auftragsgebern in Deutschland.
