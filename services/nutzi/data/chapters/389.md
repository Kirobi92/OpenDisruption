# Daten Historisieren 
Daten Historisieren 

Für die Historisierung gibt es im Navigator >> System >> Administration >> Daten Historisierung den Eintrag Daten historisieren . Hier können Historisierungsläufe (ähnlich den Mahnläufen) definiert werden. Die Ausführung der Historisierung kann manuell oder per Job erfolgen. Dafür stehen die Buttons und Job starten zur Verfügung. 

Pro Historisierungs-Lauf werden immer die ganzen Tage historisiert. Es wird immer chronologisch rückwärts gearbeitet, d.h. ältere zuerst. Ein Historisierungs-Lauf umfasst die entsprechenden Einstellungen pro Tabelle in der Registerkarte Allgemein . Es wird immer versucht den gesamten Lauf abzuschließen, es gibt allerdings entsprechend definierte Abbruch-Kriterien, um z.B. die Historisierung immer nachts laufen zu lassen. Sollte dies zu lange dauern, kann der Vorgang entsprechend unterbrochen werden. Damit wird das Tagesgeschäft am Morgen nicht belastet. Die Historisierung beginnt immer beim ältesten Datensatz in der Tabelle, welcher noch nicht historisiert ist und prüft dann pro Tag alle Datensätze, welche historisiert werden könnten. Achten Sie daher darauf, ob z.B. ein offenes Angebot von vor 10 Jahren in Ihrem System ist – dieses kann nicht historisiert werden, erzeugt aber eine längere Laufzeit im Vorgang der Historisierung, da immer mit diesem Datum begonnen wird. 

Belege, die historisiert sind, werden im System nur noch dann angezeigt, wenn auf Historisierung umgeschaltet ist.

Dieser Button öffnet die Job Verwaltung. So können Mahnungen auch über den Job-Server abgewickelt werden. So können z.B. alle überfälligen Lieferungen den Lieferanten per Email angemahnt werden.

### Button Suche 

Mit diesem Button starten Sie die Suche. 

Sie können ganz allgemein suchen, indem Sie den Button drücken, gleich nach Öffnen des Dialogs. Oder Sie geben im Kopf des Dialogs die Selektionskriterien vor und drücken dann diesen Button.

Dieser Button leert die Daten in allen Feldern, nicht gespeicherte Änderungen gehen verloren. 

Ist in den Parametern für die Sachbearbeiter die Checkbox 2 Schritt Neu aktiviert, bleiben die Suchkriterien in den Stammdialogen erhalten.

### Button Speichern 

Dieser Button speichert bzw. verarbeitet die Eingaben. Vor einer endgültigen Speicherung prüft das System die Eingabe. Wenn fehlende oder fehlerhafte Daten gefunden werden, erhalten Sie eine Meldung und die Eingabe muss entsprechend korrigiert bzw. vervollständigt werden. 

### Button Löschen

### Button Schließen

### Combobox Definition 
Aus dieser Combobox wählen Sie die entsprechende, bereits gespeicherte Historisierungsdefinition aus. So können bereits eingestellte Settings überarbeitet werden. 

Tooltip: 

Nummer und Bezeichnung der Historisierungsdefinition 

### Feld Beschreibung 
Hier wird eine genaue Beschreibung angegeben. Sie können bis zu 32000 Zeichen angeben (bei Oracle: 2000). 

Tooltip: 

Angabe einer Beschreibung 

### Feld Datum bis 
Im Feld Bis Datum geben Sie ein Datum für die Historisierung ein. Alle Datensätze, die älter sind, werden historisiert. Die jüngeren sind noch im Produktiveinsatz. Dieses Datum ist eine Über-Definition der Angaben pro Tabelle, wird aber ignoriert, wenn das Datum pro Tabelle älter ist als das angegebene Bis-Datum. 

Tooltip: 

Bis zu diesem Datum ist definiert, dass jüngere Datensätze im Produktivteil sein sollten, ältere in der Historie. 

### Feld Dauer Max. 
In dieses Feld geben Sie einen Wert ein, wie lange die Historisierung dauern darf / soll. In der Combobox daneben können Sie Minuten, Stunden und Tag(e) auswählen. Ist die entsprechende Dauer erreicht wird der darauffolgende Tag nicht mehr historisiert. 

Tooltip: 

Maximale Dauer als Fänger für Langläufer. 

### Combobox Einheit 
Aus dieser Combobox können Sie Minuten, Stunden und Tag(e) auswählen. 

### Feld Anzahl max. 
Im Feld Max Anzahl geben Sie an, wie viele Datensätze maximal historisiert werden sollen. Sobald der angegebene Wert erreicht ist, wird der darauffolgende Tag nicht mehr historisiert. 

Tooltip: 

Maximale Anzahl der Datensätze als Fänger für Langläufer. Sobald die Anzahl der historisierten Datensätze die max. Anzahl überschreitet, wird der nächste Tag nicht mehr historisiert.
