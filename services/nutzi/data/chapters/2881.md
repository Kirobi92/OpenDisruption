# Generischer IO Adapter Registerkarte Import 
Generischer IO Adapter Registerkarte Import 

Auf dieser Registerkarte definieren Sie die Einstellungen für den Import der Dateien ins Belegjournal. 

Jeder Import-Eintrag besteht aus Identifier, X-Mappper-Mapping, Prozessobjekt (Logik) und Prozessregel. 

Der Identifier steht für einen Teil des Dateinamens, damit die zu importierende Datei aus dem Importverzeichnis der Konfiguration zugeordnet werden kann. 

Über den X-Mapper erfolgt die Zuordnung der Daten zum eNVenta-Objekt. 

Über die Auswahl des Prozessobjekts und die Prozesseinstellung erfolgt die Zuordnung der Prozessregeln für die Verarbeitung vom Prozessobjekt zum eNVenta -Objekt und damit zum fertigen Import in die entsprechenden Vorgänge innerhalb von eNVenta . 

Import der Kassenbelege 
Die Kassenbelege werden über den Dialog Auszug-/Kassenimport eingelesen. Hierbei kann in den Prozessregeln für den Import bestimmt werden, ob die Kassenimport-Daten der Quelldatei pro Kassenkonto gesplittet werden sollen oder nicht. Je nach Einstellung werden die Daten entsprechend im Dialog Auszug-/Kassenimport angezeigt. 

Pro Import erfolgt immer ein neuer Eintrag in der Registerkarte Kassenübernahme mit dem aktuellen Datum inkl. Uhrzeit 

Import der Ausgangsrechnungen 
Die Ausgangsrechnungen werden in die OP-Übernahme Registerkarte Fakturaübernahme übernommen. Zusätzlich können mittels Mapping die Wareneinsätze in die Schnittstelle der Kostenrechnung übernommen werden. 

Import der Eingangsrechnungen 
Die Eingangsrechnungen werden in die OP-Übernahme Registerkarte Einkaufübernahme übernommen. 

Import in Kostenrechnung 
Der Eigenverbrauch von Waren kann direkt in die Kostenrechnung gebucht werden, hierfür ist es nicht notwendig einen Datensatz in die Finanzbuchhaltung zu importieren.

### Tabellenspalte Kennung 
Die Kennung dient dazu, die unterschiedlichen Typen der Datensätze zu unterscheiden. Dadurch lässt sich je nach Typ des Datensatzes (Satzart) ein eigenes Mapping und ein eigener Prozess ausführen - also eine eigene Import-Konfiguration verwenden. 

Innerhalb eines IO-Adapter Gateway-Kontext muss eine Kennung eindeutig sein - darf also nur einmal vergeben werden. 

Welche Kennungen möglich sind, ist abhängig vom verwendeten Import-Kanal und/oder der angebundenen Datenquelle. 

Wird der generische Datei-Import verwendet, kann die Kennung innerhalb des Dateinamens gesucht und dadurch die Import-Konfiguration, d.h. das X-Mapping und die Prozess-Einstellungen, ermittelt werden. 

Dazu muss der entsprechende Parameter für den Export-Kanal gesetzt werden. 

Wird die Kennung nicht verwendet, um Dateien zu identifizieren, erfolgt die Identifizierung über das X-Mapping. 

Die Kennung kann dafür beliebig vergeben werden - muss aber trotzdem eindeutig bleiben.
