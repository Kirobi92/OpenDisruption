Auf dieser Registerkarte wird dann die Abfrage ausgewählt, die zuvor gespeichert wurde. 
### Button Öffnen 

Dieser Button öffnet das Menü Online Analyse Abfragen , um eine Abfrage zu übernehmen. 

### Feld Nachricht 
In diesem Feld kann der Benutzer eine Nachricht hinterlegen, die dann als Text in der Wiedervorlage steht oder als Nachrichtentext in der E-Mail, falls keine Dokumentvorlage ausgewählt wurde. 

### Feld Dok. Vorlage 
Hier geben Sie die Dokumentvorlage ein. 

### Button Suche 

Mit diesem Button suchen Sie die Dokumentvorlage. 

Siehe Kapitel Dokument Vorlagenstamm . 

### Feld WV-Empfänger 
In diesem Feld werden mit ";" getrennt die eNVenta -Benutzernamen eingetragen, an welche die Wiedervorlage gesendet werden soll. 

Der Versand der Wiedervorlage kann aber nur über den Job-Server erfolgen. Hier wird ein Job definiert, der dann die Abfrage sofort auf der OLAP-Datenbank ausführt, den Report generiert und an die Empfänger versendet. 

### Feld E-Mail-Empfänger 
Alternativ / zusätzlich können in diesem Feld E-Mail Adressen eingetragen werden, welche dann eine E-Mail mit dem Report (PDF) erhalten. 

Der Versand der Email kann aber nur über den Job-Server erfolgen. Hier wird ein Job definiert, der dann die Abfrage sofort auf der OLAP-Datenbank ausführt, den Report generiert und an die Empfänger versendet. 

Um sicherzugehen, dass eine E-Mail an die richtige Adresse gesendet wird, prüft eNVenta beim Speichern oder Ändern einer Mailadresse, ob die Mail-Adresse der „Norm“ entspricht. Falls nicht wird ein Hinweis ausgegeben. Dabei wird das Speichern nicht grundsätzlich verhindert. Auf diese Weise werden z.B. Outlook-Adressbucheinträge wie z.B. „#EW_ERP“ bei Bedarf gespeichert und genutzt. 

Dabei läuft die Prüfung ausschließlich bei Anlage und Änderung einer Mail-Adresse. Werden andere Einträge, z.B. des Kundenstammsatzes geändert, so kommt diese Meldung nicht. 

Werden Stammdaten über das Gateway importiert, so werden u.U. fehlerhafte Mailadressen importiert (ggfs. mit Warnung). Bei einem Import unter Verwendung des Jobservers erfolgt dies ebenfalls in Form einer Warnung (im Jobprotokoll).
