# Parameter Einkauf Registerkarte Intercompany 
Parameter Registerkarte Intercompany 

Auf dieser Registerkarte wird das Originallager der Position für jeden beteiligten Mandanten auf das virtuelle Streckenlager (Combobox Lager Virtuell ) umgestellt. 

Siehe Kapitel Werkzeuge / Intercompany . 

Die Verschlüsselung der Einkaufarten muss einzeln gepflegt werden! 
### Combobox Lager Virtuell 
Hier geben Sie ein virtuelles Lager für Umlagerungen ein zwischen eigenen bzw. IC-Lagern. Dieses Lager muss ein statisches Lager sein und wird für Zwischenbuchungen im Intercompany-Prozess genutzt. Ist der gesamte Prozess Intercompany abgeschlossen, so ist das virtuelle Lager wieder entlastet. 

Mehr zum Thema Intercompany erfahren Sie im Kapitel Intercompany . 

Tooltip: 

Virtuelles Lager für Umlagerungen zwischen eigenen bzw. IC-Lagern 

### Checkbox Automatische Rechnungsprüfung 
Ist dieser Parameter für eine automatische Rechnungsprüfung gesetzt, löst der IC-Adapter eine komplette Verbuchung der Rechnungsprüfung aus. Hierbei wird angenommen, dass die exportierte Rechnung des einen Mandanten immer stimmt und somit eine Freigabe der Rechnung automatisch erfolgen kann. 

Tooltip: 

Ist diese Checkbox für eine automatische Rechnungsprüfung gesetzt, löst der IC-Adapter eine komplette Verbuchung aus. 

### Checkbox EK-Aktualisierung 
Ist dieser Parameter gesetzt, wird automatisch der EK aktualisiert, wenn die Rechnungsprüfung automatisch durchgeführt wird. 

Dieser Parameter kann in der einzelnen Rechnungsprüfung übersteuert werden. 

Siehe Kapitel Rechnungsprüfung . 

Tooltip: 

Ist diese Checkbox gesetzt, wird automatisch der EK aktualisiert, wenn die Rechnungsprüfung automatisch durchgeführt wird. 

### Checkbox Akzeptiere auto. Retoure im IC-Verbund 
Ist dieser Parameter gesetzt, wird im Falle eines IC-Auftrages, welcher retourniert wird, automatisch im Fremdmandanten geprüft, ob dort dieser Parameter gesetzt ist und– wenn gesetzt - ein Umlagerungsvorschlag im Bestellautomaten (im Mandant 2) erzeugt. Lauf 'Auto. IC- Retoure'. 

Tooltip: 

Ist dieser Parameter gesetzt, wird im Falle eines IC-Auftrages, welcher retourniert wird automatisch im Fremdmandanten geprüft, ob DORT dieser Parameter gesetzt ist und – falls ja - ein Umlagerungsvorschlag im Bestellautomaten (im anderen Mandant) erzeugt. Lauf „Auto. IC- Retoure“. 

### Combobox Einkaufsart 
Werden IC-Retouren angelegt, erfolgt dies automatisch mit der hier gewählten Einkaufsart. Es werden jedoch nur Einkaufsarten angezeigt, die 

- die Checkbox EK-Aktualisierung nicht aktiviert - die Checkbox Umlagerung nicht aktiviert - die Checkbox Wareneingang / Lager aktiviert 
haben. 

Ist die Checkbox Akzeptiere auto. Retoure im IC-Verbund nicht gesetzt, ist diese Combobox nicht wählbar (sie ist ausgegraut). 

Bemerkung: Es geht hier um die Einkaufsarten des Mandanten. Die Combobox ist nur aktiv, wenn die Checkbox Akzeptiere auto. Retoure im IC-Verbund gesetzt ist. Die Frage, ob die Mandant1-Retoure an den Mandant2 weiter retourniert werden kann, entscheidet die Checkbox Akzeptiere auto. Retoure im IC-Verbund . Der Zeitpunkt der Frage ist der Abschluss der Wareneingang-Stufe2-Buchung. 

Tooltip: 

Werden IC-Retouren angelegt, erfolgt dies automatisch mit der hier gewählten Einkaufsart. 

### Combobox Anlegen 
Hier stehen folgende Auswahlmöglichkeiten zur Verfügung: 

- Abfragedialog in Retoure (Stufe 2) - Automatisch (Stufe 2) - Deaktiviert 
Standard-Einstellung: Deaktiviert 

Die Einstellung Abfragedialog in Retoure bewirkt beim Verbuchen der Kundenretoure in Stufe 2, dass dem Sachbearbeiter eine Abfrage Dialog angezeigt wird, um situationsbedingt entscheiden zu können, ob eine IC-Rückführung sinnvoll ist oder nicht. 

Die Einstellung Automatisch bewirkt beim Verbuchen der Retoure in Stufe 2, dass automatisch ein Umlagerungseintrag der IC-Position/(en) im anderen Mandanten angelegt wird. Vorausgesetzt, dieser akzeptiert eine IC-Retoure. Wenn nicht, erfolgt eine entsprechende Schlussmeldung nach Abschluss der Wareingang 2-Buchung. 

Die Einstellung Deaktiviert , deaktiviert die Einstellung Autom. IC-Retourenlogik. 

Bemerkung: In der Retoure ist weiterhin eine beliebige Mischung mehrerer Aufträge möglich. 

Meldung bei Einstellung Abfragedialog in Retoure : 

Die erfasste Retoure beinhaltet eine oder mehrere Positionen, deren Bedarf über einen Fremdmandanten gedeckt wurde. Möchten Sie eine IC-Umlagerung / IC-Rücknahme im Ursprungsmandanten beantragen?“ JA / NEIN

### Combobox Lager 
Hier können alle eigenen Lagerorte, die nicht als Streckenlager / Chaoslager gekennzeichnet und nicht IC-verschlüsselt sind, ausgewählt werden. Eine Auswahl ist hier Pflicht. 

Für IC-Retouren kann hier das Standardlager ausgewählt werden. Auf dieses Lager werden IC-Retouren umgebucht. Voraussetzung hierfür ist eine gültige IC-Verschlüsselung des hier gewählten Lagerortes im Fremdmandanten. Damit der Prozess und eine korrekte Umlagerung erfolgen kann, muss hier eine Auswahl getroffen werden. Ist keine Auswahl getroffen worden, kann keine IC-Retoure erfolgen. 

Regeln: Grundvoraussetzung für das Weiterreichen von Retouren sind: 

- In beiden Mandanten MUSS ein gültiges ‚Bevorzugtes IC-Retourenlager‘ definiert sein. Grund dafür ist es für M1, zu verhindern, dass der dortige Bestand fälschlicherweise von neu erfassten Aufträgen ‚angezapft‘ werden könnte und damit einer späteren Rückbuchung an den M2 im Wege stände. Bei M2 landet dann die Ware auf einem genau definierten Ziellager. - Im Mandant2 MUSS eine Rückverschlüsselung auf das konkrete Retourenlager des Mandanten1 definiert sein. - Fehlt eine dieser Voraussetzungen, kann kein Weiterreichen der Retoure stattfinden. - Mandant1: ist hier ein ‚Bevorzugtes IC-Retourenlager‘ definiert, dann muss dieses bei der Retouren-Wareneingang 2-Buchung – ggf. nach Rückfrage als Retourenpositionslager eingespielt werden. - Mandant2: ist auch hier ein ‚Bevorzugtes IC-Retourenlager‘ definiert, dann sollte dieses – anstatt des Positionslagers der UA-Position – als Ziellager für den Bestellvorschlag fungieren. - Lagerverschlüsselung im Mandant2 an der Bestellvorschlagsposition: Ziellager = Positionslager der UA, Quelllager = eigene Lagernummer für das rückverschlüsselte Retourenlager des Mandant1. 
Mit der Hilfe einer neuen Event Komponente kann auf Wunsch über den Dialog Ereignisdefinition festgelegt werden, welche Sachbearbeiter eine Wiedervorlage erhalten, wenn im anderen Mandanten eine IC-Retoure über den Bestellautomaten gelöscht wird. 

Tooltip: 

Für IC-Retouren, kann hier das Standardlager ausgewählt werden. 
Auf dieses Lager werden IC-Retouren umgebucht. 
Voraussetzung hierfür ist eine gültige IC-Verschlüsselung des hier gewählten Lagerortes im Fremdmandanten. 
Wird keine Auswahl getroffen, erfolgt beim Verbuchen der Retoure eine Lagerortermittlung und automatische Umbuchung.
