# GAP Position ändern 
GAP Position 

Über den Button Ändern auf der Registerkarte Allgemein im Dialog Gutschriftsabrechnung öffnet sich der Dialog GAP Position , um eine Position zu ändern. Dieser Dialog entspricht dem Dialog Position aus dem Standard. Über diesen Dialog können die Daten für einen Gegenauftrag erfasst werden. Sollte ein Preis bzw. eine Kondition oder der Positionsbetrag nicht akzeptiert werden, so muss der Preis etc. trotzdem auf die kundenseitige Vorgabe angepasst werden; doch muss auch eine Gegen-Rechnung erstellt werden können. Hierzu müssen der Differenzbetrag und die Menge sowie ein Memo erfasst werden können. Dies erfolgt auf der Registerkarte Korrektur . Diese Registerkarte ist nur aus der Gutschriftsabrechnung heraus sichtbar und umfasst die entsprechenden Felder. 

Mit Freigabe der Gutschriftsabrechnung wird automatisch ein Ausgleichsauftrag (Auftragsart ohne Lieferung; dient nur der Faktura) generiert. Dazu ist zuvor im Dialog Parameter Verkauf die entsprechende Auftragsart auszuwählen. 

Zur besseren Einschätzung ist unterhalb der Eingabefelder die Preishistorie zur Auftragsposition sichtbar. In dieser Tabelle werden die Originalpreise zum Zeitpunkt der Erfassung der Gutschriftsposition angezeigt und evtl. bereits vorgenommene Preisanpassungen in der Gutschriftsposition (siehe Kapitel Preishistorie GAP ). 

Hierbei gibt es 2 Möglichkeiten: 

Die neue Position mit der Überschussmenge erhält den Preis von 0,00, wenn die Gutschrift des Kunden nicht ausgenutzt werden soll und nur die gelieferte Menge berechnet wird. 

Die neue Position wird mit demselben Preis berechnet, allerdings wird diese Position auf ein "Gutschrifts-Ausgleichslager" (in den VK-Parametern hinterlegt) gedreht. Hierzu muss beim Duplizieren die Checkbox Ausgleichslager gesetzt werden. 

Mit dem o.g. Begriff 'Preis' ist immer die Gesamtzahl aller Auftragspositions-Konditionsfelder gemeint. 

Beispiel: 

10 Stück geliefert von Lager A à 10,00 € = 100,00 € 
15 Stück berechnet via Gutschrift à 10,00 € = 150,00 € 

In der Gutschrift erfolgt nun der Splitt der Position wie folgt: 

10 Stück geliefert à 10,00 € = 100,00 € 
5 Stück geliefert à 10,00 € = 50,00 € auf Ausgleichslager 

Der OP an die Finanzbuchhaltung erfolgt dann über die 150,00 €. 

Ist die kundenseitig angegebene Menge auf der Gutschrift kleiner als die Menge_geliefert der Auftragsposition, so muss der Sachbearbeiter die Position ebenfalls manuell splitten. Die korrekte Menge wird mit dem Preis weiterbearbeitet und die neue Position muss separat behandelt werden. (Organisatorisch durch Löschen der Position aus der aktuellen Gutschriftsabrechnung oder anderen Betrag oder manuelle Rechnung). 

Durch den Splitt müssen die Chargen- oder Seriennummer-Zuordnungen entsprechend automatisch gesplittet werden. Die Gesamtmenge der Zuordnungen darf nicht geändert werden, da diese bereits geliefert wurden. Hier ist eine entsprechende Automatik zu hinterlegen. 

Achtung: Insbesondere das Handling der Lieferscheinnummer muss so berücksichtigt werden, dass die Original-Position und die Splittposition die gleiche Lieferscheinnummer haben. 

Ist die kundenseitig angegebene Menge auf der Gutschrift größer als die Menge_geliefert der Auftragsposition, so muss der Sachbearbeiter die Position manuell duplizieren. D.h. es entsteht eine neue Auftragsposition mit der überschüssigen Menge. Die Ursprungsposition kann dann mit ihrer unverändert korrekten Menge (und den kundenseitigen Konditionen) weiterbearbeitet werden. Die neue Position muss anschließend separat behandelt werden.

### Button Entbinden 

Ist in einem Auftrag ein Beschaffungsartikel für einen RAB (= Reservierte, Auftragsbezogene Bestellung) erfasst, dieser wird aber dringend zur Auslieferung für einen anderen Auftrag benötigt, so kann dieser Beschaffungsartikel aus dem eigentlichen Auftrag gelöst (entbunden) werden. Die entbundene Ware ist dann auch zwingend für den neuen Auftrag zu verwenden! Zu diesem Zweck steht der Button Entbinden zur Verfügung. 

Die Bestellung, der im Lager befindlichen Beschaffungsware wird vom Auftrag entbunden. Die Bestellung selbst bleibt als Lagerbestellung erhalten. 

Es ist unbedingt notwendig, dass dieser „entbundene“ Beschaffungsartikel neu bestellt wird. 

Jeder Schritt dieses Prozesses wird dokumentiert. 

Nach Drücken des Buttons Entbinden und bestätigen der Meldung wird der Auftragsbezug in der Bestellposition gelöscht. 

Das ist nur möglich, solange für die Position noch keine Packliste, Lieferschein oder Rechnung gedruckt worden ist. Es darf auch kein Verbundprozess sein. Damit ist auch sichergestellt, dass in Kommissionierung befindliche Positionen nicht mehr entbunden werden.

Wenn Sie diese Checkbox aktivieren, wird eine Liefersperre für diese Auftragsposition gesetzt. 

Diese Sperre kann gesetzt werden, so lange der Status der Position < 3 ist. 

Solange die Sperre gesetzt ist, kann für die Position weder eine Packliste noch ein Lieferschein gedruckt werden.

Wenn Sie diese Checkbox aktivieren, werden die Rabatte von der Faktura ausgeschlossen. 

Ist diese Checkbox zu einem Artikel gesetzt, für den eine Gutschrift erzeugt werden soll, so wird der Druck der Gutschrift verhindert. Grund dafür ist, dass in eNVenta die Möglichkeit der Teil-Gutschrift nicht vorgesehen ist!
