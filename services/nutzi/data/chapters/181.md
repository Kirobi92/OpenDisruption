# Basisform und Funktionen 
Um die Programmierung von Scannerdialogen zu erleichtern, wurden unterschiedliche Basisklassen geschaffen. Diese stellen, auf Basis des Bedienkonzeptes, einfache Einsprungspunkte zur Verfügung, um mit wenig Aufwand Masken zu erstellen. 

Aktuell existieren folgende Basisdialoge für verschiedene Einsatzzwecke. 

Basisform Einsatzzweck cfrmScannerBase cfrmScannerList cfrmScannerFilteredList 
Das Basisform enthält Funktionen zur Filterung dieser Listeneinträge, um die Anzeige der Liste einzuschränken. 
| |
| --- |
| Einfache Dialoge mit einzelnen Feldern oder Dialoge, für die es sonst keine passende Basisklasse gibt. |
| Dialoge mit einfachen Listen mit einer "handhabbaren" Anzahl von Einträgen. Diese werden direkt beim Öffnen des Dialogs geladen. |
| Dialoge mit Listen, deren potentielle Einträge in großer Anzahl auftreten können. |

Wichtig: Um alle Funktionen der Scannerbasisfunktionalitäten voll nutzen zu können, dürfen die Workflows von Scannerdialogen nicht einfach neu erstellt werden, sondern müssen von einem dazu passenden Workflow ableiten. 

Basisform Workflow zum Ableiten 
| | |
| --- | --- |
| cfrmScannerBase | wflcfrmScannerBase |
| cfrmScannerList | wflcfrmScannerBase |
| cfrmScannerFilteredList | wflcfrmScannerFilteredList |

## Zusätzliche technologische Grundlagen 
Zusätzlich zu den Basisformklassen gibt es noch weitere Logiken, welche das Programmieren vereinfachen sollen. 
Dazu gehören das Hauptmenü und die Scannereinstellungen. 

### Die Scannereinstellungen 
Es gibt Einstellungen, welche zur Laufzeit aus dem Sachbearbeiter vorbelegt werden, aber übersteuert werden sollten. Das bilden wir zukünftig über die cScannerSettings ab. Diese hängt am Global und wird nach dem Login bestückt. Dies geschieht in der Methode InitByScannerClerk. 

Die Einstellungen können aus allen Dialogen aufgerufen werden. 

Soll das nicht so sein, so können Sie mit der Methode DontShowSettings den Button deaktivieren lassen. 

Es gibt eine Methode, welche alle Elemente deaktiviert. 

Alle Felder, die außerhalb des Hauptmenüs nicht angezeigt werden sollen, müssen in der Methode HideElementsWhenNotMenu ausgeblendet werden.
