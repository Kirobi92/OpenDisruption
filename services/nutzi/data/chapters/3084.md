# Datenimport für Artikel Registerkarte Format 
Datenimport Registerkarte Format 

Auf dieser Registerkarte werden die Import-Daten der Quelldatei den eNVenta -Datenfeldern zugeordnet. 

In der linken Tabelle sind die eNVenta -Felder der ausgewählten Komponente zu sehen. In die rechte Tabelle werden die Spalten der Importdatei eingegeben, manuell oder über den Button F. Import , wenn die Quelldatei eine CSV-Datei mit Kopfzeile ist. 

Wenn die Import-Datei keine Kopfzeile erhält, ist diese Spalte leer. Dann verwenden Sie die Spalte 'Pos' (für die Reihenfolge der Import-Datei), um die Eigenschaften der eNVenta -Felder zuzuweisen. 

Vorgehensweise: 
Markieren Sie die Zeile der Quelldatei und die Zeile des eNVenta -Felds, in das der Wert geschrieben werden soll und weisen Sie das eNVenta -Feld über den Button Zuweisen der Position aus der Quelldatei zu. Das eNVenta -Feld wird in die Spalte 'Eigenschaften' übernommen, d.h. beim späteren Einlesen der Quelldaten (Katalog) werden die Werte aus der Position in die Quelldatei (in das zugewiesene eNVenta -Feld) übernommen. 

Im linken Bereich ( eNVenta -Felder) sollte nur die oberste Hierarchie, so wie sie der Dialog beim Aufruf der eNVenta -Felder anbietet, für das Zuweisen der Eigenschaften ausgewählt werden. 

Ob ein Import-Datensatz Daten überschreibt oder nicht, hängt von der Festlegung auf der Registerkarte Allgemein (rechts), den Datensatzoperationen ab. 

Bitte beachten Sie: 
Die Artikelnummer im Schattenstamm muss NICHT die eNVenta -Artikelnummer sein! Dies kommt zu einem späteren Zeitpunkt. 

Der Wert, der auf das eNVenta -Feld Bestellnummer gemappt wird, muss der Wert sein, mit dem später beim Lieferanten bestellt wird (besonders beim EDI). 

Der Wert Spalte wird mit den Spaltenwerten der Kopfzeile gefüllt, die Pos mit den Werten der jeweiligen Position aus der Quelldatei. Aufgrund dieser Positionsangabe werden später die entsprechenden Werte aus der Quelldatei (Katalog) gezogen. 

Bitte beachten Sie bei ASCII-Formaten, dass Werte in die Spalten 'Spalte', 'Start' ( eNVenta startet bei 0!, nicht bei 1) und 'Länge' manuell eingegeben werden müssen. 

Schlüssel 
Für den Import innerhalb des Data-Stagings ( Artikel Import Manager ) werden Schlüssel je Mapping gesetzt. D.h. hier muss je Artikel ein eindeutiger Schlüssel innerhalb des Data-Stagings gefunden werden. 

Sollte in der Quelldatei kein eindeutiger Schlüssel gefunden werden, können auch mehrere Felder als Schlüssel markiert werden. Diese werden dann als Main ImportKey zusammengesetzt. 

Beim nächsten Einlesen der Quelldatei in dasselbe Data-Staging wird nur noch nach dem Schlüssel gesucht (Spalte'Schl.'). Ist der Schlüssel vorhanden, werden die Daten überschrieben. Ist kein Schlüssel vorhanden, wird ein neuer Satz angelegt. 

Beim Import von Preisen in den Preisgruppen muss derselbe Schlüssel gesetzt werden, damit die Preisgruppe innerhalb des Data-Staging im Schattenstamm dem Artikel zugeordnet werden kann. 

Wenn Sie z.B. Artikelbeschreibungen mit Textumbrüchen über einen Datenimport importieren möchten, markieren Sie in der Formatdefinition die entsprechende Spalte und klicken auf den Button Umsetzung . Sie gelangen in den Dialog Daten Umsetzung . Dort wählen Sie auf der Registerkarte Autom. Umsetzung die Funktion 'Ersatz' aus der Combobox Umsetz. Funktion aus. Als erster Parameter wird die Spalte ausgewählt, als 2. das Zeichen bzw. die Zeichenfolge, die zum Zeilenumbruch ersetzt werden soll, z.B. "\n" und im 3. Parameter wählen Sie in der Combobox den Eintrag "3 : Zeilenumbruch" aus. Siehe nachfolgende Abbildung.

### Baumansicht Eigenschaften 

In diesem Feld befinden sich die Stammdaten / eNVenta -Felder. 

Symbol für Schlüssel = : 

Diese Kennzeichnung erfolgt aufgrund der Verschlüsselung als Primary Key in der Datenbank. Der Primary Key ist der eindeutige Schlüssel. 

Achtung: Es gibt Datenbank-Tabellen, die mehrere Primary Keys enthalten. Erst die Verwendung aller Primary Keys ergibt den eindeutigen Schlüssel! 

Der aufrufende Dialog steuert, welche Datenbank-Tabelle im Folgenden bearbeitet wird. 

Vergewissern Sie sich, dass beim Import alle Primary Keys zugewiesen sind, damit Ihr Import eindeutig die Datenbank bestückt! 

Beispiel: 

Im Sachkonto ist nicht die Kontonummer kennzeichnend, sondern erst die Kombination von Kontonummer, Typ und Jahr ergibt den eindeutigen Schlüssel; alle 3 Spalten sind Primary Keys! 

Führen Sie über einen Import eine Änderung vorhandener Daten durch, ohne absolut eindeutigen Schlüssel, z.B. nur Kontonr 3, wird der zufällig letzte Datenbankeintrag mit Kontonr 3 zu diesem "unvollständigen" Schlüssel geändert. Dies könnte auch ein Debitor sein, obwohl Sie Sachkonten importieren (das ist der Datenbank egal). 

Erfolgt die Zuweisung eines Schlüssels in die Tabelle (rechts), wird automatisch die Tabellenspalte 'Schl.' aktiviert. 

eNVenta-Felder 'individual read only': 

Diese Felder können exportiert, aber nicht importiert werden! Beim Import erzeugt dies einen Hinweis. Das sind z.B. Felder, die irgendwoher nachgelesen werden. Beispiel: Anlage , Registerkarte Wartung , Feld Firma1 : Es wird die Firma1 des gespeicherten Lieferanten angezeigt.
