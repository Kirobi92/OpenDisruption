# Historisierungsübersicht 
Historisierung Übersicht 

Für die Historisierung von Daten gibt es im Navigator >> System >> Administration >> Daten Historisierung den Eintrag Historisierungsübersicht . 

Hier werden alle Tabellen dargestellt und abgelegt, welche für die Historisierung in Frage kommen. 

Bei der Auslieferung von eNVenta ERP erhalten Sie bereits eine Konfiguration einiger Vorgangstabellen, wie z.B. Auftragskopf und Bestellkopf, um Aufträge und Bestellungen zu historisieren. 
## Steuerelemente des Formulars 

### Bezeichnungsfeld Historisierung Übersicht 
Tooltip: 

In diesem Dialog werden alle Tabellen dargestellt und abgelegt, welche für die Historisierung in Frage kommen. 
Für mehr Informationen drücken Sie F1. 

### Button Form-Menü 

Mit Klick auf diesen Button werden je nach Kontext und Berechtigung die unterschiedlichen Optionen des jeweiligen Dialogs hinter dem Button zur Auswahl angeboten. 

Die Funktionen für das Client Side Customizing sowie die Online-Hilfe stehen hier ebenfalls zur Verfügung.

### Button Initialisieren 

Mit diesem Button werden alle in der DBTable zur Historisierung eingestellten Tabellen gelesen. Dies sind die Tabellen welche in eNVenta zum Zeitpunkt der Auslieferung bereits markiert sind und diejenigen, welche in anderen Packages (auch Kunde) markiert wurden. 

Für Tabellen, welche noch nicht gespeichert sind, werden Datensätze (deaktiviert) eingetragen und können dann gespeichert werden. Für Datensätze, welche nicht mehr historisierbar sind, kommt eine Meldung. 

Tooltip:

### Feld Transferdatum 
Das Datum im Feld Transferdatum zeigt an, bis zu welchem Datum Daten bereits historisiert worden sind. 

Tooltip: 

Datensätze welche älter sind, wurden bereits in die Historie ausgelagert. 

### Feld Where-Bedingung 
Hier legen Sie das Kriterium fest, das zur Historisierung führt. Mit dem Button daneben öffnet sich der Dialog Suchbedingung , wo Sie eine Suchbedingung definieren können. 

Beispiel für Auftragskopf: ([lngOrderID] = 150831204136 AND [sOrderType] = '1') 

Tooltip: 

Nur Datensätze, welche dieses Kriterium erfüllen, stehen zur Historisierung zur Verfügung.

### Checkbox Aktivieren 
Ist die Checkbox Aktivieren angehakt, kann die Tabelle auch tatsächlich historisiert werden. (Siehe Dialog Daten historisieren ) 

Tooltip: 

Bei aktivierter Checkbox wird die Historisierung für diese Tabelle ausgeführt. 

### Feld Datum bis 
Das Datum, das Sie in das Feld Bis Datum eintragen, definiert das Datum, bis zu dem die Datensätze historisiert werden sollen. Alle Angaben, welche jünger sind als das angegebene Datum, können nicht historisiert werden. Achten Sie darauf, hier nur ein Datum anzugeben, das den gesetzlichen Grundlagen entspricht. 

Tooltip: 

Bis zu diesem Datum ist definiert, dass jüngere Datensätze im Produktivteil sind, ältere in der Historie. 

### Combobox Historienfeld 
Über diese Combobox legen Sie fest, nach welchem Datum die Historisierung vorgenommen werden soll. Die Auswahl bezieht sich hier immer auf alle verfügbaren Datumsfelder aus der jeweiligen Tabelle. 

Tooltip: 

Definiert, welches DB-Datumsfeld zur Historisierung herangezogen wird (Erfassungsdatum, Änderungsdatum). 

### Checkbox Null historisieren 
Wenn die Checkbox NULL historisieren gesetzt ist, werden NULL-Datumssätze berücksichtigt und historisiert. Dies bedeutet, sollte kein Eintrag im Datumsfeld vorhanden sein, z.B. Änderungsdatum ist leer, weil es nie geändert wurde, so ist es trotzdem ein älterer Datensatz und soll historisiert werden. Wird die Checkbox nicht gesetzt, so werden Datumsangaben ohne Wert als aktuell betrachtet und nicht historisiert. 

Tooltip: 

Wenn die Checkbox gesetzt ist, werden NULL Datumswerte berücksichtigt und historisiert. 

### Combobox Prüfung 
Der Eintrag, den Sie in dieser Combobox auswählen, gibt an aufgrund welcher Prüfung die eingestellte Prüfung übernommen wird. Momentan ist hier nur eine Datumsprüfung verfügbar, welche prüft ob das Datum des Vorgangs jeweils dem angegebenen Datum entspricht. 

Tooltip: 

Die Daten werden unter Berücksichtigung der eingestellten Prüfung übernommen. 

### Combobox Protokollierung 
In dieser Combobox stehen die Einträge Keine , Info , Info Details und Nur Fehler zur Verfügung. So können Sie steuern, unter welcher Voraussetzung eine Protokollierung stattfinden soll. 

- Info: alle Historisierungen werden pro Transaktion protokolliert. - Info detailliert: jede einzelne Historisierung wird protokolliert. - Nur Fehler: es werden nur Fehler protokolliert. 

Tooltip: 

Es kann eine Protokollierung konfiguriert werden.
