# Rabatt Einkauf 
Rabatt Einkauf 

Dieses Menü erlaubt die Erfassung und Pflege von Lieferantenkonditionen und -preisen. 

Es können sehr spezielle Konditionen auf Artikel- und Lieferantenbasis oder auch sehr allgemeine Konditionen über frei definierbare Rabattgruppen festgelegt werden. 

Das Modul Einkauf bietet folgende Möglichkeiten zur Rabattverwaltung: 
- Spezielle Kondition für Lieferanten - Spezielle Konditionen für einzelne Kunden oder Kundengruppen - Artikelkonditionen - Berücksichtigung von Streckenbestellungen - Kombination von Konditionen - Berücksichtigung von Verpackungseinheiten - Staffelkondition - Aktionspreise mit festem Gültigkeitszeitraum 
Diese Arten von Konditionen können auch miteinander kombiniert werden. 

So kann z.B. die zum Artikel hinterlegte Mengenstaffel mit einem Sonderrabatt eines Lieferanten ergänzt werden. Artikel, Kunden und Einkaufarten können bei der Festlegung von Konditionen beliebig kombiniert werden. 

Jede Rabattstaffel kann bis zu 999 Stufen und einen abweichenden Basispreis (auch in Fremdwährung) enthalten. Absolute Zu- und Abschläge je Mengeneinheit und Staffel sind möglich. Eine Staffelung ist alternativ nach Menge, Wert oder Gewicht möglich. Eine Assortierung über Kumulationsgruppen wird ebenfalls unterstützt. 

Je Kumulationsgruppe wird die Menge je Positionsmengenheit summiert und festgehalten. So können diese (Teil-)mengen in der Preisfindung pro Position verwendet werden. 

Wenn andere Positionen eine andere Positionsmengeneinheit haben, so können sie nicht zur Assortierungsmenge der aktuellen Position hinzugerechnet werden, da hier nicht in allen Fällen (unterschiedliche Artikel) definiert ist, wie die Mengeneinheiten ineinander umgerechnet werden sollen. 

Über den Button können Sie einen Rabattsatz kopieren. Dabei wird das Feld Lieferant geleert und die neue Lieferantennummer ist zwingend manuell zu erfassen. Beim Kopieren eines Datensatzes erkennt das System, ob es sich um einen doppelten Eintrag handelt. Sie erhalten eine entsprechende Fehlermeldung. 

Für das Anlegen von Rabattkonditionen über den Artikelpreisimport ist zu beachten, dass nur dann eine neue Rabattkondition angelegt wird, wenn mindestens eines der folgenden Felder gefüllt ist: 
- Grundrabatt 1/2 - Basispreis - Mindestmenge - Bestellnummer - Rabattstaffel Felder 

### Ändern und Löschen einer Rabattkondition 
Um eine vorhandene Kondition zu ändern, wird diese zunächst gesucht und angezeigt. Dazu tragen Sie bitte im Kopfbereich des Dialogs die gewünschte Rabattnummer in das Datenfeld Rabatt ein und klicken dann auf den Button . Der Rabattsatz wird angezeigt und kann verändert werden. 

Nach Abschluss der Änderungen speichern Sie die Daten mit dem Button in der Datenbank. 

Der Button entfernt die angezeigte Kondition aus der Datenbank, nachdem die nun erscheinende Meldung, ob Sie den Datensatz wirklich löschen möchten, mit einem Klick auf den Button Ja bestätigt wurde. 

### Assortierung in der Rabattverwaltung 
Dies ist eine spezielle Option für Artikelgruppenrabatte. Normalerweise werden die Rabatte nur für eine einzige Bestellposition berechnet, d.h. die Rabattstaffel wird auf die Menge dieser einzelnen Position angewendet. Ist die Option jedoch eingeschaltet, werden die Mengen aller Positionen einer Bestellung, die zu einer Kumulationsgruppe gehören, zusammengefasst und darauf wird für jede einzelne Position die jeweilige Rabattstaffel angewendet. Der sich daraus ergebende Rabatt wird dann in die Position eingetragen. 

Ist die Option Assortierung eingeschaltet, hat dies Auswirkungen auf die Geschwindigkeit der Bestellerfassung. Die Rabattvergabe erfolgt nur einmalig zum Zeitpunkt der Ersterfassung der Bestellung (Button ). 

Vorgehensweise für den Aufbau einer Assortierung: 
- Der Schalter Assortierung Eink im Programmteil Parameter muss eingeschaltet werden. - Die möglichen Kumulationsgruppen müssen geplant und mit der Picklistenverwaltung vorgegeben werden. - Jedem Artikel, der an der Assortierung teilnehmen soll, muss eine Kumulationsgruppe im Artikelstamm (Infos) zugeordnet werden. - In den Konditionen muss ebenfalls die Assortierung eingeschaltet sein.

### Feld Kunde 
Hier wird die Kundennummer angegeben. Dann können Sie hier Konditionen anlegen, welche dann ziehen, wenn der Bestellung ein Auftrag auf diesen Kunden vorliegt. 

Beispiel: 

Im Rabatt EK ist ein Datensatz mit Artikel 'TEST' und Lieferant '33' und Kunde '123' angelegt 
=> mit 15% Grundrabatt 

Im Rabatt EK ist ein Datensatz mit Artikel 'TEST' und Lieferant '33' und Kunde '456' angelegt 
=> mit 20% Grundrabatt 

Wenn Sie einen Auftrag erfassen mit Artikel 'TEST' auf Kunde '123' und aus diesem Auftrag bei Lieferanten '33' bestellen, bekommen Sie in der Bestellpos 15% Rabatt. 

Wenn Sie einen Auftrag erfassen mit Artikel 'TEST' auf Kunde '456' und aus diesem Auftrag bei Lieferanten '33' bestellen, bekommen Sie in der Bestellpos 20% Rabatt.

Über diesen Menüeintrag ist es möglich für alle Staffelkonditionen mit Abschlag das Kennzeichen Mengenskaliert zu setzen. Dies erfolgt, nachdem Sie eine entsprechende Abfrage bestätigt haben.

Hinweis: 
Der Rabattbetrag kann nicht bearbeitet werden, wenn das Kennzeichen Mengenskaliert gesetzt ist.
