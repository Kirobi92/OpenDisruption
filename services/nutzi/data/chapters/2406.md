# Verwaltung von Gutscheinen mit OP-führenden Konten 
Nachfolgend wird beschrieben, wie der Prozess „Gutscheine“ in der Finanzbuchhaltung mit einem OP-führenden Konto abgebildet werden kann. 

### Einstellungen 
Das OP-führende Konto muss folgende Einstellungen haben: 

- Im Sachkontenstamm auf der Registerkarte Allgemein wird die Check-Box OP-Führend aktiviert. - Im Sachkontenstamm auf der Registerkarte Allgemein wird der Radiobutton Keine Steuer aktiviert. 

### 1. Verkauf des Gutscheins 

#### Buchung wie folgt: 

| Buchungskreis | Finanzbuchhaltung |
| --- | --- |
| BelegnummerD | Nr. des Gutscheins (Empfehlung) |
| Konto Soll | z.B. Kasse |
| Konto Haben | OP-führendes Konto |
| Betrag | Wert des Gutscheins |

Auf dem OP-führenden Konto wird ein OP mit einem negativen Betrag erzeugt. 

Beispiel: Beleg G101 / 1000 Kasse an 1730 Gutscheine im Umlauf / 100,-- 

Auf dem Konto ist nun ein OP mit BelegNr „G101“ und Betrag -100,-- zu sehen. 

### 2. Kunde kauft ein 

#### Buchung wie folgt: 

| Buchungskreis | Debitoren-Rechnung |
| --- | --- |
| Belegnummer | Rechnungsnummer |
| Konto Soll | Kunden |
| Konto Haben | Kunde |
| Betrag | Zahlbetrag |

Beispiel: Beleg 2007100 / 10010 Mayer an 8010 Umsatz Werkzeug / 1000,-- 

### 3. Kunde zahlt Rechnung unter Abzug des Gutscheins 

#### Buchung wie folgt: 

| Buchungskreis | Debitoren-Ausgleich |
| --- | --- |
| Belegnummer | evtl. Kontoauszug / Rechnungsnummer |
| Konto Soll | z.B. Bank |
| Konto Haben | Kunde |
| Betrag | Zahlbetrag |

Bei einer OP-Zuordnung über den Restbetrag (Wert des Gutscheins) führen Sie eine OP-Anlage durch. Dabei achten Sie auf folgende Einstellungen: 

Belegnummer: Nr. des Gutscheins (Empfohlen wegen besserer Zuordnung und späterem Ausgleich) 

Zahlungsbedingung: eine Zahlungsbedingung ohne Skonto (Empfohlen wegen einfachen Ausgleichs) 

Beispiel: Der Kunde Mayer zahlt 900,-- Euro. 

Beleg KSK197 / 1210 Bank an 10010 Mayer / 900,-- 

In der OP-Zuordnung wird die OP-Anlage über € 100,-- mit folgenden Werten erfasst: 

Belegnummer: G101 

Zahlungsbedingung: Netto 30 Tage 

### 4. Ausbuchen der OP-Anlage gegen Konto "Gutscheine im Umlauf" 

#### Buchung wie folgt: 

| Buchungskreis | Debitoren-Ausgleich |
| --- | --- |
| Belegnummer | evtl. Kontoauszug oder Gutscheinnummer |
| Konto Soll | OP-führendes Konto |
| Konto Haben | Kunde |
| Betrag | Wert der Gutschrift |

Beispiel: Beleg KSK197 / 1730 Gutscheine im Umlauf an 10010 Mayer / 100,-- 

Bei der OP-Zuordnung ganz normal die OP Nr. G101 ausgleichen. 

Auf dem Konto 1730 wird nun ein OP mit BelegNr „G101“ (Belegnummer vom Kunden-OP) mit Betrag 100,-- erzeugt. 

--> Da die Belegnummer identisch mit der Gutschriftnummer ist, und die Beträge sich aufheben, werden auf dem Konto 1730 beide OPs automatisch als bezahlt markiert. D.h., sie sind nicht mehr offen und zu sehen. 

Somit ist der Prozess „Gutscheine“ beendet. 

### 4/1. Falls bei der OP-Anlage Belegnummer ungleich Gutschriftnummer ist: 
Hier müssen zusätzlich beim OP-führenden Konto beide OPs (also Gutschein und Ausgleich) durch eine zusätzliche Buchung miteinander ausgebucht werden. 

Buchung wie folgt: 

| Buchungskreis | Finanzbuchhaltung |
| --- | --- |
| Belegnummer | Nummer des Gutscheins (Empfehlung) |
| Konto Soll | z.B. OP-führendes Konto |
| Konto Haben | z.B. durchlaufender Posten |
| Betrag | 0 |

Beispiel: G101 / 1730 Gutscheine im Umlauf an 1590 durchlaufender Posten / 0,-- 

Bei der OP-Zuordnung werden nun beide OPs ausgeziffert.
