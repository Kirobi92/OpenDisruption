# Einsatzgenerierung im Service 
Einsatzgenerierung 

Dieser Dialog öffnet sich im Serviceeinsatz >> Optionen >> Einsatzgenerierung . 

Anhand der Suchkriterien werden dabei automatisch Serviceeinsätze für die Zukunft generiert. 

Geben Sie hierzu im Bereich Vorgaben den Zeitraum an, den Monteur, der den Serviceeinsatz durchführen soll und zu welcher Einsatzart der Serviceeinsatz erfasst werden soll. 

Die überzähligen Positionen, welche nicht freigegeben sind, werden aus dem Serviceeinsatz gelöscht und in einen neuen Serviceeinsatz eingefügt. In diesem neuen Serviceeinsatz sind nur diese Positionen enthalten. 

Wird ein Auftrag, der aus einem Serviceeinsatz generiert wurde, gelöscht oder storniert, so werden der zugehörige Serviceeinsatz und alle seine Positionen wieder geöffnet. Der Status des Serviceeinsatzes wird wieder auf „offen“ gesetzt. Der Status der einzelnen Positionen wird ebenfalls auf „offen“ gesetzt. 

Wird eine einzelne Auftragsposition gelöscht, so erscheint eine entsprechende Warnmeldung, der Serviceeinsatz bleibt allerdings unverändert. 

Beim Löschen einer einzelnen Position im Verkaufsauftrag erscheint nur eine Warnmeldung, dass der Bezug zum Serviceeinsatz noch vorhanden ist.

### Button Job-Server 

Dieser Button steht für die automatische Generierung von Service-/Wartungseinsätzen zur Verfügung. 

In diesem Zusammenhang stehen auch die nachfolgenden Felder im Bereich Vorgaben zur Verfügung. 

- Datum/Zeit von (Tag des Monats), Pflichtfeld - Datum/Zeit bis (Tag des Monats), Pflichtfeld - Zuständiger Monteur, optional - Einsatzart, Pflichtfeld 
Falls am Serviceobjekt eine Einsatzart hinterlegt ist, wird diese genutzt. Wenn nicht, wird die Einsatzart des Parameters verwendet. Als Monteur wird der zuständige Mitarbeiter aus dem Serviceobjekt ermittelt, wenn dort nicht hinterlegt aus dem Artikelstamm des Serviceartikels und als letzten Rückfall sonst aus den Parametern des Jobs.
