# Auftragsarten 
Auftragsarten 

Diesen Dialog erreichen Sie im Navigator über das Menü System >> Verkauf >> Auftragsarten . Er ist von besonderer Bedeutung für die optimale Organisation und Abwicklung der Auftragsbearbeitung. 

Es können bis zu 999 verschiedene Auftragsarten eingerichtet werden. Für jede Auftragsart wird festgelegt, welchen Status die Positionen bei der Auftragserfassung erhalten sollen, welche Formulare wie oft auf welchen Druckern gedruckt werden sollen, ob die Auftragsnummer automatisch oder manuell vergeben werden soll und ob für den Auftrag eine Lagerbuchung bzw. Reservierung erfolgt. 

Bei einer Auftragsart ohne Fakturierung kann der Status der Position nicht geändert werden, wenn eine Sperrung (egal welche) auf dem Auftrag liegt, unabhängig ob Liefer-, Faktura- oder Gesamtsperre. 

Hier einige Beispiele typischer Auftragsarten: 

Sofortauftrag bzw. Abholauftrag: 
Auftrag zur sofortigen Auslieferung und Rechnungsstellung. Die erfassten Auftragspositionen werden automatisch zur Lieferung und zur Fakturierung freigegeben. Der Lieferschein und eventuell eine Packliste sollen sofort erstellt werden. 

Lagerauftrag bzw. Terminauftrag: 
Es wird ein Auftrag mit einem bestimmten Liefertermin erfasst oder die Waren sind am Lager noch nicht verfügbar. Die Positionen sind zunächst nicht zur Lieferung freigegeben. Zu gegebener Zeit erfolgen die Lieferfreigabe und der Druck des Lieferscheins. Nach Prüfung der Verfügbarkeit im Lager werden die Waren ausgeliefert. Es erfolgt die Freigabe zur Fakturierung. 

Streckenauftrag: 
Die bestellten Waren werden nicht vom eigenen Lager ausgeliefert, sondern beispielsweise vom Lager des Lieferanten. 

Der Lieferschein wird sofort erstellt, es erfolgt jedoch noch keine Fakturafreigabe. Nachdem der Lieferant die Auslieferung gemeldet hat, wird der Auftrag zur Fakturierung freigegeben. Bei der Fakturierung werden jedoch keine Lagerbuchungen durchgeführt! 

Definieren Sie eigene Auftragsarten entsprechend der Organisation Ihres Unternehmens. Sparen Sie möglichst viele Arbeitsschritte mit Hilfe einer optimalen Anpassung der Auftragsarten ein. 

Bei einer Auftragsart OHNE Lagerbuchung findet nach einer Mengeneingabe keine Verfügbarkeitsprüfung mehr statt! 

Wenn eine Auftragsart ohne Lagerbuchung gewählt ist, kann trotz gesetztem Parameter Strikte Seriennummernzuordnung ein Auftrag ohne Seriennummer gebucht werden. 

Hinweis: es findet keine Lagerbuchung statt, daher ist eine Seriennummernzuordnung nicht nötig! 

Außer der Steuerung von Prozessen über die Auftragsart, kann auch eine Auswertung der Prozesse stattfinden. Z.B. Umsatz nach Lagerauftrag oder Beschaffungsauftrag.

### Feld Auftragsart 
Hier wird die Nummer der markierten Auftragsart aus der Tabelle angezeigt. 

Sie können auch eine neue Auftragsart definieren. Dazu geben Sie in dieses Feld eine weitere Nummer ein. Da alle bereits angelegten Auftragsarten auf der Registerkarte Tabelle angezeigt werden, können Sie leicht erkennen, welche Nummern noch frei sind. 

Eine separate Auftragsart ist immer dann zu verwenden, wenn der Nutzen einer neuen Auftragsart gegenüber dem dadurch entstehenden Overhead im Know-how-Aufbau überwiegt oder aber ein komplett eigenständiger Prozess damit abgewickelt werden soll. 

Grundsätzlich gilt: 

So viele Auftragsarten wie nötig, aber so wenige wie möglich! 

Beispiel: 
Bei einer Lagerauftragsart geht man normalerweise davon aus, dass ausreichend Ware verfügbar ist. Also wird eine solche Auftragsart üblicherweise so konfiguriert, dass bei der Erfassung vorhandene Ware bereits im Status 2, also Lieferfreigabe, erfasst wird. Ist die Ware bei der Erfassung aber nicht verfügbar, erhalten Sie eine Verfügbarkeitsabfrage. 

Das bedeutet zusätzliche Klicks. 

Bei einem Beschaffungsauftrag geht man davon aus, dass die Ware nicht verfügbar ist. Sie könnten das nun über die normale Lagerauftragsart abbilden, müssten dann aber bei jeder Position die Verfügbarkeitsabfrage durchlaufen. 

Alternativ dazu kann aber auch eine eigene Auftragsart Beschaffungsauftrag definiert werden, bei der die Erfassung der Positionen im Status 1 erfolgt und kein Verfügbarkeitsdialog aufgeht. 

Im Projekt ist nun zu entscheiden, was sinnvoller ist.
