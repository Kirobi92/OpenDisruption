# X-Mapping Editor Registerkarte Spezifisch 
Registerkarte Spezifisch 

Wenn Sie auf der Registerkarte X Knoten den Button Spezifisch drücken, wird die Registerkarte Spezifisch zugänglich. 

Auf dieser Registerkarte wird angegeben, nach welchem Kriterium die Daten innerhalb dieses Knotens verarbeitet werden. 

Die Kriterien können als Element (Nr. 1 oder 2 oder ...) angegeben werden oder über einen Filtereintrag in der Tabelle darunter. 

Hier als Beispiel, wenn z.B. der Inhalt des Felds BUYER_PID_buyer_specific den Wert "buyer_specific" beinhaltet. 

Inhalt der Datei: 

<bmecat:BUYER_PID type="buyer_specific">Kunden-Artikelnr</bmecat:BUYER_PID> 
### Feld Spezif.Name 
Hier können Sie den Namen eines spezifischen Knotens definieren. Er hat keine technische Bedeutung, sondern dient lediglich als Hilfe für die Person, die das Mapping konfiguriert, den Knoten zu identifizieren. 

Tooltip: 

Der Name eines spezifischen Knotens lässt sich frei wählen. 
Er hat keine technische Bedeutung, sondern dient lediglich als Hilfe für die Person, die das Mapping konfiguriert, den Knoten zu identifizieren. 

### Feld Element Nr. 
Tooltip: 

Gibt an, welches Element in einer Elementenliste gemeint ist. 
Ist eine Bedingung zu dem spezifischen Element hinterlegt, bezieht sich diese Nummerierung auf das entsprechende Element unter allen Elementen, auf die die Bedingung zutrifft (Die Bedingung kann auf mehrere Elemente zutreffen). 
Ist die CheckBox "von hinten" aktiv, bezieht sich die Nummerierung vom letzten Element rückwärts. Das Element 1 von hinten ist somit das letzte Element und das Element 2 von hinten ist das vorletzte Element. 

Beim Export kann dieses Feld leer gelassen werden. 

### Checkbox Von hinten 
Wenn Sie diese Checkbox aktivieren, wird die Elementnummer von hinten (vom letzten Element) rückwärts gezählt. 

Element Nr. 1 = letztes Element 
Element Nr. 2 = vorletztes Element 
usw. 

Tooltip:

In dieser Spalte geben Sie an, wie der Wert verglichen werden soll (gleich oder ungleich).

### Tabellenspalte Negieren 
In dieser Spalte kann der Wert umgekehrt werden (negieren).

Der Vergleich wird umgekehrt (negiert). 

### Tabellenspalte Vergleichswert 
In dieser Spalte kann ein Vergleichswert angegeben werden. 

### Tabellenspalte X Path 
Hier geben Sie den Vergleichswert als XPath-Ausdruck an. 

Tooltip: 

Der Vergleichswert ist ein XPath-Ausdruck.
