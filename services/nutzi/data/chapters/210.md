# Suchbedingung / Suche mit SQL 
Suchbedingung 

Die Suche mit SQL bietet den großen Vorteil, dass alle Daten aus einer Datenbank gesucht werden können, ohne erforderlichen Zugriff auf die Datenbank. D.h. es ist keine Programmierung nötig bzw. vom Sachbearbeiter sind nicht erst entsprechende Rechte bei Vorgesetzten etc. anzufordern. 

Die Suche mit SQL ist die flexibelste und mächtigste Suchmöglichkeit innerhalb von eNVenta ERP . Manche Suchanforderungen, wie zum Beispiel die Suche von PLZ X bis PLZ Y, eine ODER- und eine NICHT-Verknüpfung, können aber alleine durch die Funktionen auf der Oberfläche nicht realisiert werden. Die Anforderungen an die Selektionen wachsen aber ständig. 

Die Suche mit SQL bietet die Möglichkeit, eine Suchanfrage unter Verwendung aller Felder (zum Beispiel die Kundenfelder) und vieler Operatoren zu formulieren. Dazu wird z.B. im Dialog Kunde über das Menü Optionen >> Suchen mit SQL der zugehörige Dialog aufgerufen. Die Suche kann kombiniert werden mit den Optionen Mit Partnern bzw. Mit Merkmalen . 

Solch eine Suche mit SQL kann jederzeit für die Selektion von Daten verwendet werden. Sie erhalten eine Ad-Hoc-Anzeige der Daten in eNVenta ERP und müssen hierfür keine Abfrage direkt auf der Datenbank ausführen. In Verbindung mit dem Client Side Customizing können Sie auch die Ergebnisanzeige in der Registerkarte Tabelle (z.B. im Kundenstamm) anpassen, um sich die passenden Spalten anzeigen zu lassen. 

Die Erstellung der Abfrage kann über die Registerkarte Bedingungen und die entsprechenden Operatoren erfolgen oder für erfahrenere Sachbearbeiter auf der Registerkarte Expertenmodus . 

Bei der Suche über das Erfassungsdatum wird der aktuelle Tag der Abfrage nicht automatisch berücksichtigt. Dies hängt damit zusammen, dass die Speicherung eines Vorgangs inkl. Uhrzeit erfolgt, die Abfrage aber nur mit Datum (ohne Uhrzeit).
