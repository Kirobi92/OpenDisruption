# Broker-Sitzungen 
Broker-Sitzungen 

Diesen Dialog finden Sie unter System >> Administration >> Broker-Sitzungen . 

Dieses Werkzeug ist dazu gedacht, bei möglichen Fehlern auf dem Broker, wie z.B. hängengebliebenen Sessions, einzugreifen. Des Weiteren können Sitzungen beendet werden; so können blockierte Speicher wieder freigegeben werden. Damit lässt sich der Neustart des Servers verhindern. 

Sessions, die länger inaktiv sind, werden automatisch gelöscht. Dies ist eine Logik aus Administration / Broker-Sitzungen. Beim Start einer neuen Sitzung erfolgt eine Prüfung pro Broker. Diese Einstellung kann durch den Administrator direkt auf der Datenbank eingestellt werden und nicht über die Oberfläche. 

Im Bereich Befehle und Connects werden die Datenbank-Aktionen gezählt und gemessen. Das bezieht sich auf die ausgewählte Sitzung: 

- Befehle : bezieht sich auf die abgesetzten SQL-Befehle - Connects : bezieht sich auf die durchgeführten Datenbank-Connects 
Die Konfiguration erfolgt per SQL in der Daten-DB SYSTEM_BASIS:BrokerTimeout. Dieser Wert wird in Stunden angegeben. 

Dieses Werkzeug kann nur Informationen von dem Broker anbieten, auf dem das Werkzeug gestartet wird. 

Da dieser Dialog nur in wirklichen Ausnahmefällen zum Einsatz kommen soll und die darin enthaltenen Funktionen sehr kritisch sind, ist es mit einem grell-gelben Hintergrund versehen.

### Button Löschen 

Dieser Button beendet die in der Tabelle ausgewählte Sitzung. Das ist z.B. nützlich, wenn der Client unkontrolliert beendet wurde und somit die Sitzung selber nicht mehr beenden kann. Der von dieser Sitzung blockierte Speicher kann so wieder freigegeben werden. 

ACHTUNG: 

Dabei wird keine Rücksicht darauf genommen, ob der Client noch Kontakt zum Broker hat oder nicht. Dieser Vorgang kann nicht rückgängig gemacht werden. 

Falls der Client noch aktiv ist, erhält er nach dem Beenden der Sitzung bei der nächsten Anfrage die Meldung "Session has been terminated" und beendet sich. 

Es ist nicht möglich seine eigene Sitzung zu beenden. 

Tooltip: 

Sitzung beenden 

### Button Schließen 

Mit diesem Button schließen Sie den Dialog. Bitte denken Sie daran, Ihre Eingaben / Änderungen vorher zu speichern! 

### Button Collect GC 

Dieser Button ruft den Garbage-Collector von .NET auf. Das kann hilfreich sein, um eine bessere Abgabe im .NET-Speicher zu erhalten. Normalerweise ruft .NET diese Funktion selber auf, das kann sich aber je nach Auslastung des Servers etwas verzögern. So kann es z.B. sein, dass durch das Beenden einer Sitzung der Speicher nicht sofort wieder freigegeben wird. Mit diesem Button kann das erzwungen werden.

### Button Blockieren Job 

Dieser Button entfernt automatisch alle Datensatzsperren, welche im System hinterlegt wurden. Dies vor dem Hintergrund, dass es in eNVenta die Möglichkeit gibt, bestimmte Daten für die weitere Bearbeitung zu sperren. 

Wird dieser Button gedrückt, wird manuell der Job gestartet, welcher die Tabelle der blockierten Werte überprüft und alle Datensätze löscht, welche laut Zeitstempel nicht mehr blockiert werden müssen.

### Feld Broker Server 
Der Name des Servers wird angezeigt. 

### Feld Broker gestartet 
Hier wird die Zeit angezeigt, wann der Broker-Prozess gestartet wurde. So können Sie z.B. erkennen, wann das letzte Mal ein IIs-reset durchgeführt wurde. 

### Feld GC Bytes 
Der Speicher wird angezeigt, der vom Garbage-Collector (also von der .Net-Runtime) verwendet wird. 

### Feld Prozess Bytes 
Der vom Prozess reservierte Speicher. Dieser kann deutlich höher sein, als die GC-Bytes. 

### Feld Aktualisierung 
Hier können Sie sehen, wann die letzte Aktualisierung erfolgt ist. 

### Tabelle Aktuellen Sitzungen 

### Tabellenspalte Sitzung 
Die Bezeichnung der Sitzung wird angezeigt. 

### Tabellenspalte Benutzer 
Der angemeldete Benutzer wird angezeigt. 

### Tabellenspalte Anfragen 
Die Anzahl der Anfragen wird angezeigt, die in dieser Sitzung an den Broker gesendet wurden. 

### Tabellenspalte Ø ms 
Hier wird die durchschnittliche Antwortzeit bei Anfragen in Millisekunden angezeigt.

Durchschnittliche Antwortzeit bei Anfragen in Millisekunden 

### Tabellenspalte Ø ms / DB

Tooltip: 

Durchschnittliche Antwortzeit der Datenbank bei Anfragen in Millisekunden. 

### Tabellenspalte Letzter Zugriff 
Hier wird der Zeitpunkt der letzten Anfrage an den Broker angezeigt. 

### Tabellenspalte Vor 
Hier wird die Zeit angezeigt, die seit der letzten Anfrage verstrichen ist. 

### Tabellenspalte Start 
Hier wird der Zeitpunkt angezeigt, zu dem die Sitzung gestartet wurde. 

### Tabellenspalte Anfrage 
Diese Spalte zeigt an, ob im Moment eine Anfrage vom Broker bearbeitet wird. 

### Tabellenspalte Anzahl Objekte 
Hier wird angezeigt, wie viele Objekte im Moment von der Sitzung verwendet werden. Diese Zahl kann ggf. höher sein, als die tatsächlich verwendeten Objekte, wenn der Garbage Collector des .Net-Frameworks noch nicht alle überflüssigen Objekte weggeräumt hat. Dabei werden nur die Objekte vom Typ "DevFrameworkBaseObject" gezählt. Die Zahl lässt keinen konkreten Rückschluss auf den Speicher-Verbrauch zu. Sie kann aber als Anhaltspunkt dafür dienen, speicherintensive Sitzungen zu identifizieren. 

### Tabellenspalte Objekte + 
Hier wird angezeigt, wie viele Objekte im Lauf der gesamten Sitzung erzeugt wurden. 

### Feld Client-Infos 
Dabei handelt es sich um die Informationen, die der Java-Client bei der ersten Anfrage mitteilt. Beispielsweise werden Informationen wie Rechnername, IP-Adresse, Java-Version, Client-Betriebssystem ausgegeben. Diese Informationen können je nach verwendetem Client abweichen. 

### Feld Offene Fenster 
Hier werden die momentan offenen Forms in der Sitzung angezeigt. Es handelt sich um die technischen Namen der Forms. 

### Feld Anfragen 
Anzahl der Anfragen, welche an den Broker gesendet wurden. 

### Feld Gesamt ms 
Gesamte Dauer der Anfragen, welche an den Broker gesendet wurden. 

### Feld Ø ms 
Hier wird die durchschnittliche Antwortzeit bei einer Anfrage eingetragen. 

### Feld Anz. DB 
Hier wird die Anzahl der Befehle angezeigt. 

### Feld Anzahl Connects 
Hier wird die Anzahl der Connects angezeigt. 

### Feld Gesamt ms 
Hier wird die Gesamtzeit in Millisekunden angezeigt. 

### Feld Gesamtzeit 
Hier wird die Gesamtzeit in Millisekunden angezeigt. 

### Feld Ø ms / Anfrage 
Hier wird die Summe der beiden Zeiten auf eine Broker-Anfrage heruntergerechnet. 

### Feld Ø ms / DB 
Hier wird die Gesamtzeit auf einen einzelnen Befehl heruntergerechnet. 

### Feld Gesamtzeit 
Hier wird die Gesamtzeit auf einen einzelnen Connect heruntergerechnet.
