# Debitoren Factoring 
Debitoren Factoring 

eNVenta unterstützt das verdeckte Factoring (auch stilles Factoring genannt). 

Verdecktes Factoring bedeutet, dass offene Forderungen an eine Factoringbank verkauft werden, ohne dass der Debitor darüber informiert wird. Diese Factoring-Variante wird einerseits genutzt, um die Beziehungen zu den Kunden nicht zu belasten und andererseits, um Abtretungsverbote zu umgehen. Das Mahnwesen bleibt im Unternehmen angesiedelt. 

Das Debitoren Factoring öffnet sich im Buchen >> OP >> Debitoren Factoring und steht für den Verkauf von offenen Debitoren-Posten an eine Factoring Bank zur Verfügung. Ihr Unternehmen bekommt die OP-Summe ausbezahlt, reduziert um den Auszahlungs- und Gebührensatz. Die Überwachung der Zahlungseingänge sowie das Mahnwesen werden weiterhin von Ihnen übernommen. 

Folgende Aufgaben werden über das Debitoren Factoring in eNVenta abgebildet: 

- Kundenstammdaten melden - OPs melden (täglich / Bestand) - darauf achten, dass nur verkaufsfähige OPs gemeldet werden - Änderungen von Kundendaten melden - Berücksichtigung bei der Liquiditätsübersicht - Tausch der Bankverbindung im Rechnungsdruck 
In diesem Dialog ist zuerst eine Bank aus der Combobox Factoring Bank auszuwählen. Anschließend wird der Button zugänglich. 

Der Dialog öffnet sich nur, wenn in den Parametern der Fibu auf der Registerkarte Debitoren + Kreditoren die Checkbox Factoring aktiv gesetzt ist. 

Die Standard-Abschlusslogik setzt für alle gemeldeten OPs das Melde-Datum und die Factoring Bank, an die sie gemeldet wurden. Zusätzlich werden, falls die Checkbox Umbuchung aktiviert ist, während des Exports neue Daten für das Journal vorbereitet. Dieses enthält für jedes vorhandene Obligo in den OPs 2 neue Sätze; eine Umbuchung auf das Factoringkonto findet statt. Diese Sätze erscheinen dann in der Sachkontenübernahme. 

Der Versand von E-Mails ist nur über SMTP möglich! 

Meldung neuer OPs für die Factoring Bank Teamfactoring 

Es ist möglich, für die Factoring Bank Teamfactoring csv-Dateien für Kundendaten und neue OPs zu melden. 

Meldung von Kundenstammdaten: 
Bei Meldung der Kundendaten soll die Namensgebung der Datei nach folgendem Schema aufgebaut werden: 

| Bezeichnung Teil 1 | DEBITOREN 6 |

| Trennzeichen | '_' (Unterstrich) |

| Trennzeichen | '_' (Unterstrich) |
| Bezeichnung Teil 3 | Datum im Format 'JJJJMMTT' (Jahr, Monat, Tag) |
| Format: | .CSV |

„BELEGE5_2010206_20161110.csv ”

### Button Suche 

Mit diesem Button starten Sie die Suche. 

Sie können ganz allgemein suchen, indem Sie den Button drücken, gleich nach Öffnen des Dialogs. Oder Sie geben im Kopf des Dialogs die Selektionskriterien vor und drücken dann diesen Button. 

Dieser Button muss zu jeder gewählten Registerkarte erneut gedrückt werden, um Daten einzulesen. 

### Button Neu 

Dieser Button leert die Daten in allen Feldern, nicht gespeicherte Änderungen gehen verloren. 

Ist in den Parametern für die Sachbearbeiter die Checkbox 2 Schritt Neu aktiviert, bleiben die Suchkriterien in den Stammdialogen erhalten. 

Siehe Kapitel Parameter Sachbearbeiter . 

### Button Speichern 

Dieser Button speichert bzw. verarbeitet die Eingaben. Vor einer endgültigen Speicherung prüft das System die Eingabe. Wenn fehlende oder fehlerhafte Daten gefunden werden, erhalten Sie eine Meldung und die Eingabe muss entsprechend korrigiert bzw. vervollständigt werden. 

### Button Löschen 

Dieser Button löscht Ihre Eingaben. Genauer gesagt, die markierte Zeile wird mit dem versehen und erst endgültig gelöscht, wenn Sie den Button drücken. 

### Button Schließen 

Dieser Button schließt den Dialog. In manchen Fällen werden Sie gefragt, ob Sie die Eingaben / Änderungen noch vorher speichern möchten. 

### Button Import Meldung 

Dieser Button öffnet den Dialog CSV Import . Der Button ist nur zugänglich, wenn zu der Bank im Dialog Factoring Parameter (Registerkarte Allgemein ) die Checkbox Zahl. erhalten gesetzt ist. Hier kann nun die Rückmeldedatei der Bank eingelesen werden. Die Felder werden entsprechend der gewählten Bank gefüllt. Die Verarbeitung findet mit Einlesen der Daten statt.
