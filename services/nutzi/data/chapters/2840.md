# Tupel-Generierung 
Tupel-Generierung 

Dieser Dialog öffnet sich, wenn Sie im Dialog Monitor Empfehlungen den Button Generieren drücken. 

Für die Ermittlung der Daten müssen sogenannte Tupel (Paare) von Artikeln erstellt werden. Hierfür werden alle notwendigen Parameter eingestellt und im Anschluss werden die Daten analysiert. 

Folgende Parameter stehen zur Einstellung zur Verfügung: 

- Anzahl Wochen 
Zeitraum der historischen Auswertung. Je kürzer der Zeitraum, desto höher die Aktualität, jedoch desto niedriger der Umfang der Empfehlungen. Maximum: 48 Wochen, Standard: 18 Wochen - Faktor nächster Auftrag 
Es werden auch Tupel der Artikel eines Auftrags zu den Artikeln des nächsten folgenden Auftrags gebildet. Diese Tupel können variabel gewichtet werden, da sie möglicherweise eine geringere Relevanz haben, als die Artikel innerhalb des gleichen Auftrags. Wird keine Angabe oder 0 gemacht, so werden diese Tupel nicht ermittelt. Standard: 0,2. - Shop ID 
Ist eNVenta eGate vorhanden und die Verbindung konfiguriert, kann hier die Nummer des anzusprechenden Webshops angegeben werden. In diesem Fall werden zusätzlich auch Tupel aus den Produktseitenaufrufen der Shop-Benutzer generiert. Dies erhöht Umfang und Qualität der möglichen Produktvorschläge. 
Nach der Einstellung der Parameter kann mittels Button Neu Generieren die Analyse gestartet werden. Hierbei werden alle bestehenden Daten gelöscht und die Daten komplett neu ermittelt. Sollten Sie keinen Parameter geändert haben, so können Sie die Analyse mittels Job Server ausführen. Die Daten werden entsprechend ergänzt, anstatt komplett neu generiert. 

Achtung: Laufzeit der Generierung ca. 1 Minute pro 1000 Artikel. Die Generierung sollte daher eher in weniger arbeitsintensiven Stunden erfolgen. Bei der Ausführung via Job Server sollten die Daten 1x monatlich neu ermittelt werden. 

Nach der Generierung wird eine Statistik der ermittelten Werte angezeigt: 

Sie gibt Auskunft darüber, wie viele Tupel (d.h. mögliche Vorschläge) auf den 3 Ebenen generiert wurden (Anzahl). Die anderen 3 Werte geben Auskunft über die ermittelten Gewichtungen der Tupel. Auf Ebene Cluster und Total werden nur Tupel mit einem Mindestgewicht von 1,6 erzeugt. 

Nach der erfolgreichen Generierung der Tupel können diese Daten nun mittels des Empfehlungsagenten analysiert werden, damit der Sachbearbeiter diese Daten im täglichen Geschäft nutzen kann. 

Die Generierung erfolgt hierbei mittels eines Empfehlungsagenten. Hierzu müssen Sie im Dialog Monitor Empfehlungen im oberen Bereich einen neuen Agenten anlegen. (Button neben der Tabelle) Bei der Markierung eines Agenten können Sie darunter über eine zeitliche Beobachtung den Erfolg der Empfehlungen betrachten. 

Die Generierung der Empfehlungen erfolgt über den Dialog Empfehlungsagent .
