# Kundenschnellerfassung 
Kunde erfassen 

Wenn Sie in einem Vorgang (z.B. Auftrag) nach einem Kunden (Feld Kunde ) suchen bzw. in das Feld Suchname einen Namen eingeben, der nicht im Kundenstamm erfasst ist, öffnet sich der Dialog Kunde erfassen , um "schnell" den Kontakt anzulegen. Die komplette Anlage und ggf. Verifizierung des Kunden kann zu einem späteren Zeitpunkt im Kundenstamm erfolgen. 

Bei der Erfassung eines neuen Kunden müssen die Felder Suchname , Gruppe und Firma1 auf jeden Fall gefüllt werden. Die Felder Währung , MwSt. , Preisgruppe und Sprache werden per Default gefüllt, falls nichts angegeben wird. 

Über die Combobox Gruppe wird die Zugehörigkeit zu einer Kundengruppe angegeben. So kann über die Kundengruppe eine erste Konditionierung stattfinden, ohne dass dieser Kunde bereits komplett durchkonditioniert ist. 

Wenn Sie die Checkbox Einmalkunde aktivieren, handelt es sich um einen nicht dauerhaften Kunden. Dieser wird angelegt und bleibt so lange im System erhalten bis alle offenen Posten in der Finanzbuchhaltung ausgeglichen sind. Anschließend kann dieser Einmalkunde wieder aus dem System entfernt werden.

### Feld E-Mail 
Hier wird die E-Mail-Adresse angegeben. Bei der Angabe mehrerer Adressen, müssen diese durch Semikolon getrennt werden. 

Beim Import von E-Mails, beim Vergleich / der Prüfung der E-Mail-Adressen, werden Leerzeichen am Ende und Beginn ignoriert. So können Absender leichter ermittelt werden und die E-Mail wird problemlos importiert. Ebenso werden bereits beim Speichern neuer oder geänderter E-Mail-Adressen automatisch die unnötigen Leerzeichen am Beginn und Ende der Mail-Adresse entfernt. 

Sammelverteiler wie z.B. Mail-Adresse Mahnung im Kundenstamm sind davon ausgenommen. 

Um sicherzugehen, dass eine E-Mail an die richtige Adresse gesendet wird, prüft eNVenta beim Speichern oder Ändern einer Mailadresse, ob die Mail-Adresse der „Norm“ entspricht. Falls nicht wird ein Hinweis ausgegeben. Dabei wird das Speichern nicht grundsätzlich verhindert. Auf diese Weise werden z.B. Outlook-Adressbucheinträge wie z.B. „#EW_ERP“ gespeichert und genutzt. Die Auswertung des Eintrags erfolgt wie gewohnt im Outlook Client. 

Dabei läuft die Prüfung ausschließlich bei Anlage und Änderung einer Mail-Adresse. Werden andere Einträge, z.B. des Kundenstammsatzes geändert, so kommt diese Meldung nicht. 

Werden Stammdaten über das Gateway importiert, so werden u.U. fehlerhafte Mailadressen importiert (ggfs. mit Warnung). Bei einem Import unter Verwendung des Jobservers erfolgt dies ebenfalls in Form einer Warnung (im Jobprotokoll). 

Tooltip: 

Angabe der E-Mail-Adresse. 
Bei der Angabe mehrerer Adressen, müssen diese durch Semikolon getrennt sein.

### Feld Kreditlimit 
Hier kann ein Standard-Kreditlimit für den Kunden (in System-Währung) festgelegt werden. Falls der Betrag überschritten wird, erhalten Sie eine Meldung, wenn Sie einen Auftrag zu diesem Kunden erfassen. Sie erhalten die Möglichkeit, eine Begründung für die Überschreitung einzugeben. 

Die absoluten Kopfrabatte werden in der Kreditlimitprüfung nicht berücksichtigt. 

Das Kreditlimit ist immer in "Haus-Währung" anzugeben!

### Checkbox Striktes Kreditlimit 
Ist zusätzlich zu einem Eintrag im Feld Kreditlimit WW der Parameter Strikt gesetzt, können bei Überschreitung des Kreditlimits keine Aufträge erfasst werden. Im Dialog Sachbearbeiter kann für einen Sachbearbeiter die Checkbox Striktes KL gesetzt werden. Dieses übersteuert dann die Einstellung des strikten Kreditlimits aus den allgemeinen Parametern, wenn dies nicht gesetzt ist.
