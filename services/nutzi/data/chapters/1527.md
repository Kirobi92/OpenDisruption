# Erweiterte Technische Entladung 
Notwendige, technische Entladungsmittel können auch in den Stammdaten des Artikels hinterlegt werden. Dazu erreichen Sie den Dialog Techn. Entl. Zuordnung im Artikel >> Stamm >> Techn. Entl. Zuordnung . 

Dazu gibt es Funktionalitäten auf Artikel(haupt-)gruppen und Artikelebene. Für jede Ebene können eine oder mehrere technische Entladungsinformationen zugeordnet werden. Eine Zuordnung auf Artikelebene „sticht“ eine Zuordnung auf Artikelgruppenebene. Eine Zuordnung auf Artikelgruppenebene „sticht“ ihrerseits eine auf Hauptgruppenebene. 

Um die technische Entladung aus dem Dialog Artikelgruppe aufrufen zu können, steht der Button Tech. Entladungsmittel zur Verfügung, um in den Dialog Techn. Entl. Zuordnung zu gelangen. Vor dem Öffnen des Dialogs muss eine Artikelgruppe eingelesen sein. 

Bevor sich im Artikelstamm der Dialog Tech. Entl. Zuordnung öffnet, ist zwingend ein Artikel auszuwählen! 

Siehe Kapitel Technische Entladungszuordnung . 

Ist eine technische Entladung zwingend vorgeschrieben, so werden solche Artikel nur auf passende Touren mit entsprechend ausgestattetem Fahrzeug eingetourt. Diese Zuordnungen werden automatisch aus dem Artikelstamm übernommen und sorgen, je nach Konfiguration, für eine abweichende Tourfindung auf Positionsebene. 

Auf diese Weise können z.B. SHK- und Stahlartikel in einem Auftrag erfasst, aber automatisch in unterschiedliche Touren eingetourt werden, da für Stahlartikel z.B. ein Kran benötigt wird. 

Wählt der Sachbearbeiter im Auftragskopf manuell eine Tour aus, die nicht zur technischen Entladung eines Artikels im Auftrag passt, wird diese Tour nicht auf den Artikel übertragen. 

Wird für einen Artikel direkt eine Tour ausgewählt, die aufgrund ihrer technischen Entladung nicht zum Auftrag passt, wird dies verweigert. 

Wenn an einer Tour kein Fahrzeug hinterlegt ist, können alle Vorgänge dieser Tour zugeordnet werden. Es erfolgt keine Prüfung gegen technische Entladungsmittel! 

### Ladehilfsmittel (Transportgeräte) in den Stammdaten hinterlegen 
Zur Entladung oder Abholung einer Transportaufgabe werden ggfs. Transportgeräte benötigt. Dazu können Fahr-, Hebe- und Tragehilfen (z.B. Kran, Gurte etc.) verwendet werden. Über diese Liste können Sie die bei Ihnen verwendeten Transportgeräte pflegen. Ebenso können über die technischen Entladungen Einschränkungen bzw. Eigenschaften des Fahrzeugs definiert werden, damit die Entladung am Zielort sichergestellt wird. 

Nicht jede Abhol-/Abladestelle des Kunden/Lieferanten ist mit jedem Transportgerät ausgestattet. Daher muss bei der Zuweisung von Vorgängen auf etwaige Transportgeräte der jeweiligen Abhol-/Abladestelle Rücksicht genommen werden. Dies ist essentiell für eine optimale Tourendisposition. 

Beispiel: 
- Das Fahrzeug darf nur eine bestimmte Länge haben. - Bei der Abladestelle xy kann nur seitlich entladen werden. 
Um hier unabhängiger zu werden, gibt es für die technische Entladung in den Stammdaten (Kunde, Versandanschrift, Projekt, Lieferant, Bestellanschrift) Kategorien. Diese stehen in der Combobox Kategorie zur Verfügung, mit den Ausprägungen 

- optional (0) - notwendig (1) - darf nicht (2) 
Die Überprüfung der gültigen Zuordnungen aus Stammdaten und Tour erfolgt immer zum Zeitpunkt der Eintourung eines Vorgangs. 

### Kundenstamm 
Die technische Entladung kann an den Adressen des Kunden definiert werden. 

Dazu stehen folgende Ebenen zur Verfügung: 

- Kundenstamm - Versandanschrift - Projekt 
In den genannten Ebenen müssen eine oder mehrere technische Entladungsinformationen zugeordnet werden. 

Eine Zuordnung auf Versandanschrift übersteuert eine Zuordnung auf Kundenebene. 

Diese Zuordnungen müssen automatisch in den Auftrag und die Retoure übernommen werden. Sie sorgen dort je nach Konfiguration für eine abweichende Tourenfindung. 

Dazu steht im Kundenstamm auf der Registerkarte Logistik der Button TE-Zuordnung zur Verfügung. Dort können dem Kunden die Ladehilfsmittel zugeordnet werden. 

### Versandanschrift 
Dazu steht in der Versandanschrift der Button TE-Zuordnung zur Verfügung. Dort können die Transportgeräte dem Kunden zugeordnet werden. 

### Projekt 
Dazu steht im Projekt der Button TE-Zuordnung auf der Registerkarte Anschrift zur Verfügung. Dort können die Transportgeräte dem Projekt zugeordnet werden. 

### Auftrag 
Im Auftrag kann die technische Entladung mit einer manuellen Änderung der Lieferadresse überschrieben werden. 

Beim Kopieren eines Auftrags in einen anderen Auftrag mit demselben Kunden, wird die technische Entladung mitkopiert. 

### Lieferantenstamm 
Es ist möglich, die technische Entladung an den Adressen des Lieferanten zu definieren. 

Dazu gibt es folgende Ebenen: 

- Lieferanten - Bestellanschrift - Projekt 
In den Ebenen müssen eine oder mehrere technische Entladungsinformationen zugeordnet werden können. 

Eine Zuordnung auf Bestellanschrift "sticht" eine Zuordnung auf Lieferantenebene. 

Diese Zuordnungen werden automatisch in die Bestellung übernommen und sorgen dort je nach Konfiguration für eine abweichende Tourfindung bei einer Abholung. 

Dazu gibt es im Lieferant auf der Registerkarte Warenwirtschaft den Button TE-Zuordnung . Dort können die Transportgeräte dem Lieferanten zugeordnet werden. 

### Bestellanschrift 
In der Bestellanschrift steht der Button TE-Zuordnung zur Verfügung. Dort können die Transportgeräte dem Kunden zugeordnet werden. 

### Technische Entladung vom Lieferant in Verkaufsvorgänge 
Bei der Übernahme der Materialbeistellung aus der Retoure muss die technische Entladung aus dem Lieferantenstamm in den Auftragskopf vom Auftraggeber übernommen werden. 

Erzeugen einer Materialbeistellung aus der Bestellung (Optionen Materialbeistellung / Materialrücksendung ). 

Drucken der Materialrücksendung aus der Belastung (Optionen Materialbeistellung / Materialrücksendung ). 

### Tourendisposition 
In der Tourendisposition wird bei der Eintourung eines Vorgangs zu einer Tour die Zuordnung der technischen Entladung geprüft. Hat z.B. eine Lieferadresse die Kennzeichnung „Kran“ als notwendig deklariert, so muss das Fahrzeug der Tour über das technische Entladungsmittel „Kran“ verfügen, ansonsten erfolgt keine Einplanung auf diese Tour. Umgekehrt wenn die Lieferadresse das Kennzeichen „seitliche Entladung“ als „Darf nicht“ hinterlegt hat, so darf diese Eigenschaft nicht auf dem Fahrzeug hinterlegt sein. 

In der Tourendisposition wird die Kategorie aus Transportauftrag/Bestellung/Retoure und der technischen Entladung in der Versand-/Abholadresse verglichen. Ist die technische Entladung im Vorgang notwendig, aber an der Adresse vorhanden, wird die technische Entladung als optional bewertet. Damit muss die technische Entladung nicht dem Fahrzeug zugeordnet sein. Ist die technische Entladung an der Versand-/Abholadresse nicht vorhanden, muss die technische Entladung auch am Fahrzeug zugeteilt sein. Sonst wird der Vorgang nicht zugeteilt. 

Folgende Zuordnungen sind hierbei grundsätzlich möglich: 

Adresse / Fahrzeug Optional Notwendig darf nicht Keine Angabe 
| | | | | |
| --- | --- | --- | --- | --- |
| Optional | x | x | - | x |
| Notwendig* | x | x | - | x |
| darf nicht | - | - | x | x |
| Keine Angabe | x | x | - | x |

* Notwendig = Wenn im Auftrag eine technische Entladung hinzugefügt wird z.B., weil ein Artikel dies erfordert, so muss an Fahrzeug oder der Lieferadresse die gleiche technische Entladung mindestens als optional vorhanden sein, damit der Auftrag auf die Tour eingeplant wird.
