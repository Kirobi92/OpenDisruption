# IKOMSYS - Schnittstelle EUROBAUSTOFF 
IKOMSYS 

Unter System >> Allgemein >> Schnittstellen >> EUROBAUSTOFF >> IKOMSYS finden Sie den Dialog IKOMSYS . 

Die Bestellungen werden über das Internet (HTTPS-POST-Request) an eine definierte Adresse übermittelt, wobei die einzelnen Zentralläger (MDT) als Lieferanten angelegt sind. 

Hierzu ist es möglich, für die Schnittstelle die definierte Service-Adresse, den Benutzernamen und das Passwort anzugeben. 

Das Versenden der Bestellung an das EBZL (= EUROBAUSTOFF Zentrallager) erfolgt über den EDI Adapter. Dafür ist es möglich ORDERS-Nachrichten an den EBZL-Webservice zu versenden. 

Der Button Einheiten verzweigt in den Dialog ISO-Einheiten zur Umsetzungstabelle für Datenim- und –exporte zwischen EUROBAUSTOFF und eNVenta . 

Der Button Test prüft die Übertragung einer EUROBAUSTOFF Bestellung. 

Per X-Mapping Editor kann eine Text-Mapping-Konfiguration vorgenommen werden. Das erste Mapping zieht, wenn kein Text-Mapping angegeben wurde. 

In diesem Zusammenhang gibt es im Dialog EDI Teilnehmer das Nachrichtenformat Text . Außerdem steht die Combobox Übertrag-Kanal zur Verfügung, welche die Verbindung zu IKOMSYS schafft. 

In den Picklisten gibt es dazu den Eintrag EUROBAUSTFOFF-Zentrallager. 

Nunmehr kann die Bestellung (ORDERS) über den EDI-Adapter exportiert werden. 

Siehe Kapitel EDI Teilnehmer . 

### Elektronischer Datenaustausch von Rechnungsdaten 
Ein Gesellschafter bestellt (ORDERS) per Webservice über das EUROBAUSTOFF-Zentrallager (EBZL). 

Das Zentrallager stellt elektronische Rechnungsdaten bereit: 
- Eine EDI INVOICE im EDI- oder ASCII-Format - Die Rechnung als PDF oder TIF Datei für die Archivierung - Eine INDEX-Datei mit den Metadaten für die PDF-/TIF Datei 
(Information: welche PDF/TIF-Datei gehört zu welcher Rechnung) 

Die EDI Datei im EDI- oder ASCII-Format wird normal über den INVOICE-Import des EDI-Adapters importiert. Dabei wird der X-Mapper genutzt, um ein Mapping zwischen Prozess-Daten für die Rechnungsprüfung und dem EDI- oder ASCII-Dokument herzustellen. Der Import läuft analog zum EDI XML INVOICE Import. Allerdings kann eine Nachricht mehrere Rechnungen enthalten. Es wird daher die Nachricht INVLST benötigt. 

Außerdem stellt das EBZL auch PDF-Rechnungsdaten für die Archivierung bereit. Dafür gibt es eine Textdatei, die Informationen für die Verschlagwortung und die Namen der Dateien für die Archivierung enthält. 

### Elektronischer Datenaustausch DESADV 
Über die Schnittstelle EUROBAUSTOFF ist auch ein elektronischer Datenaustausch von DESADV (Lieferavis), d.h. Lieferscheindaten, möglich. Das Zentrallager stellt elektronische Lieferscheindaten im EDI- oder ASCII-Format bereit. Der Import erfolgt über den EDI Adapter mit Hilfe des X-Mappers.

### Button Test 

Dieser Button prüft die Übertragung einer EUROBAUSTOFF Bestellung (Bestellung muss davor ausgewählt werden). 

Die Übertragung wird im Modus "Dateitest" durchgeführt, d.h. der Server meldet, ob die übertragene Bestellung gültig ist oder nicht. 

Durch diesen Test werden natürlich auch die Zugangsdaten mitgeprüft. 

Tooltip:
