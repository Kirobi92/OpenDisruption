# Mahnparameter in der Finanzbuchhaltung 
Parameter Mahnen 

Dieses Menü öffnet sich im Navigator >> System >> Finanzbuchhaltung >> Parameter Mahnen oder über den Button Parameter im Dialog Mahnwesen . 

In Abhängigkeit von der Mahngruppe hinterlegen und definieren Sie hier alle individuellen Funktionen, welche beim Mahnlauf zum Tragen kommen sollen. Anhand der Fälligkeit der OP´s und der hier definierten Kriterien werden dann die Rechnungen zur Mahnung herangezogen. Eine Mahngruppe ist entsprechend im Kundenstamm des Kunden zu hinterlegen, damit dieser im Mahnwesen gemahnt werden kann. Es ziehen dann auch nur die Funktionen, welche in dieser Mahngruppe definiert wurden! Außerdem benötigt der Kunde im Kundenstammsatz auch noch den Parameter Mahnen , damit die eindeutige Klassifizierung für das Mahnwesen vorgenommen ist. Es können bis zu fünf Mahnstufen mit Texten hinterlegt werden. Stufe 1 bis 3 werden separat festgelegt. Im Feld Übergabe wird dann der Zeitraum in Tagen eingetragen, ab wann die vierte und jede weitere Mahnstufe übergeben werden soll. In diesem Fall werden die Mahngebühren aus dem Feld Gebühren der 3. Mahnung gezogen. 

Hier werden auch die Mahntexte in allen angelegten Sprachen hinterlegt. 

Im Block Karenzzeit + Gebühren können Sie die Fristen für die jeweiligen Mahnstufen sowie die zugehörigen Gebühren bestimmen.

### Feld E-Mail für Test 
Bei Eintrag einer E-Mail-Adresse werden im Mahnlauf hinterlegte Kunden-E-Mail-Adressen mit dieser E-Mail-Adresse übersteuert / ersetzt. 

Um sicherzugehen, dass eine E-Mail an die richtige Adresse gesendet wird, prüft eNVenta beim Speichern oder Ändern einer Mailadresse, ob die Mail-Adresse der „Norm“ entspricht. Falls nicht wird ein Hinweis ausgegeben. Dabei wird das Speichern nicht grundsätzlich verhindert. Auf diese Weise werden z.B. Outlook-Adressbucheinträge wie z.B. „#EW_ERP“ bei Bedarf gespeichert und genutzt. 

Dabei läuft die Prüfung ausschließlich bei Anlage und Änderung einer Mail-Adresse. Werden andere Einträge, z.B. des Kundenstammsatzes geändert, so kommt diese Meldung nicht. 

Werden Stammdaten über das Gateway importiert, so werden u.U. fehlerhafte Mailadressen importiert (ggfs. mit Warnung). Bei einem Import unter Verwendung des Jobservers erfolgt dies ebenfalls in Form einer Warnung (im Jobprotokoll). 

Tooltip: 

Geben Sie eine E-Mail-Adresse zum Test ein.

### Feld Mindestbetrag 
In diesem Feld kann eine Untergrenze festgelegt werden, ab der ein Kunde gemahnt wird. Dieser wird in der Landes- bzw. Hauswährung festgelegt. Bei Beträgen in Fremdwährungen wird auf den parallel geführten Betrag in der Landeswährung zurückgegriffen und dann ab dem entsprechenden Mindestbetrag in der Fremdwährung gemahnt. Es können also keine individuellen Mindestbeträge für jede Währung festgelegt werden. 

Tooltip: 

Mindestbetrag: Ab diesem Betrag wird die Mahnung angestoßen.

Hier geben Sie die Anzahl an Ausdrucken für Mahnungen an.

### Feld Mahntext 
In diesem Feld sind für jede Sprache eigene Mahntexte zu hinterlegen, da von allen Konten aus auf die sprachspezifischen Mahntexte zurückgegriffen wird. 

Für jede Mahnstufe können individuelle Texte vergeben werden. Wählen Sie den Radiobutton für die Mahnstufe an und geben Sie einen neuen Text ein. 

Über den Button Txt löschen können Sie den Text im Feld Mahntext löschen. 

Wenn für eine Mahnstufe kein Mahntext eingetragen ist, kann auch kein Mahnformular in der entsprechenden Mahnstufe erstellt werden. 

Vorsicht: Wenn Sie im Block Mahntexte in der Combobox ein Land mit Fremdwährung auswählen, dürfen Sie deshalb nicht im Block Zinsen + Mindestbetrag + Druckoption fremdwährungsspezifische Einstellungen vornehmen. Alle Veränderungen dort gelten dann auf Basis der Landeswährung!
