# CSC-Bereinigungen 
CSC-Bereinigungen 

Dieser Dialog öffnet sich im Update-Assistent über den Button CSC-Bereinigung . Hier wird die Analyse gestartet. Dieser Vorgang kann mehrere Minuten dauern. Anschließend werden alle Bereinigungsmaßnahmen aufgelistet. 
## Steuerelemente des Formulars 

### Button Ausführen 

Mit diesem Button werden die Bereinigungsmaßnahmen durchgeführt. 

Dieser Vorgang löscht alle Einstellungen, die mit der aktuellen eNVenta -Version nicht mehr nutzbar sind. Die Daten für eine ältere eNVenta -Version gehen damit verloren. 

Dieser Vorgang überträgt aber auch Einstellungen in das neue Layout-System, die ansonsten nicht nutzbar wären. 

Dialoge, die vom Entwickler noch nicht auf das neue Layout-System umgestellt wurden, können nicht bereinigt werden. Der Bereinigungsvorgang überspringt diese. Nach Umstellung des Dialogs kann der Bereinigungsvorgang erneut gestartet werden und somit auch dieser Dialog bereinigt werden. 

Übersicht der beständigen Eigenschaften 

Im Folgenden wird aufgelistet, welche Einstellungen im Bereinigungsvorgang erhalten bleiben. 

Sollte eine Beschriftung (Label) nicht eindeutig einem Control zugeordnet werden können (z.B. wegen Abweichung von den Design-Richtlinien), können Einstellungen für diese Beschriftung unter Umständen verloren gehen. 

Erhalten bleiben Einstellungen an den folgenden Eigenschaften (Properties): 

- Foreground - Background, Background 2 - GridColor - Text - ToolTipText - Visible - Enabled - Shortcut - Default Value - Required - Hinzugefügte Grid-Spalten - ShowRootHanles - Font eingeschränkt: nur bold (fett) und italic (kursiv) 
Nicht erhalten bleiben Einstellungen an den folgenden Eigenschaften (Properties): 

- Location - Size - Position (Position von Gridspalten) - Opaque - TabIndex - Zorder - HorizontalAlignment, VerticalAlignment 
Alle diese Eigenschaften sind im neuen Layout-System irrelevant.
