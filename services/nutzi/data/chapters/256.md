# E-Mail senden 
E-Mail senden 

Wenn Sie in der Tabelle des Dialogs Vorlagenauswahl auf einen E-Mail-Eintrag doppelklicken, öffnet sich der Dialog E-Mail senden . Hier wird Ihnen der Text der gewählten Vorlage angezeigt. Sie haben die Möglichkeit einen Text für diese E-Mail zu erfassen. 

Um den Adressdialog von Microsoft Outlook anzeigen zu können, ist Voraussetzung, dass die sogenannten "Collaboration Data Objects" (CDO) installiert sind. Ab Outlook 2007 werden diese bei der Installation nicht mehr standardmäßig mit installiert. Um eine E-Mail aus einer Vorlage versenden zu können, muss das Add-In installiert sein. Danach erscheint der Dialog mit der Auswahl der Mail-Adressen aus dem Outlook-Adressbuch. 

Es darf nicht vorkommen, dass beim Versand einer E-Mail an Personen, welche die gleiche E-Mail-Adresse haben, der falsche Ansprechpartner gezogen wird. Damit das funktioniert, ist es notwendig, erst eine E-Mail-Vorlage zu wählen und anschließend die Empfänger-Adresse zu setzen. 

Vor dem Senden einer E-Mail erfolgt eine Validierung der E-Mail-Adressen, damit schon der Versuch eine solche E-Mail zu verschicken, verhindert wird. Aufgrund dessen ist die Eingabe von Kürzeln oder ähnlich hinterlegten Adressen im lokalen Outlook nicht zulässig. 

Nach dem Versand eines Pakets kann dem Kunden eine Tracking-Nummer, bzw. der zugehörige Link in einer E-Mail gesendet werden. Diese Tracking-Nummer wird im Dialog Versandarten in der Spalte 'Paketverfolgung URL' hinterlegt. Dafür lässt sich bei Bedarf auch eine entsprechende E-Mail Vorlage generieren. 

Mögliche Ereignisse (Events) für die Versendung einer Tracking-Nummer sind 
- Versand einer E-Mail über die Benachrichtigungen im Auftrag (Registerkarte Abschluss ) - Versand einer E-Mail aus dem Event Management (Gateway-Event „Lieferschein gedruckt“).

Beim E-Mail-Versand können auch Dokumente direkt aus dem Archiv versendet werden. Dazu gibt es im Menü E-Mail-Vorlagen ein Feld, um den Dateipfad anzugeben. 

Der Anhang einer E-Mail kann gelöscht werden. Beim Speichern des Dialogs wird der Anhang dann endgültig gelöscht. 

Beim E-Mail-Versand werden immer die Variablen innerhalb der Mail-Vorlage gefüllt. 

Dazu gibt es im Dialog Archiv-Mappe ein Feld, welches den Archivpfad des Dokuments angibt (wird nach Doppelklick bzw. öffnen des Archivdokuments gefüllt). Dieser Pfad kann in eine E-Mail übernommen werden. 

Der Pfad des Archivdokuments kann im Feld Anhang eingefügt werden, in diesem Fall wird eine Kopie des Archivdokuments versendet. 

Um einen Anhang zu löschen, steht Ihnen der Button zur Verfügung. Dazu ist es notwendig, diesen zu markieren. Vor dem endgültigen Löschen erscheint eine Meldung, die Sie fragt, ob Sie den Anhang auch wirklich löschen möchten. 

Wenn der E-Mail-Versand nicht möglich ist, muss u.U. im Outlook der Zugriff durch eNVenta ERP gestattet werden. Um Zugriff zu erlauben, ist Voraussetzung, dass z.B. die Software Advanced Security for Outlook installiert ist und im Outlook die Aktionenliste geöffnet wird ( Extras >> Advanced Security >> Aktionenliste ). Hier müssen Sie den Zugriff für die „Remote Procedure Call Runtime“ (C:Windowssystem32rpcrt4.dll) erlauben (Rechtsklick auf diesen Eintrag >> Aktion >> Zugriff erlauben ). 

Im Dialog Meldung erfolgt ein Eintrag, wenn der Versand erfolgreich war bzw. Fehler während des Versands aufgetreten sind. 

E-Mail Vorlagen können auch im Text-Wizard generiert werden. 

Im Text-Wizard auf der Registerkarte Text wird der Textaufbau angezeigt. 

Im Text-Wizard auf der Registerkarte Vorschau erhalten Sie eine Übersicht über den Aufbau der E-Mail-Vorlage. Als Datengrundlage dienen die aktuellen Daten aus dem Vorgang, aus dem der Text-Wizard aufgerufen wurde. 

Im Text-Wizard auf der Registerkarte Struktur wird der Aufbau der E-Mail angezeigt.

### Feld Absender 
Hier wird die E-Mail-Adresse angegeben. Bei der Angabe mehrerer Adressen, müssen diese durch Semikolon getrennt werden. 

Beim Import von E-Mails, beim Vergleich / der Prüfung der E-Mail-Adressen, werden Leerzeichen am Ende und Beginn ignoriert. So können Absender leichter ermittelt werden und die E-Mail wird problemlos importiert. Ebenso werden bereits beim Speichern neuer oder geänderter E-Mail-Adressen automatisch die unnötigen Leerzeichen am Beginn und Ende der Mail-Adresse entfernt. 

Sammelverteiler wie z.B. Mail-Adresse Mahnung im Kundenstamm sind davon ausgenommen. 

Um sicherzugehen, dass eine E-Mail an die richtige Adresse gesendet wird, prüft eNVenta beim Speichern oder Ändern einer Mailadresse, ob die Mail-Adresse der „Norm“ entspricht. Falls nicht wird ein Hinweis ausgegeben. Dabei wird das Speichern nicht grundsätzlich verhindert. Auf diese Weise werden z.B. Outlook-Adressbucheinträge wie z.B. „#EW_ERP“ gespeichert und genutzt. Die Auswertung des Eintrags erfolgt wie gewohnt im Outlook Client. 

Dabei läuft die Prüfung ausschließlich bei Anlage und Änderung einer Mail-Adresse. Werden andere Einträge, z.B. des Kundenstammsatzes geändert, so kommt diese Meldung nicht. 

Werden Stammdaten über das Gateway importiert, so werden u.U. fehlerhafte Mailadressen importiert (ggfs. mit Warnung). Bei einem Import unter Verwendung des Jobservers erfolgt dies ebenfalls in Form einer Warnung (im Jobprotokoll).
