# Clustering der Daten 
Da die optimale Anzahl der Cluster vom jeweiligen Kundendatenbestand und Datenumfang abhängig ist, sollte diese zunächst evaluiert werden. Hierzu werden anhand der ermittelten Kundenprofile (Clustering der Kunden) der Gesamtumsatz durch die Angabe maximaler Bedarfsgruppen (entspricht hier den Artikelgruppen) geteilt. Daraus ergibt sich z.B. bei der Angabe von 100, dass alle Artikelgruppenumsätze ermittelt werden, die mehr als 1% zum Gesamtumsatz beigetragen haben. Anschließend werden die wichtigsten Artikelgruppen ermittelt, deren jeweiliger Umsatz größer ist als der Anteil der Bedarfsgruppen. Also hier alle Artikelgruppen, welche mehr als 1% zum Gesamtumsatz beigetragen haben. Die Anzahl der Artikelgruppen variiert entsprechend. 

Neben der Angabe der maximalen Bedarfsgruppen (Artikelgruppen) müssen Sie noch angeben, in wie viele Cluster die Kunden eingeteilt werden sollen. Die Cluster sollen möglichst gleichartige Kunden auf deren Einkaufsverhalten einteilen. Die Einteilung erfolgt dabei über die Umsätze der jeweiligen Artikelgruppe. 

Voraussagen Registerkarte Bedarfscluster 

Die Einteilung der Kunden in Cluster erfolgt hierbei anhand des eNVenta Machine Learning. Dieser Wert kann über die Anzahl Max. Cluster beeinflusst werden. Dazu kann in das Feld Max. Cluster im Dialog Voraussagen eine maximale Zahl eingegeben werden (z.B. 80). Nach Klicken des Buttons Evaluieren wird nach einigen Minuten Wartezeit eine Graphik (siehe oben) angezeigt. Der optimale Wert wird durch das rote Kreuz markiert und stellt die maximale Entfernung eines Punktes auf dem Bogen zur Sehne dar. 

Der Wert kann dann wieder in das Feld Max. Cluster eingegeben werden. Nun kann ein Job zur Generierung dieser Cluster gestartet werden (Button Job Server auf der Registerkarte Bedarfscluster einmalig, sofort) oder im Kopf des Dialogs für eine regelmäßige Generierung mit diesen Daten. 

Achtung: Wird der Wert Max. Cluster für die Evaluation zu hoch angegeben, bricht die Evaluation nach einiger Zeit mit einer Fehlermeldung ab. Bitte mit einem niedrigeren Wert noch einmal versuchen. 

Achtung: In JEDEN Kundenstammsatz wird die ermittelte Clusternummer oder NULL (Feld ClusterID ) eingetragen und der Stammsatz wird gespeichert. Sollte es einen oder mehrere Stammsätze mit inkonsistenten Daten geben, ist ein Speichern nicht möglich. Die Verarbeitung bricht ab. Die Fehlermeldung gibt Aufschluss über die Ursache. Der Vorgang muss nach Datenbereinigung erneut gestartet werden. 

Die so gewonnene Cluster-Zuteilung eines Kunden kann nun für verschiedene Zwecke genutzt werden. eNVenta Analytics nutzt dies für 

- Kundenspezifische Produktempfehlungen (Recommendation Engine) - Kundenspezifische Preisvorschläge (Price Prediction)
