# Voraussagen 
Voraussagen 

Dieser Dialog öffnet sich im Navigator >> Voraussagen . 

Hier geben Sie die Verbindungsdaten zu eNVenta ML an. 

Der Ziel-Pfad ist das Speicherverzeichnis der ermittelten Daten. Dieser Ziel-Pfad muss zwingend der Unterordner wwwrootData des eNVenta ML -Diensts sein. Am besten hinterlegen Sie im Server Datei-Manager eine entsprechende neue Verbindung, welche direkt in den Unterordner Data von eNVenta ML verweist. 

Die http-Basisadresse ist die Verbindung zum eNVenta ML -Dienst. Die Adresse wird Ihnen direkt in der Konsole der Anwendung angezeigt. 

Noch bevor der Kunde den Kontakt zu Ihrem Unternehmen als seinem bisherigen Lieferanten abbricht, sollten Sie mögliche Anzeichen für seine Abwanderungstendenzen erkennen. Oft lässt sich dann die Kundenabwanderung durch Gegenmaßnahmen verhindern. Kundenrückgewinnung beginnt vor dem Kundenverlust. 

Die Verhinderung von Kundenabwanderung zahlt sich allein deswegen aus, weil die Kosten für die Neukundenakquise sehr hoch sind. Wer die Ursachen für die Kundenabwanderung kennt, kann nicht nur die Abwanderung einzelner Kunden verhindern. Auch langfristig kann die Qualität des eigenen Angebots dadurch verbessert und exakt auf die Bedürfnisse der Kunden abgestimmt werden. 

Moderne Methoden der Datenanalyse, verbunden mit maschinellem Lernen können helfen, Wechselstimmungen zu erkennen. Sind abwanderungswillige Kunden identifiziert, ist es wichtig sie zu qualifizieren. Nicht bei allen Kunden mit Wechselvorhaben lohnt sich der Aufwand, um sie zu halten, gleichermaßen. Aus diesem Grund muss die berechnete Abwanderungswahrscheinlichkeit mit weiteren Kennzahlen ergänzt werden, die eine Entscheidung erleichtern. 
## Steuerelemente des Formulars 

### Button Form-Menü 

Mit Klick auf diesen Button werden je nach Kontext und Berechtigung die unterschiedlichen Optionen des jeweiligen Dialogs hinter dem Button zur Auswahl angeboten. 

Die Funktionen für das Client Side Customizing sowie die Online-Hilfe stehen über diesen Button ebenfalls zur Verfügung. 

## Menüeinträge 

### Menü Belegfolge 
Über dieses Menü öffnet sich die Belegfolge. Alle Vorgänge der Belegkette inkl. der Druckbelege werden auf einem Zeitstrahl angezeigt. 

Siehe Kapitel Belegfolge . 

### Menü Html Report 
Über dieses Menü öffnet sich der Dialog Html Report . 

Siehe Kapitel Html Report . 

### Menü Html Report Wahl 
Über dieses Menü öffnet sich der Dialog Html Report Auswahl . Dort stehen sämtliche, bereits angelegte Html Reports zur Auswahl. 

Siehe Kapitel Html Report Auswahl . 

### Menü Hilfe 
Über dieses Menü gelangen Sie in die Online-Hilfe und Sie erhalten Informationen zu dem Dialog, aus dem heraus Sie die Hilfe geöffnet haben. 

### Menü Customizing 
Über dieses Menü gelangen Sie ins Client Side Customizing. 

Siehe Kapitel Client Side Customizing . 

### Menü Zusatzfelder konfigurieren 
Über dieses Menü erreichen Sie den Dialog Zusatzfelder definieren . Die dynamischen Zusatzfelder dienen der Ablage zusätzlicher Informationen zum Vorgang, ohne dass eine neue Programmierung notwendig ist. 

Siehe Kapitel Zusatzfelder definieren .

Dieser Button schließt den Dialog. In manchen Fällen werden Sie gefragt, ob Sie die Eingaben / Änderungen noch vorher speichern möchten. 

### Feld Zielpfad 
Hier wird der Pfad für die Datenübergabe an das KI-System angegeben. 

Tooltip: 

Pfad für die Datenübergabe an das KI-System. 

### Button Suche 

Über diesen Button kann der Bild-Pfad gesucht werden. 

Es ist notwendig, dass dieser Pfad auch für den Webshop erreichbar ist, damit der Export der Bilder funktioniert und die Bilder innerhalb des Webshops angezeigt werden können. 

### Feld Http Basisadresse 
Hier wird die Basisadresse des eNVenta ML Service (z.B. http://10.75.20.45/api/) angegeben. 

Tooltip: 

Basisadresse des eNVenta ML Service (z.B. http://10.75.20.45/api/)
