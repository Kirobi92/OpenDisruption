# Parameter Allgemein Registerkarte Allgemein 
Parameter Allgemein Registerkarte Allgemein 

Auf dieser Registerkarte definieren Sie allgemeine Einstellungen, die in eNVenta ERP gelten sollen. 
### Dock Panel Länder-Version 

Hier kann festgelegt werden, welche Länderversion von eNVenta ERP genutzt werden soll. Je nach Wahl werden in einigen Programmbereichen (z.B. Finanzbuchhaltung) andere Berechnungen bzw. Vorgehensweisen ausgeführt. Zur Verfügung stehen: 

- Deutschland - Schweiz - Österreich 

Tooltip: 

Hier legen Sie die Länderversion für die Arbeit in eNVenta fest. Zur Verfügung stehen 
- Deutschland 
- Schweiz 
- Österreich 

### Combobox Aktuelles Jahr 
Hier wird das fürs gesamte System gültige Buchungsjahr ausgewählt. Im Dialog Buchen in der Finanzbuchhaltung kann diese Einstellung jederzeit für den entsprechenden Arbeitsplatz geändert werden und ist nur solange gültig, bis eNVenta ERP verlassen wird. 

Bei abweichendem Wirtschaftsjahr ist das aktuelle Buchungsjahr immer das Jahr, in dem das Geschäftsjahr endet. 

Tooltip: 

Wählen Sie das gültige Buchungsjahr. 

### Feld Erster Monat 
Für ein vom Kalenderjahr abweichendes Geschäftsjahr können Sie hier den Beginn Ihres Geschäftsjahres eingeben. 

Weicht das Geschäftsjahr vom Kalenderjahr ab, so bedeutet der Wert „1“ nicht Januar, sondern ist gleichbedeutend mit dem 1. Monat des Geschäftsjahres (1 = erste Periode des Geschäftsjahres). 

Tooltip: 

Definieren Sie den ersten Monat des Geschäftsjahres. 

### Feld Letzter Monat 
Für ein vom Kalenderjahr abweichendes Geschäftsjahr können Sie hier das Ende (den letzten Monat) Ihres Geschäftsjahres eingeben. 

Weicht das Geschäftsjahr vom Kalenderjahr ab, so bedeutet der Wert „12“ nicht Dezember, sondern ist gleichbedeutend mit dem letzten Monat des Geschäftsjahres. 

Tooltip: 

Definieren Sie den letzten Monat des Geschäftsjahrs. 

### Feld Datum gültig +/- 
Wenn Sie hier eine Anzahl Tage eingeben, haben Sie eine bessere Kontrolle, ob ein Rechnungs- oder Belegdatum korrekt eingetragen ist, bzw. in einem „sinnvollen“ Zeitrahmen liegt. 

Die erfasste Anzahl an Tagen wird vom Systemdatum subtrahiert oder addiert. Liegt das Buchungs-/Belegdatum außerhalb dieses Bereichs erhalten Sie einen Hinweis. Dieser Hinweis kann jederzeit ignoriert werden. 

Dieser Parameter wird in folgenden Dialogen berücksichtigt: 

- Einkauf: Rechnungsprüfung , Felder R-Datum / Valuta , Registerkarte Kopfdaten Feld Buchungsdatum - Verkauf : Massendruck , Felder Valuta und Druckdatum - Fibu: Übernahme , Feld Buchungsdatum - Fibu: Kreditorenvorerfassung (bei Datenübernahme) - Kore: Übernahme Feld Buchungsdatum 

Tooltip: 

Tagesdatum +/- Anzahl Tage 
Mit dieser Angabe haben Sie eine bessere Kontrolle, ob ein Rechnungs- oder Belegdatum korrekt eingetragen ist, bzw. in einem „sinnvollen“ Zeitrahmen liegt. 

Die erfasste Anzahl an Tagen wird vom Systemdatum subtrahiert oder addiert. Liegt das Buchungs-/Belegdatum außerhalb dieses Bereichs erhalten Sie einen Hinweis. Dieser Hinweis kann jederzeit ignoriert werden. 

### Feld Druck Timeout 
Mit einem Eintrag in dieses Feld wird festgelegt, nach wie vielen Minuten ein offener Druck-Job als überfällig betrachtet werden soll. Wenn der Timeout erreicht ist, wird dieser in der Liste der Druck-Jobs als Warnung kenntlich gemacht. Wenn in diesem Feld kein Wert definiert ist, wird ein Druck-Job nie mit einem Timeout versehen. 

Tooltip: 

Timeout für Druck-Aufträge in Minuten 
Nach Ablauf dieser Zeit werden sie als fehlerhaft angezeigt. 

### Feld Anzahl max. 
In diesem Feld legen Sie fest, wie viele Datensätze grundsätzlich in eine Tabelle in eNVenta (z.B. Kunde Registerkarte Tabelle ) eingelesen werden sollen. Auf diese Weise wird Übersichtlichkeit gewährleistet und auch die Zeit zum Einlesen der Daten wird auf ein vernünftiges Maß beschränkt. 

An manchen Stellen in eNVenta ist es möglich, diesen Wert zu überschreiben. 

Wenn Sie dieses Feld leer lassen oder eine 0 eingeben, werden alle Datensätze angezeigt. 

Tooltip: 

Anzahl Sätze, die maximal eingelesen werden. 

### Combobox Reporting-Software 
Hier können Sie die Reporting-Software (Crystal Reports oder DevExpress) auswählen, die für den Druck verwendet werden soll. Falls DevExpress ausgewählt ist, aber kein DevExpress-Report vorhanden ist, findet ein Fallback auf CrystalReports statt. 

Beispiel: 
Ein typischer Anwendungsfall ist die Übertragung von fertig generierten Lieferscheinen im Rahmen des Intercompany-Prozesses für Streckenlieferungen. Ein weiterer Anwendungsfall findet sich in der Lieferantenerklärung, die beim Rechnungsdruck mitgeliefert wird. Dazu muss eine PDF aus einem Report generiert werden. 

Tooltip: 

Auswahl der Reporting-Software (Crystal Reports oder DevExpress), die für den Druck verwendet werden soll. 
Falls DevExpress ausgewählt ist, aber kein DevExpress-Report vorhanden ist, findet ein Fallback auf CrystalReports statt. 

### Checkbox Briefkopf 
Im Rahmen von Geschäftsprozessen ist es immer wieder notwendig, Belege physisch oder digital als Ausdrucke bereitzustellen. Aus rechtlichen Gründen muss dies bei offiziellen Dokumenten mit bestimmten Informationen (Anschrift, BLZ, …) – in Kopf- und Fußzeile - geschehen. Diese Informationen werden im Folgenden als Briefkopf bezeichnet. 

Dies können Kunden nutzen, die ihre Firmeninformationen direkt mit dem Druck des Belegs auf ein Blanko-Papier drucken möchten. 

Die Checkbox Briefkopf dient in den Dialogen als Vorbelegung und im automatisierten Druck als Einstellung für den Druck. 

Im Einzelfall kann in den Druck-Dialogen der einzelnen Vorgänge die Checkbox Briefkopf deaktiviert werden, falls Sie ausnahmsweise auf ein Briefpapier drucken möchten. 

Tooltip: 

Dieser Parameter gilt als Vorbelegung für Druckdialoge und automatische Druckprozesse. 
Wenn aktiv, wird der Briefkopf auf den Beleg gedruckt bzw. die entsprechende Option im Druckdialog vorausgewählt. 

### Combobox Prüfung 
In der Regel sind dem Unternehmen die Lieferadressen der Kunden bekannt. Eine Ausnahme bilden Situationen, in denen dem bestellenden Handwerker die Adresse nicht bekannt ist. Hierzu zählen z.B. Baustellenadressen oder Adressen des Privatkunden, für den der Handwerker Material benötigt. In solchen Situationen kommt es häufiger vor, dass Postleitzahlen genannt werden, die gar nicht zur Straße passen. 

Es kommt ebenso vor, dass der Erfasser des Auftrags die PLZ nach bestem Wissen einträgt, diese aber falsch ist. 

In der Folge wird die Ware mit der falschen Tour gefahren und erreicht den Handwerker nicht wie erwartet. 

Die Daten, die Sie dafür benötigen, erhalten Sie von der Deutschen Post (die Datei heißt „Streetcode“). 

In dieser Combobox stehen 5 Einträge zur Adressprüfung zur Auswahl: 
- Leer - Länderspezifischer Provider - Postleitzahlenspezifischer Provider - Rudimentärer Provider - Straßenspezifischer Provider 
Wird nichts ausgewählt oder deaktiviert, passiert folgendes: 
- Bei Pflege oder Neuanlage einer Anschrift erfolgt die Prüfung gegen den Parameter der Adressprüfung. - Bei der Ermittlung, ob die Lieferadresse gültig ist oder die Rechnungsadresse für die Versendung übernommen werden soll, erfolgt immer mindestens eine rudimentäre Prüfung (Firma 1, Straße, PLZ und Ort), auch wenn die Parameter der Adressprüfung deaktiviert sind! 
In Zusammenhang mit der Checkbox Adresse gültig in den Versandanschriften , prüft die länderpsezifische Prüfung neben den allgemeinen Angaben immer, ob ein LKZ angegeben ist. Sollte kein LKZ angegeben sein, so erfolgt in jedem Fall eine rudimentäre Adressprüfung. 

Siehe Kapitel Adressprüfung in eNVenta . 

Tooltip: 

Hier wählen Sie den Provider für die Adressprüfung aus. 

### Combobox Vervollständigung 
In dieser Combobox stehen die Einträge Postleitzahlspezifischer Provider und Straßenspezifischer Provider zur Verfügung. 

Bei der Angabe des Straßenspezifischen Providers erscheint bei der Eingabe einer ungültigen Adresse der Dialog Adressvervollständigung zur Bearbeitung bzw. Auswahl der korrekten Adresse. 

Bei Auswahl des Postleitzahlenspezifischen Providers erscheint bei der Eingabe einer ungültigen Adresse ebenfalls der Dialog Adressvervollständigung bei nicht korrekter Adresse. Die beiden Dialoge unterscheiden sich vom Inhalt der Datenquelle, haben aber die gleiche Grundfunktionalität zur Prüfung der Daten. 

Die straßenspezifische Adressvalidierung zieht sich durch das ganze eNVenta , überall wo bisher mit der Adressprüfung gearbeitet wird. 

Je nachdem welche Auswahl Sie in dieser Combobox treffen, findet eine andere Art der Adressprüfung statt. 

Siehe Kapitel Adressvervollständigung . 

Tooltip: 

Aus dieser Combobox wählen Sie einen Provider für die automatische Adressvervollständigung aus. 

### Combobox Res. Methode 
Bei der Terminplanung für Ressourcen gibt es die Auswahl, ob ein Betriebskalender geprüft werden soll, welcher die Arbeitszeit vorgibt oder "ohne Kalender" und damit ein 24-Stunden-Tag zur Planung zur Verfügung steht. 

Hier haben Sie die Möglichkeit Ressourcen-Reservierungen in den Betriebsaufträgen, Projektaufgaben und im Serviceeinsatz ohne Berücksichtigung des Betriebskalenders vorzunehmen. Bei Buchungen ohne Betriebskalender werden auch Termine berücksichtigt, welche nachts oder an Sonn- und Feiertagen beginnen bzw. enden. 

- Es wird ausschließlich die Zeitspanne der Projektaufgabe bzw. des Serviceeinsatzes betrachtet. - Dies setzt voraus, dass dann alle Ereignisse wie z.B. Pausen, täglicher Arbeitsbeginn- und Arbeitsende per BDE gebucht werden müssen, um die korrekte Arbeitszeit für die Kalkulation zu erhalten. Jede Ressource hat damit grundsätzlich 24 Stunden Kapazität. Auf diese Weise kann unterschiedlichen Organisationsgraden Rechnung getragen werden. 

Tooltip: 

Mit Auswahl aus dieser Combobobx nehmen Sie Ressourcen-Reservierungen in den Betriebsaufträgen, Projektaufgaben und im Serviceeinsatz ohne Berücksichtigung des Betriebskalenders vor. 

### Combobox Job-Server Grp. 
Hier können Sie eine Job-Server Gruppe auswählen. Wenn der Benutzer oder der Job keiner Job-Server-Gruppe zugeordnet ist, so werden alle neu angelegten Jobs automatisch dieser Gruppe zugewiesen. 

Über die Job-Server-Gruppen können die verschiedenen Jobs mehreren verschiedenen Job-Servern zugewiesen werden, um die Last entsprechend zu verteilen. 

Tooltip: 

Mit Auswahl aus dieser Combobox werden alle neu angelegten Jobs automatisch dieser Gruppe zugewiesen. 

### Checkbox Kunde/Lieferant Konto gleich 
Bei Kunden und Lieferanten werden bei der Neuanlage für die Felder Kontonr und Kunden- bzw. Lieferantennr identische Nummern vergeben. 

Kunden bzw. Lieferanten werden über einen entsprechenden Nummernkreis definiert. 

Diese Einstellung wird empfohlen, da sich sonst der Pflegeaufwand erhöht und keine Übersichtlichkeit gewährleistet ist. 

Das Feld Kontonr ist bei der Neuanlage von Kunden bzw. Lieferanten deaktiviert, da diese automatisch vergeben wird. 

Tooltip: 

Kunde/Lieferant mit Personenkonto gleich(setzen)

### Checkbox Provision für bezahlte OP 
Ist dieser Parameter gesetzt, wird in der Vertreterabrechnung im Verkauf eine Steuerung aktiviert, mit der Sie die Auswertung auf Nur bezahlte Rechnungen setzen können. Dazu ist zwingend die Checkbox Nur bezahlte Rechnungen im Dialog Provisionsauswertung zu aktivieren. 

Weitere Voraussetzung: die Finanzbuchhaltung in eNVenta muss lizenziert sein! 

Tooltip: 

Aktiviert eine Steuerung in der Vertreterabrechnung auf 'Nur bezahlte Rechnungen'.

Diese Checkbox kann auch deaktiviert werden! 

Allgemein und Warenwirtschaft: Findet die Überprüfung einen Fehler, erhalten Sie eine Meldung, können aber trotzdem eine abweichende Nummer eingeben. 

Der verwendete Webservice hat folgende Adresse: 
http://ec.europa.eu/taxation_customs/vies/services/checkVatService 

Der Webservice wird bereits beim Veröffentlichen der Anwendung (Publish) angegeben. 

Finanzbuchhaltung >> Buchen: Die Angabe der UST-ID ist in den Vorgängen Pflicht, falls ein Steuerschlüssel im Vorgang enthalten ist, welcher Keine ZM deaktiviert hat. 

Siehe Kapitel Steuerschlüssel .

Wenn die UST-ID nicht korrekt ist, verweigert das Finanzamt unter Umständen die Anerkennung und es wird zu einer Rückforderung kommen. Bitte wenden Sie sich im Zweifelsfall immer an Ihren Steuerberater.

### Checkbox Cursor in Suchname 
Wenn Sie diesen Parameter aktivieren, wird der Cursor in den Dialogen 
- Auftrag - Angebot - Gutschrift - Rahmenauftrag - Kunde - Vertreter - Lieferant - Bestellung - Anfrage - Rahmenbestellung - Belastung 
nach Öffnen des Dialogs unverzüglich in das Feld Suchname gesetzt, anstatt in die Kundennummer. Da die Kunden- bzw. Lieferantennummer nun als Suchkriterium deklariert werden kann, ergibt sich hierdurch kein Verlust an Suchkomfort. 

Tooltip: 

Setzt beim Öffnen eines Dialogs den Cursor sofort in das Feld 'Suchname'. 

### Checkbox Automatisch Tabelle zeigen 
Wenn Sie diese Checkbox aktivieren, wird bei einer undefinierten Suche, z.B. nach einem Kunden, automatisch auf die Registerkarte Tabelle eines Dialogs gewechselt, wenn mehr als ein Suchtreffer gefunden wurde. So stehen Ihnen alle Treffer unverzüglich zur Auswahl zur Verfügung. 

Diese Funktion ist in jedem Dialog mit der Registerkarte Tabelle verfügbar. 

Tooltip: 

Zeigt nach einer Suche automatisch die Tabseite "Tabelle" bei mehr als einem Treffer. 

### Checkbox Adressfelder zusammenfassen 
Über den Parameter Adressfelder zusammenfassen kann systemweit entschieden werden, ob alle Adressfelder in einem Multiline-Feld zusammengefasst angezeigt werden sollen oder in einer zeilenweise getrennten Anzeige. Bei der Anzeige als Multiline-Feld ist die Bearbeitung der Adresse nur über den Dialog möglich, bei der getrennten Anzeige ist eine Bearbeitung direkt im Dialog und zusätzlich über den Pflege-Dialog möglich. 

Siehe Kapitel Adresse anlegen . 

Tooltip: 

Alle Anschriftfelder werden zu einem Feld zusammengefasst. 

### Checkbox Zoom im Vordergrund öffnen 
Wenn Sie diesen Parameter aktivieren, werden für den Benutzer beim Starten der Anwendung die Zoom-Elemente als erster Dialog angezeigt. 

Dies können vom Benutzer selbst angelegte oder vom Administrator eingerichtete Zoom-Elemente sein, die auf "Automatisch sichtbar" gesetzt sind. 

Wenn über die Favoriten Einträge für den Autostart eingerichtet sind, dann haben diese höhere Priorität und werden trotz dieser Einstellung über den Zoom-Dialog gelegt. 

Ist der Parameter nicht aktiv, wird der Start-Bildschirm von eNVenta als erster Dialog beim Start angezeigt. 

Tooltip: 

Beim Start der Anwendung wird der Zoom-Dialog im Vordergrund angezeigt, sofern es sichtbare Zoom-Elemente gibt.

Siehe Kapitel Erw. Call-Suche . 

Dies funktioniert allerdings nur, wenn ein entsprechender Volltext-Katalog in der Datenbank angelegt wurde. (Tabelle CC_Calltext , Spalte 'Text') 

Siehe Kapitel Volltext-Index Anlegen Microsoft SQL-Server 2005 .

Aktiviert die Volltextsuche im Call Center. 

### Checkbox E-Mail BCC an Versender 
Wenn Sie diesen Parameter aktivieren, ist es möglich den Benutzer beim Email-Versand (Dialog E-Mail senden ) über das Feld BCC in Kopie zu nehmen. Dies erfolgt dann immer, wenn der Versand von Emails über SMTP erfolgt und dies nicht im Dialog Parameter Sachbearbeiter übersteuert wird. Damit ist eine Dokumentation des erfolgreichen Versandes feststellbar (bei einem Versand über das lokale Outlook wäre dies über den Ordner "gesendete Objekte" möglich). 

Siehe Kapitel Parameter Sachbearbeiter . 

Tooltip: 

Per Parameter kann jede E-Mail an den „Versender“ per BCC geschickt werden, 
damit der Versender auch über den Versand informiert wird.

### Feld TeamViewer-URL 
In dieses Feld geben Sie die URL ein für den Download der Fernwartungssoftware Teamviewer. Nach der Eingabe erfolgt eine Prüfung, ob der Pfad bzw. das Ziel erreichbar ist. Wenn das Ziel nicht erreichbar ist, erhalten Sie eine Fehlermeldung und die URL wird zurückgesetzt. 

Der TeamViewer kann im Navigator >> ? >> TeamViewer geöffnet werden. 

Tooltip: 

Hier geben Sie die URL für den Team-Viewer ein. 

### Feld Hilfe-URL 
Hier wird der Pfad für die Online-Hilfe eingetragen, inkl. Platzhalter. 

Platzhalter: 
{0} - Name des Forms incl. Namespace 
{1} - ISO-Code der Sprache 

Beispiel: 
http://NVApp01/eNVentaHelp/eNVenta_{1}_38ws/?load={0} 

Sollte es Probleme mit der Online-Hilfe (z.B. wenn nicht alle Funktionalitäten eines Dialogs angezeigt werden) geben, nachdem Sie das Feld gefüllt haben, empfehlen wir, zunächst den Browser zu leeren! 

Tooltip: 

Geben Sie den Pfad für die Online-Hilfe an.
