# Zoom-Element Produktion 

## Betriebsaufträge in Produktion 

### Inhalt 
Hier werden alle Betriebsaufträge angezeigt, die aktuell in Produktion sind. Dies wird bestimmt durch Startdatum und Bis Datum des Betriebsauftrags, sowie durch den Status "freigegeben" oder "teilgefertigt". 
Zusätzlich wird der erste in Bearbeitung befindliche Arbeitsgang angezeigt (BDE-Buchung erforderlich). 
### Ziel / Nutzen 
Dieses Element bietet in komprimierter Form eine Information/Überblick, was aktuell gerade in der Produktion in Bearbeitung ist. 
### Darstellung 
Die Darstellung erfolgt als Tabelle sortiert nach Priorität, Lieferdatum aufsteigend (älteste zuerst). 
### Parameter 
Es können die Produktionsart und das Produktionslager als Parameter gesetzt werden. 
### Drill Down 
Es wird der ausgewählte Betriebsauftrag geöffnet. 
### Aktualisierung 
Standard / Empfehlung: halbstündlich 
### Berechtigung 
Zugangsberechtigt sind alle Benutzer, die auch das Recht für den Dialog Betriebsauftrag (frmWorkOrder) besitzen. 

## Überfällige Betriebsaufträge 

### Inhalt 
Hier werden alle Betriebsaufträge angezeigt, die überfällig sind. Erkannt werden diese Betriebsaufträge anhand des Lieferdatums und/oder Datum Bis. Die Verzugszeit wird in Tagen angezeigt und folgendermaßen berechnet: 

Sind das Lieferdatum und das Datum Bis in der Vergangenheit wird die Verzugszeit auf das Datum Bis berechnet. Ist nur eines der Daten in der Vergangenheit wird die Verzugszeit auf dieses Datum berechnet. 

#### Ziel / Nutzen 
Auf einen Blick können die größten Verzögerungen und damit die potentiell kritischen Produktionsvorgänge erkannt werden. 
### Darstellung 
Die Darstellung erfolgt als Tabelle, sortiert nach Verzugszeit in Tagen. 

### Parameter 
Es können der Produktionstyp und das Produktionslager als Parameter gesetzt werden. 
### Drill Down 
Es wird der ausgewählte Betriebsauftrag geöffnet. 
### Aktualisierung 
Standard / Empfehlung: einmal täglich 
### Berechtigung 
Zugangsberechtigt sind alle Benutzer, die auch das Recht für den Dialog Betriebsauftrag (frmWorkOrder) besitzen. 

## Engpässe Ressourcen 

### Inhalt 
Hier wird der Auslastungsgrad der Ressourcen angezeigt. Die Ressourcen mit der stärksten Auslastung stehen oben. 
### Ziel / Nutzen 
Die größten Ressourcenengpässe können auf einen Blick erfasst werden. Dies können Maschinen in der Produktion, Servicetechniker, Außendienst-Mitarbeiter etc. sein. 
Es ist zu empfehlen, die Ressourcengruppen zur Abgrenzung anzugeben. 
Durch die Angabe der Anzahl Prozesse kann erkannt werden, ob die Auslastung von einem großen Vorgang oder einer Vielzahl von Vorgängen verursacht wird. 
### Darstellung 
Die Anzeige erfolgt in einer Tabelle. Die Auslastung der Ressource wird als prozentualer Balken angezeigt. Der Wert berechnet sich anhand der Auslastung und des Limits der Ressource. 
Farben: Bis 60% grün, ab 90% rot, sonst gelb. 
Die Anzahl der Einträge ist auf 50 begrenzt. 
### Parameter 
Es können Vorbelegungen für Ressourcengruppe sowie Ressourcentyp getroffen werden. 
Zusätzlich kann ein Zeitraum angelegt werden. Zur Auswahl stehen hier: Heute, Morgen, Woche, nächste Woche, Monat, nächster Monat 
### Drill Down 
Die ausgewählte Ressource wird geöffnet. 
#### Aktualisierung 
Standard / Empfehlung: Einmal täglich. 
### Berechtigung 

Zugangsberechtigt sind alle Benutzer, die auch das Recht für den Dialog Auslastung Ressource (frmUtitlisationResources) besitzen. 

## Abweichungen Betriebsaufträge 

### Inhalt 
Es wird eine Liste der Betriebsaufträge mit den höchsten Soll-Ist-Abweichungen im angegebenen Zeitraum ermittelt. 
Maßgeblich ist das Startdatum der Produktion. Der Betriebsauftrag muss den Status "abgeschlossen" haben. 
Es werden maximal 100 Einträge angezeigt. 
### Ziel / Nutzen 
Auf einen Blick können die größten Kostenabweichungen in der Produktion der letzten Tage erkannt werden. Der direkte Durchgriff per Drill-Down auf die betreffenden Betriebsaufträge ist möglich. Dort kann dann die Ursache in der Detailkalkulation ermittelt werden. 
### Darstellung 
Tabelle sortiert nach dem absoluten Wert der Abweichung absteigend. 

Farben der prozentualen Abweichung: 
grün: <= 0%, gelb <10%, rot >= 10% 
### Parameter 
Es muss der Betrachtungszeitraum in Tagen angegeben werden (Standard: 14 Tage). 
Weitere Einschränkungsmöglichkeiten nach Produktionsarten und Produktionslagern. 
### Drill Down 

Aufruf des Betriebsauftrags. 
### Aktualisierung 

Standard / Empfehlung: täglich 
### Berechtigung 
Zugangsberechtigt sind alle Benutzer, die auch das Recht für den Dialog Kalkulation (frmPPSCalculation) besitzen.
