# Adapter Basisdaten 
Adapter Basisdaten 

Dieser Dialog öffnet sich im Gateway Adapter Kontext über den Button neben der oberen Tabelle. Der Stammdaten Adapter ermöglicht den Import/Export der folgenden Stammdaten: 

- Artikel - Kunden - Lieferanten 
### Konfiguration 
Bitte die exakte Schreibweise des Adapternamens ‚AdapterBaseData‘ beachten. 

### Gateway Adapter Kontext 
Öffnen des Dialogs: 

Markieren Sie in der oberen Tabelle den Eintrag ‚AdapterBaseData‘ und drücken Sie rechts neben der Tabelle den Button . Damit öffnet sich der Dialog Adapter Basisdaten . 

Im Bereich Datenaustausch legen Sie fest, ob Daten importiert oder exportiert werden sollen. 

Im Bereich Datenauswahl legen Sie die Stammdaten für den Datenaustausch fest (Artikel, Kunden oder Lieferanten). 

Der Datenaustausch erfolgt mittels einer CSV-Datei, wobei Sie das Trennzeichen auswählen können (Bereich Trennzeichen: Tab-Stopp, Semikolon, Komma). 

Geben Sie im Feld Datei eine gültige Datei ein bzw. wählen Sie eine Datei über den Button aus. 

Starten Sie den Datenaustausch mit dem Button Ausführen . Nach erfolgter Ausführung erhalten Sie eine Mitteilung, wie viele Datensätze importiert bzw. exportiert worden sind. Detaillierte Informationen über den Vorgang können Sie im Gateway-Protokoll nachsehen (siehe Werkzeuge >> Gateway >> Optionen im Gateway >> Gateway Protokoll ).

### Dock Panel Trennzeichen 

In diesem Bereich wählen Sie das Trennzeichen aus. 

### Checkbox Überschrift ignorieren 
Ist diese Checkbox aktiviert, wird die erste Zeile der CSV-Datei nicht importiert. 

### Checkbox Daten aktualisieren 
Ist diese Checkbox aktiviert, werden vorhandene Daten in eNVenta ERP mit dem Inhalt aus der CSV-Datei überschrieben. 

### Feld Select...Where 
Hier haben Sie die Möglichkeit, die zu exportierenden Stammdaten einzugrenzen. Andernfalls werden immer alle Daten exportiert. Verwenden Sie den Button um komfortabel die SQL-Where-Bedingung zu erstellen (siehe eNVenta ERP >> Allgemeine Funktionalitäten >> Suchfunktionen >> Suchbedingung / Suche mit SQL ).
