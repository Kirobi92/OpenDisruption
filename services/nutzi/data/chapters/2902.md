Pro Knoten in der Zuordnung kann eine Transformation eingestellt werden. Hierbei können auch mehrere Transformationen in Reihe geschaltet werden, z.B. Sonderzeichen entfernen und dann Leerzeichen entfernen. 

Im X-Mapping kann der Ausdruckseditor über eine Transformation aufgerufen werden. 

Dieser Dialog dient zum Erstellen und Bearbeiten von Ausdrücken. Je nachdem über welchen Dialog der Ausdruckseditor geöffnet wird, werden die Spalten entsprechend angepasst angezeigt. Im X-Mapping Editor kann der Ausdruckseditor jedoch nur beim Export genutzt werden. Hier existiert ein eNVenta -Objekt, welches mit Hilfe dieses Editors entsprechend dargestellt werden kann. Beim Import ist das Gegenstück ein XPath-Ausdruck. Hier gibt es kein eNVenta -Objekt, daher muss das XML entsprechend angesteuert werden. 

Der Ausdruckseditor wird genutzt, um an einem eNVenta -Quellobjekt Operationen durchzuführen und als Ergebnis einen einfachen Datentyp (String, Bool, Decimal, ...) zurück zu bekommen.

Transformation der Lieferantennummer bei X-Mapper 

Um Lieferantennummern im System mit denen von Datenlieferanten (z.B. EUROBAUSTOFF) kompatibel zu bekommen, gehen Sie vor wie nachfolgend beschrieben. 

Sie benötigen eine weitere Transformation Import-Schlüssel , welche die automatische Umsetzung von externen Lieferanten-IDs in eNVenta -Lieferantennummern beinhaltet. Diese Umsetzung legen Sie im Dialog Daten Umsetzung fest. 

Ein Zugriff erfolgt über die Import-Schlüssel Verwaltung . Dazu wählen Sie den Stammdatentyp Lieferant aus und hinterlegen den internen Schlüssel (die Lieferantennummer innerhalb eNVenta ERP , den Typ (Klassifizierung gleichartiger Datenherkunft) und den externen Schlüssel (externe ID der Lieferanten, welche gemappt werden). 

Die Import-Schlüssel-Zuordnungen gelten nur für die Richtung Import , d.h. externer Schlüssel zu internem Schlüssel. Jedoch ist es möglich, mehrere externe Schlüssel für denselben internen Schlüssel zu definieren. In diesem Fall wird der erste gefundene Eintrag verwendet. 

Folgende Parameter müssen bei der Nutzung der Transformation hinterlegt werden: 

- Richtung - Interner Schlüssel nach Externer Schüssel oder Externer Schlüssel nach Interner Schlüssel. - Stammdatentyp - Auf welche Stammdaten bezieht sich die Transformation? - Externer Typ - Auf welchen externen Schlüsseltyp bezieht sich die Transformation? - Bei fehlenden Werten - Verhalten bei nicht vorhandenen Zuordnungen: - 'error': Fehler, - 'error if input not empty': Fehler nur, wenn der angekommene Wert nicht leer ist, - 'output = input': der angekommene Wert wird wieder ausgegeben, - 'empty': es wird ein leerer Wert zurückgegeben. 
Wenn kein gültiger Wert angegeben wurde, verhält sich die Transformation wie bei der Option 'error'.

Werte anhängen / Transformation Anhängen (Join) 

Es ist möglich, einer Eigenschaft im X-Mapping Editor Werte aus mehreren Feldern zuzuordnen. Das heißt, dass der bestehende Wert beim Import der Daten nicht ersetzt, sondern der neue Wert hinten angehängt werden kann; getrennt durch ein Trennzeichen. Folgende Trennzeichen stehen derzeit in der Transformation „Anhängen (Join)“zur Verfügung: 

- Für ein Leerzeichen geben Sie „\ “ ein 
( Achtung: bitte genau so, wie es hier vorgegeben ist!). - Für einen Zeilenumbruch geben Sie „\n“ ein. - Für einen Tabulator geben Sie „\t“ ein.
