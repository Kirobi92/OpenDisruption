# Besuchsbericht Registerkarte Calls 
Besuchsbericht Registerkarte Calls 

Auf dieser Registerkarte werden standardmäßig alle Calls zum gewählten Besuchsbericht angezeigt. Über die Radiobuttons Besuch , Vertreter und Kunde kann die Anzeige der Daten eingeschränkt werden. Die angezeigten Calls sind unabhängig von diesem Besuchsbericht. 

Voraussetzung für die Verfügbarkeit der Registerkarte Calls ist, dass das Call Center lizenziert ist. 

Der Status eines Calls kann über die Spalte ‚Status‘ angepasst werden. 
### Tabelle Calls/Aufgaben 

In dieser Tabelle können Calls und Aufgaben angelegt werden. Hier können auch gleich die wichtigsten Daten eingetragen werden. Nach dem Speichern kann der generierte Call / die generierte Aufgabe aufgerufen und bearbeitet werden. 
### Tabellenspalte Call-Betreff 
Hier wird der Betreff des Calls angegeben (Thema). 

Tooltip: 

Angabe des Call-Betreffs (worum gehts?) 

### Tabellenspalte Status 
Hier wird der Status eines Calls angegeben. Zur Verfügung stehen: 
0- nicht gesetzt 
1 - offen 
2 - in Bearbeitung 
3 - erledigt 

Tooltip: 

Angabe des Call-Status 

### Tabellenspalte Kategorie 
Hier wird die Nummer der Call-Kategorie angegeben. 

Tooltip: 

Angabe einer Call-Kategorie 

### Tabellenspalte Priorität 
Hier wird die Call-Priorität angegeben. 

Tooltip: 

Angabe der Call-Priorität 

### Tabellenspalte Erfassung 
Hier werden vom Programm automatisch das Datum und die Uhrzeit der Erfassung eingetragen. 

Tooltip: 

Datum und Uhrzeit der Erfassung 

### Tabellenspalte Erfasser 
Hier wird vom Programm automatisch der Sachbearbeiter eingetragen, der den Datensatz zuletzt geändert hat. 

Tooltip: 

Zeigt den Sachbearbeiter, der als letztes den Datensatz bearbeitet hat. 

### Button Satz hinzufügen 

Mit diesem Button wird eine neue Zeile in der Tabelle zugänglich. 

### Button Löschen 

Mit diesem Button wird ein Datensatz der Tabelle Calls/Aufgaben entfernt. Aber nur solange der Call den Status offen hat. 

### Tabelle WV-Historie 

In dieser Tabelle werden zu jedem Call die entsprechenden Wiedervorlagen historisch angezeigt. Die aktuellste Wiedervorlage steht an erster Stelle. So kann sofort erkannt werden, welchen Bearbeitungsstand der Call hat bzw. bei wem sich der Call zur Bearbeitung befindet. 
### Tabellenspalte Sachbearbeiter 
Hier wird vom Programm automatisch der Sachbearbeiter eingetragen, der den Datensatz zuletzt geändert hat. 

Tooltip: 

Zeigt den Sachbearbeiter, der als letztes den Datensatz bearbeitet hat. 

### Tabellenspalte Nachricht 
Hier wird der Text aus dem Feld Nachricht angezeigt. 

### Tabellenspalte Termin 
Hier können Sie eingeben, wann die Wiedervorlage beim Empfänger ankommen soll. Das Tagesdatum ist voreingestellt. 

### Tabellenspalte Eigner 
Bei der Erfassung eines neuen Termins mit der Kennzeichnung zur Bearbeitung kann eine Zuständigkeit in Form eines Eigners hinterlegt werden. Der Eigner wird immer mit dem aktuellen Sachbarbeiter unzugänglich vorbelegt. 

Wird eine Warnung mit Warnfrist erfasst, kann der Eigner geändert werden. Der Eigner erhält nach Ablauf der Warnfrist eine Wiedervorlage zur Terminüberwachung. 

Der hinterlegte Eigner und die zur Erledigung des Termins vorgegebene Zeit können nachträglich nicht mehr geändert werden. 

Tooltip: 

Zeigt den Sachbearbeiter, der als letztes den Datensatz bearbeitet hat. 

### Button Satz hinzufügen 

Mit diesem Button kann eine Wiedervorlage erzeugt werden. Dabei wird der Betreff des Calls übernommen. 

### Button Satz hinzufügen 

Über diesen Button können Sie einen neuen Eintrag für das Feld Übersicht erstellen.
