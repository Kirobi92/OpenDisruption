# Ressource 
Ressource 

Die Ressourcenverwaltung zählt zu den Basisfunktionalitäten. Mit ihr lassen sich Werkzeuge, Arbeitsplätze, Formen, Vorrichtungen und Prüfmittel auf einfache Weise verwalten. Den einzelnen Ressourcen lassen sich Belastungsgrenzen, Übergangszeiten oder auch Wartezeiten vor und nach der Nutzung zuordnen. Des Weiteren kann die Ressourcenverwaltung auch für die Planung des Technikereinsatzes im Service, der Kapazitäten des Außendienstes, für die Verwaltung von Fahrzeugen oder die Vermietung von Geräten genutzt werden. 

Sie haben nicht nur die Möglichkeit sich diese graphische Darstellung mit dem Menü Ressourcen aufzurufen und zu betrachten, sondern Sie können in diesem Dialog Termin-Änderungen vornehmen. 

Die Ressourcenverwaltung ist allgemein gültig und kann unabhängig von der Produktion für die unterschiedlichsten Zwecke zusammen mit dem Betriebskalender eingesetzt werden. 

Jede Ressource erhält eine eindeutige, vom System fortlaufend automatisch vergebene Nummer. Eine Ressourcennummer kann auch manuell vergeben werden. 

Jede Ressource muss einer Ressourcengruppe zugeordnet werden. 

### Terminierung einer Ressource 
Folgende Parameter werden bei der Terminierung einer Ressource berücksichtigt: 

Bei Simulation werden alle Reservierungen der gleichen Simulationsnummer sowie die Reservierungen ohne Simulation (0) berücksichtigt. 
| Kalender | Der zu verwendende Kalender wird aus der Ressourcengruppe ermittelt, sonst Kalendernr. 0 |
| --- | --- |
| Startzeit | Minutengenaue Startzeit für die Terminierung. |
| Simulationsnr | Ohne Simulation: 0 |
| Unbegrenzte Kapaztität | Ist dieser Schalter in der Ressource gesetzt, so berücksichtigt die Terminierung keine bestehenden Reservierungen, d.h. im Rahmen der verfügbaren Kalender-Zeit einer Ressource können unbegrenzt viele Terminierungen gemacht werden. |
| Wartezeit vorher | Eine Wartezeit kann auf Ebene der Ressource und/oder auf Ebene des Arbeitsgangs festgelegt werden. Die beiden Zeiten werden kumuliert. Um die eigentliche Startzeit der Terminierung zu ermitteln, wird die Wartezeit vorher zur angegebenen Startzeit hinzuaddiert. Die Wartezeit wird nicht reserviert. Der Wert ist dezimal und in Industrieminuten anzugeben: hh,mm |
| Wartzeit nachher | Eine Wartezeit kann auf Ebene der Ressource und/oder auf Ebene des Arbeitsgangs festgelegt werden. Die beiden Zeiten werden kumuliert. Um die eigentliche Endzeit der Terminierung zu ermitteln, wird die Wartezeit nach der ermittelten Endzeit hinzuaddiert. Die Wartezeit wird nicht reserviert. Der Wert ist dezimal und in Industrieminuten anzugeben: hh, mm |
| Faktor Wartezeit vorher / nachher | Die jeweilige Wartezeit wird mit dem Faktor multipliziert. Damit können diese Zeiten zum Zweck einer optimierten Einplanung manipuliert werden. |
| Belastung % | In der Ressource kann eine Belastungsgrenze in % vorgegeben werden. Dies bezieht sich auf die über den Kalender ermittelte Kapazität der Ressource. Es sind auch mehr als 100% möglich. In diesem Fall wird die Terminierung über die kalendermäßig ermittelte Endzeit des Tages bis zur spätesten Zeit von 23.59 Uhr fortgeführt. |

Eine frisch angelegte Resource kann gelöscht werden. Auf der Datenbank erhält diese Ressourcennummer das „deleted“-Kennzeichen = 1. 

Die gelöschte Ressourcennummer kann nicht neu vergeben werden und wird im Dialog Ressource mit einem Lösch-Kennzeichen versehen.

Geben Sie die Zeit an, welche nach dem Einsatz der Ressource berücksichtigt wird, bevor die nächste Ressource in einem Vorgang genutzt werden kann. Die Zeit wird in Stunden und Industrieminuten eingegeben. 

Tooltip: 

Zeit nach (hh,mm) 

### Feld Belastung % 
In der Ressource kann eine Belastungsgrenze in % vorgegeben werden. Dies bezieht sich auf die über den Kalender ermittelte Kapazität der Ressource. Es kann auch ein Wert von > 100% angegeben werden. In diesem Fall wird die Ressource immer bis zum Tagesende (23:59 Uhr) des letzten Tages reserviert. 

### Checkbox Unbegrenzte Kapazität 
Ist diese Checkbox in der Ressource gesetzt, so berücksichtigt die Terminierung keine bestehenden Reservierungen, d.h. im Rahmen der verfügbaren Kalender-Zeit einer Ressource können unbegrenzt viele Terminierungen gemacht werden. 

### Checkbox Allgemein bebuchbar 
Über diese Checkbox kann die Leistungserfassung für jedermann zugänglich gemacht werden. Das macht Sinn, wenn es sich bei der Ressource um eine allgemein verwendete Ressource handelt, z.B. um die Leistungsart ‚KFZ‘. 

Ansonsten kann in der Leistungserfassung jeder Sachbearbeiter nur Leistungen für seine eigene Ressource erfassen. 

Tooltip: 

Allgemein bebuchbar 

### Checkbox Gelöscht 
Wenn Sie diese Checkbox aktivieren, werden nur gelöschte Ressourcen gesucht.
