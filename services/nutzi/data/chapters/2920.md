# Beispiele für den Ausdruckseditor mit X Mapping Editor 

### Beispiel für Rabatt (Zuschlag, Abschlag) – Überprüfung im X Mapping Editor 
Randinformationen zu diesem Beispiel: 

- Editor ist aus dem X-Mapping (ORDERS Export) heraus geöffnet --> Info: der Ausdruckseditor kann nur beim Export angewandt werden. - Je nach Rabatt-Typ (Zuschlag, Abschlag), soll dieser entsprechend ausgegeben werden (im Bestellkopf). - Auf den Positionen gibt es das Property “this”, da die Bestellpositionen eine Collection darstellen, der Kopf besteht nur aus einem Datensatz, daher kann auf diesen direkt gemappt werden (ein Mapping muss immer auf das Objekt stattfinden). 
Bedingungen: 

- Wenn der Rabattbetrag und Rabatt% „null“ zurückliefert, soll der Typ nicht gefüllt werden. - Wenn der Rabattbetrag oder Rabatt% größer 0 ist, wird „allowance“ als Abschlag/Rabatt ausgegeben. - Andernfalls wird „surcharge“--> Zuschlag ausgegeben (negativer Rabatt = Zuschlag). 

Im Beispiel wird das root-Element zweimal angegeben. Es wird jedoch genau das Gleiche abgebildet. Ein Unterschied liegt darin, dass im zweiten Fall der Datentyp nicht angezeigt wird. Dies dient lediglich als Information. 

Vorgehen: 

Zuerst stellt sich die Frage, ob ein Wert im Rabattbetrag/Rabatt% übertragen wird. Wenn ein Wert übergeben wird, muss eine Überprüfung stattfinden, da er zwei verschiedene Typen (Abschlag/Zuschlag) darstellen kann. Zuerst kann anhand eines Baumes dargestellt werden, welche Abfragen durchgeführt werden: 

Der if-Weg wird zuerst gegangen. 

Es wird nun die erste if-Bedingung betrachtet. Hier geht es darum zu prüfen, ob in Rabattbetrag/ Rabatt% „null“ steht. Sobald eines dieser zwei Felder (Rabattbetrag bzw. Rabatt%) gefüllt ist, ist ein Wert vorhanden. Daher müssen bei der if-Bedingung beide Felder auf „null“ geprüft werden--> AND. 

In der Registerkarte Auswahl Vorlagen des Editors findet sich eine Vorlage zur Bedingung (if, then, else). 

Sobald diese Vorlage ausgewählt ist, ist sie im Ausdruckseditor ersichtlich. Da die Bedingung noch nicht gefüllt ist, wird #ERROR# ausgegeben. Die Kondition erwartet 1 Element. 

Nun müssen zwei Felder geprüft werden. Daher wird eine logische AND-Verknüpfung aus den Vorlagen gewählt (nun werden 2 Elemente erwartet). Um die AND-Verknüpfung an der gewünschten Stelle auszugeben, wird die Condition auf der linken Seite im Editor markiert. Wenn sie eingefügt ist, sieht das Ganze wie folgt aus: 

Anschließend findet die Prüfung auf „null“ statt. Da nun zwei Elemente erwartet werden, muss aus der Vorlagenauswahl, zweimal „isNull“ gewählt werden. 

Nun werden die beiden Bedingungen eingefügt. Dazu wechseln Sie von der Registerkarte Auswahl Vorlagen zu Auswahl Properties . Value bedeutet dabei, der Wert, der in eNVenta im entsprechenden Property eingetragen ist. Über “OrgValue” gibt es die Möglichkeit auf den Wert der letzten Datenbankspeicherung zurückzugreifen, da die Möglichkeit besteht, dass der Wert in eNVenta bereits wieder geändert wurde. Wurde keine Änderung vorgenommen, entspricht der Value dem OrgValue. 

Das Feld kann über einen Doppelklick oder über den Button <-- Value gemappt werden. 

Wenn Sie auf die Elemente klicken, wird das #ERROR# durch den entsprechenden Typ aktualisiert. 

In der then -Anweisung wird nun eine leere Konstante ausgegeben. Die Konstante finden Sie unter Auswahl Vorlagen (Constant). Ist diese hinzugefügt, gibt es einen automatischen Wechsel in die Registerkarte Allgemein . Hier wird der Datentyp „FSstring“ eingegeben, ohne einen Wert. 

Die Reihenfolge der einzelnen Elemente kann über die Pfeiltasten angepasst warden. 

Im Knoten else ( else -Anweisung) wird nun eine neue if-then-else-Bedingung durchgeführt. Das Prinzip funktioniert wieder genau gleich. Es wurde nun darauf geprüft, dass kein Wert vorhanden ist. Else wird im „anderen“ Fall benutzt. Das bedeutet, dass hier nun auf einen der zwei Auswahlwerte geprüft werden muss. Daher eröffnet sich wieder eine if-Anweisung, in der (siehe Baumstruktur) entweder im Feld Rabattbetrag oder Rabatt% auf einen Wert > 0 geprüft wird. Wichtig ist hier, dass in einem Feld ein Wert drinsteht, daher wird OR benutzt. 

Sobald ein positiver Wert erkannt wird, soll in der then -Anweisung „allowance“ (als Konstante) ausgegeben werden, da es somit ein Rabatt ist. Im anderen Fall (else) wird „surcharge“ vom Datentyp 'FSstring' als Konstante ausgegeben.
