# Änderungsprotokoll Rewe 
Änderungsprotokoll Rewe 

Dieser Dialog öffnet sich über System >> Fibu >> Änderungsprotokoll Rewe und bietet die Möglichkeit Vorgänge in der Finanzbuchhaltung, in der Anlagenbuchhaltung und in der Kostenrechnung zu protokollieren. Die Felder/Comboboxen im Kopf des Dialogs bieten die Möglichkeit die Daten zu selektieren. 
## Steuerelemente des Formulars 

### Bezeichnungsfeld Änderungsprotokoll Rewe 
Tooltip: 

Dieser Dialog bietet die Möglichkeit Vorgänge in der Finanzbuchhaltung, in der Anlagenbuchhaltung und in der Kostenrechnung zu protokollieren. 
Für mehr Informationen drücken Sie F1. 

### Button Form-Menü 

Mit Klick auf diesen Button werden je nach Kontext und Berechtigung die unterschiedlichen Optionen des jeweiligen Dialogs hinter dem Button zur Auswahl angeboten. 

Die Funktionen für das Client Side Customizing sowie die Online-Hilfe stehen über diesen Button ebenfalls zur Verfügung.

### Button Suche 

Dieser Button startet die Suche nach Daten; entweder ganz allgemein oder wenn Sie im Kopf des Dialogs entsprechende Selektionskriterien festgelegt haben, gezielt. 

### Button Neu 

Dieser Button leert die Daten in allen Feldern, nicht gespeicherte Änderungen gehen verloren. 

Ist in den Parametern für die Sachbearbeiter die Checkbox 2 Schritt Neu aktiviert, bleiben die Suchkriterien in den Stammdialogen erhalten. 

Siehe Kapitel Parameter Sachbearbeiter . 

### Button Speichern 

Dieser Button speichert bzw. verarbeitet die Eingaben. Vor einer endgültigen Speicherung prüft das System die Eingabe. Wenn fehlende oder fehlerhafte Daten gefunden werden, erhalten Sie eine Meldung und die Eingabe muss entsprechend korrigiert bzw. vervollständigt werden. 

### Button Neuer Satz 

Mit diesem Button können Sie der Tabelle einen neuen Eintrag hinzufügen. 

### Button Löschen 

Dieser Button löscht Ihre Eingaben. Genauer gesagt, die markierte Zeile wird mit dem versehen und erst endgültig gelöscht, wenn Sie den Button drücken. 

Dieser Button steht nur zur Löschung „alter“ Datensätze zur Verfügung, damit die Tabelle nicht „überläuft“. Dazu müssen die entsprechenden Datensätze markiert werden. 

### Button Schließen 

Dieser Button schließt den Dialog. In manchen Fällen werden Sie gefragt, ob Sie die Eingaben / Änderungen noch vorher speichern möchten. 

### Combobox Aktion 
Hier wird angezeigt, ob eine Änderung (Changes) oder Löschung (Delete) vorgenommen wurde. 

### Combobox Objekt 
Hier wird das eNVenta -Objekt angegeben, in welchem die Änderung stattgefunden hat. Z.B. frmJournalDetail ist der Dialog Journal Detail .

### Feld Anzahl max. 
Über dieses Feld ist in den Suchdialogen die Anzahl der einzulesenden Datensätze limitiert. Die einzelnen Datensätze werden im Dialog entsprechend gesucht und nach Erreichen der maximalen Anzahl Datensätze abgebrochen. Auf diese Weise kann eine Suche über alle Datensätze verhindert werden, welche die gesamte Performance beeinträchtigen könnte. 

Diese Einstellung nehmen Sie im Dialog Parameter Allgemein auf der Registerkarte Allgemein vor. 

Siehe Kapitel Parameter Allgemein Registerkarte Allgemein . 

Diese Einstellung lässt sich in den entsprechenden Suchdialogen (z.B. Artikelsuche, Kundenauswahl) ändern bzw. ausschalten (ausgenommen Stammdatenprogramme). 

Tooltip: 

Anzahl Sätze, die maximal eingelesen werden. 

### Feld Anzahl max. 
In diesem Feld wird die Anzahl der tatsächlich gefundenen Datensätze angezeigt.
