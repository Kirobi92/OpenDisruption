# OP-Übernahme Registerkarte Sachkontenübernahme 
OP-Übernahme Registerkarte Sachkontenübernahme 

Anlagenbuchhaltung: 

Bei Einsatz der Anlagenbuchhaltung haben Sie hier die Möglichkeit, die Abschreibungen automatisch in die Finanzbuchhaltung zu übernehmen. 

Permanente Lagerbewertung: 

Bei eingeschalteter permanenter Lagerbewertung werden hier die Buchungen für die Warenbestandsveränderungen übergeben. 

Fremdsysteme: 

Ebenfalls kann hier auch die Übernahme von Sachkontenbuchungen aus Fremdsystemen (z.B. Lohnbuchhaltung) durchgeführt werden. 

Hierbei muss jedoch beachtet werden, dass zuvor eine entsprechende Schnittstelle programmiert werden muss! 

Übernahme in Fremdwährung: 

Es können auch Sachkonten mit Fremdwährung in die OP-Übernahme eingelesen werden, wenn diese mittels einer CSV-Datei importiert werden. 

Bei Buchungen mit Fremdwährung werden der FW-Betrag, FW-Steuerbetrag, Währung und Kurs geprüft. Die Prüfung funktioniert folgendermaßen: 

FW-Betrag x Kurs = LW-Betrag 

Weicht dieser Betrag bei der Prüfung ab, wird die Übernahme abgelehnt. Die Beträge können vom Sachbearbeiter in der Importdatei angepasst werden, wenn notwendig. In diesem Fall muss der Import der Datei nochmals erfolgen. 

Bei OP-führenden Sachkonten werden FW-Beträge ignoriert, da bei diesen nur die Hauswährung relevant sind. 

Innerhalb der gesamten OP-Übernahme können einzelne Elemente einer Verbuchung verändert werden. Es ist möglich, das Sachkonto, die Kostenstelle und den Kostenträger zu verändern. Zur Sicherheit werden alle manuellen Veränderungen hier in der Datenbank protokolliert. 

Ebenfalls kann die Sachkontenübernahme für die Übernahme von Sachkontenbuchungen aus Fremdsystemen (z.B. Lohnbuchhaltung) genutzt werden. 

Bei der Übernahme eines Datensatzes in die Finanzbuchhaltung erfolgt eine Prüfung, falls dieser Datensatz die falsche Niederlassungsnummer hat. Bei einem Fehler erfolgt ein Eintrag in die Spalte ‚Fehlertext‘ und die Spalte 'Niederlassung' erhält den Eintrag ##nnn###. Die Buchung wird abgewiesen. 

Die Spalte 'Niederlassung' ist für Benutzer ohne Niederlassung zugänglich. Der Wert in dieser Spalte (Combobox) kann geändert und gespeichert werden. 

Die Felder Kontonr. von / bis stehen zur Suche nach Soll- bzw. Habenkonten zur Verfügung. Über die Buttons kann nach Konten gesucht werden. 

Buchungen aus der Sachkontenübernahme ohne Steuerschlüssel können auf Konten mit Steuer übernommen werden (z.B. Anlagenabgang, Wareneinsatzbuchungen). 

Analog zur Faktura- bzw. Einkaufsübernahme, ist es in der Sachkontenübernahme möglich Buchungen temporär oder für immer aus der Übernahme auszuschließen. 

Zusätzlich ist auch eine Selektion nach Sachkonten (Kontonummer von, Kontonummer bis) möglich. 

Um bei einer Vielzahl von Lagerbuchungen (die sich aus der permanenten Lagerbewegung ergeben) nur die wirklich relevanten Buchungen in die Finanzbuchhaltung zu übernehmen, können Sie die Lagerbuchungen über den Button Kompensation kompensieren. D.h. die sich ausgleichenden Buchungen werden zusammengefasst, sortiert und markiert. Mit den Buttons Nicht Übernehmen bzw. Ausschluss kann der Sachbearbeiter entscheiden, was mit den markierten Buchungen passieren soll. 

Die Kompensation bezieht sich hierbei immer auf einen Tag und erfolgt nur über Sachkonten-Buchungen. 

Zur Sachkontenübernahme steht Ihnen im Kopf des Dialogs die Checkbox Steuer rechnen zur Verfügung. Wenn Sie diese aktivieren, wird der Steuerbetrag neu aus dem übermittelten Betrag ermittelt. 

Beachten Sie aber hierbei, dass zuvor eine entsprechende Schnittstelle vorhanden bzw. programmiert werden muss. 
### Button Nicht übernehmen 

Dieser Button löscht die Datensätze nur temporär für diese Übernahme. 

Bei einer erneuten Selektion stehen Ihnen diese Daten wieder zur Verfügung. 

Tooltip: 

Von diesem Lauf ausschließen 

### Button Ausschluss 

Über diesen Button werden die selektierten Datensätze endgültig für die Übernahme in die Finanzbuchhaltung gelöscht. 

Tooltip: 

Nie übernehmen 

### Button Kompensation 

Um bei einer Vielzahl von Lagerbuchungen (die sich aus der permanenten Lagerbewegung ergeben) nur die wirklich relevanten Buchungen in die Finanzbuchhaltung zu übernehmen, können Sie die Lagerbuchungen über diesen Button kompensieren. 

Tooltip: 

Kompensation - selektiert sich ausgleichende Buchungen
