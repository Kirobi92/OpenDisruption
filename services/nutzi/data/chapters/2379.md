# Automatische Verbuchung von PayPal Zahlungen 
Für die automatische Verbuchung von PayPal-Zahlungen gibt es in eNVenta die entsprechenden Funktionalitäten, auf die nachfolgend näher eingegangen wird. 

Der PayPal-Transaktionscode wird in eNVenta in der Spalte ‚Transaktionskennung‘ in den folgenden Dialogen ausgewiesen: 

- OP-Übernahme - Auszug-/Kassenimport - OP Debitor - OP Debitor Detail - OP Ausbuchung (DA) bzw. (DZ) 
Die OP-Übernahme schreibt die Transaktionskennung in den Dialog Offene Posten Debitoren . 

Über den Auszug-/Kassenimport werden die Kontoauszüge von PayPal automatisiert verarbeitet. Das heißt, die Kontoauszüge können über den CSV Import eingelesen werden. 

Über die Standardfunktionen: z.B. Button Automatik oder Halbautomatik werden die OPs anhand der Transaktionskennung identifiziert und können automatisch verbucht werden. 

Die Transaktionskennung ist das Identifizierungsmerkmal für den Ursprungs-OP und den Ausgleichs-OP. 

Für die Angabe der Transaktionskennung gibt es ein neues Feld mit 50 Zeichen (alphanumerisch). 

### Anforderungen an CSV Import 
Die Zahlungseingänge/-ausgänge werden von PayPal im CSV-Format zur Verfügung gestellt. Diese CSV-Datei wird über den Dialog CSV Import im Dialog Auszug-/Kassenimport eingelesen. Für den Datenimport muss ein Format gespeichert sein. 

Die PayPal-Transaktionsgebühren werden in einer neuen Spalte im Dialog Auszug-/Kassenimport dargestellt. 

In der Firmen- Bankverbindung ist es möglich, in der Spalte ‚Gebührenkonto‘ ein Sachkonto für die Verbuchung der Gebühren zu hinterlegen. Dabei greift die Steuerverschlüsselung auf dem hinterlegten Sachkonto für die Verbuchung der Gebühren. 

Im Auszug-/Kassenimport wird die Transaktionskennung über den CSV Import eingelesen und in der gleichnamigen Spalte des Auszug-/Kassenimports auf der Registerkarte Importdaten in der Tabelle angezeigt. Wenn diese Spalte gefüllt ist, greift folgende Automatik-Funktion: 

Über die Automatikfunktion werden die eingelesenen PayPal-Sätze den Ursprungs-OP’s, anhand der Transaktionskennung und des Betrags, zugeordnet und über Buchen (F3) ausgeglichen. Zusätzlich wird bei Verbuchung ein separater FB-Buchungssatz für die Gebühren erstellt. Wenn die Transaktionskennung gefüllt ist, wird weder über die Spalten ‚Name‘ und ‚Zweck 1‘, ‚Zweck 2‘, ‚Zweck 3‘, noch über die Tabelle Absender abgeglichen. 

Wenn der Betrag abweichend ist, wird nach gewohnter Vorgehensweise über die Halbautomatik die OP-Ausbuchung geöffnet. 

In der OP-Ausbuchung wird ebenfalls die Transaktionskennung zum OP angezeigt, um eine Zuordnung zu finden, z.B. bei abweichendem Betrag. Bei Verbuchung über die Halbautomatik wird die Gebühr aus dem Auszug-/Kassenimport gemerkt und bei Verbuchung ebenfalls über eine separate FB-Buchung mit verbucht. 

Die Import-Datei von PayPal enthält eine Spalte, die mit CR oder DR gefüllt wird. Wenn Sie im Format des Datenimports diese Spalte der Eigenschaft Ukategorie zuweisen, wird der Wert der Gebühr bei CR gedreht.
