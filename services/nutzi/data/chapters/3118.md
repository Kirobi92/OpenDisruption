# Leistungserfassung 
Leistungserfassung 

Aus den Modulen Projektmanagement , Produktion und Werkzeuge (über den Navigator) kann auf das Menü Leistungserfassung zugegriffen werden. Dazu ist es zwingend notwendig das Modul Kostenrechnung zu lizenzieren. 

Mit diesem Menü ist es möglich eine Weiterverrechnung von Kosten eines Leistenden (Personal) zu erfassen und weiter zu verrechnen. Sämtliche Tätigkeiten können als Leistungsart vordefiniert werden. 

Buchungen aus dem Projektmanagement und Call Center werden berücksichtigt. 

Jeder Leistende wird als Ressource dargestellt, d.h. im Menü Ressourcen angelegt. 

Der Leistende ist immer eine Kostenstelle, die entlastet wird. 

Der Empfänger einer Leistung kann eine Kostenstelle oder ein Kostenträger sein. 

Die Leistungserfassung kann aus dem Projekt / Call über den Button Leistung geöffnet werden. Bei einem Klick auf diesen Button öffnet sich die Leistungserfassung; diese ist bereits mit den nötigen Daten wie Projektnummer bzw. Call vorbelegt. 

Beim Nachtragen von Zeiten bleibt das Datum so lange auf dem gewählten Tag, bis der Dialog erneut geöffnet wird. 

Wenn eine Leistungsart ausgewählt ist und diese die Leistungseinheit „Stunden“ hat, werden die Stunden aus Zeit von und Zeit bis ausgerechnet und in das Feld Leistungsmenge eingetragen. 

Sie erhalten eine Hinweismeldung, falls ein oder mehrere Arbeitstage nicht in der Leistungserfassung eingetragen sind. Diese Meldung erscheint einmalig beim Öffnen der Leistungserfassung (solange Sie sich in der gleichen Sitzung befinden). Erst wenn Sie sich erneut in eNVenta ERP anmelden, erscheint diese Hinweismeldung wieder. Es wird immer bis zu sechs Monate in der Vergangenheit geprüft, an welchen Tagen KEINE Leistungserfassung erfolgt ist. 

Jeder Ressourcengruppe kann im Menü Ressourcengruppen eine übergeordnete Ressource zugewiesen werden (Spalte ‚Übergeordn. Ress.‘). Damit können Benutzer einer übergeordneten Ressource die Leistungen aller Ressourcen aus dieser Gruppe einsehen. Die Ressourcengruppe steht als Selektionskriterium in der Combobox LA-Gruppennr in der Leistungserfassung zur Verfügung. Auf diese Weise kann z.B. ein Abteilungsleiter alle erfassten Leistungen der Mitarbeiter seiner Abteilung betrachten, hat aber keinen Zugriff auf Leistungen von Mitarbeitern anderer Abteilungen. 
### Felder Datum/Zeit von - Datum / Zeit bis: 
Das Tagesdatum wird automatisch eingetragen. 

Bei Bedarf können Sie auch die genauen Zeiten für die Leistungserbringung eintragen. 

Weiterhin ist es möglich einen Zeitraum, z.B. 01.08.-15.08. für Urlaub einzutragen und in die Leistungsmenge nur einmalig 8 Stunden (wenn in Ihrem Unternehmen die 40-Stunden-Woche gilt). Das System rechnet anschließend selbst die Stunden für den gesamten eingetragenen Zeitraum aus. Wochenenden, Feiertage, Heiligabend etc. werden automatisch berücksichtigt! 

Dazu ist Voraussetzung, dass Feiertage u.ä. im Betriebskalender akkurat gepflegt werden! Beim Verbuchen auf mehrere Tage wird geprüft, was alles ein Arbeitstag ist und auf diese Tage wird gebucht. Ist weder ein gültiger Betriebskalender noch eine gültige Kalenderschicht angelegt, können Arbeitstage nicht ermittelt werden und demnach kann auch kein Tag verbucht werden. 

Beim Nachtragen von Zeiten bleibt das Datum so lange auf dem gewählten Tag, bis der Dialog erneut geöffnet wird. 

Der Report kann erstellt werden, selektiert nach Summe Menge/Tag und Summe/Menge Ressource: 

- Gruppierung nach Ressourcennummer - Gruppierung nach Leistungseinheit - Gruppierung nach Tag 

### Leistungserfassung im Projektmanagement 
Der Aufgabe wird eine Leistungsart zugeordnet, die bei der BDE-Buchung vorgeschlagen wird. Die Buchungen erfolgen im BDE-Journal; zusätzliche Datensätze werden in das Leistungs-Journal geschrieben. 

Wenn aus dem Projektmanagement (in Verbindung mit einer Aufgabe) die Leistung erfasst wird, dann dürfen bei den IST-Stunden nur Leistungsarten berücksichtigt werden, die im Typ eine „1“ stehen haben. 

### Leistungserfassung aus dem Callcenter 
Zum Call sollen Arbeitszeiten per BDE erfasst werden können. Die hier erfassten Leistungen sollen ebenfalls von der Leistungserfassung ‚Personal‘ berücksichtigt werden. Die Buchungen erfolgen im BDE-Journal; zusätzliche Datensätze werden in das Leistungs-Journal geschrieben. 

Bei der Erfassung aus dem Call wird der Kostenträger aus dem Call übernommen. 

### Leistungserfassung aus der Produktion 
3 Leistungsarten sind notwendig, die evtl. in den Parametern der Produktion definiert werden können (Rüsten, Fortlauf, Nacharbeit, Material). 

### Leistungserfassung aus dem Projekt 
Dieser erweiterte Dialog Leistungserfassung öffnet sich nur, wenn er aus dem Projekt (Button Leistung auf der Registerkarte Phasen ) geöffnet wird! 

Wenn Sie den Button Genehmig . aktivieren, wird die Leistungserfassung genehmigt und der Dialog geschlossen. Dazu brauchen Sie eine entsprechende Berechtigung. Diese wird im Dialog Berechtigungsgruppe zugeordnet. 

Wenn Sie den Button Zu prüfen aktivieren, bekommt diese Leistungserfassung den Status zu prüfen und der Dialog wird geschlossen. 

Wenn Sie den Button Verarbeitet aktivieren, bekommt diese Leistungserfassung den Status verarbeitet und der Dialog wird geschlossen. 

Fakturierbar: 
Angabe der Menge, welche fakturierbar ist und entsprechend an einen Auftrag übergeben werden kann. 

Termin-ID: 
Angabe des Termins, also eines Teils der Projektaufgabe, wofür die Leistungserfassung gemacht wird. 

Fakt. Betrag: 
Berechneter Betrag, welcher in den Auftrag übernommen wird. Setzt sich aus dem Kostensatz und der fakturierten Menge zusammen. 

Nicht akzeptiert: 
Hier können Sie eine Zeit eingeben, für welche die Leistungserfassung nicht akzeptiert werden kann. 

Terminbeschr: 
Anzeige der Terminbeschreibung aus der Projektaufgabe heraus. 

Status: 
Der Status wird angezeigt.

### Combobox Typ 
Diese Checkbox wird sichtbar, wenn Sie die Leistungserfassung aus dem Projekt öffnen. 

Um eine Leistungsart genauer definieren zu können bzw. damit ein entsprechender Auftrag korrekt erfasst werden kann, können Sie hier noch den Typ der Leistungseinheit auswählen. 

Beispiel: 
Dass bei einer Leistungsart, die Kilometer als Leistungseinheit hat, automatisch von Reisezeit ausgegangen wird, macht wenig Sinn, da die Reisezeit oft auch mit Stunden als Leistungseinheit gebucht wird.
