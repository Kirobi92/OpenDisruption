# Zoom-Element Projekte 

## Überfällige Projektaufgaben 

### Inhalt 
Hier werden die fälligen und / oder überfälligen Projektaufgaben angezeigt. Es werden nur Projekte mit dem Status "offen" und die Projektaufgaben im Status 
- offen - zugewiesen - in QA - in Bearbeitung berücksichtigt. 
Es werden nur die gepflegten Werte "Datum von" / "Datum bis" berücksichtigt. 
### Ziel / Nutzen 
Terminüberwachung der Projekte. Überfällige Projektaufgaben sollen erkannt werden. Es kann rechtzeitig reagiert werden. Fehler werden vermieden. Gleichzeitig dient dieses Element der verlässlichen Statuspflege. 
Wird die Checkbox "Fällige Aufgaben" in Verbindung mit dem Verantwortlichen gesetzt, kann dieses Element auch zur Anzeige und Übersicht der eigenen, zu bearbeitenden Aufgaben eingesetzt werden. 
### Darstellung 
Liste der Projekte sortiert nach "Zeit / Datum bis" absteigend, damit die aktuellsten Fälligkeiten zuerst erscheinen. 
Die Anzeige ist auf 100 Einträge beschränkt. 
### Parameter 
Einschränkungen auf Projektarten, Projektkennungen, Projektleiter, Verantwortliche, Bereiche, überfällige Aufgaben, fällige Aufgaben sind möglich. 
Benutzerfilter: ist diese Option aktiviert, so bekommt jeder Benutzer nur die Aufgaben zu sehen, für die er verantwortlich ist. Vorteil: das Element kann zentral bereitgestellt werden und muss nicht für jeden Benutzer separat erstellt und ausgeführt sein. 
### Drill Down 
Per Doppelklick auf eine Zeile kann die Projektaufgabe aufgerufen werden. 
### Aktualisierung 
Standard / Empfehlung: täglich 
### Berechtigung 
Zugangsberechtigt sind alle Benutzer, die auch das Recht für Projektinformationen (auProject_Information) besitzen. 

## Überfällige Projekt-ToDo's 

### Inhalt 
Hier werden die fälligen und / oder überfälligen Projekt-ToDo's angezeigt. Es werden nur Projekte mit dem Status "offen" und die Projekt-ToDo's im Status 
- neu - offen - wartet auf - genehmigt berücksichtigt. 
Fällige und / oder überfällige Projekt-ToDo's werden nur berücksichtigt, wenn die Fertigstellungstermine gepflegt sind. ToDo's ohne Fertigstellungstermin können angezeigt werden, wenn der entsprechende Parameter "Kein Fertig-Datum" gesetzt ist. 
### Ziel / Nutzen 
Terminüberwachung der Projekte. Überfällige ToDo's sollen erkannt werden. Es kann rechtzeitig reagiert werden. Fehler werden vermieden. Gleichzeitig dient dieses Element der verlässlichen Statuspflege. Optional können auch neue ToDo's ohne Fertigstellungstermin zur Bearbeitung gelistet werden. 
### Darstellung 
Liste der ToDo's sortiert nach Fälligkeitstermin absteigend, damit die aktuellsten Fälligkeiten zuerst erscheinen. 
Ein überschrittener Termin wird rot unterlegt, sonst gelb. 
Die Anzeige ist auf 100 Einträge beschränkt. 
### Parameter 
Einschränkungen auf Projekte, Projektarten, Projektkennungen, Projektleiter, Verantwortliche, überfällige ToDo's, fällige ToDo's und Kein Fertig-Datum sind möglich. 
Die Vorwarnzeit bezieht sich auf die fälligen ToDo's. 
Benutzerfilter: ist diese Option aktiviert, so bekommt jeder Benutzer nur die Aufgaben zu sehen, für die er verantwortlich ist. Vorteil: das Element kann zentral bereitgestellt werden und muss nicht für jeden Benutzer separat erstellt und ausgeführt sein. 
### Drill Down 
Per Doppelklick auf eine Zeile kann das Projekt aufgerufen werden. 
### Aktualisierung 
Standard / Empfehlung: täglich 
### Berechtigung 
Zugangsberechtigt sind alle Benutzer, die auch das Recht für Projektinformationen (auProject_Information) besitzen. 

## Meine Projekttermine 

### Inhalt 
Hier werden alle fälligen, zur Verfügung stehenden Projekttermine angezeigt. Es werden nur Projekte mit Status "offen" berücksichtigt. 
### Ziel / Nutzen 
Terminüberwachung der Projekte. Überfällige Projekte sollen erkannt und rechtzeitig die weitere Vorgehensweise geplant werden. Fehler werden vermieden und gleichzeitig dient dieses Element der verlässlichen Statuspflege. 
### Darstellung 
Liste der Projekte nach Fälligkeit aufsteigend sortiert, damit die überfälligen Projekte sofort erkannt werden. Überfällige Termine werden rot, die heutigen Termine gelb und zukünftige Termine grün angezeigt. Die Anzeige berücksichtigt maximal 100 Einträge. 
### Parameter 
Anzahl Tage ausgehend von heute in die Zukunft (Forecast in Tagen) 
Auswahl des Termintyps - es werden alle Felder angezeigt, die zur Verfügung stehen. 
Optionale Auswahl des Projektleiters 
### Drill Down 
Per Doppelklick auf eine Zeile kann das Projekt aufgerufen werden. 
### Aktualisierung 

Standard / Empfehlung: einmal täglich 
### Berechtigung 
Zugangsberechtigt sind alle Benutzer, die auch das Recht für die Projektverwaltung besitzen.
