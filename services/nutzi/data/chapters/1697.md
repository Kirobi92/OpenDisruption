# Doppelte Anlieferung ausschließen

Lagerverwaltungssystem (LVS) 

# Doppelte Anlieferung ausschließen 
Um eine doppelte Anlieferung zu vermeiden, wird der Sachbearbeiter beim Scannen, z.B. eines Pakets, darauf hingewiesen (mit einem akustischen Signal). D.h. er wird aufgefordert die Tour zu ändern. So soll verhindert werden, dass ein Kunde an einem Tag z.B. ein Paket geliefert bekommt und zusätzlich eine Lieferung per LKW. Dieses Kapitel steht für das 1-stufige LVS zur Verfügung. 

Der Sachbearbeiter hat in dem o.g. Fall die Möglichkeit das Paket dem LKW mitzugeben. 

#### Folgende Einstellungen liegen diesem Vorgang zugrunde: 
- Bei der automatischen Tourenzuteilung wird grundsätzlich die nächste mögliche Tour ausgewählt. - Ist im Auftragskopf eine Tourenart eingetragen, hat diese Priorität. Ist dort nichts eingetragen, und die Einstellungen im Artikel- bzw. Kundenstamm lassen es zu, wird die Pakettour gewählt. Ist der Artikel nicht paketfähig, wird die Tourenart ‚Fuhrpark' gewählt (auch wenn im Auftragskopf 'Paketdienst' ausgewählt ist). - Der Sachbearbeiter kann folgende Tourenarten manuell auswählen: - Paketdienst - Fuhrpark - Spedition - Nachtexpress - Pro Mandant wird nur ein Paketdienstanbieter unterstützt. - Wenn ein Artikel über Fuhrpark ausgeliefert wird, sollen alle anderen Artikel für den Kunden an die gleiche Adresse ebenfalls über Fuhrpark geliefert werden. 

#### Folgende Varianten sind denkbar: 
- Ein Paketdienst-Auftrag ist bereits eingetourt, ein weiterer Fuhrpark-Auftrag kommt dazu: Jetzt werden die Tour und der Stopp ermittelt. Dabei wird der Stopp der Paketdienst-Tour gefunden. Jetzt wird der Stopp von der Tourenart Paketdienst auf Fuhrpark umgestellt. In diesem Zusammenhang wird geprüft, ob der alte Stopp der Pakettour auf die neue Tour darf. D.h. Abmessungen, Gewicht, Anzahl Stopps etc. müssen geprüft werden. Die Tourennummer wird in der Auftragsposition aktualisiert. Der Paketdienst-Auftrag wird, bei Versandetikettierung, Lieferscheindruck oder Ladelistenerfassung gescannt und dem Sachbearbeiter signalisiert, dass der Auftrag auf eine andere Tour gepackt werden muss. Dabei wird dem Benutzer die neue Tour aus dem Stopp angezeigt. Wurde die Versandetikettierung bereits durchgeführt, wird ein Storno der Sendung ausgelöst. Der Sachbearbeiter bringt den Auftrag an den Versandplatz der neuen Tour. - Ein Fuhrpark-Auftrag wurde bereits eingetourt, ein weiterer Paketdienst-Auftrag liegt zum Eintouren vor. Für die Zuordnung einer Tour zu diesem Auftrag wird die Tour mittels Paketdienst ermittelt. Anschließend wird der Stopp ermittelt (anhand der Kundendaten). Ein Stopp mit der Fuhrpark-Tour wird gefunden. Jetzt wird die Tour des Stopps für die Paketsendung übernommen. - Ein Fuhrpark-Auftrag wurde bereits eingetourt, ein weiterer Fuhrpark-Auftrag liegt zum Eintouren vor. Anschließend wird die Tour Fuhrpark ermittelt. Dann wird der Stopp ermittelt (anhand der Kundendaten). Der Stopp der Fuhrpark-Tour wurde gefunden. Jetzt wird geprüft, ob dem 1. Stopp ein Tourzyklus hinterlegt ist. Ist kein Tourzyklus hinterlegt, wird geprüft, ob der Eilt-Zeitpunkt der Tour des 1. Pakets bereits überschritten wurde. Ist das der Fall, wird das 1. Paket ausgeschleust und auf die Tour der 2. Sendung geändert. Ist der Eilt-Zeitpunkt noch nicht erreicht, wird der Stopp des 1. Auftrags für den 2. übernommen. 
Sobald die Lieferscheine / Packstücke für die Ladeliste gescannt und diese bestätigt wurden, gilt der Stopp als endgültig und wird mit Status 2 versehen. Jetzt kann die Tour nicht mehr geändert werden. Kommt ein neuer Auftrag für den Kunden, wird ein neuer Stopp erzeugt, auch wenn dieser am gleichen Tag ist.
