# Bezahlen 
Bezahlen 

Wenn Sie im Dialog Kassieren auf den Button Bezahlen (Funktionstaste F3 ) drücken, öffnet sich der Dialog Bezahlen . Vor dem endgültigen Abschluss eines Verkaufs in der Kasse müssen die Art der Bezahlung, die kassierte Summe und die Währung festgestellt und verbucht werden. 

Ein Kassieren auf Lieferschein ist nur möglich, wenn in der Kasse eine Kundennummer angegeben wurde. 

Ein Kassieren auf Rechnung ist nur möglich, wenn in der Kasse eine Kundennummer angegeben wurde. 

Hat ein Kunde sein Kreditlimit überschritten, werden weder ein Lieferschein noch eine Rechnung gedruckt; auch wenn in den Auftragsarten die Checkbox Sperrung und KL ignorieren gesetzt ist! 

Siehe Kapitel Auftragsarten . 

Nach Abschluss der Buchung wird automatisch der Bon auf dem Bondrucker ausgedruckt, wenn der Parameter Auto Druck in den Kasseneinstellungen aktiviert ist, ansonsten erscheint der Standard-Druckdialog. 

Auf dem Kassenbon werden alle Positionen aufgelistet, inkl. des kassierten Betrags und Rückgeld. 

Ist der Parameter MwSt. pro Pos aktiviert, wird die Steuer pro Position ausgewiesen. 

In das Kassenjournal werden die verkauften Artikel, die unterschiedlichen Zahlarten/Summen und das Wechselgeld eingetragen. 

Rechnung und Auftrag werden freigegeben und gespeichert (und damit ein OP für die Übernahme an die FIBU generiert). 

Beim Lieferschein-Identitätspreisverfahren ist dies nicht möglich! 

Siehe Kapitel Parameter Verkauf (Registerkarte Allgemein , Checkbox Lieferschein-Identitätspreisverfahren ). 

Um nach dem Belegdruck die Erfassung weiterer Zahlungen zu verhindern, werden alle Felder und der Button Bezahlen auf unzugänglich gesetzt. 

Diese Funktionalität ist aktiv, solange Sie den Parameter Dialog Bezahlen autom. schließen im Dialog Kasseneinstellungen auf der Registerkarte Allgemein nicht setzen. 

Wenn der Kassier-Vorgang komplett abgeschlossen ist, schließt der Sachbearbeiter den Dialog manuell. 

Siehe Kapitel Kasseneinstellungen Registerkarte Allgemein .

### Bezahl-Buttons 

Diese Buttons bekommen ein Label für die Art der Bezahlung. Dieses wird im Dialog Zahlungsarten definiert. Die Position der Bezahl-Buttons erfolgt von links nach rechts und von oben nach unten. (Um ein Vertippen zu vermeiden, können zwischen den Buttons auch Leerräume bleiben). 

Wurde eine Positionsnummer nicht vergeben, so wird der entsprechende Button ausgeblendet. Entsprechend ist eine individuelle Einstellung möglich, um die Bedienung zu vereinfachen. 

Die Bedienung der Buttons zum Bezahlen erfolgt über die Tasten F1 – F12 , je nach Positionsnummer. 

Achtung: Da die Taste F3 nicht für einen Bezahlvorgang zur Verfügung steht, wird über die Taste Einfg (einfügen) gespeichert. Diese Taste ist bei fast allen Tastaturen im Ziffernblock integriert und daher auch schnell bedienbar. 

Damit ein Kassierer auf einen Blick erkennen kann, welche Zahlungsart gewählt ist, wird diese farbig ( grün ) angezeigt. Damit kann ein Fehler bei der Zuordnung der Zahlungsart vermieden werden. 

Siehe Kapitel Zahlungsart . 

### Combobox Währung 
Hier kann die Währung, in welcher gezahlt wird, eingestellt werden. Wird diese Option verändert, so werden die Werte der Felder Preis und noch zu bezahlen ebenfalls in die eingestellte Währung umgerechnet. Diese Umrechnung kann allerdings durch die Einstellung nur Kassenwährung für bestimmte Zahlungsarten deaktiviert werden. Dann wird das Feld automatisch mit der Kassenwährung (Kasseneinstellung) vorbelegt und die Auswahl gesperrt. 

Die Combobox ist generell gesperrt, wenn die Kasseneinstellung Nur in Systemwährung kassieren gesetzt ist. 

Tooltip: 

Angabe der Währung 

### Feld Kurs 
In diesem Feld wird der zu kassierende Gesamtpreis eingetragen. Dieser Preis wird in der Währung der aktuellen Zahlung ausgegeben (d.h. wechselt die Währung der Zahlungsart, so werden die Preise neu berechnet). 

Tooltip: 

Hinterlegen Sie den aktuellen Kurs zur Landeswährung. 
- Wenn Sie den Kurs als 'Preisnotierung' pflegen wollen, dürfen Sie die Checkbox 'Division' nicht setzen. 
- Wenn Sie aber den Kurs als 'Mengennotierung' pflegen wollen, dann müssen Sie die Checkbox 'Division' setzen. 
Bemerkung: Die 'Preisnotierung' gibt an, wieviel EINE Fremd-Währungseinheit in Landeswährung wert ist. 
(Beispiel: LW = Euro, FW = US-Dollar: 1 US-Dollar kostet 0,80 Euro, d.h. Preisnotierung = 0,8) 
(Die Mengennotierung ist der Kehrwert der Preisnotierung, hier also 1/0,8 = 1,25)

### Feld Preis 
In diesem Feld wird der gesamte zu kassierende Gesamtpreis eingetragen. Dieser Preis wird in der Währung der aktuellen Zahlung ausgegeben (d.h. wechselt die Währung der Zahlungsart, so werden die Preise neu berechnet).

### Feld Zu bezahlen 
In diesem Feld wird der Restbetrag aus dem Kassiervorgang berechnet und rot angezeigt. Der Betrag wird in der Währung ausgegeben, wie er in der Combobox Währung eingetragen ist (wie der Gesamtpreis). 

Tooltip:

### Feld Betrag 
Hier wird der Teilzahlungsbetrag in der eingestellten Währung eingetragen. Der Betrag wird immer in der ausgewählten Währung kassiert. Nach verlassen dieses Feldes wird die Zahlung in die Tabelle übergeben. 

Tooltip: 

Zahlbetrag 

### Feld Akzept. Betrag 
Dieses Feld ist nur sichtbar, wenn zuvor im Dialog Kassieren ein Kunde, der nicht der Standard-Kassenkunde ist, erfasst wurde. 

Wird akzeptiert, dass nicht der ausgewiesene Wert im Feld Preis zu bezahlen ist, muss dieser akzeptierte Betrag hier im gleichlautenden Feld erfasst werden. 

Aufgrund der Eingabe im Feld Akzept. Betrag wird der Wert im Feld Noch zu bezahlen neu berechnet. 

Wenn Sie hier eine Summe einfügen, wird das Feld Begründung eingeblendet. Hier können Sie einen Kommentar für Ihre Buchhaltung zum akzeptierten Betrag erfassen, damit entsprechende Differenzen in der Buchhaltung beim Ausgleich berücksichtigt werden können. 

Achtung: Rechnungen können an der Kasse lediglich einmal zur Zahlung herangezogen werden, d.h. eine bereits herangezogene Rechnung (auch nur mit einem Teilbetrag) kann nicht mehr herangezogen werden. 

Es kommt die Meldung: „Diese Rechnung wurde bereits bezahlt“. Eine Teilzahlungsverwaltung ist hier nicht gegeben. 

Tooltip: 

Akzeptierter Betrag 

### Feld Begründung 
Wenn Sie im Feld Akzept. Betrag eine Summe einfügen, wird das Feld Begründung eingeblendet. Hier können Sie einen Kommentar für Ihre Buchhaltung zum akzeptierten Betrag erfassen, z.B. Teilzahlung auf Rechnung Nr. 123. 

### Combobox Wechselwährung 
In dieser Combobox können Sie die Währung, in welcher das Wechselgeld angezeigt werden soll, einstellen. Ist die Option Nur in Systemwährung kassieren aktiviert bzw. Rückgeld in Fremdwährung deaktiviert, so ist diese Combobox ebenfalls deaktiviert und mit der Kassenwährung vorbelegt. Dieser Wert wird auch ins Kassenjournal gespeichert. 

Wird die Währung geändert, wird der Betrag des Wechselgelds umgerechnet. 

Tooltip: 

Angabe der Währung 

### Feld Kurs 
Hier wird der aktuelle Kurs zur Landeswährung hinterlegt. Der Kurs bezieht sich immer auf eine Einheit der Währung. Der Kurs für die Landeswährung ist immer exakt 1, da er die Vergleichsbasis für die anderen Kurse darstellt. 

Die Terminprüfung für die Berechnung des Kurses einer Fremdwährung erfolgt in der Finanzbuchhaltung auf das Belegdatum! 

Tooltip: 

Hinterlegen Sie den aktuellen Kurs zur Landeswährung. 
- Wenn Sie den Kurs als 'Preisnotierung' pflegen wollen, dürfen Sie die Checkbox 'Division' nicht setzen. 
- Wenn Sie aber den Kurs als 'Mengennotierung' pflegen wollen, dann müssen Sie die Checkbox 'Division' setzen. 
Bemerkung: Die 'Preisnotierung' gibt an, wieviel EINE Fremd-Währungseinheit in Landeswährung wert ist. 
(Beispiel: LW = Euro, FW = US-Dollar: 1 US-Dollar kostet 0,80 Euro, d.h. Preisnotierung = 0,8) 
(Die Mengennotierung ist der Kehrwert der Preisnotierung, hier also 1/0,8 = 1,25)
