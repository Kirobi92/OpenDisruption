# Parameter MIS Registerkarte Analysis Services 
Parameter MIS Registerkarte Analysis Service 

Auf dieser Registerkarte wird mit Klick auf den Button Verarbeiten ein Sofort-Job erstellt, analog zur Registerkarte Integration Services . 
### Button Verarbeiten 

Mit diesem Button erstellen Sie einen Sofort-Job. 

### Feld Server 
In den MIS-Parametern wird die Verbindung über den MISRemoteService hergestellt, damit die Cubes auch angesprochen werden können. 

Die SSAS arbeiten eigentlich mit definierten Instanzen, deshalb wird in den MIS-Parametern der Instanz-Name angegeben - nicht die Server-IP. Oft sind der Name des BI-Servers und der Instanz-Name aber identisch (Vorbelegung bei der Installation des SQL-Server). 

Die Online-Analyse bzw. Planung kommuniziert dagegen über ADOMD.NET mit den SSAS. Das ist vergleichbar mit einer Datenbankverbindung über ODBC bzw. OleDB. Tatsächlich basiert die Kommunikation auf OleDB. Für die Verbindung wird die Client-API von Microsoft eingesetzt. Bei der Verbindungsaufnahme wird ein Connection-String generiert, und dieser nutzt den in den Parametern hinterlegten Server/Instanznamen, d.h., nicht den "localhost". 

### Button Cubes aktualisieren 

Mit diesem Button werden die Cubes aktualisiert. 

### Baumansicht Cube 

In diesem Feld werden alle Cubes angezeigt. 

Cubes, die ein + vor dem Namen haben, beinhalten weitere Cubes. 

### Feld Beschreibung 
Wenn Sie einen Cube markieren, erscheint in diesem Feld die entsprechend hinterlegte Beschreibung dazu. 

### Feld Letzte Verarb. 
Hier wird das Datum der letzten Verarbeitung angezeigt. 

### Feld Schema Akt. 
Hier wird das Datum angezeigt, an dem die letzte Aktivierung des Schemas erfolgte. 

### Combobox Verarb. Opt. 
Hier wählen Sie die Verarbeitungsmethode für den Analysis Service des SQL-Servers aus. Zur Verfügung stehen: 

- Löschen verarbeiten 
Löscht die Daten im angegebenen Objekt und sämtliche untergeordneten Komponentenobjekte. Nach dem Löschen der Daten wird das Objekt nicht erneut geladen. - Standard verarbeiten 
Erkennt den Verarbeitungsstatus von Datenbankobjekten und führt die Verarbeitung durch, mit der nicht verarbeitete oder teilweise verarbeitete Objekte in den vollständig verarbeiteten Status versetzt werden. Wenn Sie eine Datenbindung ändern, führt die Option Standard verarbeiten für das betroffene Objekt den Schritt Vollständig verarbeiten aus. - Vollständig verarbeiten 
Verarbeitet ein Analysis Services-Objekt und alle in ihm enthaltenen Objekte. Wenn die Verarbeitungsmethode Vollständig verarbeiten für ein bereits verarbeitetes Objekt ausgeführt wird, löscht Analysis Services alle Daten im Objekt und verarbeitet anschließend das Objekt. Dieser Verarbeitungstyp ist erforderlich, wenn eine strukturelle Änderung am Objekt vorgenommen wurde, z. B. wenn eine Attributhierarchie hinzugefügt, gelöscht oder umbenannt wurde.
