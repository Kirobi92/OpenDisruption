# Chargen in Bezug auf Längenverwaltung 
Damit längengeführte Artikel (z.B. Fensterbänke) im Wareneingang mit den tatsächlichen Längen eingebucht werden können, gibt es in eNVenta entsprechende Funktionen. Mit Hilfe dieser Funktionen ist es auch möglich, diese Längenartikel optimiert in Bezug auf Arbeit und Kosten zu verkaufen. 

Dazu ist notwendig, dass der Längenartikel im Artikelstamm (auf der Registerkarte Allgemein ) mit einer Längeneinheit (Combobox Einheit hinter dem Feld Höhe ) versehen wird. 

Längengeführte Artikel werden zur Klassifizierung mit Chargen versehen. So können verschiedene Längen mit entsprechenden Chargen vereint werden. So kann der interne Weg des Artikels verfolgt werden, ohne Rückverfolgung zum Hersteller. 

Wird der Längenartikel im Artikelstamm aber NICHT als Chargenartikel gekennzeichnet, dann entfällt jegliche Logik wie die Längenkontrolle (bzw. Längenoptimierung im VK). 

Da vor dem Verkauf eines Längenartikels evtl. noch ein Konfektionierungsprozess stattfindet (z.B. sägen, montieren), bekommt der Artikel eine neue (verkaufsseitige) Charge zugeordnet. Dazu stehen im Dialog Chargenumbuchung entsprechende Funktionalitäten zur Verfügung.

#### Dialog Chargenzugang 
In Bezug auf „neue“ Chargen für Längenartikel steht im Dialog Chargenzugang die Checkbox alte Chargen ausblenden zur Verfügung. Die Vorbelegung der Checkbox stammt aus dem Artikelstamm. Werden alte Chargen nicht angezeigt, so werden in der Tabelle nur die Chargen angezeigt, die für diese Wareneingangsposition erfasst oder generiert worden sind. 

Im Bereich Chargen generieren wird im Feld Gesamt die Menge der Wareneingangsposition angezeigt. Der Button Generieren generiert die Chargen. 

Die Summe der gesamt-generierten Chargen wird zur Kontrolle im Feld gen. Menge angezeigt. 

Danach können Sie die Eingabe wiederholen; wieder über den Button Generieren und die nächsten Größen können eingebucht werden. Dies wird solange wiederholt, bis die komplette Wareneingangsposition vereinnahmt ist. 

Über den Button Verwerfen werden alle bis dahin erzeugten Chargen verworfen. Diese Funktion ist aber nur solange vorhanden, bis der Dialog verlassen wurde. 

Dabei muss die „Chargenmenge" nicht mit der „Buchungsmenge" übereinstimmen. Für die Wareneingangsbuchung ist die Chargenmenge relevant. Es werden die „Menge_geliefert" und die „Menge_akzeptiert" der Wareneingangsposition angepasst und verbucht. 

Beim Generieren der Chargen wird in der Charge die gebuchte Menge zusätzlich in das Feld Ursprungsmenge übertragen und fixiert. 

Über die Ursprungsmenge können Sie zu jeder Zeit feststellen, ob es sich um eine „Ganze Charge" handelt (Ursprungsmenge = Chargenmenge) oder ob es sich um eine Restmenge handelt (Ursprungsmenge > Chargenmenge). 

In der linken Tabelle ('Altchargen' plus 'Neuchargen') wird eine Statusspalte visualisiert, so dass hier unmittelbar zu sehen ist, ob es sich um eine neugenerierte Charge handelt oder ob eine bereits existierende Charge als Vorlage verwendet und zugeordnet worden ist. 

Ist an einem bestimmten Chargenartikel eine 'Handelslänge' gesetzt, dann wird bei der Chargenzuordnung im Wareneingang sichergestellt, dass auf keine bereits existierende Charge zugeordnet werden kann, sondern IMMER eine neue, eindeutige, interne Chargennummer verwendet werden muss. Dies gilt auch dann, wenn der Einkaufsparameter UrsprungsChargeStrikt gesetzt ist. 

Etikettendruck: Es wird ein Etikett pro Charge in Abhängigkeit der Anzahl Etiketten erzeugt.
