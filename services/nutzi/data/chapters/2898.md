# Staging Anbindung an DB-Komponenten via X-Mapper 
Staging ist ein Prozess oder Bereich, in dem Daten in einem Datenbereich gesammelt und temporär zwischengespeichert werden. Nach Aufbereitung der Daten werden alle Import-Quellen für den Import von Artikel- und Konditionsdaten im Artikel Import Manager verwaltet. 

Der Import der Daten in den Artikel Schattenstamm erfolgt über die Dateiform (csv, ASCII, XML) und ebenfalls über den Import Typ mittels Datenbank. Hierfür muss eine entsprechende SQL-Server-Datenbank mit den notwendigen Daten angebunden werden. Diese Anbindung erfolgt über Framework Studio durch Auswahl der Database Connection „cnEDC“. 

Grundsätzlich kann jede Datenbank angebunden werden. Im Standard unterstützt wird momentan nur die Anbindung an eine „eDC-Datenbank“. Diese Datenbank ist entsprechend normiert und wird vom eDC zur Verfügung gestellt. Der Datenimport erfolgt über die definierten Sichten der SQL-Server-Datenbank. 

Die Daten aus den SQL-Sichten werden über das Staging ( Artikel Import Manager ) sowie den Artikel Schattenstamm dem Artikelstamm und somit den Prozessen zur Verfügung gestellt. 

Für den Import muss in der Importkonfiguration auf der Registerkarte Allgemein der Import Typ Datenbank ausgewählt werden. 

Vor dem Öffnen des X-Mappers wird der Quell-Komponentenname aus einer Combobox ausgewählt und dem Mapping als Quelle fest vorgegeben. 

Import Konfiguration 

Hier drückt der Nutzer den -Button und erstellt dadurch eine neue Konfigurationsdatei. Nun kann er ein bereits vorhandenes Mapping auswählen oder ein neues erstellen. Im Fall der Neuerstellung wird aus der Combobox eine mögliche Komponente ausgewählt und anschließend der Button X-Mapping gedrückt. Die Quell-Komponente ist hierbei fix hinterlegt und entsprechend auszuwählen. 

Importkonfiguration Registerkarte Datenquelle 

Wenn der neue Adapter ausgewählt ist, öffnet sich der Dialog X-Mapping Editor mit weiteren Parametern. Hier muss über die Suche im Dialog X-Mapping Auswahl die passende Vorlage „EDC Default-Mapping“ vom Typ „Object-to-Object“ ausgewählt werden. 

Nach der Hinterlegung des Mappings kann der Import in den Staging-Bereich des Artikelschattenstamms erfolgen. Da hier oftmals große Datenmengen importiert werden, kann mittels des Feldes „Select… Where“ in der Importkonfiguration, die einzulesende Anzahl an Datensätzen eingeschränkt werden.
