# Protokollverfolgung 
Protokollverfolgung 

Diesen Dialog finden Sie unter System >> Administration >> Protokollverfolgung . Über diesen Dialog können die definierten Änderungen verfolgt werden: 

Sie können alle Änderungen, welche die Tabelle und die Spalten betreffen (nur die zur Überwachung definiert sind) anzeigen lassen. 

Die Überwachung kann für jede vorhandene Tabelle und für jede ausgewählte Spalte erfolgen. 

Dabei kann der Anwender (Administrator) frei definieren, welche Daten überwacht werden sollen. 

Über entsprechende Dialoge können Änderungen pro Tabelle und Key gesichtet werden. 

Unter Oracle werden Lösch-, Update- und Insert-Aktionen dokumentiert. Unter SQLServer werden Insert- und Update-Aktionen überwacht.

Bei diesem Feld handelt sich um ein dynamisches Feld. Dieses wird je nach gewählter Tabelle genutzt.

### Feld Anzahl max. 
Über dieses Feld ist in den Suchdialogen die Anzahl der einzulesenden Datensätze limitiert. Die einzelnen Datensätze werden im Dialog entsprechend gesucht und nach Erreichen der maximalen Anzahl Datensätze abgebrochen. Auf diese Weise kann eine Suche über alle Datensätze verhindert werden, welche die gesamte Performance beeinträchtigen könnte. 

Diese Einstellung nehmen Sie im Dialog Parameter Allgemein auf der Registerkarte Allgemein vor. 

Siehe Kapitel Parameter Allgemein Registerkarte Allgemein . 

Diese Einstellung lässt sich in den entsprechenden Suchdialogen (z.B. Artikelsuche, Kundenauswahl) ändern bzw. ausschalten (ausgenommen Stammdatenprogramme). 

Tooltip:
