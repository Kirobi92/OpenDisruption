# X-Mapping Editor Registerkarte X Knoten 
Registerkarte X Knoten 

In der Registerkarte X Knoten werden die Eigenschaften eines Knotens der Datei festgelegt. Hierbei wird dies vornehmlich für den Export definiert. Sollte eine Vorgabe beim Import beinhaltet sein, so wird dieses als Filterkriterium verwendet. 

Der NodeName gibt die Bezeichnung des Knotens in der Datei an. Über die Combobox dahinter kann entschieden werden, ob es sich um ein Element (<Element>) handelt oder ein Attribut. 

HINWEIS FÜR DEN EXPORT 

Für den Export wird eine "Vorlage" für den spezifischen Knoten definiert, wenn die Bedingungen nicht absolut sind (z.B. bei Verwendung des "nicht gleich" Operators). Die Vorlage muss natürlich den Bedingungen des Knotens entsprechen.

Hier wird der NodeName angegeben. Dieser gibt die Bezeichnung des Knotens in der Datei an.

### Feld Min. Vork. 
Ist die Checkbox Mehrmalig aktiviert, kann der Knoten mehrmals in der Datei mit der gleichen Struktur vorkommen. Diese Angabe erfordert dann auch eine Eingabe des Min- und/oder Max-Vorkommens in einer Datei. Ist hier kein Wert in Min. bzw. Max. vorhanden, so gilt dies als niemals oder unendliche Anzahl. 

Ist ein Knoten als mehrmalig markiert, so wird dies mit dem entsprechenden Symbol im TreeView darüber gekennzeichnet. Ebenso kann dies bei der Zuordnung der Eigenschaften als Symbol im mittleren Bereich des Dialogs erkannt werden.

### Button Neuen spezifischen Knoten hinzufügen 

Über diesen Button wird bei Spezifisch der XML-Knoten als spezifisch markiert. Ein spezifischer Knoten hat die gleiche Struktur in wiederholender Form, aber mit unterschiedlichen Inhalten. Kommt ein Element mehrmals vor, ist es unter Umständen notwendig ein BESTIMMTES Element zu spezifizieren. 

Ein spezifisches Element kann z.B. einfach das ERSTE oder das LETZTE Element der Liste sein. Es kann aber auch durch bestimmte Bedingungen eingeschränkt werden. 

Zum Beispiel: Das erste Element bei dem der Wert einer bestimmten Eigenschaft einem bestimmten Wert entspricht. 

Es können mehrere Bedingungen definiert werden. Da es nicht ausgeschlossen ist, dass die definierten Bedingungen auf mehrere Elemente zutreffen, ist die Bestimmung über die "Element Nr." immer notwendig. 

Die Spezifikation eines spezifischen Elements sieht also immer wie folgt aus: 

Das x-te Element von Anfang oder vom Ende bei dem folgende Bedingung zutrifft. 

HINWEISE FÜR DEN EXPORT 

Für den Export sollte eine "XML-Vorlage" für den Spezifischen Knoten definiert werden, wenn die Bedingungen nicht absolut sind (z.B. bei Verwendung des "nicht gleich" Operators). 

Die XML-Vorlage hat zwingend den Bedingungen des Knotens zu entsprechen. 

Spezifische Elemente werden beim Datenexport nicht unterstützt.
