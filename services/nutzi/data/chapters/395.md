# Aktionsdefinition Registerkarte Bedingung 
Aktionsdefinition Registerkarte Bedingung 

Auf dieser Registerkarte können Sie noch Bedingungen definieren, wann bzw. bei welchem Event die Aktion ausgeführt werden soll. So kann die Aktion eingeschränkt werden. 

Bedingungen für Events - Beispiele: 

Auftragspos-Wertgrenze: 
Das Event soll ausgeführt werden, wenn ein Kunde der Kundengruppe 'privater Kunde' eine Auftragsposition über einem Wert von 10.000 € bestellt. 

Event: Auftragsposition aktualisiert 

A-Artikel gelöscht: 
Wird ausgeführt, wenn ein Artikel, dessen ABC-Bewertung "A" war gelöscht, wurde. 

Event: Artikel gelöscht 

Kreditlimit Check 
Der Kunde überschreitet sein Kreditlimit. 
Soll ausgeführt werden, wenn nur noch x% des verfügbaren Kreditlimits nicht verbraucht sind. 
z.B. soll immer gewarnt werden, wenn nur noch 10% des Kreditlimits verfügbar sind. 
Kunde Meyer hat ein Kreditlimit von 1.000 € 
Er bestellt nun für 950€. 
Nun sind nur noch 50€ frei -> kleiner als 10% von 1000€ -> Meldung 

Event: Auftragsabschluss ausgelöst 

### Feld Bedingung 
Hier kann eine Bedingung erstellt werden, mit welcher das Auslösen der Aktion auf bestimmte Fälle eingegrenzt werden kann.
