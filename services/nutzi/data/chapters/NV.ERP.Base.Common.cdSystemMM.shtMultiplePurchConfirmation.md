# Kapitel NV.ERP.Base.Common.cdSystemMM.shtMultiplePurchConfirmation

Mit diesem Parameter ist es möglich, für einzelne Positionen einer Bestellung mehrere Bestell-Bestätigungen zu pflegen. 

Dieser Parameter dient als Schalter zwischen den beiden nachfolgend aufgeführten Logiken. 

Logiken: 

Wenn der Parameter nicht gesetzt ist, wird pro Bestellposition nur die letzte Bestellbestätigung gespeichert. Wenn Sie im Dialog Bestellung auf den Button Bestätigen drücken, öffnet sich der Dialog Lieferbestätigung . 

Siehe Kapitel Lieferbestätigung . 

Wenn der Parameter gesetzt ist, werden alle Bestellbestätigungen chronologisch gespeichert und pro Bestellposition können mehrere Liefertermine und Teilmengen gepflegt werden.
