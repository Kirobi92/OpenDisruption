# Schlagwort-Zuordnung Registerkarte Auswahl 
Schlagwort-Zuordnung Registerkarte Auswahl 

Auf dieser Registerkarte stehen die logischen Schlagworte zur Verfügung und im TreeView werden alle Properties des Objekts zur Auswahl angeboten. Werden beide markiert (Eintrag aus Tabelle und TreeView), kann die Zuordnung mit den Buttons daneben erzeugt werden (Buttons Zuordnen 0 , Zuordnen 1 , Zuordnen 2 ). Die erste Zuordnung erfolgt mit dem Button Zuordnen 0 , weitere mit den Buttons Zuordnen 1 bzw. Zuordnen 2 . 

In der linken Tabelle wird der Schlagworttyp in die Spalte ‚Schlagw. Typ‘ eingetragen. So lässt sich sofort erkennen, um welchen Typ Schlagwort es sich handelt. Dies erleichtert die Übersicht und die Entscheidung, welches logische Schlagwort zum Archiv-Schlagwort passt. 

Für die Archivierung von Sammelbelegen hat ein logisches Schlagwort unbedingt ein Eintrag aus der Collection zu sein, damit die Archivierung für Sammelbelege auch nur einen Sammelbeleg ergibt. 

Wird ein Text-Property mit einem Ganzzahl-Schlagwort verknüpft, wird der Text in eine Ganzzahl geparsed. Wenn das funktioniert, wird der Wert übernommen. 

Es ist erforderlich, dass für jede Zeile des Registers Allgemein ein Property-0 vorhanden ist. 

Wenn Sie Barcodes für die Archivierung nutzen, können Sie das logische Schlagwort [Barcode] einem Property zuordnen, z.B. den Lieferschein-Barcode der Lieferscheinnummer. 

Beim Erzeugen eines Archivbelegs ist es möglich, automatisch mehrere Archiv-Mappen zu erzeugen. Wenn beim Einfügen der Schlagworte erkannt wird, dass hier mehrere Belege zusammengefasst wurden, wird automatisch dafür gesorgt, dass ein Satz Schlagworte mit den entsprechenden Werten erzeugt wird. 

Beim Speichern des Archivbelegs wird dann für jeden Schlagwort-Satz eine Archiv-Mappe bzw. ein Archivpool-Eintrag erzeugt. 

Das funktioniert nur bei einer automatischen Verarbeitung, z.B. beim Druck. 

Bei Schlagwort-Zuordnungen mit Reports können Sie auch die Collections aufklappen. Darunter werden die Properties mit einem ":" angeboten. Das bedeutet, das sind Properties, die an Einträgen in dieser Liste zur Verfügung stehen. (Das ist analog den Listen in der Text-Automatisierung.) 

Die Zuordnung ist nur von einer Collection möglich. 

Bei der Archiv-Zuordnung erhalten Sie einen Hinweis, falls solche Sammelbelege zugrunde liegen.
