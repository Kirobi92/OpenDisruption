# Kombination von Konditionen im Rabatt im Verkauf 
Grundsätzlich besteht die Möglichkeit 2 Konditionen miteinander zu kombinieren. Dies erleichtert vor allem die Pflege der Konditionen. Dazu stehen 3 Konditionsarten zur Verfügung: 

#### Die Exklusivkondition 
Sie ist grundsätzlich nicht kombinierbar. 
#### Die Standardkondition 
Die Standardkondition besteht im Normalfall aus der Kundennummer und einem Rabatt. Sie kann mit einer Basiskondition kombiniert werden (falls vorhanden). 

Das System selektiert - ausgehend von den jeweiligen Selektionskriterien - zunächst ALLE in Frage kommenden VK-Rabatte, sortiert nach der entsprechenden Hierarchiestufe. Ausgehend von der so ggf. gefundenen Collection gelten weitere Regeln: 

- Der erste Eintrag ist IMMER der führende Rabatt. - Ist der erste Rabatt KEIN Standardrabatt, dann wird sofort abgebrochen, andernfalls wird noch der zweite Collection-Eintrag ausgewertet. - Ist der zweite Coll-Eintrag (sofern überhaupt existent) eine Basiskondition, dann wird diese dazu kombiniert, andernfalls wird abgebrochen. - Weitere Rabatte (Coll-Einträge > 2) werden generell ignoriert. 
#### Die Basiskondition 
Die Basiskondition besteht im Normalfall aus einer Artikelnummer und z.B. einer Mengenstaffel. 

Staffeln werden in der Basiskondition hinterlegt. In der Standardkondition können dann kunden- bzw. artikelspezifische Abweichungen, in aller Regel Basispreise oder Grundrabatte eingepflegt werden, ohne dass immer die Staffel mit gepflegt werden muss. 

Beispiel: 
Für eine bestimmte Rabattgruppe gibt es einen gemeinsamen Grundrabatt. Für manche Artikel soll zusätzlich noch ein Staffelrabatt vergeben werden. 

Lösung: 
Da nur entweder die Rabattgruppe oder die Artikelnummer ausgewählt werden kann, wird zur Rabattgruppe, die lückenlos frei definierbar ist, eine Basiskondition hinterlegt. 

Zu den einzelnen Artikeln werden nun Artikelstaffeln als Standardkondition erfasst. 

eNVenta ERP kombiniert die in der Priorität höhere Artikelkondition mit der in der Reihenfolge nächsten Basiskondition. 

Wichtige Regeln: 
- Oberstes Kriterium ist in jedem Fall die Prioritätenreihenfolge. - Falls die erste gefundene Kondition eine Standardkondition ist, wird nach einer evtl. vorhandenen Basiskondition gesucht, sonst nicht. - Zu einer Standardkondition sind grundsätzlich nur entweder die Grundrabatte oder die Staffel kombinierbar - keine weiteren Kriterien. - Kombiniert werden können nur die in den Standardkonditionen nicht vorhandenen Daten (beispielsweise können 2 Staffeln nicht kombiniert werden).
