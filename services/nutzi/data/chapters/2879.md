# Generischer IO Adapter 
Generischer IO Adapter 

Dieser Dialog öffnet sich im Gateway Adapter Kontext über den Button rechts von der Tabelle, wenn ein AdapterGeneralIO markiert ist. Hier werden Einstellungen definiert für den Import bzw. Export per CSV. Dafür stehen jeweils die beiden Registerkarten Export und Import zur Verfügung. 

Für den Import von Dateien werden programmierte Import-Objekte zur Verfügung gestellt. Diese stehen in der Combobox Prozessobjekt zur Auswahl. Mittels des nebenstehenden Button können die Import-Einstellungen (Quell-, Zielpfad, Codierung, etc.) des Adapters verwaltet werden. 

Auf der Registerkarte Export definieren Sie die Einstellungen für den Export der Dateien. Die zur Verfügung stehenden Daten werden über den Transformer geladen. Hierbei stehen alle Komponenten mit Ihren entsprechenden Lade-Bedingungen aus eNVenta zur Verfügung. Ein Eingriff in die Lade-Bedingung ist hierbei nicht möglich und muss daher ggf. programmiert werden. 

Über die Einstellung X-Mapper erfolgt die Zuordnung der einzelnen Properties zum Exportobjekt (XML, csv, etc.). 
### Grundlagen 
Im eNVenta Modul Gateway gibt es den Gateway-Adapter General IO-Adapter (IO = Input / Output), mit dem Sie unterschiedliche Importe und Exporte durchführen können. 

Hier lassen sich die Exporte und Importe von Daten flexibel konfigurieren, sodass unterschiedliche Daten in unterschiedlichen Formaten und über unterschiedliche Kanäle, d.h. Dateien, Datenbanken usw. realisiert werden. 

Viele dieser Importe und Exporte lassen sich bereits „out-of-the-box“ über eine entsprechende Konfiguration nutzen. Weitere Datenübertragungskanäle (Datei, Datenbank, WebService etc.) können über eine Programmierung hinzugefügt werden. Dazu stellt der IO-Adapter ein entsprechendes Framework bereit, welches im Absatz Eigene Programmierung beschrieben wird. 

Im Folgenden werden die Möglichkeiten von Importen und Exporten des IO-Adapters vorgestellt. 
#### Exporte 
Sollen Daten aus einem System exportiert werden, stellen sich folgende Fragen: 

- Wann soll exportiert werden, d.h. was ist der Auslöser (Trigger) für den Export? - Benutzer klickt auf den Jetzt exportieren -Button - Zeitgesteuert über einen Job, z.B. alle 10 Minuten oder jeden Tag 7:00 Uhr - Sobald ein bestimmtes Ereignis (Event) auftritt, z.B. Auftragsabschluss - Was soll exportiert werden? - Daten werden von einem Benutzer selektiert - Job-Parameter merken sich die Auswahl der zu exportierenden Daten - Daten sind Event-bezogen, z.B., wenn Auftrag 4711 abgeschlossen wurde, soll dieser exportiert werden - Wohin soll exportiert werden (welches Medium)? - Datei(en) schreiben - Datensätze in eine Datenbank - Aufruf eines Web-Services - Per E-Mail, usw. - Wie sollen die Daten exportiert werden (d.h. welches Format)? - XML-Format - CSV - Textformat mit fixen Positionen / Längen der Datenfelder - Bei Export in andere Datenbank: welche Daten sollen in welche Tabelle / Spalte eingefügt werden? - Soll der Export dokumentiert werden, d.h. sollen die exportierten Daten nach dem Export verändert werden, z.B. ein Export-Datum setzen oder ähnliches. 
#### Auslöser (Trigger) 
Da der IO-Adapter ein Gateway Adapter ist, wird der Export asynchron durch ein Event (Ereignis) ausgelöst. Tritt das Trigger-Event auf, wird zunächst nur ein Event-Datensatz in die Gw_EventExport Tabelle geschrieben. Dieser Datensatz enthält die Referenz zum Quell-Datensatz und die Info. um was für ein Event es sich handelt. 

Erst wenn zu einem späteren Zeitpunkt (manuell durch Benutzer oder per Zeit-gesteuertem Job) der IO-Adapter ausgeführt wird, führt das Event zu einem Daten-Export. 

In eNVenta gibt es zwei Arten von Events, die im Gateway verarbeitet werden können: 

- Programmierte Events 
(Welche Arten von Events es gibt und wann sie auftreten wird programmatisch festgelegt -> Änderungen müssen programmatisch gemacht werden) - Konfigurierte Events (siehe Event Management ) 
Events lassen sich ohne Programmierung und individuell konfigurieren. 
Bisher lassen sich im IO-Adapter nur die konfigurierbaren Events nutzen. Es ist jedoch geplant auch die Nutzbarkeit der programmierten Events zu implementieren. 
#### Laden der Daten für den Export 
Ein wesentlicher Teil der Konfiguration ist es die Daten, die exportiert werden sollen, bereitzustellen, d.h. diese zu laden / zu filtern / zu sortieren und diese evtl. zu standardisieren. 

Die Ausgangsdaten für den Export ist der vom konfiguriertem Event referenzierte Datensatz. Dieser kann im X-Mapping-Editor genutzt werden, um Daten daraus in das gewünschte Format zu übertragen. 

Es besteht aber auch die Möglichkeit anhand der Daten aus dem Event-Datensatz ein anderes Datenobjekt zu laden, welches dann als Datengrundlage für den Export (d.h. X-Mapping-Editor) genutzt wird. 

Beispiel: 
Das konfigurierte Event „Auftragsposition Lieferfreigabe“ bezieht sich auf eine einzelne Auftragsposition (Komponente: cdSalesOrderDetail). 

Wird eine IO-Adapter Export-Konfiguration auf dieses Event eingerichtet, kann entweder der Datensatz der Auftragsposition als Datengrundlage für den Export genutzt werden oder Sie könnten anhand der Daten dieses Datensatzes z.B. den ganzen Auftrag laden und diesen als Export-Daten nutzen. 

Für weitere Informationen lesen Sie das Kapitel Generischer IO-Adapter - Einrichtung und Weiterentwicklung .
