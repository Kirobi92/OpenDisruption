# Anbindung der Telefonie in eNVenta ERP 

## Allgemeine Informationen 
TAPI (Telephony Application Programming Interface) ist eine Schnittstelle zwischen den Telefonie-Funktionen von Windows und der Telefonie-Anwendung. 

TAPI stellt dem Entwickler Standardfunktionen zur Entwicklung eigener Telefonie-Anwendungen zur Verfügung. 

Der TSP (TAPI Service Provider) ist die Schnittstelle zur Telefonie-Hardware. Er übersetzt die gewünschten TAPI-Funktionen in eine Sprache, die von der Hardware verstanden wird. Statusänderungen eines Gesprächs werden vom TSP an die Windows-TAPI übermittelt. 

## Umsetzung der Telefonie 
Im Fall von eNVenta ERP stellt der lokale Java-Client die TAPI-Anwendung dar. Der Client greift auf die NVTopTapi zu, welche wiederum die TopTapi verwendet, um die Telefonie anzusprechen. 

## Installation und Anbindung der NVTopTapi 
Bei der Installation der NVTopTapi werden die nötigen DLLs im gewählten Installationsverzeichnis gespeichert und entsprechende Einträge in der Windows-Registry vorgenommen. Der Java-Client greift dann direkt über die Registry auf diese DLLs zu. Die NVTopTapi wird dadurch als sogenanntes COM-Objekt (Component Object Model) verwendet – genau wie das bei MS Word oder MS Excel der Fall ist. Dadurch ist es dann möglich auf die lokalen Telefonie-Funktionen zuzugreifen und z.B. die Lines in den Telefon-Parametern anzuzeigen. 

## Anmerkungen in der Telefonie 
Die NVTopTapi funktioniert nur auf Windows-Systemen! 

Welche Version NVTopTapi eingesetzt wird, sehen Sie in der Auflistung der installierten Programme in der Systemsteuerung („Software" unter Windows XP, „Programme und Funktionen“ unter Windows 7). 

Beim Initialisieren der NVTopTapi wird erst versucht, die Durchwahl einer Line zu erkennen, um diese dann für die Telefonie zu verwenden. Empfehlung: die Line in den Telefon-Parametern bekommt folgendes Format: 

'Line-ID': 'Line-Name': 'Durchwahl' 

Beispiel: 
1: Test: 999 

Kann die Durchwahl nicht erkannt werden, wird die Line-ID zur Initialisierung verwendet. 

## Tipps und Tricks in der Telefonie 
:Verwenden Sie stets die neueste Version der NVTopTapi! 

Wenn Sie die NVTopTapi installiert haben, gibt es im Startmenü unter dem Ordner NVTopTapi den Menüpunkt „NVTopTapi testen“. Damit können Sie ein kleines Tool starten, mit dem Sie die Funktionen Ihrer lokalen Telefonie testen können. In diesem Tool werden die reinen TopTapi-Funktionen verwendet, ohne die Erweiterungen der NVTopTapi. D.h. wenn hier etwas nicht funktioniert, hat bereits die TopTapi ein Problem und die NVTopTapi kann dann entsprechend auch nicht fehlerfrei arbeiten. 

Standardmäßig gibt es auf Windows-Systemen die sogenannte Wählhilfe. Wenn die Telefonie korrekt eingerichtet ist, können Sie damit über das eingestellte Telefon beliebige Telefonnummern anrufen. Man kann die Wählhilfe direkt über Startmenü >> Ausführen >> dialer.exe starten. Wenn die Wählhilfe nicht fehlerfrei funktioniert, lässt das auf eine fehlerhafte Installation der Telefonie schließen (z.B. falsche oder fehlerhafte TAPI-Treiber, falsche TAPI-Einstellungen, etc.).
