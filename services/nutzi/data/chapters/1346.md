# Gutschriftsabrechnung Registerkarte Allgemein 
Gutschriftsabrechnung Registerkarte Allgemein 

Auf dieser Registerkarte werden die Auftragspositionen eingelesen, entsprechend der Selektion im oberen Teil des Dialogs. 
### Button Neu 

Mit diesem Button legen Sie eine neue Position an. 

### Button Ändern 

Mit diesem Button können Sie eine Position ändern. Dafür öffnet sich der Dialog GAP Position . 

Siehe Kapitel GAP Position ändern . 

### Button Rückstand 

Der Button Rückstand stellt eine markierte Position aus der GAP zurück. Die Kennung der Gutschrift wird wieder aus der Auftragsposition entfernt, damit diese Auftragsposition später wieder in eine andere Gutschriftsabrechnung übernommen werden kann. 

Gesplitte Mengen können nicht in Rückstand genommen werden. 

Tooltip: 

Mit diesem Button werden markierte Position aus der GAP entfernt. 

### Button Rest rückst. 

Der Button Rest rückst. entfernt alle nicht markierten Positionen aus der GAP. Die Kennung der Gutschrift wird wieder aus den Auftragspositionen entfernt, damit diese Auftragspositionen später wieder in eine andere Gutschriftsabrechnung übernommen werden können. 

Tooltip: 

Alle nicht markierten Positionen aus der GAP entfernen. 

### Button Splitt 

Mit diesem Button können Sie eine Position splitten. Dafür öffnet sich der Dialog Position splitten GAP . 

Hier wird die Menge angegeben, welche nicht geliefert wurde. Dann übernehmen Sie die Einstellungen in die Gutschriftsabrechnung mit dem Button . Anschließend drücken Sie den Button Rückstand , damit diese Menge aus der Gutschriftsabrechnung entfernt wird. Dabei behält die Splitt-Position die Original-Lieferscheinnummer. 

Siehe Kapitel Gutschriftsposition splitten . 

Tooltip: 

Markierte Position splitten 

### Button Vorgang 

Der Button Vorgang öffnet den zugrundeliegenden Vorgang. Dies kann ein Auftrag oder auch eine interne Gutschrift sein. 

Tooltip: 

Auftrag / Gutschrift öffnen 

### Button Preishistorie 

Der Button Preishistorie öffnet den Dialog Preishistorie GAP zur Ansicht der Entwicklung der Konditionen. 

Siehe Kapitel Preishistorie GAP . 

Tooltip: 

Preishistorie GAP

Wenn dieses Feld gefüllt ist, wird in der Auftragserfassung die hier hinterlegte Auftragsart voreingestellt. Sie kann jederzeit geändert werden. 

Dieses Feld steht in direktem Zusammenhang mit dem Parameter Auftragsart fix im Dialog Auftragsarten .

Tooltip:

### Tabellenspalte S 
Hier wird der Positionsstatus angezeigt. Zur Verfügung stehen: 

1 - Offen 
2 - Offen, zur Lieferung freigegeben 
3 - Offen, zur Faktura freigegeben 
4 - Fakturiert 
5 - Geliefert 
6 - Storniert

### Tabellenspalte Bezeichnung 
Die Bezeichnung dient der einfachen Identifikation des Artikels und kann auch zur Suche verwendet werden. 

Bei Eingabe der Artikelnummer wird das Feld Bezeichnung automatisch mit dem Namen des Artikels gefüllt. 

Tooltip: 

Artikelbezeichnung 

### Tabellenspalte Menge geliefert 
Hier wird die gewünschte Liefermenge eingetragen. Dies ist auch der Wert, der für die Fakturierung und die Lagerabbuchung maßgebend ist. Nach der Positionserfassung enthält das Feld automatisch den gleichen Wert wie die Spalte 'Bestellt'.

### Tabellenspalte Menge geliefert 
Hier wird die gelieferte Menge angegeben. D.h. die tatsächlich durch den Lieferscheindruck als gelieferte, im Auftrag eingetragene, Menge. 

Tooltip: 

Menge geliefert: 

Die tatsächlich durch den Lieferscheindruck als gelieferte, im Auftrag eingetragene Menge. 

### Tabellenspalte Einheit 
Hier wird die Beschreibung der Einheit aus der Pickliste angezeigt. Dies dient vor allem der Optimierung der Reports, da Abkürzungen leicht missverstanden werden können. Bei vorhandenem Customizing mit dem Feld Einheit ist dies ggf. nachzuziehen. 

Ist in der Pickliste keine Beschreibung gepflegt, so wird wie bisher die Bezeichnung aus eNVenta angezeigt. 

Tooltip: 

Einheit VK 

### Tabellenspalte Preis 
Hier wird der Preis angegeben. 

Tooltip: 

Preis 

### Tabellenspalte VK pro 
Hier wird eine Preiseinheit angegeben. Dieser Preis kann sich auf 10, 100, 1000 oder eine andere Anzahl Artikeleinheiten beziehen. Standard ist der Wert 1. 

Dieser Wert spiegelt die Anzahl an Artikeleinheiten wider, auf welche sich der Verkaufspreis bezieht. 

Beispiel: 
- 1 Einheit - je 10 Einheiten - je 100 Einheiten
