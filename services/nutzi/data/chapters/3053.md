Werkzeuge 

# Datenimport in den Artikel Schattenstamm 
Im Zeitalter der Globalisierung müssen immer größere Artikelmengen (im Hinblick auf Preise und Konditionen) zeitnah aktualisiert werden. Dafür stellen die Hersteller und Verbände digitale Kataloge zur Verfügung, die über den eNVenta Artikel Schattenstamm bequem importiert werden können. Dies erspart viel Zeit, und Fehler bei der Dateneingabe werden auf ein Mindestmaß reduziert. 

Um den unterschiedlichen Katalogformaten gerecht zu werden, gibt es keine fest vorgegebenen Feldzuordnungen. Die einzelnen Felder des digitalen Katalogs (Quelldatei) werden den jeweiligen Datenfeldern in eNVenta einmalig je Katalog zugeordnet. Dies wird MAPPING genannt. 

Da unterschiedliche Kataloge gelegentlich dasselbe Format haben, können einzelne oder mehrere Mappings unabhängig von Katalog oder Datenlieferant in einer Importkonfiguration zusammengefasst werden. 

Es gibt Lieferanten für Daten und Lieferanten für Ware! Diese müssen nicht identisch sein! 

Die einzelnen Artikeldatensätze werden durch eigenständiges Setzen des Importschlüssels innerhalb des Artikel Schattenstamms eindeutig. 

Der Import der einzelnen Kataloge und der Konditionsdatei erfolgt jeweils in einem separaten organisatorischen Lauf, bis hinein in den Artikel Schattenstamm. Dieser Lauf wird getrennt über den Datenbereich (Feld SBScope in der Artikel-Preisvorerfassung und Data Import Konfiguration ). D.h. es können Daten mit denselben Import-Schlüsseln eingespielt werden, ohne sich gegenseitig zu überschreiben, wenn diese Dateien in unterschiedliche Datenbereiche eingespielt werden. Dies nennt man Daten Staging und wird über den Dialog Artikel Import Manager ausgeführt. 

Dieser Datenbereich kann mit maximal 20 Stellen gepflegt werden! 

Im Staging besteht die Möglichkeit, Daten aus dem letzten Import mit den neuen Importdaten zu vergleichen; Abweichungen werden farbig gekennzeichnet. 

Der Schattenstamm enthält Artikel aus externen Datenquellen, welche dem Verkauf zur Verfügung stehen, bisher aber noch nicht genutzt wurden. Somit bietet der Artikel Schattenstamm eine weitere Recherchemöglichkeit, welche über den Artikelstamm hinausgeht. Soll ein Artikel in den Artikelstamm übernommen werden, muss er zuerst im Artikel Schattenstamm für die Übernahme gekennzeichnet werden (Checkbox in der Spalte 'Transfer'). 

Datenimport- Vorgehensweise 

- Mapping für den Import erstellen oder hochladen. - Einlesen in den Artikel Import Manager (Data-Staging). - Bearbeiten im Artikel-Import-Manager (Data-Staging). - Übernehmen in den Artikel Schattenstamm (Tabelle Artpreisvorerfass ) - Übernehmen aus dem Artikel Schattenstamm in den Echtstamm (Artikelstamm) 
Was ist vor dem Mapping zu tun? 

Da die Datenlieferanten nicht immer die Warenlieferanten sind, werden teilweise in Katalogen die Warenlieferanten je Artikel aufgelistet. Diese Warenlieferantenbezeichnung oder -codierung ist beim Einlesen der Daten zwingend in die eNVenta -Lieferantennummer umzusetzen. Dazu wird im Vorfeld die Verschlüsselung wie folgt hinterlegt: 

- Falls zum Artikel zugehörige Nummern als Fremdartikelnummer eingelesen werden sollen, legen Sie in der Pickliste den Fremdartikelnummerntyp an. Dazu markieren Sie in der linken Tabelle den Eintrag Fremdlieferantennr. Typ und geben in der rechten Tabelle die Verschlüsselung an (z.B. Lieferantennummer). - Anschließend wechseln Sie in den Dialog Import-Schlüssel Verwaltung (über System >> Allgemein ), um einen internen Schlüssel zum angelegten Fremdlieferantennummern-Typ (EDE-Lieferantennummern) zu vergeben. 
Falls die Kataloge (Quelldaten) auf einem Server hinterlegt sind, richten Sie einen Verbindungsmanager ein.
