# Protokolle Aufräumen Registerkarte Tabelle 
Protokolle Aufräumen Registerkarte Tabelle 

Auf dieser Registerkarte erhalten Sie eine Übersicht über bereits angelegte Protokoll-Tabellen. 
### Tabelle 

### Tabellenspalte Aktiv 
Die Löschung erfolgt bei Ausführung des Lösch-Jobs für alle Definitionen, welche die Kennung Aktiv gesetzt haben. Wird z.B. der Lauf 1x monatlich ausgeführt, so werden die ältesten Daten regelmäßig aus der Datenbank gelöscht. 

### Tabellenspalte Alter in Tagen 
Als Filter wird eine Anzahl an Tagen angeben, die das Alter der Daten festlegen, die bereinigt werden sollen. 

Wird eine Lösch-Konfiguration gespeichert, bei der die Tage so gewählt sind, dass extrem viel gelöscht wird (> 50.000), während gleichzeitig das Protokoll aktiviert ist, erhält der Nutzer eine Warnung. 

Tooltip: 

Es werden nur Einträge gelöscht, die älter sind als die hier angegebene Anzahl von Tagen. 

### Feld Anzahl max. 
Über dieses Feld ist in den Suchdialogen die Anzahl der einzulesenden Datensätze limitiert. Die einzelnen Datensätze werden im Dialog entsprechend gesucht und nach Erreichen der maximalen Anzahl Datensätze abgebrochen. Auf diese Weise kann eine Suche über alle Datensätze verhindert werden, welche die gesamte Performance beeinträchtigen könnte. 

Diese Einstellung nehmen Sie im Dialog Parameter Allgemein auf der Registerkarte Allgemein vor. 

Siehe Kapitel Parameter Allgemein Registerkarte Allgemein . 

Diese Einstellung lässt sich in den entsprechenden Suchdialogen (z.B. Artikelsuche, Kundenauswahl) ändern bzw. ausschalten (ausgenommen Stammdatenprogramme). 

Tooltip: 

Anzahl Sätze, die maximal eingelesen werden. 

### Feld Anzahl Sätze 
Dieses Feld zeigt die Anzahl der tatsächlich eingelesenen Datensätze.
