# Kreditlimit - Allgemeine Informationen 
In diesem Kapitel erhalten Sie ganz allgemein Informationen zum Thema Kreditlimit . 

Im Dialog Sachbearbeiter kann für jeden einzelnen Sachbearbeiter definiert werden, ob er bei der Auftragserfassung ein bestimmtes Kreditlimit überschreiten darf oder nicht. Diese Einstellung nehmen Sie auf der Registerkarte Verkauf mit der Checkbox Striktes Kreditlimit vor. 

Siehe Kapitel Sachbearbeiter . 

Das Kreditlimit selbst wird einem Kunden im Kundenstamm auf der Registerkarte Rechnungswesen im Feld Kreditlimit WW zugeordnet. Wenn Sie dazu die Checkbox Strikt aktivieren, kann dieses Kreditlimit beim Erfassen eines Auftrags nicht überschritten werden. 

Siehe Kapitel Kunde Registerkarte Rechnungswesen . 

Aber: Ist in den Parametern Allgemein (Registerkarte Wawi ) ein Passwort für das strikte Kreditlimit hinterlegt, kann mit Eingabe dieses Passworts das Kreditlimit überschrieben werden. Dazu benötigt der Sachbearbeiter die entsprechende Berechtigung. 

Zum Kunden kann auch der Parameter Kreditlimit auf Verbandsebene gesetzt werden. 

Siehe Kapitel Parameter Allgemein Registerkarte Wawi . 

Ein gesetztes Kreditlimit kann aber über die Auftragsarten umgangen werden. Dazu stehen die Checkboxen Sperrung und KL ignorieren zur Verfügung. Wenn Sie diesen Parameter aktivieren, findet keine Prüfung auf Kreditlimit oder versichertes Kreditlimit statt. 

Ist das Kreditlimit überschritten, erscheint bei Erfassung eines neuen Auftrags lediglich eine Hinweismeldung, dass das Kreditlimit überschritten wurde. 

Ist dem Kunden ein striktes Kreditlimit hinterlegt, kann der Auftrag nicht erfasst werden. 

Hat der Sachbearbeiter die Berechtigung, das Kreditlimit zu erhöhen, öffnet sich der Dialog Kreditlimit . Hier kann das neue Kreditlimit eingetragen werden. 

Siehe Kapitel Kreditlimit . 

Das Kreditlimit, das einem Kunden hinterlegt ist, wird auch im Dialog Bonität angezeigt. 

Siehe Kapitel Bonität . 

Ist die Abschlussprüfung aktiviert ( Parameter Verkauf Registerkarte Allgemein ) kann ein Auftrag jederzeit erfasst und geändert werden, unabhängig davon, ob eine Kreditlimitsperre eingetragen ist. Wenn das Kreditlimit überschritten ist, öffnet sich der Dialog Kreditlimit Überschreitung . Allerdings kann der Auftrag nicht abgeschlossen werden und somit kein logistischer Prozess gestartet werden. 

Im Menü Optionen im Kundenstamm steht Ihnen der Eintrag Kreditlimit Ausnutzung zur Verfügung. Der Dialog öffnet sich auch in der Bonität über den Button neben dem Feld Kreditlimit . Hier erhalten Sie eine Übersicht über die Zusammensetzung des Kreditlimits für einen Kunden und Sie haben die Möglichkeit ein neues Kreditlimit zu erfassen. 

Das Kreditlimit eines Kunden kann auch versichert werden. Die Einstellungen dafür nehmen Sie im Kundenstamm auf der Registerkarte Versicherung vor. Die hinterlegte Versicherungssumme wird ebenfalls in der Bonitätsinformation angezeigt. 

In diesem Zusammenhang ist es notwendig Kreditversicherer und Kreditversicherungsarten anzulegen. Diese Einstellungen finden Sie im Kundenstamm unter Optionen . 

Im Dialog Kreditversicherer hinterlegen Sie die Versicherungsgesellschaften, bei denen Sie Kundenkredite versichern. 

Siehe Kapitel Kreditversicherer . 

In den Kreditversicherungsarten hinterlegen Sie Ihre Versicherungsarten. D.h. Sie legen fest, wie die Kredite versichert werden. Z.B. pauschal, namentlich versichert etc. 

Siehe Kapitel Kreditversicherungsarten . 

Diese stehen dann im Kundenstamm auf der Registerkarte Versicherung in der Combobox Beurteilungsart zur Verfügung. 

Vergleichen Sie dazu auch das Kapitel Erweitertes Kreditlimit . 

### Kreditlimitprüfung im Auftrag 
Auftragserfassung mit aktiviertem Belegabschluss: 
Da hier noch keine Veranlassung besteht, beim Speichern auf eine KL-Überschreitung zu regieren, führt dies zunächst nur zu einem Eintrag in der Tabelle in der Registerarte Abschluss . Die eigentliche Reaktion auf die Kreditlimit-Überschreitung (Begründung mit oder ohne Passwort) erfolgt im Event 'Belegabschluss'. 

Änderungen für das einfache Kreditlimit: 
Sonderfall: Es existiert ein Auftrag und der Belegabschluss ist bereits durchgeführt. Jetzt wird das Kreditlimit überschritten (z.B. durch Änderung der Positionen). Dann führen alle Versuche den Auftrag zu speichern zu einer Kreditlimitprüfung und einem Abbruch des Speicherns bis eine Begründung für die Überschreitung erfasst worden ist. 

Änderungen für das erweiterte Kreditlimit: 
Falls Auftragsabschluss aktiv: Der Auftrag wird nicht auf Kreditsperre gesetzt, sobald bei überschrittenem Kreditlimit gespeichert wird, sondern erst wenn der Auftrag abgeschlossen wird, oder bereits abgeschlossen ist und bei einem erneuten Speichern das Kreditlimit überschritten wird. 

### Tourenplanung nach Auflösen der Kreditlimitsperre 
Die Neueinplanung von Auftragspositionen für eine Tour erfolgt automatisch, wenn der Grund der Kreditlimitsperre nicht mehr gegeben ist.
