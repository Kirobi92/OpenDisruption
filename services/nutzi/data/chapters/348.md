# Manueller Datenimport 
Um den Datenimport manuell durchzuführen, öffnet der Sachbearbeiter den Dialog Datenimport Adapter . 

Der Sachbearbeiter wählt das Importformat (Adapter) aus. 

Dann wählt er aus der Combobox Import Format die Konfiguration aus, bzw. legt die Bezeichnung für eine neue fest. 

Mit Klick auf den Button F. Import wird die Datei auf den Server übertragen. Der Manager übergibt die Datei an den Adapter mit SetFile und setzt die Versionsinformation über die Methode GetImportVersion(). Wenn keine Information vorhanden ist, wird der aktuelle Tag als Versionsnummer genommen. 

Mit Klick auf den Verarbeiten-Button werden die Positionen wie beschrieben verarbeitet. Nach dem Verarbeiten wird die DataImportConfiguration mit den Werten LastImportDate, LastImportCreationDate (Erstelldatum der Datei), LastImportVersion aktualisiert. 

Nach dem Import der zu ändernden Daten ist zu analysieren, welche Felder von einer Änderung betroffen sind. Die Werte CanInsert, CanUpdate, CanDelete aus der Adapter-Konfiguration steuern hierbei die Vorbelegung. Ist ein Datensatz bereits im Schattenstamm vorhanden, und ist der Wert abweichend zu dem neuen Wert, wird in die Tabelle DataImportDetailSelection ein Datensatz eingetragen. Bei der Verarbeitung werden später nur Werte berücksichtigt / in den Schattenstamm übernommen, welche hier einen Eintrag haben. 

Der Fortschritts-Balken informiert während des Imports über den aktuellen Verarbeitungsfortschritt. 

Ist die Verarbeitung abgeschlossen, müssen die Datensätze mit Änderungen vom Sachbearbeiter nachbearbeitet werden. Ziel des Entwurfs ist, eine möglichst effiziente Weiterverarbeitungsmöglichkeit zu schaffen.
