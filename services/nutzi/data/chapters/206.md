# AKL Konfiguration 
AKL Konfig. Matchcode 

Dieser Dialog öffnet sich im Kundenstamm, Lieferantenstamm, Adressstamm und Vertreterstamm über das Menü Stamm . 

Dies ist ein allgemeiner Konfigurationsdialog, der eine Liste mit Feldnamen des jeweiligen Objekts (Kunde, Lieferant, Adresse oder Vertreter) anzeigt. Sie können hier eine Liste mit Feldnamen für die Indexierung zusammenstellen. 

Mit dem Button Indizieren wird der ganze Index nach einer Änderung komplett neu aufgebaut (auch nach einer evtl. Datenübernahme) und die Datenbank-Tabelle wird angepasst. 

Nach jeder Änderung an den Feldern (neue Felder hinzu, löschen) müssen Sie die Daten über den Button Indizieren neu aufbereiten. 

Je nach Anzahl der Kunden dauert dies entsprechend lang. Aus diesem Grund wird hier eine Fortschrittsanzeige eingeblendet und der Vorgang kann ggf. über den Button Abbrechen abgebrochen werden. 

Ist ein Memofeld in der Indizierung vorhanden, so ist zwingend ein Zeilenumbruch (Enter) wie ein Leerzeichen zu behandeln. 

Bei einer Änderung der Feldliste ist dies zwingend vor dem Indizieren zu speichern. 

Bei der Suche im Suchfeld können mehrere Begriffe hintereinander eingegeben werden. Diese müssen mit einem Leerzeichen getrennt werden, damit sie als separate Suchbegriffe (mit UND-Verknüpfung) behandelt werden. Die Reihenfolge ist unerheblich. Markieren Sie hierzu das gewünschte Feld links in der Tabelle und klicken auf den Button , um das Feld in die Feldliste zu übernehmen. Die verschiedenen Felder werden automatisch getrennt eingetragen. 

Manuelle Eingriffe sind nicht notwendig und sollten daher vermieden werden. 

Siehe Kapitel Parameter Allgemein . 

Alle "Suchname"-Felder müssen den konfigurierten Matchcode unterstützen. Dafür setzen Sie die Checkbox Cursor in Suchname im Dialog Parameter Allgemein auf der Registerkarte Allgemein . 

Um die Kompatibilität mit vorhandenen Installationen sicherzustellen, wird geprüft, ob eine Konfiguration vorhanden ist. Wenn nicht, verhält sich das „Suchname“-Feld wie gewohnt. 

In den Suchdialogen ist es weiterhin möglich, den konfigurierten Matchcode mit anderen Suchfeldern zu kombinieren. Auch in den Stammdatendialogen. 

Es ist zu empfehlen, sowohl für Kunden als auch für Adressen eine Matchcodekonfiguration anzugeben und nicht nur für eines von beiden. 

Die Konfigurationen für Kunden, Adressen und Lieferanten müssen nicht übereinstimmen. Es können problemlos unterschiedliche Felder verwendet werden. 

Es ist auch möglich numerische Felder (z.B. Kundennr) in die Suche mit aufzunehmen. 

In einem Suchfeld ist keine Volltextsuche möglich. Die Abfrage ist immer im Format LIKE "XXX%". 

Wenn in einem Suchfeld auch Leerzeichen existieren, so kann nur nach dem ersten Wort gesucht werden, da das Leerzeichen als Trennzeichen für einen neuen Suchbegriff interpretiert wird.
