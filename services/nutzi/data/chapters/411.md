# Ereignis Komponenten 
Komponenten 

Die Ereignis Komponenten werden in der Ereignisdefinition über Stamm >> Ereignis Komponenten aufgerufen. 

Komponentenname: Hier kann ein beliebiger Komponenten-Name (Beispiel: Artikelstamm ) eingetragen werden. 

Komponententyp: Hier muss der vollständige Pfad der Komponente angegeben werden (Beispiel: NV.ERP.Base.Article.cdArticle). 

Achtung: Es muss die genaue Schreibweise (Groß- und Kleinschreibung) berücksichtigt werden. Diese kann entsprechend aus Framework Studio kopiert werden. 

Beschreibung: Hier kann eine beliebige Beschreibung angegeben werden. Beispiel: Artikelstamm. 

Ist eine Komponente registriert, kann auf diese Komponente ein Ereignis erfasst werden. 

Beispiel: 
Bei jedem Erfassen eines Artikels soll eine Wiedervorlage an den Webshop-Beauftragten geschickt werden, damit Artikelbilder hinzugefügt werden. Das Ereignis ist also das Erfassen des Artikels. Die Komponente "cdArtikel" wurde bereits registriert, somit kann auf diese zugegriffen werden. 

Achtung: Es sind nur Komponenten zulässig, welche einen Datenbank-Bezug haben. 

Achtung: Nach dem Hinzufügen neuer Komponenten, ist ein Neustart der Anwendung notwendig. Dies kann auch bei Job Servern oder weiteren eNVenta -Diensten notwendig sein.

### Button Suche 

Mit diesem Button starten Sie die Suche. 

Sie können ganz allgemein suchen, indem Sie den Button drücken, gleich nach Öffnen des Dialogs. Oder Sie geben im Kopf des Dialogs die Selektionskriterien vor und drücken dann diesen Button. 

### Button Neu 

Dieser Button leert den Dialog für eine neue Eingabe. 

Ist in den Parametern für die Sachbearbeiter die Checkbox 2 Schritt Neu aktiviert, bleiben die Suchkriterien in den Stammdialogen erhalten. 

Siehe Kapitel Parameter Sachbearbeiter . 

### Button Speichern 

Mit diesem Button speichern Sie Ihre Eingaben / Änderungen. 

### Button Neuer Satz 

Mit diesem Button fügen Sie weitere Daten in den Dialog ein. 

### Button Löschen 

Dieser Button löscht Ihre Eingaben. Genauer gesagt, die markierte Zeile wird mit dem versehen und erst endgültig gelöscht, wenn Sie den Button drücken. 

### Button Schließen 

Dieser Button schließt den Dialog. 

Vergessen Sie nicht Ihre Eingaben zuvor zu speichern!
