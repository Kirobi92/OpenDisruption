# Kapitel NV.ERP.Base.Common.cdSystemMM.shtPermanentOrderProposal

Die Einstellung für den permanenten Bestellvorschlag legen Sie hier fest, indem Sie diesen Parameter aktivieren. Beim permanenten Bestellvorschlag wird bei jedem relevanten Speichervorgang (Auftragsabschluss, Änderungen im Artikelstamm etc.) der Bestellvorschlag für den betroffenen Artikel neu berechnet. Dies kann zu geringen Laufzeit-Einbußen führen, schließt aber die zeitliche Differenz zwischen dem Lagerabgang und der Berechnung zum Auffüllen des Lagerbestands aus. 

Siehe Kapitel Bestellvorschlag Absatz Permanenter Bestellvorschlag . 

Bitte vor Aktivierung der Checkbox unbedingt das Kapitel Permanenter Bestellvorschlag lesen, da sich hieraus organisatorische Änderungen im Unternehmen ergeben können. 

Automatischer Bestellvorschlag 

Ist die Checkbox gesetzt und damit der automatische Bestellvorschlag aktiviert, wird die definierte Lauf-Nummer „Vorschlag“ auch als Lauf-Nummer im Dialog Bestellvorschlag für eine manuelle Berechnung vorbelegt. 

Dadurch wird das Problem reduziert, dass weitere Bestellvorschläge mit anderen Lauf-Nummern ungewollt berechnet werden, mit dem Ergebnis mehrfacher Bestellvorschläge! 

Zusätzlich werden in der Bestellung, bei Klick auf den Button B-Vorschl. die Lieferantennummer und die Lauf-Nummer an den Bestellvorschlag übergeben, sofern eine vorhanden ist. Diese kann dann auch direkt ausgewählt werden. 

Tooltip: 

Hier legen Sie die Einstellung für den permanenten Bestellvorschlag fest.
