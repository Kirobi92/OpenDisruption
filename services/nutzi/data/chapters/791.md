# Kunde Registerkarte Kundenverweise 
Kunde Registerkarte Kundenverweise 

eNVenta bietet die Möglichkeit Rahmenaufträge auch für einen Konzern zu erstellen. Einzelne Niederlassungen dürfen aus diesem Rahmenauftrag abrufen, oder aber der Konzern, um die Mengen einzelnen Niederlassungen zuzuordnen. 

Hier ist zu beachten, dass mehrere Kunden (bspw. mehrere Niederlassungen eines Konzerns) auf einen Rahmenauftrag zugreifen können. Da hier aber nicht immer alle Kunden der Kundengruppe auf den Rahmenauftrag zugreifen dürfen, muss der Rahmenauftrag auf eine Kundennummer geschrieben werden, unter der der Vertrag abgeschlossen wird. 

Dafür steht diese Registerkarte zur Verfügung. Hier können Sie im Feld Rahmen-Kunde eine „Rahmenauftrag-Kundennummer" erfassen. Diese bewirkt, dass beim Erfassen einer Auftragsposition nicht über die Kundennummer, sondern über die Rahmenkundennummer nach offenen Rahmenaufträgen gesucht wird. 

Dabei sind folgende Einschränkungen zu beachten: 

- Ist eine Rahmenkundennummer beim Kunden hinterlegt, wird nur über diese Kundennummer in den Rahmenaufträgen gesucht. - Aus den Rahmenaufträgen können Aufträge nur auf die Vertragskundenummer automatisch angelegt werden. 
Achtung: Auch die bestehenden Verweise auf andere Kunden werden hier angezeigt. Bei vorhandenem Customizing ist das Layout der bestehenden Registerkarten zu prüfen und ggf. nachzuarbeiten. 

Im Rahmenauftrag werden die abweichenden Konditionen hinterlegt. Diese abweichenden Konditionen werden in den Auftrag übernommen. 

Wird nun z.B. ein Auftrag auf Kunde 30091 (Rahmen-Kunde = 30090) erfasst, wird der Rahmenauftrag zu Kunde 30090 angezeigt. 
### Feld Verband 
Gehört der Kunde zu einem Verband und soll die Zahlung der offenen Posten über diesen Verband erfolgen, bzw. die Verbandsanschrift auf der Rechnung gedruckt werden (wenn dies im Report entsprechend eingestellt ist), muss hier die Kundennummer des Verbands eingetragen werden. 

Im Kundenstamm wird im Feld Verband immer der zum Tagesdatum passende Verband gespeichert. Der Button neben dem Feld Verband öffnet den Dialog Verbandshistorie . Damit kann eine Historie auf Verbände zur Verfügung gestellt werden.

### Feld Rabattkunde 
Hier kann die Kundennummer des Kunden hinterlegt werden, dessen Rabatte für den aktuellen Kunden in den Vorgängen gezogen werden sollen. 

Dies ist sinnvoll, wenn mehrere Kunden (z.B. mehrere Filialen) gleiche Konditionen haben, da sich durch die einmalige Anlage der Rabatte die Laufzeiten der Preisfindung reduzieren.

### Feld Ref.Nr A.-Suche 
Dieses Feld spielt eine Rolle in der Zusammenführung von Unternehmen in einer Gruppe. 

Wird ein Unternehmen in eine Gruppe integriert, so kommt es häufig vor, dass die Materialstämme zusammengeführt werden sollen. Hierbei kann es vorkommen, dass zum Teil noch mit alten Materialnummern bestellt wird und zum Teil schon mit Materialnummern des Mutter-Unternehmens. 

Aus diesem Grund ist es möglich, im Kundenstamm des Kunden A eine Referenz auf einen anderen Kunden B zu hinterlegen. So wird z.B. bei der Eingabe einer Materialnummer in der Auftragsposition, zunächst die Liste der Kundenartikelnummern des gewählten Kunden A durchsucht. Sollte hier keine Übereinstimmung gefunden werden, so wird zusätzlich in den Kundenartikelnummern des Kunden B nach der eingegebenen Kundenartikelnummer gesucht. 

Tooltip: 

Referenz-Kundennummer für die Artikelsuche über die Kundenartikelnummern.

### Feld Zentralzahler 
Das Feld Zentralzahler ist wie das Feld Verband (Registerkarte Warenwirtschaft ) für die Verbandsabrechnung zuständig. Es bietet eine zusätzliche Auswahlmöglichkeit in der Verbandsabrechnung für Debitoren, ohne dass die Funktionalität des Felds Verband zur Verfügung steht. (Änderung der Rechnungsanschrift) 

### Feld ZZ-Mitglied 
Hier wird die Mitgliedsnummer beim Zentralzahler eingetragen, wenn Sie mit dieser arbeiten. Diese Nummer wird in die Verbandsabrechnung übernommen. 

Tooltip: 

Gibt die Mitgliedsnummer beim Zentralzahler an. 

### Field Panel #fpnAccountancyAssociationChargeOff# 

Verbands-Ausbuchungskonten

Wenn die Checkbox Vorzeichen aktiviert wird, wird der importierte Wert zu diesem Konto beim Datenimport umgekehrt.
