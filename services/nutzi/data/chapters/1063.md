# Anzahlung 
Anzahlung 

Der Dialog Anzahlung öffnet sich im Auftrag >> Optionen >> Vorauszahlung . Bei einem Anzahlungsauftrag leistet der Kunde eine Teilzahlung auf den Rechnungsbetrag bevor die Lieferung erfolgt. Dabei wird automatisch eine Gegenposition generiert und eine Verrechnung ermöglicht. 

Bei der Erfassung der Artikelpositionen werden eine Position mit positivem Betrag und eine Position mit negativem Betrag erstellt. Die Position mit positivem Betrag wird direkt im Status 3 erfasst, damit die Rechnung erstellt werden kann und damit der Kunde einen Beleg für die geleistete Anzahlung erhält. 

Die Lieferung dieser Artikelpositionen erfolgt allerdings erst, nachdem die Anzahlung geleistet wurde. 

Damit die Anzahlung ordnungsgemäß verwaltet werden kann, muss hierfür ein entsprechender Artikel in den Auftrag eingefügt werden. Die Angabe der Artikelnummern erfolgt im Dialog Parameter Verkauf . 

Wenn Sie die Anzahlung speichern, resultieren folgende Schritte: 

- Das evtl. vorhandene Kennzeichen Sammelrechnung wird entfernt. - Es werden 2 Positionen generiert: eine mit positivem Betrag und eine mit einem negativen Betrag. Dabei wird die entsprechende Artikelnummer für Anzahlungen verwendet. - Die positive Position erhält immer automatisch den Status 3. 
Bei einer kompletten Vorauszahlung wird ein Anzahlungsartikel verwendet, welcher den gleichen Preis hat wie der gesamte Auftrag. 

Dieser „Verrechnungsartikel“ muss zuvor im Artikelstamm angelegt sein. Dieser Verrechnungsartikel ist ein Diversartikel und ohne Lagerhaltung. Er wird nur für die Verwaltung der Teilzahlungen verwendet. 

Im Auftrag wird der Verrechnungsartikel eingetragen und entsprechend der Betrag über die Teilzahlung. In der Bezeichnung der Auftragsposition können die entsprechenden Anmerkungen hinterlegt werden. Diese Auftragsposition muss dann anschließend fakturiert werden, um die Rechnung zu erzeugen und den Betrag an die Buchhaltung zu übergeben. 

Die Lieferung dieser Positionen erfolgt allerdings erst, nachdem die Abschlagszahlung geleistet wurde. 

Durch die automatische Anpassung der vorhandenen Auftragspositionen können auch weitere Positionen in den Auftrag hinzugefügt oder auch gelöscht werden. 

Mit dem Auftragsabschluss werden automatisch die entsprechenden Anzahlungspositionen generiert (eine Position positiv über den gesamten Anzahlungsbetrag im Status 3, die andere Position im Status 1 mit negativem Betrag). 

Bei der anteiligen Verrechnung wird bei jeder Rechnungslegung automatisch der entsprechende Teilbetrag mit fakturiert, d.h. wird eine Teilrechnung von 2.000 EUR bei einem Auftragswert von 10.000 EUR und einer Anzahlungssumme von 2.000 EUR erstellt, so wird die Rechnung um die Anzahlung reduziert. 
#### Fakturierung 
Enthält ein Auftrag eine oder mehrere Anzahlungspositionen und es werden "normale" Positionen fakturiert, gilt für die Abschlussrechnung: „alle Nicht-Anzahlungspositionen werden fakturiert“. In diesem Fall müssen auch alle Gegenpositionen zur Faktura freigegeben und mit fakturiert werden. 

#### Anzahlungen im Zusammenhang mit Erweitertem Kreditlimit 
Wenn eine Anzahlung fakturiert und bezahlt ist, zieht das 'späte Kreditlimit' nicht mehr! 

Lesen Sie dazu auch das Kapitel Anzahlung und Abschlagszahlung .

### Checkbox Anteilige Verteilung 
Wenn Sie diese Checkbox aktivieren, ist zwingend ein Anteil der Gegenpositionen zur Faktura freizugeben. Dazu muss eine weitere Gegenposition mit entsprechendem Betrag erzeugt und vorhandene Gegenpositionen entsprechend im Wert reduziert werden (eine oder mehrere). 

Bei der anteiligen Verrechnung wird bei jeder Rechnungslegung automatisch der entsprechende Teilbetrag mit fakturiert, d.h. wird eine Teilrechnung von 2.000 EUR bei einem Auftragswert von 10.000 EUR und einer Anzahlungssumme von 2.000 EUR erstellt, so wird die Rechnung um die anteilige Anzahlung reduziert. 

Beispiel: 
Gesamtbetrag des Belegs vor der Teilfaktura 200 
Anzahlung dazu 60 
Teilfaktura über Betrag von 90 
=> Es entstehen zwei Gegenpositionen zum Anzahlungsbetrag: 
1. Über 60 * 90 / 200 = 27 (anteilsmäßiger Anzahlungsbetrag bezogen auf teilfakturierten Betrag) 
2. Über 60 - 27 = 33 

Tooltip: 

Anteilige Verteilung der Anzahlung bei Teilfaktura. 

Der ursprüngliche Anzahlungsbetrag, der sich auf den Gesamtbetrag des Belegs bezieht, 
wird auf den teilfakturierten Betrag umgerechnet. 
Dabei entstehen dann aus einer Anzahlungsgegenposition zwei. 

Bsp.: 
Gesamtbetrag des Belegs vor der Teilfaktura 200 
Anzahlung dazu 60 
Teilfaktura über Betrag von 90 
=> Es entstehen zwei Gegenpositionen zum Anzahlungsbetrag: 
1. Über 60 * 90 / 200 = 27 (anteilsmäßiger Anzahlungsbetrag bezogen auf teilfakturierten Betrag) 
2. Über 60 - 27 = 33

### Feld Anzahlung Max. 
Die maximale Anzahlung ist ein vom System errechneter Wert, welcher angibt, wie hoch die maximale Anzahlung sein kann. 

Dies ist immer der Bruttowert des Auftrags minus aller bereits erhaltenen Anzahlungen minus aller Teilrechnungen. 

Tooltip: 

Angabe des Betrags 

### Feld Anzahlung % 
Dieses Feld steht zur Verfügung, damit eine Anzahlung für einen Auftrag nicht nur als Betrag, sondern auch prozentual berechnet und ausgewiesen werden kann. 

Wenn hier ein Prozentwert eingetragen wird, erfolgt die Berechnung automatisch und der Betrag im Feld Anzahlung wird angepasst.
