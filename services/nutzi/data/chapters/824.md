# Versandanschriften verwalten 
Versandadresse 

Dieser Dialog öffnet sich im Kunden- und Adressstamm über den Button Versand . Beim Öffnen des Dialogs werden die Felder AKL-Typ, -nummer und AnschriftNr . vorbelegt und können nicht geändert werden. Die restlichen Felder im Kopf des Dialogs dienen der Selektion. 

In diesem Dialog können Sie viele verschiedene Versandanschriften (auch Lieferanschriften und Bestellanschriften) anlegen und verwalten. 

Um eine neue Versandanschrift anzulegen, drücken Sie den Button und geben die Daten in die Kopffelder ein. 
#### Lieferanschriften im Zusammenhang mit Sammelpackliste / -lieferschein 
Existieren für einen Kunden mehrere Lieferanschriften und der Parameter Sammelpackliste (im Dialog Parameter Verkauf ) ist für diesen Kunden enbenfalls gesetzt, so werden Lieferanschriften als 'identisch' angesehen, wenn diese bezügl. der Felder Firma1 und Ort übereinstimmen. D.h. gibt es mehrere Aufträge zu einem Kunden an verschiedene Lieferanschriften innerhalb einer Stadt, so landen diese auch auf einem gemeinsamen Sammellieferschein. 

Beim Druck einer Packliste aus Auftrag 1 wird nun eine Sammelpackliste über die anderen zuvor angelegten Aufträge erzeugt, trotz unterschiedlicher Lieferadresse. Geht man einen Schritt weiter und erzeugt einen Lieferschein, wird ebenfalls ein Sammellieferschein generiert, auf welchem beide Aufträge aufgeführt sind wieder trotz unterschiedlicher Lieferadresse. 

Siehe Kapitel Parameter Verkauf .

### Checkbox Adresse gültig 
Diese Checkbox kann nur manuell gesetzt werden und dient dazu, die Adresse als manuell geprüft zu markieren. 

Sie übersteuert die Logik der Adressprüfung für Adressen, z.B. welche nicht in den Stammdaten enthalten sind (Stammdaten veraltet, Neubaugebiete, Angabe von Baustellen etc.). 

Wird die Checkbox gesetzt, erfolgt keine weitere Adressprüfung im weiteren Prozess, z.B. bei Auftragsabschluss und Lieferscheindruck. 

Wird eine Adresse nachträglich geändert, so wird die Checkbox Adresse gültig automatisch wieder deaktiviert, damit eine erneute Validierung der Adresse erfolgen kann. 

Ist die Checkbox nicht gesetzt, so erfolgt immer wieder eine Adressprüfung bei Auftragsabschluss, Bestellabschluss, Speichern der Retoure, sowie bei der Anlage von Stammdaten (Kunde, Lieferant, Lagerort, Projektanschrift, Adresse, Vertreter, Versandanschrift, Bestellanschrift, Wettbewerber). 

Hinweis: 
Die länderspezifischen Provider (cAddressCheckProviderCountry) delegieren die Prüfung je nach Länderkennzeichen an unterschiedliche Methoden. 

Je nach Land wird das Format der PLZ geprüft. So ist es zwingend erforderlich, dass die deutsche PLZ 5-stellig numerisch ist. Es erfolgt hier keine Prüfung gegen Stammdaten, lediglich der Aufbau ist entscheidend. 

Siehe Kapitel Adressprüfung in eNVenta . 

Tooltip: 

Wenn aktiviert, ist die Adresse gültig.
