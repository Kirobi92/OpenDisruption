# Artikelkalkulation Registerkarte Allgemein 
Artikelkalkulation Registerkarte Allgemein 

Auf dieser Registerkarte geben Sie detailliert die Daten für die Kalkulation ein. D.h. Sie geben den Einkaufspreis ein, die Zuschläge (wie. z.B. Verwaltungskosten-Zuschlag), die hinzugerechnet werden sollen und die Marge. Dazu wählen Sie dann noch eine entsprechende Rechenart aus (z.B. ein + für Addition, ein - für Subtraktion). 
### Button Neu 

Dieser Button fügt eine neue Zeile in die Tabelle ein. 

Tooltip: 

Neue Zeile 

### Button Löschen 

Mit diesem Button können Sie eine Zeile aus der Tabelle entfernen. 

Tooltip: 

Zeile löschen 

### Button Faktor 

Dieser Button ruft den Dialog Kalkulationsfaktor auf, in dem Sie Faktoren definieren, korrigieren, löschen oder übernehmen können (in Kalkulationen oder Kalkulationsschemas). Dieser Faktor bietet eine weitere Möglichkeit, einen Artikel zu kalkulieren. 

Ein Kalkulationsfaktor ergibt sich, indem der Listenverkaufspreis durch den Einstandspreis dividiert wird. 

Wenn Sie in eine bereits bestehende Kalkulation einen neuen Faktor aufnehmen möchten, müssen Sie zuvor in der Kalkulation diejenige Zeile markieren, ab der Sie den neuen Faktor einfügen möchten. 

Siehe Kapitel Kalkulationsfaktoren . 

Tooltip: 

Faktor hinzufügen 

### Button Schema 

Dieser Button öffnet den Dialog Kalkulationsschema . Das Kalkulationsschema zeigt auf, aus welchen Kostenbestandteilen (z.B. Herstellkosten, Selbstkosten, Gemeinkosten) sich die Kosten des Produkts zusammensetzen. 

Dort werden Standard-Kalkulationsschemas definiert, korrigiert, gelöscht oder in eine Kalkulation übernommen. 

Siehe Kapitel Verwaltung des Standard-Kalkulationsschemas . 

Tooltip: 

Schema hinzufügen 

### Button InputNeu 

Dieser Button holt die aktuellen Werte / Preise anhand der Input-Variablen aus dem Artikelstamm und den Lieferantenkonditionen in die aktuelle Kalkulation. Diese Funktion ist in Kalkulationen für einzelne Artikel möglich (nicht bei Artikel- und Artikel Kalkulationsgruppen). 

Diese Werte werden dann automatisch im Feld Betrag LW vor dem Buchen sichtbar, aber nicht gespeichert, damit Sie die Möglichkeit haben, die Kalkulation vor dem Buchen noch mal zu prüfen. 

Die Eigenschaft dieses Buttons kann an eine Jobdefinition für die Ausführung der Artikelkalkulation per Job Server übergeben werden. Diese Eigenschaft muss direkt im Dialog Job-Verwaltung gesetzt werden. 

Dabei wird berücksichtigt, dass das Verhalten bei Einzel- bzw. Gruppenartikelkalkulation verschieden ist: 

- Die Einzelartikelkalkulation unterscheidet Input Neu , Kalkulation und Buchen . - Die Gruppenartikelkalkulation führt immer alle Schritte ( Input Neu, Kalkulation und Buchen ) zusammen aus. - Buchen der Einzelartikelkalkulation funktioniert nur, wenn der Status kalkuliert ist. - Beim Buchen per Job wird der Job als ganzes betrachtet. - Ist der Job-Parameter Input Neu gesetzt, werden ALLE Kalkulationen durchgebucht ( Input Neu, Kalkulation und Buchen ). - Ist der Job-Parameter Input Neu nicht gesetzt, werden ALLE Kalkulationen nur gebucht, wenn sie noch nicht im Status gebucht sind. 

Tooltip: 

Update Kalkulationspositionen bei einer Kalkulation für einen einzelnen Artikel 

### Button Kalkulieren 

Für einzelne Artikel gilt: Alle Werte der gewählten Kalkulation werden am Bildschirm berechnet und in der Datenbank gespeichert. Der Status wird auf kalkuliert gesetzt. 

Lesen Sie in diesem Zusammenhang auch die Informationen zum nachfolgenden Button Buchen ! 

Tooltip: 

Positionen kalkulieren 

### Button Buchen 

Dieser Button aktualisiert alle Datensätze mit den kalkulierten Werten, die sich aus den ermittelten Werten ergeben. Diese werden in der Datenbank gespeichert, der Status der Kalkulation wird auf verbucht gesetzt. 

Es werden nur Daten im Artikelstamm, den Konditionen etc. geändert, wenn diese Zeilen die Checkbox in der Spalte 'Buchen' gesetzt haben. 

Vor dem Verbuchen muss immer eine Kalkulation stattfinden (über den Button Kalkulation ). Wir empfehlen, diese mit dem Button InputNeu zu bestätigen, damit die aktuellen Werte in der Kalkulation angezeigt werden und evtl. noch korrigiert werden können. 

Ein einzelner Artikel kann nur manuell gebucht werden, wenn der Artikel den Status kalkuliert hat. 

Bei der Kalkulation von Artikel- oder Artikel Kalkulationsgruppen ist dieser Button nicht aktiv, da die Kalkulation automatisch mit dem Verbuchen erfolgt (d.h. mit dem Button Kalkulieren ). 

Der Button Buchen wird nur dann aktiv, wenn mindestens für eine Artikelkalkulationsposition die Checkbox in der Spalte 'Buchen' gesetzt ist! 

Tooltip: 

Kalkulation verbuchen 

Die Funktion 'Buchen' ist nur aktiv, wenn mindestens eine Artikelkalkulationsposition den Haken 'Buchen' gesetzt hat. 

### Button Alle buchen

### Tabellenspalte Rechenart 
Hier geben Sie die Rechenart ein. Zur Verfügung stehen +, -, /, *, =. 

Die Auswahl der Rechenart bestimmt, wie die angegebenen Prozente oder Faktoren berücksichtigt werden. 

Bei den Rechenarten Division und Multiplikation werden die Angaben in der Spalte ‚Faktor‘ berücksichtigt. Bei den Rechenarten Addition und Subtraktion werden die Angaben in den Spalten ‚Betrag‘ und ‚Prozent‘ berücksichtigt. 

### Tabellenspalte Input-Variable 
Hier geben Sie die Bezeichnung der Variablen an. 

Bei den Variablen handelt es sich um Datenfelder aus dem Artikelstamm und den Lieferantenkonditionen. Sie dienen dem Programm entweder als Input-Variable (bei Erstellung einer Kalkulation oder über den Button InputNeu werden die betreffenden Werte ab Artikel und Lieferantenkondition in die Kalkulation übernommen) oder als Update-Variable (der kalkulierte Wert wird bei der Verbuchung der Kalkulation in den Artikel übernommen). Das Programm verfügt über die folgenden Input- und Update-Variablen:

### Tabellenspalte Preisgruppe 
Hier kann eine Preisgruppe eingetragen werden. Diese Spalte ist nur zugänglich, wenn die Input-Variable Mehrfach Preisgruppe ausgewählt ist. 

Tooltip: 

Hier wählen Sie eine Preisgruppe aus, die bestimmt auf welchen Verkaufspreis zugegriffen wird. 

### Tabellenspalte Text 
Hier kann ein individueller Text zur Variablen eingetragen werden. Z.B. 20.% Aufschlag für Verwaltungsgemeinkosten. 

### Tabellenspalte Betrag LW 
Hier wird der Betrag in Landeswährung über die Variable eingelesen, kann geändert werden und ist das Ergebnis, welches gebucht wird. 

Tooltip: 

Menge in Landeswährung 

### Tabellenspalte Prozent 
Hier kann ein Prozentwert eingetragen werden. 

Ein Prozentsatz kann nur bei den Rechenarten Addition bzw. Subtraktion eingetragen werden und wird auf den Betrag in Landeswährung berechnet. Er bezieht sich auf den vorhergehenden Faktorenwert. 

### Tabellenspalte Faktor 
Hier kann ein Faktor eingetragen werden. 

Ein Faktor kann nur in Verbindung mit den Rechenarten Multiplikation bzw. Division verwendet werden und bezieht sich auf den zuvor ermittelten Faktorenwert in Landeswährung. 

### Tabellenspalte Menge 
Hier geben Sie die Menge ein, wenn eine Variable „AbMengeP“ oder „AbMengeR“ ausgewählt wurde. In diesem Fall wird der Rabattsatz aus der Rabattstaffel EK übernommen. 

Diese Option ist auch für Preisgruppen, wenn in den Preisgruppen Details mehrere 'ab Mengen' (ab 0,000, ab 10,000 etc.) vorhanden sind und ein Ergebnis der Kalkulation einen neuen Eintrag erzeugt. 

Tooltip: 

Angabe der Menge 

### Tabellenspalte Runden 
Hier geben Sie an, ob der Betrag gerundet werden soll. 

### Tabellenspalte Buchen

Der Button Buchen wird nur dann aktiv, wenn mindestens für eine Artikelkalkulationsposition die Checkbox gesetzt ist!

Angabe des Positionsstatus
