# BMECat

E-Commerce Katalog 

# BMECat 
BMECat bedeudet: Bundesverband Materialwirtschaft-, Einkauf- und Logistik-Catalog. 

BMEcat ist ein XML-basiertes standardisiertes Austauschformat für Katalogdaten im B2B Bereich. 

Durch den Import sind verschiedene Module von eNVenta ERP adressiert: 

- Artikelstamm - EK-Rabatt - Preisgruppen - Merkmalsverwaltung - Katalogverwaltung - Lieferantenstamm 
Die Funktionalität ist ähnlich der des Schattenstamms. Der Schattenstamm unterstützt den Import von Artikeldaten. 

Es gibt wie im Schattenstamm eine Steuermöglichkeit, ob die Preise in eine EK-Kondition, direkt in den Artikelstamm oder in eine Preisgruppe übernommen werden. 

Der allgemeingültige Mechanismus überprüft die betriebswirtschaftlichen Auswirkungen von Änderungen an den Stammdaten, dabei wird der Benutzer auf mögliche Probleme hingewiesen. 

Beispiel: 
Die Stammdaten im Artikelstamm dürfen nicht beliebig geändert werden, da laufende Vorgänge damit ungültig werden können. (z.B. Anpassung von EK-pro, Einheiten…) 

Der Übernahme-Mechanismus ist erweitert worden, damit auch Merkmalswerte, Lieferantendaten, Katalogdaten importiert werden können. Der Mechanismus ist kein eigener Dialog, sondern eine Komponente (API), welche mit entsprechenden Import-Daten angesteuert wird. 

Die BMEcat Import Funktionalität ist standardisiert, so dass eine Verarbeitung von Lieferantenkatalogen und Kundenkatalogen möglich ist. 

Die zwei Szenarien werden unterschieden, da es bei Lieferantenkatalogen um Stammdatenpflege und den Einkauf geht, bei den Kundenkatalogen dagegen um den Reimport von Katalogen, welche durch externe Werkzeuge auf Basis der aktuellen ERP-Daten erstellt wurden. 

Durch einen Import-Vorgang werden automatisiert massenhaft Stammdaten verändert. Allein durch diese Tatsache ist ein Import immer mit Vorsicht zu genießen. 

Die Änderungen können nicht einfach rückgängig gemacht werden. Deshalb gibt es einen Kompromiss zwischen Datensicherheit und Automatisierung. Der BMECat geht in der Definition davon aus, dass durch einen Import bestehende Daten gelöscht / überschrieben werden können. Allerdings können in einem ERP-System die Artikel nicht einfach gelöscht werden. Es braucht genaue Regeln, welche Operationen in den Zuständen Neu / Aktualisierung / Löschen zulässig sind. Ein Problem ist auch die Tatsache, dass Artikel, welche in einem früheren Katalog enthalten waren, nicht gezielt abgekündigt werden, sondern einfach nicht mehr in dem neuen Katalog enthalten sind. Gleiches trifft natürlich auch für Merkmale, Bilder, Preise etc. zu. 

Um die Import-Funktionalität in eNVenta ERP zu vereinheitlichen, erfolgt der Import über den Artikel-Schattenstamm. Der Schattenstamm folgt der Logik, dass die Daten im Schattenstamm ohne weitere Bearbeitung übernommen werden können. Wenn von der inhaltlichen Korrektheit der Daten ausgegangen wird, werden die Daten mit einer festen Programmierlogik in eNVenta übernommen. Der Benutzer hat keine Möglichkeit, die Veränderungen an den Stammdaten einzeln zu überprüfen und freizugeben bzw. zu verhindern. 

### Szenario: Lieferantenkatalog 
Der Lieferantenkatalog wird durch einen Hersteller / Einkaufsverband / Großhändler erstellt. Da der Katalog die Preise auf der Einkaufsseite enthält, ist eine automatische Ermittlung der VK-Preise nur über eine automatische Kalkulation möglich. 

Der Katalog wird als Sortiment des Lieferanten betrachtet. Es wird davon ausgegangen, dass die Artikel, welche in dem Katalog enthalten sind, von dem Lieferanten bezogen werden können. Siehe auch: 

- BMECat - Import - BMEcat - Einheiten - BMEcat - Export - BMEcat - Katalogexport - BMEcat - Preiskonfiguration
