# Aufgabe ändern Registerkarte Allgemein 
Aufgabe ändern Registerkare Allgemein 

Auf dieser Registerkarte definieren Sie allgemeine Einstellungen zur Aufgabe. 

Es ist möglich einen Verantwortlichen festzulegen sowie ein Datum / Zeitraum für die Erledigung der Aufgabe. Diese Änderungen werden automatisch in die Ressource übernommen. 

Entsprechend werden die Projektaufgaben in der Registerkarte Phasen inkl. Positionsnummer angezeigt. 

Beim Verschieben einer Aufgabe in eine andere Phase wird diese Positionsnummer nur noch zur Information mitgeführt. 

Wurde eine Aufgabe auf Wiedervorlage gelegt und von dort aufgerufen, so öffnet sich nicht nur der Dialog Projekt , sondern auch das Menü zum Ändern einer Aufgabe. 

Es kann sinnvoll sein, nach Abschluss einer Aufgabe eine neue Folgeaufgabe zu erzeugen. Dazu geben Sie im Bereich Automatische Folgeaufgabe bei Abschluss eine Aufgabennummer ein. 
### Feld Datum/Zeit von 
In diesem Feld geben Sie ein Datum ein, ab wann die Aufgabe bearbeitet werden soll.

### Feld Datum/Zeit bis

Tooltip:

### Feld Sollzeit (hh,mm) 
Hier geben Sie eine geplante Sollzeit für die Bearbeitung der Aufgabe ein. 

### Feld Ist-Zeit (hh,mm) 
Hier kann die tatsächlich benötigte Zeit manuell eingetragen werden oder sie wird aufgrund der Leistungserfassung automatisch addiert und hier eingetragen. Bei jeder Leistungserfassung auf die Projektaufgabe wird die Ist-Zeit entsprechend erhöht. Die Erhöhung der Ist-Zeit erfolgt nur bei Leistungsarten mit dem Leistungstyp "Projektmanagement" um z.B. Fahrzeiten / Fortbildung anderer Kollegen nicht zu berücksichtigen.

### Combobox Bereich 
Wenn Sie hier einen Eintrag auswählen, wird die Struktur sofort entsprechend dem ausgewählten Bereich eingelesen. Dadurch lassen sich Aufgaben filtern. Dies dient der übersichtlicheren Darstellung bei größeren Projekten. Die Combobox wird aus dem Aufgabenstamm vorbelegt. 

### Combobox Klassifizierung 
In dieser Combobox werden die bereits eingegebenen Klassifizierungen dieses Projekts zur Auswahl angeboten. Es kann aber auch immer ein beliebiger Text als (neue) Klassifizierung eingegeben werden. Die Klassifizierung wird auch entsprechend in der Aufgabenübersicht angezeigt und als Selektionskriterium zur Verfügung gestellt.

### Tabelle Checkliste 

In dieser Tabelle sehen Sie die Einträge, die dieser Aufgabe zugeordnet wurden. Wird eine Aufgabe aus dem Stamm in ein Projekt übernommen, so wird auch die Checkliste übernommen. Im Aufgabenstamm vordefinierte Checklisten erleichtern die lückenlose Abarbeitung der Aufgabe in den Projekten. 

Über das Menü System >> Allgemein >> Sachbearbeiter Parameter kann die Checkliste individuell ein- oder ausgeblendet werden.

### Button Suche 

An den Checkboxen in der Tabelle Checkliste hängen oft auch ganze Prozesse. Diese Checkliste hängt an der jeweiligen Vorlage im Aufgabenstamm und wird in die aktuelle Projektaufgabe bei der ersten Erfassung übernommen. 

Mit diesem Button können Sie die entsprechende Aufgabe aus dem Aufgabenstamm (es öffnet sich der Dialog Aufgabe auswählen ) auswählen und dessen Checkliste in die aktuelle Aufgabe übernehmen. So kann komfortabel nach der richtigen Checkliste gesucht werden. Fehler bei der Übernahme werden vermieden. 

Mit Übernahme der Checkliste werden die bestehenden Einträge ergänzt, nicht aber gelöscht, da evtl. bereits Daten eingetragen waren. Hierbei erfolgt allerdings weiterhin eine Prüfung, ob die Punkte der Checkliste doppelt vorhanden sind. Dies muss vor der Speicherung entsprechend manuell korrigiert bzw. angepasst werden. Eine Sortierung anhand der Positionsnummer hilft, Dubletten schnell zu erkennen und zu entfernen. 

Tooltip:
