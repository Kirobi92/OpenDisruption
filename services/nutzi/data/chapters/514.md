# Niederlassungen 
Niederlassung 

Das Ziel der Niederlassungsverwaltung ist es, Unternehmen, die mit mehreren Niederlassungen arbeiten, durch ein weiteres Kriterium (der Abgrenzung der Niederlassungen innerhalb eines Mandanten) besser zu unterstützen. Damit besteht u.a. die Möglichkeit, Daten zu segmentieren, den Zugriff einzelner Benutzer auf Vorgänge seiner Niederlassung zu beschränken, Statistiken niederlassungsbezogen durchzuführen und ähnliches mehr. 

Um eine neue Niederlassung anzulegen, drücken Sie den Button und füllen die Felder des Dialogs. 

Sämtliche Anzeige- und Auswahldialoge werden entsprechend der benutzerspezifischen Niederlassungs-Beschränkungslogik eingegrenzt. 

Jede Niederlassung erhält eine Nummer, eine Bezeichnung sowie eine Adresse. Die Niederlassungs-Nummer wird automatisch vom System vergeben und wird selbstverständlich in allen niederlassungsrelevanten Vorgängen zusätzlich mitgeführt. 

Mit der Reportprogrammierung können die Niederlassungsdaten auf jedem entsprechenden Formular mit angedruckt werden. 

Die Niederlassungszuordnung ist optional und dient ausschließlich der Datensegmentierung, d.h. ein Benutzer der Niederlassung 1 kann nur Aufträge der Niederlassung 1 sehen. 

Ist ein Mitarbeiter keiner Niederlassung zugeordnet, so kann er uneingeschränkt alle Kundendaten einsehen und bearbeiten. 

Aufgrund der Niederlassungsnummer kann selektiert werden, Statistiken können abgegrenzt werden, Daten segmentiert werden etc. 

Es gelten folgende Regeln: 
- Jeder Kunde im Kundenstamm ist zwingend genau einer Niederlassung zuzuordnen (Pflichtfeld!). - Jedes Lager wird genau einer Niederlassung zugewiesen (Pflichtfeld). - Lagergruppen sind ebenfalls einer Niederlassung zuzuordnen. Lagergruppen können nur Lager mit gleicher Niederlassungsnummer enthalten. - Die Niederlassung eines Kunden ist nicht änderbar. - Die Niederlassung von Lagern sowie Lagergruppen ist nicht änderbar. - Die Artikel sind für alle Niederlassungen gleich. - Ist einem Sachkonto, das kein Sammelkonto ist, eine Niederlassung zugeordnet, wird diese bei jeder Buchung verwendet, egal ob es ein Steuerkonto ist oder nicht. 
Für das Feld Sachbearbeiter einiger Dialoge gilt: Ist dieses Feld editierbar und kann ein Vertreter, anstatt des aktuellen Benutzers eingegeben werden, so gilt dennoch die auf den Benutzer bezogene Zugriffslogik. 

Bestellungen erhalten eine Niederlassungsnummer auf Positionsebene. Wenn der User einer Niederlassung zugeordnet ist, wird die Niederlassungsnummer auch im Bestellkopf eingetragen.

### Feld E-Mail 
Hier wird die E-Mail-Adresse angegeben. Bei der Angabe mehrerer Adressen, müssen diese durch Semikolon getrennt werden. 

Beim Import von E-Mails, beim Vergleich / der Prüfung der E-Mail-Adressen, werden Leerzeichen am Ende und Beginn ignoriert. So können Absender leichter ermittelt werden und die E-Mail wird problemlos importiert. Ebenso werden bereits beim Speichern neuer oder geänderter E-Mail-Adressen automatisch die unnötigen Leerzeichen am Beginn und Ende der Mail-Adresse entfernt. 

Sammelverteiler wie z.B. Mail-Adresse Mahnung im Kundenstamm sind davon ausgenommen. 

Um sicherzugehen, dass eine E-Mail an die richtige Adresse gesendet wird, prüft eNVenta beim Speichern oder Ändern einer Mailadresse, ob die Mail-Adresse der „Norm“ entspricht. Falls nicht wird ein Hinweis ausgegeben. Dabei wird das Speichern nicht grundsätzlich verhindert. Auf diese Weise werden z.B. Outlook-Adressbucheinträge wie z.B. „#EW_ERP“ gespeichert und genutzt. Die Auswertung des Eintrags erfolgt wie gewohnt im Outlook Client. 

Dabei läuft die Prüfung ausschließlich bei Anlage und Änderung einer Mail-Adresse. Werden andere Einträge, z.B. des Kundenstammsatzes geändert, so kommt diese Meldung nicht. 

Werden Stammdaten über das Gateway importiert, so werden u.U. fehlerhafte Mailadressen importiert (ggfs. mit Warnung). Bei einem Import unter Verwendung des Jobservers erfolgt dies ebenfalls in Form einer Warnung (im Jobprotokoll). 

Tooltip: 

Angabe der E-Mail-Adresse. 
Bei der Angabe mehrerer Adressen, müssen diese durch Semikolon getrennt sein.

Hier geben Sie die SEPA-Gläubiger ID ein. Jedes Unternehmen muss diese ID bei der Deutschen Bundesbank beantragen, um SEPA-Einzug durchführen zu können. 

Dies ist die eindeutige Kennnummer des SEPA-Gläubigers.

https://extranet.bundesbank.de/scp/lizenz.do 

Tooltip: 

SEPA-Gläubiger ID 

SEPA-weit ist die Gläubiger-Identifikationsnummer maximal 35 Stellen lang und hat folgenden Aufbau: 
Stellen 1–2: jeweiliger ISO-Ländercode (2 Stellen, z. B. DE, AT usw.), 
Stellen 3–4: Prüfzahl (2 Stellen analog zur IBAN-Prüfzahl gemäß ISO 13616:2007), 
Stellen 5–7: alphanumerische, dreistellige Geschäftsbereichskennung (Creditor Business Code), die von dem Gläubiger selbst z. B. zur Kennzeichnung einzelner Geschäftsbereiche, Filialen etc. vergeben/beantragt werden kann, und die nicht in die Prüfziffernberechnung eingeht, 
Stellen 8–35 (max.): nationales Identifikationsmerkmal. 

Die Länge des nationalen Identifikationsmerkmals kann variieren und beträgt maximal 28 Stellen. Ab der Stelle 5 dürfen folgende Zeichen vorkommen: 
Buchstaben a-z A-Z 
Ziffern 0-9 
weitere Zeichen: + ? / - : ( ) . , '
