# .opencode/commands

Slash-Commands für die OpenCode-Workbench — jede Markdown-Datei definiert einen aufrufbaren `/command`.
KeyCodi delegiert Missions-Schritte über diese Commands an spezialisierte Agents.

## Wichtige Dateien

- `mission.md` — Haupt-Einstiegspunkt für neue Coding-Missionen
- `agents.md` — Agent-Registry abfragen und steuern
- `architect.md` — Architektur-Entscheidungen und Systemdesign
- `autonomous.md` — Autonomen Ausführungsmodus starten
- `backlog.md` — Task-Backlog verwalten
- `deploy.md` — Deployment-Workflows
- `kidi.md` — KiDi-Kontext und Wissensdatenbank
- `refactor.md` — Code-Refactoring
- `review.md` — Code-Review anstoßen
- `scan.md` — Repository-Scan
- `status.md` — System-Status abfragen
- `test.md` — Test-Ausführung
- `upgrade.md` — Dependency-Upgrades
